이번 차시는 pandas로 정리한 데이터를 그래프로 바꾸는 수업이다. 엑셀에서 표를 만든 뒤 차트 삽입을 누르는 것과 비슷하지만, 파이썬에서는 차트를 만들기 전에 "어떤 질문을 어떤 표로 줄이고, 어떤 차트로 보여줄 것인가"를 코드로 분명히 적는다.

수업은 한글폰트 설정에서 시작한다. 한글 데이터로 차트를 만들 때 제목과 축이 깨지면 분석 결과가 전달되지 않는다. 그래서 matplotlib의 rcParams에서 글꼴과 unicode_minus를 먼저 설정한다.

이후에는 세 가지 실데이터를 다룬다. 서울 지하철 199,424행은 원자료를 그대로 그리지 않고 승차 데이터만 골라 역별·시간대별로 집계한 뒤 히트맵을 만든다. 강남구 아파트 실거래 3,754행은 히스토그램, 박스플롯, 산점도와 추세선으로 본다. FRED 한국 CPI 월별 데이터는 2000년 이후 287개 월로 줄여 시계열 라인차트를 만든다. 마지막 미션에서는 카페 매출 데이터를 월별로 집계해 라인차트와 CSV로 저장한다.

학습 목표

  1. 한글폰트를 설정하고 axes.unicode_minus=False가 필요한 이유를 설명할 수 있다.
  2. 숫자 분포를 히스토그램과 박스플롯으로 확인할 수 있다.
  3. 두 숫자 변수의 관계를 산점도와 추세선으로 표현하고, 상관계수를 보조 정보로 해석할 수 있다.
  4. 대용량 데이터를 먼저 집계한 뒤 히트맵이나 라인차트로 시각화할 수 있다.
  5. 차트에 사용한 정제·집계 결과를 to_csv()로 저장해 BI 도구에 넘길 수 있다.

핵심 개념

1. 차트는 표를 예쁘게 꾸미는 일이 아니라 질문에 답하는 일이다

엑셀에서는 표를 선택한 뒤 차트 종류를 고르는 방식에 익숙하다. 파이썬에서도 그래프를 그릴 수 있지만, 입문 단계에서 먼저 익혀야 할 순서는 반대에 가깝다.

  1. 무엇을 알고 싶은지 정한다.
  2. 그 질문에 필요한 열만 고른다.
  3. 큰 데이터는 집계하거나 필터링한다.
  4. 질문에 맞는 차트 종류를 고른다.
  5. 제목, 축, 단위, 저장 파일을 확인한다.

예를 들어 지하철 원본 199,424행을 그대로 화면에 찍는 것은 분석이 아니다. "어느 역의 어느 시간대 승차가 많은가"라는 질문으로 바꾸고, 승차 데이터만 골라 역별·시간대별 합계로 줄여야 히트맵이 의미를 가진다.

2. 한글폰트 설정은 차트의 전처리다

한글 CSV를 잘 읽었더라도 그래프 제목과 축 글자가 깨질 수 있다. 데이터가 틀린 것이 아니라 matplotlib이 한글 글꼴을 찾지 못한 상태일 수 있다.

이번 차시에서는 아래 두 가지를 반드시 확인한다.

plt.rcParams["font.family"] = "NanumGothic"
plt.rcParams["axes.unicode_minus"] = False

font.family는 한글을 표시할 글꼴을 정한다. axes.unicode_minus=False는 음수 부호가 네모로 깨지는 문제를 줄인다. 오늘 다루는 핵심 그래프에는 음수가 많지 않지만, 시각화 환경을 처음 잡을 때 함께 설정해두는 습관을 들이면 좋다.

3. 히스토그램과 박스플롯은 분포를 보는 도구다

히스토그램은 숫자가 어느 구간에 많이 몰려 있는지 보여준다. 엑셀의 도수분포표와 막대그래프를 합친 느낌이다. 강남구 아파트 거래금액을 히스토그램으로 보면 거래금액이 낮은 구간부터 높은 구간까지 어떻게 퍼져 있는지 볼 수 있다.

박스플롯은 중앙값, 중간 50% 범위, 이상치를 한 번에 보여준다. 엑셀에서는 익숙하지 않을 수 있지만, 여러 그룹의 퍼짐을 비교할 때 유용하다. 이번 수업에서는 전용면적을 40㎡ 이하, 40-60㎡, 60-85㎡, 85-120㎡, 120㎡ 초과로 나누고, 면적구간별 거래금액 범위를 비교한다.

4. 산점도와 추세선은 두 숫자 사이의 관계를 본다

산점도는 한 행을 점 하나로 표현한다. x축은 전용면적, y축은 거래금액이다. 점들이 오른쪽 위로 올라가는 모양이면 전용면적이 클수록 거래금액도 높아지는 경향이 있다는 뜻이다.

이번 강남구 아파트 데이터에서 전용면적과 거래금액의 상관계수는 r = 0.806, 강한 양의 상관이다. 다만 상관은 인과가 아니다. 같은 면적이라도 단지, 위치, 층, 연식, 거래 시점에 따라 가격이 달라질 수 있으니 "면적만이 가격을 결정한다"고 말하지는 않는다.

5. 히트맵은 행과 열이 있는 숫자표를 색으로 읽는 방법이다

히트맵은 피벗 테이블에 색을 입힌 것과 비슷하다. 엑셀의 조건부 서식 색조와도 연결해서 설명할 수 있다.

이번 지하철 실습에서는 행에 역, 열에 시간대를 놓고, 값에는 연간 승차인원을 둔다. 색이 진한 칸은 해당 역과 시간대의 승차 인원이 많다는 뜻이다. 원본 파일이 크기 때문에 먼저 승차 데이터만 필터링하고, 역별·시간대별 합계로 줄인 뒤, 총 승차 상위 10개 역만 그린다.

6. 라인차트는 시간 순서가 있는 데이터에 쓴다

FRED CPI 데이터는 월별 시계열이다. 카테고리를 비교하려는 것이 아니라 시간이 흐르며 지수가 어떻게 움직였는지 보려는 것이므로 라인차트가 적합하다.

카페 매출 미션도 같은 원리다. 주문 10,424행을 월별 매출 6행으로 집계한 뒤 라인차트를 그린다. 일별 매출보다 월별 매출을 쓰는 이유는 이번 차시의 미션이 "추세를 저장해 BI에 넘길 수 있는 집계표 만들기"이기 때문이다.

7. to_csv()는 BI로 넘기는 인수인계 파일이다

시각화 도구로 넘어갈 때 원본 전체를 넘길 수도 있지만, 처음에는 차트에 쓰인 집계 결과를 따로 저장하는 것이 안전하다.

monthly_sales.to_csv("s14_monthly_sales.csv", index=False, encoding="utf-8-sig")

index=False는 pandas 인덱스를 불필요한 열로 저장하지 않기 위한 설정이다. encoding="utf-8-sig"는 Excel이나 일부 BI 도구가 한글 열 이름을 더 안정적으로 읽게 해준다. 데이터 원본은 UTF-8로 정규화되어 있지만, 전달 파일은 수강생 환경의 Excel·BI 호환성까지 고려해 저장한다.

엑셀과 연결해서 설명할 지점

  • sns.histplot()은 엑셀의 히스토그램 또는 도수분포표와 연결해서 설명한다.
  • sns.boxplot()은 엑셀 입문자에게 낯설 수 있으므로 "그룹별 중앙값과 퍼짐을 한 번에 보는 요약 그림"으로 설명한다.
  • sns.regplot()의 추세선은 엑셀 산점도에서 "추세선 추가"를 누르는 것과 비슷하다.
  • sns.heatmap()은 피벗 테이블에 조건부 서식 색조를 입힌 결과와 비슷하다.
  • to_csv()는 분석자가 BI 담당자에게 넘기는 정리된 표라고 설명한다.
  • Power BI나 Looker Studio에서 좋은 대시보드를 만들려면 원본보다 차트용 집계표가 더 적절할 때가 있다.

따라하기 1: 폴더, 패키지, 한글폰트 설정

먼저 공통 설정을 실행한다.

from pathlib import Path

import matplotlib.pyplot as plt
import matplotlib.ticker as mticker
import pandas as pd
import seaborn as sns
from matplotlib import font_manager

ROOT = Path("/mnt/d/ai-data-analysis-course")
DATA = ROOT / "02_datasets"
OUT = ROOT / "06_answers/S14_outputs"
OUT.mkdir(parents=True, exist_ok=True)

font_candidates = [
    "NanumGothic",
    "NanumGothicCoding",
    "Noto Sans CJK KR",
    "Malgun Gothic",
    "AppleGothic",
    "DejaVu Sans",
]

available_fonts = {font.name for font in font_manager.fontManager.ttflist}
selected_font = next(
    (font for font in font_candidates if font in available_fonts),
    "DejaVu Sans",
)

sns.set_theme(style="whitegrid")
plt.rcParams["font.family"] = selected_font
plt.rcParams["axes.unicode_minus"] = False

print("선택된 글꼴:", selected_font)

한글이 네모로 보이면 나눔폰트를 설치해야 한다. Ubuntu, WSL, Colab에서는 아래 명령을 참고한다.

sudo apt-get update

Colab에서는 명령 앞에 !를 붙인다. 설치 후에도 한글이 깨지면 런타임을 다시 시작하거나 matplotlib 폰트 캐시를 지운 뒤 다시 실행한다.

예상 질문과 답변

Q1. 한글이 계속 깨지면 어떻게 하나요?

A. 먼저 나눔폰트가 설치되어 있는지 확인한다. 설치 후에도 깨지면 matplotlib 폰트 캐시가 이전 상태를 기억하고 있을 수 있다. Colab에서는 런타임을 다시 시작하고, 로컬에서는 캐시 삭제 후 다시 실행한다. 수업에서는 selected_font 출력값이 NanumGothic 또는 NanumGothicCoding인지 확인한다.

Q2. 지하철 데이터는 왜 전체 역을 다 그리지 않나요?

A. 전체 역을 모두 그리면 글자가 겹쳐 읽을 수 없다. 좋은 시각화는 많은 데이터를 한 화면에 억지로 넣는 것이 아니라 질문에 맞게 줄이는 것이다. 오늘은 총 승차 상위 10개 역의 시간대 패턴을 보는 것이 목표다.

Q3. 산점도에 점이 너무 많이 겹치면 어떻게 하나요?

A. alpha로 투명도를 낮추거나, 필요한 경우 표본을 뽑거나, 구간별 집계로 바꿀 수 있다. 오늘 아파트 데이터는 3,754행이라 투명도 조정만으로도 관계를 읽을 수 있다.

Q4. 상관계수 0.806이면 전용면적이 가격의 원인인가요?

A. 아니다. 상관은 함께 움직이는 정도를 말할 뿐 원인을 증명하지 않는다. 같은 면적이라도 단지, 위치, 층, 연식, 거래 시점 등이 가격에 영향을 준다.

Q5. 왜 CSV 저장에 utf-8-sig를 쓰나요?

A. 원본 데이터는 UTF-8로 정규화되어 있다. 다만 수강생이 Excel이나 BI 도구에서 파일을 열 때 한글 열 이름이 깨지는 경우가 있어 전달용 CSV는 utf-8-sig로 저장한다. 데이터 내용이 바뀌는 것은 아니다.

Q6. Power BI에서도 같은 차트를 다시 만들 수 있나요?

A. 가능하다. 오늘 저장한 CSV는 BI에서 바로 불러올 수 있는 차트용 표다. S15 이후에는 Power BI나 Looker Studio에서 이 집계표를 불러와 대시보드 차트로 배치할 수 있다.

from pathlib import Path

import matplotlib.pyplot as plt
import matplotlib.ticker as mticker
import pandas as pd
import seaborn as sns

OUT.mkdir(parents=True, exist_ok=True)

font_candidates = [
    "NanumGothic",
    "NanumGothicCoding",
    "Noto Sans CJK KR",
    "Malgun Gothic",
    "AppleGothic",
    "DejaVu Sans",
]
available_fonts = {font.name for font in font_manager.fontManager.ttflist}

    (font for font in font_candidates if font in available_fonts),
    "DejaVu Sans",
)

sns.set_theme(style="whitegrid")

plt.rcParams["axes.unicode_minus"] = False

    .reset_index()
    .rename(columns={"date": "월"})

monthly_sales.to_csv(

    color="#A3D576",

    ax=ax,

ax.set_title("카페 월별 매출 추세")

4. 카페 월별 매출 미션

사용 파일:

02_datasets/cafe_sales/cafe_sales_clean.csv

전처리:

  • date를 날짜형으로 변환한다.
