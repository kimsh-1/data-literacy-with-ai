S03. 피벗테이블로 질문에 답하기

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S02
  • 난이도: 2
  • 시간: 50분
  • 데이터: /mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.xlsx
  • 시트: sales
  • 확인된 데이터 규모: 10,424행 × 12열

이번 차시의 핵심은 "표를 보는 법"에서 "표에 질문하는 법"으로 넘어가는 데 있다. 원본 데이터는 주문 한 건이 한 줄씩 쌓여 있다. 피벗테이블은 이 긴 주문 목록을 지점별, 카테고리별, 요일별, 시간대별로 접어서 보여 주는 요약 도구다.

학습 목표

수강생은 이 차시가 끝나면 다음을 할 수 있다.

  1. 피벗테이블의 행, 열, 값 영역이 각각 어떤 역할을 하는지 설명할 수 있다.
  2. 매출 합계, 판매수량 합계, 주문건수처럼 질문에 맞는 집계함수를 고를 수 있다.
  3. 지점×카테고리, 요일×시간대 교차표를 보고 어느 구간이 큰지 해석할 수 있다.
  4. "가장 잘 팔리는 시간대는 언제인가?"라는 질문을 피벗테이블로 답할 수 있다.

데이터 이해

사용 파일은 cafe_sales_clean.xlsx이다. 같은 내용의 CSV 파일도 있지만, 수업에서는 엑셀 조작을 위해 xlsx 파일을 연다.

주요 컬럼은 다음과 같다.

컬럼 의미 피벗에서 쓰는 방식
date 주문 날짜 이후 차시에서 날짜 분석에 사용
weekday 요일 행 또는 열
hour 주문 시간대 행 또는 열
store 지점 행
category 상품 카테고리 열 또는 행
item 상품명 세부 분석
qty 판매수량 값, 합계
amount 주문금액 값, 합계
order_id 주문번호 값, 개수

이번 차시에서는 계산식을 새로 만들지 않는다. 이미 있는 amount, qty, order_id를 이용해 피벗테이블이 자동으로 요약하게 한다.

핵심 개념

1. 피벗테이블은 긴 영수증 묶음을 요약표로 접는 도구다

카페의 하루 영수증을 모두 펼쳐 놓으면 주문 내역은 자세하지만 전체 흐름은 보이지 않는다. 점장이 알고 싶은 것은 보통 "강남점과 홍대점 중 어디가 많이 팔렸는가", "커피와 디저트 중 무엇이 매출을 만들었는가", "점심시간과 저녁시간 중 언제 바쁜가" 같은 요약 질문이다.

피벗테이블은 이 질문을 표 구조로 바꾼다.

  • 행: 무엇별로 나눌 것인가
  • 열: 무엇과 무엇을 교차해 볼 것인가
  • 값: 무엇을 계산할 것인가
  • 필터: 어떤 범위만 볼 것인가

예를 들어 "지점별 카테고리별 매출"은 다음처럼 번역된다.

  • 행: store
  • 열: category
  • 값: amount의 합계

2. 행과 열은 분류 기준이고, 값은 계산 대상이다

입문자가 가장 자주 헷갈리는 대목이 "어떤 필드를 값에 넣어야 하는가"이다. 행과 열은 그룹을 나누는 기준이고, 값은 더하거나 세거나 평균낼 숫자다.

엑셀의 일반 표에서 여러 조건을 겹쳐 일일이 계산하려면 조건마다 수식을 새로 짜야 한다. 피벗테이블은 그 일을 마우스만으로 구성한다. store=강남점, category=커피 같은 조건 조합을 엑셀이 알아서 만들고, 그 안의 amount를 합산한다.

3. 집계함수는 질문의 뜻을 정한다

"많이 팔렸다"는 말은 한 가지 뜻이 아니다.

  • 매출이 큰가: amount의 합계
  • 수량이 많은가: qty의 합계
  • 주문이 자주 들어왔는가: order_id의 개수
  • 객단가가 높은가: amount의 평균 또는 별도 계산

이번 미션에서는 "가장 잘 팔리는 시간대"를 매출액 합계 기준으로 판단한다. 다만 판매수량과 주문건수도 함께 확인해, 같은 시간대가 반복해서 1위인지 비교한다.

4. 교차표는 한 줄씩 읽지 말고 패턴을 본다

요일×시간대 피벗은 숫자가 많아서 칸을 하나하나 읽으려 들면 오히려 길을 잃는다. 큰 숫자가 몰리는 시간대를 먼저 찾고, 그다음 특정 요일에서 유난히 큰 칸이 있는지 확인한다.

이 데이터에서는 매출액 합계 기준으로 13시가 전체 1위다. 요일과 시간대를 함께 보면 일요일 13시가 가장 큰 조합이다. 이 결론은 실제 데이터로 검증한 값이다.

엑셀 기능 비교

하고 싶은 일 일반 엑셀 방식 피벗테이블 방식
지점별 매출 합계 SUMIF 또는 필터 후 합계 행에 store, 값에 amount
지점×카테고리 매출 여러 조건 조합을 일일이 계산 행에 store, 열에 category, 값에 amount
시간대별 주문건수 COUNTIF 또는 필터 행에 hour, 값에 order_id 개수
요일×시간대 비교 조건별 표를 직접 작성 행에 weekday, 열에 hour, 값에 amount

피벗테이블은 수식을 몰라도 시작할 수 있다. 대신 "어떤 질문을 어떤 필드 배치로 바꿀 것인가"는 스스로 생각해야 한다.

50분 수업 흐름

시간 내용 진행 방식
0-5분 S02 복습과 오늘의 질문 제시 원본 데이터가 주문 한 건 단위임을 확인
5-12분 피벗테이블 개념 설명 행, 열, 값, 필터를 영수증 요약 비유로 설명
12-25분 따라하기 1: 지점×카테고리 피벗 store×category, amount 합계
25-38분 따라하기 2: 요일×시간대 피벗 weekday×hour, amount 합계
38-47분 직접 해보기: 가장 잘 팔리는 시간대 hour별 매출, 수량, 주문건수 비교
47-50분 정리와 체크포인트 질문을 필드 배치로 번역하는 법 확인

수업 중 강조할 원칙

  1. 피벗테이블은 원본을 바꾸지 않고 요약표를 만든다.
  2. 값 영역에 숫자 컬럼을 넣으면 보통 합계가 기본값이다.
  3. 값 영역에 문자 또는 주문번호를 넣으면 개수가 나올 수 있다.
  4. "합계", "개수", "평균" 중 무엇을 골랐는지에 따라 답이 달라진다.
  5. 결과를 말할 때는 기준을 함께 말한다. 예: "매출액 합계 기준으로 13시가 1위입니다."

차시 평가 기준

  • 상: 피벗테이블을 새로 만들고, 시간대별 매출 1위를 찾아 기준과 함께 해석한다.
  • 중: 따라하기 피벗을 재현하고, 강사의 안내를 받아 미션 결과를 찾는다.
  • 하: 피벗테이블 생성 또는 값 필드 설정에서 막혀 결과를 재현하지 못한다.

오늘의 목표

피벗테이블을 이용해 다음 두 질문에 답한다.

  1. 지점별로 어떤 카테고리가 매출을 많이 만드는가?
  2. 요일과 시간대에 따라 매출은 어떻게 달라지는가?

마지막에는 직접 "가장 잘 팔리는 시간대"를 찾는다.

피벗테이블 기본 구조

피벗테이블 필드 목록에는 보통 네 영역이 있다.

영역 넣는 것 예시
행 세로로 나눌 기준 store, weekday, hour
열 가로로 나눌 기준 category, hour
값 계산할 숫자 amount 합계, qty 합계
필터 일부만 볼 조건 특정 지점, 특정 카테고리

중요한 질문은 "무엇별로 나눌 것인가"와 "무엇을 계산할 것인가"이다.

따라하기 1. 지점×카테고리 매출 피벗 만들기

질문: 지점별로 커피, 음료, 디저트 매출이 어떻게 다른가?

1단계. 피벗테이블 삽입

  1. sales 시트에서 데이터 안의 아무 셀이나 클릭한다.
  2. 상단 메뉴에서 삽입을 클릭한다.
  3. 피벗 테이블을 클릭한다.
  4. 표/범위가 자동으로 잡히는지 확인한다.
  5. 위치는 새 워크시트를 선택한다.
  6. 확인을 클릭한다.

2단계. 필드 배치

오른쪽의 피벗테이블 필드 목록에서 다음처럼 끌어다 놓는다.

필드 놓을 영역
store 행
category 열
amount 값

값 영역에 합계: amount가 보이면 맞게 설정된 것이다.

3단계. 금액 형식 정리

  1. 피벗테이블 안의 매출 숫자 중 하나를 오른쪽 클릭한다.
  2. 값 필드 설정을 클릭한다.
  3. 집계 방식이 합계인지 확인한다.
  4. 표시 형식을 클릭한다.
  5. 숫자를 선택하고 1000 단위 구분 기호 사용을 켠다.
  6. 소수 자릿수는 0으로 설정한다.

4단계. 결과 읽기

실제 데이터 기준 결과는 다음과 같다.

지점 디저트 음료 커피 총합계
강남점 11,604,000 15,281,500 26,539,500 53,425,000
홍대점 7,732,500 10,268,500 19,066,500 37,067,500
총합계 19,336,500 25,550,000 45,606,000 90,492,500

해석 문장 예시는 다음과 같다.

강남점과 홍대점 모두 커피 매출이 가장 크다. 전체 매출은 강남점이 53,425,000원으로 홍대점을 앞선다.

따라하기 2. 요일×시간대 매출 피벗 만들기

질문: 요일과 시간대에 따라 매출이 어떻게 달라지는가?

1단계. 새 피벗테이블 만들기

  1. 다시 sales 시트로 이동한다.
  2. 데이터 안의 아무 셀이나 클릭한다.
  3. 삽입 > 피벗 테이블을 클릭한다.
  4. 위치는 새 워크시트를 선택한다.
  5. 확인을 클릭한다.

2단계. 필드 배치

필드 놓을 영역
weekday 행
hour 열
amount 값

값 영역은 합계: amount로 둔다. hour 값은 8부터 21까지 보인다. 여기서는 각 숫자를 8시, 9시, 10시처럼 해석한다.

3단계. 숫자 표시 형식 정리

따라하기 1과 같은 방식으로 매출 숫자에 1000 단위 구분 기호를 적용한다.

4단계. 교차표 읽기

교차표는 한 칸씩 외우는 표가 아니라 흐름을 읽는 표다. 다음 순서로 본다.

  1. 숫자가 큰 시간대가 어느 열에 몰리는지 본다.
  2. 특정 요일에서 유난히 큰 값이 있는지 본다.
  3. 총합계를 보고 전체 흐름을 확인한다.

실제 데이터에서는 13시 매출이 가장 크다. 요일과 시간대를 함께 보면 일요일 13시가 2,260,500원으로 가장 큰 조합이다.

요일 값은 원본 데이터에 영어 약어로 들어 있다.

Tue 화요일
Wed 수요일
Thu 목요일
Fri 금요일
Sat 토요일
Sun 일요일

직접 해보기. 가장 잘 팔리는 시간대 찾기

질문: 매출액 합계 기준으로 가장 잘 팔리는 시간대는 언제인가?

조건

새 피벗테이블을 만들어 아래처럼 구성한다.

필드 놓을 영역
hour 행
amount 값
qty 값
order_id 값

설정은 다음처럼 맞춘다.

  • amount: 합계
  • qty: 합계
  • order_id: 개수

amount 합계를 기준으로 내림차순 정렬한다.

제출할 답

아래 형식으로 한 문장을 작성한다.

매출액 합계 기준 가장 잘 팔리는 시간대는 __시이며, 매출은 ______원, 판매수량은 ______개, 주문건수는 ______건이다.

추가 질문

시간이 남으면 다음도 확인한다.

  1. 판매수량 기준 1위 시간대도 같은가?
  2. 주문건수 기준 1위 시간대도 같은가?
  3. 요일×시간대를 함께 볼 때 가장 큰 조합은 무엇인가?

자주 생기는 문제

값이 합계가 아니라 개수로 나오는 경우

값 영역의 필드 이름 옆 화살표를 누르고 값 필드 설정으로 들어간다. 합계, 개수, 평균 중 원하는 계산 방식을 선택한다.

요일 순서가 월, 화, 수 순서가 아닌 경우

이번 차시에서는 정렬 순서보다 값의 크기를 읽는 것이 중요하다. 순서가 달라도 해석에는 지장이 없다. 날짜와 요일 정렬은 이후 차시에서 더 다룬다.

피벗테이블 필드 목록이 보이지 않는 경우

피벗테이블 안의 아무 셀이나 클릭한다. 그래도 보이지 않으면 상단의 피벗 테이블 분석 메뉴에서 필드 목록을 켠다.

0-5분. 도입

강사 멘트:

"지난 시간에는 깨끗한 표가 왜 중요한지 봤습니다. 오늘은 그 표를 가지고 질문에 답하겠습니다. 데이터분석에서 중요한 것은 표를 오래 바라보는 것이 아니라, 질문을 표 구조로 바꾸는 것입니다."

"오늘의 질문은 두 가지입니다. 지점별로 어떤 카테고리가 매출을 만드는가, 그리고 어떤 시간대가 잘 팔리는가입니다. 이 질문을 피벗테이블로 풀어 보겠습니다."

시연 포인트:

  • sales 시트를 열고 행이 주문 한 건 단위로 쌓여 있음을 보여준다.
  • amount는 주문금액, qty는 판매수량, order_id는 주문번호임을 짚는다.
  • 원본 표를 직접 수정하지 않고 피벗테이블을 새 시트에 만든다고 설명한다.

5-12분. 개념 설명

강사 멘트:

"피벗테이블은 긴 영수증 묶음을 요약표로 접는 도구입니다. 원본에는 주문 한 건 한 건이 있지만, 우리는 보통 전체 흐름을 알고 싶습니다."

"피벗테이블 필드에는 행, 열, 값, 필터가 있습니다. 행과 열은 나누는 기준입니다. 값은 계산할 숫자입니다. 필터는 일부만 보는 조건입니다."

"예를 들어 지점별 카테고리별 매출을 보고 싶다면, 지점은 행, 카테고리는 열, 매출액은 값으로 들어갑니다. 이 번역이 피벗테이블의 핵심입니다."

판서 또는 화면 강조:

  • 질문: 지점별 카테고리별 매출은?
  • 행: store
  • 열: category
  • 값: amount 합계

주의해서 말할 점:

  • 값 영역에 무엇을 넣었는지가 답의 기준이다.
  • amount 합계는 매출 기준이고, qty 합계는 수량 기준이다.
  • order_id 개수는 주문건수 기준이다.

12-25분. 따라하기 1: 지점×카테고리 매출

시연 절차:

  1. sales 시트에서 데이터 안의 아무 셀이나 클릭한다.
  2. 삽입 > 피벗 테이블을 클릭한다.
  3. 새 워크시트를 선택하고 확인한다.
  4. store를 행에 넣는다.
  5. category를 열에 넣는다.
  6. amount를 값에 넣는다.
  7. 값 필드 설정이 합계인지 확인한다.
  8. 숫자 표시 형식을 1000 단위 구분, 소수 0자리로 바꾼다.

강사 멘트:

"이 표는 원본 데이터를 지운 것이 아니라, 같은 데이터를 다른 방식으로 요약한 것입니다. 행에는 강남점과 홍대점이 생기고, 열에는 디저트, 음료, 커피가 자리를 잡았습니다. 각 칸은 그 지점과 카테고리가 만나는 주문금액의 합계입니다."

"강남점의 커피 매출은 26,539,500원입니다. 홍대점의 커피 매출은 19,066,500원입니다. 이렇게 한 칸은 두 조건이 만나는 자리입니다."

확인 질문:

"강남점에서 가장 매출이 큰 카테고리는 무엇인가요?"

기대 답:

"커피입니다."

시연 포인트:

  • "총합계"가 행과 열 양쪽에 생기는 것을 설명한다.
  • 강남점 총합계 53,425,000원, 홍대점 총합계 37,067,500원을 보여준다.
  • 두 지점 모두 커피 비중이 가장 크다는 해석을 말하게 한다.

25-38분. 따라하기 2: 요일×시간대 매출

시연 절차:

  1. sales 시트로 돌아간다.
  2. 새 피벗테이블을 만든다.
  3. weekday를 행에 넣는다.
  4. hour를 열에 넣는다.
  5. amount를 값에 넣는다.
  6. 숫자 표시 형식을 정리한다.

강사 멘트:

"이번에는 행과 열을 바꿔서 시간 패턴을 보겠습니다. 행은 요일, 열은 시간대입니다. 값은 여전히 매출액 합계입니다."

"교차표는 모든 칸을 읽으려고 하면 어렵습니다. 큰 숫자가 어느 열에 몰리는지부터 봅니다. 이 데이터는 12시, 13시, 14시가 크게 보이고, 그중 13시가 가장 큽니다."

"요일과 시간대를 동시에 보면 일요일 13시가 가장 큰 칸입니다. 이 말은 '일요일 13시에 주문이 가장 많았다'가 아니라, 정확히는 '일요일 13시의 매출액 합계가 가장 컸다'입니다."

시연 포인트:

  • hour가 8부터 21까지 보이는 것을 확인한다.
  • weekday는 영어 약어이므로 Mon은 월요일, Sun은 일요일이라고 짚는다.
  • 요일 순서가 월화수목금토일로 보이지 않아도 이번 차시의 핵심은 값 읽기라고 말한다.

38-47분. 직접 해보기: 가장 잘 팔리는 시간대

미션 안내 멘트:

"이제 여러분이 직접 하나의 질문에 답해 보겠습니다. 질문은 '가장 잘 팔리는 시간대는 언제인가'입니다. 기준을 먼저 정해야 합니다. 이번에는 매출액 합계 기준으로 보겠습니다."

수강생 작업:

  1. 새 피벗테이블을 만든다.
  2. hour를 행에 넣는다.
  3. amount를 값에 넣고 합계로 둔다.
  4. qty를 값에 넣고 합계로 둔다.
  5. order_id를 값에 넣고 개수로 바꾼다.
  6. amount 합계 기준으로 내림차순 정렬한다.
  7. 한 문장으로 답을 쓴다.

정답 안내:

"매출액 합계 기준 가장 잘 팔리는 시간대는 13시입니다. 매출은 11,626,000원, 판매수량은 2,220개, 주문건수는 1,348건입니다. 이 데이터에서는 판매수량 기준과 주문건수 기준으로 봐도 13시가 1위입니다."

47-50분. 정리

강사 멘트:

"오늘 배운 내용은 피벗테이블 버튼 하나가 아니라 질문을 배치로 바꾸는 방법입니다. 지점별 카테고리별 매출이라는 질문은 행에 지점, 열에 카테고리, 값에 매출 합계로 바뀝니다. 가장 잘 팔리는 시간대라는 질문은 행에 시간대, 값에 매출 합계로 바뀝니다."

"다음 차시부터는 여러 표를 연결하거나 계산 컬럼을 만드는 방식으로 질문의 범위를 넓혀 갑니다."

완료 판단 체크포인트:

수강생이 hour별 피벗테이블을 만들고 "매출액 합계 기준 13시가 1위이며 매출은 11,626,000원"이라고 말하면 이번 차시를 완료한 것으로 본다.

예상 질문과 답변

Q1. 값이 합계가 아니라 개수로 나옵니다.

A. 값 영역의 필드 옆 화살표를 누르고 값 필드 설정으로 들어가 합계를 선택한다. 숫자처럼 보이지 않는 필드이거나 빈 값이 섞이면 엑셀이 개수로 잡을 수 있다.

Q2. order_id는 왜 합계가 아니라 개수로 보나요?

A. 주문번호는 더해 봐야 의미가 없다. 주문이 몇 건 있었는지 세는 식별자이므로 개수가 맞다.

Q3. 매출 1위와 수량 1위가 다를 수도 있나요?

A. 다를 수 있다. 비싼 상품이 적게 팔린 시간대는 매출이 높게, 싼 상품이 많이 팔린 시간대는 수량이 높게 나오기 때문이다. 그래서 "잘 팔린다"는 말을 쓸 때는 기준을 함께 밝혀야 한다.

Q4. 요일 순서가 이상합니다.

A. 원본의 weekday 값이 영어 약어라 엑셀이 자동으로 월화수목금토일 순서로 정렬하지 않을 수 있다. 이번 차시에서는 순서보다 교차표를 읽는 법이 중요하다. 이후 날짜 차시에서 정렬을 더 다룬다.

Q5. 원본 데이터가 바뀌면 피벗테이블도 자동으로 바뀌나요?

A. 원본을 수정한 뒤 피벗테이블을 오른쪽 클릭하고 새로 고침을 눌러야 반영된다. 피벗테이블은 원본을 참조하되, 늘 그 자리에서 곧바로 다시 계산되지는 않는다.

강의 운영 메모

  • 수강생이 필드 배치를 외우려 하면 질문으로 되돌린다. "무엇별로 나눌까요?", "무엇을 계산할까요?"라고 묻는다.
  • 첫 피벗에서는 필드가 어디에 들어가는지 천천히 보여준다.
  • 두 번째 피벗부터는 수강생이 먼저 예측하게 한다.
  • 미션에서는 정답 숫자보다 기준을 말하는 습관을 더 중요하게 본다.

S03. 모범답안

미션 정답

미션 질문은 "매출액 합계 기준으로 가장 잘 팔리는 시간대는 언제인가?"이다.

정답:

매출액 합계 기준 가장 잘 팔리는 시간대는 13시이며, 매출은 11,626,000원, 판매수량은 2,220개, 주문건수는 1,348건이다.

이 데이터에서는 판매수량 기준과 주문건수 기준으로 봐도 13시가 1위다.

점심 직후인 13시가 매출액, 판매수량, 주문건수 모두에서 가장 크다. 따라서 이 데이터만 놓고 보면 13시를 가장 잘 팔리는 시간대로 판단할 수 있다.

따라하기 1 검증 결과: 지점×카테고리 매출

따라하기 2 검증 결과: 요일×시간대

요일×시간대를 함께 보았을 때 매출액 합계가 가장 큰 조합은 Sun 13시이며, 매출액은 2,260,500원이다.

Sun은 일요일을 뜻한다.

검증용 pandas 코드

아래 코드는 엑셀 피벗테이블 결과와 같은 계산을 pandas로 재현한 것이다. 작성 시 실제로 실행해 결과를 확인했다.

from pathlib import Path

import pandas as pd

data_path = Path("/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.csv")
df = pd.read_csv(data_path)

assert df.shape == (10424, 12)
assert df["amount"].isna().sum() == 0
assert df["qty"].isna().sum() == 0

store_category = pd.pivot_table(
    df,
    index="store",
    columns="category",
    values="amount",
    aggfunc="sum",
    margins=True,
    margins_name="총합계",
).fillna(0).astype(int)

hour_summary = (
    df.groupby("hour")
    .agg(
        주문건수=("order_id", "count"),
        판매수량=("qty", "sum"),
        매출액=("amount", "sum"),
    )
    .reset_index()
    .sort_values("매출액", ascending=False)
)

weekday_order = ["Mon", "Tue", "Wed", "Thu", "Fri", "Sat", "Sun"]
weekday_hour = (
    pd.pivot_table(
        df,
        index="weekday",
        columns="hour",
        values="amount",
        aggfunc="sum",
    )
    .reindex(weekday_order)
    .fillna(0)
    .astype(int)
)

top_hour = hour_summary.iloc[0]
top_weekday_hour_raw = weekday_hour.stack().sort_values(ascending=False).index[0]
top_weekday_hour = (top_weekday_hour_raw[0], int(top_weekday_hour_raw[1]))
top_weekday_hour_amount = int(weekday_hour.stack().sort_values(ascending=False).iloc[0])

print("데이터 행열:", df.shape)
print("전체 매출:", int(df["amount"].sum()))
print("전체 판매수량:", int(df["qty"].sum()))
print()
print("[지점 x 카테고리 매출]")
print(store_category)
print()
print("[시간대별 상위 5개]")
print(hour_summary.head(5).to_string(index=False))
print()
print(
    "매출 1위 시간대:",
    f"{int(top_hour['hour'])}시",
    f"매출 {int(top_hour['매출액']):,}원",
    f"판매수량 {int(top_hour['판매수량']):,}개",
    f"주문건수 {int(top_hour['주문건수']):,}건",
)
print(
    "요일 x 시간대 최고 조합:",
    top_weekday_hour,
    f"{top_weekday_hour_amount:,}원",
)
