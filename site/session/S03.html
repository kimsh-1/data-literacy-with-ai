S03. 피벗테이블로 질문에 답하기

차시 개요

피벗테이블은 긴 주문 목록을 질문에 맞게 접어서 보는 엑셀의 요약 도구입니다. 원본 데이터에는 주문 한 건이 한 줄씩 쌓여 있습니다. 이렇게 자세한 표는 기록으로는 좋지만, “어느 지점이 많이 팔렸나요?”, “어느 시간대가 바쁜가요?” 같은 질문에 바로 답하기는 어렵습니다.

그래서 이번 차시에서는 표를 한 줄씩 읽는 방식에서 벗어나, 표에 질문을 던지고 피벗테이블로 답을 찾는 방식으로 넘어가 봅니다.

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S02
  • 난이도: 2
  • 시간: 50분
  • 데이터: 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 엑셀 파일
  • 시트: sales
  • 확인된 데이터 규모: 10,424행 × 12열

카페 매출 데이터는 주문 내역이 길게 이어진 표입니다. 피벗테이블을 쓰면 이 긴 주문 목록을 지점별, 카테고리별, 요일별, 시간대별로 접어서 볼 수 있습니다. 종이를 접으면 한눈에 보이는 면이 달라지듯, 같은 데이터도 어떻게 접느냐에 따라 보이는 질문과 답이 달라집니다.

학습 목표

수강생은 이 차시가 끝나면 아래 내용을 직접 설명하고 실행할 수 있습니다.

  1. 피벗테이블의 행, 열, 값 영역이 각각 어떤 역할을 하는지 설명할 수 있다.
  2. 매출 합계, 판매수량 합계, 주문건수처럼 질문에 맞는 집계함수를 고를 수 있다.
  3. 지점×카테고리, 요일×시간대 교차표를 보고 어느 구간이 큰지 해석할 수 있다.
  4. "가장 잘 팔리는 시간대는 언제인가?"라는 질문을 피벗테이블로 답할 수 있다.

여기서 중요한 점은 버튼을 누르는 순서를 외우는 것이 아닙니다. “내가 알고 싶은 질문을 엑셀의 행, 열, 값에 어떻게 나누어 넣을까?”를 생각하는 힘을 기르는 것입니다.

데이터 이해

사용할 데이터는 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일입니다. 같은 내용은 엑셀로도 열리는 표 파일인 CSV 형태로도 받을 수 있지만, 초보자는 엑셀 파일(.xlsx)을 받아서 시작해도 괜찮습니다. 수업에서는 엑셀에서 바로 조작하기 쉬운 엑셀 파일을 기준으로 봅니다.

주요 컬럼은 아래 표로 확인해 봅니다.

컬럼 의미 피벗에서 쓰는 방식
date 주문 날짜 이후 차시에서 날짜 분석에 사용
weekday 요일 행 또는 열
hour 주문 시간대 행 또는 열
store 지점 행
category 상품 카테고리 열 또는 행
item 상품명 세부 분석
qty 판매수량 값, 합계
amount 주문금액 값, 합계
order_id 주문번호 값, 개수

이번 차시에서는 계산식을 새로 만들지 않습니다. 이미 들어 있는 amount, qty, order_id를 사용합니다. amount는 돈의 크기를 볼 때, qty는 몇 개가 팔렸는지 볼 때, order_id는 주문이 몇 번 들어왔는지 셀 때 씁니다.

엑셀에게 “이 숫자들을 지점별로 더해 주세요”, “이 주문번호들을 시간대별로 세어 주세요”라고 맡기는 것이 피벗테이블의 기본 흐름입니다.

꼭 잡고 가야 할 개념

1. 피벗테이블은 긴 영수증 묶음을 요약표로 접는 도구다

카페의 하루 영수증을 모두 펼쳐 놓으면 주문 내역은 아주 자세합니다. 누가 언제 무엇을 주문했는지는 보이죠. 하지만 점장이 정말 궁금한 것은 보통 조금 다릅니다.

“강남점과 홍대점 중 어디가 많이 팔렸을까요?” “커피와 디저트 중 무엇이 매출을 더 만들었을까요?” “점심시간과 저녁시간 중 언제 더 바쁠까요?”

이런 질문은 한 줄짜리 주문 내역만 봐서는 바로 답하기 어렵습니다. 피벗테이블은 이 질문을 요약표 구조로 바꾸어 줍니다. 마치 키를 잴 때 사람을 한 명씩 보는 것이 아니라 반별 평균 키를 보는 것처럼, 자세한 기록을 질문에 맞게 묶어서 보여 주는 방식입니다.

피벗테이블에는 네 가지 자리가 있습니다.

  • 행: 무엇별로 나눌 것인가
  • 열: 무엇과 무엇을 교차해 볼 것인가
  • 값: 무엇을 계산할 것인가
  • 필터: 어떤 범위만 볼 것인가

예를 들어 “지점별 카테고리별 매출”을 보고 싶다면 질문을 엑셀 말로 바꾸어야 합니다.

  • 행: store
  • 열: category
  • 값: amount의 합계

이렇게 배치하면 엑셀은 지점과 카테고리의 조합을 자동으로 만들고, 각 조합 안에 들어 있는 주문금액을 더해 줍니다.

2. 행과 열은 분류 기준이고, 값은 계산 대상이다

입문자가 가장 자주 헷갈리는 부분은 “어떤 필드를 값에 넣어야 하나요?”입니다. 여기서는 역할을 나누어 생각하면 쉽습니다.

행과 열은 나누어 보는 기준입니다. store를 행에 넣으면 지점별로 나뉩니다. category를 열에 넣으면 카테고리별로 가로 방향에 펼쳐집니다. 이 둘은 계산할 숫자가 아니라, 데이터를 어떤 상자에 나누어 담을지 정하는 이름표입니다.

값은 실제로 더하거나 세거나 평균을 낼 대상입니다. amount를 값에 넣으면 금액을 계산합니다. qty를 값에 넣으면 판매수량을 계산합니다. order_id를 값에 넣으면 주문번호를 세어서 주문건수를 볼 수 있습니다.

일반 엑셀 표에서 store=강남점, category=커피 같은 조건을 하나씩 만들어 계산하려면 수식을 여러 번 써야 합니다. 피벗테이블은 이 조건 조합을 대신 만들어 줍니다. 사전에서 단어를 찾듯이, 엑셀이 각 조건에 맞는 행들을 찾아 묶고 그 안의 값을 계산해 줍니다.

3. 집계함수는 질문의 뜻을 정한다

“많이 팔렸다”라는 말은 생각보다 여러 뜻을 가질 수 있습니다. 돈을 많이 벌었다는 뜻일 수도 있고, 상품이 많이 나갔다는 뜻일 수도 있습니다. 주문이 자주 들어왔다는 뜻일 수도 있죠.

  • 매출이 큰가: amount의 합계
  • 수량이 많은가: qty의 합계
  • 주문이 자주 들어왔는가: order_id의 개수
  • 객단가가 높은가: amount의 평균 또는 별도 계산

그래서 피벗테이블에서는 값 영역에 무엇을 넣는지만큼, 어떤 계산 방식을 고르는지도 중요합니다. 합계를 고르면 전체 크기를 봅니다. 개수를 고르면 몇 번 발생했는지를 봅니다. 평균을 고르면 한 번당 크기를 봅니다.

이번 미션에서는 “가장 잘 팔리는 시간대”를 매출액 합계 기준으로 판단합니다. 다만 판매수량과 주문건수도 함께 확인합니다. 같은 시간대가 매출에서도 1위이고, 수량이나 주문건수에서도 높은지 비교하면 결과를 더 정확하게 읽을 수 있습니다.

4. 교차표는 한 줄씩 읽지 말고 패턴을 본다

요일×시간대 피벗은 숫자가 많이 나옵니다. 이때 칸을 하나하나 읽으려고 하면 금방 길을 잃습니다. 교차표는 모든 숫자를 외우는 표가 아니라, 큰 흐름을 찾는 표입니다.

먼저 큰 숫자가 몰리는 시간대를 봅니다. 그다음 특정 요일에서 유난히 큰 칸이 있는지 확인합니다. 마지막으로 총합계를 보면서 전체 흐름과 맞는지 살펴봅니다.

이 데이터에서는 매출액 합계 기준으로 13시가 전체 1위입니다. 요일과 시간대를 함께 보면 일요일 13시가 가장 큰 조합입니다. 이 결론은 실제 데이터로 검증한 값입니다.

엑셀 기능 비교

하고 싶은 일 일반 엑셀 방식 피벗테이블 방식
지점별 매출 합계 SUMIF 또는 필터 후 합계 행에 store, 값에 amount
지점×카테고리 매출 여러 조건 조합을 일일이 계산 행에 store, 열에 category, 값에 amount
시간대별 주문건수 COUNTIF 또는 필터 행에 hour, 값에 order_id 개수
요일×시간대 비교 조건별 표를 직접 작성 행에 weekday, 열에 hour, 값에 amount

피벗테이블은 수식을 몰라도 시작할 수 있습니다. 대신 질문은 사람이 먼저 정해야 합니다. “무엇별로 나누어 볼까?”, “무엇을 계산할까?”, “합계를 볼까, 개수를 볼까?”를 정하면 엑셀이 계산을 맡아 줍니다.

50분 수업 흐름

시간 내용 진행 방식
0-5분 S02 복습과 오늘의 질문 제시 원본 데이터가 주문 한 건 단위임을 확인
5-12분 피벗테이블 개념 설명 행, 열, 값, 필터를 영수증 요약 비유로 설명
12-25분 따라하기 1: 지점×카테고리 피벗 store×category, amount 합계
25-38분 따라하기 2: 요일×시간대 피벗 weekday×hour, amount 합계
38-47분 직접 해보기: 가장 잘 팔리는 시간대 hour별 매출, 수량, 주문건수 비교
47-50분 정리와 체크포인트 질문을 필드 배치로 번역하는 법 확인

수업은 먼저 원본 데이터가 주문 한 건 단위라는 점을 확인하는 데서 출발합니다. 그다음 같은 데이터를 지점별, 카테고리별, 요일별, 시간대별로 접어 보면서 피벗테이블의 감각을 익힙니다.

마지막에는 직접 “가장 잘 팔리는 시간대”를 찾아봅니다. 여기서는 단순히 1위 시간대를 찾는 데서 끝내지 않고, 그 기준이 매출인지, 수량인지, 주문건수인지 함께 말해 보는 연습을 합니다.

수업 중 강조할 원칙

  1. 피벗테이블은 원본을 바꾸지 않고 요약표를 만든다.
  2. 값 영역에 숫자 컬럼을 넣으면 보통 합계가 기본값이다.
  3. 값 영역에 문자 또는 주문번호를 넣으면 개수가 나올 수 있다.
  4. "합계", "개수", "평균" 중 무엇을 골랐는지에 따라 답이 달라진다.
  5. 결과를 말할 때는 기준을 함께 말한다. 예: "매출액 합계 기준으로 13시가 1위입니다."

여기서 특히 조심할 점은 원본과 요약표를 구분하는 것입니다. 피벗테이블은 원본 데이터를 고치는 기능이 아닙니다. 원본은 그대로 두고, 그 위에 질문에 맞는 요약표를 하나 더 만드는 기능입니다.

또 하나는 기준을 함께 말하는 습관입니다. “13시가 제일 잘 팔립니다”라고만 말하면 조금 부족합니다. “매출액 합계 기준으로 13시가 1위입니다”라고 말해야 무엇을 보고 판단했는지 분명해집니다.

차시 평가 기준

  • 상: 피벗테이블을 새로 만들고, 시간대별 매출 1위를 찾아 기준과 함께 해석한다.
  • 중: 따라하기 피벗을 재현하고, 강사의 안내를 받아 미션 결과를 찾는다.
  • 하: 피벗테이블 생성 또는 값 필드 설정에서 막혀 결과를 재현하지 못한다.

평가에서는 예쁜 표를 만드는 것보다 질문을 표 구조로 바꾸는 과정을 봅니다. 행에는 무엇을 넣었는지, 열에는 무엇을 넣었는지, 값에는 무엇을 계산했는지 설명할 수 있으면 피벗테이블의 큰 원리를 잡은 것입니다.

S03. 피벗테이블로 질문에 답하기

실습 파일 받기

수업 홈의 데이터 받기 페이지에서 카페 매출 파일을 내려받아 엑셀에서 여세요. 초보자는 엑셀 파일(.xlsx)로 받으면 됩니다. CSV, 즉 엑셀로도 열리는 표 파일로 받아도 같은 데이터를 볼 수 있지만, 손으로 따라 하기에는 엑셀 파일이 더 편합니다.

  • 시트 이름: sales
  • 데이터 규모: 10,424행 × 12열
  • 이번 차시에서 주로 사용할 컬럼: store, category, weekday, hour, amount, qty, order_id

무엇을 답할까요?

피벗테이블은 긴 주문 목록을 질문에 맞게 접어서 보는 도구입니다. 주문 한 건 한 건을 모두 읽는 대신, “지점별로”, “요일별로”, “시간대별로” 묶어서 합계나 개수를 바로 봅니다.

오늘은 피벗테이블을 이용해 두 가지 질문에 답해 봅니다.

  1. 지점별로 어떤 카테고리가 매출을 많이 만드는가?
  2. 요일과 시간대에 따라 매출은 어떻게 달라지는가?

마지막에는 직접 "가장 잘 팔리는 시간대"를 찾습니다. 여기서 “잘 팔린다”는 말은 느낌이 아니라, 매출액 합계라는 기준으로 판단합니다.

피벗테이블 기본 구조

피벗테이블 필드 목록에는 보통 네 영역이 있습니다. 처음에는 이름이 낯설 수 있지만, “표를 어떤 기준으로 나눌지”와 “무엇을 계산할지”만 구분하면 됩니다.

영역 넣는 것 예시
행 세로로 나눌 기준 store, weekday, hour
열 가로로 나눌 기준 category, hour
값 계산할 숫자 amount 합계, qty 합계
필터 일부만 볼 조건 특정 지점, 특정 카테고리

행과 열은 기준입니다. 키를 잴 때 사람 이름이 기준이 되고 키가 숫자인 것처럼, 피벗테이블에서도 store, category, weekday는 나누는 기준이 됩니다.

값은 계산 대상입니다. 매출을 알고 싶으면 amount, 판매수량을 알고 싶으면 qty, 주문건수를 알고 싶으면 order_id를 값 영역에 넣고 개수로 셉니다.

여기서 꼭 잡아야 하는 질문은 두 가지입니다. “무엇별로 나눌 것인가?” 그리고 “무엇을 계산할 것인가?”입니다.

따라하기 1. 지점×카테고리 매출 피벗 만들기

질문: 지점별로 커피, 음료, 디저트 매출이 어떻게 다른가?

이 질문을 피벗테이블 말로 바꾸면, 지점은 행에 놓고 카테고리는 열에 놓고 매출액은 값에 놓는다는 뜻입니다. 사전에서 단어를 찾듯이, 질문 속 단어를 피벗테이블 영역에 하나씩 맞춰 넣어 봅니다.

1단계. 피벗테이블 삽입

이 단계는 원본 주문 목록을 건드리지 않고, 새 요약표를 만들기 위해 필요합니다. 피벗테이블은 원본 데이터를 바꾸는 기능이 아니라 원본을 바라보는 요약 창이라고 생각하면 됩니다.

  1. sales 시트에서 데이터 안의 아무 셀이나 클릭한다.
  2. 상단 메뉴에서 삽입을 클릭한다.
  3. 피벗 테이블을 클릭한다.
  4. 표/범위가 자동으로 잡히는지 확인한다.
  5. 위치는 새 워크시트를 선택한다.
  6. 확인을 클릭한다.

표/범위가 자동으로 잡힌다는 것은 엑셀이 “이 표 전체를 분석 대상으로 쓰면 되겠구나” 하고 알아본 상태입니다. 새 워크시트를 고르면 원본 표와 결과표가 섞이지 않아서 훨씬 보기 쉽습니다.

2단계. 필드 배치

이 단계는 질문을 표 모양으로 번역하는 과정입니다. “지점별”은 행, “카테고리별”은 열, “매출”은 값으로 옮겨 놓습니다.

오른쪽의 피벗테이블 필드 목록에서 아래처럼 끌어다 놓아 보세요.

필드 놓을 영역
store 행
category 열
amount 값

값 영역에 합계: amount가 보이면 맞게 설정된 것입니다. amount는 주문금액이므로, 여러 주문을 묶었을 때 합계를 내야 지점별 매출을 볼 수 있습니다.

만약 개수: amount처럼 보이면 매출을 더한 것이 아니라 주문 줄 수를 센 상태일 수 있습니다. 그럴 때는 값 필드 설정에서 합계로 바꿔 주세요.

3단계. 금액 형식 정리

이 단계는 계산 결과를 읽기 쉽게 만들기 위해 필요합니다. 숫자에 쉼표가 없으면 53425000처럼 보여서 한눈에 금액 크기를 판단하기 어렵습니다. 가격표에 천 단위 쉼표를 찍어 두는 것과 같은 일입니다.

  1. 피벗테이블 안의 매출 숫자 중 하나를 오른쪽 클릭한다.
  2. 값 필드 설정을 클릭한다.
  3. 집계 방식이 합계인지 확인한다.
  4. 표시 형식을 클릭한다.
  5. 숫자를 선택하고 1000 단위 구분 기호 사용을 켠다.
  6. 소수 자릿수는 0으로 설정한다.

금액은 보통 소수점 없이 읽습니다. 그래서 소수 자릿수를 0으로 두면 매출표처럼 깔끔하게 보입니다.

4단계. 결과 읽기

이 단계는 숫자를 문장으로 바꾸는 연습입니다. 피벗테이블을 만드는 것에서 끝내지 말고, 그 표가 무엇을 말하는지 읽어야 분석이 됩니다.

실제 데이터 기준 결과는 아래와 같습니다.

지점 디저트 음료 커피 총합계
강남점 11,604,000 15,281,500 26,539,500 53,425,000
홍대점 7,732,500 10,268,500 19,066,500 37,067,500
총합계 19,336,500 25,550,000 45,606,000 90,492,500

여기서 뭐가 보이시나요? 먼저 각 지점의 가로줄을 봅니다. 강남점도 홍대점도 커피 숫자가 가장 큽니다. 그다음 오른쪽의 총합계를 봅니다. 전체 매출은 강남점이 53,425,000원으로 홍대점보다 큽니다.

해석 문장은 이렇게 써 볼 수 있습니다.

강남점과 홍대점 모두 커피 매출이 가장 크다. 전체 매출은 강남점이 53,425,000원으로 홍대점을 앞선다.

좋은 해석 문장은 기준을 함께 말합니다. “강남점이 좋다”가 아니라 “전체 매출 기준으로 강남점이 53,425,000원이다”처럼 말해야 듣는 사람이 같은 표를 떠올릴 수 있습니다.

따라하기 2. 요일×시간대 매출 피벗 만들기

질문: 요일과 시간대에 따라 매출이 어떻게 달라지는가?

이번에는 지점과 카테고리 대신 요일과 시간을 봅니다. 카페 매출은 하루 종일 고르게 나오지 않습니다. 출근 전, 점심시간, 퇴근 후처럼 사람이 몰리는 시간이 다르기 때문입니다. 피벗테이블로 그 흐름을 확인해 봅니다.

1단계. 새 피벗테이블 만들기

이 단계는 앞에서 만든 지점×카테고리 표와 다른 질문을 따로 보기 위해 필요합니다. 질문이 달라지면 표의 모양도 달라지므로, 새 피벗테이블을 하나 더 만드는 편이 안전합니다.

  1. 다시 sales 시트로 이동한다.
  2. 데이터 안의 아무 셀이나 클릭한다.
  3. 삽입 > 피벗 테이블을 클릭한다.
  4. 위치는 새 워크시트를 선택한다.
  5. 확인을 클릭한다.

원본 데이터 안의 아무 셀이나 클릭하는 이유는 엑셀이 분석할 표 범위를 자동으로 잡게 하려는 것입니다. 새 워크시트를 선택하면 두 번째 결과표를 독립적으로 볼 수 있습니다.

2단계. 필드 배치

이 단계는 “요일별, 시간대별 매출”이라는 질문을 교차표로 만드는 과정입니다. 교차표는 두 기준이 만나는 칸에 값을 보여 주는 표입니다. 예를 들어 일요일과 13시가 만나는 칸에는 일요일 13시의 매출 합계가 들어갑니다.

필드 놓을 영역
weekday 행
hour 열
amount 값

값 영역은 합계: amount로 둡니다. hour 값은 8부터 21까지 보입니다. 여기서는 각 숫자를 8시, 9시, 10시처럼 해석합니다.

행에는 요일이 세로로 놓이고, 열에는 시간이 가로로 놓입니다. 값에는 매출액이 들어가므로, 각 칸은 “해당 요일의 해당 시간대에 발생한 매출 합계”가 됩니다.

3단계. 숫자 표시 형식 정리

이 단계는 큰 금액을 빠르게 비교하기 위해 필요합니다. 숫자 모양이 정리되어 있어야 어느 시간이 큰지 눈으로 훑기 쉽습니다.

따라하기 1과 같은 방식으로 매출 숫자에 1000 단위 구분 기호를 적용합니다.

표를 읽을 때는 정확한 금액 하나만 보는 것이 아니라, 큰 숫자가 어느 쪽에 몰려 있는지도 함께 봅니다. 쉼표가 들어가면 큰 값과 작은 값을 구분하기가 훨씬 편해집니다.

4단계. 교차표 읽기

이 단계는 많은 숫자 속에서 패턴을 찾는 연습입니다. 교차표는 한 칸씩 외우는 표가 아니라 흐름을 읽는 표입니다. 지도에서 길을 찾듯이 큰 숫자가 모여 있는 방향을 먼저 보세요.

교차표는 한 칸씩 외우는 표가 아니라 흐름을 읽는 표입니다. 아래 순서로 봅니다.

  1. 숫자가 큰 시간대가 어느 열에 몰리는지 본다.
  2. 특정 요일에서 유난히 큰 값이 있는지 본다.
  3. 총합계를 보고 전체 흐름을 확인한다.

실제 데이터에서는 13시 매출이 가장 큽니다. 요일과 시간대를 함께 보면 일요일 13시가 2,260,500원으로 가장 큰 조합입니다.

요일 값은 원본 데이터에 영어 약어로 들어 있습니다. 엑셀이 틀린 것이 아니라, 데이터가 이렇게 기록되어 있는 것입니다.

값 의미
Mon 월요일
Tue 화요일
Wed 수요일
Thu 목요일
Fri 금요일
Sat 토요일
Sun 일요일

영어 약어가 보이면 위 표처럼 읽으면 됩니다. 예를 들어 Sun은 일요일이고, Mon은 월요일입니다.

직접 해보기. 가장 잘 팔리는 시간대 찾기

질문: 매출액 합계 기준으로 가장 잘 팔리는 시간대는 언제인가?

이 실습에서는 선생님이 모든 값을 먼저 알려 주지 않습니다. 대신 여러분이 피벗테이블을 하나 더 만들어서 답을 찾아봅니다. 중요한 점은 “가장 잘 팔리는”의 기준을 분명히 하는 것입니다. 여기서는 매출액 합계 기준으로 찾습니다.

조건

새 피벗테이블을 만들어 아래처럼 구성합니다.

필드 놓을 영역
hour 행
amount 값
qty 값
order_id 값

설정은 아래처럼 맞춥니다.

  • amount: 합계
  • qty: 합계
  • order_id: 개수

amount 합계를 기준으로 내림차순 정렬합니다.

이렇게 배치하면 시간대별로 매출, 판매수량, 주문건수를 한 번에 비교할 수 있습니다. amount는 돈이므로 합계로 더하고, qty도 판매된 개수이므로 합계로 더합니다. order_id는 주문번호라서 더하는 숫자가 아니라, 주문이 몇 번 있었는지 개수로 세어야 합니다.

내림차순 정렬은 큰 값이 위로 오게 하는 기능입니다. 매출액 합계를 기준으로 내림차순 정렬하면, 맨 위에 오는 시간대가 매출 기준 1위가 됩니다.

제출할 답

아래 형식으로 한 문장을 작성합니다.

매출액 합계 기준 가장 잘 팔리는 시간대는 __시이며, 매출은 ______원, 판매수량은 ______개, 주문건수는 ______건이다.

빈칸에는 피벗테이블에서 직접 확인한 값을 넣으세요. 답을 쓸 때는 시간, 매출, 판매수량, 주문건수를 모두 함께 적습니다. 그래야 “어떤 기준으로 1위를 골랐는지”가 분명해집니다.

추가 질문

시간이 남으면 아래도 확인해 보세요.

  1. 판매수량 기준 1위 시간대도 같은가?
  2. 주문건수 기준 1위 시간대도 같은가?
  3. 요일×시간대를 함께 볼 때 가장 큰 조합은 무엇인가?

세 질문의 답이 모두 같지 않을 수도 있습니다. 매출이 큰 시간대, 판매수량이 많은 시간대, 주문이 자주 들어온 시간대는 서로 다를 수 있습니다. 비싼 메뉴가 적게 팔려도 매출은 클 수 있고, 저렴한 메뉴가 많이 팔리면 수량은 클 수 있습니다.

자주 생기는 문제

값이 합계가 아니라 개수로 나오는 경우

값 영역의 필드 이름 옆 화살표를 누르고 값 필드 설정으로 들어갑니다. 합계, 개수, 평균 중 원하는 계산 방식을 선택합니다.

엑셀이 값 영역에 들어온 필드를 보고 자동으로 계산 방식을 고를 때가 있습니다. 숫자처럼 보이면 보통 합계를 내지만, 주문번호처럼 식별자에 가까운 값은 개수로 세는 편이 자연스럽습니다. 원하는 답이 매출이면 합계, 주문건수이면 개수를 고르세요.

요일 순서가 월, 화, 수 순서가 아닌 경우

이번 차시에서는 정렬 순서보다 값의 크기를 읽는 것이 중요합니다. 순서가 달라도 해석에는 지장이 없습니다. 날짜와 요일 정렬은 이후 차시에서 더 다룹니다.

요일이 월요일부터 차례대로 보이지 않아도 당황하지 마세요. 지금은 “어느 요일이 더 큰가”, “어느 시간대가 더 큰가”를 읽는 연습을 하고 있습니다. 순서를 예쁘게 맞추는 작업은 나중에 해도 됩니다.

피벗테이블 필드 목록이 보이지 않는 경우

피벗테이블 안의 아무 셀이나 클릭합니다. 그래도 보이지 않으면 상단의 피벗 테이블 분석 메뉴에서 필드 목록을 켭니다.

필드 목록은 피벗테이블을 클릭했을 때 나타나는 조작 패널입니다. 표 바깥의 셀을 클릭하면 엑셀이 “지금은 피벗테이블을 편집하지 않는구나”라고 보고 필드 목록을 숨길 수 있습니다. 먼저 피벗테이블 안쪽을 다시 클릭해 보세요.

S03. 피벗테이블로 질문에 답하기 강사대본

수업 전 확인

  • 파일: 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 엑셀 파일
  • 시트: sales
  • 데이터 규모: 10,424행 × 12열
  • 주요 컬럼: store, category, weekday, hour, amount, qty, order_id
  • 검증된 미션 정답: 매출액 합계 기준 13시, 11,626,000원, 판매수량 2,220개, 주문건수 1,348건

0-5분. 도입

강사 멘트:

"지난 시간에는 데이터가 표처럼 가지런해야 분석을 시작할 수 있다는 것을 봤습니다. 오늘은 그 표를 가지고 질문에 답해 봅니다. 데이터분석은 표를 오래 바라보는 일이 아닙니다. 내가 궁금한 질문을 엑셀이 계산할 수 있는 표 모양으로 바꾸는 일입니다."

"카페 매출 파일에는 주문 한 건 한 건이 줄마다 쌓여 있습니다. 한 줄씩 보면 자세한 주문 내역은 알 수 있지만, 전체 흐름은 잘 보이지 않습니다. 그래서 오늘은 두 가지를 물어봅니다. 지점별로 어떤 카테고리가 매출을 많이 만드는지, 그리고 어떤 시간대에 매출이 커지는지 보겠습니다."

"이 질문을 피벗테이블로 풀어 봅니다. 피벗테이블은 긴 주문 목록을 접어서 요약표로 만들어 주는 엑셀 기능입니다. 영수증을 한 장씩 다 읽는 대신, 지점별 합계표나 시간대별 합계표로 바꿔 보는 도구라고 생각하시면 됩니다."

시연 포인트:

  • sales 시트를 열고 행이 주문 한 건 단위로 쌓여 있음을 보여준다.
  • amount는 주문금액, qty는 판매수량, order_id는 주문번호임을 짚는다.
  • 원본 표를 직접 수정하지 않고 피벗테이블을 새 시트에 만든다고 설명한다.

5-12분. 피벗테이블 개념 설명

강사 멘트:

"피벗테이블은 긴 영수증 묶음을 요약표로 접는 도구입니다. 원본 표에는 주문 한 건 한 건이 모두 들어 있습니다. 그런데 우리가 알고 싶은 것은 보통 '전체적으로 어디가 많이 팔렸나', '어떤 시간대가 바빴나' 같은 흐름입니다."

"피벗테이블 필드에는 행, 열, 값, 필터가 있습니다. 행과 열은 데이터를 나누는 기준입니다. 값은 실제로 계산할 숫자입니다. 필터는 그중 일부만 보고 싶을 때 쓰는 조건입니다."

"예를 들어 지점별 카테고리별 매출을 보고 싶다고 해 보겠습니다. 지점은 세로 방향의 기준이므로 행에 둡니다. 카테고리는 가로 방향으로 비교할 기준이므로 열에 둡니다. 매출액은 더해야 하는 숫자이므로 값에 둡니다. 이렇게 질문을 행, 열, 값으로 바꿔 주는 것이 피벗테이블을 쓰는 방법입니다."

"여기서 뭐가 중요할까요? 값 영역에 무엇을 넣었는지에 따라 답의 뜻이 달라집니다. amount를 합계로 보면 매출 기준입니다. qty를 합계로 보면 판매수량 기준입니다. order_id를 개수로 보면 주문건수 기준입니다. 같은 데이터라도 기준을 바꾸면 1위가 달라질 수 있습니다."

판서 또는 화면 강조:

  • 질문: 지점별 카테고리별 매출은?
  • 행: store
  • 열: category
  • 값: amount 합계

주의해서 말할 점:

  • 값 영역에 무엇을 넣었는지가 답의 기준이다.
  • amount 합계는 매출 기준이고, qty 합계는 수량 기준이다.
  • order_id 개수는 주문건수 기준이다.

12-25분. 따라하기 1: 지점×카테고리 매출

시연 절차:

  1. sales 시트에서 데이터 안의 아무 셀이나 클릭한다.
  2. 삽입 > 피벗 테이블을 클릭한다.
  3. 새 워크시트를 선택하고 확인한다.
  4. store를 행에 넣는다.
  5. category를 열에 넣는다.
  6. amount를 값에 넣는다.
  7. 값 필드 설정이 합계인지 확인한다.
  8. 숫자 표시 형식을 1000 단위 구분, 소수 0자리로 바꾼다.

강사 멘트:

"이 표는 원본 데이터를 지운 것이 아닙니다. 같은 데이터를 다른 방식으로 접어서 보여 주는 요약표입니다. 원본은 주문 한 건씩 길게 쌓여 있었고, 지금 만든 피벗테이블은 지점과 카테고리 기준으로 합계를 낸 표입니다."

"행에는 강남점과 홍대점이 생겼습니다. 열에는 디저트, 음료, 커피가 자리를 잡았습니다. 각 칸은 두 조건이 만나는 자리입니다. 예를 들어 강남점 행과 커피 열이 만나는 칸은, 강남점에서 팔린 커피 주문금액을 모두 더한 값입니다."

"강남점의 커피 매출은 26,539,500원입니다. 홍대점의 커피 매출은 19,066,500원입니다. 이렇게 한 칸은 두 조건이 동시에 맞는 주문들을 모아서 더한 결과입니다. 사전에서 단어를 찾듯이, 행에서 지점을 찾고 열에서 카테고리를 찾으면 그 교차 지점에 답이 있습니다."

확인 질문:

"강남점에서 가장 매출이 큰 카테고리는 무엇인가요?"

기대 답:

"커피입니다."

시연 포인트:

  • "총합계"가 행과 열 양쪽에 생기는 것을 설명한다.
  • 강남점 총합계 53,425,000원, 홍대점 총합계 37,067,500원을 보여준다.
  • 두 지점 모두 커피 비중이 가장 크다는 해석을 말하게 한다.

25-38분. 따라하기 2: 요일×시간대 매출

시연 절차:

  1. sales 시트로 돌아간다.
  2. 새 피벗테이블을 만든다.
  3. weekday를 행에 넣는다.
  4. hour를 열에 넣는다.
  5. amount를 값에 넣는다.
  6. 숫자 표시 형식을 정리한다.

강사 멘트:

"이제 같은 피벗테이블 기능으로 시간 패턴을 보겠습니다. 행에는 요일을 놓고, 열에는 시간대를 놓습니다. 값은 그대로 매출액 합계입니다. 이렇게 하면 요일과 시간대가 만나는 칸마다 매출 합계가 계산됩니다."

"교차표는 모든 칸을 하나씩 읽으려고 하면 어렵습니다. 시험지 채점하듯이 한 칸씩 보는 표가 아니라, 먼저 큰 숫자가 어디에 몰려 있는지 보는 표입니다. 이 데이터는 12시, 13시, 14시가 크게 보이고, 그중 13시가 가장 큽니다."

"요일과 시간대를 동시에 보면 일요일 13시가 가장 큰 칸입니다. 여기서 문장을 정확하게 말하는 연습을 해 보겠습니다. 이 말은 '일요일 13시에 주문이 가장 많았다'가 아닙니다. 지금 값에 넣은 것은 amount이고, 계산 방식은 합계입니다. 그래서 정확히는 '일요일 13시의 매출액 합계가 가장 컸다'입니다."

"weekday 값은 영어 약어로 보입니다. Mon은 월요일, Tue는 화요일, Sun은 일요일입니다. 요일 순서가 우리가 익숙한 월화수목금토일 순서로 보이지 않을 수 있습니다. 그래도 오늘은 정렬 순서보다, 값 영역에 무엇을 넣었고 그 값을 어떻게 읽는지가 더 중요합니다."

시연 포인트:

  • hour가 8부터 21까지 보이는 것을 확인한다.
  • weekday는 영어 약어이므로 Mon은 월요일, Sun은 일요일이라고 짚는다.
  • 요일 순서가 월화수목금토일로 보이지 않아도 이번 차시의 중요한 연습은 값 읽기라고 말한다.

38-47분. 직접 해보기: 가장 잘 팔리는 시간대

미션 안내 멘트:

"이제 여러분이 직접 하나의 질문에 답해 보겠습니다. 질문은 '가장 잘 팔리는 시간대는 언제인가'입니다. 여기서 바로 피벗테이블을 만들기 전에 기준을 먼저 정해야 합니다. 잘 팔린다는 말은 매출이 크다는 뜻일 수도 있고, 수량이 많다는 뜻일 수도 있고, 주문건수가 많다는 뜻일 수도 있습니다."

"이번 미션에서는 매출액 합계 기준으로 보겠습니다. 그러니까 hour별로 나누고, amount를 더해서 가장 큰 시간대를 찾습니다. 키를 잴 때 센티미터라는 기준이 있어야 비교할 수 있듯이, 데이터도 기준을 정해야 비교할 수 있습니다."

수강생 작업:

  1. 새 피벗테이블을 만든다.
  2. hour를 행에 넣는다.
  3. amount를 값에 넣고 합계로 둔다.
  4. qty를 값에 넣고 합계로 둔다.
  5. order_id를 값에 넣고 개수로 바꾼다.
  6. amount 합계 기준으로 내림차순 정렬한다.
  7. 한 문장으로 답을 쓴다.

정답 안내:

"매출액 합계 기준 가장 잘 팔리는 시간대는 13시입니다. 매출은 11,626,000원, 판매수량은 2,220개, 주문건수는 1,348건입니다. 이 데이터에서는 판매수량 기준과 주문건수 기준으로 봐도 13시가 1위입니다."

"답을 말할 때는 꼭 기준을 붙여 주세요. 그냥 '13시가 제일 잘 팔립니다'라고 말하면 무엇을 기준으로 본 것인지 흐려집니다. '매출액 합계 기준으로 13시가 1위입니다'라고 말하면, 어떤 숫자를 보고 판단했는지 분명해집니다."

47-50분. 마무리

강사 멘트:

"오늘 배운 것은 피벗테이블 버튼을 누르는 순서만이 아닙니다. 질문을 표의 배치로 바꾸는 방법입니다. 지점별 카테고리별 매출이라는 질문은 행에 지점, 열에 카테고리, 값에 매출 합계로 바뀝니다. 가장 잘 팔리는 시간대라는 질문은 행에 시간대, 값에 매출 합계로 바뀝니다."

"피벗테이블을 만들 때마다 스스로 이렇게 물어보면 됩니다. 무엇별로 나눌까요? 무엇을 계산할까요? 합계로 볼까요, 개수로 볼까요? 이 세 가지를 정하면 긴 주문 목록도 읽을 수 있는 요약표가 됩니다."

"다음 차시부터는 여러 표를 연결하거나 계산 컬럼을 만드는 방식으로 질문의 범위를 넓혀 갑니다."

완료 판단 체크포인트:

수강생이 hour별 피벗테이블을 만들고 "매출액 합계 기준 13시가 1위이며 매출은 11,626,000원"이라고 말하면 이번 차시를 완료한 것으로 본다.

예상 질문과 답변

Q1. 값이 합계가 아니라 개수로 나옵니다.

A. 값 영역의 필드 옆 화살표를 누르고 값 필드 설정으로 들어가 합계를 선택해 보세요. 엑셀이 어떤 필드를 숫자로 인식하지 못하거나 빈 값이 섞여 있으면, 자동으로 합계가 아니라 개수로 잡을 수 있습니다. 이럴 때는 값 필드 설정에서 직접 계산 방식을 골라 주면 됩니다.

Q2. order_id는 왜 합계가 아니라 개수로 보나요?

A. order_id는 주문번호입니다. 주문번호는 더해도 의미가 없습니다. 예를 들어 영수증 번호 100번과 101번을 더해서 201번이라고 말하지 않는 것과 같습니다. 우리가 알고 싶은 것은 주문번호의 총합이 아니라 주문이 몇 건 있었는지이므로 개수로 보는 것이 맞습니다.

Q3. 매출 1위와 수량 1위가 다를 수도 있나요?

A. 다를 수 있습니다. 비싼 상품이 적게 팔린 시간대는 판매수량은 많지 않아도 매출이 높게 나올 수 있습니다. 반대로 싼 상품이 많이 팔린 시간대는 판매수량은 높지만 매출은 1위가 아닐 수 있습니다. 그래서 "잘 팔린다"는 말을 쓸 때는 매출 기준인지, 수량 기준인지, 주문건수 기준인지 함께 밝혀야 합니다.

Q4. 요일 순서가 이상합니다.

A. 원본의 weekday 값이 영어 약어라 엑셀이 자동으로 월화수목금토일 순서로 정렬하지 않을 수 있습니다. 이번 차시에서는 순서보다 교차표를 읽는 법이 중요합니다. 어느 열과 어느 행이 만나는지 보고, 값이 큰 칸을 해석하는 연습에 집중하면 됩니다. 날짜와 요일 정렬은 이후 날짜 차시에서 더 다룹니다.

Q5. 원본 데이터가 바뀌면 피벗테이블도 자동으로 바뀌나요?

A. 원본을 수정한 뒤 피벗테이블을 오른쪽 클릭하고 새로 고침을 눌러야 반영됩니다. 피벗테이블은 원본을 참고해서 만든 요약표이지만, 원본이 바뀌는 순간마다 항상 즉시 다시 계산되지는 않습니다. 원본을 바꾼 뒤에는 새로 고침을 한 번 눌러 준다고 기억해 두면 됩니다.

강의 운영 메모

  • 수강생이 필드 배치를 외우려 하면 질문으로 되돌린다. "무엇별로 나눌까요?", "무엇을 계산할까요?"라고 묻는다.
  • 첫 피벗에서는 필드가 어디에 들어가는지 천천히 보여준다.
  • 두 번째 피벗부터는 수강생이 먼저 예측하게 한다.
  • 미션에서는 정답 숫자보다 기준을 말하는 습관을 더 중요하게 본다.

S03. 모범답안

미션 질문에 대한 답

미션 질문은 "매출액 합계 기준으로 가장 잘 팔리는 시간대는 언제인가?"입니다.

이 질문에서 기준은 분명히 매출액 합계입니다. 판매수량이 많은 시간이나 주문건수가 많은 시간이 아니라, 각 시간대의 주문금액을 모두 더했을 때 가장 큰 시간을 찾는 문제입니다. 키를 잴 때 자를 하나 정하듯이, 여기서는 amount의 합계를 기준자로 삼는다고 보면 됩니다.

정답:

매출액 합계 기준 가장 잘 팔리는 시간대는 13시이며, 매출은 11,626,000원, 판매수량은 2,220개, 주문건수는 1,348건이다.

이 데이터에서는 판매수량 기준과 주문건수 기준으로 봐도 13시가 1위입니다. 그래서 "매출은 13시가 큰데 주문은 다른 시간대가 많았다"는 상황이 아니라, 매출액·판매수량·주문건수가 모두 13시에 가장 높게 모인 결과라고 해석할 수 있습니다.

해석:

여기서 뭐가 보이시나요? 점심 직후인 13시가 매출액, 판매수량, 주문건수 모두에서 가장 큽니다. 12시와 14시도 높은 편이지만, 매출액 합계로 비교하면 13시가 가장 앞섭니다.

따라서 이 데이터만 놓고 보면 13시를 가장 잘 팔리는 시간대로 판단할 수 있습니다. "잘 팔린다"는 말을 감으로 말하지 않고, 매출액 합계라는 기준으로 확인했다는 점이 중요합니다.

따라하기 1 검증 결과: 지점별·카테고리별 매출

해석:

강남점과 홍대점 모두 커피 매출이 가장 큽니다. 표를 가로로 읽어 보면 각 지점 안에서 어떤 카테고리가 큰지 볼 수 있고, 세로로 읽어 보면 같은 카테고리에서 어느 지점이 더 큰지 볼 수 있습니다.

전체 매출은 강남점이 53,425,000원으로 홍대점보다 큽니다. 즉, 이 표에서는 "강남점이 전체 매출을 더 많이 만들고 있고, 두 지점 모두 커피가 가장 큰 매출 카테고리다"라고 말할 수 있습니다.

따라하기 2 검증 결과: 요일별·시간대별 매출

요일×시간대를 함께 보았을 때 매출액 합계가 가장 큰 조합은 Sun 13시이며, 매출액은 2,260,500원입니다.

Sun은 일요일을 뜻합니다. 그러니까 "전체 시간대만 보면 13시가 가장 크고, 요일까지 함께 나누어 보면 일요일 13시가 가장 큰 칸이다"라고 이해하면 됩니다. 사전에서 단어를 찾듯이, Sun이라는 요일과 13시라는 시간대를 교차해서 해당 칸의 금액을 읽은 것입니다.

검증용 pandas 코드

아래 코드는 엑셀 피벗테이블에서 한 계산을 pandas로 다시 확인한 것입니다. pandas는 파이썬에서 표 데이터를 다룰 때 많이 쓰는 도구입니다. 수업에서는 엑셀 피벗테이블로 답을 찾지만, 이렇게 코드로도 같은 결과가 나오는지 검산할 수 있습니다.

학생 입장에서는 이 코드를 외울 필요는 없습니다. 여기서는 "엑셀에서 만든 피벗 결과가 실제 원본 데이터와 맞는지 한 번 더 확인했다"는 뜻으로 보면 됩니다.

from pathlib import Path

import pandas as pd

data_path = Path("/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.csv")
df = pd.read_csv(data_path)

assert df.shape == (10424, 12)
assert df["amount"].isna().sum() == 0
assert df["qty"].isna().sum() == 0

store_category = pd.pivot_table(
    df,
    index="store",
    columns="category",
    values="amount",
    aggfunc="sum",
    margins=True,
    margins_name="총합계",
).fillna(0).astype(int)

hour_summary = (
    df.groupby("hour")
    .agg(
        주문건수=("order_id", "count"),
        판매수량=("qty", "sum"),
        매출액=("amount", "sum"),
    )
    .reset_index()
    .sort_values("매출액", ascending=False)
)

weekday_order = ["Mon", "Tue", "Wed", "Thu", "Fri", "Sat", "Sun"]
weekday_hour = (
    pd.pivot_table(
        df,
        index="weekday",
        columns="hour",
        values="amount",
        aggfunc="sum",
    )
    .reindex(weekday_order)
    .fillna(0)
    .astype(int)
)

top_hour = hour_summary.iloc[0]
top_weekday_hour_raw = weekday_hour.stack().sort_values(ascending=False).index[0]
top_weekday_hour = (top_weekday_hour_raw[0], int(top_weekday_hour_raw[1]))
top_weekday_hour_amount = int(weekday_hour.stack().sort_values(ascending=False).iloc[0])

print("데이터 행열:", df.shape)
print("전체 매출:", int(df["amount"].sum()))
print("전체 판매수량:", int(df["qty"].sum()))
print()
print("[지점 x 카테고리 매출]")
print(store_category)
print()
print("[시간대별 상위 5개]")
print(hour_summary.head(5).to_string(index=False))
print()
print(
    "매출 1위 시간대:",
    f"{int(top_hour['hour'])}시",
    f"매출 {int(top_hour['매출액']):,}원",
    f"판매수량 {int(top_hour['판매수량']):,}개",
    f"주문건수 {int(top_hour['주문건수']):,}건",
)
print(
    "요일 x 시간대 최고 조합:",
    top_weekday_hour,
    f"{top_weekday_hour_amount:,}원",
)
