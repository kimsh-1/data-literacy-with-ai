S10. Colab과 파이썬 기초

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S01 데이터 분석 작업 흐름과 환경 세팅
  • 난이도: 3
  • 시간: 90분
  • 사용 데이터: 없음. 문법 예제만 사용한다.

이번 차시는 엑셀에서 파이썬으로 넘어가는 첫 수업이다. 코딩 실력을 키우자는 것이 아니라, 다음 차시부터 쓸 pandas 문법이 낯설지 않을 만큼 기본 언어 감각을 익히는 데 목표가 있다.

수강생은 Colab 노트북을 열어 셀을 실행하고, 변수에 값을 담고, 리스트와 딕셔너리로 여러 값을 표현하고, 조건문과 반복문으로 행을 골라낸다. 끝에서는 "다음 차시 pandas의 필터는 오늘 배운 조건문을 열 전체에 한 번에 적용하는 것"이라는 연결고리를 잡아 준다.

학습 목표

  1. Colab 노트북을 만들고 코드 셀을 실행할 수 있다.
  2. 변수, 자료형, 리스트, 딕셔너리의 역할을 엑셀 표와 비교해 설명할 수 있다.
  3. if 조건문으로 원하는 경우만 골라낼 수 있다.
  4. for 반복문으로 여러 값을 하나씩 처리하고 합계와 평균을 계산할 수 있다.
  5. 함수는 "자주 쓰는 절차에 이름을 붙이는 것"이라는 수준으로 이해한다.

핵심 개념

1. Colab은 브라우저에서 쓰는 파이썬 작업장이다

Colab은 구글 계정으로 사용하는 온라인 노트북이다. 엑셀 파일이 셀 격자로 되어 있다면, Colab 노트북은 코드 셀과 텍스트 셀로 되어 있다.

엑셀에서의 감각 Colab에서의 감각
셀에 수식 입력 코드 셀에 파이썬 코드 입력
Enter로 셀 이동 Shift + Enter로 코드 실행 후 다음 셀 이동
워크시트 저장 구글 드라이브에 노트북 저장
수식 결과가 셀에 표시 코드 실행 결과가 셀 아래에 표시
잘못된 수식은 오류 표시 잘못된 코드는 빨간 오류 메시지 표시

Colab에서 중요한 점은 코드 셀을 위에서 아래로 실행한다는 것이다. 위 셀에서 만든 변수는 아래 셀에서 다시 쓸 수 있다. 반대로 위 셀을 실행하지 않았거나 런타임을 재시작했다면, 아래 셀에서 그 변수를 찾지 못한다.

2. 변수는 값에 붙이는 이름이다

엑셀에서는 B2 같은 셀 주소가 값을 가리킨다. 파이썬에서는 사람이 읽을 수 있는 이름을 붙인다.

store_name = "강남점"
sales = 170000

store_name은 "강남점"이라는 글자 값을 가리킨다. sales는 170000이라는 숫자 값을 가리킨다.

변수를 만들 때의 등호 =는 수학의 "같다"가 아니라 "오른쪽 값을 왼쪽 이름에 담는다"에 가깝다. 조건에서 같은지 비교할 때는 ==를 쓴다.

sales = 170000      # sales에 170000을 담는다
sales == 170000     # sales가 170000인지 비교한다

3. 자료형은 값의 종류다

파이썬은 값의 종류를 구분한다. 엑셀에서도 숫자처럼 보이는 글자와 진짜 숫자가 다르게 동작하듯, 파이썬도 자료형이 맞아야 계산이 된다.

자료형 예시 엑셀과 비교
정수 int 170000 소수점 없는 숫자
실수 float 0.15 소수점 있는 숫자
문자열 str "강남점" 텍스트
불리언 bool True, False 조건 결과, 체크 여부

아래 코드는 값의 종류를 확인한다.

type(170000)
type("강남점")
type(True)

입문 단계에서 자료형 이름을 다 외울 필요는 없다. 다만 계산이 안 될 때 "숫자인 줄 알았는데 글자였나"부터 확인하는 습관은 꼭 들여 두자.

4. 리스트는 한 열처럼 여러 값을 담는다

리스트는 여러 값을 순서대로 담는 상자다. 엑셀에서 한 열에 매출이 아래로 쌓여 있는 모습과 비슷하다.

sales_list = [120000, 98000, 135000, 170000, 155000]

리스트에서 값의 개수는 len()으로 센다.

len(sales_list)

합계는 sum()으로도 구하지만, 오늘은 반복문을 익히려고 일부러 직접 더해 본다. 손으로 한 번 더해 보면 다음 차시 pandas의 집계가 무엇을 대신 해주는지 감이 잡힌다.

5. 딕셔너리는 한 행처럼 이름 붙은 값을 담는다

딕셔너리는 값마다 이름표가 붙어 있는 구조다. 엑셀의 한 행을 떠올리면 쉽다.

order = {
    "요일": "월",
    "매출": 120000,
    "지점": "강남점",
}

딕셔너리에서 특정 값을 꺼낼 때는 열 이름처럼 키를 쓴다.

order["요일"]
order["매출"]

리스트 안에 딕셔너리를 여러 개 넣으면 작은 표처럼 다룰 수 있다.

daily_sales = [
    {"요일": "월", "매출": 120000},
    {"요일": "화", "매출": 98000},
    {"요일": "수", "매출": 135000},
]

이 구조가 다음 차시의 pandas DataFrame으로 이어진다. DataFrame은 이런 행과 열 구조를 훨씬 편하게 다루도록 만든 도구다.

6. 조건문은 필터의 출발점이다

엑셀 필터에서 요일 = 금 조건을 걸면 금요일 행만 보인다. 파이썬에서는 if로 조건을 쓴다.

day = "금"

if day == "금":
    print("금요일입니다.")

if 뒤에 조건을 쓰고, 조건 끝에는 콜론 :을 붙인다. 그 아래 실행할 코드는 들여쓰기한다. 파이썬에서 들여쓰기는 문법이다. 보기 좋으라고 넣는 장식이 아니라 어떤 코드가 조건 안에 들어가는지 알려주는 표시다.

7. 반복문은 같은 작업을 여러 번 하는 방법이다

엑셀에서는 수식을 한 셀에 만든 뒤 아래로 채운다. 파이썬에서는 for 반복문이 비슷한 역할을 한다.

for amount in sales_list:
    print(amount)

sales_list 안의 값을 하나씩 꺼내 amount라는 이름으로 부르는 코드다. 첫 번째 반복에서 amount는 120000, 두 번째 반복에서는 98000이 된다.

합계를 직접 구하려면 누적할 변수를 먼저 만든다.

total = 0

for amount in sales_list:
    total = total + amount

total = total + amount는 처음 보면 어색하다. 오른쪽의 기존 total에 이번 amount를 더한 다음, 그 값을 다시 total에 담으라는 뜻이다.

8. 함수는 자주 쓰는 절차에 이름을 붙이는 것이다

이번 차시에서 함수는 깊게 다루지 않는다. 다만 다음 차시부터 read_csv(), head(), mean() 같은 함수를 계속 만나게 되니 개념만 미리 맛본다.

def average(numbers):
    total = 0
    for number in numbers:
        total = total + number
    return total / len(numbers)

함수는 여러 줄짜리 절차에 이름을 붙여 두고 다시 꺼내 쓰는 방법이다. 엑셀의 SUM()이나 AVERAGE()처럼 이름만 부르면 정해진 계산을 해준다.

엑셀과 연결해서 설명할 지점

  • 변수는 셀 주소보다 읽기 쉬운 이름이다.
  • 리스트는 값이 세로로 쌓인 한 열과 비슷하다.
  • 딕셔너리는 열 이름이 붙은 한 행과 비슷하다.
  • 반복문은 아래로 채우기처럼 같은 계산을 여러 값에 적용한다.
  • 조건문은 엑셀 필터의 조건식과 비슷하다.
  • pandas 필터는 if 조건문을 표 전체 열에 한 번에 적용하는 방식이다.

수업 후 수강생이 말할 수 있어야 하는 문장

출력에서 확인할 것:

따라하기 3: 리스트로 매출 여러 개 담기

새 코드 셀에 아래 코드를 입력한다.

sales_list = [120000, 98000, 135000, 170000, 155000]

print(sales_list)
print(len(sales_list))

sales_list는 월요일부터 금요일까지 매출 5개를 담은 리스트라고 보면 된다. len()은 값이 몇 개인지 세는 함수다.

따라하기 4: 반복문으로 매출 합계 구하기

이번에는 sum() 함수 대신 반복문으로 하나씩 더해 본다. 새 코드 셀에 아래 코드를 입력한다.

total_sales = 0

for amount in sales_list:
    total_sales = total_sales + amount

print(total_sales)

기대 결과:

678000

코드를 읽는 순서는 다음과 같다.

  1. total_sales = 0으로 합계를 담을 변수를 만든다.
  2. sales_list에서 값을 하나씩 꺼내 amount라고 부른다.
  3. 꺼낸 값을 total_sales에 계속 더한다.
  4. 반복이 끝나면 total_sales에는 전체 합계가 남는다.

주의할 점:

  • for 줄 끝에는 콜론 :을 붙인다.
  • 반복할 코드는 반드시 들여쓰기한다.
  • 들여쓰기는 보통 스페이스 4칸이다.

따라하기 5: 평균 구하기

새 코드 셀에 아래 코드를 입력한다.

average_sales = total_sales / len(sales_list)

print(average_sales)

기대 결과:

135600.0

엑셀의 AVERAGE 함수가 속으로 하던 일을 손으로 옮겨 쓴 셈이다.

평균 = 합계 / 개수

따라하기 6: 딕셔너리로 한 행 표현하기

새 코드 셀에 아래 코드를 입력한다.

one_day = {
    "요일": "월",
    "매출": 120000,
}

print(one_day)
print(one_day["요일"])
print(one_day["매출"])

딕셔너리는 이름표를 붙여 값을 담는다. 엑셀 표의 한 행을 떠올리면 쉽다.

요일 매출
월 120000

따라하기 7: 조건문으로 금요일만 확인하기

새 코드 셀에 아래 코드를 입력한다.

day = "금"

if day == "금":
    print("금요일 데이터입니다.")

=와 ==를 구분한다.

  • =: 값을 변수에 담는다.
  • ==: 두 값이 같은지 비교한다.

따라하기 8: 리스트와 딕셔너리로 작은 표 만들기

새 코드 셀에 아래 코드를 입력한다.

daily_sales = [
    {"요일": "월", "매출": 120000},
    {"요일": "화", "매출": 98000},
    {"요일": "수", "매출": 135000},
    {"요일": "목", "매출": 170000},
    {"요일": "금", "매출": 155000},
    {"요일": "토", "매출": 210000},
    {"요일": "일", "매출": 190000},
]

for row in daily_sales:
    print(row["요일"], row["매출"])

요일과 매출이 한 줄씩 출력된다. 여기서 row는 반복문이 지금 보고 있는 한 행이다.

직접 해보기: 평일 데이터만 필터링하기

아래 조건을 만족하는 코드를 작성한다.

목표:

  1. daily_sales에서 평일인 월, 화, 수, 목, 금 행만 고른다.
  2. 고른 행을 weekday_rows 리스트에 담는다.
  3. 평일 매출 합계와 평균을 계산한다.
  4. 평일 행수, 평일 합계, 평일 평균을 출력한다.

힌트 코드:

weekday_names = ["월", "화", "수", "목", "금"]
weekday_rows = []
weekday_total = 0

for row in daily_sales:
    if row["요일"] in weekday_names:
        weekday_rows.append(row)
        weekday_total = weekday_total + row["매출"]

weekday_average = weekday_total / len(weekday_rows)

print(weekday_rows)
print(weekday_total)
print(weekday_average)

def average(numbers):
    total = 0
    for number in numbers:
        total = total + number
    return total / len(numbers)

예상 질문과 답변

Q1. 파이썬 문법을 다 외워야 하나요?

아니다. 오늘 외울 것은 많지 않다. 변수는 이름, 리스트는 여러 값, 딕셔너리는 이름표 붙은 값, 조건문은 필터, 반복문은 같은 작업 반복. 이 감각만 잡으면 된다. 정확한 문법은 실습하면서 찾아보면 된다.

Q2. 왜 sum()을 쓰지 않고 반복문으로 직접 더하나요?

실무에서는 sum()을 쓰면 된다. 오늘 직접 더해 보는 건 반복문이 값을 하나씩 어떻게 처리하는지 눈으로 보기 위해서다. 이 감각이 있어야 나중에 pandas가 알아서 해주는 일이 무엇인지 알 수 있다.

Q3. =와 ==가 계속 헷갈립니다.

처음에는 다들 헷갈린다. =는 변수에 값을 넣을 때, ==는 같은지 물어볼 때라고 기억하자. 오류가 나면 이 둘부터 확인한다.

Q4. 들여쓰기는 꼭 해야 하나요?

파이썬에서는 반드시 해야 한다. 들여쓰기가 코드의 소속을 나타내기 때문이다. if나 for 아래에서 실행할 코드는 한 단계 안쪽으로 들여쓴다.

Q5. Colab을 닫으면 변수가 사라지나요?

노트북 파일 자체는 저장되지만, 실행 중이던 변수는 런타임이 꺼지면 사라진다. 다시 열면 위에서부터 필요한 셀을 순서대로 실행해 주어야 한다.

Q6. 오늘 배운 것으로 실제 데이터 분석을 할 수 있나요?

오늘 문법만으로도 작은 리스트는 처리할 수 있다. 다만 실제 CSV는 행과 열이 많으니 다음 차시부터 pandas를 쓴다. 오늘 배운 조건문과 반복문이 그 pandas 문법을 이해하는 바탕이 된다.

해설

daily_sales는 리스트 안에 딕셔너리가 들어 있는 구조다. 리스트 전체는 작은 표이고, 딕셔너리 하나는 표의 한 행이라고 볼 수 있다.

{"요일": "월", "매출": 120000}

이 한 행에서 row["요일"]은 요일 값을 꺼내고, row["매출"]은 매출 값을 꺼낸다.

핵심 조건은 아래 한 줄이다.

if row["요일"] in weekday_names:

이 문장은 현재 행의 요일이 weekday_names 목록 안에 있으면 아래 코드를 실행하라는 뜻이다. 엑셀에서 요일 필터를 열고 월요일부터 금요일까지 체크하는 상황과 같다.

조건을 만족한 행은 두 가지 처리를 한다.

weekday_rows.append(row)
weekday_total = weekday_total + row["매출"]

첫 줄은 평일 행을 별도 리스트에 모은다. 둘째 줄은 평일 매출을 누적한다. 반복이 끝난 뒤 weekday_rows에는 평일 5행만 남고, weekday_total에는 평일 매출 합계가 남는다.

평균은 합계를 행수로 나누어 계산한다.

weekday_average = weekday_total / len(weekday_rows)

반복문 코드:

total_sales = 0

for amount in sales_list:
    total_sales = total_sales + amount

average_sales = total_sales / len(sales_list)

이 코드는 엑셀의 SUM과 AVERAGE를 내부 절차로 풀어 쓴 것이다. 실제 분석에서는 sum()이나 pandas의 .sum(), .mean()을 더 자주 쓰지만, 입문 단계에서는 반복문으로 직접 더해 보는 것이 도움이 된다.

함수 맛보기

함수로 평균 계산 절차에 이름을 붙이면 아래와 같다.

def average(numbers):
    total = 0
    for number in numbers:
        total = total + number
    return total / len(numbers)

print(average(sales_list))

즉, pandas 필터는 오늘 배운 조건문을 열 전체에 적용하는 방법이다.
