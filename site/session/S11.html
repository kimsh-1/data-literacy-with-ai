S11. Pandas 입문: Series, DataFrame, CSV 읽기와 필터

  • 선수 차시: S10 Colab과 파이썬 기초
  • 난이도: 3
  • 시간: 60분
  • 사용 데이터: 02_datasets/cafe_sales/cafe_sales_clean.csv

이번 차시는 파이썬 문법을 벗어나 실제 표 데이터를 다루는 첫 시간이다. 수강생은 pandas로 CSV 파일을 읽고, DataFrame과 Series를 엑셀 시트와 열에 견주어 이해한다. 그다음 head, info, describe로 데이터를 살펴본 뒤, 불리언 필터로 원하는 행만 남기고 금액순으로 정렬한다.

핵심 브리지는 "엑셀 필터 = 불리언 마스크"이다. 엑셀에서는 필터 창에서 강남점을 체크하지만, pandas에서는 df["store"] == "강남점"이라는 True/False 열을 만든 뒤 True인 행만 남긴다. 이 생각을 잡으면 이후 정제, 집계, 시각화 차시에서 나오는 대부분의 pandas 코드가 덜 낯설어진다.

학습 목표

  1. pandas의 DataFrame을 엑셀 시트, Series를 엑셀의 한 열에 비유해 설명할 수 있다.
  2. pd.read_csv()로 UTF-8 CSV 파일을 읽고, 한글 CSV에서 인코딩 문제가 생길 수 있음을 인지한다.
  3. head, info, describe로 데이터의 첫 행, 컬럼, 타입, 기초 통계를 확인할 수 있다.
  4. 불리언 조건식을 만들어 원하는 행만 필터링할 수 있다.
  5. .loc, .iloc의 차이를 라벨 기준 선택과 위치 기준 선택으로 구분할 수 있다.
  6. sort_values로 금액이 큰 주문부터 정렬할 수 있다.

핵심 개념

1. pandas는 표를 다루는 파이썬 도구다

S10에서는 변수, 리스트, 딕셔너리, 조건문, 반복문을 배웠다. pandas는 그 문법을 표 데이터에 맞게 확장한 도구다.

import pandas as pd

여기서 pd는 pandas를 짧게 부르는 별명이다. 엑셀에서 SUM, AVERAGE, FILTER 같은 기능을 쓰듯, pandas에서는 read_csv, head, describe, sort_values 같은 기능을 쓴다.

2. DataFrame은 시트, Series는 열이다

CSV를 읽으면 pandas는 보통 DataFrame을 만든다. DataFrame은 행과 열이 있는 표다. 엑셀의 워크시트 한 장을 떠올리면 된다.

df = pd.read_csv(path, encoding="utf-8-sig")

DataFrame에서 열 하나를 꺼내면 Series가 된다.

amount_series = df["amount"]

엑셀로 비교하면 다음과 같다.

엑셀 감각 pandas 감각
워크시트 전체 DataFrame
열 하나 Series
행 번호 인덱스
열 제목 컬럼명
필터 조건 불리언 마스크

처음에는 DataFrame과 Series 이름보다 "표 전체"와 "열 하나"라는 감각이 더 중요하다.

3. CSV 읽기는 파일 경로와 인코딩을 확인하는 일이다

CSV는 쉼표로 구분된 텍스트 파일이다. 엑셀 파일처럼 보이지만 실제로는 글자 파일이므로, 파일 경로와 글자 인코딩이 중요하다.

df = pd.read_csv(path, encoding="utf-8-sig")

이번 과정에서 배포하는 CSV는 대부분 UTF-8로 정규화되어 있어 encoding="utf-8-sig"나 기본값으로 읽힌다. 다만 한국 공공데이터나 오래된 업무 파일에는 cp949 인코딩이 남아 있기도 한데, 이런 파일은 글자가 깨지거나 UnicodeDecodeError를 낸다. cp949 원본을 직접 여는 실습은 S12에서 한다.

S11에서는 인코딩을 깊게 해결하지 않는다. 오늘은 "CSV는 글자 저장 방식이 맞아야 열린다"는 정도를 기억한다.

4. head, info, describe는 데이터 첫 점검 도구다

데이터를 읽자마자 분석을 시작하지 않는다. 먼저 어떤 표인지 확인한다.

확인 도구 하는 일 엑셀 비교
df.head() 앞쪽 몇 행 보기 시트 맨 위 훑어보기
df.info() 행수, 컬럼, 결측, 타입 보기 열 목록과 데이터 형식 확인
df.describe() 숫자 열의 기초 통계 보기 평균, 최소, 최대 빠른 확인
df.shape 행과 열 개수 보기 시트 크기 확인
df.columns 컬럼명 보기 머리글 행 확인

이번 데이터의 info()를 보면 10,424행이 모두 채워져 있고 amount, qty, unit_price가 숫자형으로 읽힌다. date만 문자열이다. 이 문자열 날짜를 실제 날짜 타입으로 바꾸는 일은 다음 차시 정제 주제로 미룬다.

5. 불리언 마스크는 True/False로 된 필터 조건이다

엑셀 필터에서 지점 열을 열고 강남점만 체크한다고 생각해 보자. pandas에서는 먼저 행마다 조건을 검사한다.

gangnam_mask = df["store"] == "강남점"

이 코드는 각 행의 store 값이 강남점인지 비교한다. 결과는 True 또는 False로 된 Series다. 이 True/False 열을 불리언 마스크라고 부른다.

gangnam = df.loc[gangnam_mask]

df.loc[gangnam_mask]는 마스크가 True인 행만 남긴다. 즉, 엑셀 필터에서 조건에 맞는 행만 보이게 한 상태와 같다.

조건이 여러 개일 때는 괄호와 &를 쓴다.

mask = (
    (df["store"] == "강남점")
    & (df["category"] == "커피")
    & (df["amount"] >= 10000)
)

초보자가 자주 틀리는 지점은 두 가지다. 첫째, 파이썬의 and가 아니라 pandas 조건 결합에는 &를 쓴다. 둘째, 각 조건을 괄호로 감싼다.

6. sort_values는 정렬이다

엑셀에서 금액 열을 기준으로 내림차순 정렬하듯, pandas에서는 sort_values를 쓴다.

gangnam_sorted = gangnam.sort_values("amount", ascending=False)

ascending=False는 큰 값이 위로 오게 하라는 뜻이다. 같은 금액이 많을 때는 날짜나 주문번호를 보조 기준으로 추가할 수 있다.

gangnam.sort_values(
    ["amount", "date", "order_id"],
    ascending=[False, True, True],
)

처음에는 정렬 기준 열 이름과 오름차순/내림차순만 이해하면 된다.

7. loc는 이름 기준, iloc는 위치 기준이다

pandas에서 행과 열을 고를 때 자주 보는 도구가 .loc와 .iloc다.

도구 기준 예시
.loc 인덱스 라벨과 컬럼명 df.loc[0, "store"]
.iloc 숫자 위치 df.iloc[0, 4]

이번 데이터에서 첫 행의 store는 두 방식으로 모두 꺼낼 수 있다.

df.loc[0, "store"]
df.iloc[0, 4]

둘 다 홍대점을 반환한다. 하지만 의미는 다르다. .loc는 "0번 라벨 행의 store 열"이고, .iloc는 "0번째 행, 4번째 열"이다. 입문 단계에서는 컬럼명을 직접 쓰는 .loc가 읽기 쉽다.

  • df.loc[mask]는 True인 행만 남기는 필터 결과다.

따라하기 9: loc와 iloc 비교하기

.loc는 이름 기준, .iloc는 위치 기준이다.

print(df.loc[0, "store"])
print(df.iloc[0, 4])

둘 다 아래 값을 출력한다.

홍대점

하지만 뜻은 다르다.

  • df.loc[0, "store"]: 인덱스 라벨 0번 행의 store 열
  • df.iloc[0, 4]: 0번째 행, 4번째 열

from pathlib import Path
import pandas as pd

root = Path("/mnt/d/ai-data-analysis-course")
path = root / "02_datasets/cafe_sales/cafe_sales_clean.csv"

df = pd.read_csv(path, encoding="utf-8-sig")

    (df["store"] == "강남점")
    & (df["category"] == "커피")

)

)

print(f"미션 평균금액: {mission['amount'].mean():.2f}")
print(mission.head(10))

해설

핵심은 조건을 먼저 True/False로 만든 뒤, 그 조건이 True인 행만 남기는 것이다.

mission_mask = (
    (df["store"] == "강남점")
    & (df["category"] == "커피")
    & (df["amount"] >= 10000)
)

이 코드는 행마다 세 가지 질문을 한다.

  1. 지점이 강남점인가
  2. 카테고리가 커피인가
  3. 금액이 10,000원 이상인가

세 조건을 모두 만족해야 하므로 조건 사이에 &를 쓴다. pandas에서는 각 조건이 True/False 하나가 아니라 True/False로 된 Series 전체이기 때문에 and가 아니라 &를 사용한다. 각 조건을 괄호로 감싸는 것도 중요하다.

조건을 적용하는 줄은 아래와 같다.

df.loc[mission_mask, display_cols]

앞쪽의 mission_mask는 True인 행만 남긴다. 뒤쪽의 display_cols는 화면에 보여줄 열만 고른다. 엑셀로 비유하면 필터를 적용한 뒤 필요한 열만 남겨 보는 것이다.

정렬은 아래 줄에서 처리한다.

sort_values(["amount", "date", "order_id"], ascending=[False, True, True])

첫 번째 기준인 amount는 내림차순이므로 금액이 큰 주문이 위에 온다. 같은 금액이 여러 개 있을 때는 날짜와 주문번호를 오름차순으로 정렬해 결과가 일정하게 보이게 했다.

검증 통과

입문자에게 중요한 오류 포인트

=와 ==를 구분한다.

df["store"] == "강남점"

위 코드는 같은지 비교하는 조건이다. df["store"] = "강남점"처럼 쓰면 필터가 아니라 값을 바꾸는 코드가 된다.

조건 여러 개를 연결할 때는 and가 아니라 &를 쓴다.

(df["store"] == "강남점") & (df["category"] == "커피")

각 조건을 괄호로 감싼다. 괄호를 빼면 연산 순서 때문에 오류가 나거나 의도와 다른 결과가 나올 수 있다.

df.info()는 화면에 직접 출력하는 명령이다. print(df.info())라고 쓰면 정보 아래에 None이 추가로 보일 수 있다. 오류는 아니지만 입문자에게 혼란스럽기 때문에 수업에서는 df.info()만 실행한다.
