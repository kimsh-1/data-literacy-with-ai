S11. Pandas 입문: Series, DataFrame, CSV 읽기와 필터

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S10 Colab과 파이썬 기초
  • 난이도: 3
  • 시간: 60분
  • 사용 데이터: 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일

pandas는 파이썬에서 표 데이터를 다루게 해 주는 도구입니다. 이 차시에서는 CSV, 즉 엑셀로도 열리는 표 파일을 pandas로 읽어 봅니다. 초보자는 같은 데이터를 엑셀 파일(.xlsx)로도 받아서 눈으로 먼저 확인해도 됩니다. 다만 파이썬 실습 코드는 CSV 파일을 기준으로 진행합니다.

S10에서 변수, 리스트, 딕셔너리, 조건문을 배웠다면, 이제는 그 문법을 실제 표에 적용해 봅니다. 카페 매출 데이터를 읽고, DataFrame과 Series를 각각 엑셀 시트와 엑셀의 한 열에 견주어 이해합니다. 그런 다음 head, info, describe로 데이터를 먼저 살펴보고, 불리언 필터로 원하는 행만 남긴 뒤 금액순으로 정렬해 봅니다.

가장 중요한 연결 고리는 "엑셀 필터 = 불리언 마스크"입니다. 엑셀에서는 필터 창에서 강남점을 체크하죠. pandas에서는 df["store"] == "강남점"이라는 True/False 열을 먼저 만듭니다. 그다음 True인 행만 남깁니다. 사전에서 원하는 단어만 표시해 두고 그 줄만 읽는 것과 비슷합니다. 이 감각을 잡으면 이후 정제, 집계, 시각화 차시에서 나오는 pandas 코드가 훨씬 덜 낯설어집니다.

실제 데이터 확인 결과

수업 자료 작성 전 python3와 pandas 3.0.3으로 실제 카페 매출 파일을 열어 확인했습니다. 여기서는 데이터가 어떤 모양인지 먼저 봅니다. 분석을 시작하기 전에 키를 재듯이, 표의 크기와 값의 범위를 확인하는 과정입니다.

order_id, date, weekday, hour, store, category, item, unit_price, qty, amount, payment, member

이 차시에서는 이미 정리된 카페 매출 파일을 사용합니다. 날짜는 문자열로 읽히지만 그대로 둡니다. 문자열은 컴퓨터가 글자로 보고 있는 값이라는 뜻입니다. 날짜처럼 보이지만 아직 날짜 계산을 할 수 있는 형태는 아닙니다. 이 문자열 날짜를 실제 날짜 타입으로 바꾸는 일은 S12 정제 차시에서 다룹니다.

학습 목표

  1. pandas의 DataFrame을 엑셀 시트, Series를 엑셀의 한 열에 비유해 설명할 수 있습니다.
  2. pd.read_csv()로 UTF-8 CSV 파일을 읽고, 한글 CSV에서 인코딩 문제가 생길 수 있음을 알아차릴 수 있습니다.
  3. head, info, describe로 데이터의 첫 행, 컬럼, 타입, 기초 통계를 확인할 수 있습니다.
  4. 불리언 조건식을 만들어 원하는 행만 필터링할 수 있습니다.
  5. .loc, .iloc의 차이를 라벨 기준 선택과 위치 기준 선택으로 구분할 수 있습니다.
  6. sort_values로 금액이 큰 주문부터 정렬할 수 있습니다.

꼭 잡아야 할 개념

1. pandas는 표를 다루는 파이썬 도구입니다

S10에서는 변수, 리스트, 딕셔너리, 조건문, 반복문을 배웠습니다. pandas는 그 문법을 표 데이터에 맞게 확장해 주는 도구입니다. 엑셀에서 셀과 열을 다루듯이, pandas에서는 행과 열이 있는 표를 코드로 다룹니다.

import pandas as pd

여기서 pd는 pandas를 짧게 부르는 별명입니다. 매번 pandas라고 길게 쓰지 않고 pd라고 부르겠다는 약속입니다. 엑셀에서 SUM, AVERAGE, FILTER 같은 기능을 쓰듯, pandas에서는 read_csv, head, describe, sort_values 같은 기능을 씁니다.

처음에는 pandas를 어렵게 생각하지 않아도 됩니다. "엑셀에서 하던 표 확인과 필터를 파이썬 코드로 한다"라고 잡으면 충분합니다.

2. DataFrame은 시트, Series는 열입니다

CSV를 읽으면 pandas는 보통 DataFrame을 만듭니다. DataFrame은 행과 열이 있는 표입니다. 엑셀의 워크시트 한 장을 떠올리면 됩니다.

df = pd.read_csv(path, encoding="utf-8-sig")

여기서 df는 보통 DataFrame을 담을 때 많이 쓰는 이름입니다. 이름은 바꿀 수 있지만, 입문 단계에서는 많은 예제가 df를 쓰기 때문에 그대로 따라가면 읽기 편합니다.

DataFrame에서 열 하나를 꺼내면 Series가 됩니다.

amount_series = df["amount"]

엑셀로 비교하면 아래처럼 보면 됩니다.

엑셀 감각 pandas 감각
워크시트 전체 DataFrame
열 하나 Series
행 번호 인덱스
열 제목 컬럼명
필터 조건 불리언 마스크

처음부터 DataFrame과 Series라는 이름을 완벽하게 외우려고 하지 않아도 됩니다. 먼저 "DataFrame은 표 전체", "Series는 열 하나"라는 감각을 잡아 보세요. 엑셀 파일을 열었을 때 화면 전체가 DataFrame이고, 그중 금액 열 하나만 손가락으로 짚으면 Series입니다.

3. CSV 읽기는 파일 선택과 인코딩을 확인하는 일입니다

CSV는 쉼표로 값을 구분해 저장한 텍스트 파일입니다. 엑셀로 열면 표처럼 보이지만, 실제로는 글자 파일입니다. 그래서 pandas로 읽을 때는 어떤 파일을 읽을지와 글자가 어떤 방식으로 저장되어 있는지를 확인해야 합니다. 이 글자 저장 방식을 인코딩이라고 부릅니다.

df = pd.read_csv(path, encoding="utf-8-sig")

위 코드에서 path는 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일을 가리키는 이름이라고 보면 됩니다. encoding="utf-8-sig"는 한글이 깨지지 않도록 글자 저장 방식을 맞춰 읽으라는 뜻입니다.

이 과정에서 사용하는 CSV는 대부분 UTF-8로 정규화되어 있어 encoding="utf-8-sig"나 기본값으로 읽힙니다. 다만 한국 공공데이터나 오래된 업무 파일에는 cp949 인코딩이 남아 있기도 합니다. 이런 파일은 글자가 깨지거나 UnicodeDecodeError를 냅니다. 오류 이름이 낯설어도 뜻은 단순합니다. "글자 저장 방식이 맞지 않아 읽지 못했다"는 말입니다.

S11에서는 인코딩을 깊게 해결하지 않습니다. 오늘은 "CSV는 표처럼 보여도 글자 파일이고, 글자 저장 방식이 맞아야 열린다"는 정도를 기억하면 됩니다. cp949 원본을 직접 여는 실습은 S12에서 합니다.

4. head, info, describe는 데이터 첫 점검 도구입니다

데이터를 읽자마자 분석을 시작하지 않습니다. 먼저 어떤 표인지 확인합니다. 택배 상자를 받으면 바로 쓰기 전에 주소와 내용물을 확인하듯이, 데이터도 먼저 크기와 열 이름과 값의 상태를 봐야 합니다.

확인 도구 하는 일 엑셀 비교
df.head() 앞쪽 몇 행 보기 시트 맨 위 훑어보기
df.info() 행수, 컬럼, 결측, 타입 보기 열 목록과 데이터 형식 확인
df.describe() 숫자 열의 기초 통계 보기 평균, 최소, 최대 빠른 확인
df.shape 행과 열 개수 보기 시트 크기 확인
df.columns 컬럼명 보기 머리글 행 확인

df.head()는 앞쪽 몇 줄을 보여 줍니다. 엑셀 파일을 열고 맨 위 행들을 눈으로 훑는 행동과 같습니다.

df.info()는 행수, 컬럼, 비어 있는 값, 데이터 타입을 보여 줍니다. 여기서 데이터 타입은 숫자인지, 글자인지 같은 형식을 말합니다.

df.describe()는 숫자 열의 평균, 최소, 최대 같은 기초 통계를 보여 줍니다. 오늘은 이 결과를 보고 금액이 어느 정도 범위 안에 있는지 빠르게 확인합니다.

이번 데이터의 info()를 보면 10,424행이 모두 채워져 있고 amount, qty, unit_price가 숫자형으로 읽힙니다. date만 문자열입니다. 날짜처럼 생겼지만 아직 컴퓨터에게는 글자입니다. 이 문자열 날짜를 실제 날짜 타입으로 바꾸는 일은 다음 차시 정제 주제로 미룹니다.

5. 불리언 마스크는 True/False로 된 필터 조건입니다

엑셀 필터에서 지점 열을 열고 강남점만 체크한다고 생각해 볼까요? pandas에서는 먼저 행마다 조건을 검사합니다.

gangnam_mask = df["store"] == "강남점"

이 코드는 각 행의 store 값이 강남점인지 비교합니다. 맞으면 True, 아니면 False가 됩니다. 결과는 True 또는 False로 된 Series입니다. 이 True/False 열을 불리언 마스크라고 부릅니다.

불리언은 True와 False처럼 참과 거짓을 나타내는 값입니다. 마스크는 필요한 부분만 보이게 가리는 도구라고 생각하면 됩니다. 얼굴팩 마스크처럼 필요한 곳만 덮는 것이 아니라, 데이터에서는 조건에 맞는 행만 통과시키는 표시라고 보면 됩니다.

gangnam = df.loc[gangnam_mask]

df.loc[gangnam_mask]는 마스크가 True인 행만 남깁니다. 엑셀 필터에서 조건에 맞는 행만 보이게 한 상태와 같습니다. 여기서 gangnam에는 지점이 강남점인 주문만 들어갑니다.

조건이 여러 개일 때는 괄호와 &를 씁니다.

mask = (
    (df["store"] == "강남점")
    & (df["category"] == "커피")
    & (df["amount"] >= 10000)
)

이 조건은 지점이 강남점이고, 카테고리가 커피이고, 금액이 10,000원 이상인 주문만 찾습니다. 세 조건을 모두 만족해야 하므로 &를 씁니다. 엑셀에서 필터를 여러 열에 동시에 거는 상황과 같습니다.

초보자가 자주 틀리는 부분은 두 가지입니다. 하나는 파이썬의 and가 아니라 pandas 조건 결합에는 &를 쓴다는 점입니다. 또 하나는 각 조건을 괄호로 감싼다는 점입니다. 괄호는 조건 하나하나를 묶어 주는 울타리입니다. 울타리가 없으면 pandas가 어느 조건부터 계산해야 하는지 헷갈릴 수 있습니다.

6. sort_values는 정렬입니다

엑셀에서 금액 열을 기준으로 내림차순 정렬하듯, pandas에서는 sort_values를 씁니다. 정렬은 표의 순서를 바꾸는 일입니다. 값이 큰 주문부터 보고 싶다면 금액 열을 기준으로 정렬하면 됩니다.

gangnam_sorted = gangnam.sort_values("amount", ascending=False)

ascending=False는 큰 값이 위로 오게 하라는 뜻입니다. ascending은 오름차순이라는 뜻인데, 여기에 False를 주었으니 오름차순이 아니라 내림차순으로 보겠다는 말입니다.

같은 금액이 많을 때는 날짜나 주문번호를 보조 기준으로 추가할 수 있습니다.

gangnam.sort_values(
    ["amount", "date", "order_id"],
    ascending=[False, True, True],
)

이 코드는 먼저 amount를 큰 금액부터 봅니다. 금액이 같으면 date를 빠른 날짜부터 보고, 날짜도 같으면 order_id를 작은 번호부터 봅니다. 처음에는 정렬 기준 열 이름과 오름차순/내림차순만 이해하면 됩니다.

7. loc는 이름 기준, iloc는 위치 기준입니다

pandas에서 행과 열을 고를 때 자주 보는 도구가 .loc와 .iloc입니다. 둘 다 데이터를 고르는 도구이지만 기준이 다릅니다.

도구 기준 예시
.loc 인덱스 라벨과 컬럼명 df.loc[0, "store"]
.iloc 숫자 위치 df.iloc[0, 4]

이번 데이터에서 첫 행의 store는 두 방식으로 모두 꺼낼 수 있습니다.

df.loc[0, "store"]
df.iloc[0, 4]

둘 다 홍대점을 반환합니다. 하지만 의미는 다릅니다. .loc는 "0번 라벨 행의 store 열"입니다. 컬럼명을 직접 보고 고르는 방식입니다. .iloc는 "0번째 행, 4번째 열"입니다. 위치 번호로 고르는 방식입니다.

엑셀로 비유하면 .loc는 "store라는 제목의 열을 찾아서 보세요"에 가깝고, .iloc는 "왼쪽에서 5번째 열을 보세요"에 가깝습니다. 입문 단계에서는 컬럼명을 직접 쓰는 .loc가 읽기 쉽습니다. 위치 번호는 열 순서가 바뀌면 헷갈릴 수 있으니, 처음에는 이름으로 고르는 습관을 들이면 좋습니다.

  • df.head()는 시트 맨 위 몇 줄을 눈으로 확인하는 행동입니다. 데이터가 제대로 열렸는지 먼저 보는 습관입니다.
  • df.info()는 열 이름, 빈칸 여부, 데이터 형식을 한 번에 확인하는 점검표입니다. 분석 전에 표의 건강 상태를 보는 과정입니다.
  • df.describe()는 숫자 열의 평균, 최소, 최대를 빠르게 보는 요약 통계입니다. 금액이 말이 되는 범위인지 확인할 수 있습니다.
  • df["store"] == "강남점"은 엑셀 필터에서 강남점만 체크하기 전, 각 행이 조건에 맞는지 True/False로 표시한 것입니다.
  • df.loc[mask]는 True인 행만 남기는 필터 결과입니다. 엑셀 필터를 적용한 뒤 보이는 행만 남긴 상태와 같습니다.
  • sort_values("amount", ascending=False)는 금액 열을 큰 값부터 정렬하는 것입니다. 매출 금액이 큰 주문을 위에서부터 보고 싶을 때 씁니다.

수업 후 수강생이 말할 수 있어야 하는 문장

  • "DataFrame은 표 전체이고, Series는 그 안의 열 하나다."
  • "CSV를 읽은 뒤에는 바로 분석하지 않고 head, info, describe로 먼저 확인한다."
  • "엑셀 필터에서 조건을 체크하는 일은 pandas에서 True/False 마스크를 만드는 일과 같다."
  • "df.loc[mask]는 조건이 True인 행만 남긴다."
  • "금액이 큰 순서로 보려면 sort_values에 정렬할 열과 내림차순 옵션을 준다."

오늘 사용할 데이터

pandas는 파이썬에서 표 데이터를 다루는 도구입니다. 오늘은 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일을 사용합니다.

CSV는 엑셀로도 열리는 표 파일입니다. 처음에는 CSV가 낯설 수 있으니, 눈으로 먼저 확인하고 싶다면 엑셀 파일(.xlsx)을 받아 열어 보셔도 됩니다. 다만 오늘 실습 코드는 CSV 파일을 읽는 방식으로 진행합니다.

이 카페 매출 파일은 한글이 깨지지 않도록 UTF-8로 정리된 데이터이고, 실제 확인 결과 10,424행 x 12열입니다.

오늘 해 볼 일은 pandas로 CSV를 열고, 표의 구조를 확인하고, 엑셀 필터처럼 원하는 행만 골라내는 것입니다. 데이터를 바로 계산하기보다, 먼저 “이 표가 어떻게 생겼는지” 확인하는 습관을 잡아 봅니다.

따라하기 2: DataFrame과 Series 확인하기

DataFrame은 표 전체이고, Series는 열 하나입니다. 엑셀로 보면 DataFrame은 시트 한 장, Series는 그 안의 금액 열 하나라고 생각하면 됩니다.

이 구분을 먼저 확인해야 나중에 “표 전체를 다루는 코드인지, 열 하나를 다루는 코드인지” 덜 헷갈립니다.

print(type(df))

amount_series = df["amount"]
print(type(amount_series))
print(len(amount_series))

확인할 점:

  • df는 DataFrame입니다.
  • df["amount"]는 Series입니다.
  • amount_series의 길이는 전체 행수와 같은 10,424입니다.

엑셀로 비유하면 df는 시트 전체이고, amount_series는 금액 열 하나입니다. 표 전체와 열 하나를 구분하는 감각이 pandas의 출발점입니다.

따라하기 3: 처음 몇 행 보기

데이터를 읽은 뒤에는 바로 분석하지 말고, 앞쪽 몇 줄을 먼저 봅니다. 엑셀 파일을 열었을 때 맨 위 행들을 눈으로 훑어보는 것과 같습니다.

head()는 데이터가 제대로 읽혔는지 빠르게 보는 도구입니다. 사람이 표를 처음 볼 때 위에서부터 몇 줄 읽어 보는 것과 같은 역할을 합니다.

따라하기 4: 표 구조 확인하기

이번에는 표의 크기, 열 이름, 각 열의 데이터 종류를 확인합니다. 키를 재기 전에 줄자를 확인하듯, 분석 전에 표의 구조를 먼저 확인하는 단계입니다.

따라하기 5: 숫자 열 요약 보기

이 단계에서는 숫자 열의 대략적인 범위를 봅니다. 금액이 어느 정도인지, 수량이 몇 개까지 있는지 먼저 알아야 이상한 값이 있는지도 눈에 들어옵니다.

따라하기 7: 강남점만 필터링하기

1단계. 조건 만들기

필터링을 하려면 먼저 어떤 행을 남길지 조건을 만들어야 합니다. 엑셀 필터에서 강남점만 체크하기 전에, pandas에서는 각 행이 강남점인지 아닌지를 True와 False로 표시합니다.

gangnam_mask = df["store"] == "강남점"

print(gangnam_mask.head(10))

gangnam_mask에는 True와 False가 들어 있습니다. 각 행의 지점이 강남점이면 True, 아니면 False입니다.

이 True/False 열을 불리언 마스크라고 부릅니다. “조건에 맞는 행에는 표시를 해 둔 체크 목록”이라고 생각하면 됩니다.

2단계. True인 행만 남기기

이제 방금 만든 체크 목록을 사용해서 True인 행만 남깁니다. 엑셀에서 필터를 적용하면 조건에 맞는 행만 보이는 것과 같은 과정입니다.

gangnam = df.loc[gangnam_mask]

print(gangnam.shape)
print(gangnam.head())

확인할 값:

(6196, 12)

엑셀에서 지점 필터를 열고 강남점만 체크한 결과와 같습니다. 6,196개의 행이 남고, 열은 원래처럼 12개입니다.

따라하기 8: 강남점 주문을 금액 큰 순서로 정렬하기

강남점 주문만 남겼다면, 이제 금액이 큰 주문부터 보겠습니다. 정렬은 큰 값이나 작은 값을 먼저 보고 싶을 때 쓰는 기능입니다.

display_cols = ["order_id", "date", "store", "category", "item", "qty", "amount"]

gangnam_sorted = gangnam.sort_values(
    ["amount", "date", "order_id"],
    ascending=[False, True, True],
)

print(gangnam_sorted.loc[:, display_cols].head(10))

해석:

  • amount는 큰 금액부터 봅니다.
  • 같은 금액이면 date, order_id 순서로 보기 좋게 정렬합니다.
  • display_cols는 화면에 보여줄 열만 고른 것입니다.

sort_values()는 엑셀에서 특정 열을 기준으로 오름차순이나 내림차순 정렬하는 기능과 같습니다. 여기서는 금액을 가장 먼저 보고 싶기 때문에 amount를 정렬 기준으로 둡니다.

따라하기 9: loc와 iloc 비교하기

.loc는 이름 기준, .iloc는 위치 기준입니다. 사전에서 단어 이름으로 찾는 방식이 .loc에 가깝고, 책장에서 몇 번째 칸인지 세어서 찾는 방식이 .iloc에 가깝습니다.

print(df.loc[0, "store"])
print(df.iloc[0, 4])

둘 다 아래 값을 출력합니다.

홍대점

하지만 뜻은 다릅니다.

  • df.loc[0, "store"]: 인덱스 라벨 0번 행의 store 열
  • df.iloc[0, 4]: 0번째 행, 4번째 열

여러 행과 여러 열을 고를 수도 있습니다.

print(df.loc[0:2, ["date", "store", "amount"]])
print(df.iloc[0:3, [1, 4, 9]])

주의할 점:

  • .loc[0:2]는 라벨 기준이라 0, 1, 2가 포함됩니다.
  • .iloc[0:3]는 위치 기준이라 0, 1, 2번째 행까지만 포함되고 3번째는 제외됩니다.

처음에는 열 이름이 직접 보이는 .loc가 더 읽기 쉽습니다. 다만 위치 번호로 골라야 할 때는 .iloc를 씁니다.

51-56분. loc와 iloc 비교하기

시연 코드:

둘 다 홍대점이 나옵니다. 하지만 고르는 방식은 다릅니다. .loc는 이름 기준이고, .iloc는 위치 기준입니다.

시연 코드:

주의:

.loc[0:2]는 2가 포함되고, .iloc[0:3]은 3이 포함되지 않습니다. 오늘은 이 차이를 억지로 외우기보다, .loc는 이름 기준, .iloc는 위치 기준이라는 점을 먼저 잡습니다.

미션 모범답안

미션:

강남점 주문 중 카테고리가 커피이고, 주문 금액이 10,000원 이상인 행만 추출한다.
그 결과를 금액 큰 순서로 정렬하고, 행수와 총금액과 평균금액을 출력한다.

모범 코드:

from pathlib import Path
import pandas as pd

root = Path("/mnt/d/ai-data-analysis-course")
path = root / "02_datasets/cafe_sales/cafe_sales_clean.csv"

df = pd.read_csv(path, encoding="utf-8-sig")

display_cols = ["order_id", "date", "store", "category", "item", "qty", "amount"]

mission_mask = (
    (df["store"] == "강남점")
    & (df["category"] == "커피")
    & (df["amount"] >= 10000)
)

mission = (
    df.loc[mission_mask, display_cols]
    .sort_values(["amount", "date", "order_id"], ascending=[False, True, True])
    .copy()
)

print(f"미션 행수: {len(mission)}")
print(f"미션 총금액: {mission['amount'].sum()}")
print(f"미션 평균금액: {mission['amount'].mean():.2f}")
print(mission.head(10))

해설

핵심은 조건을 먼저 True/False로 만든 뒤, 그 조건이 True인 행만 남기는 것이다.

mission_mask = (
    (df["store"] == "강남점")
    & (df["category"] == "커피")
    & (df["amount"] >= 10000)
)

이 코드는 행마다 세 가지 질문을 한다.

  1. 지점이 강남점인가
  2. 카테고리가 커피인가
  3. 금액이 10,000원 이상인가

세 조건을 모두 만족해야 하므로 조건 사이에 &를 쓴다. pandas에서는 각 조건이 True/False 하나가 아니라 True/False로 된 Series 전체이기 때문에 and가 아니라 &를 사용한다. 각 조건을 괄호로 감싸는 것도 중요하다.

조건을 적용하는 줄은 아래와 같다.

df.loc[mission_mask, display_cols]

앞쪽의 mission_mask는 True인 행만 남긴다. 뒤쪽의 display_cols는 화면에 보여줄 열만 고른다. 엑셀로 비유하면 필터를 적용한 뒤 필요한 열만 남겨 보는 것이다.

정렬은 아래 줄에서 처리한다.

sort_values(["amount", "date", "order_id"], ascending=[False, True, True])

검증 통과

입문자에게 중요한 오류 포인트

=와 ==를 구분한다.

df["store"] == "강남점"

위 코드는 같은지 비교하는 조건이다. df["store"] = "강남점"처럼 쓰면 필터가 아니라 값을 바꾸는 코드가 된다.

조건 여러 개를 연결할 때는 and가 아니라 &를 쓴다.

(df["store"] == "강남점") & (df["category"] == "커피")

각 조건을 괄호로 감싼다. 괄호를 빼면 연산 순서 때문에 오류가 나거나 의도와 다른 결과가 나올 수 있다.

df.info()는 화면에 직접 출력하는 명령이다. print(df.info())라고 쓰면 정보 아래에 None이 추가로 보일 수 있다. 오류는 아니지만 입문자에게 혼란스럽기 때문에 수업에서는 df.info()만 실행한다.
