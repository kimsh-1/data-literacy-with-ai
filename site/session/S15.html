S15. Power BI 설치, 로드, 첫 차트

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S03 피벗테이블과 기본 집계
  • 난이도: 3
  • 시간: 60분
  • 사용 데이터: /mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.csv
  • 보조 데이터 파일: /mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.xlsx

이번 차시는 BI 도구를 처음 열어 보는 시간이다. BI는 무언가를 새로 계산해 주는 도구가 아니다. 이미 정리된 표를 화면에 배치해, 다른 사람이 계속 들여다볼 수 있는 대시보드로 바꿔 준다. 엑셀 피벗테이블이 "내가 빠르게 집계해 보는 작업장"이라면, Power BI는 "다른 사람이 필터를 바꿔 가며 보는 보고 화면"에 가깝다.

수강생은 Power BI Desktop을 설치하거나 실행하고, 정제된 카페 매출 CSV를 불러온 뒤, 첫 페이지에 총매출 카드, 카테고리별 매출 막대그래프, 일자별 매출 꺾은선그래프를 만든다. Mac 수강생은 Power BI Desktop 설치 대신 화면 구조와 개념을 따라오고, S18에서 Looker Studio 경로로 직접 실습한다.

학습 목표

  1. Power BI Desktop이 Windows용 보고서 제작 도구라는 점을 알고 설치 또는 실행할 수 있다.
  2. CSV 파일과 엑셀 파일을 Power BI에 불러오는 경로를 설명할 수 있다.
  3. 필드 목록, 시각화 패널, 캔버스의 역할을 엑셀 피벗테이블과 비교해 이해한다.
  4. category, date, amount 필드를 사용해 막대그래프와 꺾은선그래프를 만들 수 있다.
  5. amount의 합계를 KPI 카드로 표시하고, 표시 단위를 조정해 총매출을 확인할 수 있다.

핵심 개념 1. Power BI Desktop은 보고서 제작 도구이다

Power BI라는 이름은 여러 형태로 등장한다. 입문자가 처음에 헷갈리는 지점도 대개 여기다.

구분 이번 차시에서의 의미
Power BI Desktop Windows PC에 설치해 보고서 파일을 만드는 프로그램
.pbix 파일 Power BI Desktop에서 저장하는 보고서 파일
Power BI Service 웹에 게시하고 공유하는 서비스. S18에서 공유 개념을 다룬다
Power BI Pro, Fabric, Copilot 조직 공유와 고급 기능에 필요한 유료 또는 조직 환경. 오늘은 쓰지 않는다

이번 차시는 Power BI Desktop에서 로컬 파일을 열고 보고서 한 페이지를 만드는 데 집중한다. 계정 공유, 게시, 권한 관리는 오늘의 목표가 아니다.

Power BI Desktop은 Windows용 도구다. Microsoft 공식 다운로드 페이지도 Windows용 Desktop 설치만 안내하고, 강의 환경은 Windows 10이나 Windows 11 PC를 기준으로 잡는다. Mac 사용자는 억지로 우회 설치할 필요 없이 S18에서 Looker Studio로 대체 실습한다.

핵심 개념 2. 로드는 표를 Power BI 안으로 가져오는 일이다

엑셀은 파일을 열면 바로 셀이 보인다. Power BI는 그렇지 않다. 데이터를 먼저 가져와야 하고, 이 과정을 로드라고 부른다.

데이터를 가져오는 감각은 피벗테이블을 만들기 전에 원본 표를 지정하던 일과 비슷하다. 차이가 있다면, Power BI는 원본을 그대로 펼쳐 보여주는 프로그램이 아니라 그 표를 바탕으로 시각화 화면을 짜는 프로그램이라는 점이다.

엑셀에서의 감각 Power BI에서의 감각
엑셀 파일 열기 데이터 가져오기
표 범위 선택 CSV 또는 엑셀 시트 선택
피벗테이블 필드 목록 데이터 패널의 필드 목록
행, 열, 값 영역 X축, Y축, 값, 범례 영역
피벗 차트 시각화 개체

오늘 다루는 cafe_sales_clean은 이미 깨끗하게 정리돼 있다. 덕분에 Power Query에서 손대는 정제 작업은 없다. 대신 date가 날짜로, amount가 숫자로 제대로 인식되는지만 확인하면 된다.

핵심 개념 3. 시각화는 필드를 놓는 방식으로 만든다

화면 가운데 흰 영역이 캔버스다. 카드, 막대그래프, 꺾은선그래프 같은 시각화 개체를 여기에 올린다. 오른쪽에는 시각화 패널과 데이터 패널이 붙어 있다.

시각화는 대체로 세 가지 질문에서 출발한다.

  1. 무엇을 비교할 것인가: category, store, date
  2. 어떤 숫자를 볼 것인가: amount, qty, order_id
  3. 어떤 모양이 적절한가: 카드, 막대, 꺾은선

막대그래프는 항목을 나란히 견줄 때 좋다. 카테고리별 매출이 그렇다. category마다 amount 합계를 세워 비교한다. 시간 흐름에는 꺾은선그래프가 맞다. 일자별 매출이라면 date 순서대로 amount 합계를 이어 본다. KPI 카드는 핵심 숫자 하나를 크게 보여줄 자리이니, 총매출이 여기에 어울린다.

핵심 개념 4. KPI 카드는 큰 셀 하나라고 생각한다

엑셀에서 =SUM(매출범위)를 큰 글씨로 박아 두면 보고받는 사람이 제일 먼저 보는 숫자가 된다. Power BI의 카드도 딱 그 역할이다.

이번 미션은 amount 합계를 카드로 띄우는 것이고, 맞는 값은 총매출 90,492,500이다. Power BI는 기본값에서 이 숫자를 90M이나 90.49M처럼 줄여 보여줄 때가 있다. 정확히 검산하려면 표시 단위를 없음(None)으로 바꾸고 소수 자릿수를 0으로 맞춘다.

엑셀과 연결해서 설명할 지점

  • Power BI의 데이터 패널은 피벗테이블 필드 목록과 비슷하다.
  • 막대그래프의 X축은 피벗테이블의 행 영역처럼 비교 기준을 놓는 자리이다.
  • 막대그래프의 Y축 또는 값 영역은 피벗테이블의 값 영역처럼 합계나 개수를 놓는 자리이다.
  • KPI 카드는 엑셀에서 SUM 결과를 크게 보여주는 셀과 비슷하다.
  • Power BI는 화면을 만드는 도구이므로, 원본 데이터를 고치는 일은 엑셀 정제, Power Query, 파이썬과 역할을 나누어 생각한다.

Power BI Desktop은 Windows용 프로그램이다. Mac 수강생은 오늘 화면 구조와 개념만 눈으로 따라오면 되고, 실습은 S18의 Looker Studio에서 이어 간다.

공식 설치 경로:

  • Microsoft Power BI 다운로드: https://www.microsoft.com/en-us/power-platform/products/power-bi/downloads
  • Microsoft Store Power BI Desktop: https://apps.microsoft.com/detail/9ntxr16hnw1t

실습 기준은 cafe_sales_clean.csv다. 엑셀 파일은 같은 데이터를 담은 보조 파일로, 엑셀 로드 경로를 확인할 때만 쓴다. 한 Power BI 파일에 CSV와 엑셀을 동시에 불러오지는 않는다.

따라하기 1: Power BI Desktop 열기

1단계. 설치 확인

  1. Windows 시작 메뉴를 연다.
  2. Power BI Desktop을 검색한다.
  3. 프로그램이 있으면 실행한다.
  4. 프로그램이 없으면 강사의 안내에 따라 Microsoft Store 또는 Microsoft 공식 다운로드 페이지에서 설치한다.

처음 실행하면 로그인 화면이 뜰 수 있다. 오늘은 로컬 파일로 보고서를 만드는 실습이니, 강사 안내에 맞춰 닫거나 건너뛰면 된다. 조직 PC라면 보안 정책 탓에 설치나 로그인이 막히기도 한다.

2단계. 빈 보고서 화면 확인

Power BI Desktop을 열면 다음 영역을 확인한다.

화면 영역 볼 것
가운데 흰 영역 차트를 놓는 캔버스
오른쪽 데이터 패널 불러온 표와 열 목록
오른쪽 시각화 패널 막대그래프, 꺾은선그래프, 카드 선택
위쪽 리본 메뉴 데이터 가져오기, 저장, 새 시각적 개체

따라하기 2: CSV 파일 불러오기

1단계. 데이터 가져오기

  1. 위쪽 메뉴에서 홈(Home)을 누른다.
  2. 데이터 가져오기(Get data)를 누른다.
  3. 텍스트/CSV(Text/CSV)를 선택한다.
  4. 아래 파일을 선택한다.
/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.csv
  1. 미리보기 화면이 열리면 열 이름을 확인한다.
  2. 로드(Load)를 누른다.

2단계. 데이터 패널 확인

오른쪽 데이터 패널에 cafe_sales_clean 표가 생겼는지 확인한다. 표를 펼쳐 아래 열이 보이면 정상이다.

order_id
date
weekday
hour
store
category
item
unit_price
qty
amount
payment
member

확인할 것:

  • amount는 숫자 열이어야 한다.
  • date는 날짜 열로 인식되는 것이 좋다.
  • 오늘은 정제 실습이 아니므로 복잡한 변환은 하지 않는다.

date가 날짜로 인식되지 않으면, 강사 화면을 따라 변환 데이터(Transform data)에서 형식을 날짜로 바꾼 뒤 닫기 및 적용(Close & Apply)을 누른다.

따라하기 3: 엑셀 파일 로드 경로 확인하기

이 단계는 경로만 눈에 익히는 용도다. 이미 CSV를 불러온 파일에는 엑셀을 덧붙이지 않는다.

새 Power BI 파일에서 엑셀을 불러올 때는 다음 순서를 따른다.

  1. 홈(Home)을 누른다.
  2. 데이터 가져오기(Get data)를 누른다.
  3. Excel 통합 문서(Excel workbook)를 선택한다.
  4. 아래 파일을 선택한다.
/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.xlsx
  1. 탐색기(Navigator)에서 sales 시트를 선택한다.
  2. 로드(Load)를 누른다.

CSV와 엑셀은 같은 데이터다. 실습 결과를 맞추기 위해, 이후 단계는 CSV로 불러온 cafe_sales_clean 표를 기준으로 삼는다.

따라하기 4: 카테고리별 매출 막대그래프 만들기

1단계. 막대그래프 추가

  1. 시각화 패널에서 묶은 세로 막대형 차트(Clustered column chart)를 선택한다.
  2. 캔버스에 빈 막대그래프가 생기면 오른쪽 필드 영역을 본다.
  3. 데이터 패널에서 category를 X축(X-axis)으로 끌어다 놓는다.
  4. 데이터 패널에서 amount를 Y축(Y-axis) 또는 값(Values)으로 끌어다 놓는다.

2단계. 집계 방식 확인

amount가 합계(Sum)로 들어갔는지 확인한다. 평균이나 개수로 잡히면 총매출 비교가 어긋난다.

따라하기 5: 일자별 매출 꺾은선그래프 만들기

1단계. 꺾은선그래프 추가

  1. 캔버스의 빈 공간을 클릭한다.
  2. 시각화 패널에서 꺾은선형 차트(Line chart)를 선택한다.
  3. date를 X축(X-axis)으로 끌어다 놓는다.
  4. amount를 Y축(Y-axis) 또는 값(Values)으로 끌어다 놓는다.

2단계. 날짜 축 확인

Power BI가 date를 날짜 계층 구조(Date hierarchy)로 넣을 때가 있다. 일자별 흐름을 보려면 X축 필드의 드롭다운을 눌러 date 자체를 고른다. 화면이나 버전에 따라 메뉴 이름은 조금씩 다르다.

검증용 Python 코드

S15는 Power BI 차시이므로 별도 S15_code.py 파일을 요구하지 않는다. 아래 코드는 모범답안 수치 검증을 위해 python3와 pandas 3.0.3으로 실제 실행한 코드이다.

from pathlib import Path
import pandas as pd

path = Path("/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.csv")
df = pd.read_csv(path)

print("shape:", df.shape)
print("date_min:", df["date"].min())
print("date_max:", df["date"].max())
print("total_amount:", int(df["amount"].sum()))
print("order_count:", len(df))
print("null_total:", int(df.isna().sum().sum()))

category_summary = (
    df.groupby("category", as_index=False)["amount"]
    .sum()
    .sort_values("amount", ascending=False)
)
print(category_summary.to_string(index=False))

df["date_dt"] = pd.to_datetime(df["date"])
monthly = df.groupby(df["date_dt"].dt.to_period("M"))["amount"].sum().reset_index()
monthly["date_dt"] = monthly["date_dt"].astype(str)
print(monthly.to_string(index=False))
