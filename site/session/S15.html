S15. Power BI 설치, 로드, 첫 차트

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S03 피벗테이블과 기본 집계
  • 난이도: 3
  • 시간: 60분
  • 사용 데이터: 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 CSV, 즉 엑셀로도 열리는 표 파일
  • 보조 데이터 파일: 수업 홈의 데이터 받기 페이지에서 내려받은 같은 카페 매출 엑셀 파일(.xlsx)

BI 도구는 데이터를 새로 만들어 내는 도구가 아니라, 이미 정리된 표를 사람이 보기 좋은 보고 화면으로 바꿔 주는 도구입니다. 여기서 BI는 Business Intelligence의 줄임말이고, 회사나 가게의 숫자를 보면서 판단할 수 있게 도와주는 화면이라고 생각하면 됩니다.

엑셀 피벗테이블은 내가 빠르게 합계를 내 보고 기준을 바꿔 보는 작업장에 가깝습니다. Power BI는 그 결과를 다른 사람이 계속 열어 보고, 필터를 바꾸고, 그래프를 보면서 이해할 수 있는 보고 화면에 가깝습니다. 엑셀에서 계산한 숫자를 종이에 적어 두는 것과, 벽에 붙인 현황판으로 만들어 두는 것의 차이를 떠올리면 됩니다.

수강생은 Power BI Desktop을 설치하거나 실행한 뒤, 수업 홈의 데이터 받기 페이지에서 내려받은 정제된 카페 매출 파일을 불러옵니다. 그리고 첫 보고서 페이지에 총매출 카드, 카테고리별 매출 막대그래프, 일자별 매출 꺾은선그래프를 만듭니다. Mac 수강생은 Power BI Desktop을 직접 설치하기 어렵기 때문에, 이번 시간에는 화면 구조와 개념을 따라오고 S18에서 Looker Studio로 직접 실습합니다.

학습 목표

  1. Power BI Desktop이 Windows용 보고서 제작 도구라는 점을 알고 설치 또는 실행할 수 있다.
  2. CSV 파일과 엑셀 파일을 Power BI에 불러오는 경로를 설명할 수 있다.
  3. 필드 목록, 시각화 패널, 캔버스의 역할을 엑셀 피벗테이블과 비교해 이해한다.
  4. category, date, amount 필드를 사용해 막대그래프와 꺾은선그래프를 만들 수 있다.
  5. amount의 합계를 KPI 카드로 표시하고, 표시 단위를 조정해 총매출을 확인할 수 있다.

데이터 확인 결과

수업에서 쓰는 카페 매출 파일은 python3와 pandas 3.0.3으로 열어 미리 확인한 데이터입니다. 여기서 확인한다는 말은, 표가 몇 줄인지, 기간이 어디부터 어디까지인지, 비어 있는 값이 있는지를 미리 검사했다는 뜻입니다. 키를 재기 전에 줄자가 제대로 놓였는지 보는 것과 같습니다.

항목 확인값
행 수 10,424
열 수 12
기간 2025-01-01부터 2025-06-30까지
총매출 amount 합계 90,492,500
주문수 10,424
수량 qty 합계 17,241
결측값 총합 0

행 수는 주문 기록이 몇 줄 있는지를 뜻합니다. 이 데이터에는 10,424줄이 있고, 주문수도 10,424건입니다. 기간은 2025-01-01부터 2025-06-30까지이므로, 반년 동안의 카페 매출 흐름을 보는 자료라고 이해하면 됩니다. 결측값은 비어 있는 값이라는 뜻인데, 결측값 총합이 0이므로 이번 실습에서는 비어 있는 칸 때문에 그래프가 어긋날 가능성이 낮습니다.

주요 열은 이렇게 읽으면 됩니다.

열 이름 의미 Power BI에서 주로 쓰는 위치
date 주문일 꺾은선그래프의 X축
category 상품군 막대그래프의 X축
amount 주문금액 그래프 값, KPI 카드 값
store 지점 선택 필터 또는 범례
order_id 주문 번호 주문 건수 확인

date는 언제 팔렸는지를 보는 열입니다. 시간의 흐름을 봐야 하므로 꺾은선그래프의 X축에 놓습니다. category는 커피, 음료, 디저트처럼 무엇이 팔렸는지를 나누는 기준입니다. 항목끼리 비교해야 하므로 막대그래프의 X축에 놓습니다. amount는 주문금액이므로 그래프의 높이나 카드의 큰 숫자로 들어갑니다.

핵심 개념 1. Power BI Desktop은 보고서 제작 도구입니다

Power BI라는 이름은 여러 곳에서 나옵니다. 처음 배우는 사람은 여기서 가장 많이 헷갈립니다. 같은 이름 안에 보고서를 만드는 프로그램, 웹에 올려 공유하는 서비스, 조직에서 쓰는 유료 기능이 함께 있기 때문입니다.

구분 이번 차시에서의 의미
Power BI Desktop Windows PC에 설치해 보고서 파일을 만드는 프로그램
.pbix 파일 Power BI Desktop에서 저장하는 보고서 파일
Power BI Service 웹에 게시하고 공유하는 서비스. S18에서 공유 개념을 다룬다
Power BI Pro, Fabric, Copilot 조직 공유와 고급 기능에 필요한 유료 또는 조직 환경. 오늘은 쓰지 않는다

이번 차시에서 손으로 다룰 대상은 Power BI Desktop입니다. 내 컴퓨터에서 데이터 파일을 열고, 차트를 배치하고, 보고서 한 페이지를 만드는 데 집중합니다. 계정으로 공유하기, 웹에 게시하기, 다른 사람에게 권한을 주는 일은 오늘의 목표가 아닙니다.

Power BI Desktop은 Windows용 도구입니다. Microsoft 공식 다운로드 페이지도 Windows용 Desktop 설치를 안내하고, 강의 환경도 Windows 10이나 Windows 11 PC를 기준으로 봅니다. Mac 사용자는 억지로 우회 설치를 시도하지 않아도 됩니다. 이번 시간에는 어떤 화면에서 어떤 일을 하는지 눈으로 익히고, S18에서 Looker Studio를 사용해 같은 보고서 개념을 직접 실습합니다.

핵심 개념 2. 로드는 표를 Power BI 안으로 가져오는 일입니다

엑셀은 파일을 열면 바로 셀이 보입니다. Power BI는 조금 다릅니다. 먼저 데이터를 가져와야 하고, 이 과정을 로드라고 부릅니다. 로드는 표를 Power BI 보고서 안에서 사용할 수 있게 등록하는 일입니다.

피벗테이블을 만들 때 원본 표 범위를 먼저 지정했던 일을 떠올려 보세요. Power BI에서도 먼저 원본 데이터를 지정합니다. 다만 Power BI는 원본 표를 그대로 고치는 프로그램이라기보다, 그 표를 바탕으로 카드와 그래프가 놓인 화면을 만드는 프로그램입니다.

엑셀에서의 감각 Power BI에서의 감각
엑셀 파일 열기 데이터 가져오기
표 범위 선택 CSV 또는 엑셀 시트 선택
피벗테이블 필드 목록 데이터 패널의 필드 목록
행, 열, 값 영역 X축, Y축, 값, 범례 영역
피벗 차트 시각화 개체

오늘 다루는 카페 매출 데이터는 이미 깨끗하게 정리돼 있습니다. 그래서 Power Query에서 행을 지우거나 열을 새로 만드는 정제 작업은 하지 않습니다. 대신 date가 날짜로 인식되는지, amount가 숫자로 인식되는지만 확인합니다. 날짜가 날짜로 잡혀야 시간 흐름을 제대로 그릴 수 있고, 금액이 숫자로 잡혀야 합계를 낼 수 있습니다.

CSV 파일로 불러와도 되고, 초보자는 같은 내용의 엑셀 파일(.xlsx)을 받아서 실습해도 됩니다. 다만 한 보고서 안에 둘을 동시에 넣으면 같은 데이터가 두 번 들어간 것처럼 헷갈릴 수 있으니, 수업에서는 하나를 기준으로 진행합니다.

핵심 개념 3. 시각화는 필드를 놓는 방식으로 만듭니다

Power BI 화면 가운데의 흰 영역이 캔버스입니다. 캔버스는 보고서 종이라고 보면 됩니다. 카드, 막대그래프, 꺾은선그래프 같은 시각화 개체를 이 종이 위에 올려놓습니다. 오른쪽에는 시각화 패널과 데이터 패널이 있습니다. 시각화 패널에서는 그래프 모양을 고르고, 데이터 패널에서는 표의 열을 골라 그래프에 넣습니다.

시각화를 만들 때는 세 가지를 차례로 생각해 봅니다.

  1. 무엇을 비교할 것인가: category, store, date
  2. 어떤 숫자를 볼 것인가: amount, qty, order_id
  3. 어떤 모양이 적절한가: 카드, 막대, 꺾은선

막대그래프는 항목을 나란히 견줄 때 좋습니다. 카테고리별 매출은 커피, 음료, 디저트처럼 항목을 나눠 비교하는 일이므로 막대그래프에 잘 맞습니다. 이때 category마다 amount 합계를 세워서 높이를 비교합니다.

시간 흐름에는 꺾은선그래프가 어울립니다. 일자별 매출이라면 date 순서대로 amount 합계를 이어 봅니다. 선이 올라가면 매출이 늘어난 날이고, 선이 내려가면 매출이 줄어든 날입니다. 날짜를 따라 체온을 기록하면 몸 상태의 흐름이 보이는 것과 같은 감각입니다.

KPI 카드는 핵심 숫자 하나를 크게 보여주는 자리입니다. KPI는 Key Performance Indicator의 줄임말로, 꼭 봐야 하는 대표 지표라는 뜻입니다. 이번 보고서에서는 총매출이 그 역할을 합니다. 여러 그래프를 보기 전에 “그래서 전체 매출이 얼마인가요?”라는 질문에 바로 답해 주는 숫자입니다.

핵심 개념 4. KPI 카드는 큰 셀 하나라고 생각합니다

엑셀에서 =SUM(매출범위)를 계산한 뒤, 그 셀을 크게 키워 보고서 맨 위에 놓으면 보는 사람이 제일 먼저 총매출을 확인할 수 있습니다. Power BI의 카드도 같은 역할을 합니다. 많은 숫자 중에서 가장 먼저 봐야 하는 숫자 하나를 크게 보여줍니다.

이번 미션은 amount 합계를 카드로 띄우는 것입니다. 맞는 값은 90,492,500입니다. 이 숫자는 카페 매출 파일의 모든 주문금액을 더한 총매출입니다. 한 주문씩 영수증 금액을 전부 더한 값이라고 생각하면 됩니다.

Power BI는 기본 설정에서 큰 숫자를 90M이나 90.49M처럼 줄여 보여줄 때가 있습니다. 여기서 M은 million, 즉 백만 단위입니다. 화면을 간단하게 보이게 하려는 기능이지만, 처음 검산할 때는 오히려 헷갈립니다. 정확히 맞는지 확인하려면 표시 단위를 없음(None)으로 바꾸고 소수 자릿수를 0으로 맞춥니다. 그러면 총매출을 원래 숫자 그대로 확인할 수 있습니다.

앞부분에서는 Power BI가 어떤 도구인지 보고, 중간에는 데이터를 가져와 그래프를 만듭니다. 후반에는 총매출 카드를 직접 만들면서 값이 맞는지 확인합니다. 여기서 중요한 것은 예쁜 화면을 꾸미는 일이 아닙니다. 어떤 열을 어디에 놓으면 어떤 질문에 답할 수 있는지를 익히는 일입니다.

엑셀과 연결해서 설명할 지점

  • Power BI의 데이터 패널은 피벗테이블 필드 목록과 비슷하다.
  • 막대그래프의 X축은 피벗테이블의 행 영역처럼 비교 기준을 놓는 자리이다.
  • 막대그래프의 Y축 또는 값 영역은 피벗테이블의 값 영역처럼 합계나 개수를 놓는 자리이다.
  • KPI 카드는 엑셀에서 SUM 결과를 크게 보여주는 셀과 비슷하다.
  • Power BI는 화면을 만드는 도구이므로, 원본 데이터를 고치는 일은 엑셀 정제, Power Query, 파이썬과 역할을 나누어 생각한다.

엑셀을 이미 써 봤다면 Power BI가 완전히 낯선 도구는 아닙니다. 피벗테이블에서 행 영역에 기준을 놓고, 값 영역에 숫자를 놓았던 경험을 떠올리면 됩니다. Power BI에서는 그 자리가 X축, Y축, 값, 범례 같은 이름으로 바뀌어 있을 뿐입니다.

다만 Power BI는 원본 데이터를 고치는 데 초점을 둔 도구가 아닙니다. 원본을 정리하는 일은 엑셀 정제, Power Query, 파이썬이 더 잘 맡습니다. Power BI는 정리된 데이터를 사람에게 보여 주는 보고 화면을 만드는 쪽에 가깝습니다. 재료 손질과 접시 담기가 다른 일인 것처럼 역할을 나누어 생각하면 쉽습니다.

수업 후 수강생이 말할 수 있어야 하는 문장

  • "Power BI Desktop은 Windows에서 보고서를 만드는 프로그램이고, Mac은 Looker Studio로 대체한다."
  • "CSV와 엑셀 파일은 데이터 가져오기 메뉴에서 불러온다."
  • "막대그래프는 항목 비교, 꺾은선그래프는 시간 흐름, 카드는 핵심 숫자에 쓴다."
  • "amount를 카드에 넣고 합계로 요약하면 총매출을 볼 수 있다."

이 문장들을 말할 수 있으면 화면에서 무엇을 누르는지만 외운 것이 아니라, 왜 그 행동을 하는지도 이해한 상태입니다. amount를 카드에 넣는 이유는 주문금액을 모두 더해 총매출을 보기 위해서입니다. category를 막대그래프에 놓는 이유는 상품군별로 매출을 비교하기 위해서입니다. date를 꺾은선그래프에 놓는 이유는 날짜가 지나며 매출이 어떻게 움직였는지 보기 위해서입니다.

완료 판단

1단계. 설치 확인

  4. 프로그램이 없으면 강사의 안내에 따라 Microsoft Store 또는 Microsoft 공식 다운로드 페이지에서 설치합니다.
    Power BI Desktop은 설치형 프로그램이라서, PC에 없으면 보고서 파일을 만들 수 없습니다.

처음 실행하면 로그인 화면이 뜰 수 있습니다. 오늘은 로컬 파일로 보고서를 만드는 실습이니, 강사 안내에 맞춰 닫거나 건너뛰면 됩니다. 조직 PC라면 보안 정책 때문에 설치나 로그인이 막히기도 합니다. 이럴 때는 프로그램 문제가 아니라 회사나 학교 PC의 설정 문제일 수 있습니다.

따라하기 2: CSV 파일 불러오기

1단계. 데이터 가져오기

  1. 위쪽 메뉴에서 홈(Home)을 누릅니다.
    Power BI에서 데이터를 불러오는 명령은 주로 홈 메뉴에 모여 있습니다.
  2. 데이터 가져오기(Get data)를 누릅니다.
    이 버튼은 Power BI에게 “이제 분석할 표를 가져오겠다”라고 알려 주는 입구입니다.
  3. 텍스트/CSV(Text/CSV)를 선택합니다.
    카페 매출 CSV 파일은 행과 열로 된 표 파일이므로 이 선택지를 사용합니다.
  4. 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 CSV 파일을 선택합니다.
    오늘 사용할 매출 표를 정확히 골라야 뒤에서 총매출과 그래프 값이 맞습니다.
  5. 미리보기 화면이 열리면 열 이름을 확인합니다.
    열 이름을 보는 이유는 잘못된 파일을 골랐는지 바로 알아차리기 위해서입니다.
  6. 로드(Load)를 누릅니다.
    로드를 눌러야 표가 Power BI 안으로 들어오고, 오른쪽 데이터 패널에서 열을 끌어다 쓸 수 있습니다.

2단계. 데이터 패널 확인

오른쪽 데이터 패널에 cafe_sales_clean 표가 생겼는지 확인합니다. 표를 펼쳐 아래 열이 보이면 정상입니다. 사전에서 낱말을 찾듯이, 이 패널에서 우리가 그래프에 넣을 열을 고르게 됩니다.

확인할 것:

  • amount는 숫자 열이어야 합니다. 매출을 더해야 하므로 글자가 아니라 숫자로 인식되어야 합니다.
  • date는 날짜 열로 인식되는 것이 좋습니다. 날짜가 제대로 잡혀야 일자별 흐름을 선으로 볼 수 있습니다.
  • 오늘은 정제 실습이 아니므로 복잡한 변환은 하지 않습니다. 이미 정리된 표를 Power BI에 올려 첫 보고 화면을 만드는 데 집중합니다.

date가 날짜로 인식되지 않으면, 강사 화면을 따라 변환 데이터(Transform data)에서 형식을 날짜로 바꾼 뒤 닫기 및 적용(Close & Apply)을 누릅니다. 날짜 형식을 맞추는 일은 달력 자를 대고 순서를 맞추는 작업이라고 생각하면 됩니다.

따라하기 3: 엑셀 파일 로드 경로 확인하기

이 단계는 엑셀 파일을 불러오는 메뉴 위치만 눈에 익히는 용도입니다. 이미 CSV를 불러온 파일에는 엑셀을 덧붙이지 않습니다. 같은 표를 두 번 넣으면 결과가 두 배로 보이거나, 어떤 표를 기준으로 봐야 하는지 헷갈릴 수 있습니다.

새 Power BI 파일에서 엑셀을 불러올 때는 아래 순서를 따릅니다.

  1. 홈(Home)을 누릅니다.
    외부 데이터를 가져오는 시작점은 CSV와 마찬가지로 홈 메뉴입니다.
  2. 데이터 가져오기(Get data)를 누릅니다.
    Power BI에 새 표를 연결하겠다는 뜻입니다.
  3. Excel 통합 문서(Excel workbook)를 선택합니다.
    엑셀 파일은 CSV와 가져오는 입구가 다르기 때문에 이 항목을 고릅니다.
  4. 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 엑셀 파일을 선택합니다.
    CSV와 같은 내용을 담은 엑셀 파일로 메뉴 흐름만 확인합니다.
  5. 탐색기(Navigator)에서 sales 시트를 선택합니다.
    엑셀 파일 안에는 여러 시트가 있을 수 있으므로, 어떤 시트를 표로 가져올지 정해야 합니다.
  6. 로드(Load)를 누릅니다.
    이 버튼을 눌러야 선택한 시트가 Power BI의 데이터 패널에 들어옵니다.

CSV와 엑셀은 같은 데이터입니다. 실습 결과를 맞추기 위해, 이후 단계는 CSV로 불러온 cafe_sales_clean 표를 기준으로 삼습니다.

따라하기 4: 카테고리별 매출 막대그래프 만들기

1단계. 막대그래프 추가

  1. 시각화 패널에서 묶은 세로 막대형 차트(Clustered column chart)를 선택합니다.
    막대그래프는 항목끼리 크기를 비교할 때 좋습니다.
  2. 캔버스에 빈 막대그래프가 생기면 오른쪽 필드 영역을 봅니다.
    빈 차트는 아직 재료가 올라가지 않은 접시와 같습니다. 이제 기준과 숫자를 넣어야 합니다.
  3. 데이터 패널에서 category를 X축(X-axis)으로 끌어다 놓습니다.
    category는 커피, 음료, 디저트처럼 무엇끼리 비교할지 정하는 기준입니다.
  4. 데이터 패널에서 amount를 Y축(Y-axis) 또는 값(Values)으로 끌어다 놓습니다.
    amount는 매출 금액이므로, 막대의 높이를 만드는 숫자입니다.

2단계. 집계 방식 확인

amount가 합계(Sum)로 들어갔는지 확인합니다. 매출 비교는 “얼마나 팔렸는가”를 보는 일이므로 합계가 맞습니다. 평균이나 개수로 잡히면 질문이 달라집니다. 예를 들어 평균은 주문 한 번당 금액을 보는 것이고, 개수는 몇 건인지를 세는 것입니다.

날짜 계층 구조는 연도, 분기, 월처럼 큰 단위로 접어 보는 방식입니다. 오늘은 하루하루의 매출 흐름을 보는 것이 목적이므로, 접힌 달력 대신 날짜 그대로를 축에 놓는다고 생각하면 됩니다.

Power BI Desktop은 Windows에서 보고서를 만드는 프로그램입니다. 오늘은 공유하거나 웹에 게시하는 단계까지 가지 않습니다. 내 PC에서 첫 보고서 파일을 만들고, 차트가 어떻게 만들어지는지만 확실히 봅니다. Mac 사용자는 설치를 억지로 따라오지 않아도 됩니다. 화면 구조와 용어만 익혀 두면, S18에서 Looker Studio로 같은 개념을 직접 실습합니다.

엑셀 피벗테이블로 치면 category를 행에 넣고, amount를 값에 넣은 뒤 차트로 보는 것과 같습니다. 여기서 중요한 점은 amount가 합계로 들어가야 한다는 것입니다. 평균이나 개수로 들어가면 우리가 보려는 매출 합계가 아닙니다.

48-56분. 직접 해보기: 총매출 KPI 카드

미션 안내:

이 카드에 들어갈 숫자는 amount의 합계입니다. 오늘 맞춰야 할 값은 90,492,500입니다. 이 숫자가 맞으면 데이터를 제대로 불러왔고, 집계 방식도 맞게 잡은 것입니다.

수강생 수행:

  1. 캔버스 빈 곳을 클릭한다.
  2. 카드(Card) 또는 카드(새)(Card new)를 선택한다.
  3. amount를 값 영역에 넣는다.
  4. 집계 방식이 합계인지 확인한다.
  5. 제목을 총매출 또는 총매출(원)으로 바꾼다.
  6. 표시 단위를 없음(None)으로 바꾼다.
  7. 소수 자릿수를 0으로 맞춘다.

기대 결과:

카드 값이 90,492,500이면 맞다.

돌아다니며 볼 것:

  • 수강생이 amount 대신 order_id를 넣지 않았는지 확인한다.
  • 카드 값이 10,424로 보이면 주문 건수를 넣었을 가능성이 높다.
  • 카드 값이 90.49M으로 보이면 표시 단위만 바꾸면 된다.
  • 값이 평균으로 들어가면 약 8,681 수준으로 보일 수 있다. 집계를 합계로 바꾸게 한다.

완료 판단 체크포인트:

수강생이 총매출 카드에 90,492,500을 띄우고, 같은 페이지에 카테고리별 매출 막대그래프와 일자별 매출 꺾은선그래프까지 배치했다면 이 차시는 완료로 본다.

예상 질문과 답변

Q1. Power BI Desktop은 꼭 설치해야 하나요?

A. Windows에서 직접 실습하려면 필요합니다. Power BI Desktop은 Windows용 보고서 제작 프로그램입니다. Mac 수강생은 설치하지 않고 화면 구조와 필드 배치 개념을 따라오면 됩니다. 직접 실습은 S18의 Looker Studio에서 이어 갑니다.

Q2. Power BI와 엑셀 차트는 무엇이 다른가요?

A. 엑셀 차트는 파일 안에서 빠르게 그려 보기 좋습니다. Power BI는 여러 시각화를 한 페이지에 모으고, 필터와 상호작용을 얹어 반복해서 보는 보고서에 강합니다. 엑셀은 계산하고 확인하는 작업장에 가깝고, Power BI는 다른 사람이 계속 들여다보는 보고 화면에 가깝다고 보면 됩니다.

Q3. 로그인해야 하나요?

A. 오늘 실습은 수업 홈의 데이터 받기 페이지에서 받은 카페 매출 파일을 불러와 Desktop에서 보고서를 만드는 과정입니다. 공유나 게시를 하지 않으니 조직 계정 기능은 건드리지 않습니다. 다만 설치 방식이나 회사 PC 정책에 따라 로그인 화면이 뜰 수는 있습니다.

Q4. CSV와 엑셀 중 무엇을 써야 하나요?

Q5. 카드 숫자가 정답보다 작게 보입니다.

검증용 Python 코드

S15는 Power BI 차시이므로 별도 S15_code.py 파일을 제출할 필요는 없습니다. 아래 코드는 모범답안의 숫자가 실제 데이터에서 맞는지 다시 계산해 본 강사용 검산 코드입니다. 학생 실습에서는 이 코드를 따라 치는 것이 목표가 아닙니다. Power BI 화면에서 같은 값이 나오는지 확인하면 됩니다.

from pathlib import Path
import pandas as pd

path = Path("/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.csv")
df = pd.read_csv(path)

print("shape:", df.shape)
print("date_min:", df["date"].min())
print("date_max:", df["date"].max())
print("total_amount:", int(df["amount"].sum()))
print("order_count:", len(df))
print("null_total:", int(df.isna().sum().sum()))

category_summary = (
    df.groupby("category", as_index=False)["amount"]
    .sum()
    .sort_values("amount", ascending=False)
)
print(category_summary.to_string(index=False))

df["date_dt"] = pd.to_datetime(df["date"])
monthly = df.groupby(df["date_dt"].dt.to_period("M"))["amount"].sum().reset_index()
monthly["date_dt"] = monthly["date_dt"].astype(str)
print(monthly.to_string(index=False))
