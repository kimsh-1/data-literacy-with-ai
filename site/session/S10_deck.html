Colab과 파이썬 기초

엑셀에서 파이썬으로 넘어가는 첫 수업으로, 다음 차시부터 쓸 pandas 문법이 낯설지 않도록 기본 언어 감각을 익힌다.

오늘 끝나면 할 수 있는 것

  • Colab 노트북을 만들고 코드 셀을 실행할 수 있다.
  • 변수, 자료형, 리스트, 딕셔너리의 역할을 엑셀 표와 비교해 설명할 수 있다.
  • if 조건문으로 원하는 경우만 골라낼 수 있다.
  • for 반복문으로 여러 값을 하나씩 처리하고 합계와 평균을 계산할 수 있다.
  • 함수는 자주 쓰는 절차에 이름을 붙이는 것이라고 이해할 수 있다.
핵심 개념 1

Colab은 브라우저에서 쓰는 파이썬 작업장이다

Colab은 구글 계정으로 사용하는 온라인 노트북이다. 엑셀 파일이 셀 격자로 되어 있다면, Colab 노트북은 코드 셀과 텍스트 셀로 되어 있다.

  • 코드 셀을 실행하면 결과가 셀 아래에 표시된다.
  • 위 셀에서 만든 변수는 아래 셀에서 다시 쓸 수 있다.
  • 위 셀을 실행하지 않았거나 런타임을 재시작했다면 아래 셀에서 그 변수를 찾지 못한다.

핵심 개념 2

변수와 자료형은 값의 이름과 종류를 구분한다

변수는 값에 붙이는 이름이다. 엑셀에서 B2 같은 셀 주소가 값을 가리키듯, 파이썬에서는 today_sales처럼 사람이 읽을 수 있는 이름을 붙인다.

  • 등호 =는 오른쪽 값을 왼쪽 이름에 담는다는 뜻이다.
  • 같은지 비교할 때는 ==를 쓴다.
  • 자료형은 값의 종류이며 str, int, float, bool을 구분한다.
  • 계산이 안 될 때는 숫자인 줄 알았는데 글자였는지부터 확인한다.
핵심 개념 3

리스트는 한 열, 딕셔너리는 한 행처럼 생각한다

리스트는 여러 값을 순서대로 담는 상자이고, 딕셔너리는 값마다 이름표가 붙어 있는 구조다. 리스트 안에 딕셔너리를 여러 개 넣으면 작은 표처럼 다룰 수 있다.

  • sales_list는 월요일부터 금요일까지 매출 5개를 담은 리스트다.
  • len()은 리스트 안의 값이 몇 개인지 세는 함수다.
  • one_day는 요일과 매출을 담은 한 행처럼 볼 수 있다.
  • 이 구조가 다음 차시의 pandas DataFrame으로 이어진다.
핵심 개념 4

조건문과 반복문은 필터와 아래로 채우기의 출발점이다

조건문은 원하는 경우만 실행하게 하는 문법이고, 반복문은 같은 작업을 여러 값에 적용하는 방법이다. 엑셀 필터와 아래로 채우기에 빗대어 이해할 수 있다.

  • if 뒤에는 조건을 쓰고 조건 끝에는 콜론을 붙인다.
  • if나 for 아래에서 실행할 코드는 한 단계 안쪽으로 들여쓴다.
  • for amount in sales_list는 리스트에서 값을 하나씩 꺼내 이번 값을 amount라고 부른다는 뜻이다.
  • pandas 필터는 if 조건문을 표 전체 열에 한 번에 적용하는 방식이다.
핵심 개념 5

함수는 자주 쓰는 절차에 이름을 붙이는 것이다

이번 차시에서 함수는 깊게 다루지 않는다. 다만 다음 차시부터 read_csv(), head(), mean() 같은 함수를 계속 만나므로, 정해진 일을 해주는 도구라는 감각을 잡는다.

  • 평균은 합계를 개수로 나눈 값이다.
  • 엑셀 AVERAGE 함수가 하던 일을 파이썬 코드로 손으로 써 본다.
  • average(numbers)는 평균 계산 절차에 이름을 붙인 예다.

오류가 났을 때 먼저 볼 것

오류흔한 원인확인할 것

정리

  • Colab은 코드 셀을 위에서 아래로 실행하는 브라우저 기반 파이썬 작업장이다.
  • 변수는 값을 다시 쓰기 위해 붙이는 이름이고, 자료형은 값의 종류다.
  • 리스트는 여러 값을 순서대로 담고, 딕셔너리는 이름표가 붙은 값을 담는다.
  • 반복문은 값을 하나씩 처리하고, 조건문은 원하는 경우만 남긴다.
  • 함수는 자주 쓰는 절차에 이름을 붙여 다시 쓰는 방법이다.
