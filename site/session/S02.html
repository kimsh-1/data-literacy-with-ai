S02. 좋은 데이터의 모양: tidy 데이터와 척도

차시 개요

  • 대상: 엑셀은 사용하지만 데이터 분석은 처음인 비개발자
  • 선수 차시: S01
  • 난이도: 1
  • 시간: 40분
  • 사용 데이터:
  • /mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.xlsx
  • /mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_dirty.xlsx
  • 확인한 데이터 규모:
  • clean: 10,424행, 12열
  • dirty: 10,439행, 12열

학습 목표

  1. 분석하기 좋은 표의 기본 모양을 설명할 수 있다.
  2. 한 셀 한 값, 한 행 한 관찰, 한 열 한 변수의 의미를 엑셀 표에서 찾을 수 있다.
  3. 병합셀, 빈 셀, 표기 혼재, 숫자와 문자의 혼합이 분석을 방해하는 이유를 이해한다.
  4. 컬럼별로 범주형, 순서형, 수치형, 날짜형, 논리형 척도를 구분할 수 있다.
  5. dirty 데이터에서 tidy 원칙을 어기는 지점을 찾아 교정 방향을 말할 수 있다.

핵심 개념

1. 분석용 표는 보기 좋은 보고서와 다르다

엑셀에서 사람에게 보여주기 좋은 표는 제목이 크고, 셀이 병합되어 있고, 중간중간 소계와 설명이 들어가는 경우가 많다. 이런 표는 회의 자료로는 읽기 편하지만, 피벗테이블이나 AI 분석, 파이썬, BI 도구로 넘기면 오히려 걸림돌이 된다.

분석용 표는 장식보다 규칙이 중요하다. 이번 차시에서 사용할 카페 매출 데이터는 주문 한 건이 한 행이다. date, store, item, qty, amount처럼 각 열은 주문을 설명하는 한 가지 속성만 담는다. 이 구조가 유지되어야 필터, 정렬, 피벗, 집계가 안정적으로 작동한다.

2. tidy 데이터의 세 가지 규칙

tidy 데이터는 분석하기 좋은 표의 기본형이다. 입문 단계에서는 다음 세 문장으로 기억하면 충분하다.

규칙 뜻 엑셀에서 깨지는 예
한 행은 한 관찰이다 한 줄이 주문 한 건, 고객 한 명, 직원 한 명처럼 같은 단위여야 한다 중간에 월별 소계 행이 끼어 있음
한 열은 한 변수다 한 열에는 날짜, 지점, 상품명, 수량처럼 하나의 의미만 있어야 한다 상품/수량처럼 한 열에 두 의미가 섞임
한 셀은 한 값이다 셀 하나에는 하나의 값만 들어가야 한다 4,500원처럼 숫자와 단위가 섞임, 강남점/카드처럼 값 두 개가 한 셀에 있음

병합셀은 이 원칙을 자주 무너뜨린다. 사람 눈에는 위아래가 같은 그룹처럼 보이지만, 컴퓨터는 병합된 첫 셀만 값이 있고 나머지 셀은 빈 칸으로 처리하는 경우가 많다. 그래서 분석용 원본 표에는 병합셀을 쓰지 않는 것이 원칙이다.

3. clean 데이터와 dirty 데이터 비교

이번 차시의 clean 데이터는 분석용 표의 기준 예시다. dirty 데이터는 같은 카페 매출 데이터에 실무에서 자주 나오는 문제를 일부러 심어 둔 파일이다.

clean 데이터의 컬럼은 다음과 같다.

컬럼 의미
order_id 주문번호
date 주문일
weekday 요일
hour 주문 시간대
store 지점
category 상품 대분류
item 상품명
unit_price 단가
qty 수량
amount 주문금액
payment 결제수단
member 회원 여부

dirty 데이터에서 확인한 대표 문제는 다음과 같다.

문제 유형 실제 예 왜 문제인가
주문번호 중복 같은 order_id가 다시 등장 주문 건수가 부풀 수 있음
결측 payment, member가 비어 있음 결제수단별, 회원 여부별 집계가 흔들림
날짜 형식 혼합 2025-03-23, 2025/03/23, 03-23-2025 월별, 요일별 분석 전 날짜 인식이 불안정함
숫자와 문자 혼합 unit_price에 4,500원 형태 포함 합계와 평균 계산 전에 숫자로 바꿔야 함
범주 표기 혼재 강남점과 강남 점, 아메리카노와 Americano 같은 대상을 다른 그룹으로 집계함
앞뒤 공백 커피, 음료 눈에는 같아 보여도 컴퓨터는 다른 값으로 볼 수 있음
이상치 qty가 999 평균과 총합을 크게 왜곡함
불가능한 값 amount가 음수 매출 데이터의 기본 규칙과 맞지 않음

4. 척도는 분석 방법을 고르는 기준이다

데이터 분석에서 컬럼을 볼 때는 값의 모양뿐 아니라 “이 값으로 무엇을 할 수 있는가”를 생각해야 한다. 이것이 척도 구분이다.

척도 설명 가능한 작업 예
범주형 이름표처럼 그룹을 구분한다 개수 세기, 그룹별 합계 지점, 상품명, 결제수단
순서형 순서가 있지만 간격이 숫자처럼 정확하지 않을 수 있다 정렬, 단계별 비교 만족도, 등급
수치형 더하기, 평균, 최솟값, 최댓값 계산이 가능하다 합계, 평균, 분포 확인 수량, 단가, 주문금액
날짜형 시간의 흐름을 나타낸다 일별, 월별, 요일별 추세 주문일
논리형 참과 거짓 또는 예와 아니오 비율 계산, 필터링 회원 여부
식별자 각 행을 구분하는 이름표다 중복 확인, 추적 주문번호

주의할 점은 숫자로 보인다고 모두 수치형은 아니라는 것이다. order_id는 숫자로 저장되어 있지만 평균 주문번호를 구해도 의미가 없다. 그러니 주문번호는 계산용 숫자가 아니라 식별자로 봐야 한다.

수업 흐름

시간 내용 진행 방식
0~5분 S01 복습과 오늘의 질문 제시 “좋은 표와 예쁜 표는 다르다”는 문제의식 공유
5~15분 tidy 데이터 원칙 설명 주문 한 건을 한 행으로 보는 비유와 엑셀 비교
15~25분 cafe_sales_dirty.xlsx에서 문제 찾기 필터, 정렬, 값 목록 확인
25~33분 컬럼별 척도 분류 clean 데이터 컬럼을 기준으로 범주형, 수치형 등 분류
33~38분 직접 해보기 미션 수강생이 척도 분류표와 교정 방향 작성
38~40분 정리와 다음 차시 연결 S03 피벗테이블에서 왜 tidy가 필요한지 연결

다음 차시와의 연결

S03에서는 피벗테이블로 지점별, 카테고리별, 요일별 매출 질문에 답한다. 피벗테이블이 제대로 작동하려면 이번 차시에서 다룬 tidy 구조가 먼저 필요하다. 같은 상품명이 여러 표기로 흩어져 있거나, 숫자 열에 문자가 섞여 있으면 피벗 결과가 맞아 보여도 실제로는 틀릴 수 있다.

S02 핸드아웃. 좋은 데이터의 모양

오늘 사용할 파일

아래 두 파일을 사용한다.

  • /mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.xlsx
  • /mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_dirty.xlsx

두 파일 모두 sales 시트가 있으며 컬럼은 12개다. clean 파일은 10,424행, dirty 파일은 10,439행이다. 엑셀에서는 머리글 행까지 포함되므로 화면상 마지막 행 번호는 각각 10,425와 10,440으로 보일 수 있다.

따라하기 1. 분석하기 좋은 표 모양 확인

  1. cafe_sales_clean.xlsx를 연다.
  2. sales 시트를 확인한다.
  3. 1행의 머리글을 왼쪽부터 읽는다. - order_id, date, weekday, hour, store, category, item, unit_price, qty, amount, payment, member
  4. 2행을 가로로 읽는다. - 한 행이 주문 한 건을 설명하는지 확인한다.
  5. 각 열을 세로로 훑어본다. - date에는 날짜만 있다. - store에는 지점명만 있다. - qty에는 수량만 있다. - amount에는 주문금액만 있다.

정리: 이 표는 “주문 한 건이 한 행, 주문의 속성이 각 열”인 구조다. 이런 구조를 피벗테이블과 분석 도구가 반긴다.

따라하기 2. dirty 표에서 문제 찾기

  1. cafe_sales_dirty.xlsx를 연다.
  2. sales 시트에서 1행 아무 셀이나 클릭한다.
  3. 엑셀 상단 메뉴에서 데이터를 누른다.
  4. 필터를 클릭한다.
  5. 각 컬럼의 필터 버튼을 눌러 값 목록을 확인한다.

2-1. 지점 표기 문제 찾기

  1. store 컬럼의 필터 버튼을 누른다.
  2. 값 목록에서 다음 값이 함께 있는지 확인한다. - 강남점 - 강남 점 - 홍대점
  3. 강남점과 강남 점은 같은 지점으로 보아야 한다.

기록할 내용: store는 범주형 컬럼이지만 같은 범주의 표기가 섞여 있다.

2-2. 상품명 표기 문제 찾기

  1. item 컬럼의 필터 버튼을 누른다.
  2. 검색창에 아메를 입력한다.
  3. 다음처럼 비슷하지만 다른 값이 있는지 확인한다. - 아메리카노 - 아메리카 - 아메 리카노
  4. 필터 검색창에 Americano도 입력해 본다.

기록할 내용: 같은 상품이 여러 이름으로 기록되면 상품별 매출이 나뉘어 계산된다.

2-3. 카테고리 앞뒤 공백 찾기

  1. category 컬럼의 필터 버튼을 누른다.
  2. 값 목록에서 커피와 커피처럼 앞뒤 공백이 있는 값이 따로 보이는지 확인한다.
  3. 음료, 디저트도 같은 방식으로 확인한다.

기록할 내용: 사람 눈에는 거의 같아도 컴퓨터는 공백이 있으면 다른 값으로 처리할 수 있다.

2-4. 단가 컬럼의 숫자와 문자 혼합 찾기

  1. unit_price 컬럼의 필터 버튼을 누른다.
  2. 검색창에 원을 입력한다.
  3. 4,500원, 5,500원처럼 숫자와 문자가 함께 들어간 값을 확인한다.

기록할 내용: 단가 열은 계산에 쓰는 수치형이어야 하므로 원과 쉼표를 제거하고 숫자로 맞춰야 한다.

2-5. 날짜 형식 혼합 찾기

  1. date 컬럼의 값을 훑어본다.
  2. 다음처럼 날짜 형식이 섞여 있는지 확인한다. - 2025-03-23 - 2025/03/23 - 03-23-2025

기록할 내용: 날짜가 여러 형식으로 섞이면 월별 추세나 요일 분석 전에 날짜형으로 통일해야 한다.

2-6. 결측값 찾기

  1. payment 컬럼의 필터 버튼을 누른다.
  2. 값 목록에서 (빈 셀) 또는 비어 있는 항목을 선택한다.
  3. 결제수단이 비어 있는 행을 확인한다.
  4. 필터를 해제한다.
  5. member 컬럼에서도 같은 방식으로 빈 셀을 확인한다.

기록할 내용: 비어 있는 값은 무시할지, 별도 범주로 둘지, 원자료를 확인해 채울지 결정해야 한다.

2-7. 수량 이상치 찾기

  1. qty 컬럼의 필터 버튼을 누른다.
  2. 값 목록에서 999가 있는지 확인한다.
  3. 999만 선택해 해당 행을 본다.

기록할 내용: 카페 주문 수량이 999인 값은 입력 오류일 가능성이 높다. 평균과 총합을 왜곡할 수 있다.

2-8. 음수 금액 찾기

  1. amount 컬럼의 필터 버튼을 누른다.
  2. 숫자 필터에서 보다 작음을 선택한다.
  3. 기준값에 0을 입력한다.
  4. 음수 금액 행이 있는지 확인한다.

기록할 내용: 환불 데이터라는 별도 설명이 없다면 매출 금액이 음수인 행은 확인이 필요하다.

예상 질문과 답변

질문 1. 병합셀을 이미 많이 쓰고 있는데 전부 없애야 하나요?

답변:

보고용 시트와 분석용 원본 시트를 분리하면 된다. 보고용 시트에서는 병합셀을 사용할 수 있지만, 피벗테이블이나 분석에 연결할 원본 시트에서는 병합셀을 피하는 것이 좋다.

질문 2. 4,500원처럼 쓰면 엑셀에서도 계산되는 경우가 있는데 왜 문제인가요?

답변:

엑셀 화면에서는 계산되는 것처럼 보여도 다른 도구로 옮기거나 CSV로 저장하면 문자로 처리될 수 있다. 분석용 원본에는 숫자만 저장하고, 원 표시는 셀 서식으로 처리하는 편이 안전하다.

질문 3. 결측값은 그냥 삭제하면 되나요?

답변:

항상 삭제하는 것은 아니다. 결측이 적고 분석에 큰 영향이 없으면 삭제할 수 있지만, 특정 지점이나 특정 결제수단에서만 결측이 많다면 중요한 신호일 수 있다. 삭제, 대체, 별도 범주 처리, 원자료 확인 중에서 목적에 맞게 결정한다.

질문 4. weekday는 범주형인가요, 순서형인가요?

답변:

요일 이름은 그룹을 나누는 값이므로 범주형이다. 다만 월요일부터 일요일까지 자연스러운 순서가 있으므로 “순서가 있는 범주형”으로 다룰 수 있다. 평균 요일을 계산하는 수치형은 아니다.

질문 5. member는 참과 거짓인데 범주형인가요?

답변:

회원과 비회원 두 그룹을 나누므로 범주형으로 볼 수 있고, 값이 True와 False 두 개뿐이므로 논리형으로도 부른다. 분석에서는 회원 비율, 회원 여부별 매출 비교에 자주 쓴다.
