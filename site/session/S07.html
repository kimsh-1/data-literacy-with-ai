S07. 엑셀을 AI에 올려 자연어로 분석

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S03
  • 난이도: 2
  • 시간: 50분
  • 데이터: /mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_dirty.xlsx
  • 보조 검증 파일: /mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_dirty.csv
  • 시트: sales
  • 확인된 데이터 규모: 10,439행 × 12열

이번 차시의 핵심은 "AI에게 파일을 맡기면 분석이 끝난다"가 아니다. 엑셀 파일을 업로드한 뒤 AI에게 무엇을 어떻게 물어야 하는지, 그 방법을 익히는 것이 목표다. AI는 피벗테이블과 차트, 요약문을 순식간에 만들어 주지만, 데이터 품질을 확인하지 않고 집계 기준도 정하지 않으면 그럴듯하지만 틀린 답을 내놓기 쉽다.

S03에서 수강생은 피벗테이블로 store, category, hour, amount를 요약했다. 이번에는 같은 요약을 자연어로 요청한다. 대신 이번 파일은 일부러 지저분하게 만든 dirty 데이터라, AI가 내놓은 답을 그대로 옮겨 적기보다 어떤 전처리 기준을 적용했는지 짚어 봐야 한다.

학습 목표

수강생은 이 차시가 끝나면 다음을 할 수 있다.

  1. 엑셀 파일을 AI 분석 도구에 업로드하고, 분석 대상 시트를 명확히 지정할 수 있다.
  2. 자연어로 데이터 요약, 월별 집계, 추세, 이상치 확인을 요청할 수 있다.
  3. AI가 만든 표와 차트를 받을 때 집계 기준과 제외 기준을 함께 확인할 수 있다.
  4. 두 AI 답안을 비교해 더 믿을 만한 답을 고르고, 그 이유를 설명할 수 있다.

사용할 AI 도구

표준 도구로는 파일 업로드와 데이터 분석 기능을 갖춘 ChatGPT 계정을 안내한다. 계정이나 플랜에 따라 업로드가 막힐 수 있으니 강사는 수업 전에 되는지 미리 확인해 둔다.

대안은 두 가지다.

  1. 파일 업로드가 가능한 다른 AI 도구를 사용한다.
  2. 업로드 도구가 없는 수강생은 강사가 제공한 샘플 답안과 자신의 AI 답안을 비교한다.

실무 데이터에는 개인정보나 고객 정보, 내부 매출 자료가 섞여 있기 마련이다. 이번 수업은 합성 데이터만 쓴다. 실제 회사 파일은 허가 없이 외부 AI에 올리지 않는다는 원칙도 함께 일러 둔다.

데이터 이해

사용 파일은 cafe_sales_dirty.xlsx다. 같은 내용을 담은 CSV도 있는데, 강사용 검증은 이 CSV로 했다.

주요 컬럼은 다음과 같다.

컬럼 의미 이번 차시에서 쓰는 방식
order_id 주문번호 중복 주문 확인
date 주문 날짜 월별 추세 집계
weekday 요일 보조 해석
hour 주문 시간대 필요 시 세부 분석
store 지점 표기 흔들림 확인
category 상품 카테고리 표기 공백 확인
item 상품명 표기 흔들림 확인
unit_price 단가 금액 검산
qty 판매수량 수량 이상치 확인
amount 주문금액 월별 매출 집계

실제로 확인한 데이터 품질 이슈는 다음과 같다.

항목 확인값
전체 행 수 10,439
전체 열 수 12
완전 중복 행 13
중복 order_id 추가 행 15
payment 결측 313

store 값 강남 점 310

date는 2025-01-01, 2025/01/01, 01-01-2025처럼 형식이 뒤섞여 있다. 날짜 형식이 섞여 있을 수 있다고 미리 일러 주지 않으면 AI가 일부 날짜를 놓친다.

핵심 개념

1. AI 업로드는 파일을 주는 것이고, 프롬프트는 업무 지시서다

엑셀에서는 사용자가 피벗테이블 필드를 손으로 끌어다 놓지만, AI 분석에서는 말로 지시한다. 지시가 모호하면 AI가 하는 일도 그만큼 모호해진다.

예를 들어 "월별 매출 분석해 줘"라고만 하면 AI가 중복을 어떻게 처리했는지, 음수 매출을 뺐는지, 날짜 오류를 어떻게 읽었는지 알 길이 없다. 반대로 "중복 주문번호는 한 건만 남기고, 수량 999와 음수 금액은 이상치로 따로 보여 준 뒤 월별 매출에서 제외해 달라"라고 쓰면 결과를 검토할 수 있다.

2. AI는 피벗테이블을 대신 만들 수 있지만, 기준은 사용자가 줘야 한다

S03의 피벗테이블에서 했던 일은 다음과 같다.

  • 행: 월
  • 값: amount 합계
  • 보조 값: order_id 개수, qty 합계

AI에게도 같은 말을 자연어로 해야 한다. "월별 매출 표"만 요청하지 말고, "월별 주문건수, 판매수량, 매출액, 전월대비 증감액과 증감률"처럼 필요한 열을 구체적으로 말한다.

3. 이상치는 "큰 값"만 뜻하지 않는다

입문자는 이상치를 보통 유난히 큰 매출로 생각한다. 그러나 실무에서는 다음도 이상치다.

  • 수량이 현실적으로 불가능하게 큰 값
  • 금액이 음수인데 환불 여부가 없는 값
  • 단가×수량과 주문금액이 맞지 않는 값
  • 같은 주문번호가 여러 번 나온 값
  • 날짜 형식이 섞여 월별 집계에서 빠질 수 있는 값

이번 데이터에서는 일별 매출을 IQR 기준으로 보면 통계적 일별 이상치는 없었다. 그러나 행 단위에서는 수량 999, 음수 금액, 단가×수량 불일치가 있어 반드시 별도 표시해야 한다.

4. 표와 차트는 결론이 아니라 중간 산출물이다

AI가 표와 선그래프를 만들어 주면 겉보기에는 분석이 끝난 듯 느껴진다. 하지만 표와 차트는 결론 자체가 아니라 결론을 세우기 위한 재료일 뿐이다. 강사는 수강생에게 다음을 묻는다.

  • 어떤 행을 제외했는가?
  • 어떤 날짜 형식을 인식했는가?
  • 가장 큰 달과 가장 작은 달은 어디인가?
  • 전월대비 흐름이 결론 문장과 맞는가?
  • 이상치를 표로 따로 보여 주었는가?

엑셀 기능 비교

하고 싶은 일 엑셀 방식 AI 방식
파일 열기 엑셀에서 xlsx 열기 AI 채팅창에 파일 업로드
데이터 구조 확인 첫 행, 필터, 상태 표시줄 확인 행 수, 열 수, 컬럼 목록을 요청
월별 매출 집계 날짜 열에서 월 추출 후 피벗테이블 "월별 주문건수, 판매수량, 매출액 표" 요청
이상치 확인 필터, 조건부 서식, 정렬 "수량 999, 음수 금액, 중복 주문번호를 찾아라" 요청
차트 만들기 피벗차트 또는 선그래프 삽입 "월별 매출 선그래프를 만들어라" 요청
결과 검증 원본 행과 피벗 기준 확인 AI의 전처리 기준과 수치를 재질문

AI는 작업 속도를 끌어올려 주지만 분석의 책임까지 대신 져 주지는 않는다. 입문 단계에서 익혀야 할 것은 "AI가 말한 숫자를 믿는 법"이 아니라 "AI가 그 숫자를 어떻게 만들었는지 되묻는 법"이다.

5-12분 AI 업로드와 보안 원칙 합성 데이터만 업로드, 개인정보 업로드 금지
12-22분 따라하기 1: 파일 구조와 품질 확인 행 수, 컬럼, 결측, 중복, 이상값 요청
22-35분 따라하기 2: 월별 매출 추세와 차트 요청 정밀 프롬프트로 표와 선그래프 받기
35-44분 따라하기 3: AI 답 재질문 이상치 기준, 제외 기준, 결론 문장 확인
44-49분 직접 해보기: 두 AI 답 비교 신뢰할 답 고르기와 이유 작성
49-50분 정리 AI 답은 기준과 함께 읽는다는 원칙 확인

수업 중 강조할 원칙

  1. 파일을 올리기 전에 민감 정보가 없는지 확인한다.
  2. 첫 질문은 분석이 아니라 데이터 구조 확인으로 시작한다.
  3. 추세 분석은 기간 단위를 명시한다. 이번 차시는 월별이다.
  4. 이상치는 기준을 요구한다. "이상해 보이는 값"만으로는 부족하다.
  5. AI가 만든 차트도 원본 데이터와 전처리 기준을 확인해야 한다.
  6. 두 AI 답이 다르면 더 그럴듯한 문장이 아니라 더 검증 가능한 답을 고른다.

차시 평가 기준

  • 상: 두 AI 답안을 비교하고, 행 수·전처리 기준·월별 매출 수치·이상치 목록을 근거로 더 신뢰할 답을 고른다.
  • 중: 따라하기 프롬프트를 재현하고, 월별 매출 표와 차트를 받아 주요 흐름을 말한다.
  • 하: 파일 업로드 또는 프롬프트 입력 단계에서 막혀 월별 표를 재현하지 못한다.

S07. 엑셀을 AI에 올려 자연어로 분석

사용할 파일

AI 분석 도구에 아래 파일을 업로드한다.

/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_dirty.xlsx

  • 시트 이름: sales
  • 데이터 규모: 10,439행 × 12열
  • 주요 컬럼: order_id, date, store, category, item, unit_price, qty, amount, payment, member

업로드 기능이 없는 계정이라면 강사가 나눠 준 샘플 답안과 비교하면 된다. 실제 회사 파일이나 고객 정보, 개인정보가 담긴 파일은 허가 없이 외부 AI에 올리지 않는다.

오늘의 목표

AI에게 엑셀 파일을 업로드하고 다음 세 가지를 받아 본다.

  1. 데이터 구조와 품질 점검
  2. 월별 매출 추세 표
  3. 월별 매출 선그래프와 이상치 설명

마지막에는 두 AI 답안을 비교해 어느 답을 더 믿을 수 있는지 판단한다.

따라하기 1. 파일 업로드 후 데이터 구조 확인하기

1단계. 파일 업로드

  1. 파일 업로드와 데이터 분석이 가능한 AI 채팅창을 연다.
  2. cafe_sales_dirty.xlsx 파일을 업로드한다.
  3. 업로드가 끝나면 아래 프롬프트를 그대로 입력한다.
첨부한 cafe_sales_dirty.xlsx 파일의 sales 시트를 먼저 점검해 주세요.

아직 결론을 내리지 말고 다음 항목만 표로 알려 주세요.
1. 행 수와 열 수
2. 컬럼 목록
3. 각 컬럼의 의미를 한 줄씩 추정한 설명
4. 결측값이 있는 컬럼과 결측 개수
5. 중복 order_id 개수
6. 월별 매출 분석 전에 주의해야 할 데이터 품질 이슈

2단계. 확인할 결과

AI 답에서 최소한 다음 내용이 나와야 한다.

확인 항목 기대값
행 수 10,439
열 수 12
payment 결측 313
member 결측 209
중복 order_id 추가 행 15

AI가 행 수를 말하지 않거나, 바로 매출 결론부터 말하면 다시 질문한다.

결론보다 검증이 먼저입니다. 행 수, 열 수, 결측값, 중복 order_id, 날짜 형식 문제를 먼저 확인한 뒤 다시 답해 주세요.

따라하기 2. 월별 매출 추세와 이상치 요청하기

아래 프롬프트를 그대로 입력한다. 이번 차시에서 가장 중요한 부분이다.

이제 월별 매출 추세를 분석해 주세요.

반드시 아래 기준을 적용해 주세요.
- date는 YYYY-MM-DD, YYYY/MM/DD, MM-DD-YYYY 형식이 섞여 있을 수 있으니 모두 날짜로 해석해 주세요.
- order_id는 주문 고유번호이므로 중복 order_id는 한 건만 남겨 주세요.
- unit_price에 쉼표나 '원' 문자가 붙어 있으면 숫자로 바꿔 주세요.
- amount가 0 이하인 행, qty가 20보다 큰 행, unit_price * qty와 amount가 맞지 않는 행은 이상치로 따로 보여 주고 월별 매출 집계에서는 제외해 주세요.
- payment와 member 결측은 매출 집계에서 제외하지 마세요.

결과는 다음 순서로 보여 주세요.
1. 적용한 전처리 기준 요약
2. 월별 주문건수, 판매수량, 매출액, 전월대비 증감액, 전월대비 증감률 표
3. 월별 매출 선그래프
4. 이상치로 제외한 행 목록
5. 분석 결론 3문장

2단계. AI 답에서 확인할 핵심 수치

전처리 기준을 위와 같이 적용했다면 핵심 흐름은 다음과 같아야 한다.

월 주문건수 매출액 전월대비 증감률
2025-01 1,685 14,421,000
2025-02 1,523 13,151,500 -8.8%
2025-03 1,716 14,735,500 12.0%
2025-04 1,760 15,257,500 3.5%
2025-05 1,800 16,094,000 5.5%
2025-06 1,937 16,816,500 4.5%

해석의 핵심은 다음이다.

  • 2025년 2월에 매출이 1월보다 줄었다.
  • 2025년 3월부터 6월까지는 매출이 매월 증가했다.
  • 2025년 6월 매출이 16,816,500원으로 가장 크다.

따라하기 3. 차트를 다시 요청하기

AI가 차트를 만들었더라도 축과 기준이 불분명하면 다시 요청한다.

월별 매출액을 선그래프로 다시 만들어 주세요.

차트 조건:
- x축은 2025-01부터 2025-06까지 월입니다.
- y축은 매출액 합계입니다.
- 각 점에 매출액 값을 표시해 주세요.
- 그래프 아래에 "2월 하락 후 3월부터 6월까지 증가"라는 해석을 붙여 주세요.

차트가 표와 다른 값을 가리키면 표의 전처리 기준부터 다시 확인한다. 표와 차트, 결론에 나온 숫자는 서로 어긋나선 안 된다.

따라하기 4. 이상치 설명을 요청하기

AI가 이상치를 "매출이 큰 달"만으로 설명하면 부족하다. 아래처럼 행 단위 이상치를 다시 묻는다.

또한 일별 매출 합계를 기준으로 통계적으로 유난히 높은 날이나 낮은 날이 있었는지도 확인해 주세요. 기준을 함께 설명해 주세요.

확인해야 할 행 단위 이상치는 다음 3건이다.

order_id 문제

13013 amount가 -5,500원

AI 답마다 숫자가 조금 다른 경우

전처리 기준이 달라졌을 공산이 크다. 중복을 지웠는지, 이상치를 뺐는지, 날짜 형식을 빠짐없이 읽었는지 확인한다.

AI가 "이상치 없음"이라고 말하는 경우

어떤 기준의 이상치인지 물어본다. 일별 매출 합계 기준으로는 이상치가 없을 수 있지만, 행 단위에서는 수량 999와 음수 금액이 있다.

12-22분. 시연 1: 데이터 구조부터 확인

프롬프트:

첨부한 cafe_sales_dirty.xlsx 파일의 sales 시트를 먼저 점검해 주세요.

아직 결론을 내리지 말고 다음 항목만 표로 알려 주세요.
1. 행 수와 열 수
2. 컬럼 목록
3. 각 컬럼의 의미를 한 줄씩 추정한 설명
4. 결측값이 있는 컬럼과 결측 개수
5. 중복 order_id 개수
6. 월별 매출 분석 전에 주의해야 할 데이터 품질 이슈

말할 것:

AI에게 대뜸 "결론을 말해 달라"고 하면 그럴듯한 문장부터 튀어나온다. 분석은 순서가 생명이다. 첫 질문은 언제나 이 파일이 어떤 구조이고 어디가 지저분한지부터 확인하는 것이어야 한다.

확인할 값:

  • 전체 데이터는 10,439행 × 12열이다.
  • payment 결측은 313개다.
  • member 결측은 209개다.
  • 중복 order_id 추가 행은 15개다.
  • 날짜 형식이 섞여 있다.

강조할 것:

결측이 있다고 무턱대고 행을 버리지는 않는다. 오늘의 질문은 월별 매출이다. payment와 member 결측은 매출액 계산에 직접 쓰이지 않으니 집계에서 빼지 않는다. 반면 date, order_id, qty, amount, unit_price의 문제는 월별 매출을 곧바로 흔든다.

22-35분. 시연 2: 월별 매출 추세와 차트 요청

프롬프트:

이제 월별 매출 추세를 분석해 주세요.

반드시 아래 기준을 적용해 주세요.
- date는 YYYY-MM-DD, YYYY/MM/DD, MM-DD-YYYY 형식이 섞여 있을 수 있으니 모두 날짜로 해석해 주세요.
- order_id는 주문 고유번호이므로 중복 order_id는 한 건만 남겨 주세요.
- unit_price에 쉼표나 '원' 문자가 붙어 있으면 숫자로 바꿔 주세요.
- amount가 0 이하인 행, qty가 20보다 큰 행, unit_price * qty와 amount가 맞지 않는 행은 이상치로 따로 보여 주고 월별 매출 집계에서는 제외해 주세요.
- payment와 member 결측은 매출 집계에서 제외하지 마세요.

결과는 다음 순서로 보여 주세요.
1. 적용한 전처리 기준 요약
2. 월별 주문건수, 판매수량, 매출액, 전월대비 증감액, 전월대비 증감률 표
3. 월별 매출 선그래프
4. 이상치로 제외한 행 목록
5. 분석 결론 3문장

시연 포인트:

  1. AI가 전처리 기준을 먼저 말하는지 확인한다.
  2. 월별 표에 2025-01부터 2025-06까지 6개월이 모두 있는지 확인한다.
  3. 2025-06 매출이 16,816,500원으로 가장 큰지 확인한다.
  4. 2025-02 매출이 13,151,500원으로 가장 작은지 확인한다.
  5. 차트가 표와 같은 값을 쓰는지 확인한다.

말할 것:

35-44분. 시연 3: 이상치 재질문

프롬프트:

이번 분석에서 제외한 이상치 행을 order_id, date, item, unit_price, qty, amount, 제외 이유 순서로 표로 보여 주세요.

또한 일별 매출 합계를 기준으로 통계적으로 유난히 높은 날이나 낮은 날이 있었는지도 확인해 주세요. 기준을 함께 설명해 주세요.

말할 것:

이상치에는 두 층위가 있다. 하나는 행 단위 이상치로, 수량 999나 음수 금액처럼 원본 행 자체가 이상한 경우다. 다른 하나는 집계 단위 이상치로, 어느 하루 매출이 다른 날보다 유난히 튀는 경우다.

이번 데이터에는 행 단위 이상치가 3건 있다. order_id 11496과 16235는 수량이 999이고, 13013은 금액이 -5,500원이다. 반면 일별 매출 합계는 IQR 기준으로 통계적 이상치가 없었다. 그래서 "이상치 없음"이라는 답은 어떤 기준을 썼느냐에 따라 맞을 수도, 틀릴 수도 있다.

완료 판단 체크포인트:

수강생이 AI 답의 문장 유창함이 아니라 행 수, 전처리 기준, 이상치 처리, 월별 매출 수치를 근거로 더 신뢰할 답을 고르면 완료로 본다.

예상 질문과 답변

Q1. AI가 제 답과 다른 숫자를 냈는데 틀린 건가요?

곧바로 틀렸다고 단정하지 않는다. 먼저 전처리 기준을 확인한다. 중복 order_id를 지웠는지, 수량 999와 음수 금액을 뺐는지, 날짜 형식을 모두 읽었는지에 따라 숫자는 얼마든지 달라진다.

Q2. 결측값이 있으면 모두 삭제해야 하나요?

아니다. 그 컬럼이 질문에 필요한지부터 본다. 월별 매출에서는 date와 amount가 중요하다. payment와 member 결측은 이번 집계에서 제외 기준으로 삼지 않는다.

Q3. AI가 "이상치 없음"이라고 했습니다. 어떻게 해야 하나요?

"어떤 기준으로 이상치가 없다고 봤는지" 되묻는다. 일별 매출 합계로 보면 이상치가 없을 수 있다. 하지만 행 단위로는 수량 999와 음수 금액이 있으니 이상치가 있다고 봐야 한다.

Q4. 차트만 보고 결론을 써도 되나요?

차트는 흐름을 훑어보기에 좋지만, 최종 결론에는 표의 숫자가 있어야 한다. 차트와 표가 같은 전처리 기준을 썼는지 확인한다.

Q5. 회사 데이터를 AI에 올려도 되나요?

허가 없이는 안 된다. 특히 개인정보나 고객 정보, 매출 원장, 계약 정보, 내부 보고서는 조직의 보안 정책부터 확인해야 한다. 오늘 수업은 합성 데이터로만 진행한다.

S07. 모범답안

미션 해설

미션은 "두 AI 답을 비교해 더 믿을 만한 답을 고르는 것"이다. 이 차시의 정답은 특정 AI 이름이 아니라, 판단 근거다. 더 믿을 만한 답은 다음 조건을 충족해야 한다.

  1. 원본 파일의 행 수와 열 수를 먼저 확인한다.
  2. 날짜 형식이 섞여 있음을 인식한다.
  3. 중복 order_id를 어떻게 처리했는지 말한다.
  4. 수량 999, 음수 금액, 단가×수량 불일치 행을 이상치로 따로 보여 준다.
  5. 월별 매출 표와 차트의 숫자가 서로 맞는다.
  6. 결론이 표의 숫자에서 바로 나온다.

기준 답안

전처리 기준:

  • date는 혼합 날짜 형식으로 변환했다.
  • order_id는 주문 고유번호로 보고 중복 order_id는 첫 행만 남겼다.
  • unit_price의 쉼표와 원 문자를 제거해 숫자로 변환했다.
  • amount <= 0, qty > 20, unit_price * qty != amount인 행은 이상치로 표시하고 월별 매출 집계에서는 제외했다.
  • payment, member 결측은 월별 매출 집계에서 제외하지 않았다.

이 기준을 적용하면 분석용 데이터는 10,421행이다.

검증용 pandas 코드

아래 코드는 AI가 만든 표가 맞는지 검산하기 위한 코드다. 작성 시 실제로 실행해 결과를 확인했다.

from pathlib import Path

import pandas as pd

data_path = Path("/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_dirty.csv")
df = pd.read_csv(data_path)

expected_columns = [
    "order_id",
    "date",
    "weekday",
    "hour",
    "store",
    "category",
    "item",
    "unit_price",
    "qty",
    "amount",
    "payment",
    "member",
]

assert df.shape == (10439, 12)
assert list(df.columns) == expected_columns

date_parsed = pd.to_datetime(df["date"], format="mixed", errors="coerce")
unit_price_num = pd.to_numeric(
    df["unit_price"]
    .astype(str)
    .str.replace(",", "", regex=False)
    .str.replace("원", "", regex=False)
    .str.strip(),
    errors="coerce",
)

work = df.assign(
    date_parsed=date_parsed,
    unit_price_num=unit_price_num,
)
work["amount_mismatch"] = work["amount"] != work["unit_price_num"] * work["qty"]

date_format_counts = {
    "YYYY-MM-DD": int(df["date"].str.match(r"^\d{4}-\d{2}-\d{2}$").sum()),
    "YYYY/MM/DD": int(df["date"].str.match(r"^\d{4}/\d{2}/\d{2}$").sum()),
    "MM-DD-YYYY": int(df["date"].str.match(r"^\d{2}-\d{2}-\d{4}$").sum()),
}

quality_summary = {
    "전체 행 수": len(df),
    "전체 열 수": len(df.columns),
    "완전 중복 행": int(df.duplicated().sum()),
    "중복 order_id 추가 행": int(df["order_id"].duplicated().sum()),
    "payment 결측": int(df["payment"].isna().sum()),
    "member 결측": int(df["member"].isna().sum()),
    "store 값 '강남 점'": int((df["store"] == "강남 점").sum()),
    "앞 공백 category": int(df["category"].str.startswith(" ", na=False).sum()),
    "unit_price에 원 문자 포함": int(df["unit_price"].astype(str).str.contains("원", regex=False).sum()),
    "qty > 20": int((df["qty"] > 20).sum()),
    "amount <= 0": int((df["amount"] <= 0).sum()),
    "단가x수량과 금액 불일치": int(work["amount_mismatch"].sum()),
}

analysis = work.drop_duplicates("order_id", keep="first").copy()

anomaly_mask = (
    analysis["date_parsed"].isna()
    | (analysis["qty"] > 20)
    | (analysis["amount"] <= 0)
    | (analysis["amount_mismatch"])
)

anomalies = analysis.loc[
    anomaly_mask,
    ["order_id", "date", "item", "unit_price_num", "qty", "amount", "amount_mismatch"],
].sort_values("order_id")

cleaned = analysis.loc[~anomaly_mask].copy()
cleaned["month"] = cleaned["date_parsed"].dt.to_period("M").astype(str)

monthly = (
    cleaned.groupby("month")
    .agg(
        주문건수=("order_id", "count"),
        판매수량=("qty", "sum"),
        매출액=("amount", "sum"),
    )
    .reset_index()
)
monthly["전월대비 증감액"] = monthly["매출액"].diff().fillna(0).astype(int)
monthly["전월대비 증감률"] = monthly["매출액"].pct_change().mul(100).round(1)

daily = (
    cleaned.groupby("date_parsed")
    .agg(
        주문건수=("order_id", "count"),
        판매수량=("qty", "sum"),
        매출액=("amount", "sum"),
    )
    .reset_index()
    .rename(columns={"date_parsed": "date"})
)

q1 = daily["매출액"].quantile(0.25)
q3 = daily["매출액"].quantile(0.75)
iqr = q3 - q1
lower_bound = q1 - 1.5 * iqr
upper_bound = q3 + 1.5 * iqr
daily_outliers = daily[(daily["매출액"] < lower_bound) | (daily["매출액"] > upper_bound)]
top_daily = daily.sort_values("매출액", ascending=False).head(3)

assert len(cleaned) == 10421
assert int(cleaned["amount"].sum()) == 90476000
assert int(monthly.loc[monthly["month"] == "2025-06", "매출액"].iloc[0]) == 16816500
assert len(anomalies) == 3
assert len(daily_outliers) == 0

print("데이터 행열:", df.shape)
print("날짜 형식 수:", date_format_counts)
print("품질 점검:", quality_summary)
print()
print("[월별 매출 추세]")
print(monthly.to_string(index=False))
print()
print("[이상치로 제외한 행]")
print(anomalies.to_string(index=False))
print()
print("일별 매출 IQR 상한:", int(upper_bound))
print("일별 매출 이상치 수:", len(daily_outliers))
print("[일별 매출 상위 3일]")
print(top_daily.to_string(index=False))
