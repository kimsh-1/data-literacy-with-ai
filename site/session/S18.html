S18. 배포와 공유: Looker Studio 주력, Power BI Copilot 선택

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S17 시각화, 상호작용, DAX 기초
  • 난이도: 4
  • 시간: 60분
  • 기본 실습 도구: Looker Studio
  • 선택 시연 도구: Power BI Service, Power BI Copilot
  • 사용 데이터: /mnt/d/ai-data-analysis-course/02_datasets/survey_responses/survey_responses_clean.csv

이번 차시는 보고서 제작에서 한 걸음 더 나아가 그 보고서를 다른 사람이 볼 수 있는 링크로 공유한다. S15부터 S17까지는 Power BI Desktop에서 보고서와 대시보드를 만드는 흐름을 다뤘다. 다만 Power BI Service 게시와 Copilot 기능은 조직 계정, 유료 라이선스, Fabric이나 Premium 용량이 얽혀 있다 보니, 개인 Gmail만 가진 입문자가 표준 실습 경로로 삼기에는 무리가 있다.

그래서 이번 차시의 표준 경로는 Looker Studio다. 웹에서 CSV를 연결해 보고서를 짜고 공유 링크까지 뽑을 수 있으니, Windows를 쓰지 않는 수강생도 그대로 따라올 수 있다. Power BI Copilot은 강사나 수강생 쪽에 조직 환경이 갖춰져 있을 때만 선택 시연으로 다룬다.

학습 목표

  1. 로컬 파일 보고서와 웹 공유 보고서의 차이를 설명할 수 있다.
  2. Looker Studio에 CSV 파일을 업로드하고 데이터 원본을 연결할 수 있다.
  3. 설문 응답 데이터로 응답 수, 평균 NPS, 평균 만족도, NPS 점수 카드를 만들 수 있다.
  4. 계산 필드 NPS_그룹, NPS_점수_행을 만들어 대시보드에 활용할 수 있다.
  5. 공유 링크를 만들고, 다른 사용자가 보기 권한으로 열 수 있는지 검증할 수 있다.
  6. Power BI Service 게시와 Power BI Copilot이 무료 개인 계정 실습 범위가 아니라는 점을 구분할 수 있다.

핵심 개념 1. 공유는 파일을 보내는 것이 아니라 권한을 여는 일이다

엑셀 파일을 이메일로 보내면 받는 사람마다 사본이 하나씩 생긴다. 그중 누군가 손을 대는 순간 어느 것이 최신본인지 헷갈리기 시작한다. BI 보고서 공유는 이와 다르다. 원본은 웹에 하나만 있고, 받는 사람은 링크로 그 원본을 함께 본다.

엑셀과 비교하면 다음과 같다.

엑셀에서의 감각 Looker Studio에서의 감각
파일을 첨부해 보낸다 보고서 링크를 보낸다
받는 사람에게 사본이 생긴다 같은 보고서를 함께 본다
수정본이 여러 개 생길 수 있다 권한에 따라 보기 또는 편집을 나눈다
차트가 정적인 경우가 많다 필터와 상호작용으로 화면을 바꿔 볼 수 있다

링크를 만들었다고 아무나 볼 수 있는 것은 아니다. 보고서 권한이 닫혀 있으면 링크를 받은 사람도 문 앞에서 막힌다. 그래서 이번 차시의 미션은 링크 생성에서 끝나지 않는다. 다른 계정이나 시크릿 창에서 정말 열리는지 확인하는 데까지가 미션이다.

핵심 개념 2. Looker Studio는 무료 경로의 표준 대안이다

Power BI Desktop은 Windows에서 돌아가는 보고서 제작 도구다. Looker Studio는 이와 달리 브라우저 안에서 움직인다. 그래서 Mac을 쓰는 수강생도 Gmail 계정과 브라우저만 있으면 실습에 참여할 수 있다.

CSV 업로드 자체는 무료다. 올린 파일은 데이터셋이 되고, 그 데이터셋을 바탕으로 데이터 원본을 만든다. 한 가지 주의할 점은 같은 데이터셋에 파일을 또 올리면 기존 데이터에 그대로 얹힌다는 것이다. 중복 업로드로 숫자가 뻥튀기되지 않도록 수업에서는 딱 한 번만 올린다.

이번 수업에서 쓰는 것은 개인정보나 민감 정보가 없는 합성 설문 데이터다. 만약 실제 업무 데이터였다면, 공유 버튼을 누르기 전에 개인정보나 고객명, 이메일, 전화번호, 내부 매출 같은 항목이 섞여 있지 않은지 반드시 눈으로 확인해야 한다.

핵심 개념 3. NPS는 점수를 그룹으로 바꿔 읽어야 한다

NPS는 고객이 얼마나 추천할 마음이 있는지를 보는 지표다. 원점수는 0점부터 10점까지 매기지만, 해석할 때는 보통 세 그룹으로 묶어서 읽는다.

NPS 점수 그룹 의미
9-10 추천 고객 적극 추천 가능성이 높은 응답
7-8 중립 고객 나쁘지는 않지만 강한 추천은 아닌 응답
0-6 비추천 고객 불만 또는 이탈 위험이 있는 응답

NPS 점수는 다음 식으로 계산한다.

NPS 점수 = 추천 고객 비율 - 비추천 고객 비율

이번 데이터에서는 추천 고객이 481명, 비추천 고객이 552명이다. 전체 응답이 1,500명이니 NPS 점수는 약 -4.73으로 떨어진다. Looker Studio에서는 계산 필드를 하나 만들어 추천 고객은 100, 중립 고객은 0, 비추천 고객은 -100으로 바꾼 뒤 평균을 내면 이 값이 그대로 나온다.

핵심 개념 4. 게시와 공유는 보안 판단이 포함된 작업이다

대시보드는 그저 보기 좋은 차트 모음이 아니다. 누군가 그 화면을 보고 판단을 내리는 자료다. 그러니 공유하기 전에 아래를 짚고 넘어가야 한다.

확인 항목 질문
데이터 범위 보여도 되는 행과 열만 들어 있는가
숫자 검산 핵심 지표가 원본 데이터와 맞는가
보기 권한 링크 수신자가 편집자가 아니라 보기 권한인가
접근 검증 다른 브라우저, 시크릿 창, 동료 계정에서 열리는가
맥락 보고서 제목과 차트 제목만 보고 의미를 알 수 있는가

이번 차시의 기준은 공개 웹 게시가 아니라 제한된 링크 공유다. 수업 환경에서는 링크가 있는 모든 사용자 보기 가능을 써도 되지만, 실제 회사 데이터라면 조직 내부 사용자나 특정 이메일에게만 보기 권한을 여는 것이 보통이다.

Power BI Service와 Copilot의 위치

Power BI Desktop에서 만든 .pbix 파일을 웹에 올리려면 Power BI Service를 쓴다. 이 길은 개인 무료 계정만으로는 막히는 데가 많고, 조직 계정과 라이선스 정책에 좌우된다. Copilot은 문턱이 더 높다. Microsoft 문서에 따르면, Power BI Service에서 Copilot을 쓰려면 Premium Power BI 용량이나 유료 Fabric 용량이 붙은 워크스페이스가 있어야 하고, Desktop에서 쓸 때도 Copilot이 열린 워크스페이스에 접근할 수 있어야 한다.

따라서 이번 차시의 원칙은 다음과 같다.

구분 수업에서의 처리
Looker Studio 전원 표준 실습
Power BI Service 게시 조직 계정이 있는 경우만 개념 설명 또는 선택 시연
Power BI Copilot Fabric 또는 Premium 환경이 준비된 경우만 선택 시연
개인 Gmail 수강생 Looker Studio 공유 링크 생성으로 미션 완료

60분 수업 흐름

시간 내용 진행 방식
0-7분 로컬 보고서와 웹 공유 보고서 차이 설명 강의
7-15분 설문 데이터 구조와 검산 숫자 확인 강의, 시연
15-27분 Looker Studio에 CSV 업로드, 데이터 원본 연결 따라하기
27-40분 계산 필드와 기본 대시보드 작성 따라하기
40-50분 공유 링크 생성과 권한 설정 따라하기
50-57분 링크 검증 미션 직접 해보기
57-60분 Power BI Copilot 조건, 다음 차시 연결 정리

엑셀과 연결해서 설명할 지점

  • Looker Studio 데이터 원본은 엑셀 피벗테이블의 원본 표와 비슷하다.
  • 차트의 차원은 피벗테이블의 행 또는 열 영역과 비슷하다.
  • 차트의 측정항목은 피벗테이블의 값 영역과 비슷하다.
  • 계산 필드는 엑셀의 새 계산 열 또는 IF 함수와 비슷하다.
  • 공유 링크는 엑셀 파일 첨부가 아니라, 같은 보고서를 볼 권한을 여는 일이다.

따라하기 1. Looker Studio 열기

1단계. 로그인

  1. 브라우저를 연다.
  2. 아래 주소로 이동한다.
https://lookerstudio.google.com/
  1. Google 계정으로 로그인한다.
  2. 처음 접속한 경우 약관이나 국가 설정 화면이 나올 수 있다. 강사의 안내에 따라 진행한다.

2단계. 빈 보고서 만들기

  1. 왼쪽 위 또는 시작 화면에서 만들기(Create)를 누른다.
  2. 보고서(Report) 또는 빈 보고서(Blank report)를 선택한다.
  3. 데이터 추가 화면이 나타나면 다음 단계로 넘어간다.

따라하기 2. CSV 파일 연결하기

1단계. 파일 업로드 커넥터 선택

  1. 데이터 연결 화면에서 파일 업로드(File Upload) 또는 CSV 파일 업로드(CSV file upload)를 찾는다.
  2. Google 제공 커넥터인지 확인한다.
  3. 커넥터를 선택한다.

2단계. CSV 업로드

  1. 파일 업로드 또는 데이터세트 추가(Add data set)를 누른다.
  2. 아래 파일을 선택한다. 브라우저 파일 선택 창에서는 Windows 경로를 사용한다.
Windows: D:\ai-data-analysis-course\02_datasets\survey_responses\survey_responses_clean.csv
WSL 참고: /mnt/d/ai-data-analysis-course/02_datasets/survey_responses/survey_responses_clean.csv
  1. 업로드가 끝나면 연결(Connect)을 누른다.
  2. 필드 목록이 보이면 보고서에 추가(Add to report)를 누른다.

공식 문서에 따르면 CSV 업로드 파일은 UTF-8로 인코딩된 표 형식 데이터여야 한다. 수업 배포본은 이미 UTF-8로 맞춰 두었으니 따로 변환할 것은 없다.

따라하기 3. 계산 필드 만들기

1단계. NPS 그룹 만들기

데이터 원본 또는 보고서 편집 화면에서 필드 추가(Add a field)를 누른다.

필드 이름:

NPS_그룹

수식:

CASE
  WHEN nps >= 9 THEN "추천 고객"
  WHEN nps >= 7 THEN "중립 고객"
  ELSE "비추천 고객"
END

CASE

  • nps와 satisfaction이 합계가 아니라 평균으로 들어갔는지 본다.
  • NPS_점수_행이 평균으로 들어갔는지 본다.
  • NPS_그룹 막대그래프의 세 그룹 수가 552, 481, 467 근처로 보이는지 본다.

40-50분. 공유 링크 생성

시연:

  1. 보고서 제목을 S18_설문_공유_대시보드_강사로 바꾼다.
  2. 오른쪽 위 보기(View)를 눌러 보기 화면을 확인한다.
  3. 다시 수정(Edit)으로 돌아온다.
  4. 공유(Share)를 누른다.
  5. 수업 상황에 맞게 특정 사용자 또는 링크가 있는 사용자에게 보기 가능(Viewer) 권한을 준다.
  6. 링크 복사(Copy link)를 누른다.

말할 것:

여기서 관건은 권한이다. 링크만 복사하고 권한을 열어 두지 않으면 다른 사람은 아무것도 못 본다. 거꾸로 편집자 권한을 열어 주면 남이 보고서를 마음대로 고칠 수 있다. 오늘은 보기 권한으로만 공유한다.

보안 안내:

오늘 데이터는 합성 설문 데이터라서 수업용으로 마음 놓고 공유할 수 있다. 하지만 실제 회사 데이터였다면 링크가 있는 모든 사용자 설정을 함부로 켜서는 안 된다. 고객명, 이메일, 전화번호, 내부 매출 같은 민감 정보가 끼어 있다면 특정 사용자나 조직 내부로 접근을 좁혀야 한다.

50-57분. 직접 해보기 미션

미션 안내:

각자 만든 보고서에서 공유 링크를 뽑는다. 그런 다음 시크릿 창이나 동료 계정에서 직접 열어 본다. 그냥 열리는지만 보고 끝내지 말고, 편집이 아닌 보기 화면인지, 응답 수가 1,500인지, NPS 점수가 약 -4.73인지까지 확인한다.

  • 평균 만족도 카드 값
  • NPS 점수 카드 값
  • 보기 권한 여부

완료 판단 체크포인트:

수강생이 Looker Studio 공유 링크를 보기 권한으로 만들고, 그것을 다른 환경에서 열어 응답 수 1,500과 NPS 점수 약 -4.73까지 확인했다면 이번 차시는 완료된 것으로 본다.

57-60분. Power BI Copilot 조건과 다음 차시 연결

말할 것:

Power BI Copilot은 분명 탐나는 기능이지만, 오늘 수업의 필수 도구는 아니다. Microsoft 문서에 따르면 Copilot은 조직의 Fabric이나 Premium 용량, 관리자 설정, 워크스페이스 권한이 다 맞아떨어져야 쓸 수 있다. 개인 무료 계정으로 무조건 된다는 식으로 안내하면 곤란하다.

선택 시연이 가능한 경우:

Power BI Service에서 미리 준비한 보고서를 열고 Copilot에게 요약을 시켜 본다. 가령 "이 보고서의 핵심 변화를 한 문단으로 요약해줘" 정도면 된다. 답을 받고 나면 반드시 카드 숫자, 원본 집계와 나란히 대조한다.

예상 질문과 답변

Q1. Looker Studio는 무료인가요?

A. 수업에서 다루는 CSV 업로드와 기본 보고서 작성은 무료 경로로 충분하다. 다만 조직용 고급 기능이나 연결 서비스로 넘어가면 별도 조건이 붙기도 한다.

Q2. Gmail 계정이면 충분한가요?

A. 오늘 Looker Studio 실습은 Gmail 계정으로 진행 가능하다. 회사 보안 정책이 적용된 계정은 외부 공유가 제한될 수 있다.

Q3. CSV를 다시 올렸더니 응답 수가 3,000이 되었습니다.

A. 같은 데이터셋에 파일이 하나 더 붙었을 가능성이 크다. CSV 업로드는 새 파일을 기존 데이터에 얹는 식으로 동작하기도 한다. 새 데이터 원본을 만들거나, 중복으로 올라간 데이터셋을 정리하면 된다.

Q4. NPS 점수가 왜 평균 NPS와 다른가요?

A. 평균 NPS는 0점부터 10점까지 원점수를 그대로 평균 낸 값이다. 반면 NPS 점수는 추천 고객 비율에서 비추천 고객 비율을 뺀 값이다. 이름은 비슷해도 서로 다른 지표다.

Q5. 링크를 보냈는데 상대가 못 엽니다.

A. 링크만 복사하고 권한을 열지 않았을 때가 대부분이다. 공유 설정에서 상대 이메일을 Viewer로 넣었는지, 아니면 링크 접근 권한 자체를 열었는지 확인한다.

Q6. 상대가 보고서를 수정할 수 있습니다.

A. 권한이 Editor로 되어 있는지 확인한다. 오늘 미션은 Viewer 권한 공유이다.

Q7. Power BI로 만든 대시보드도 이렇게 공유할 수 있나요?

A. 할 수는 있다. 다만 Power BI Service 게시와 공유는 조직 계정, 라이선스, 테넌트 정책에 발목이 잡힌다. 그래서 오늘 무료 표준 경로는 Looker Studio로 잡았다.

Q8. Power BI Copilot은 왜 안 하나요?

A. Copilot은 Fabric이나 Premium 용량에 관리자 설정까지 있어야 돌아가는 기능이다. 조직 환경이 갖춰져 있으면 선택 시연이야 할 수 있지만, 개인 계정 수강생의 필수 실습으로 못 박으면 재현이 어렵다.

검증용 Python 코드

S18은 Looker Studio 공유 차시이므로 별도 S18_code.py 파일을 요구하지 않는다. 아래 코드는 모범답안 수치 검증을 위해 python3와 pandas 3.0.3으로 실제 실행한 코드이다.

from pathlib import Path
import pandas as pd

path = Path("/mnt/d/ai-data-analysis-course/02_datasets/survey_responses/survey_responses_clean.csv")
df = pd.read_csv(path, parse_dates=["submitted_at"])

print("shape:", df.shape)
print("columns:", list(df.columns))
print("submitted_min:", df["submitted_at"].min())
print("submitted_max:", df["submitted_at"].max())
print("unique_respondents:", df["respondent_id"].nunique())
print("missing_total:", int(df.isna().sum().sum()))

promoters = (df["nps"] >= 9).sum()
passives = df["nps"].between(7, 8).sum()
detractors = (df["nps"] <= 6).sum()
nps_score = ((promoters - detractors) / len(df)) * 100

print("avg_nps:", round(df["nps"].mean(), 4))
print("avg_satisfaction:", round(df["satisfaction"].mean(), 4))
print("promoters:", int(promoters))
print("passives:", int(passives))
print("detractors:", int(detractors))
print("nps_score:", round(nps_score, 2))

channel_summary = (
    df.groupby("channel")
    .agg(
        responses=("respondent_id", "count"),
        avg_nps=("nps", "mean"),
        avg_satisfaction=("satisfaction", "mean"),
    )
    .sort_values("responses", ascending=False)
)
print(channel_summary.round(2).to_string())

age_summary = (
    df.groupby("age_group")
    .agg(
        responses=("respondent_id", "count"),
        avg_nps=("nps", "mean"),
        avg_satisfaction=("satisfaction", "mean"),
    )
    .sort_values("responses", ascending=False)
)
print(age_summary.round(2).to_string())
