S18. 배포와 공유: Looker Studio 주력, Power BI Copilot 선택

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S17 시각화, 상호작용, DAX 기초
  • 난이도: 4
  • 시간: 60분
  • 기본 실습 도구: Looker Studio
  • 선택 시연 도구: Power BI Service, Power BI Copilot
  • 사용 데이터: 수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 파일. CSV는 엑셀로도 열리는 표 파일입니다.

보고서는 만드는 것에서 끝나지 않습니다. 다른 사람이 열어 보고, 같은 숫자를 확인하고, 필요한 판단을 할 수 있어야 보고서의 역할이 완성됩니다. S15부터 S17까지는 Power BI Desktop에서 보고서와 대시보드를 만드는 흐름을 다뤘습니다. 여기까지는 내 컴퓨터 안에서 보고서를 잘 만드는 연습이었다고 보면 됩니다.

그런데 보고서를 공유하려면 이야기가 조금 달라집니다. Power BI Service에 게시하거나 Copilot 기능을 쓰려면 조직 계정, 유료 라이선스, Fabric 또는 Premium 용량 같은 조건이 얽힙니다. 이런 조건은 회사 환경에서는 자연스러울 수 있지만, 개인 Gmail 계정만 가진 입문자가 표준 실습으로 따라가기에는 문턱이 높습니다.

그래서 이 차시의 표준 실습은 Looker Studio로 진행합니다. Looker Studio는 브라우저에서 보고서를 만들고 링크로 공유할 수 있는 도구입니다. Windows가 아니어도 되고, 별도 프로그램을 설치하지 않아도 됩니다. 수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 파일을 연결해 보고서를 만들고, 다른 사람이 볼 수 있는 링크까지 확인해 봅니다.

초보자는 같은 데이터의 엑셀 파일(.xlsx)을 받아 먼저 열어 봐도 됩니다. 엑셀에서 열어 보면 행과 열이 어떤 모양인지 눈에 들어옵니다. 다만 Looker Studio에 파일 업로드로 연결하는 실습에서는 CSV 파일을 사용한다고 이해하면 됩니다.

Power BI Copilot은 강사나 수강생 쪽에 조직 환경이 갖춰져 있을 때만 선택 시연으로 다룹니다. 오늘의 기본 미션은 Looker Studio에서 공유 가능한 설문 대시보드를 완성하는 것입니다.

학습 목표

  1. 로컬 파일 보고서와 웹 공유 보고서의 차이를 설명할 수 있습니다.
  2. Looker Studio에 CSV 파일을 업로드하고 데이터 원본을 연결할 수 있습니다.
  3. 설문 응답 데이터로 응답 수, 평균 NPS, 평균 만족도, NPS 점수 카드를 만들 수 있습니다.
  4. 계산 필드 NPS_그룹, NPS_점수_행을 만들어 대시보드에 활용할 수 있습니다.
  5. 공유 링크를 만들고, 다른 사용자가 보기 권한으로 열 수 있는지 검증할 수 있습니다.
  6. Power BI Service 게시와 Power BI Copilot이 무료 개인 계정 실습 범위가 아니라는 점을 구분할 수 있습니다.

데이터 확인 결과

수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 파일을 python3와 pandas 3.0.3으로 열어 확인한 값입니다. 여기서 pandas는 표 데이터를 읽고 계산하는 데 쓰는 파이썬 도구입니다. 학생이 직접 외울 필요는 없지만, 강사가 숫자를 미리 검산해 두었다는 뜻으로 보면 됩니다.

항목 확인값
파일 수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 파일
행 수 1,500
열 수 8
고유 응답자 수 1,500
결측값 총합 0
제출 기간 2025-03-01 09:35:00부터 2025-05-31 21:26:00까지

추천 고객 수, NPS 9-10 481

이 표는 우리가 대시보드를 만들고 나서 맞춰 볼 기준표입니다. 예를 들어 응답 수 카드가 1,500이 아니라 3,000으로 보인다면, 같은 파일을 두 번 올렸을 가능성을 의심해야 합니다. 평균 NPS가 6.86 근처가 아니라 아주 큰 숫자로 보인다면, 평균이 아니라 합계로 계산했을 가능성이 있습니다. 키를 잴 때 줄자가 맞는지 먼저 확인하듯이, 데이터 분석에서도 기준 숫자를 먼저 확인해 둡니다.

주요 열은 아래와 같이 읽으면 됩니다. 열은 엑셀에서 세로로 내려가는 항목 이름이라고 보면 됩니다.

열 이름 의미 Looker Studio에서의 용도
respondent_id 응답자 ID 응답 수 집계, 표 식별자
age_group 연령대 막대그래프, 필터
gender 성별 필터 또는 분해 기준

satisfaction 만족도, 1-5 평균 만족도
channel 유입 채널 채널별 표, 막대그래프
free_text 주관식 의견 상세 표, S08과 연결
submitted_at 제출 시각 기간 필터, 시계열

여기서 nps와 satisfaction은 숫자로 계산해야 하는 열입니다. age_group, gender, channel은 사람을 나누어 보는 기준입니다. 예를 들어 채널별 평균 만족도를 본다는 말은, 네이버 검색으로 들어온 사람과 인스타그램으로 들어온 사람을 나누어 만족도 평균을 비교한다는 뜻입니다.

개념 1. 공유는 파일을 보내는 것이 아니라 권한을 여는 일입니다

엑셀 파일을 이메일로 보내면 받는 사람마다 사본이 하나씩 생깁니다. 처음에는 괜찮아 보이지만, 누군가 파일을 고치기 시작하면 어느 파일이 최신본인지 헷갈립니다. “최종”, “최종수정”, “진짜최종” 같은 파일 이름이 생기는 이유가 바로 여기에 있습니다.

BI 보고서 공유는 이 방식과 다릅니다. 원본 보고서는 웹에 하나만 있고, 받는 사람은 링크를 통해 그 보고서를 함께 봅니다. 파일을 복사해서 나눠 갖는 것이 아니라, 같은 보고서를 볼 수 있도록 문을 열어 주는 일에 가깝습니다.

엑셀에서 익숙한 감각과 Looker Studio에서의 감각을 비교해 볼까요?

엑셀에서의 감각 Looker Studio에서의 감각
파일을 첨부해 보낸다 보고서 링크를 보낸다
받는 사람에게 사본이 생긴다 같은 보고서를 함께 본다
수정본이 여러 개 생길 수 있다 권한에 따라 보기 또는 편집을 나눈다
차트가 정적인 경우가 많다 필터와 상호작용으로 화면을 바꿔 볼 수 있다

여기서 중요한 부분은 권한입니다. 링크가 있다고 해서 아무나 볼 수 있는 것은 아닙니다. 보고서 권한이 닫혀 있으면 링크를 받은 사람도 보고서 앞에서 막힙니다. 반대로 권한을 너무 넓게 열면 보지 말아야 할 사람까지 볼 수 있습니다.

그래서 이 차시의 미션은 링크를 만드는 데서 끝나지 않습니다. 다른 계정이나 시크릿 창에서 정말 열리는지 확인해야 합니다. 시크릿 창은 브라우저가 기존 로그인 정보를 쓰지 않는 새 창이라고 생각하면 됩니다. 내가 만든 보고서가 내 계정에서만 보이는지, 다른 사람도 보기 권한으로 들어올 수 있는지 확인하는 데 도움이 됩니다.

개념 2. Looker Studio는 무료 실습 경로로 쓰기 좋습니다

Power BI Desktop은 Windows에서 실행하는 보고서 제작 도구입니다. 앞 차시에서 Power BI Desktop을 사용했다면, 내 컴퓨터 안에서 보고서를 만드는 감각을 익힌 것입니다. 하지만 웹 공유로 넘어가면 Power BI Service라는 별도 환경이 필요하고, 조직 계정이나 라이선스 조건이 걸릴 수 있습니다.

Looker Studio는 브라우저 안에서 움직입니다. Chrome이나 Edge 같은 브라우저와 Google 계정만 있으면 실습할 수 있습니다. Mac을 쓰는 수강생도 같은 화면을 보고 따라올 수 있다는 장점이 있습니다.

Looker Studio에서는 수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 파일을 올려 데이터 원본으로 연결합니다. 데이터 원본은 차트를 만들 때 참조하는 표라고 보면 됩니다. 엑셀 피벗테이블을 만들 때 원본 표를 먼저 지정하는 것과 비슷합니다.

한 가지 조심할 점이 있습니다. 같은 데이터셋에 파일을 또 올리면 기존 데이터 위에 그대로 얹힐 수 있습니다. 그러면 응답 수가 1,500이 아니라 3,000처럼 늘어나 보일 수 있습니다. 같은 학생 명단을 두 번 출력해서 출석 인원이 두 배로 보이는 상황과 비슷합니다. 수업에서는 파일을 딱 한 번만 올리고, 대시보드의 응답 수가 1,500으로 나오는지 확인합니다.

이번 수업에서 쓰는 데이터는 개인정보나 민감 정보가 없는 합성 설문 데이터입니다. 합성 데이터는 실제 고객 정보가 아니라 수업용으로 만든 데이터라는 뜻입니다. 그래도 실제 업무 데이터로 보고서를 만들 때는 공유 버튼을 누르기 전에 반드시 확인해야 합니다. 고객명, 이메일, 전화번호, 내부 매출, 계약 정보처럼 밖으로 나가면 안 되는 항목이 섞여 있지 않은지 눈으로 봐야 합니다.

개념 3. NPS는 점수를 그룹으로 바꿔 읽습니다

NPS는 고객이 우리 제품이나 서비스를 다른 사람에게 추천할 마음이 얼마나 있는지를 보는 지표입니다. 원점수는 0점부터 10점까지입니다. 하지만 해석할 때는 0부터 10까지 각각 따로 보는 것보다 세 그룹으로 묶어 읽는 경우가 많습니다.

NPS 점수 그룹 의미
9-10 추천 고객 적극 추천 가능성이 높은 응답
7-8 중립 고객 나쁘지는 않지만 강한 추천은 아닌 응답
0-6 비추천 고객 불만 또는 이탈 위험이 있는 응답

여기서 뭐가 보이시나요? 7점과 8점은 꽤 좋아 보이는 점수지만, NPS 계산에서는 추천 고객으로 보지 않습니다. “나쁘지는 않아요”와 “꼭 추천하고 싶어요”는 다르기 때문입니다. 그래서 7점과 8점은 중립 고객으로 둡니다.

NPS 점수는 추천 고객 비율에서 비추천 고객 비율을 뺀 값입니다.

NPS 점수 = 추천 고객 비율 - 비추천 고객 비율

이번 데이터에서는 추천 고객이 481명, 비추천 고객이 552명입니다. 전체 응답은 1,500명입니다. 추천 고객보다 비추천 고객이 더 많기 때문에 NPS 점수는 약 -4.73으로 떨어집니다. 평균 NPS가 6.8553이라고 해서 추천 분위기가 강하다고 바로 말하면 안 됩니다. NPS는 평균 점수만 보는 지표가 아니라, 추천 고객과 비추천 고객의 비율 차이를 함께 봐야 합니다.

Looker Studio에서는 계산 필드를 만들어 이 값을 구합니다. 계산 필드는 원래 데이터에 없던 새 열을 규칙으로 만들어 내는 기능입니다. 사전에서 단어 뜻을 찾아 붙이듯이, NPS 점수를 보고 “추천 고객”, “중립 고객”, “비추천 고객”이라는 이름을 붙입니다.

NPS_그룹은 0-10점짜리 원점수를 세 그룹으로 나눕니다. NPS_점수_행은 추천 고객을 100, 중립 고객을 0, 비추천 고객을 -100으로 바꿉니다. 그다음 NPS_점수_행의 평균을 내면 NPS 점수가 됩니다. 이렇게 하면 Looker Studio의 카드 하나로 약 -4.73이라는 값을 보여 줄 수 있습니다.

개념 4. 게시와 공유에는 보안 판단이 들어갑니다

대시보드는 보기 좋은 차트 모음이 아닙니다. 누군가 그 화면을 보고 회의에서 말하고, 의사결정을 하고, 다음 행동을 정합니다. 그래서 공유 전에는 “잘 보이는가”만 보면 안 됩니다. “보여도 되는가”, “숫자가 맞는가”, “권한이 맞는가”를 함께 봐야 합니다.

공유 전에 아래 질문을 하나씩 확인해 보세요.

확인 항목 질문
데이터 범위 보여도 되는 행과 열만 들어 있는가
숫자 검산 핵심 지표가 원본 데이터와 맞는가
보기 권한 링크 수신자가 편집자가 아니라 보기 권한인가
접근 검증 다른 브라우저, 시크릿 창, 동료 계정에서 열리는가
맥락 보고서 제목과 차트 제목만 보고 의미를 알 수 있는가

수업에서는 공개 웹 게시가 아니라 제한된 링크 공유를 기준으로 봅니다. 공개 웹 게시는 인터넷에 올려 누구나 볼 수 있게 만드는 방식입니다. 수업 데이터는 합성 데이터라 위험이 낮지만, 실제 회사 데이터라면 그렇게 하면 안 되는 경우가 많습니다.

수업 환경에서는 링크가 있는 모든 사용자 보기 가능을 써도 됩니다. 다만 실제 업무에서는 조직 내부 사용자나 특정 이메일에게만 보기 권한을 여는 것이 보통입니다. 편집 권한도 조심해야 합니다. 보기 권한은 보고서 화면을 열어 보는 권한이고, 편집 권한은 차트나 데이터 연결을 바꿀 수 있는 권한입니다. 학생 실습에서는 링크를 받은 사람이 편집자가 아니라 보는 사람으로 들어오는지 확인합니다.

Power BI Service와 Copilot의 위치

Power BI Desktop에서 만든 .pbix 파일을 웹에 올리려면 Power BI Service를 사용합니다. .pbix는 Power BI Desktop 보고서 파일입니다. 엑셀 파일에 .xlsx가 붙는 것처럼, Power BI 보고서 파일에는 .pbix가 붙는다고 보면 됩니다.

Power BI Service 게시 경로는 개인 무료 계정만으로는 막히는 부분이 많습니다. 조직 계정이 필요할 수 있고, 회사나 학교의 라이선스 정책에 따라 가능한 기능이 달라집니다. 그래서 이 차시에서 Power BI Service는 모든 학생이 반드시 따라 하는 표준 실습으로 두지 않습니다.

Copilot은 문턱이 더 높습니다. Microsoft 문서에 따르면, Power BI Service에서 Copilot을 쓰려면 Premium Power BI 용량이나 유료 Fabric 용량이 붙은 워크스페이스가 있어야 합니다. Desktop에서 쓸 때도 Copilot이 열린 워크스페이스에 접근할 수 있어야 합니다. 쉽게 말해, Copilot 버튼이 보이려면 개인의 의지만이 아니라 조직 환경과 유료 조건이 맞아야 합니다.

이 차시에서는 도구의 위치를 이렇게 구분합니다.

구분 수업에서의 처리
Looker Studio 전원 표준 실습
Power BI Service 게시 조직 계정이 있는 경우만 개념 설명 또는 선택 시연
Power BI Copilot Fabric 또는 Premium 환경이 준비된 경우만 선택 시연
개인 Gmail 수강생 Looker Studio 공유 링크 생성으로 미션 완료

학생 입장에서는 “Power BI Copilot을 못 쓰면 수업을 못 따라가는 것 아닌가?”라고 걱정할 필요가 없습니다. 표준 미션은 Looker Studio에서 끝납니다. Power BI Copilot은 환경이 되는 경우에만, 보고서를 요약하거나 낮은 지표를 설명해 달라고 요청하는 선택 시연으로 봅니다. Copilot의 답도 최종 결론이 아니라 초안입니다. 숫자는 반드시 카드와 원본 데이터 기준으로 다시 확인해야 합니다.

60분 수업 흐름

아래 흐름대로 진행하면 보고서 제작, 공유, 검증까지 한 번에 이어서 경험할 수 있습니다. 앞부분에서는 왜 공유가 필요한지 이해하고, 중간에는 Looker Studio에서 직접 만들고, 마지막에는 다른 환경에서 링크가 열리는지 확인합니다.

시간 내용 진행 방식

엑셀과 연결해서 설명할 지점

엑셀을 써 본 사람이라면 Looker Studio의 용어가 처음에는 낯설 수 있습니다. 새 도구를 외우려고 하기보다, 엑셀에서 이미 알던 감각에 연결해 보면 훨씬 편합니다.

  • Looker Studio 데이터 원본은 엑셀 피벗테이블의 원본 표와 비슷합니다.
  • 차트의 차원은 피벗테이블의 행 또는 열 영역과 비슷합니다.
  • 차트의 측정항목은 피벗테이블의 값 영역과 비슷합니다.
  • 계산 필드는 엑셀의 새 계산 열 또는 IF 함수와 비슷합니다.
  • 공유 링크는 엑셀 파일 첨부가 아니라, 같은 보고서를 볼 권한을 여는 일입니다.

예를 들어 차원을 channel로 두고 측정항목을 respondent_id 개수로 두면, 채널별 응답 수를 볼 수 있습니다. 엑셀 피벗테이블에서 행에 채널을 넣고 값에 응답자 수를 넣는 것과 같은 감각입니다. 계산 필드는 IF 함수처럼 조건을 보고 값을 바꿉니다. NPS가 9 이상이면 추천 고객, 7 이상이면 중립 고객, 나머지는 비추천 고객으로 나누는 식입니다.

완료 판단

수강생이 Looker Studio에서 설문 대시보드를 완성하고 응답 수 1,500, 평균 NPS 약 6.86, 평균 만족도 약 3.62, NPS 점수 약 -4.73을 확인했다면 숫자 검산은 통과한 것입니다.

그다음 보기 권한 공유 링크를 만들고, 시크릿 창이나 다른 계정에서 직접 열어 검증해야 합니다. 내 화면에서만 잘 보이는 보고서는 아직 공유가 끝난 것이 아닙니다. 다른 사람이 보기 권한으로 열 수 있고, 카드 숫자가 기준값과 맞는 것까지 확인하면 이 차시는 완료된 것으로 봅니다.

따라하기 3. 계산 필드 만들기

1단계. NPS 그룹 만들기

이 단계는 0점부터 10점까지 흩어져 있는 NPS 점수를 추천, 중립, 비추천 세 묶음으로 나누는 과정입니다.

데이터 원본 또는 보고서 편집 화면에서 필드 추가(Add a field)를 누른다.

필드 이름:

NPS_그룹

수식:

CASE
  WHEN nps >= 9 THEN "추천 고객"
  WHEN nps >= 7 THEN "중립 고객"
  ELSE "비추천 고객"
END

저장한다. 이 필드는 0-10점짜리 NPS 원점수를 세 그룹으로 갈라 줍니다. 키를 잴 때 171.2cm, 168.5cm를 그대로 외우기보다 “160cm대, 170cm대”로 묶어 보면 더 빨리 보이는 것과 같은 원리입니다.

2단계. NPS 점수 계산용 필드 만들기

이 단계는 NPS 공식에 맞게 각 응답을 100, 0, -100으로 바꿔 평균을 낼 준비를 하는 과정입니다.

다시 필드 추가(Add a field)를 누른다.

필드 이름:

NPS_점수_행

수식:

CASE
  WHEN nps >= 9 THEN 100
  WHEN nps <= 6 THEN -100
  ELSE 0
END

저장한다. 추천 고객은 100, 중립 고객은 0, 비추천 고객은 -100으로 바뀝니다. 그런 다음 이 필드의 평균을 내면 NPS 점수가 됩니다. 사람마다 점수를 먼저 붙이고 반 평균을 내는 방식이라고 보면 됩니다.

3단계. 만족도 구간 만들기

이 단계는 만족도 1점부터 5점까지의 숫자를 만족, 보통, 불만족이라는 말로 읽기 쉽게 바꾸는 과정입니다.

필수는 아니지만, 나중에 필터나 색상 구분에 요긴하게 쓸 수 있다.

필드 이름:

만족도_구간

수식:

CASE
  WHEN satisfaction >= 4 THEN "만족"
  WHEN satisfaction = 3 THEN "보통"
  ELSE "불만족"
END

숫자만 있을 때보다 말로 된 구간이 있으면 차트를 읽는 사람이 빠르게 이해합니다. 사전에서 단어 뜻을 찾아 붙이듯, 숫자에 이름표를 붙이는 작업입니다.

따라하기 4. 카드 4개 만들기

1단계. 응답 수 카드

이 단계는 이 대시보드가 몇 명의 응답을 바탕으로 만들어졌는지 맨 위에서 바로 보이게 하는 과정입니다.

  1. 상단 메뉴에서 차트 추가(Add a chart)를 누른다.
  2. 스코어카드(Scorecard)를 선택한다.
  3. 캔버스 상단 왼쪽에 배치한다.
  4. 측정항목에 respondent_id를 넣는다.
  5. 집계 방식을 고유 개수(Count Distinct) 또는 개수(Count)로 설정한다.
  6. 제목을 아래처럼 바꾼다.
응답 수

정답 기준은 1,500입니다. 이 숫자가 다르면 파일을 잘못 올렸거나 같은 데이터를 중복으로 올렸을 가능성이 있습니다.

2단계. 평균 NPS 카드

이 단계는 전체 응답자의 추천 의향이 평균적으로 어느 정도인지 한눈에 보이게 하는 과정입니다.

nps는 0점부터 10점까지 매긴 추천 의향 점수입니다. "이 서비스를 다른 사람에게 추천할 마음이 얼마나 있나요?"를 숫자로 받은 값이라고 보면 됩니다. satisfaction은 1점부터 5점까지의 만족도입니다. channel은 응답자가 어디를 거쳐 들어왔는지를 나타내고, free_text는 주관식 의견을 담고 있습니다.

검산 숫자:

대시보드가 제대로 만들어졌는지 보려면 기준 숫자를 옆에 두고 맞춰 봐야 합니다. 오늘 확인할 값은 응답 수 1,500, 평균 NPS 약 6.86, 평균 만족도 약 3.62, NPS 점수 약 -4.73입니다. 이 숫자가 다르게 나오면 공유부터 하면 안 됩니다. 먼저 차트의 집계 방식이 맞는지 확인해야 합니다.

NPS 설명:

NPS는 평균 점수 하나만 보는 지표가 아닙니다. 9점과 10점은 추천 고객, 7점과 8점은 중립 고객, 0점부터 6점까지는 비추천 고객으로 나눠 봅니다. 사전에서 단어를 찾아 뜻을 분류하듯이, 점수 구간에 따라 응답자를 세 그룹으로 나누는 겁니다. 여기서 추천 고객 비율에서 비추천 고객 비율을 뺀 값이 NPS 점수입니다. 이번 데이터는 추천 고객보다 비추천 고객이 조금 많아서 NPS 점수가 음수로 내려갑니다.

15-27분. Looker Studio에 설문 응답 파일 업로드

시연:

  1. https://lookerstudio.google.com/에 접속한다.
  2. 만들기(Create)를 누른다.
  3. 보고서(Report) 또는 빈 보고서(Blank report)를 선택한다.
  4. 데이터 연결에서 파일 업로드(File Upload) 또는 CSV 파일 업로드를 선택한다.

수강생이 CSV라는 말을 어려워하면 이렇게 안내합니다. CSV는 엑셀로도 열리는 표 파일입니다. 행과 열로 된 데이터를 담는 단순한 파일이라고 보면 됩니다. 데이터 받기 페이지에서 엑셀 파일(.xlsx)을 받은 수강생도 같은 표 데이터를 쓰는 것이므로, 강사 안내에 맞춰 업로드하면 됩니다.

주의:

같은 데이터셋에 파일을 또 올리면 데이터가 겹쳐서 붙을 수 있습니다. 응답 수가 3,000쯤으로 보인다면 같은 파일을 두 번 올렸다고 봐야 합니다. 이럴 때는 억지로 고치려고 시간을 쓰기보다 새 데이터 원본으로 다시 시작하는 편이 빠릅니다.

27-40분. 계산 필드와 기본 대시보드 작성

시연 1. 계산 필드:

NPS_그룹 수식:

CASE
  WHEN nps >= 9 THEN "추천 고객"
  WHEN nps >= 7 THEN "중립 고객"
  ELSE "비추천 고객"
END

NPS_점수_행 수식:

CASE
  WHEN nps >= 9 THEN 100
  WHEN nps <= 6 THEN -100
  ELSE 0
END

말할 것:

이 수식은 엑셀의 IF와 닮았습니다. 조건을 하나씩 따져 보고, 맞는 값이 나오면 그 결과를 돌려줍니다. NPS_그룹은 응답자를 추천 고객, 중립 고객, 비추천 고객으로 나누는 이름표 역할을 합니다. 그래서 차원으로 씁니다.

NPS_점수_행은 계산용 숫자입니다. 추천 고객은 100, 중립 고객은 0, 비추천 고객은 -100으로 바꿔 둡니다. 그런 다음 평균을 내면 NPS 점수가 됩니다. 여기서 조심할 점이 있습니다. NPS_점수_행을 합계로 잡으면 안 됩니다. 반드시 평균이어야 합니다.

시연 2. 카드 4개:

  1. 응답 수 카드: respondent_id 개수 또는 고유 개수, 값 1,500
  2. 평균 NPS 카드: nps 평균, 값 약 6.86
  3. 평균 만족도 카드: satisfaction 평균, 값 약 3.62
  4. NPS 점수 카드: NPS_점수_행 평균, 값 약 -4.73

시연 3. 차트:

  1. NPS 그룹별 응답 수 막대그래프
  2. 채널별 설문 요약 표
  3. 연령대별 평균 만족도 막대그래프

말할 것:

카드와 차트를 만들 때는 차원과 측정항목을 나눠 생각해야 합니다. 차원은 무엇을 기준으로 쪼개 볼지입니다. 채널, 연령대, NPS 그룹이 여기에 들어갑니다. 측정항목은 그렇게 쪼갠 뒤 계산할 숫자입니다. 응답 수, 평균 NPS, 평균 만족도가 여기에 들어갑니다.

예를 들어 채널별 평균 만족도를 본다면, 채널은 차원이고 평균 만족도는 측정항목입니다. 엑셀 피벗테이블에서 행 영역에 채널을 넣고 값 영역에 만족도 평균을 넣는 것과 같은 감각입니다.

돌아다니며 볼 것:

  • 응답 수가 1,500인지 확인한다.
  • nps와 satisfaction이 합계가 아니라 평균으로 들어갔는지 본다.
  • NPS_점수_행이 평균으로 들어갔는지 본다.
  • NPS_그룹 막대그래프의 세 그룹 수가 552, 481, 467 근처로 보이는지 본다.

예상 질문과 답변

Q1. Looker Studio는 무료인가요?

A. 수업에서 다루는 파일 업로드와 기본 보고서 작성은 무료 경로로 충분합니다. 다만 조직용 고급 기능이나 외부 서비스 연결로 넘어가면 별도 조건이 붙을 수 있습니다.

Q2. Gmail 계정이면 충분한가요?

A. 오늘 Looker Studio 실습은 Gmail 계정으로 진행할 수 있습니다. 회사 보안 정책이 적용된 계정은 외부 공유가 제한될 수 있습니다. 그럴 때는 계정 문제가 아니라 조직 보안 설정 때문에 막히는 경우가 많습니다.

Q3. CSV를 다시 올렸더니 응답 수가 3,000이 되었습니다.

A. 같은 데이터셋에 파일이 하나 더 붙었을 가능성이 큽니다. 파일 업로드는 새 파일을 기존 데이터에 얹는 식으로 동작하기도 합니다. 응답 수가 1,500이어야 하는데 3,000으로 보이면 중복 업로드를 의심해 보세요. 새 데이터 원본을 만들거나, 중복으로 올라간 데이터셋을 정리하면 됩니다.

Q4. NPS 점수가 왜 평균 NPS와 다른가요?

A. 평균 NPS는 0점부터 10점까지 원점수를 그대로 평균 낸 값입니다. 반면 NPS 점수는 추천 고객 비율에서 비추천 고객 비율을 뺀 값입니다. 이름은 비슷해 보여도 서로 다른 지표입니다. 평균 NPS는 점수의 평균이고, NPS 점수는 고객 그룹 비율의 차이라고 보면 됩니다.

Q5. 링크를 보냈는데 상대가 못 엽니다.

A. 링크만 복사하고 권한을 열지 않았을 때가 많습니다. 공유 설정에서 상대 이메일을 Viewer로 넣었는지, 아니면 링크 접근 권한 자체를 열었는지 확인합니다. 링크 주소와 접근 권한은 따로 움직인다고 생각하면 됩니다.

Q6. 상대가 보고서를 수정할 수 있습니다.

A. 권한이 Editor로 되어 있는지 확인합니다. 오늘 미션은 Viewer 권한 공유입니다. 상대가 보기만 해야 한다면 반드시 Viewer로 바꿔야 합니다.

Q7. Power BI로 만든 대시보드도 이렇게 공유할 수 있나요?

A. 할 수는 있습니다. 다만 Power BI Service 게시와 공유는 조직 계정, 라이선스, 테넌트 정책에 영향을 받습니다. 그래서 오늘 무료 표준 경로는 Looker Studio로 잡았습니다.

Q8. Power BI Copilot은 왜 안 하나요?

A. Copilot은 Fabric이나 Premium 용량에 관리자 설정까지 있어야 돌아가는 기능입니다. 조직 환경이 갖춰져 있으면 선택 시연은 할 수 있습니다. 하지만 개인 계정 수강생의 필수 실습으로 정하면 재현이 어렵습니다.

공유 링크는 사람마다 다릅니다. 그래서 모범답안에 고정된 URL은 없습니다. 중요한 것은 링크 주소 자체가 아니라, 그 링크가 열리는지, 보기 권한인지, 화면의 숫자가 검산값과 맞는지입니다. 줄자로 키를 재듯이, 대시보드도 기준 숫자와 맞춰 보아야 합니다.

설문 응답 파일은 수업 홈의 데이터 받기 페이지에서 내려받아 사용하면 됩니다. CSV라는 말이 나오면, 엑셀로도 열리는 표 파일이라고 생각하시면 됩니다. 엑셀 파일 형식으로 받은 경우에도 수업 안내에 맞게 같은 설문 응답 데이터를 연결하면 됩니다.

Looker Studio 계산 필드 정답

NPS_그룹

CASE
  WHEN nps >= 9 THEN "추천 고객"
  WHEN nps >= 7 THEN "중립 고객"
  ELSE "비추천 고객"
END

이 필드는 0점부터 10점까지 들어 있는 NPS 원점수를 세 그룹으로 나누는 역할을 합니다. 숫자를 그대로 보면 한눈에 감이 잘 오지 않기 때문에, 사전에서 단어 뜻을 찾듯이 점수 구간마다 이름을 붙여 주는 것입니다.

NPS 그룹 응답 수
비추천 고객 552
추천 고객 481
중립 고객 467

여기서 뭐가 보이시나요? 비추천 고객이 552명이고 추천 고객이 481명입니다. 추천 고객보다 비추천 고객이 조금 더 많기 때문에, 뒤에서 계산하는 NPS 점수가 음수로 나옵니다.

NPS_점수_행

CASE
  WHEN nps >= 9 THEN 100
  WHEN nps <= 6 THEN -100
  ELSE 0
END

이 필드는 반드시 평균으로 집계해야 합니다. 추천 고객은 100, 중립 고객은 0, 비추천 고객은 -100으로 바꾼 뒤 평균을 내면 NPS 점수가 됩니다. 합계로 계산하면 점수가 너무 크게 튀기 때문에, 꼭 평균인지 확인해 보세요.

평균값이 NPS 점수이며, 정답은 약 -4.73입니다.

만족도_구간

이 필드는 필수는 아니지만, 만족도를 더 쉽게 읽고 싶을 때 사용할 수 있습니다. 만족도 숫자 1-5를 그대로 보는 대신, 만족, 보통, 불만족처럼 말로 바꿔 주는 필드입니다.

CASE
  WHEN satisfaction >= 4 THEN "만족"
  WHEN satisfaction = 3 THEN "보통"
  ELSE "불만족"
END

검증용 Python 코드

S18은 Looker Studio 공유 차시이므로 별도 S18_code.py 파일을 요구하지 않습니다. 아래 코드는 모범답안 수치가 실제 데이터와 맞는지 검증하기 위해 python3와 pandas 3.0.3으로 실행한 코드입니다.

학생 제출용 코드는 아니고, 선생님이 기준 숫자를 확인하기 위해 사용한 검산 코드라고 보시면 됩니다. 코드 안에는 실행 환경의 파일 위치가 들어 있지만, 실습에서는 수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 파일을 Looker Studio에 연결하면 됩니다.

from pathlib import Path
import pandas as pd

path = Path("/mnt/d/ai-data-analysis-course/02_datasets/survey_responses/survey_responses_clean.csv")
df = pd.read_csv(path, parse_dates=["submitted_at"])

print("shape:", df.shape)
print("columns:", list(df.columns))
print("submitted_min:", df["submitted_at"].min())
print("submitted_max:", df["submitted_at"].max())
print("unique_respondents:", df["respondent_id"].nunique())
print("missing_total:", int(df.isna().sum().sum()))

promoters = (df["nps"] >= 9).sum()
passives = df["nps"].between(7, 8).sum()
detractors = (df["nps"] <= 6).sum()
nps_score = ((promoters - detractors) / len(df)) * 100

print("avg_nps:", round(df["nps"].mean(), 4))
print("avg_satisfaction:", round(df["satisfaction"].mean(), 4))
print("promoters:", int(promoters))
print("passives:", int(passives))
print("detractors:", int(detractors))
print("nps_score:", round(nps_score, 2))

channel_summary = (
    df.groupby("channel")
    .agg(
        responses=("respondent_id", "count"),
        avg_nps=("nps", "mean"),
        avg_satisfaction=("satisfaction", "mean"),
    )
    .sort_values("responses", ascending=False)
)
print(channel_summary.round(2).to_string())

age_summary = (
    df.groupby("age_group")
    .agg(
        responses=("respondent_id", "count"),
        avg_nps=("nps", "mean"),
        avg_satisfaction=("satisfaction", "mean"),
    )
    .sort_values("responses", ascending=False)
)
print(age_summary.round(2).to_string())
