S01. 데이터 분석이란, AI 시대의 작업 흐름 + 환경 세팅

차시 개요

  • 대상: 엑셀은 쓰지만 데이터 분석은 처음인 비개발자
  • 시간: 50분
  • 난이도: 1
  • 선수 지식: 없음
  • 사용 데이터: 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 CSV 파일. CSV는 엑셀로도 열리는 표 파일입니다. 초보자는 같은 내용의 엑셀 파일 형식으로 내려받아도 괜찮습니다.
  • 수업의 중심 질문: "내가 가진 표로 어떤 질문에 답할 수 있는가?"

데이터 분석은 어려운 프로그램을 외우는 일이 아니라, 표에 담긴 기록으로 업무 질문에 답하는 일입니다. 오늘은 계산 기술을 많이 배우기보다 “내가 가진 표로 무엇을 물어볼 수 있는지”를 보는 관점을 먼저 잡습니다.

엑셀, AI, 파이썬, BI는 서로 경쟁하는 도구가 아닙니다. 같은 질문을 다루는 방법이 서로 다를 뿐입니다. 엑셀은 눈으로 표를 보며 빠르게 확인하는 데 좋고, AI는 질문을 정리하거나 분석 방향을 떠올릴 때 도움이 됩니다. 파이썬은 같은 분석을 코드로 다시 실행할 수 있게 해 주고, BI는 여러 사람이 볼 수 있는 대시보드로 결과를 보여 주는 데 강합니다.

학습 목표

이 차시가 끝나면 수강생은 아래 세 가지를 할 수 있어야 합니다.

  1. 데이터로 답할 수 있는 질문과 답하기 어려운 질문을 구분한다.
  2. 엑셀, AI, 파이썬, BI가 데이터 분석 흐름에서 각각 어떤 역할을 하는지 설명한다.
  3. 수업에 필요한 환경을 점검한다. 운영체제, 필요한 계정, 데이터 파일 준비 여부를 확인한다. Google Colab은 웹 브라우저에서 파이썬을 실행하는 도구이고, 파이썬 파트(S10)에서 처음 사용한다. 오늘은 Google 계정이 있는지만 확인한다.

오늘 사용할 데이터

오늘은 카페 주문 데이터를 살펴봅니다. 깊게 분석하기보다, 표가 어떻게 생겼고 어떤 질문을 만들 수 있는지 보는 데 집중합니다.

수업 홈의 데이터 받기 페이지에서 카페 매출 파일을 내려받아 사용합니다. 처음에는 CSV 파일을 받아도 되고, 엑셀에 더 익숙하다면 같은 내용의 엑셀 파일 형식으로 받아도 됩니다. 중요한 것은 파일 형식이 아니라 표 안에 어떤 열이 있고, 그 열로 어떤 질문에 답할 수 있는지 보는 것입니다.

  • 실제 확인 행수: 10,424행
  • 실제 확인 열수: 12열
  • 날짜 범위: 2025-01-01부터 2025-06-30까지
  • 주요 열: order_id, date, weekday, hour, store, category, item, unit_price, qty, amount, payment, member
  • 지점: 강남점, 홍대점
  • 카테고리: 커피, 음료, 디저트

엑셀에서 열어 보면 한 행은 주문 한 건을 뜻합니다. 마트 영수증을 한 줄씩 쌓아 놓은 것처럼 보면 됩니다. 한 열은 그 주문의 속성입니다. 예를 들어 store는 어느 지점에서 팔렸는지, amount는 주문 금액이 얼마인지, date는 언제 팔렸는지를 알려 줍니다.

여기서 뭐가 보이시나요? 이 표에는 주문일, 지점, 상품명, 수량, 금액처럼 “이미 기록된 것”이 들어 있습니다. 그래서 데이터 분석은 먼저 표에 무엇이 적혀 있는지 확인하는 일에서 시작합니다.

개념 1. 데이터 분석은 질문을 표에 맞게 번역하는 일입니다

데이터 분석은 냉장고 속 재료로 요리하는 일과 비슷합니다. 냉장고에 있는 재료가 데이터이고, 만들고 싶은 음식이 질문이며, 칼과 팬이 분석 도구입니다. 재료가 없으면 그 요리를 바로 만들 수 없듯이, 표에 기록되지 않은 내용은 분석으로 바로 답하기 어렵습니다.

카페 주문 표에는 주문일, 지점, 상품명, 수량, 금액이 있습니다. 그래서 아래 질문은 이 표로 답할 수 있습니다.

  • 2025년 1월부터 6월까지 강남점과 홍대점 중 어느 지점의 매출이 더 높은가?
  • 커피, 음료, 디저트 중 총매출이 가장 큰 카테고리는 무엇인가?
  • 어느 시간대에 주문 금액이 많이 발생하는가?

이 질문들은 표 안에 필요한 열이 있습니다. 지점별 매출을 보려면 store와 amount를 보면 됩니다. 카테고리별 총매출을 보려면 category와 amount를 보면 됩니다. 시간대별 주문 금액을 보려면 hour와 amount를 보면 됩니다. 사전에서 단어를 찾듯이, 질문에 필요한 열을 표에서 찾아보는 것입니다.

반대로 아래 질문은 이 표만으로는 답하기 어렵습니다.

  • 손님이 왜 이 메뉴를 골랐는가?
  • 매장 분위기가 만족도에 어떤 영향을 주었는가?
  • 다음 달 매출이 반드시 얼마가 될 것인가?

이 질문들이 틀렸다는 뜻은 아닙니다. 업무에서는 오히려 아주 자연스러운 질문입니다. 다만 지금 표에는 구매 이유도, 만족도도, 매장 분위기도, 다음 달 결과도 담겨 있지 않습니다. 그래서 분석을 시작하기 전에 “이 질문에 답하려면 어떤 열이 더 필요한가?”를 먼저 따져야 합니다.

예를 들어 손님이 왜 이 메뉴를 골랐는지 알고 싶다면 구매 이유를 묻는 설문 열이 필요할 수 있습니다. 매장 분위기의 영향을 보려면 만족도 점수나 매장 평가 같은 자료가 더 필요합니다. 다음 달 매출을 알고 싶다면 과거 매출만으로 단정할 수 없고, 예측 모델과 추가 가정이 필요합니다. 데이터 분석은 있는 자료로 말할 수 있는 것과 아직 말할 수 없는 것을 나누는 작업입니다.

개념 2. 좋은 분석 질문은 모양을 갖추고 있습니다

입문 단계에서는 분석 질문을 아래 구조로 바꾸면 훨씬 다루기 쉬워집니다.

대상 + 기준 + 비교/집계 + 기간

예시는 이렇게 볼 수 있습니다.

  • 대상: 카페 주문
  • 기준: 지점별
  • 비교/집계: 총매출 비교
  • 기간: 2025년 1월부터 6월까지
  • 질문: 2025년 1월부터 6월까지 지점별 총매출은 어떻게 다른가?

분석 질문은 막연한 궁금증을 표의 열 이름으로 바꾸는 과정입니다. “잘 팔린다”는 말은 일상에서는 자연스럽지만, 분석에서는 먼저 뜻을 정해야 합니다. 키를 잴 때 “큰 편이다”라고만 말하지 않고 센티미터로 재듯이, 데이터에서도 기준을 숫자로 정해야 합니다.

예를 들어 “잘 팔린다”는 말을 amount 합계로 볼 수도 있습니다. 이 경우에는 매출이 큰 상품을 찾는 것입니다. qty 합계로 볼 수도 있습니다. 이 경우에는 많이 팔린 상품을 찾는 것입니다. order_id 개수로 볼 수도 있습니다. 이 경우에는 주문 건수가 많은지를 보는 것입니다.

그래서 분석을 시작하기 전에는 무엇을 기준으로 “잘 팔린다”고 할지 정해야 합니다. 기준을 정하지 않으면 같은 표를 보고도 서로 다른 답을 말하게 됩니다. 매출 1위 상품과 수량 1위 상품이 다를 수 있기 때문입니다.

개념 3. 엑셀, AI, 파이썬, BI는 역할이 다릅니다

엑셀

엑셀은 표를 직접 눈으로 확인하고, 간단한 필터와 피벗테이블로 빠르게 답을 얻는 데 강합니다. 많은 수강생이 이미 업무에서 엑셀을 써 보았기 때문에, 데이터 감각을 잡기에 좋은 출발점입니다.

엑셀에서는 행과 열을 바로 볼 수 있습니다. 필터를 눌러 강남점만 보거나, 피벗테이블로 지점별 매출 합계를 빠르게 만들 수 있습니다. 그래서 “이 데이터가 대략 어떤 모양인지”, “이 질문에 필요한 열이 있는지”를 확인하기 좋습니다.

다만 엑셀 작업은 손으로 클릭하며 진행되는 경우가 많습니다. 같은 분석을 여러 번 반복하거나 데이터가 커지면, 어느 단계에서 어떤 필터를 걸었는지 놓치기 쉽습니다. 그래서 반복성과 기록이 중요한 작업에서는 파이썬 같은 도구가 필요해집니다.

AI

AI는 질문을 정리하고, 분석 방향을 제안하고, 코드나 설명을 만들어 주는 보조자 역할을 합니다. 예를 들어 “카페 매출 데이터로 어떤 분석을 해 볼 수 있을까?”라고 물으면, AI는 지점별 매출, 시간대별 매출, 카테고리별 판매량 같은 분석 아이디어를 제안할 수 있습니다.

하지만 AI가 낸 숫자가 항상 맞는 것은 아닙니다. AI는 그럴듯한 설명을 잘 만들지만, 실제 데이터 파일을 정확히 읽었는지, 계산을 제대로 했는지는 확인해야 합니다. 특히 파일 업로드와 데이터 분석 기능은 유료 구독이 필요한 경우가 있습니다. 수업에서는 표준 AI 도구 1종을 기준으로 설명하되, 무료 대안도 함께 안내합니다.

중요한 숫자는 AI 답변만 믿지 말고 엑셀, 파이썬, BI 중 하나로 반드시 검증해야 합니다. 분석에서 숫자는 말의 장식이 아니라 판단의 근거입니다. 계산기로 다시 눌러 보듯이, 중요한 결과는 한 번 더 확인하는 습관이 필요합니다.

파이썬

파이썬과 pandas는 같은 작업을 코드로 재현할 수 있게 해 줍니다. pandas는 표 데이터를 다루기 위한 파이썬 도구입니다. 엑셀에서 필터를 누르고 합계를 내는 일을 코드로 적어 둔다고 생각하면 됩니다.

코드로 남겨 두면 누가 실행해도 같은 결과가 나옵니다. 정제, 집계, 병합, 시각화 과정을 기록으로 남기기도 쉽습니다. 처음에는 코드가 낯설게 보일 수 있습니다. 하지만 데이터가 커지고 반복 작업이 많아질수록 “다시 실행할 수 있다”는 점이 큰 장점이 됩니다.

이 수업에서는 Part 4부터 파이썬을 본격적으로 사용합니다. 오늘은 파이썬을 설치하거나 코드를 쓰는 시간이 아닙니다. 대신 나중에 파이썬을 왜 배우는지, 엑셀과 어떤 점이 다른지 큰 그림만 잡아 둡니다.

BI

BI는 Business Intelligence의 줄임말로, 데이터를 대시보드로 보고 공유하기 위한 도구라고 생각하면 됩니다. BI 도구는 한 번 만든 화면을 다른 사람이 필터를 바꾸며 볼 수 있게 해 줍니다. 회의나 보고에서 “지점별로 눌러 보세요”, “기간을 바꿔 보세요”처럼 함께 확인하기 좋습니다.

엑셀이나 파이썬이 분석을 만드는 도구라면, BI는 분석 결과를 계속 볼 수 있는 화면으로 만드는 데 강합니다. 매출 현황판처럼 자주 확인해야 하는 결과를 보여 줄 때 잘 맞습니다.

Power BI Desktop은 Windows 전용입니다. Mac 수강생은 Part 5에서 Looker Studio 경로를 대체 경로로 사용합니다. 운영체제에 따라 쓰는 도구가 조금 달라질 수 있지만, 배우는 생각의 흐름은 같습니다. 표에서 질문을 만들고, 필요한 열을 고르고, 결과를 읽는 과정은 그대로입니다.

수업 전체 흐름에서 S01의 위치

이 수업은 아래 흐름으로 진행됩니다.

  1. 엑셀에서 표의 구조와 피벗 감각을 잡는다.
  2. 통계 직관으로 평균, 분포, 상관을 해석한다.
  3. AI를 이용해 분석을 요청하고, 결과를 검증한다.
  4. 파이썬 pandas로 같은 분석을 재현 가능하게 만든다.
  5. BI로 대시보드를 만들고 공유한다.
  6. 마지막에는 분석 결과를 업무 보고 형태로 정리한다.

S01은 이 전체 흐름의 출발점입니다. 오늘의 목표는 많은 기능을 배우는 것이 아닙니다. 앞으로 배울 도구들이 어떤 문제를 해결하는지 큰 지도를 얻는 것입니다.

엑셀을 배울 때도, AI를 쓸 때도, 파이썬 코드를 볼 때도 출발점은 같습니다. “이 표로 어떤 질문에 답할 수 있는가?”를 먼저 봅니다. 이 질문이 잡히면 도구는 목적지까지 가는 이동수단이 됩니다. 반대로 질문이 없으면 도구를 많이 배워도 어디에 써야 할지 막히기 쉽습니다.

환경 체크

수강생은 아래 항목을 확인합니다.

  • 운영체제: Windows 또는 Mac
  • Power BI Desktop 사용 가능 여부: Windows 가능, Mac은 Looker Studio 대체
  • Google 계정: 준비 여부 (파이썬 파트 S10부터 Google Colab에 사용한다)
  • AI 계정: 수업에서 지정한 AI 도구 접속 가능 여부
  • 데이터 준비: 수업 홈의 데이터 받기 페이지에서 카페 매출 파일을 내려받을 수 있는지 확인
  • 오늘 파일: 카페 매출 파일을 엑셀에서 열 수 있는지 확인

실습 데이터는 수업에서 바로 사용할 수 있도록 정리된 가공본을 사용합니다. 수강생이 수업 중 공공데이터 사이트에서 직접 다운로드하지 않습니다.

오늘은 데이터 파일을 열고 표가 보이는지만 확인해도 충분합니다. 행과 열이 보이면 준비가 된 것입니다. 파일 형식 때문에 막히면 CSV 파일 대신 같은 내용의 엑셀 파일 형식으로 내려받아 열어 보세요.

시간 내용 진행 방식
0-5분 수업 목표와 전체 지도 소개 강의
5-15분 데이터로 답할 수 있는 질문과 답하기 어려운 질문 구분 사례 설명
15-25분 엑셀, AI, 파이썬, BI 역할 비교 강의와 대화
25-35분 환경 체크리스트 점검 따라하기
35-45분 카페 매출 표로 분석 질문 3개 만들기 개인 실습
45-50분 업무 표 질문화 공유와 마무리 발표와 정리

이번 차시의 완료 기준

수강생이 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일을 열어 표의 행과 열을 확인하면 첫 단계는 완료입니다. 그다음 카페 매출 표로 답할 수 있는 분석 질문 3개와 자기 업무 표에 대한 분석 질문 1개를 작성해 보세요.

질문을 만들 때는 “무엇을 보고 싶은가?”에서 멈추지 말고 “어느 열을 봐야 하는가?”까지 적어 봅니다. 예를 들어 지점별 총매출을 보고 싶다면 store와 amount가 필요합니다. 시간대별 매출을 보고 싶다면 hour와 amount가 필요합니다. 이렇게 질문과 열을 연결할 수 있으면, 데이터 분석의 출발선을 제대로 잡은 것입니다.

S01 핸드아웃. 데이터 분석이란, AI 시대의 작업 흐름 + 환경 세팅

오늘 할 일

데이터 분석은 표에 들어 있는 값으로 업무 질문에 답하는 일입니다. 오늘은 어려운 분석 기법을 배우기보다, 분석을 시작하기 전에 꼭 잡아야 하는 감각을 먼저 익혀 봅니다.

오늘 확인할 내용은 세 가지입니다.

  1. 이 표로 어떤 질문에 답할 수 있는지 판단합니다.
  2. 엑셀, AI, 파이썬, BI가 각각 언제 필요한지 큰 지도를 잡습니다.
  3. 앞으로 수업을 따라가기 위한 환경을 확인합니다.

분석은 계산부터 시작하지 않습니다. 먼저 “이 표에 내가 궁금한 답을 찾을 재료가 있나요?”를 확인합니다. 키를 재려면 줄자가 있어야 하듯이, 데이터 분석도 답에 필요한 열이 표 안에 있어야 합니다.

준비물

  • 노트북
  • Google 계정 (파이썬 파트인 S10부터 Google Colab에 사용합니다. 오늘은 준비만 해 두면 됩니다.)
  • 수업에서 지정한 AI 도구 계정
  • 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일

CSV는 엑셀로도 열리는 표 파일입니다. 열었을 때 엑셀이 더 편한 분은 같은 데이터의 엑셀 파일을 받아서 사용해도 됩니다.

확인할 점은 세 가지입니다.

  • Power BI Desktop은 Windows에서만 설치해 사용합니다. Mac 사용자는 Part 5에서 Looker Studio 경로를 따릅니다.
  • 엑셀은 Microsoft 365를 권장합니다. 뒤 차시에서 FILTER, SORT 등 동적 배열 함수를 사용합니다.
  • AI 파일 업로드와 데이터 분석 기능은 유료 구독이 필요한 경우가 있습니다. 수업에서는 표준 도구를 기준으로 설명하되, 가능한 무료 대안도 함께 안내합니다.

따라하기 1. 카페 매출 파일 내려받기

  1. 수업 홈의 데이터 받기 페이지를 엽니다.
    실습에 쓸 표를 먼저 준비해야, 뒤에서 같은 화면을 보며 따라갈 수 있습니다.

  2. 카페 매출 파일을 내려받습니다.
    오늘은 이 파일 하나로 “어떤 질문에 답할 수 있는지”를 연습합니다.

  3. 내려받은 파일을 찾기 쉬운 위치에 둡니다.
    파일을 바로 열 수 있어야 실습 흐름이 끊기지 않습니다.

  4. 파일 이름을 보고 카페 매출 데이터인지 확인합니다.
    엉뚱한 파일을 열면 열 이름과 값이 달라져서 설명을 따라가기 어렵습니다.

파일이 보이면 오늘 실습 데이터 준비는 완료입니다.

따라하기 2. 엑셀로 표 관찰하기

  1. 카페 매출 파일을 엑셀로 엽니다.
    엑셀로 먼저 보는 이유는 표의 모양을 눈으로 확인하기 쉽기 때문입니다.

  2. 첫 행의 열 이름을 확인합니다.
    열 이름은 이 표가 어떤 정보를 들고 있는지 알려 주는 표지판입니다.

order_id, date, weekday, hour, store, category, item, unit_price, qty, amount, payment, member
  1. 각 열이 무엇을 뜻하는지 옆 사람에게 설명해 봅니다.
    사전에서 단어 뜻을 찾듯이, 열 이름의 뜻을 알아야 질문을 만들 수 있습니다.
열 이름 뜻
order_id 주문 번호
date 주문일
weekday 요일
hour 주문 시간대
store 지점
category 상품 카테고리
item 상품명
unit_price 단가
qty 수량
amount 주문 금액
payment 결제수단
member 멤버십 여부
  1. 아래 질문에 답해 봅니다. 정확한 계산보다 “어느 열을 봐야 하는지”를 생각하는 것이 중요합니다.
    분석 질문은 보통 열 이름으로 바꿀 수 있어야 계산으로 이어집니다.
질문 봐야 할 열
어느 지점의 매출이 더 높은가? store, amount
어떤 카테고리가 많이 팔리는가? category, qty 또는 amount
어느 시간대에 매출이 높은가? hour, amount
멤버십 고객과 비멤버십 고객의 매출은 다른가? member, amount

여기서 뭐가 보이시나요? “지점”을 비교하려면 store가 필요하고, “매출”을 보려면 amount가 필요합니다. 질문 속 말과 표의 열 이름을 연결하는 습관이 분석의 출발점입니다.

따라하기 3. 데이터로 답할 수 있는 질문 구분하기

다음 질문을 보고, 현재 카페 주문 표만으로 답할 수 있는지 판단해 보세요.
이 단계가 필요한 이유는, 데이터에 없는 내용을 억지로 계산하려고 하면 그럴듯하지만 틀린 답이 나오기 때문입니다.

질문 판단 이유
2025년 1월부터 6월까지 지점별 총매출은? 가능 date, store, amount가 있다.
상품 카테고리별 주문 수량은? 가능 category, qty가 있다.
손님이 카페 분위기에 만족했는가? 어려움 만족도나 분위기 평가 열이 없다.
다음 달 매출은 얼마인가? 제한적 과거 데이터는 있지만 미래 값은 없다. 예측 모델과 추가 가정이 필요하다.
왜 특정 메뉴를 골랐는가? 어려움 구매 이유를 묻는 열이나 설문이 없다.

분석을 시작할 때는 “궁금한가?”보다 “이 표의 열로 답할 수 있는가?”를 먼저 확인합니다.

예를 들어 손님의 만족도를 알고 싶다면 만족도 점수나 설문 응답 열이 있어야 합니다. 구매 이유를 알고 싶다면 구매 이유를 물어본 기록이 있어야 합니다. 표에 없는 값은 사전에서 없는 단어를 찾는 것과 비슷해서, 분석 도구를 바꾼다고 바로 생기지 않습니다.

카페 매출 파일로 답할 수 있는 질문을 3개 작성합니다. 질문에는 가능한 한 다음 네 가지가 들어가야 합니다.

  • 대상
  • 기준
  • 비교하거나 집계할 값

대상은 무엇을 볼 것인지입니다. 기준은 무엇으로 나누어 볼 것인지입니다. 비교하거나 집계할 값은 합계, 개수, 평균처럼 숫자로 확인할 값입니다. 기간은 언제부터 언제까지 볼 것인지를 뜻합니다.

미션 2. 내 업무 표 1개를 질문으로 바꾸기

제출 기준

아래 세 가지가 들어가면 제출 완료입니다.

  • 카페 매출 표로 만든 분석 질문 3개
  • 각 질문에 필요한 열 이름
  • 내 업무 표 1개와 그 표로 답하고 싶은 질문 1개

제출할 때는 질문만 쓰고 끝내지 말고, 그 질문에 답하려면 어떤 열을 봐야 하는지도 함께 적어 보세요. 질문과 열 이름이 연결되어야 실제 분석으로 넘어갈 수 있습니다.

막힐 때 확인할 것

  • 파일이 안 보이면 수업 홈의 데이터 받기 페이지에서 카페 매출 파일을 다시 내려받습니다.
  • 질문이 너무 막연하면 “어떤 열을 기준으로, 어떤 숫자를 비교할지”부터 적습니다.
  • 데이터에 없는 내용을 묻고 있다면, 필요한 열을 새로 적어 봅니다.

막히는 것은 자연스럽습니다. 분석 질문을 만드는 일은 머릿속 궁금증을 표의 열 이름으로 바꾸는 연습입니다. 처음에는 느리게 하셔도 됩니다. 지금은 계산을 빨리 하는 것보다, 이 표로 답할 수 있는 질문과 답하기 어려운 질문을 구분하는 감각이 더 중요합니다.

S01 강사대본. 데이터 분석이란, AI 시대의 작업 흐름 + 환경 세팅

수업 전 확인

  • 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일이 엑셀에서 열리는지 확인한다.
  • CSV는 엑셀로도 열리는 표 파일이라고 한 번 풀어 말한다. 초보자는 CSV 대신 엑셀 파일(.xlsx)을 내려받아 열어도 된다고 안내한다.
  • 강의장 인터넷과 Colab 접속 상태를 확인한다.
  • Mac 수강생에게 Power BI Desktop은 Windows 전용이며, Part 5에서는 Looker Studio 대체 경로를 안내한다고 미리 말한다.
  • AI 도구는 파일 업로드와 데이터 분석 기능이 유료일 수 있음을 사전에 안내한다.

0-5분. 오프닝

데이터 분석 기술을 한꺼번에 많이 배우는 시간이 아니라, 앞으로 사용할 도구들이 어디에 쓰이는지 지도를 그려 보는 시간이라고 말한다.

강사 멘트 예시:

"데이터 분석은 프로그램을 잘 다루는 사람만 하는 일이 아닙니다. 이미 여러분은 엑셀에서 필터를 걸고, 합계를 보고, 피벗테이블을 만들어 본 경험이 있으실 겁니다. 그 경험을 데이터 분석이라는 말로 다시 정리해 보겠습니다. 중요한 기준은 하나입니다. 내가 가진 표로 어떤 질문에 답할 수 있는지 판단하는 것입니다. 표에 있는 열로 답할 수 있는 질문인지, 아니면 다른 정보가 더 필요한 질문인지 구분하는 힘을 먼저 기르는 겁니다."

오늘의 목표 세 가지를 화면에 보여 준다.

  1. 데이터로 답할 질문과 답하기 어려운 질문 구분
  2. 엑셀, AI, 파이썬, BI의 역할 이해
  3. 수업 환경 체크

5-15분. 데이터로 답할 수 있는 질문 구분

수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일을 연다. 처음에는 엑셀로 보여 주는 것이 좋다. 첫 행의 열 이름을 함께 읽으며, 한 행이 주문 한 건이라는 점을 설명한다.

시연 포인트:

  • 사용할 파일: 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일
  • 실제 데이터: 10,424행, 12열
  • 날짜 범위: 2025-01-01부터 2025-06-30까지
  • 주요 열: date, weekday, hour, store, category, item, qty, amount
  • 지점: 강남점, 홍대점
  • 카테고리: 커피, 음료, 디저트

강사 멘트 예시:

"이 표를 볼 때 처음부터 계산하려고 하지 마세요. 먼저 이 표가 무엇을 기록한 표인지 봐야 합니다. 여기서는 한 행이 주문 한 건입니다. 주문 한 건마다 날짜, 요일, 시간대, 지점, 상품, 수량, 금액 같은 정보가 붙어 있습니다. 그러면 이 표로 답할 수 있는 질문도 그 정보에서 출발해야 합니다. 키를 재려면 줄자가 있어야 하듯이, 매출을 보려면 금액 열이 있어야 하고, 지점별로 비교하려면 지점 열이 있어야 합니다."

화이트보드나 슬라이드에 다음 두 그룹을 나눠 쓴다.

답할 수 있는 질문:

  • 지점별 총매출은 어떻게 다른가?
  • 카테고리별 판매 수량은 어떻게 다른가?
  • 시간대별 매출은 어떻게 다른가?

답하기 어려운 질문:

  • 고객은 왜 이 메뉴를 골랐는가?
  • 고객이 매장 분위기에 만족했는가?
  • 다음 달 매출은 반드시 얼마인가?

설명할 점:

"답하기 어려운 질문은 나쁜 질문이 아닙니다. 다만 지금 표에 그 내용을 담은 열이 없을 뿐입니다. 고객이 왜 메뉴를 골랐는지 알려면 구매 이유나 설문 응답이 필요하고, 매장 분위기 만족도를 보려면 만족도 점수가 필요합니다. 데이터 분석가는 질문을 버리는 사람이 아닙니다. 질문을 표로 답할 수 있는 형태로 바꾸거나, 필요한 데이터를 추가로 요청하는 사람입니다."

15-25분. 엑셀, AI, 파이썬, BI의 역할 비교

도구 네 가지를 하나씩 설명한다. 수강생이 이미 아는 엑셀에서 출발한다.

강사 멘트 예시:

"엑셀은 표를 빠르게 눈으로 확인하고, 필터나 피벗테이블로 바로 살펴보기에 좋습니다. AI는 질문을 정리하거나 분석 방향을 제안받을 때 도움이 됩니다. 파이썬은 같은 분석을 나중에 다시 실행할 수 있도록 기록으로 남기는 데 강합니다. BI는 다른 사람이 필터를 바꾸면서 볼 수 있는 대시보드를 만들 때 좋습니다. 같은 표를 보더라도 도구마다 맡는 일이 조금씩 다른 셈입니다."

도구별 주의점도 같이 말한다.

  • 엑셀: 손작업이 많아지면 재현이 어렵다.
  • AI: 그럴듯하지만 틀린 숫자를 낼 수 있으므로 검증해야 한다.
  • 파이썬: 처음에는 낯설지만 반복 작업과 큰 데이터에 강하다.
  • BI: 공유와 대시보드에 강하지만 설치와 계정 환경을 확인해야 한다.

Power BI 안내:

"Power BI Desktop은 Windows 전용입니다. Mac을 쓰는 분도 수업을 못 따라가는 것은 아닙니다. Part 5에서 Looker Studio를 대체 경로로 사용합니다. 도구 이름은 달라도, 데이터를 연결하고 대시보드로 보여 주는 흐름은 같이 배웁니다."

AI 안내:

"AI 도구의 파일 업로드나 데이터 분석 기능은 유료 구독이 필요한 경우가 있습니다. 수업에서는 표준 도구를 기준으로 설명하되, 무료 대안이 가능한 구간은 같이 안내하겠습니다. AI가 답을 빨리 말해 줄 수는 있지만, 숫자가 맞는지는 꼭 확인해야 합니다. 계산기 화면에 숫자가 나왔다고 해서 영수증과 맞는지 안 보는 것은 아니죠. 분석에서도 검산이 필요합니다."

25-35분. 환경 체크 따라하기

수강생에게 핸드아웃의 체크리스트를 열게 한다.

체크 항목:

  • 운영체제가 Windows인지 Mac인지 확인
  • Google 계정 준비 여부 확인 (Colab은 S10부터 쓴다)
  • AI 도구 접속 가능 여부 확인
  • 수업 홈의 데이터 받기 페이지 접속 여부 확인
  • 데이터 받기 페이지에서 카페 매출 파일을 내려받을 수 있는지 확인

오리엔테이션에서는 코드를 실행하지 않는다. Colab 실행과 pandas로 파일 열기는 파이썬 파트(S10, S11)에서 처음 다루므로 여기서 시연하지 않는다. 오늘은 엑셀로 표를 열어 열 이름과 의미를 함께 살피는 데까지만 한다.

강사 멘트 예시:

"지금은 코드를 실행하는 시간이 아닙니다. 오늘은 내가 어떤 환경에서 수업을 따라갈 수 있는지 확인하고, 카페 매출 파일을 열어 표의 모양을 보는 데 집중하겠습니다. 파일은 수업 홈의 데이터 받기 페이지에서 직접 내려받습니다. CSV가 낯설면 엑셀 파일을 받아도 됩니다. 중요한 것은 파일 형식이 아니라, 표를 열고 열 이름을 읽을 수 있는지입니다."

35-45분. 미션 진행

수강생이 직접 질문을 만들게 한다. 강사는 계산을 도와주기보다 질문의 모양을 다듬어 준다.

판서할 질문 공식:

대상 + 기준 + 비교/집계 + 기간

좋은 질문 예:

  • 2025년 1월부터 6월까지 지점별 총매출은 어떻게 다른가?
  • 2025년 1월부터 6월까지 카테고리별 판매 수량은 어떻게 다른가?
  • 2025년 1월부터 6월까지 시간대별 평균 주문 금액은 어떻게 다른가?

수강생 질문을 받을 때 확인할 점:

  • 질문 안에 비교 기준이 있는가?
  • 어떤 숫자를 집계할지 정했는가?
  • 필요한 열이 현재 표에 있는가?
  • 기간이 필요한 질문이면 기간이 적혀 있는가?

강사 멘트 예시:

"질문을 만들 때는 막연하게 '잘 팔리는 메뉴가 뭐예요?'라고만 쓰면 분석하기 어렵습니다. 여기서 '잘 팔린다'는 말이 수량이 많다는 뜻인지, 금액이 크다는 뜻인지 먼저 정해야 합니다. 사전에서 단어를 찾을 때 찾을 단어를 알아야 하듯이, 표에서도 어떤 열을 볼지 정해야 합니다. 예를 들어 지점별 총매출을 보려면 store와 amount가 필요합니다. 카테고리별 판매 수량을 보려면 category와 qty가 필요합니다."

45-50분. 마무리

몇 명에게 본인의 업무 표 질문을 공유하게 한다. 정답을 평가하기보다, 표의 열과 질문이 연결되는지 확인한다.

마무리 멘트 예시:

"오늘 만든 질문은 앞으로 모든 차시에서 계속 쓰게 될 기본 습관입니다. 분석 도구를 배우기 전에 먼저 질문을 표의 열로 바꾸어야 합니다. 지금 가진 표로 답할 수 있는지, 답하려면 어떤 열이 필요한지 보는 겁니다. 다음 차시부터는 좋은 데이터의 모양을 보고, 엑셀에서 직접 분석하기 좋은 표와 그렇지 않은 표를 구분해 보겠습니다."

예상 질문과 답변

Q1. 데이터 분석을 하려면 코딩을 먼저 배워야 하나요?

A. 아닙니다. 이 수업은 엑셀에서 시작합니다. 코딩은 반복 작업과 검증이 필요해질 때 나중에 배웁니다. 오늘은 코딩보다 질문을 만드는 일이 먼저입니다. 표를 보고 어떤 열로 어떤 질문에 답할 수 있는지 판단하는 연습부터 합니다.

Q2. AI에게 파일을 올리면 분석이 끝나는 것 아닌가요?

A. AI는 분석을 도와주는 도구입니다. 하지만 숫자를 잘못 계산하거나, 데이터에 없는 내용을 그럴듯하게 말할 수 있습니다. 그래서 중요한 결과는 엑셀, 파이썬, BI 중 하나로 검증해야 합니다. AI의 답은 초안으로 보고, 숫자는 다시 확인한다고 생각하면 됩니다.

Q3. Mac이라서 Power BI를 못 쓰면 수업을 따라갈 수 없나요?

A. 따라갈 수 있습니다. Power BI Desktop은 Windows 전용이므로 Mac 수강생은 Looker Studio 경로로 대체합니다. 공유와 대시보드 개념은 그대로 배웁니다. 화면 도구가 조금 달라질 뿐, 데이터를 연결하고 필터로 살펴보는 흐름은 같습니다.

Q4. 제 업무 데이터에 필요한 열이 없으면 어떻게 하나요?

A. 그 질문은 현재 표만으로는 답하기 어렵다고 판단하면 됩니다. 질문이 틀렸다는 뜻은 아닙니다. 다음 단계는 필요한 열을 추가로 수집하거나, 현재 표로 답할 수 있는 더 좁은 질문으로 바꾸는 것입니다. 예를 들어 만족도를 알고 싶다면 만족도 점수나 설문 응답 열이 필요합니다.

Q5. 행이 많으면 엑셀로도 충분하지 않나요?

A. 엑셀로 가능한 일이 많습니다. 특히 처음 표를 열어 보고, 필터를 걸고, 피벗테이블로 빠르게 확인할 때는 엑셀이 좋습니다. 다만 작업을 반복하거나, 여러 파일을 합치거나, 정제 과정을 남겨야 할 때는 파이썬이 유리합니다. 대시보드로 공유해야 할 때는 BI가 유리합니다.

완료 판단 체크포인트

수강생이 데이터 받기 페이지에서 내려받은 카페 매출 파일의 행과 열을 확인하고, 카페 매출 표로 답할 수 있는 분석 질문 3개와 자기 업무 표 질문 1개를 작성하면 이번 차시는 완료이다.

S01 모범답안. 데이터 분석이란, AI 시대의 작업 흐름 + 환경 세팅

실제로 확인한 카페 매출 데이터

수업에서 숫자를 설명하려면 먼저 데이터가 어떤 모양인지 확인해 두어야 합니다. 여기서는 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일을 기준으로 봅니다. 이 파일은 엑셀로도 열리는 표 파일인 CSV 형식입니다. 엑셀이 더 편한 분은 같은 내용의 엑셀 파일(.xlsx)을 받아도 됩니다.

오리엔테이션에서는 수강생이 코드를 직접 실행하지 않습니다. 파이썬과 Colab은 S10부터 다룹니다. 아래 내용은 강사가 미리 같은 파일을 열어, 표가 제대로 준비되어 있는지 키를 재듯 확인한 값입니다.

  • 사용한 데이터: 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일
  • 행과 열: 10,424행, 12열
  • 날짜 범위: 2025-01-01부터 2025-06-30까지
  • 결측값: 모든 열 0개
  • 열 이름: order_id, date, weekday, hour, store, category, item, unit_price, qty, amount, payment, member

여기서 행은 주문 한 건이라고 보면 됩니다. 열은 그 주문을 설명하는 항목입니다. 예를 들어 date는 주문일, store는 지점, amount는 주문 금액입니다. 결측값이 0개라는 말은 빈칸으로 빠진 값이 없다는 뜻입니다.

간단히 집계하면 이런 숫자가 나옵니다.

항목 결과
전체 주문 건수 10,424
전체 매출 합계 90,492,500
주문 1건당 평균 금액 8,681.2

주문 1건당 평균 금액 8,681.2는 주문 한 번에 평균적으로 이 정도 금액이 결제되었다는 뜻입니다. 평균은 여러 사람의 키를 모두 더한 뒤 사람 수로 나누는 것처럼, 전체 매출을 전체 주문 건수로 나누어 봅니다.

지점별 매출 합계:

지점 매출 합계
강남점 53,425,000
홍대점 37,067,500

이 표에서는 강남점의 매출 합계가 53,425,000이고, 홍대점의 매출 합계가 37,067,500입니다. 지점별로 amount를 더했기 때문에 두 지점의 매출 규모를 바로 비교할 수 있습니다.

카테고리별 매출 합계:

카테고리 매출 합계
커피 45,606,000
음료 25,550,000
디저트 19,336,500

카테고리별로 보면 커피의 매출 합계가 가장 큽니다. 여기서 "많이 팔렸다"를 말할 때는 수량 기준인지, 금액 기준인지 구분해야 합니다. 지금 표는 매출 합계이므로 amount를 더한 값입니다.

미션 1 모범답안. 카페 매출 파일로 분석 질문 3개 만들기

모범답안은 하나로 고정되어 있지 않습니다. 다만 좋은 분석 질문은 지금 가진 표의 열로 답할 수 있어야 합니다. 사전에서 단어를 찾듯이, 질문 속의 말이 어느 열과 연결되는지 확인해 보세요.

번호 분석 질문 답을 위해 필요한 열 판단
1 2025년 1월부터 6월까지 지점별 총매출은 어떻게 다른가? date, store, amount 가능
2 2025년 1월부터 6월까지 상품 카테고리별 총매출은 어떻게 다른가? date, category, amount 가능
3 2025년 1월부터 6월까지 시간대별 매출 상위 시간은 언제인가? date, hour, amount 가능

위 질문들은 모두 카페 매출 파일에 있는 열로 답할 수 있습니다. 기간은 date, 지점은 store, 카테고리는 category, 시간대는 hour, 매출은 amount와 연결됩니다. 질문 안에 기간, 기준, 집계할 값이 들어 있다는 점도 좋습니다.

반대로 아래 질문은 현재 표만으로는 답하기 어렵습니다.

질문 어려운 이유
고객이 왜 아메리카노를 샀는가? 구매 이유 열이 없다.
고객이 매장 분위기에 만족했는가? 만족도나 매장 분위기 평가 열이 없다.
2025년 7월 매출은 반드시 얼마인가? 현재 파일은 2025년 1월부터 6월까지의 과거 주문이다. 예측에는 추가 가정이 필요하다.

이 질문들이 틀렸다는 뜻은 아닙니다. 다만 지금 표에는 구매 이유, 만족도, 매장 분위기 평가, 2025년 7월의 실제 매출이 들어 있지 않습니다. 분석을 시작할 때는 "궁금한가?"보다 먼저 "이 표에 그 답을 찾을 열이 있는가?"를 봅니다.

미션 2 모범답안. 내 업무 표 1개를 분석 질문으로 바꾸기

내 업무에서도 같은 방식으로 생각하면 됩니다. 평소 자주 보는 표를 하나 고르고, 그 표에 어떤 열이 있을지 떠올린 뒤, 그 열로 답할 수 있는 질문을 만듭니다.

예시 답안:

내 업무 표 표에 있을 것 같은 열 분석 질문
고객 문의 처리표 문의일, 채널, 문의유형, 담당자, 처리시간, 처리상태 이번 달 문의유형별 평균 처리시간은 어떻게 다른가?

이 질문은 좋은 편입니다. 무엇을 볼지, 어떤 기준으로 나눌지, 어떤 값을 계산할지가 분명하기 때문입니다.

  • 대상이 명확합니다: 고객 문의 처리표
  • 기준이 명확합니다: 문의유형별
  • 집계할 값이 명확합니다: 평균 처리시간
  • 기간이 들어 있습니다: 이번 달

단, 실제 표에 처리시간이 없다면 이 질문은 바로 답할 수 없습니다. 그 경우에는 처리 시작 시각과 완료 시각으로 처리시간을 계산하거나, 처리시간 열을 새로 만들어야 합니다. 분석 질문은 머릿속에서만 끝나는 문장이 아니라, 실제 표의 열과 연결되어야 합니다.

실습 코드

아래 파이썬 코드는 강사용 검산 코드입니다. pandas는 표 데이터를 다루는 파이썬 도구입니다. 이 코드는 카페 매출 파일을 읽고, 날짜를 날짜 형식으로 바꾼 뒤, 행 수, 열 수, 날짜 범위, 비어 있는 값, 매출 합계를 확인합니다.

S01에서 수강생이 반드시 실행해야 하는 코드는 아닙니다. 저장된 검산용 코드로 실제 실행해 정상 작동을 확인했습니다.

from pathlib import Path

import pandas as pd


ROOT = Path(__file__).resolve().parents[1]
DATA_PATH = ROOT / "02_datasets" / "cafe_sales" / "cafe_sales_clean.csv"


def main():
    df = pd.read_csv(DATA_PATH)
    df["date"] = pd.to_datetime(df["date"])

    print(f"pandas_version={pd.__version__}")
    print(f"data_path={DATA_PATH}")
    print(f"rows={len(df)}")
    print(f"columns={len(df.columns)}")
    print("column_names=" + ", ".join(df.columns))
    print(f"date_range={df['date'].min().date()} ~ {df['date'].max().date()}")
    print(f"missing_total={int(df.isna().sum().sum())}")
    print(f"total_amount={int(df['amount'].sum())}")
    print(f"average_order_amount={df['amount'].mean():.1f}")

    print("\nstore_sales")
    print(df.groupby("store")["amount"].sum().sort_values(ascending=False).to_string())

    print("\ncategory_sales")
    print(df.groupby("category")["amount"].sum().sort_values(ascending=False).to_string())

    print("\ntop_5_hours_by_amount")
    print(df.groupby("hour")["amount"].sum().sort_values(ascending=False).head(5).to_string())


if __name__ == "__main__":
    main()

python3 06_answers/S01_code.py

실행하면 여러 줄이 나오는데, 답안에서 확인할 값은 아래 부분입니다.

rows=10424는 주문 기록이 10,424행이라는 뜻입니다. columns=12는 열이 12개라는 뜻이고, missing_total=0은 비어 있는 값이 없다는 뜻입니다. total_amount=90492500은 전체 매출 합계이고, average_order_amount=8681.2는 주문 1건당 평균 금액입니다.

시간대별 매출 상위 5개는 13시, 12시, 14시, 15시, 16시 순으로 확인되었습니다. 이 말은 각 시간대의 amount를 모두 더했을 때 13시가 가장 컸다는 뜻입니다. 이 결과는 S01의 미션 답안 예시에만 사용하고, 본격적인 피벗테이블 해석은 이후 차시에서 다룹니다.
