S01. 데이터 분석이란, AI 시대의 작업 흐름 + 환경 세팅

차시 개요

  • 대상: 엑셀은 쓰지만 데이터 분석은 처음인 비개발자
  • 시간: 50분
  • 난이도: 1
  • 선수 지식: 없음
  • 사용 데이터: 02_datasets/cafe_sales/cafe_sales_clean.csv
  • 수업의 중심 질문: "내가 가진 표로 어떤 질문에 답할 수 있는가?"

이번 차시는 계산 기술보다 관점을 잡는 시간이다. 데이터 분석은 어려운 프로그램을 익히는 일이라기보다, 표에 담긴 관찰값으로 업무 질문에 답하는 일이다. 엑셀, AI, 파이썬, BI도 서로 경쟁하는 도구가 아니라 같은 질문을 저마다 다른 방식으로 다루는 도구다.

학습 목표

수강생은 이 차시가 끝나면 다음 세 가지를 할 수 있어야 한다.

  1. 데이터로 답할 수 있는 질문과 답하기 어려운 질문을 구분한다.
  2. 엑셀, AI, 파이썬, BI가 데이터 분석 흐름에서 각각 어떤 역할을 하는지 설명한다.
  3. 수업에 필요한 환경을 점검한다. 운영체제, 필요한 계정, 데이터 폴더 수령 여부를 확인한다. Google Colab은 파이썬 파트(S10)에서 처음 쓰므로, 오늘은 Google 계정이 있는지만 확인한다.

오늘 사용할 데이터

이번 차시에서는 카페 주문 데이터를 관찰한다. 분석을 깊이 파고들기보다 표의 구조를 살피고 질문을 만드는 데 집중한다.

  • 파일 경로: 02_datasets/cafe_sales/cafe_sales_clean.csv
  • 실제 확인 행수: 10,424행
  • 실제 확인 열수: 12열
  • 날짜 범위: 2025-01-01부터 2025-06-30까지
  • 주요 열: order_id, date, weekday, hour, store, category, item, unit_price, qty, amount, payment, member
  • 지점: 강남점, 홍대점
  • 카테고리: 커피, 음료, 디저트

엑셀에서 열어 보면 한 행이 주문 한 건, 한 열이 그 주문의 속성이다. 가령 store는 어느 지점에서 팔렸는지, amount는 주문 금액이 얼마인지, date는 언제 팔렸는지를 나타낸다.

핵심 개념 1. 데이터 분석은 질문을 표에 맞게 번역하는 일

데이터 분석은 냉장고 속 재료로 요리하는 일과 비슷하다. 냉장고에 있는 재료가 데이터, 만들고 싶은 음식이 질문, 칼과 팬이 분석 도구다. 재료가 없으면 그 요리를 못 만들듯, 표에 기록되지 않은 내용은 분석으로 곧장 답하기 어렵다.

예를 들어 카페 주문 표에는 주문일, 지점, 상품명, 수량, 금액이 있다. 그래서 다음 질문은 데이터로 답할 수 있다.

  • 2025년 1월부터 6월까지 강남점과 홍대점 중 어느 지점의 매출이 더 높은가?
  • 커피, 음료, 디저트 중 총매출이 가장 큰 카테고리는 무엇인가?
  • 어느 시간대에 주문 금액이 많이 발생하는가?

반대로 다음 질문은 이 표만으로는 답하기 어렵다.

  • 손님이 왜 이 메뉴를 골랐는가?
  • 매장 분위기가 만족도에 어떤 영향을 주었는가?
  • 다음 달 매출이 반드시 얼마가 될 것인가?

이 질문들이 나쁘다는 뜻은 아니다. 다만 지금 표에는 구매 이유도, 만족도도, 매장 분위기도, 다음 달 결과도 담겨 있지 않다. 그래서 분석에 앞서 "이 질문에 답하려면 어떤 열이 더 필요한가?"를 먼저 따져야 한다.

핵심 개념 2. 좋은 분석 질문의 모양

입문 단계에서는 분석 질문을 다음 구조로 바꾸면 좋다.

대상 + 기준 + 비교/집계 + 기간

예시는 다음과 같다.

  • 대상: 카페 주문
  • 기준: 지점별
  • 비교/집계: 총매출 비교
  • 기간: 2025년 1월부터 6월까지
  • 질문: 2025년 1월부터 6월까지 지점별 총매출은 어떻게 다른가?

분석 질문은 막연한 궁금증을 표의 열 이름으로 바꾸는 과정이다. "잘 팔린다"는 말은 amount 합계로 볼 수도 있고, qty 합계로 볼 수도 있으며, order_id 개수로 볼 수도 있다. 분석을 시작하기 전에 무엇을 기준으로 "잘 팔린다"고 할지 정해야 한다.

핵심 개념 3. 엑셀, AI, 파이썬, BI의 역할

엑셀

엑셀은 표를 직접 눈으로 확인하고, 간단한 필터와 피벗테이블로 빠르게 답을 얻는 데 강하다. 이미 많은 수강생이 익숙하게 쓰는 도구이므로 데이터 감각을 잡기에 좋다. 다만 작업 과정이 수작업으로 남기 쉬워서, 같은 분석을 반복하거나 데이터가 커지면 실수 관리가 어려워진다.

AI

AI는 질문을 정리하고, 분석 방향을 제안하고, 코드나 설명을 만들어 주는 보조자 역할을 한다. 하지만 AI가 낸 숫자가 항상 맞는 것은 아니다. 특히 파일 업로드와 데이터 분석 기능은 유료 구독이 필요한 경우가 있다. 수업에서는 표준 AI 도구 1종을 기준으로 설명하되, 무료 대안도 함께 안내한다. 중요한 숫자는 엑셀, 파이썬, BI 중 하나로 반드시 검증해야 한다.

파이썬

파이썬과 pandas는 같은 작업을 코드로 재현할 수 있게 해 준다. 누가 실행해도 같은 결과가 나오며, 정제, 집계, 병합, 시각화 과정을 기록으로 남기기 쉽다. 처음에는 낯설지만 데이터가 커지고 반복 작업이 많아질수록 강점이 커진다. 이 수업에서는 Part 4부터 본격적으로 사용한다.

BI

BI 도구는 대시보드와 공유에 강하다. 한 번 만든 화면을 다른 사람이 필터를 바꾸며 볼 수 있고, 회의나 보고에 적합하다. Power BI Desktop은 Windows 전용이다. Mac 수강생은 Part 5에서 Looker Studio 경로를 대체 경로로 사용한다.

수업 전체 흐름에서 S01의 위치

이 수업은 다음 흐름으로 진행된다.

  1. 엑셀에서 표의 구조와 피벗 감각을 잡는다.
  2. 통계 직관으로 평균, 분포, 상관을 해석한다.
  3. AI를 이용해 분석을 요청하고, 결과를 검증한다.
  4. 파이썬 pandas로 같은 분석을 재현 가능하게 만든다.
  5. BI로 대시보드를 만들고 공유한다.
  6. 마지막에는 분석 결과를 업무 보고 형태로 정리한다.

S01은 이 전체 흐름의 출발점이다. 오늘의 목표는 많은 기능을 배우는 것이 아니라, 앞으로 배울 도구들이 어떤 문제를 해결하는지 큰 지도를 얻는 것이다.

환경 체크

수강생은 다음 항목을 확인한다.

  • 운영체제: Windows 또는 Mac
  • Power BI Desktop 사용 가능 여부: Windows 가능, Mac은 Looker Studio 대체
  • Google 계정: 준비 여부 (파이썬 파트 S10부터 Google Colab에 사용한다)
  • AI 계정: 수업에서 지정한 AI 도구 접속 가능 여부
  • 데이터 폴더: 강사가 배포한 02_datasets 폴더 수령 여부
  • 오늘 파일: 02_datasets/cafe_sales/cafe_sales_clean.csv 존재 여부

실데이터는 강사가 UTF-8로 정리한 가공본을 사전 배포한다. 수강생이 수업 중 공공데이터 사이트에서 직접 다운로드하지 않는다.

오늘은 데이터 분석 기술을 잔뜩 배우는 날이 아니라, 앞으로 다룰 도구들의 지도를 그려 보는 날이라고 말한다.

강사 멘트 예시:

"데이터 분석은 프로그램을 잘 다루는 사람만 하는 일이 아닙니다. 이미 여러분은 엑셀에서 필터를 걸고, 합계를 보고, 피벗테이블을 만들어 본 경험이 있습니다. 오늘은 그 경험을 데이터 분석이라는 언어로 다시 정리하겠습니다. 핵심은 한 가지입니다. 내가 가진 표로 어떤 질문에 답할 수 있는지 판단하는 것입니다."

오늘의 목표 세 가지를 화면에 보여 준다.

  1. 데이터로 답할 질문과 답하기 어려운 질문 구분
  2. 엑셀, AI, 파이썬, BI의 역할 이해
  3. 수업 환경 체크

5-15분. 데이터로 답할 수 있는 질문 구분

cafe_sales_clean.csv를 연다. 엑셀로 먼저 보여 주는 것이 좋다. 첫 행의 열 이름을 읽으며 한 행이 주문 한 건이라는 점을 설명한다.

시연 포인트:

  • 파일 경로: 02_datasets/cafe_sales/cafe_sales_clean.csv
  • 실제 데이터: 10,424행, 12열
  • 날짜 범위: 2025-01-01부터 2025-06-30까지
  • 주요 열: date, weekday, hour, store, category, item, qty, amount
  • 지점: 강남점, 홍대점
  • 카테고리: 커피, 음료, 디저트

강사 멘트 예시:

"이 표를 볼 때 처음부터 계산을 하려고 하지 마십시오. 먼저 이 표가 무엇을 관찰한 표인지 봐야 합니다. 여기서는 한 행이 주문 한 건입니다. 그러면 이 표로 답할 수 있는 질문은 주문 한 건의 속성에서 출발해야 합니다."

화이트보드나 슬라이드에 다음 두 그룹을 나눠 쓴다.

답할 수 있는 질문:

  • 지점별 총매출은 어떻게 다른가?
  • 카테고리별 판매 수량은 어떻게 다른가?
  • 시간대별 매출은 어떻게 다른가?

답하기 어려운 질문:

  • 고객은 왜 이 메뉴를 골랐는가?
  • 고객이 매장 분위기에 만족했는가?
  • 다음 달 매출은 반드시 얼마인가?

핵심 설명:

"답하기 어려운 질문은 틀린 질문이 아닙니다. 다만 지금 표에 그 내용을 담은 열이 없을 뿐입니다. 데이터 분석가는 질문을 버리는 사람이 아니라, 질문을 답할 수 있는 형태로 바꾸거나 필요한 데이터를 추가로 요청하는 사람입니다."

15-25분. 엑셀, AI, 파이썬, BI의 역할 비교

도구 네 가지를 하나씩 설명한다. 수강생이 이미 아는 엑셀에서 출발한다.

강사 멘트 예시:

"엑셀은 빠르게 눈으로 확인하고 피벗테이블을 만들기에 좋습니다. AI는 질문을 정리하고 분석 방향을 제안하는 데 좋습니다. 파이썬은 반복 가능한 분석 기록을 남기는 데 강합니다. BI는 다른 사람이 필터를 바꾸며 볼 수 있는 대시보드를 만들 때 좋습니다."

도구별 주의점도 같이 말한다.

  • 엑셀: 손작업이 많아지면 재현이 어렵다.
  • AI: 그럴듯하지만 틀린 숫자를 낼 수 있으므로 검증해야 한다.
  • 파이썬: 처음에는 낯설지만 반복 작업과 큰 데이터에 강하다.
  • BI: 공유와 대시보드에 강하지만 설치와 계정 환경을 확인해야 한다.

Power BI 안내:

"Power BI Desktop은 Windows 전용입니다. Mac을 쓰는 분은 수업을 못 듣는 것이 아닙니다. Part 5에서 Looker Studio를 대체 경로로 사용합니다."

AI 안내:

"AI 도구의 파일 업로드나 데이터 분석 기능은 유료 구독이 필요한 경우가 있습니다. 수업에서는 표준 도구 기준으로 설명하되, 무료 대안이 가능한 구간은 같이 안내하겠습니다."

35-45분. 미션 진행

수강생이 직접 질문을 만들게 한다. 강사는 계산을 도와주기보다 질문의 모양을 다듬어 준다.

판서할 질문 공식:

대상 + 기준 + 비교/집계 + 기간

좋은 질문 예:

  • 2025년 1월부터 6월까지 지점별 총매출은 어떻게 다른가?

수강생 질문을 받을 때 확인할 점:

  • 질문 안에 비교 기준이 있는가?
  • 어떤 숫자를 집계할지 정했는가?
  • 필요한 열이 현재 표에 있는가?
  • 기간이 필요한 질문이면 기간이 적혀 있는가?

마무리 멘트 예시:

"오늘 만든 질문은 앞으로 모든 차시에서 계속 쓰게 될 기본 습관입니다. 분석 도구를 배우기 전에 먼저 질문을 표의 열로 바꾸어야 합니다. 다음 차시부터는 좋은 데이터의 모양을 보고, 엑셀에서 직접 분석하기 좋은 표와 그렇지 않은 표를 구분하겠습니다."

예상 질문과 답변

Q1. 데이터 분석을 하려면 코딩을 먼저 배워야 하나요?

A. 아니다. 이 수업은 엑셀에서 시작한다. 코딩은 반복 작업과 검증이 필요해질 때 나중에 배운다. 오늘은 질문을 만드는 일이 먼저다.

Q2. AI에게 파일을 올리면 분석이 끝나는 것 아닌가요?

A. AI는 분석을 도와주는 도구이다. 하지만 숫자를 잘못 계산하거나 데이터에 없는 내용을 그럴듯하게 말할 수 있다. 중요한 결과는 엑셀, 파이썬, BI 중 하나로 검증해야 한다.

Q3. Mac이라서 Power BI를 못 쓰면 수업을 따라갈 수 없나요?

A. 따라갈 수 있다. Power BI Desktop은 Windows 전용이므로 Mac 수강생은 Looker Studio 경로로 대체한다. 공유와 대시보드 개념은 그대로 배운다.

Q4. 제 업무 데이터에 필요한 열이 없으면 어떻게 하나요?

A. 그 질문은 현재 표만으로는 답하기 어렵다고 판단하면 된다. 다음 단계는 필요한 열을 추가로 수집하거나, 현재 표로 답할 수 있는 더 좁은 질문으로 바꾸는 것이다.

Q5. 행이 많으면 엑셀로도 충분하지 않나요?

A. 엑셀로 가능한 일이 많다. 다만 작업을 반복하거나, 여러 파일을 합치거나, 정제 과정을 남겨야 할 때는 파이썬이 유리하다. 대시보드로 공유해야 할 때는 BI가 유리하다.

미션 2 모범답안. 내 업무 표 1개 질문화

예시 답안:

내 업무 표 표에 있을 것 같은 열 분석 질문
고객 문의 처리표 문의일, 채널, 문의유형, 담당자, 처리시간, 처리상태 이번 달 문의유형별 평균 처리시간은 어떻게 다른가?

이 질문은 좋은 편이다. 이유는 다음과 같다.

  • 대상이 명확하다: 고객 문의 처리표
  • 기준이 명확하다: 문의유형별
  • 집계할 값이 명확하다: 평균 처리시간
  • 기간이 들어 있다: 이번 달

단, 실제 표에 처리시간이 없다면 이 질문은 바로 답할 수 없다. 그 경우에는 처리 시작 시각과 완료 시각으로 처리시간을 계산하거나, 처리시간 열을 새로 만들어야 한다.

실습 코드

아래 코드는 06_answers/S01_code.py로도 저장되어 있다. 저장된 코드를 실제 실행해 정상 작동을 확인했다.

from pathlib import Path

import pandas as pd


ROOT = Path(__file__).resolve().parents[1]
DATA_PATH = ROOT / "02_datasets" / "cafe_sales" / "cafe_sales_clean.csv"


def main():
    df = pd.read_csv(DATA_PATH)
    df["date"] = pd.to_datetime(df["date"])

    print(f"pandas_version={pd.__version__}")
    print(f"data_path={DATA_PATH}")
    print(f"rows={len(df)}")
    print(f"columns={len(df.columns)}")
    print("column_names=" + ", ".join(df.columns))
    print(f"date_range={df['date'].min().date()} ~ {df['date'].max().date()}")
    print(f"missing_total={int(df.isna().sum().sum())}")
    print(f"total_amount={int(df['amount'].sum())}")
    print(f"average_order_amount={df['amount'].mean():.1f}")

    print("\nstore_sales")
    print(df.groupby("store")["amount"].sum().sort_values(ascending=False).to_string())

    print("\ncategory_sales")
    print(df.groupby("category")["amount"].sum().sort_values(ascending=False).to_string())

    print("\ntop_5_hours_by_amount")
    print(df.groupby("hour")["amount"].sum().sort_values(ascending=False).head(5).to_string())


if __name__ == "__main__":
    main()
