S09. AI 답 검증하기: 환각과 계산오류 잡기

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S07, S03
  • 난이도: 3
  • 시간: 50분
  • 사용 데이터
  • 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 clean 엑셀 파일
  • 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 dirty 엑셀 파일
  • 검증 코드에서는 같은 내용의 CSV, 즉 엑셀로도 열리는 표 파일을 사용한다. 초보자는 엑셀 파일(.xlsx)을 받아도 실습에 충분하다.

이 차시는 AI에게 데이터 분석을 맡긴 뒤, 그 답을 업무에 쓰기 전에 직접 검산하는 방법을 다룹니다. 여기서 검산은 계산이 맞는지 다시 확인하는 일입니다. 마트 영수증 금액을 한 번 더 더해 보듯이, AI가 말한 숫자도 원본 데이터와 엑셀 피벗테이블로 다시 확인해야 합니다.

수강생은 먼저 깨끗한 카페 매출 데이터로 AI 답과 엑셀 피벗테이블을 대조하는 기본 루틴을 익힙니다. 그다음 일부러 오류를 심어 둔 카페 매출 데이터로 넘어가 봅니다. 이때 AI와 피벗테이블이 같은 숫자를 내놓더라도, 데이터 자체가 지저분하면 그 숫자를 바로 결론으로 믿으면 안 된다는 점을 확인합니다.

AI를 쓰지 말자는 수업이 아닙니다. AI가 빠르게 요약해 주는 능력은 충분히 활용합니다. 다만 숫자로 결론을 내릴 때는 피벗테이블과 원본 점검으로 확인하는 습관을 만듭니다. 그래야 “AI가 그렇게 말했습니다”가 아니라 “원본 데이터로도 확인했습니다”라고 말할 수 있습니다.

학습 목표

수강생은 이 차시가 끝나면 아래 일을 직접 할 수 있어야 합니다.

  1. AI가 낸 총매출, 지점별 매출, 카테고리별 매출을 엑셀 피벗테이블로 교차확인한다.
  2. 행수, 기간, 사용 열, 집계 기준을 확인해 AI의 계산오류와 환각을 구분한다.
  3. dirty 데이터에서 중복, 결측, 표기 흔들림, 이상값이 숫자 답을 왜곡하는 방식을 설명한다.
  4. 실제 업무에서 쓸 수 있는 AI 답 검증 체크리스트를 작성한다.

여기서 환각은 AI가 데이터에 없는 내용을 있는 것처럼 말하는 현상입니다. 예를 들어 파일에 없는 매출이익 열을 만들어 말하거나, 실제로 없는 지점을 언급하는 식입니다. 계산오류는 말 그대로 더하기, 묶어 계산하기, 필터 적용 같은 계산 과정에서 숫자가 틀어지는 경우입니다. 두 문제는 대응 방법이 다르기 때문에 구분해서 봐야 합니다.

데이터 이해

이 수업에서 다루는 데이터는 카페 주문 데이터입니다. 주문 한 건이 한 행으로 들어 있습니다. 엑셀에서 한 줄이 영수증 한 장이라고 생각하면 이해하기 쉽습니다. 한 주문마다 주문번호, 주문 날짜, 지점, 상품 카테고리, 단가, 수량, 주문금액 같은 정보가 들어 있습니다.

컬럼 의미 검증에서 보는 이유
order_id 주문번호 중복 주문 확인
date 주문 날짜 기간과 월별 집계 확인
store 지점 지점별 매출 검산
category 상품 카테고리 카테고리별 매출 검산
unit_price 단가 금액 계산 검증
qty 수량 수량 이상값 확인
amount 주문금액 총매출과 피벗의 핵심 값
payment 결제수단 결측 확인
member 회원 여부 결측 확인

amount는 이 수업에서 가장 자주 검산할 금액 열입니다. 총매출을 확인할 때는 이 열을 더합니다. 지점별 매출이나 카테고리별 매출도 결국 amount를 지점이나 카테고리 기준으로 나누어 더한 값입니다.

실제로 세어 본 데이터 규모는 이렇습니다. 아래 표의 파일명은 검증용 표 파일 이름입니다. 실습할 때는 수업 홈의 데이터 받기 페이지에서 같은 카페 매출 엑셀 파일을 내려받아도 됩니다.

파일 행수 열수 특징
cafe_sales_clean.csv 10,424 12 2025-01-01부터 2025-06-30까지의 깨끗한 주문 데이터
cafe_sales_dirty.csv 10,439 12 clean에 중복, 결측, 날짜 형식 혼재, 표기 흔들림, 이상값을 넣은 데이터

깨끗한 데이터의 검산 기준값은 아래처럼 잡습니다. 이 값들은 AI 답을 확인할 때 기준 저울처럼 사용합니다.

  • 총매출: 90,492,500원
  • 판매수량 합계: 17,241개
  • 지점별 매출: 강남점 53,425,000원, 홍대점 37,067,500원
  • 카테고리별 매출: 커피 45,606,000원, 음료 25,550,000원, 디저트 19,336,500원

dirty 데이터를 그대로 합산하면 총매출은 90,600,500원으로 clean보다 108,000원 큽니다. 그런데 이 값은 “원본에 적힌 금액을 그대로 더한 값”일 뿐입니다. 중복 주문, 이상한 수량, 음수 금액 같은 문제가 남아 있다면 최종 확정 매출이라고 말하면 안 됩니다.

꼭 이해할 개념

1. AI 답은 결론이 아니라 검토 대상입니다

AI가 “총매출은 9천만 원입니다”라고 답하면 꽤 그럴듯해 보입니다. 하지만 데이터 분석에서 그럴듯한 문장은 증거가 아닙니다. 숫자는 원본 데이터에서 어떻게 계산했는지 확인되어야 합니다.

AI 답은 신입 동료가 빠르게 만들어 온 요약표처럼 다루면 됩니다. 방향을 잡는 데는 도움이 됩니다. 다만 보고서에 넣거나 의사결정에 쓰기 전에는 원본 데이터로 검산해야 합니다. 특히 매출, 수량, 비율처럼 업무 판단에 바로 쓰이는 숫자는 더 조심해야 합니다.

엑셀 사용자에게 가장 가까운 검산 도구는 피벗테이블입니다. AI에게 자연어로 질문하고, 엑셀 피벗테이블로 같은 질문을 다시 계산합니다. 두 결과가 같으면 다음 점검으로 넘어갑니다. 두 결과가 다르면 AI가 어떤 열을 썼는지, 어떤 필터를 적용했는지, 몇 행을 분석했는지 확인해야 합니다.

2. 검증 루틴은 같은 질문을 두 번 계산하는 일입니다

AI 답 검증은 어려운 기술이 아니라 순서의 문제입니다. 같은 질문을 AI에게 한 번, 엑셀 피벗테이블에게 한 번 물어보는 일이라고 보면 됩니다.

  1. 질문을 고정한다. 예: amount의 총합계, 지점별 합계, 카테고리별 합계
  2. AI에게 같은 질문을 한다.
  3. 엑셀 피벗테이블로 같은 질문을 계산한다.
  4. AI 답과 피벗 기준값의 차이를 표로 적는다.
  5. 차이가 있으면 사용 열, 필터, 행수, 기간, 단위를 확인한다.
  6. dirty 데이터라면 중복, 결측, 표기 흔들림, 이상값을 별도로 확인한다.

여기서 중요한 일은 “AI가 틀렸다”라고만 말하고 끝내지 않는 것입니다. 어떤 기준이 달랐는지 찾아야 합니다. amount를 더해야 하는데 qty를 더했는지, 전체 기간을 봐야 하는데 일부 월만 봤는지, 지점명을 잘못 묶었는지 확인해 보세요. 그래야 AI에게 다시 질문할 때 더 정확하게 요청할 수 있습니다.

3. 엑셀 피벗은 AI 답의 기준 저울입니다

피벗테이블은 원본 데이터를 놓고 합계를 직접 계산합니다. 마치 물건 무게를 저울에 올려 확인하듯이, AI가 말한 숫자를 피벗테이블에 올려 보는 셈입니다.

AI는 내부에서 어떤 방식으로 계산했는지 바로 보이지 않을 때가 있습니다. 어떤 열을 골랐는지, 일부 행만 읽었는지, 날짜를 제대로 읽었는지 확인이 필요합니다. 반면 피벗테이블은 사용자가 행, 열, 값 영역을 직접 지정합니다. 그래서 초보자도 “나는 amount를 합계로 계산했다”라고 기준을 분명히 말할 수 있습니다.

확인할 질문 AI에서 생길 수 있는 문제 피벗에서 확인할 기준
총매출은 얼마인가 amount 대신 qty를 합산하거나 평균을 말함 값에 amount, 집계는 합계
지점별 매출은 얼마인가 지점명을 잘못 합치거나 누락함 행에 store, 값에 amount 합계
카테고리별 매출은 얼마인가 앞뒤 공백이 있는 범주를 놓침 행에 category, 값에 amount 합계
월별 추세는 어떤가 날짜 일부를 파싱하지 못함 month 열을 만들고 월별 합계
몇 행을 분석했는가 일부 행만 읽고 전체처럼 말함 원본 행수와 피벗 총합계 확인

이 표는 AI 답을 볼 때의 질문 목록입니다. “총매출이 맞나요?”라고만 묻지 말고, 어떤 열을 더했는지와 어떤 기준으로 묶었는지를 같이 봐야 합니다.

4. clean에서 맞은 방식이 dirty에서도 충분하지는 않습니다

clean 데이터에서는 AI 답과 피벗 기준값을 대조하는 일이 비교적 단순합니다. 예를 들어 총매출 90,492,500원, 강남점 53,425,000원, 홍대점 37,067,500원이 맞는지 확인하면 됩니다. 데이터가 깨끗하기 때문에 같은 열을 같은 기준으로 더하면 같은 숫자가 나와야 합니다.

dirty 데이터에서는 한 단계 더 봐야 합니다. dirty 원본을 그대로 피벗하면 총매출 90,600,500원이 나옵니다. AI도 같은 값을 낼 수 있습니다. 여기서 “AI와 피벗이 같으니 통과”라고 끝내면 위험합니다. 같은 숫자가 나왔다는 말은 같은 방식으로 더했다는 뜻이지, 데이터가 믿을 만하다는 뜻은 아닙니다.

실제 dirty 데이터에는 아래 문제가 확인됩니다.

  • 주문번호 중복 행: 15행
  • payment 결측: 313행
  • member 결측: 209행
  • 날짜 YYYY/MM/DD 형식: 206행
  • 날짜 MM-DD-YYYY 형식: 104행
  • unit_price에 원 또는 쉼표가 들어간 행: 157행
  • store가 강남 점으로 들어간 행: 310행
  • category 앞뒤 공백: 313행
  • 아메리카노 표기 변형: 106행
  • qty=999: 2행
  • amount 음수: 1행
  • unit_price × qty와 amount 불일치: 3행

이런 문제가 있으면 숫자는 쉽게 흔들립니다. 같은 강남점인데 강남점과 강남 점으로 나뉘면 피벗테이블은 두 지점을 다른 지점으로 봅니다. 수량이 999인 행이 섞이면 판매수량 합계가 크게 튈 수 있습니다. 음수 금액이 들어가면 총매출이 실제보다 줄어들 수도 있습니다.

따라서 dirty 파일에서 올바른 결론은 “총매출은 90,600,500원이다”가 아닙니다. “원본을 그대로 합산하면 90,600,500원이지만, 중복과 이상값이 있어 정제 기준을 정한 뒤 확정해야 한다”라고 말해야 합니다.

5. 환각과 계산오류는 다르게 대응합니다

AI 답의 문제는 한 가지가 아닙니다. 없는 내용을 만들어 내는 경우도 있고, 있는 데이터를 잘못 계산하는 경우도 있습니다. 그래서 답이 이상할 때는 “어떤 종류의 문제인가?”를 먼저 나눠 봐야 합니다.

유형 예시 대응
환각 파일에 없는 profit 열을 만들거나, 없는 지점을 언급함 사용한 열 이름과 원본 근거를 요구한다
계산오류 총합계, 그룹합계, 월별 합계가 피벗과 다름 같은 열과 같은 필터로 다시 계산하게 한다
기준오류 총매출을 합계가 아니라 평균으로 말함 "집계함수는 합계"처럼 기준을 명시한다
데이터품질 누락 중복과 이상값을 무시하고 결론만 말함 답변 전에 품질 점검을 요구한다

환각이라면 원본에 그런 열이나 값이 있는지부터 확인해야 합니다. 계산오류라면 같은 열과 같은 필터로 다시 계산하게 해야 합니다. 기준오류라면 “합계인지 평균인지”처럼 계산 기준을 분명히 적어 줘야 합니다. 데이터품질 누락이라면 AI가 숫자를 말하기 전에 중복, 결측, 이상값을 먼저 보게 해야 합니다.

AI에게 다시 물을 때는 “다시 해 주세요”라고만 쓰면 부족합니다. “내 피벗 기준값은 90,492,500원입니다. amount 열 합계만 사용했는지 확인하고, 적용한 필터와 행수를 함께 알려 주세요”처럼 구체적으로 요청해야 합니다. 이렇게 물으면 AI가 무엇을 다시 확인해야 하는지 분명해집니다.

수업 흐름은 먼저 clean 데이터로 검산 방법을 익히고, 그 방법을 dirty 데이터에 적용해 보는 순서입니다. clean 데이터에서는 숫자 대조에 집중합니다. dirty 데이터에서는 숫자가 맞아 보여도 원본 품질을 확인해야 한다는 점을 몸으로 익힙니다.

엑셀과 연결해서 설명할 지점

  • AI 답의 총매출은 엑셀에서 SUM(amount) 또는 피벗테이블의 총합계와 비교한다. 같은 계산을 엑셀에서 다시 해 보는 것입니다.
  • 지점별, 카테고리별 답은 피벗테이블의 행 영역과 값 영역으로 검산한다. 행 영역은 “무엇별로 나눌지”, 값 영역은 “무엇을 더할지”를 정하는 자리입니다.
  • 강남점과 강남 점처럼 사람 눈에는 비슷한 값도 엑셀과 AI 모두 다른 범주로 볼 수 있다. 컴퓨터는 띄어쓰기 하나도 다른 값으로 볼 수 있습니다.
  • 값이 맞아도 행수, 기간, 필터, 단위가 다르면 같은 분석이 아니다. 10,424행 전체를 봤는지, 2025-01-01부터 2025-06-30까지를 봤는지 같이 확인해야 합니다.
  • dirty 데이터에서는 피벗과 AI가 같은 값을 내도, 중복과 이상값을 점검하기 전에는 확정 결론으로 쓰지 않는다. 같은 숫자는 출발점이지 최종 승인 도장이 아닙니다.

수업 후 수강생이 말할 수 있어야 하는 문장

  • "AI가 총매출을 말하면 먼저 amount 합계인지, 전체 행을 썼는지, 피벗 총합계와 맞는지 확인해야 한다."
  • "clean 파일의 총매출은 90,492,500원이고, 지점별 매출 합계도 이 총합계와 맞아야 한다."
  • "dirty 파일은 원본 합계가 90,600,500원이지만 중복, 표기 흔들림, 이상값이 있으므로 정제 기준 없이 최종 매출이라고 단정하면 안 된다."
  • "AI 답 검증은 AI를 의심하는 태도가 아니라, 숫자를 업무에 쓸 수 있게 만드는 절차다."

이 문장들을 말할 수 있으면 단순히 버튼을 따라 누른 것이 아니라, 검증이 왜 필요한지 이해한 것입니다. 숫자는 맞는지 확인해야 하고, 데이터는 믿을 만한 상태인지 확인해야 합니다. 두 가지가 함께 통과해야 업무에 쓸 수 있습니다.

차시 평가 기준

  • 상: AI 답과 피벗 기준값을 대조하고, dirty 데이터의 품질 문제까지 반영해 결론을 보류하거나 수정한다.
  • 중: clean 파일에서 AI 답과 피벗 총매출을 비교하고, dirty 파일의 주요 오류를 일부 찾는다.
  • 하: AI 답을 그대로 제출하거나, 피벗 기준값과의 차이를 설명하지 못한다.

사용할 파일

수업 홈의 데이터 받기 페이지에서 카페 매출 파일 두 개를 내려받아 사용합니다. 데이터 받기 페이지에 CSV가 함께 보인다면, CSV는 엑셀로도 열리는 표 파일입니다. 처음 실습하는 분은 엑셀 파일(.xlsx)을 받아서 따라 해도 됩니다.

  1. clean 실습 파일 - 카페 매출 clean 파일
  2. dirty 실습 파일 - 카페 매출 dirty 파일

AI 도구의 파일 업로드 기능은 계정이나 요금제에 따라 다를 수 있습니다. 파일을 올릴 수 없으면 AI에는 프롬프트와 검증 체크리스트 작성만 맡기고, 실제 계산은 엑셀 피벗테이블로 해 보세요.

오늘의 목표

AI가 데이터 파일을 분석해 준 답을 그대로 믿지 않고, 엑셀 피벗테이블로 다시 계산해서 맞는지 확인합니다. 숫자는 말로 들으면 그럴듯해 보이지만, 업무에서는 한 번 더 재는 과정이 필요합니다. 키를 눈대중으로 보지 않고 줄자로 재듯이, AI 답도 피벗테이블로 다시 재 봅니다.

검증할 질문은 네 가지입니다.

  1. 전체 행수와 기간은 맞는가?

엑셀 필터를 켭니다.

  2. qty=999인 행이 2개 있는지 확인합니다.
  3. amount 열 필터에서 작은 값 순으로 정렬합니다.
  4. amount=-5500인 행이 1개 있는지 확인합니다.
  5. store 필터에서 강남 점이 따로 있는지 확인합니다.
  6. category 필터에서 앞뒤 공백이 있는 항목이 따로 보이는지 확인합니다.

직접 해보기. AI 답 검증 체크리스트 만들기

이제 직접 판단하는 연습을 해 봅니다. AI가 말한 문장이 그럴듯해 보여도, 보고서에 넣기 전에는 어떤 근거로 통과시킬지 정해야 합니다.

0-5분. 도입: AI가 말한 숫자를 바로 믿지 않기

말할 것:

지난 S07에서는 AI에게 엑셀 파일을 올리고 자연어로 분석을 요청했습니다. 이제는 그다음이 중요합니다. AI가 내놓은 답을 그대로 받아 적는 것이 아니라, 업무에 써도 되는 숫자인지 확인하는 방법을 봅니다.

AI가 숫자를 말하면 사람은 쉽게 안심합니다. "총매출은 90,492,500원입니다"라는 답은 꽤 그럴듯해 보이죠. 그런데 데이터 분석에서는 그럴듯한 문장을 바로 결론으로 쓰면 안 됩니다. 숫자는 키를 잴 때처럼 기준이 분명해야 합니다. 어떤 열을 더했는지, 전체 행을 다 봤는지, 필터가 걸려 있었는지 확인해야 합니다.

오늘은 같은 질문을 AI와 엑셀 피벗테이블에 각각 던져 봅니다. 그리고 두 결과가 같은지, 다르다면 왜 다른지 비교합니다. 여기서 피벗테이블은 AI 답을 검산하는 저울 역할을 합니다.

강조할 문장:

AI 답은 결론이 아니라 검토 대상입니다. 피벗테이블은 그 답을 검산하는 기준 저울입니다.

5-12분. 개념 설명: 같은 질문을 두 번 계산하기

말할 것:

검증은 어려운 기술이 아닙니다. 순서대로 확인하는 일입니다. 먼저 질문을 고정합니다. 예를 들어 "총매출"이라고 말할 때는 amount 열의 합계를 뜻한다고 정합니다. amount는 주문금액이 들어 있는 열입니다. 그러니까 총매출을 보려면 이 열을 더해야 합니다.

그다음 AI에게 답을 받습니다. 그리고 엑셀 피벗테이블로 같은 계산을 다시 합니다. AI가 계산한 값과 피벗테이블 값이 같으면 계산 검산은 통과한 것입니다. 다르면 바로 "AI가 틀렸네"라고 끝내지 않습니다. 어떤 열을 썼는지, 필터를 걸었는지, 전체 행을 읽었는지 차근차근 물어봐야 합니다.

오늘 쓸 검증 루틴은 다섯 단계입니다.

  1. 질문을 숫자로 바꾼다.
  2. AI 답을 받는다.
  3. 피벗 기준값을 만든다.
  4. 차이표를 작성한다.
  5. dirty 데이터에서는 데이터품질 문제를 추가로 점검한다.

엑셀 비교:

엑셀에서 SUM 함수나 피벗테이블은 계산 과정이 눈에 보입니다. SUM은 선택한 숫자를 모두 더하는 엑셀 함수입니다. 피벗테이블은 지점별, 카테고리별처럼 묶어서 합계를 볼 때 쓰는 도구입니다.

AI는 자연어로 답해 주기 때문에 편합니다. 하지만 계산 기준이 화면에 바로 보이지 않을 때가 있습니다. 사전에서 단어 뜻을 찾듯이, 숫자도 근거를 다시 찾아봐야 합니다. 그래서 AI를 분석 보조자로 쓰고, 최종 숫자는 엑셀로 확인합니다.

만약 다르면 원인을 찾아야 합니다. AI가 다른 열을 썼을 수도 있고, 일부 행만 읽었을 수도 있고, 합계 대신 평균을 말했을 수도 있습니다. 이때는 AI를 비난하는 시간이 아닙니다. 질문을 더 좁혀서 다시 확인하는 시간입니다. "amount 열만 합산했나요?", "분석한 행수가 10,424행인가요?"처럼 물어보면 됩니다.

말할 것:

여기서 여러분의 역할은 AI 답을 혼내는 사람이 아닙니다. 보고서에 들어가도 되는 숫자인지 확인하는 사람입니다. 숫자가 맞는지, 맞더라도 조건이 붙어야 하는지 판단해야 합니다.

"총매출은 90,600,500원"이라는 문장은 원본 합계로는 맞을 수 있습니다. 그런데 이 데이터에는 중복, 표기 흔들림, 이상값이 있습니다. 그래서 최종 보고서에 그대로 넣어도 되는지는 따로 판단해야 합니다.

수강생에게 요구할 것:

  1. 검증 체크리스트를 최소 8개 항목으로 작성한다.
  2. 각 항목에 통과, 불일치, 확인 필요를 쓴다.
  3. 마지막에 3문장으로 보고서 사용 가능 여부를 판단한다.

돌아다니며 볼 포인트:

  • 총매출 90,600,500원을 무조건 틀렸다고 쓰지 않았는가. 원본 합계로는 맞는 값이다.
  • 그러나 최종 확정값이라고 쓰면 안 된다는 판단이 들어 있는가.
  • 중복, 표기 흔들림, 이상값 중 최소 3가지를 언급했는가.

도와줄 때 할 말:

총매출이 맞는지 보는 것과, 그 총매출을 보고서에 확정값으로 써도 되는지 보는 것은 다릅니다. 계산 검산은 계산기 두 대로 같은 답이 나오는지 보는 일이고, 데이터품질 검증은 계산기에 넣은 재료가 믿을 만한지 보는 일입니다.

체크리스트에는 행수, 사용한 금액 열, 총매출, 지점별 합계, 카테고리별 합계, 주문번호 중복, 수량 이상값, 음수 금액 같은 항목이 들어가면 좋습니다. 각 항목 옆에는 통과, 불일치, 확인 필요 중 하나를 적어 보세요.

47-50분. 마무리: 검산을 통과한 숫자만 보고서에 넣기

말할 것:

오늘 가져갈 문장은 짧습니다. AI 답은 빠르지만 검산이 필요합니다. 깨끗한 데이터에서는 피벗테이블과 숫자가 맞는지 확인합니다. 오류가 들어 있는 데이터에서는 숫자가 맞아도 데이터 자체가 믿을 만한지 확인합니다.

AI가 말한 숫자를 그대로 복사하면 빠르기는 합니다. 하지만 보고서에 들어가는 숫자는 나중에 누가 물어봐도 설명할 수 있어야 합니다. "AI가 그렇게 말했습니다"가 아니라, "amount 열을 전체 10,424행 기준으로 합산했고, 피벗테이블과 일치했습니다"라고 말할 수 있어야 합니다.

마지막 문장:

보고서에 들어갈 숫자는 "AI가 말한 숫자"가 아니라 "검증 과정을 통과한 숫자"여야 합니다.

완료 판단 체크포인트:

수강생이 clean 총매출 90,492,500원은 피벗으로 검산 통과한 값이고, dirty 원본 총매출 90,600,500원은 중복·표기 흔들림·이상값 때문에 최종 확정값으로는 보류해야 한다고 설명하면 완료로 본다.

예상 질문과 답변

Q1. AI 답과 피벗 값이 같으면 바로 믿어도 되나요?

깨끗한 데이터라면 계산 검산은 통과했다고 볼 수 있습니다. 같은 질문을 AI와 피벗테이블에 던졌고 같은 숫자가 나왔기 때문입니다.

하지만 dirty 데이터, 즉 문제가 섞여 있는 데이터라면 데이터품질 점검이 남아 있습니다. 같은 원본을 그대로 더하면 AI와 피벗테이블이 둘 다 같은 문제를 안고 같은 숫자를 낼 수 있습니다. 계산이 맞는 것과 결론이 안전한 것은 구분해야 합니다.

Q2. AI가 피벗과 다른 총매출을 말하면 어떻게 하나요?

먼저 AI에게 사용한 열 이름, 필터, 행수를 물어봅니다. 예를 들어 "amount 열만 합산했는지, 분석한 행수가 10,424행인지 확인해 주세요"라고 다시 요청합니다.

그래도 다르면 피벗 기준값을 우선합니다. 피벗테이블은 우리가 직접 amount 열을 값 영역에 넣고 합계를 확인한 결과입니다. AI 답은 그 기준값에 맞춰 다시 계산하도록 요청하면 됩니다.

Q3. 오류가 들어 있는 데이터의 총매출 90,600,500원은 맞는 값인가요, 틀린 값인가요?

원본을 아무 정제 없이 그대로 합산한 값으로는 맞습니다. 그래서 이 숫자를 무조건 틀렸다고 하면 안 됩니다.

다만 중복과 이상값이 있으므로 최종 매출로 확정할 수는 없습니다. 실제 업무라면 중복 제거 기준, 이상값 처리 기준, 음수 금액 처리 기준을 먼저 정해야 합니다. 그 기준을 적용한 뒤에 최종 보고서 숫자를 다시 계산해야 합니다.

Q4. 주문번호 중복을 엑셀에서 보면 왜 30행처럼 보이나요?

COUNTIF로 중복 주문번호를 찾으면 원본 행과 복제된 행이 모두 표시되므로 30행이 보일 수 있습니다. 예를 들어 같은 주문번호가 두 번 있으면, 두 행 모두 "중복"으로 잡힙니다.

"두 번째 이후 중복 행"으로 세면 15행이고, "중복에 걸린 전체 행"으로 세면 30행입니다. 둘 다 볼 수 있는 숫자입니다. 그래서 중복을 말할 때는 어떤 기준으로 셌는지 함께 말해야 합니다.

Q5. AI에게 데이터 정리까지 맡겨도 되나요?

초안은 맡길 수 있습니다. 예를 들어 AI에게 "어떤 품질 문제가 있는지 목록으로 찾아 달라"고 요청할 수 있습니다. 하지만 정리 기준은 사람이 정해야 합니다.

강남 점을 강남점으로 합칠지, qty=999를 삭제할지 수정할지는 업무 규칙의 문제입니다. AI는 후보를 제안할 수 있지만, 최종 기준은 분석 목적과 회사 규칙에 맞춰 사람이 결정해야 합니다.

Q6. 파일 업로드가 안 되는 AI 도구를 쓰는 수강생은 어떻게 하나요?

계산은 엑셀 피벗테이블로 하면 됩니다. AI에는 검증 체크리스트를 만들게 하거나, 이미 계산한 숫자를 바탕으로 보고서 문장을 다듬게 하면 됩니다.

이 차시에서 중요한 것은 특정 AI 도구 사용법이 아닙니다. AI가 말한 숫자를 업무에 쓰기 전에 행수, 사용 열, 합계, 데이터품질을 확인하는 절차입니다.

S09 모범답안

미션 모범답안

동료가 AI로 낸 숫자를 보고서에 바로 넣기 전에, 먼저 “계산은 맞는가”와 “그 계산을 믿어도 되는가”를 나누어 봐야 합니다. 계산기가 숫자를 맞게 더했더라도, 저울 위에 올린 재료가 잘못되어 있으면 결론도 조심해야 하는 것과 같습니다.

상황:

동료가 AI로 cafe_sales_dirty.xlsx를 분석했고,
"총매출은 90,600,500원이며 6월 매출이 가장 높습니다"라고 보고했다.
이 답을 그대로 보고서에 넣어도 되는지 검토해야 한다.

모범 판단:

원본을 그대로 합산한 총매출 90,600,500원은 엑셀 피벗 기준값과 일치한다.
그러나 dirty 파일에는 주문번호 중복, 지점명과 카테고리 표기 흔들림, qty=999, amount 음수, 단가×수량과 금액 불일치가 확인된다.
따라서 이 값은 "정제 전 원본 합계"로는 말할 수 있지만, 정제 기준을 정하기 전에는 보고서의 최종 매출로 쓰면 안 된다.

이 답안에서 중요한 부분은 “90,600,500원이 맞다”에서 멈추지 않는 점입니다. 엑셀 피벗테이블로 원본을 그대로 더하면 그 숫자가 나옵니다. 그래서 계산 자체는 통과로 볼 수 있습니다.

하지만 dirty 파일은 일부러 지저분하게 만든 데이터입니다. 주문이 중복되어 있거나, 같은 강남점이 강남점과 강남 점처럼 다르게 적혀 있거나, 수량이 비정상적으로 큰 값으로 들어가 있습니다. 이런 상태에서는 “원본을 그대로 더한 값”과 “보고서에 써도 되는 최종 매출”을 반드시 구분해야 합니다.

검증 체크리스트 작성 예

검증 체크리스트는 AI 답을 채점하는 표라고 보면 됩니다. 답이 맞는지 틀리는지만 보는 것이 아니라, 어떤 기준으로 계산했는지도 함께 봅니다. 예를 들어 금액 열로 amount를 썼는지, 전체 행을 다 썼는지, 중복이나 이상값을 확인했는지를 하나씩 확인합니다.

좋은 답안은 단순히 숫자를 다시 적는 답안이 아닙니다. AI가 말한 값과 피벗 기준값이 일치하는 부분은 인정하고, 그래도 최종 결론으로 쓰면 안 되는 이유를 설명해야 합니다.

  • AI 답과 피벗 기준값이 일치하는 항목을 구분한다.
  • "원본 그대로 합산한 값"과 "최종 확정값"을 구분한다.
  • dirty 데이터의 품질 문제를 최소 3가지 이상 언급한다.
  • 최종 판단을 사용 가능, 조건부 사용, 사용 보류 중 하나로 명확히 쓴다.

여기서는 사용 보류에 가깝습니다. 원본 합계는 말할 수 있지만, 보고서의 최종 매출로 확정하려면 중복과 이상값을 어떻게 처리할지 먼저 정해야 합니다.

카테고리도 같은 문제가 있습니다. 앞뒤에 공백이 붙어 있으면 커피와 커피가 서로 다른 카테고리처럼 잡힙니다. 사람은 대충 같은 말로 읽지만, 데이터는 대충 읽어 주지 않습니다.

해석:

dirty 파일은 원본을 그대로 합산하면 90,600,500원이 맞다. 그러나 지점명 `강남 점`, category 앞뒤 공백, 중복 주문, qty=999, 음수 금액이 있어 이 값을 최종 확정 매출이라고 말하면 안 된다. 피벗 검산은 "AI가 원본을 제대로 더했는가"를 확인하고, 데이터품질 점검은 "그 원본 합계를 믿어도 되는가"를 확인한다.

피벗 검산과 데이터품질 점검은 역할이 다릅니다. 피벗 검산은 AI가 계산을 제대로 했는지 보는 과정입니다. 데이터품질 점검은 그 계산에 사용한 원본이 믿을 만한지 보는 과정입니다.

AI에게 다시 물을 때의 모범 프롬프트

AI 답이 엑셀 피벗 기준값과 다를 때는 “틀렸어요”라고만 말하지 말고, 비교 기준을 같이 알려 주는 것이 좋습니다. 그래야 AI가 어떤 열을 다시 봐야 하는지, 어떤 행을 빠뜨렸는지 점검할 수 있습니다.

실행한 검증 코드

아래 코드는 수업 답안 자료에 있는 검증 코드입니다. 초보자는 코드를 모두 이해하지 않아도 괜찮습니다. 여기서는 “엑셀 피벗으로 확인한 값과 코드로 확인한 값이 같은지”를 보는 용도로 사용합니다.

실행 명령:

검산에 사용한 코드:

from pathlib import Path

import pandas as pd


ROOT = Path("/mnt/d/ai-data-analysis-course")
CLEAN_CSV = ROOT / "02_datasets/cafe_sales/cafe_sales_clean.csv"
DIRTY_CSV = ROOT / "02_datasets/cafe_sales/cafe_sales_dirty.csv"

clean = pd.read_csv(CLEAN_CSV, encoding="utf-8-sig")
dirty = pd.read_csv(DIRTY_CSV, encoding="utf-8-sig")

for df in [clean, dirty]:
    df["amount_num"] = pd.to_numeric(df["amount"], errors="coerce")
    df["qty_num"] = pd.to_numeric(df["qty"], errors="coerce")
    df["unit_price_num"] = pd.to_numeric(
        df["unit_price"].astype(str).str.replace(",", "", regex=False).str.replace("원", "", regex=False),
        errors="coerce",
    )
    df["expected_amount"] = df["unit_price_num"] * df["qty_num"]
    df["date_parsed"] = pd.to_datetime(df["date"], format="mixed", errors="coerce")
    df["month"] = df["date_parsed"].dt.to_period("M").astype(str)
    df["store_norm"] = df["store"].astype(str).str.replace(" ", "", regex=False)
    df["category_norm"] = df["category"].astype(str).str.strip()

clean_total = int(clean["amount_num"].sum())
dirty_total = int(dirty["amount_num"].sum())

clean_store = clean.pivot_table(index="store", values="amount_num", aggfunc="sum")
clean_category = clean.pivot_table(index="category", values="amount_num", aggfunc="sum")
clean_monthly = clean.groupby("month")["amount_num"].sum()

dirty_store_raw = dirty.pivot_table(index="store", values="amount_num", aggfunc="sum")
dirty_category_raw = dirty.pivot_table(index="category", values="amount_num", aggfunc="sum")
dirty_store_norm = dirty.pivot_table(index="store_norm", values="amount_num", aggfunc="sum")
dirty_category_norm = dirty.pivot_table(index="category_norm", values="amount_num", aggfunc="sum")

dirty_issues = {
    "order_id_duplicated_after_first": int(dirty["order_id"].duplicated().sum()),
    "payment_missing": int(dirty["payment"].isna().sum()),
    "member_missing": int(dirty["member"].isna().sum()),
    "store_spaced": int((dirty["store"] == "강남 점").sum()),
    "category_spaced": int((dirty["category"].astype(str) != dirty["category"].astype(str).str.strip()).sum()),
    "qty_999": int((dirty["qty_num"] == 999).sum()),
    "amount_negative": int((dirty["amount_num"] < 0).sum()),
    "amount_mismatch": int((dirty["expected_amount"] != dirty["amount_num"]).sum()),
}

print(clean_total)
print(dirty_total)
print(clean_store)
print(clean_category)
print(clean_monthly)
print(dirty_store_raw)
print(dirty_category_raw)
print(dirty_store_norm)
print(dirty_category_norm)
print(dirty_issues)

코드가 확인한 값도 엑셀 피벗으로 본 값과 맞습니다. 그래서 이 수업에서는 피벗테이블과 코드 검산이 서로 같은 기준을 보고 있다고 이해하면 됩니다.
