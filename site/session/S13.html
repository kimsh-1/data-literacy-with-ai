S13. 그룹화·집계·재구조화

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S12 pandas 파일 읽기, 기본 전처리, 인코딩 처리
  • 난이도: 4
  • 시간: 70분
  • 사용 데이터:
  • 수업 홈의 데이터 받기 페이지에서 내려받은 주문 내역 파일
  • 수업 홈의 데이터 받기 페이지에서 내려받은 고객 정보 파일
  • 수업 홈의 데이터 받기 페이지에서 내려받은 상품 정보 파일
  • 수업 홈의 데이터 받기 페이지에서 내려받은 서울 음식점 상가업소 파일
  • 수업 홈의 데이터 받기 페이지에서 내려받은 법정동 연령·성별 인구 파일
  • 수업 홈의 데이터 받기 페이지에서 내려받은 서울 자치구별 생활인구 일별 파일

여러 행을 기준별 요약표로 바꾸는 방법을 배웁니다. 엑셀에서 피벗 테이블, 부분합, VLOOKUP을 써 보셨다면 이미 중요한 감각은 가지고 계신 겁니다. 파이썬의 표 다루기 도구인 pandas에서는 그 일을 groupby, merge, pivot_table, melt라는 이름으로 합니다.

데이터 받기 페이지에서는 CSV를 받을 수 있습니다. CSV는 엑셀로도 열리는 표 파일입니다. 초보자라면 같은 데이터가 엑셀 파일(.xlsx)로 함께 제공될 때 엑셀 파일을 받아도 됩니다. 중요한 것은 파일 형식보다 “이 표에 어떤 열이 있고, 어떤 표와 붙여야 하는가”입니다.

앞선 차시에서 다룬 카페 매출 데이터는 잠시 내려놓습니다. 이 차시에서는 여러 표를 함께 다루어야 하므로 주문 내역, 고객 정보, 상품 정보로 나누어진 데이터를 사용합니다.

수업의 중심 데이터는 주문·고객·상품 3개 표입니다. 주문 표만 보면 고객 지역과 상품 카테고리가 없습니다. 매출 컬럼도 바로 들어 있지 않습니다. 그래서 고객 표와 상품 표를 주문 표에 붙인 뒤, 단가와 수량과 할인율을 이용해 매출을 계산해야 합니다. 엑셀로 비유하면 주문표에 고객정보와 상품정보를 VLOOKUP으로 끌어온 뒤, 그 결과로 피벗 테이블을 만드는 흐름입니다.

후반에는 실제 공공데이터를 다룹니다. 231열짜리 인구 파일은 melt로 세로형 데이터로 바꾸어 봅니다. 서울 음식점 상가업소 119,158행은 업종별로 집계합니다. 마지막 미션에서는 서울 생활인구 77,792행을 자치구 단위로 요약합니다. 큰 표를 바로 그래프로 그리는 것이 아니라, 먼저 읽을 수 있는 크기의 요약표로 줄이는 연습을 합니다.

학습 목표

  1. groupby()로 두 개 이상의 기준 열을 묶고 여러 집계값을 한 번에 계산할 수 있습니다.
  2. merge()를 엑셀의 VLOOKUP 또는 XLOOKUP과 연결해 이해하고, 병합 후 누락값을 확인할 수 있습니다.
  3. pivot_table()로 행 기준과 열 기준이 모두 있는 요약표를 만들 수 있습니다.
  4. melt()로 가로로 넓은 wide 데이터를 세로로 긴 long 데이터로 바꿀 수 있습니다.
  5. 수만 행 이상의 실데이터를 바로 시각화하지 않고, 먼저 필터링하거나 집계해 줄이는 습관을 갖습니다.

아래 표는 수업에서 실제로 확인한 데이터의 크기입니다. 표의 이름은 데이터 받기 페이지에서 어떤 자료를 내려받아야 하는지 구분하기 위한 이름으로 보면 됩니다. 학생 입장에서 더 중요한 것은 “이 파일이 무슨 역할을 하는가”입니다.

꼭 알아야 할 개념

1. groupby는 부분합과 피벗의 출발점입니다

엑셀에서 지역별 합계, 카테고리별 평균, 부서별 인원수를 구할 때는 부분합이나 피벗 테이블을 씁니다. pandas에서는 먼저 “무엇을 기준으로 묶을지”를 정합니다. 사람 키를 잴 때 반별로 줄을 세운 뒤 평균 키를 구하는 것과 비슷합니다. 먼저 반을 나누고, 그다음에 평균을 계산하는 것이죠.

df.groupby("region")

여기서 df는 pandas에서 표를 담고 있는 데이터프레임입니다. region은 지역 열입니다. 이 한 줄은 “지역별로 묶을 준비를 하라”는 뜻입니다. 아직 합계나 평균을 계산한 것은 아닙니다.

기준이 하나가 아니라 둘일 수도 있습니다.

df.groupby(["region", "category"])

이 코드는 “지역별로 나누고, 그 안에서 다시 카테고리별로 나누라”는 뜻입니다. 엑셀 피벗 테이블에서 행 영역에 region과 category를 차례로 넣는 것과 비슷합니다. 예를 들어 서울특별시 안의 디지털, 서울특별시 안의 패션, 경기도 안의 디지털처럼 묶음이 더 잘게 나뉩니다.

그 다음에는 무엇을 계산할지 정합니다.

df.groupby(["region", "category"]).agg(
    주문건수=("order_id", "nunique"),
    판매수량=("qty", "sum"),
    매출합계=("sales", "sum"),
    평균주문금액=("sales", "mean"),
)

agg()는 여러 집계를 한 번에 만드는 도구입니다. 엑셀 피벗에서 값 영역에 “주문건수”, “수량 합계”, “매출 합계”, “매출 평균”을 함께 넣는 상황과 같습니다.

여기서 주문건수=("order_id", "nunique")는 주문번호가 몇 종류인지 세라는 뜻입니다. 같은 주문번호가 여러 번 나올 수 있을 때 단순히 행 개수를 세는 것보다 고유한 주문번호 개수를 세는 편이 맞습니다. 판매수량=("qty", "sum")은 수량을 모두 더합니다. 매출합계=("sales", "sum")은 매출을 모두 더합니다. 평균주문금액=("sales", "mean")은 주문 한 건이 평균적으로 얼마인지 보는 값입니다.

2. merge는 pandas의 VLOOKUP입니다

주문 내역 파일에는 고객 지역이 없습니다. 대신 customer_id가 있습니다. 고객 정보 파일에는 같은 customer_id와 함께 region이 있습니다. 두 표를 customer_id로 붙이면 주문마다 고객 지역을 가져올 수 있습니다. 사전에서 단어를 찾듯이, 주문표의 고객ID를 들고 고객표에서 해당 고객의 정보를 찾아 붙이는 일입니다.

엑셀식으로 말하면 이렇게 연결됩니다.

엑셀 작업 pandas 작업
주문표에 고객ID 기준으로 고객 지역을 VLOOKUP orders.merge(customers, on="customer_id", how="left")
주문표에 상품ID 기준으로 상품 카테고리와 단가를 VLOOKUP .merge(products, on="product_id", how="left")
조회 실패한 행 확인 병합 후 isna().sum() 확인

입문 단계에서 가장 중요한 일은 병합 뒤 행수가 달라졌는지, 필요한 값이 비지는 않았는지 확인하는 것입니다. 표를 붙이는 일은 겉으로는 간단해 보여도, 키가 잘못되면 행이 늘어나거나 값이 비어 버릴 수 있습니다.

이번 데이터는 주문 20,000행이 병합 후에도 20,000행으로 유지되고, 지역·카테고리·단가 누락값이 0개임을 확인했습니다. 이 말은 주문표의 모든 주문이 고객 정보와 상품 정보를 제대로 찾았다는 뜻입니다.

3. 매출은 병합 뒤 계산합니다

주문 내역 파일에는 매출 컬럼이 없습니다. 단가는 상품 표에 있고, 수량과 할인율은 주문 표에 있습니다. 그래서 상품 표까지 붙인 뒤에야 매출을 계산할 수 있습니다.

수식으로 쓰면 이렇게 됩니다.

sales = price * qty * (1 - discount_rate)

엑셀에서도 흐름은 같습니다. 먼저 VLOOKUP으로 단가를 가져옵니다. 그다음에 단가 * 수량 * (1 - 할인율) 수식을 만듭니다.

여기서 1 - discount_rate는 할인 후 실제로 받는 비율입니다. 할인율만큼 깎고 남은 금액을 계산하는 부분입니다. 그래서 매출은 단가와 수량만 곱하는 것이 아니라, 할인율까지 반영해서 계산해야 합니다.

4. pivot_table은 결과를 가로형 요약표로 펼칩니다

groupby 결과는 보통 세로로 긴 표입니다. 지역과 카테고리가 행으로 쌓여 있기 때문에 컴퓨터가 다루기에는 좋지만, 사람이 한눈에 비교하기에는 불편할 수 있습니다. 그럴 때 지역을 행에 두고 카테고리를 열에 둔 피벗 형태가 편합니다.

pd.pivot_table(
    df,
    index="region",
    columns="category",
    values="sales",
    aggfunc="sum",
    fill_value=0,
)

여기서 index는 피벗의 행입니다. columns는 피벗의 열입니다. values는 계산할 값입니다. aggfunc는 계산 방식입니다. fill_value는 비어 있는 칸을 어떤 값으로 채울지 정합니다.

엑셀 피벗 테이블과 거의 같은 사고방식입니다. 지역을 행 영역에 넣고, 카테고리를 열 영역에 넣고, 매출을 값 영역에 넣은 뒤 합계를 구한다고 보면 됩니다. 결과를 보면 “서울특별시의 디지털 매출”, “경기도의 패션 매출”처럼 행과 열이 만나는 칸마다 값이 들어갑니다.

5. melt는 넓은 표를 긴 표로 바꿉니다

공공데이터에는 열이 지나치게 많은 파일이 자주 있습니다. 이번 인구 파일은 231열입니다. 0세남자, 1세남자, 2세남자처럼 연령과 성별이 열 이름 안에 들어 있습니다.

눈으로 훑어볼 때는 넓은 표가 익숙합니다. 엑셀 보고서처럼 오른쪽으로 계속 펼쳐져 있기 때문입니다. 하지만 분석과 시각화에는 긴 표가 더 다루기 쉽습니다. 나이와 성별을 열 이름 속에 숨겨 두지 않고, 각각 값으로 내려놓아야 필터링하고 묶기 쉬워집니다.

wide 형태 long 형태
0세남자, 1세남자, 2세남자가 각각 별도 열 연령성별 열에 0세남자, 1세남자가 값으로 들어감
열이 너무 많아 필터와 그래프가 어려움 연령, 성별, 인구수 열로 필터와 그룹화가 쉬움
엑셀 보고서처럼 보임 데이터베이스와 BI 도구가 좋아하는 형태

이번 수업에서는 전체 파일을 바로 길게 만들지 않고, 시도명 == "서울특별시" 행 452개만 골라 melt합니다. 그 결과 100,344행의 long 데이터가 됩니다. 원리는 전체 파일에도 똑같이 적용됩니다.

여기서 행수가 늘어나는 것은 이상한 일이 아닙니다. 예전에는 한 행 안에 여러 나이·성별 값이 가로로 들어 있었고, melt 후에는 그 값들이 세로로 내려오기 때문입니다. 책장 한 칸에 옆으로 꽂혀 있던 책들을 바닥에 한 줄로 늘어놓는 것처럼, 보관 방식만 바뀐다고 보면 됩니다.

6. 큰 파일은 먼저 줄이고 봅니다

서울 음식점 상가업소 파일은 119,158행이고, 생활인구 파일은 77,792행입니다. 이 정도는 pandas로 읽을 수 있습니다. 하지만 입문 수업에서 원자료를 그대로 화면에 많이 출력하거나 바로 그래프로 그리면 오히려 무엇을 봐야 하는지 흐려집니다.

큰 파일은 지도 전체를 한 번에 들여다보는 것과 비슷합니다. 처음부터 모든 골목을 보려고 하면 길을 잃기 쉽습니다. 먼저 필요한 지역이나 기간만 고르고, 그다음에 업종별 개수나 자치구별 평균처럼 요약해서 봐야 의미가 드러납니다.

큰 파일을 다룰 때의 순서는 아래 흐름으로 잡아 보세요.

  1. 행수와 열 이름을 확인합니다.
  2. 필요한 열만 고릅니다.
  3. 필요한 지역이나 기간만 필터링합니다.
  4. groupby로 먼저 요약합니다.
  5. 요약표를 보고 해석하거나 시각화합니다.

이 순서를 지키면 데이터가 커져도 덜 흔들립니다. 화면에 많은 행을 출력하는 것이 분석이 아니라, 질문에 맞게 줄인 표를 만드는 것이 분석의 시작입니다.

엑셀과 연결해서 설명할 지점

  • merge는 VLOOKUP 또는 XLOOKUP입니다. 엑셀에서는 한 열씩 찾아오는 느낌이 강하지만, pandas는 여러 열을 한꺼번에 붙일 수 있습니다.
  • 병합 키는 엑셀의 조회값과 기준열입니다. 키가 틀리면 원하는 값이 붙지 않습니다.
  • groupby는 부분합 또는 피벗의 계산 단계입니다. 어떤 기준으로 나누어 계산할지 정하는 일입니다.
  • pivot_table은 groupby 결과를 사람이 보기 좋은 행×열 표로 펼치는 단계입니다.
  • melt는 여러 열에 흩어진 값을 한 열로 모으는 작업입니다. 엑셀의 “파워쿼리 열 피벗 해제”와 비슷합니다.
  • 큰 공공데이터는 원본을 그대로 보려 하지 말고, 집계표로 줄여서 봅니다.

수업 후 수강생이 말할 수 있어야 하는 문장

  • “주문표에 없는 고객 지역과 상품 단가는 merge로 붙입니다.”
  • “병합 후에는 행수와 누락값을 확인해야 합니다.”
  • “groupby는 어떤 기준으로 묶을지 정하고, agg는 무엇을 계산할지 정합니다.”
  • “pivot_table은 지역×카테고리 같은 교차표를 만드는 방법입니다.”
  • “melt는 열이 너무 많은 wide 데이터를 분석하기 쉬운 long 데이터로 바꾸는 방법입니다.”

따라하기 2: 매출 계산하기

계산식은 아래와 같다.

매출 = 단가 * 수량 * (1 - 할인율)

1 - 할인율은 실제로 고객이 결제하는 비율입니다. 예를 들어 할인율이 0.1이면 10% 할인이라는 뜻이고, 고객은 원래 가격의 90%를 냅니다. 그래서 단가에 수량을 곱한 뒤 할인 후 비율을 다시 곱합니다.

전체 매출 합계가 1,938,489,803으로 나오면 계산이 잘 된 것입니다. 이 숫자는 20,000건 주문의 매출을 모두 더한 값입니다.

따라하기 3: 지역×카테고리별 다중 집계하기

이 단계에서는 20,000건의 주문을 지역과 카테고리 기준으로 요약합니다. 원자료를 한 줄씩 보는 대신, "서울특별시의 디지털 상품 매출은 얼마인가?", "경기도의 패션 주문은 몇 건인가?"처럼 비교하기 쉬운 표로 줄이는 작업입니다.

읽는 법:

  • groupby(["region", "category"]): 지역별로 나눈 뒤 카테고리별로 다시 나눈다.
  • 주문건수=("order_id", "nunique"): 주문번호의 고유 개수를 센다.
  • 판매수량=("qty", "sum"): 수량을 합한다.
  • 매출합계=("sales", "sum"): 매출을 합한다.
  • 평균주문금액=("sales", "mean"): 주문 한 건의 평균 매출을 구한다.

nunique는 같은 주문번호를 중복해서 세지 않고, 서로 다른 주문번호가 몇 개인지 세는 방법입니다. 사전에서 같은 단어를 여러 번 만나도 단어 종류는 한 번만 세는 것과 비슷합니다.

결과를 보면 서울특별시 디지털 카테고리는 주문건수 612건, 판매수량 658개, 매출합계 210356387입니다. 이 표는 "누가 많이 팔렸는가"보다 "어떤 지역과 어떤 카테고리 조합이 큰가"를 보는 표입니다.

따라하기 4: 지역×카테고리 피벗 테이블 만들기

이 단계에서는 같은 요약을 더 보기 편한 가로형 표로 바꿉니다. 엑셀 피벗 테이블에서 지역을 행에 놓고, 카테고리를 열에 놓는다고 생각해 보세요.

이번에는 엑셀 피벗처럼 지역을 행, 카테고리를 열에 놓습니다.

pivot = pd.pivot_table(
    merged,
    index="region",
    columns="category",
    values="sales",
    aggfunc="sum",
    fill_value=0,
    margins=True,
    margins_name="합계",
)

pivot_display = pivot.round(0).astype("int64")
print(pivot_display.head())

자주 쓰는 인자:

인자 뜻 엑셀 피벗과 비교
index 행 기준 행 영역
columns 열 기준 열 영역
values 계산할 값 값 영역
aggfunc 계산 방법 합계, 평균, 개수
fill_value 빈 칸 대체값 빈 셀에 0 표시

index="region"은 행 방향에 지역을 놓겠다는 뜻입니다. columns="category"는 열 방향에 카테고리를 놓겠다는 뜻입니다. values="sales"와 aggfunc="sum"은 매출을 합계로 계산하겠다는 뜻입니다.

fill_value=0은 해당 지역과 카테고리 조합이 없을 때 빈칸 대신 0을 넣습니다. 빈칸이 많으면 표를 읽기 어렵기 때문에, 집계표에서는 이렇게 0으로 채우는 일이 자주 있습니다.

margins=True와 margins_name="합계"는 행과 열의 전체 합계를 붙입니다. 엑셀 피벗 테이블의 총합계와 비슷합니다.

예상 질문과 답변

Q1. merge와 join은 같은 건가요?

비슷한 일을 하지만 오늘은 merge만 기억하면 됩니다. merge는 열의 값을 기준으로 두 표를 붙이는 방법입니다. 엑셀의 VLOOKUP과 연결해서 이해하기 가장 쉽습니다. 주문표의 고객ID로 고객표에서 지역을 찾아오듯이, 한 표의 값을 기준으로 다른 표의 정보를 가져옵니다.

Q2. 왜 how="left"를 쓰나요?

주문표를 기준으로 분석하기 때문입니다. 주문 20,000행을 유지한 채 고객 정보와 상품 정보를 붙이고 싶습니다. 기준을 잘못 잡으면 행이 줄어들 수도 있고, 반대로 늘어날 수도 있습니다. 그래서 병합 뒤에는 항상 행수가 그대로인지 확인해야 합니다.

Q3. groupby 뒤에 왜 reset_index()를 붙이나요?

groupby 결과의 묶음 기준은 인덱스로 들어갑니다. 화면으로 보기에는 괜찮지만, 이후에 정렬하거나 다시 병합할 때는 일반 열로 있는 편이 다루기 쉽습니다. 그래서 수업에서는 reset_index()를 붙여서 다시 평범한 표처럼 만듭니다.

Q4. pivot_table과 groupby는 무엇이 다른가요?

둘 다 집계를 합니다. groupby는 보통 세로로 긴 요약표를 만들고, pivot_table은 행과 열이 나뉜 교차표를 만듭니다. 엑셀 피벗처럼 지역을 행에 두고 카테고리를 열에 두어 비교하고 싶을 때는 pivot_table이 편합니다.

Q5. 왜 넓은 표를 굳이 긴 표로 바꾸나요?

분석 도구는 보통 “열 하나는 의미 하나”인 표를 좋아합니다. 0세남자, 1세남자처럼 열 이름 안에 값이 들어 있으면 필터링과 그래프 작성이 불편합니다. 연령, 성별, 인구수 열로 나누면 “20세만 보기”, “여자만 보기”, “연령별 합계 보기” 같은 작업이 훨씬 쉬워집니다.

Q6. 큰 파일을 전부 melt하면 안 되나요?

할 수 있는 경우도 있습니다. 다만 입문 수업에서는 먼저 줄이는 습관이 중요합니다. 이번 인구 파일 전체를 222개 연령·성별 열로 melt하면 400만 행이 넘습니다. 오늘은 서울특별시만 필터링해서 100,344행으로 연습합니다. 필요한 부분을 먼저 고른 뒤 모양을 바꾸는 순서가 안전합니다.

Q7. 평균 생활인구와 최대 생활인구는 어떻게 다르게 읽나요?

평균 생활인구는 기간 전체에서 보통 어느 정도 사람이 있었는지를 보여줍니다. 최대 생활인구는 특정일·특정 시간대의 피크를 포함한 값입니다. 그래서 일상적인 규모를 보려면 평균을 보고, 사람이 가장 많이 몰렸던 순간의 규모를 보려면 최대를 봅니다. 업무지구나 행사 같은 특성은 최대값에 더 크게 드러날 수 있습니다.

orders = pd.read_csv(DATA / "shop_orders/orders.csv")
customers = pd.read_csv(DATA / "shop_orders/customers.csv")
products = pd.read_csv(DATA / "shop_orders/products.csv")

merged = (
    orders.merge(customers, on="customer_id", how="left")
    .merge(products, on="product_id", how="left")
)

merged["sales"] = merged["price"] * merged["qty"] * (1 - merged["discount_rate"])

merge는 두 표를 공통된 열 기준으로 붙이는 기능입니다. 여기서는 customer_id로 고객 정보를 붙이고, product_id로 상품 정보를 붙입니다. 그 뒤에 price * qty * (1 - discount_rate)로 매출을 계산합니다. 단가에 수량을 곱하고, 할인율만큼 빼 주는 식입니다.

여기서 병합 후에도 20,000행이라는 점을 확인해야 합니다. 주문이 늘어나거나 줄어들면 병합 기준이 잘못되었을 수 있습니다. 병합 누락값 합계가 0이라는 것은 주문마다 필요한 고객 정보와 상품 정보가 잘 붙었다는 뜻입니다.

2. 지역×카테고리 다중 집계

이제 주문마다 계산된 매출을 지역과 카테고리 기준으로 묶어 봅니다. region 하나만 보는 것이 아니라, region 안에서 다시 category를 나누어 보는 방식입니다. 엑셀 피벗 테이블에서 행에 지역과 카테고리를 차례로 넣는 모습과 비슷합니다.

region_category_summary = (
    merged.groupby(["region", "category"])
    .agg(
        주문건수=("order_id", "nunique"),
        판매수량=("qty", "sum"),
        매출합계=("sales", "sum"),
        평균주문금액=("sales", "mean"),
    )
    .reset_index()
    .sort_values("매출합계", ascending=False)
)

주문건수는 주문번호의 고유 개수를 셉니다. 같은 주문번호가 여러 번 섞일 가능성을 생각해서, 중복을 빼고 세는 방식입니다. 판매수량은 수량을 모두 더하고, 매출합계는 매출을 모두 더합니다. 평균주문금액은 주문 한 건당 평균 매출을 보는 값입니다.

3. 지역×카테고리 피벗

같은 집계라도 표 모양을 바꾸면 더 읽기 쉬워질 때가 있습니다. pivot_table은 지역을 행에 두고, 카테고리를 열로 펼쳐서 보여 줍니다. 엑셀 피벗 테이블을 떠올리면 됩니다.

pivot = pd.pivot_table(
    merged,
    index="region",
    columns="category",
    values="sales",
    aggfunc="sum",
    fill_value=0,
    margins=True,
    margins_name="합계",
)

피벗은 같은 집계를 사람이 비교하기 쉬운 가로형 표로 펼친 것입니다. 지역은 행, 카테고리는 열, 값은 매출 합계입니다. fill_value=0은 비어 있는 칸을 0으로 채우라는 뜻입니다. margins=True는 합계 행과 합계 열을 함께 보겠다는 설정입니다.

4. 231열 인구 파일 melt

수업 홈의 데이터 받기 페이지에서 내려받은 인구 파일은 열이 매우 많습니다. 원본은 18,623행, 231열입니다. 열이 231개라는 말은 옆으로 매우 넓은 표라는 뜻입니다. 이런 표는 눈으로 보기에는 익숙할 수 있지만, 분석하거나 그래프로 그리기에는 불편합니다.

melt는 넓은 표를 긴 표로 바꾸는 기능입니다. 예를 들어 0세남자, 1세남자처럼 열 이름에 들어 있던 정보가 연령성별이라는 열의 값으로 내려옵니다. 옷장 여러 칸에 흩어진 물건을 한 바구니에 모으고, 물건 이름표를 붙이는 느낌으로 보면 됩니다.

pop = pd.read_csv(
    DATA / "real_korean/population_kosis/mois_legal_dong_gender_age_population_20260531.csv"
)

id_cols = ["법정동코드", "기준연월", "시도명", "시군구명", "읍면동명", "리명"]
age_gender_cols = [
    col
    for col in pop.columns
    if (col.endswith("남자") or col.endswith("여자")) and col not in ["남자", "여자"]
]

seoul_pop = pop[pop["시도명"] == "서울특별시"].copy()

long_pop = seoul_pop.melt(
    id_vars=id_cols,
    value_vars=age_gender_cols,
    var_name="연령성별",
    value_name="인구수",
)

long_pop["성별"] = long_pop["연령성별"].str.extract("(남자|여자)$")
long_pop["연령"] = (
    long_pop["연령성별"].str.replace("(남자|여자)$", "", regex=True).str.strip()
)

먼저 서울특별시 행만 남긴 뒤 melt합니다. 큰 데이터를 무조건 한 번에 바꾸기보다, 필요한 범위로 줄인 뒤 다루는 편이 초보자에게 훨씬 안전합니다. 연령성별에서 맨 끝의 남자 또는 여자를 뽑아 성별 열을 만들고, 나머지 부분을 연령으로 정리합니다.
