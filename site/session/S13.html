S13. 그룹화·집계·재구조화

차시 개요

  • 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
  • 선수 차시: S12 pandas 파일 읽기, 기본 전처리, 인코딩 처리
  • 난이도: 4
  • 시간: 70분
  • 사용 데이터:
  • 02_datasets/shop_orders/orders.csv
  • 02_datasets/shop_orders/customers.csv
  • 02_datasets/shop_orders/products.csv
  • 02_datasets/real_korean/cafe_restaurant/seoul_commercial_food_stores_202109_github.csv
  • 02_datasets/real_korean/population_kosis/mois_legal_dong_gender_age_population_20260531.csv
  • 02_datasets/real_korean/seoul_living_pop/seoul_living_population_gu_daily.csv

이번 차시는 여러 행을 요약표로 바꾸는 수업이다. 엑셀에서 피벗 테이블, 부분합, VLOOKUP을 써본 사람이라면 이미 핵심 감각은 가지고 있다. pandas에서는 그 일을 groupby, merge, pivot_table, melt라는 이름으로 한다.

앞선 cafe 매출 데이터 대신, 오늘은 여러 표를 함께 다루어야 하므로 잠시 주문 데이터인 shop_orders로 옮겨 간다.

수업의 중심 데이터는 shop_orders 3개 표다. 주문 표에는 고객 지역과 상품 카테고리가 없고, 매출 컬럼도 없다. 그래서 고객 표와 상품 표를 붙인 뒤 단가와 수량, 할인율로 매출을 계산해야 한다. 이 과정은 엑셀에서 주문표에 고객정보와 상품정보를 VLOOKUP으로 끌어온 뒤 피벗 테이블을 만드는 흐름과 같다.

후반에는 실데이터를 다룬다. 231열짜리 인구 파일을 melt로 세로형 데이터로 바꾸고, 서울 음식점 상가업소 119,158행을 업종별로 집계한다. 마지막 미션은 서울 생활인구 77,792행을 자치구 단위로 요약하는 것이다.

학습 목표

  1. groupby()로 두 개 이상의 기준 열을 묶고 여러 집계값을 한 번에 계산할 수 있다.
  2. merge()를 엑셀의 VLOOKUP 또는 XLOOKUP과 연결해 이해하고, 병합 후 누락값을 확인할 수 있다.
  3. pivot_table()로 행과 열 기준이 모두 있는 요약표를 만들 수 있다.
  4. melt()로 가로로 넓은 wide 데이터를 세로로 긴 long 데이터로 바꿀 수 있다.
  5. 수만 행 이상의 실데이터를 바로 시각화하지 않고, 먼저 필터링하거나 집계해 줄이는 습관을 갖는다.

핵심 개념

1. groupby는 부분합과 피벗의 출발점이다

엑셀에서 지역별 합계, 카테고리별 평균, 부서별 인원수를 구할 때는 부분합이나 피벗 테이블을 쓴다. pandas에서는 먼저 묶을 기준을 정한다.

df.groupby("region")

기준이 하나가 아니라 둘일 수도 있다.

df.groupby(["region", "category"])

이 코드는 "지역별로 나누고, 그 안에서 다시 카테고리별로 나누라"는 뜻이다. 엑셀 피벗 테이블에서 행 영역에 region과 category를 차례로 넣는 것과 비슷하다.

그 다음에는 무엇을 계산할지 정한다.

df.groupby(["region", "category"]).agg(
    주문건수=("order_id", "nunique"),
    판매수량=("qty", "sum"),
    매출합계=("sales", "sum"),
    평균주문금액=("sales", "mean"),
)

agg()는 여러 집계를 한 번에 만드는 도구다. 엑셀 피벗에서 값 영역에 "주문건수", "수량 합계", "매출 합계", "매출 평균"을 함께 넣는 상황과 같다.

2. merge는 pandas의 VLOOKUP이다

orders.csv에는 고객 지역이 없다. 대신 customer_id가 있다. customers.csv에는 같은 customer_id와 함께 region이 있다. 두 표를 customer_id로 붙이면 주문마다 고객 지역을 가져올 수 있다.

엑셀식으로 말하면 다음과 같다.

엑셀 작업 pandas 작업
주문표에 고객ID 기준으로 고객 지역을 VLOOKUP orders.merge(customers, on="customer_id", how="left")
주문표에 상품ID 기준으로 상품 카테고리와 단가를 VLOOKUP .merge(products, on="product_id", how="left")
조회 실패한 행 확인 병합 후 isna().sum() 확인

입문 단계에서 가장 중요한 일은 병합 뒤 행수가 달라졌는지, 필요한 값이 비지는 않았는지 확인하는 것이다. 이번 데이터는 주문 20,000행이 병합 후에도 20,000행으로 유지되고, 지역·카테고리·단가 누락값이 0개임을 확인했다.

3. 매출은 병합 뒤 계산한다

orders.csv에는 매출 컬럼이 없다. 단가는 상품 표에 있고, 수량과 할인율은 주문 표에 있다. 그래서 병합 후 아래 계산을 한다.

sales = price * qty * (1 - discount_rate)

엑셀에서도 VLOOKUP으로 단가를 가져온 뒤 단가 * 수량 * (1 - 할인율) 수식을 만드는 것과 같다.

4. pivot_table은 결과를 가로형 요약표로 펼친다

groupby 결과는 보통 세로로 긴 표다. 사람이 보고 비교하기에는 지역을 행에 두고 카테고리를 열에 둔 피벗 형태가 편할 때가 있다.

pd.pivot_table(
    df,
    index="region",
    columns="category",
    values="sales",
    aggfunc="sum",
    fill_value=0,
)

여기서 index는 피벗의 행, columns는 피벗의 열, values는 계산할 값, aggfunc는 계산 방식이다. 엑셀 피벗 테이블과 거의 같은 사고방식이다.

5. melt는 넓은 표를 긴 표로 바꾼다

공공데이터에는 열이 지나치게 많은 파일이 자주 있다. 이번 인구 파일은 231열이다. 0세남자, 1세남자, 2세남자처럼 연령과 성별이 열 이름 안에 들어 있다.

눈으로 훑어볼 때는 넓은 표가 익숙하지만, 분석과 시각화에는 긴 표가 더 다루기 쉽다.

wide 형태 long 형태
0세남자, 1세남자, 2세남자가 각각 별도 열 연령성별 열에 0세남자, 1세남자가 값으로 들어감
열이 너무 많아 필터와 그래프가 어려움 연령, 성별, 인구수 열로 필터와 그룹화가 쉬움
엑셀 보고서처럼 보임 데이터베이스와 BI 도구가 좋아하는 형태

이번 수업에서는 전체 파일을 바로 길게 만들지 않고, 시도명 == "서울특별시" 행 452개만 골라 melt한다. 그 결과 100,344행의 long 데이터가 된다. 원리는 전체 파일에도 똑같이 적용된다.

6. 큰 파일은 먼저 줄이고 본다

서울 음식점 상가업소 파일은 119,158행이고, 생활인구 파일은 77,792행이다. 이 정도는 pandas로 읽을 수 있지만, 입문 수업에서 원자료를 그대로 화면에 많이 출력하거나 바로 그래프로 그리면 이해가 흐려진다.

큰 파일을 다룰 때의 순서는 다음과 같다.

  1. 행수와 열 이름을 확인한다.
  2. 필요한 열만 고른다.
  3. 필요한 지역이나 기간만 필터링한다.
  4. groupby로 먼저 요약한다.
  5. 요약표를 보고 해석하거나 시각화한다.

엑셀과 연결해서 설명할 지점

  • merge는 VLOOKUP 또는 XLOOKUP이다. 단, pandas는 여러 열을 한꺼번에 붙일 수 있다.
  • 병합 키는 엑셀의 조회값과 기준열이다. 키가 틀리면 원하는 값이 붙지 않는다.
  • groupby는 부분합 또는 피벗의 계산 단계다.
  • pivot_table은 groupby 결과를 사람이 보기 좋은 행×열 표로 펼치는 단계다.
  • melt는 여러 열에 흩어진 값을 한 열로 모으는 작업이다. 엑셀의 "파워쿼리 열 피벗 해제"와 비슷하다.
  • 큰 공공데이터는 원본을 그대로 보려 하지 말고, 집계표로 줄여서 본다.

오늘은 엑셀의 VLOOKUP, 부분합, 피벗 테이블을 pandas로 옮긴다. 코드를 외우는 것보다 "어떤 표를 어떤 모양으로 바꾸는가"를 이해하는 것이 중요하다.

따라하기 1: 주문 3개 표 읽고 병합하기

새 파이썬 파일이나 Colab 노트북을 열고 아래 코드를 실행한다.

from pathlib import Path
import pandas as pd

ROOT = Path("/mnt/d/ai-data-analysis-course")
DATA = ROOT / "02_datasets"

orders = pd.read_csv(DATA / "shop_orders/orders.csv")
customers = pd.read_csv(DATA / "shop_orders/customers.csv")
products = pd.read_csv(DATA / "shop_orders/products.csv")

print(orders.shape)
print(customers.shape)
print(products.shape)
print(orders.head())

확인할 결과:

orders: 20,000행, 6열
customers: 3,000행, 7열
products: 200행, 5열

주문 표에는 customer_id와 product_id만 있고, 고객 지역과 상품 단가는 없다. 고객 표와 상품 표를 붙인다.

merged = (
    orders.merge(customers, on="customer_id", how="left")
    .merge(products, on="product_id", how="left")
)

print(merged.shape)
print(merged[["region", "category", "price"]].isna().sum())

확인할 결과:

병합 후: 20,000행, 17열
region      0
category    0
price       0

엑셀에서 VLOOKUP을 한 뒤 조회 실패가 있는지 확인하는 것과 같다. how="left"는 주문표의 20,000행을 기준으로 유지하겠다는 뜻이다.

따라하기 2: 매출 계산하기

orders.csv에는 매출 컬럼이 없다. 상품 단가와 주문 수량, 할인율로 만든다.

merged["sales"] = merged["price"] * merged["qty"] * (1 - merged["discount_rate"])

print(merged[["order_id", "price", "qty", "discount_rate", "sales"]].head())
print(merged["sales"].sum())

확인할 결과:

전체 매출 합계: 1,938,489,803

계산식은 아래와 같다.

매출 = 단가 * 수량 * (1 - 할인율)

따라하기 3: 지역×카테고리별 다중 집계하기

지역과 카테고리를 동시에 기준으로 묶는다.

region_category_summary = (
    merged.groupby(["region", "category"])
    .agg(
        주문건수=("order_id", "nunique"),
        판매수량=("qty", "sum"),
        매출합계=("sales", "sum"),
        평균주문금액=("sales", "mean"),
    )
    .reset_index()
    .sort_values("매출합계", ascending=False)
)

print(region_category_summary.head(10))

대표 결과:

region    category  주문건수  판매수량      매출합계
서울특별시  디지털       612   658  210356387
경기도     디지털       514   565  188110354
서울특별시  패션         990  1442   86206229
경기도     패션         884  1292   75963357

읽는 법:

  • groupby(["region", "category"]): 지역별로 나눈 뒤 카테고리별로 다시 나눈다.
  • 주문건수=("order_id", "nunique"): 주문번호의 고유 개수를 센다.
  • 판매수량=("qty", "sum"): 수량을 합한다.
  • 매출합계=("sales", "sum"): 매출을 합한다.
  • 평균주문금액=("sales", "mean"): 주문 한 건의 평균 매출을 구한다.

따라하기 4: 지역×카테고리 피벗 테이블 만들기

이번에는 엑셀 피벗처럼 지역을 행, 카테고리를 열에 놓는다.

pivot = pd.pivot_table(
    merged,
    index="region",
    columns="category",
    values="sales",
    aggfunc="sum",
    fill_value=0,
    margins=True,
    margins_name="합계",
)

pivot_display = pivot.round(0).astype("int64")
print(pivot_display.head())

핵심 인자:

인자 뜻 엑셀 피벗과 비교
index 행 기준 행 영역
columns 열 기준 열 영역
values 계산할 값 값 영역
aggfunc 계산 방법 합계, 평균, 개수

시연 코드:

region_category_summary = (
    merged.groupby(["region", "category"])
    .agg(
        주문건수=("order_id", "nunique"),
        판매수량=("qty", "sum"),

말할 것:

groupby(["region", "category"])는 지역으로 먼저 나누고, 그 안에서 카테고리로 다시 나눈다는 뜻이다. 엑셀 피벗에서 행 영역에 지역과 카테고리를 넣은 상태를 떠올리면 된다.

agg는 값 영역에 여러 계산을 넣는 부분이다. 주문건수는 주문번호의 고유 개수, 판매수량은 수량 합계, 매출합계는 매출 합계, 평균주문금액은 매출 평균이다.

시연 포인트:

  • reset_index()를 빼고 한 번 실행해 인덱스가 계층처럼 보이는 모습을 보여준다.
  • reset_index()를 다시 붙여 일반 표처럼 다루기 쉽게 만든다고 설명한다.
  • 상위 결과가 서울특별시 × 디지털임을 확인한다.

예상 결과:

40-48분. pivot_table로 교차표 만들기

시연 코드:

pivot = pd.pivot_table(
    merged,
    index="region",
    columns="category",
    values="sales",
    aggfunc="sum",
    fill_value=0,
    margins=True,
    margins_name="합계",
)

print(pivot.round(0).astype("int64").head())

말할 것:

groupby 결과는 긴 표다. 그런데 지역을 행에 두고 카테고리를 열에 두면 비교가 쉬워진다. 이것이 피벗 테이블이다.

index는 행, columns는 열, values는 계산할 숫자, aggfunc는 계산 방식이다. fill_value=0은 빈 조합을 0으로 채운다. margins=True는 합계 행과 합계 열을 만든다.

강조:

피벗은 분석 결과를 보기 좋게 펼치는 방법이다. 원자료를 피벗 형태로만 저장하는 것이 항상 좋은 것은 아니다. 다음 melt에서 그 반대 방향을 본다.

Q1. merge와 join은 같은 건가요?

비슷한 일을 하지만 오늘은 merge만 기억하면 된다. merge는 열의 값을 기준으로 두 표를 붙이는 방법이고, 엑셀의 VLOOKUP과 가장 연결하기 쉽다.

Q2. 왜 how="left"를 쓰나요?

주문표를 기준으로 분석하기 때문이다. 주문 20,000행을 유지한 채 고객 정보와 상품 정보를 붙이고 싶다. 만약 기준을 잘못 잡으면 행이 줄거나 늘어날 수 있다.

Q3. groupby 뒤에 왜 reset_index()를 붙이나요?

groupby 결과의 묶음 기준은 인덱스로 들어간다. 화면으로 보기에는 괜찮지만, 이후 정렬하거나 다시 병합할 때 일반 열로 있는 편이 쉽다. 그래서 수업에서는 reset_index()를 붙여 표처럼 다룬다.

Q4. pivot_table과 groupby는 무엇이 다른가요?

둘 다 집계한다. groupby는 긴 요약표를 만들고, pivot_table은 행과 열이 나뉜 교차표를 만든다. 엑셀 피벗처럼 비교표를 만들고 싶으면 pivot_table이 편하다.

Q5. 왜 넓은 표를 굳이 긴 표로 바꾸나요?

분석 도구는 보통 "열 하나는 의미 하나"인 표를 좋아한다. 0세남자, 1세남자처럼 열 이름에 값이 들어 있으면 필터링과 그래프 작성이 불편하다. 연령, 성별, 인구수 열로 나누면 분석하기 쉬워진다.

Q6. 큰 파일을 전부 melt하면 안 되나요?

가능할 때도 있지만 입문 수업에서는 먼저 줄이는 습관이 중요하다. 이번 인구 파일 전체를 222개 연령·성별 열로 melt하면 400만 행이 넘는다. 오늘은 서울특별시만 필터링해 100,344행으로 연습한다.

Q7. 평균 생활인구와 최대 생활인구는 어떻게 다르게 읽나요?

평균 생활인구는 기간 전체에서 보통 어느 정도 사람이 있었는지를 보여준다. 최대 생활인구는 특정일·특정 시간대의 피크를 포함한 값이므로, 일상적인 규모보다 행사나 업무지구 특성의 영향을 더 크게 받을 수 있다.

seoul_living_population_gu_daily.csv를 사용해 자치구별 생활인구 요약표를 만든다.

조건:

  • 서울시 전체 합계 행은 제외한다.
  • 시군구명별로 관측일수, 평균 생활인구, 평균 주간인구, 평균 야간인구, 최대 생활인구를 계산한다.
  • 평균 생활인구 기준으로 내림차순 정렬한다.
  • 결과가 25개 자치구인지 확인한다.

미션 모범 코드

from pathlib import Path
import pandas as pd

ROOT = Path("/mnt/d/ai-data-analysis-course")
DATA = ROOT / "02_datasets"

living = pd.read_csv(
    DATA / "real_korean/seoul_living_pop/seoul_living_population_gu_daily.csv"
)

living["기준일"] = pd.to_datetime(living["기준일ID"].astype(str), format="%Y%m%d")

gu_living = living[living["시군구명"] != "서울시"].copy()

gu_summary = (
    gu_living.groupby("시군구명")
    .agg(
        관측일수=("기준일", "nunique"),
        평균생활인구=("총생활인구수", "mean"),
        평균주간인구=("주간인구수(09~18)", "mean"),
        평균야간인구=("야간인구수(19~08)", "mean"),
        최대생활인구=("일최대인구수", "max"),
    )
    .reset_index()
    .sort_values("평균생활인구", ascending=False)
)

display = gu_summary.copy()
for col in ["평균생활인구", "평균주간인구", "평균야간인구", "최대생활인구"]:
    display[col] = display[col].round(0).astype("int64")

print(living.shape)
print(living["기준일"].min(), living["기준일"].max())
print(display.head(10))

gu_living = living[living["시군구명"] != "서울시"].copy()

이 코드는 엑셀 필터에서 서울시를 체크 해제하는 것과 같다.

그 다음 핵심은 groupby("시군구명")이다. 자치구 이름으로 행을 묶고, 각 묶음 안에서 필요한 값을 계산한다.

.agg(
    관측일수=("기준일", "nunique"),
    평균생활인구=("총생활인구수", "mean"),

)

관측일수는 단순 행수가 아니라 날짜의 고유 개수를 세었다. 같은 기간의 자치구들을 비교하는지 확인하기 위해서다. 실제 결과에서 모든 자치구가 2,992일로 확인된다.

평균생활인구 기준 1위는 강남구다. 강남구는 평균주간인구가 평균야간인구보다 크게 나타난다. 업무·상업 활동으로 낮 시간대 유입이 크다는 해석을 붙일 수 있다. 반대로 이 표만으로 원인을 단정하지는 않는다. 오늘의 목표는 원인 분석이 아니라 집계표를 정확히 만드는 것이다.
