# S18 모범답안: Looker Studio 공유 링크 생성과 검증

## 미션에서 맞아야 하는 모습

이 미션은 파일 하나를 만들어 제출하는 문제가 아닙니다. Looker Studio에서 설문 대시보드를 만들고, 다른 사람이 열 수 있는 공유 링크까지 확인하는 문제입니다. 보고서는 예쁜 화면만 만들었다고 끝이 아니라, 링크를 받은 사람이 실제로 볼 수 있어야 합니다.

정답으로 보려면 아래 기준이 맞아야 합니다.

| 항목 | 정답 기준 |
|---|---|
| 데이터 원본 | 수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 파일 |
| 보고서 제목 | `S18_설문_공유_대시보드_본인이름` |
| 공유 권한 | Viewer, 보기 가능 |
| 링크 검증 | 시크릿 창, 동료 계정, 강사 계정 중 하나로 열림 확인 |
| 응답 수 | 1,500 |
| 평균 NPS | 약 6.86 |
| 평균 만족도 | 약 3.62 |
| NPS 점수 | 약 -4.73 |

공유 링크는 사람마다 다릅니다. 그래서 모범답안에 고정된 URL은 없습니다. 중요한 것은 링크 주소 자체가 아니라, 그 링크가 열리는지, 보기 권한인지, 화면의 숫자가 검산값과 맞는지입니다. 줄자로 키를 재듯이, 대시보드도 기준 숫자와 맞춰 보아야 합니다.

설문 응답 파일은 수업 홈의 데이터 받기 페이지에서 내려받아 사용하면 됩니다. CSV라는 말이 나오면, 엑셀로도 열리는 표 파일이라고 생각하시면 됩니다. 엑셀 파일 형식으로 받은 경우에도 수업 안내에 맞게 같은 설문 응답 데이터를 연결하면 됩니다.

## Looker Studio 계산 필드 정답

### `NPS_그룹`

```text
CASE
  WHEN nps >= 9 THEN "추천 고객"
  WHEN nps >= 7 THEN "중립 고객"
  ELSE "비추천 고객"
END
```

이 필드는 0점부터 10점까지 들어 있는 NPS 원점수를 세 그룹으로 나누는 역할을 합니다. 숫자를 그대로 보면 한눈에 감이 잘 오지 않기 때문에, 사전에서 단어 뜻을 찾듯이 점수 구간마다 이름을 붙여 주는 것입니다.

| NPS 그룹 | 응답 수 |
|---|---:|
| 비추천 고객 | 552 |
| 추천 고객 | 481 |
| 중립 고객 | 467 |

여기서 뭐가 보이시나요? 비추천 고객이 552명이고 추천 고객이 481명입니다. 추천 고객보다 비추천 고객이 조금 더 많기 때문에, 뒤에서 계산하는 NPS 점수가 음수로 나옵니다.

### `NPS_점수_행`

```text
CASE
  WHEN nps >= 9 THEN 100
  WHEN nps <= 6 THEN -100
  ELSE 0
END
```

이 필드는 반드시 평균으로 집계해야 합니다. 추천 고객은 100, 중립 고객은 0, 비추천 고객은 -100으로 바꾼 뒤 평균을 내면 NPS 점수가 됩니다. 합계로 계산하면 점수가 너무 크게 튀기 때문에, 꼭 평균인지 확인해 보세요.

평균값이 NPS 점수이며, 정답은 약 `-4.73`입니다.

### `만족도_구간`

이 필드는 필수는 아니지만, 만족도를 더 쉽게 읽고 싶을 때 사용할 수 있습니다. 만족도 숫자 1-5를 그대로 보는 대신, 만족, 보통, 불만족처럼 말로 바꿔 주는 필드입니다.

```text
CASE
  WHEN satisfaction >= 4 THEN "만족"
  WHEN satisfaction = 3 THEN "보통"
  ELSE "불만족"
END
```

## 완성 대시보드의 권장 구성

대시보드는 한 장의 요약판처럼 보이면 좋습니다. 위쪽에는 전체 상황을 바로 볼 수 있는 카드 4개를 두고, 아래쪽에는 그룹별 차이를 보는 그래프와 표를 배치합니다.

| 시각화 | 차원 | 측정항목 | 정답 확인 |
|---|---|---|---|
| 응답 수 카드 | 없음 | `respondent_id` 개수 또는 고유 개수 | 1,500 |
| 평균 NPS 카드 | 없음 | `nps` 평균 | 6.86 |
| 평균 만족도 카드 | 없음 | `satisfaction` 평균 | 3.62 |
| NPS 점수 카드 | 없음 | `NPS_점수_행` 평균 | -4.73 |
| NPS 그룹별 막대그래프 | `NPS_그룹` | 응답 수 | 552, 481, 467 |
| 채널별 표 | `channel` | 응답 수, 평균 NPS, 평균 만족도 | 네이버 검색 435건 |
| 연령대별 막대그래프 | `age_group` | 평균 만족도 | 40대와 50대가 상대적으로 높음 |

차원은 “무엇별로 볼 것인가”입니다. 예를 들어 채널별로 본다면 `channel`이 차원입니다. 측정항목은 “무엇을 계산할 것인가”입니다. 응답 수, 평균 NPS, 평균 만족도처럼 숫자로 계산되는 값이 여기에 들어갑니다.

## 채널별 검산값

채널별 표는 유입 경로마다 응답 수와 평균 점수가 어떻게 다른지 보는 표입니다. 여기서는 원인까지 단정하지 않고, 화면에 나온 값이 맞는지 확인하는 용도로 봅니다.

| channel | 응답 수 | 평균 NPS | 평균 만족도 |
|---|---:|---:|---:|
| 네이버 검색 | 435 | 6.87 | 3.62 |
| 카카오톡 채널 | 259 | 6.93 | 3.61 |
| 인스타그램 | 251 | 6.94 | 3.67 |

채널별 표가 이 값과 소수점 반올림 수준에서 맞으면 정상입니다. 예를 들어 네이버 검색이 435건으로 보이고, 평균 NPS가 6.87 근처라면 데이터 연결과 집계가 제대로 된 것입니다.

연령대별 표는 나이대에 따라 평균 점수가 어떻게 보이는지 확인하는 표입니다. 이것도 원인 분석용이 아니라 검산용입니다.

## 공유 링크 검증 체크리스트

공유는 링크를 복사하는 순간 끝나는 일이 아닙니다. 링크를 받은 사람이 실제로 볼 수 있는지까지 확인해야 합니다. 문이 열리는지 직접 손잡이를 돌려 보는 과정이라고 생각하시면 됩니다.

| 확인 항목 | 통과 기준 |
|---|---|
| 링크가 열린다 | 다른 브라우저, 시크릿 창, 동료 계정 중 하나에서 열림 |
| 권한이 적절하다 | Viewer 권한이며 편집 화면이 아님 |
| 핵심 카드가 보인다 | 응답 수, 평균 NPS, 평균 만족도, NPS 점수 카드가 보임 |
| 숫자가 맞다 | 1,500, 6.86, 3.62, -4.73 근처 |
| 민감 정보가 없다 | 수업용 합성 데이터만 사용 |

특히 Viewer 권한을 확인해야 합니다. Viewer는 보기만 가능한 권한입니다. 링크를 받은 사람이 편집까지 할 수 있으면 제출 기준에 맞지 않습니다.

## 흔한 오답과 수정

아래 표는 실제로 많이 생기는 실수입니다. 숫자가 이상할 때는 차트를 새로 만들기보다, 먼저 집계 방식과 공유 권한을 확인해 보세요.

| 오답 상태 | 원인 | 수정 |
|---|---|---|
| 응답 수가 3,000 이상 | 같은 CSV를 중복 업로드함 | 새 데이터 원본으로 다시 연결하거나 중복 데이터셋을 정리한다 |
| 평균 NPS가 매우 큰 값 | `nps`가 합계로 집계됨 | 집계를 평균으로 바꾼다 |
| NPS 점수가 -7,100처럼 보임 | `NPS_점수_행`이 합계로 집계됨 | 집계를 평균으로 바꾼다 |
| 링크를 받은 사람이 못 봄 | 보고서 권한이 닫혀 있음 | Share 설정에서 Viewer 권한을 부여한다 |
| 링크를 받은 사람이 편집 가능 | Editor 권한으로 공유함 | Viewer 권한으로 낮춘다 |
| 한글이 깨짐 | 다른 인코딩 파일을 올림 | 수업 홈의 데이터 받기 페이지에서 설문 응답 파일을 다시 내려받는다 |

응답 수가 3,000 이상으로 보이면 거의 먼저 중복 업로드를 의심하면 됩니다. 같은 데이터를 두 번 올리면 같은 응답자가 두 번 들어간 것처럼 계산됩니다. 평균 NPS나 NPS 점수가 이상하게 크면 집계 방식이 합계로 되어 있는지 확인해 보세요.

## 검증용 Python 코드

S18은 Looker Studio 공유 차시이므로 별도 `S18_code.py` 파일을 요구하지 않습니다. 아래 코드는 모범답안 수치가 실제 데이터와 맞는지 검증하기 위해 `python3`와 `pandas 3.0.3`으로 실행한 코드입니다.

학생 제출용 코드는 아니고, 선생님이 기준 숫자를 확인하기 위해 사용한 검산 코드라고 보시면 됩니다. 코드 안에는 실행 환경의 파일 위치가 들어 있지만, 실습에서는 수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 파일을 Looker Studio에 연결하면 됩니다.

```python
from pathlib import Path
import pandas as pd

path = Path("/mnt/d/ai-data-analysis-course/02_datasets/survey_responses/survey_responses_clean.csv")
df = pd.read_csv(path, parse_dates=["submitted_at"])

print("shape:", df.shape)
print("columns:", list(df.columns))
print("submitted_min:", df["submitted_at"].min())
print("submitted_max:", df["submitted_at"].max())
print("unique_respondents:", df["respondent_id"].nunique())
print("missing_total:", int(df.isna().sum().sum()))

promoters = (df["nps"] >= 9).sum()
passives = df["nps"].between(7, 8).sum()
detractors = (df["nps"] <= 6).sum()
nps_score = ((promoters - detractors) / len(df)) * 100

print("avg_nps:", round(df["nps"].mean(), 4))
print("avg_satisfaction:", round(df["satisfaction"].mean(), 4))
print("promoters:", int(promoters))
print("passives:", int(passives))
print("detractors:", int(detractors))
print("nps_score:", round(nps_score, 2))

channel_summary = (
    df.groupby("channel")
    .agg(
        responses=("respondent_id", "count"),
        avg_nps=("nps", "mean"),
        avg_satisfaction=("satisfaction", "mean"),
    )
    .sort_values("responses", ascending=False)
)
print(channel_summary.round(2).to_string())

age_summary = (
    df.groupby("age_group")
    .agg(
        responses=("respondent_id", "count"),
        avg_nps=("nps", "mean"),
        avg_satisfaction=("satisfaction", "mean"),
    )
    .sort_values("responses", ascending=False)
)
print(age_summary.round(2).to_string())
```

실행 확인 결과입니다. 여기서 `shape: (1500, 8)`은 행이 1,500개, 열이 8개라는 뜻입니다. `unique_respondents: 1500`은 응답자 ID를 세어도 1,500명이라는 뜻이라서, 중복 응답자가 없다고 볼 수 있습니다.
