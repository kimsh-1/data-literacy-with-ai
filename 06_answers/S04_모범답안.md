# S04 모범답안: 등급별 객단가

## 미션 확인

고객 등급별로 주문 건수, 매출 합계, 객단가를 구합니다.

객단가는 이 실습에서 주문 1건당 평균 매출을 뜻합니다. 손님 한 명이 아니라, 주문 한 건이 평균적으로 얼마의 매출을 만들었는지 보는 값입니다. 키를 잴 때 여러 사람의 키를 더한 뒤 사람 수로 나누듯이, 매출도 전체 매출을 주문 건수로 나누어 평균을 냅니다.

```text
객단가 = 매출 합계 ÷ 주문 건수
```

## 풀이 전 확인

수업 홈의 데이터 받기 페이지에서 내려받은 쇼핑몰 주문 파일을 열고, `orders` 시트에 따라하기 결과가 이미 만들어져 있어야 합니다.

`orders` 시트에는 앞 실습에서 상품 단가, 고객 지역, 고객 등급, 주문별 매출을 붙여 두었습니다.

| 열 | 내용 |
|---|---|
| I열 | `price` |
| J열 | `region` |
| K열 | `grade` |
| L열 | `sales` |

주문별 매출은 단가에 수량을 곱하고, 할인율만큼 빼서 계산합니다. 할인율이 10%라면 실제로는 90%만 매출로 잡히는 식입니다.

주문별 매출 공식은 다음과 같습니다.

```excel
=I2*E2*(1-F2)
```

## 엑셀로 푸는 방법

새 시트 `grade_summary`를 만듭니다. 이 시트는 등급별 결과만 따로 모아 보는 요약표입니다.

A1:D1에 다음 제목을 입력해 보세요.

| 셀 | 제목 |
|---|---|
| A1 | `grade` |
| B1 | `order_count` |
| C1 | `total_sales` |
| D1 | `avg_order_value` |

A2:A5에는 등급을 직접 입력합니다. 등급 이름을 먼저 적어 두고, 엑셀 함수가 각 등급에 맞는 주문만 찾아 계산하게 만드는 방식입니다.

```text
VIP
골드
실버
브론즈
```

B2 셀에는 VIP 주문이 몇 건인지 세는 공식을 입력합니다. `COUNTIF`는 조건에 맞는 행의 개수를 세는 함수입니다. 여기서는 `orders` 시트의 K열에서 A2에 적힌 등급, 즉 VIP와 같은 값을 찾습니다.

```excel
=COUNTIF(orders!$K$2:$K$20001,A2)
```

C2 셀에는 VIP의 매출 합계를 구하는 공식을 입력합니다. `SUMIF`는 조건에 맞는 행의 값을 더하는 함수입니다. K열에서 VIP인 주문만 찾고, 그 주문들의 L열 매출을 더합니다.

```excel
=SUMIF(orders!$K$2:$K$20001,A2,orders!$L$2:$L$20001)
```

D2 셀에는 객단가를 계산합니다. 이미 C2에 매출 합계가 있고 B2에 주문 건수가 있으니, 매출 합계를 주문 건수로 나누면 됩니다.

```excel
=C2/B2
```

B2:D2를 5행까지 채워 보세요. 그러면 골드, 실버, 브론즈도 같은 방식으로 계산됩니다. 공식 안의 `A2`가 아래로 내려가면서 `A3`, `A4`, `A5`로 바뀌기 때문에 각 등급에 맞는 결과가 나옵니다.

## 정답

| grade | order_count | total_sales | avg_order_value |
|---|---:|---:|---:|
| VIP | 3,757 | 393,432,930 | 104,719.97 |
| 골드 | 5,433 | 547,524,985 | 100,777.65 |
| 실버 | 6,373 | 603,583,519 | 94,709.48 |
| 브론즈 | 4,437 | 393,948,369 | 88,787.10 |

여기서 뭐가 보이시나요? 네 등급 중 주문 1건당 평균 매출이 가장 높은 등급은 VIP입니다. VIP의 객단가는 104,719.97원이고, 브론즈의 객단가 88,787.10원보다 높습니다.

주문 건수만 보면 실버가 6,373건으로 가장 많습니다. 하지만 객단가는 VIP가 가장 높습니다. 그래서 “많이 주문한 등급”과 “주문 1건당 매출이 큰 등급”은 다를 수 있습니다.

## 검산값 확인

계산이 맞는지 확인할 때는 전체 합계와 맞춰 봅니다. 저울의 눈금을 확인하듯이, 등급별로 나눈 값들을 다시 더했을 때 전체 값과 같아야 합니다.

| 항목 | 값 |
|---|---:|
| 전체 주문 건수 | 20,000 |
| 전체 매출 | 1,938,489,803 |
| 전체 객단가 | 96,924.49 |

등급별 주문 건수 합계와 매출 합계가 위 검산값과 일치해야 합니다. 주문 건수는 3,757 + 5,433 + 6,373 + 4,437을 더했을 때 20,000이 되어야 하고, 매출 합계도 네 등급의 `total_sales`를 더했을 때 1,938,489,803이 되어야 합니다.

## 강사용 검증 코드

아래 코드는 엑셀 풀이와 같은 계산을 pandas로 재현해 검산한 코드입니다. pandas는 표 데이터를 다루는 파이썬 도구입니다. 이 차시는 엑셀 함수 차시이므로 수강생은 위의 엑셀 공식으로 실습하면 됩니다. 별도 `S04_code.py` 산출물은 만들지 않습니다.

```python
import pandas as pd
from pathlib import Path

base = Path("/mnt/d/ai-data-analysis-course/02_datasets/shop_orders")

orders = pd.read_csv(base / "orders.csv")
customers = pd.read_csv(base / "customers.csv")
products = pd.read_csv(base / "products.csv")

merged = orders.merge(
    products[["product_id", "price"]],
    on="product_id",
    how="left",
)
merged = merged.merge(
    customers[["customer_id", "grade"]],
    on="customer_id",
    how="left",
)

merged["sales"] = merged["price"] * merged["qty"] * (1 - merged["discount_rate"])

answer = (
    merged.groupby("grade", as_index=False)
    .agg(
        order_count=("order_id", "count"),
        total_sales=("sales", "sum"),
        avg_order_value=("sales", "mean"),
    )
    .sort_values("avg_order_value", ascending=False)
)

print(answer.to_string(index=False, float_format=lambda x: f"{x:.2f}"))
print("orders", len(merged))
print("total_sales", f"{merged['sales'].sum():.2f}")
```
