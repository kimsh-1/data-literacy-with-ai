# S20 모범답안

## 미션 모범답안

S19에서 만든 상권 분석 결과를 보고서로 바꾸면, 아래처럼 쓸 수 있습니다. 여기서 중요한 점은 분석 과정을 길게 설명하는 것이 아니라, 데이터를 보고 무엇을 발견했는지, 그래서 어떤 판단을 해야 하는지, 다음에 무엇을 할지까지 이어 주는 것입니다.

보고서는 길다고 좋은 것이 아닙니다. 키를 잴 때 줄자를 여러 개 들고 오는 것보다, 정확한 눈금 하나를 제대로 읽는 것이 더 중요하죠. 데이터 보고서도 같습니다. 숫자를 많이 늘어놓기보다 판단에 필요한 숫자를 골라 보여 주어야 합니다.

### 1페이지 보고서 예시

보고서 제목은 주제가 아니라 결론이어야 합니다. “서울 상권 분석”처럼 넓은 제목만 쓰면 읽는 사람이 무엇을 봐야 할지 바로 알기 어렵습니다. 아래 제목은 읽자마자 강남구가 가장 중요한 비교 대상이고, 한식과 커피점/카페 조합을 먼저 봐야 한다는 방향이 보입니다.

제목:

```text
서울 음식점·카페 상권은 강남구에 가장 집중되어 있으며, 구별 우선순위는 한식과 커피점/카페 조합부터 확인해야 한다.
```

핵심 발견은 데이터에서 실제로 확인한 사실입니다. 의견이나 예상이 아니라, 표와 그래프에서 바로 확인할 수 있는 숫자를 적는 자리입니다. 사전에서 단어 뜻을 찾듯이, 데이터에서 확인한 값만 가져온다고 생각해 보세요.

핵심 발견:

```text
분석 대상은 서울 음식점·카페 상권 데이터 119,158행이다.
구별 점포수는 강남구 12,095개, 마포구 7,396개, 송파구 7,106개 순으로 많다.
구별·업종 조합은 강남구 한식 3,917개, 마포구 한식 2,835개, 송파구 한식 2,626개, 강남구 커피점/카페 2,253개가 상위에 온다.
```

이 발견에서 눈여겨볼 숫자는 **119,158행**입니다. 분석한 데이터의 규모를 알려 주기 때문입니다. 그다음에는 강남구 12,095개, 마포구 7,396개, 송파구 7,106개처럼 구별 점포수를 비교합니다. 숫자를 넣어야 “많다”는 말이 감이 아니라 근거가 됩니다.

핵심 시각화는 결론을 가장 잘 보여 주는 차트 하나를 고르는 자리입니다. 차트를 여러 개 넣는다고 보고서가 더 좋아지는 것은 아닙니다. 읽는 사람이 “어디부터 봐야 하지?”라고 느끼지 않도록, 결론과 가장 가까운 차트 하나를 앞에 세워야 합니다.

핵심 시각화:

```text
구별·업종 조합 상위 10개 막대그래프를 사용한다.
이 차트는 어느 구와 업종 조합에 점포가 집중되어 있는지, 다음 조사 우선순위를 어디에 둘지 직접 보여준다.
```

시사점은 발견의 의미를 풀어 주는 문장입니다. 예를 들어 “강남구 점포수가 많다”는 발견이고, “그래서 상권 후보를 고를 때 강남구를 먼저 비교해야 한다”는 업무상 의미입니다. 발견과 행동 사이에 놓는 징검다리라고 보면 됩니다.

시사점:

```text
상권 후보를 고를 때는 전체 점포수가 많은 구를 먼저 보되, 실제 실행 우선순위는 업종별 밀집도까지 함께 보아야 한다. 강남구는 전체 점포수와 한식·커피점/카페 점포수가 모두 높으므로 1차 조사 후보로 적합하다.
```

권장 액션은 보고서를 읽은 사람이 실제로 할 일을 적는 부분입니다. “추가 분석 필요”라고만 쓰면 너무 막연합니다. 누가, 무엇을, 어떤 순서로 볼지까지 적어야 회의나 업무에서 바로 사용할 수 있습니다.

권장 액션:

```text
1. 상권 조사 담당자는 강남구, 마포구, 송파구를 1차 비교 대상으로 잡는다.
2. 각 구는 한식과 커피점/카페 점포수를 먼저 비교하고, 경쟁 강도가 높은 조합을 표시한다.
3. 회의 자료에는 구별 점포수 막대그래프 1개와 구별·업종 상위 10개 표 1개만 남기고, 원자료 전체 표는 부록으로 분리한다.
```

주의점은 보고서의 한계를 솔직하게 적는 자리입니다. 점포수만으로 출점 성공을 단정할 수는 없습니다. 음식점을 열지 말지 판단하려면 임대료, 유동인구, 매출, 배달 수요 같은 정보도 함께 봐야 합니다.

주의점:

```text
이 분석은 2021년 서울 음식점·카페 상권 데이터의 점포수만 사용한 교육용 예시다. 실제 출점 판단에는 임대료, 유동인구, 매출, 배달 수요, 폐업률, 접근성 같은 추가 정보가 필요하다.
```

동료 피드백은 “틀렸다”를 찾는 시간이 아니라, 보고서를 읽는 사람이 한 번에 이해할 수 있게 고치는 시간입니다. 특히 제목, 근거 숫자, 액션이 흐릿하면 아래처럼 구체적으로 고쳐 봅니다.

동료 피드백 반영 예시:

```text
받은 피드백 1: 제목은 결론이 보이지만 "무엇을 해야 하는지"가 조금 약하다.
반영 내용 1: 액션 영역에 "강남구, 마포구, 송파구를 1차 비교 대상으로 잡고 한식·커피점/카페 점포수를 먼저 비교한다"를 명확히 추가했다.

받은 피드백 2: 강남구가 왜 먼저인지 근거가 부족하다.
반영 내용 2: 구별 점포수 상위 3개와 구별·업종 조합 상위 값을 발견 문장에 추가했다.
```

### 3분 발표 원고 예시

발표는 보고서를 말로 읽는 시간이 아닙니다. 듣는 사람이 짧은 시간 안에 결론, 근거, 행동을 따라올 수 있게 순서를 잡는 시간입니다. 먼저 결론을 말하고, 그다음 숫자 근거를 두세 개만 보여 준 뒤, 마지막에 무엇을 하겠다고 말해 보세요.

```text
서울 음식점·카페 상권은 강남구에 가장 집중되어 있으며, 구별 우선순위는 한식과 커피점/카페 조합부터 확인해야 합니다.

분석 대상은 119,158행입니다. 구별 점포수는 강남구 12,095개, 마포구 7,396개, 송파구 7,106개 순으로 많습니다.

구별·업종 조합에서는 강남구 한식 3,917개, 마포구 한식 2,835개, 송파구 한식 2,626개, 강남구 커피점/카페 2,253개가 상위에 있습니다. 전체 점포수와 업종별 밀집도를 함께 보아야 한다는 뜻입니다.

따라서 상권 조사 담당자는 강남구, 마포구, 송파구를 먼저 비교하고, 각 구에서 한식과 커피점/카페의 경쟁 강도를 확인한 뒤 후보지를 좁히는 방식이 적절합니다. 보고서에는 구별·업종 조합 막대그래프 1개와 상위 10개 표 1개만 남기고 원자료 전체 표는 부록으로 분리하겠습니다.
```

## 평가 기준 해설

좋은 답안은 발견, 시사점, 액션이 서로 섞이지 않습니다. 발견에는 실제 숫자가 들어가고, 시사점에는 그 숫자가 왜 중요한지 들어가며, 액션에는 다음 행동이 들어갑니다.

상:

```text
발견, 시사점, 액션이 분리되어 있고, 발견에 실제 숫자가 2개 이상 들어 있다. 차트 선택 이유가 결론과 연결되어 있으며, 동료 피드백을 받아 제목이나 액션을 구체적으로 고쳤다.
```

중 수준의 답안은 형식은 갖추었지만, 읽는 사람이 바로 움직이기에는 조금 흐릿합니다. “추가 분석 필요”처럼 넓은 말만 쓰면 실제 업무 행동으로 이어지기 어렵습니다.

중:

```text
보고서 형식은 갖추었고 숫자도 일부 들어 있지만, 시사점이나 액션이 다소 추상적이다. 예를 들어 "추가 분석 필요"에서 멈추면 중 수준이다.
```

하 수준의 답안은 표나 차트가 있어도 결론이 보이지 않습니다. 숫자가 부족하거나, 앞 차시에서 만든 분석 결과와 연결되지 않으면 보고서라기보다 작업 메모에 가까워집니다.

하:

```text
차트나 표는 있으나 결론이 없고, 실제 숫자가 부족하거나 S19 결과물과 연결되지 않는다. 동료 피드백 반영 흔적도 없다.
```

## 검증 코드로 숫자 확인하기

아래 검증 코드는 수업 홈의 데이터 받기 페이지에서 내려받은 서울 음식점·카페 상권 파일을 읽어서, 보고서에 넣은 숫자가 맞는지 확인하는 용도입니다. 학생 입장에서는 코드를 모두 외울 필요는 없습니다. 보고서에 적은 숫자가 어디에서 나왔는지 확인하는 장치라고 보면 됩니다.

검증용 요약 자료는 엑셀로도 열리는 표 파일인 CSV 형식으로 저장될 수 있습니다. 초보자는 같은 내용의 엑셀 파일(.xlsx)이 제공되어 있다면 그것을 받아 열어도 됩니다.

실행 명령:

```bash
cd /mnt/d/ai-data-analysis-course
python3 06_answers/S20_code.py
```

핵심 코드:

```python
from pathlib import Path

import pandas as pd


ROOT = Path(__file__).resolve().parents[1]
CAFE_RESTAURANT_PATH = (
    ROOT / "02_datasets/real_korean/cafe_restaurant/seoul_commercial_food_stores_202109_github.csv"
)

cafe = pd.read_csv(CAFE_RESTAURANT_PATH)

top_gu = (
    cafe.groupby("시군구명")
    .size()
    .sort_values(ascending=False)
    .head(3)
    .reset_index(name="점포수")
)

top_gu_category = (
    cafe.groupby(["시군구명", "상권업종중분류명"])
    .size()
    .sort_values(ascending=False)
    .head(4)
    .reset_index(name="점포수")
)

print(len(cafe))
print(top_gu)
print(top_gu_category)
```

이 코드는 카페 매출 파일을 읽는 것이 아니라, 서울 음식점·카페 상권 점포 데이터를 읽습니다. `groupby`는 같은 항목끼리 묶는 기능입니다. 엑셀의 피벗테이블처럼 구별로 묶고, 다시 업종별로 묶어서 점포수를 세는 방식이라고 이해하면 됩니다.

실제 실행 출력 요약:

```text
pandas version: 3.0.3
서울 음식점·카페 상권 원자료: 119,158행
구별 점포수 상위 3개
시군구명   점포수
강남구   12095
마포구    7396
송파구    7106
구별·업종 조합 상위 4개
시군구명 상권업종중분류명  점포수
강남구        한식  3917
마포구        한식  2835
송파구        한식  2626
강남구    커피점/카페  2253
아파트 대안 예시: 강남구 아파트 실거래 3,754행, 전용면적-거래금액 상관계수 0.806
```

여기서 보고서에 사용한 숫자가 다시 확인됩니다. 서울 음식점·카페 상권 원자료는 119,158행이고, 구별 점포수는 강남구 12095, 마포구 7396, 송파구 7106 순입니다. 보고서 문장에서는 읽기 쉽도록 강남구 12,095개처럼 쉼표를 넣어 적으면 됩니다.

아파트 대안 예시도 함께 확인할 수 있습니다. 강남구 아파트 실거래 3,754행과 전용면적-거래금액 상관계수 0.806은 다른 주제로 보고서를 쓸 때 사용할 수 있는 검증 숫자입니다. 상관계수는 두 숫자가 얼마나 함께 움직이는지 보는 값입니다. 0.806처럼 1에 가까운 양수이면, 전용면적이 커질수록 거래금액도 같이 커지는 경향이 강하다고 읽습니다.
