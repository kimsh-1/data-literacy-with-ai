## 미션에서 만들 표

수업 홈의 데이터 받기 페이지에서 내려받은 서울 생활인구 파일을 사용해 자치구별 생활인구 요약표를 만듭니다. CSV는 엑셀로도 열리는 표 파일입니다. 같은 데이터가 엑셀 파일(.xlsx)로 제공되면 초보자는 그 파일을 받아서 확인해도 됩니다.

조건을 먼저 분명히 잡아 보겠습니다. 이 미션은 "서울 전체"와 "자치구"를 섞어서 보지 않고, **25개 자치구**만 비교하는 연습입니다.

- `서울시` 전체 합계 행은 제외합니다.
- `시군구명`별로 관측일수, 평균 생활인구, 평균 주간인구, 평균 야간인구, 최대 생활인구를 계산합니다.
- 평균 생활인구 기준으로 내림차순 정렬합니다.
- 결과가 25개 자치구인지 확인합니다.

## 미션 모범 코드

아래 코드는 파일을 읽고, 날짜를 바꾸고, 서울시 전체 행을 제외한 뒤 자치구별로 묶어 요약표를 만드는 전체 흐름입니다. 처음 볼 때는 한 줄씩 외우려고 하지 말고, "읽기 → 날짜 만들기 → 서울시 제외 → 자치구별 집계 → 보기 좋게 정리" 순서로 따라가 보세요.

```python
from pathlib import Path
import pandas as pd

ROOT = Path("/mnt/d/ai-data-analysis-course")
DATA = ROOT / "02_datasets"

living = pd.read_csv(
    DATA / "real_korean/seoul_living_pop/seoul_living_population_gu_daily.csv"
)

living["기준일"] = pd.to_datetime(living["기준일ID"].astype(str), format="%Y%m%d")

gu_living = living[living["시군구명"] != "서울시"].copy()

gu_summary = (
    gu_living.groupby("시군구명")
    .agg(
        관측일수=("기준일", "nunique"),
        평균생활인구=("총생활인구수", "mean"),
        평균주간인구=("주간인구수(09~18)", "mean"),
        평균야간인구=("야간인구수(19~08)", "mean"),
        최대생활인구=("일최대인구수", "max"),
    )
    .reset_index()
    .sort_values("평균생활인구", ascending=False)
)

display = gu_summary.copy()
for col in ["평균생활인구", "평균주간인구", "평균야간인구", "최대생활인구"]:
    display[col] = display[col].round(0).astype("int64")

print(living.shape)
print(living["기준일"].min(), living["기준일"].max())
print(display.head(10))
```

## 풀이 해설

생활인구 파일에는 `서울시` 전체 행과 25개 자치구 행이 함께 들어 있습니다. 여기서 `서울시`는 자치구가 아니라 전체 합계입니다. 자치구끼리 비교하려면 합계 행을 빼야 합니다. 합계까지 같이 넣으면 같은 사람 수를 두 번 보는 것처럼 비교가 흐려집니다.

```python
gu_living = living[living["시군구명"] != "서울시"].copy()
```

이 코드는 엑셀 필터에서 `서울시` 체크를 해제하는 작업과 같습니다. `서울시`가 아닌 행만 남기고, 그 결과를 `gu_living`이라는 새 표로 둡니다.

그 다음에 보는 부분이 `groupby("시군구명")`입니다. `groupby`는 같은 이름끼리 묶는 기능입니다. 사전에서 같은 단어끼리 한 칸에 모아 놓듯이, `강남구` 행은 `강남구`끼리, `송파구` 행은 `송파구`끼리 모읍니다. 그런 다음 각 묶음 안에서 평균, 최댓값, 날짜 개수 같은 계산을 합니다.

```python
.agg(
    관측일수=("기준일", "nunique"),
    평균생활인구=("총생활인구수", "mean"),
    평균주간인구=("주간인구수(09~18)", "mean"),
    평균야간인구=("야간인구수(19~08)", "mean"),
    최대생활인구=("일최대인구수", "max"),
)
```

`관측일수`에는 `nunique`를 썼습니다. 이것은 중복을 빼고 서로 다른 날짜가 몇 개인지 세는 방법입니다. 단순히 행 개수를 세는 것이 아니라 날짜의 개수를 세는 이유는, 모든 자치구가 같은 기간으로 비교되고 있는지 확인하기 위해서입니다. 실제 결과에서 모든 자치구가 2,992일로 확인됩니다.

`평균생활인구`, `평균주간인구`, `평균야간인구`는 각각 평균을 냅니다. 평균은 여러 날의 값을 하나의 대표값으로 보는 방법입니다. 키를 매일 잰 뒤 평균 키를 구하는 것처럼, 매일의 생활인구를 자치구별 대표값으로 바꾸는 과정입니다.

평균생활인구 기준 1위는 `강남구`입니다. 강남구는 평균주간인구가 평균야간인구보다 크게 나타납니다. 낮 시간에 업무나 상업 활동으로 들어오는 사람이 많을 수 있다고 조심스럽게 읽어 볼 수 있습니다. 다만 이 표만 보고 원인을 단정하지는 않습니다. 이 미션의 목적은 원인 분석이 아니라, 자치구별 요약표를 정확히 만드는 데 있습니다.

### 1. 주문 3표 병합과 매출 계산

수업 홈의 데이터 받기 페이지에서 내려받은 주문 파일, 고객 파일, 상품 파일을 함께 사용합니다. 주문 파일에는 고객 지역과 상품 단가가 바로 들어 있지 않습니다. 그래서 주문 표에 고객 표와 상품 표를 붙입니다. 엑셀로 치면 VLOOKUP이나 XLOOKUP으로 필요한 열을 가져오는 작업입니다.

```python
orders = pd.read_csv(DATA / "shop_orders/orders.csv")
customers = pd.read_csv(DATA / "shop_orders/customers.csv")
products = pd.read_csv(DATA / "shop_orders/products.csv")

merged = (
    orders.merge(customers, on="customer_id", how="left")
    .merge(products, on="product_id", how="left")
)

merged["sales"] = merged["price"] * merged["qty"] * (1 - merged["discount_rate"])
```

`merge`는 두 표를 공통된 열 기준으로 붙이는 기능입니다. 여기서는 `customer_id`로 고객 정보를 붙이고, `product_id`로 상품 정보를 붙입니다. 그 뒤에 `price * qty * (1 - discount_rate)`로 매출을 계산합니다. 단가에 수량을 곱하고, 할인율만큼 빼 주는 식입니다.

검증 결과:

| 항목 | 값 |
|---|---:|
| 주문 원본 | 20,000행, 6열 |
| 고객 원본 | 3,000행, 7열 |
| 상품 원본 | 200행, 5열 |
| 병합 후 | 20,000행, 17열 |
| 병합 누락값 합계 | 0 |
| 전체 매출 합계 | 1,938,489,803 |

여기서 병합 후에도 20,000행이라는 점을 확인해야 합니다. 주문이 늘어나거나 줄어들면 병합 기준이 잘못되었을 수 있습니다. 병합 누락값 합계가 0이라는 것은 주문마다 필요한 고객 정보와 상품 정보가 잘 붙었다는 뜻입니다.

### 2. 지역×카테고리 다중 집계

이제 주문마다 계산된 매출을 지역과 카테고리 기준으로 묶어 봅니다. `region` 하나만 보는 것이 아니라, `region` 안에서 다시 `category`를 나누어 보는 방식입니다. 엑셀 피벗 테이블에서 행에 지역과 카테고리를 차례로 넣는 모습과 비슷합니다.

```python
region_category_summary = (
    merged.groupby(["region", "category"])
    .agg(
        주문건수=("order_id", "nunique"),
        판매수량=("qty", "sum"),
        매출합계=("sales", "sum"),
        평균주문금액=("sales", "mean"),
    )
    .reset_index()
    .sort_values("매출합계", ascending=False)
)
```

`주문건수`는 주문번호의 고유 개수를 셉니다. 같은 주문번호가 여러 번 섞일 가능성을 생각해서, 중복을 빼고 세는 방식입니다. `판매수량`은 수량을 모두 더하고, `매출합계`는 매출을 모두 더합니다. `평균주문금액`은 주문 한 건당 평균 매출을 보는 값입니다.

상위 결과:

| region | category | 주문건수 | 판매수량 | 매출합계 | 평균주문금액 |
|---|---|---:|---:|---:|---:|
| 서울특별시 | 디지털 | 612 | 658 | 210,356,387 | 343,720 |
| 경기도 | 디지털 | 514 | 565 | 188,110,354 | 365,973 |
| 서울특별시 | 패션 | 990 | 1,442 | 86,206,229 | 87,077 |
| 경기도 | 패션 | 884 | 1,292 | 75,963,357 | 85,931 |

이 표에서는 서울특별시 디지털 카테고리의 매출합계가 210,356,387로 가장 위에 있습니다. 이렇게 정렬하면 어떤 지역과 카테고리 조합이 매출을 많이 만들었는지 바로 볼 수 있습니다.

### 3. 지역×카테고리 피벗

같은 집계라도 표 모양을 바꾸면 더 읽기 쉬워질 때가 있습니다. `pivot_table`은 지역을 행에 두고, 카테고리를 열로 펼쳐서 보여 줍니다. 엑셀 피벗 테이블을 떠올리면 됩니다.

```python
pivot = pd.pivot_table(
    merged,
    index="region",
    columns="category",
    values="sales",
    aggfunc="sum",
    fill_value=0,
    margins=True,
    margins_name="합계",
)
```

피벗은 같은 집계를 사람이 비교하기 쉬운 가로형 표로 펼친 것입니다. 지역은 행, 카테고리는 열, 값은 매출 합계입니다. `fill_value=0`은 비어 있는 칸을 0으로 채우라는 뜻입니다. `margins=True`는 합계 행과 합계 열을 함께 보겠다는 설정입니다.

### 4. 231열 인구 파일 `melt`

수업 홈의 데이터 받기 페이지에서 내려받은 인구 파일은 열이 매우 많습니다. 원본은 18,623행, 231열입니다. 열이 231개라는 말은 옆으로 매우 넓은 표라는 뜻입니다. 이런 표는 눈으로 보기에는 익숙할 수 있지만, 분석하거나 그래프로 그리기에는 불편합니다.

`melt`는 넓은 표를 긴 표로 바꾸는 기능입니다. 예를 들어 `0세남자`, `1세남자`처럼 열 이름에 들어 있던 정보가 `연령성별`이라는 열의 값으로 내려옵니다. 옷장 여러 칸에 흩어진 물건을 한 바구니에 모으고, 물건 이름표를 붙이는 느낌으로 보면 됩니다.

```python
pop = pd.read_csv(
    DATA / "real_korean/population_kosis/mois_legal_dong_gender_age_population_20260531.csv"
)

id_cols = ["법정동코드", "기준연월", "시도명", "시군구명", "읍면동명", "리명"]
age_gender_cols = [
    col
    for col in pop.columns
    if (col.endswith("남자") or col.endswith("여자")) and col not in ["남자", "여자"]
]

seoul_pop = pop[pop["시도명"] == "서울특별시"].copy()

long_pop = seoul_pop.melt(
    id_vars=id_cols,
    value_vars=age_gender_cols,
    var_name="연령성별",
    value_name="인구수",
)

long_pop["성별"] = long_pop["연령성별"].str.extract("(남자|여자)$")
long_pop["연령"] = (
    long_pop["연령성별"].str.replace("(남자|여자)$", "", regex=True).str.strip()
)
```

먼저 서울특별시 행만 남긴 뒤 `melt`합니다. 큰 데이터를 무조건 한 번에 바꾸기보다, 필요한 범위로 줄인 뒤 다루는 편이 초보자에게 훨씬 안전합니다. `연령성별`에서 맨 끝의 `남자` 또는 `여자`를 뽑아 `성별` 열을 만들고, 나머지 부분을 `연령`으로 정리합니다.

검증 결과:

| 항목 | 값 |
|---|---:|
| 원본 인구 파일 | 18,623행, 231열 |
| 서울특별시 wide | 452행, 231열 |
| melt 대상 연령·성별 열 | 222개 |
| 서울특별시 long | 100,344행, 10열 |

`long_pop["인구수"].sum()`은 서울특별시 wide 데이터의 `남자` 합계와 `여자` 합계를 더한 값과 일치했습니다. 이 확인은 모양을 바꾸는 과정에서 인구수가 사라지거나 중복되지 않았는지 보는 검산입니다.

### 5. 서울 음식점 상권 업종 상위 10개

수업 홈의 데이터 받기 페이지에서 내려받은 서울 음식점 상권 업소 파일은 119,158행입니다. 이런 큰 표는 원본을 그대로 오래 들여다보기보다 먼저 요약해야 합니다. 여기서는 업종 중분류별로 업소 수를 세어 상위 10개를 봅니다.

```python
stores = pd.read_csv(
    DATA / "real_korean/cafe_restaurant/seoul_commercial_food_stores_202109_github.csv"
)

top10 = (
    stores.groupby("상권업종중분류명")
    .size()
    .reset_index(name="업소수")
    .sort_values("업소수", ascending=False)
    .head(10)
)
```

`groupby("상권업종중분류명").size()`는 업종 이름별로 행이 몇 개인지 세는 코드입니다. 여기서는 행 하나가 업소 하나라고 볼 수 있으므로, 행 개수를 세면 업소 수가 됩니다.

상위 결과:

| 상권업종중분류명 | 업소수 |
|---|---:|
| 한식 | 40,448 |
| 커피점/카페 | 19,703 |
| 유흥주점 | 11,693 |
| 분식 | 10,199 |
| 일식/수산물 | 7,985 |
| 양식 | 6,519 |
| 닭/오리요리 | 5,733 |
| 패스트푸드 | 5,361 |
| 제과제빵떡케익 | 4,815 |
| 중식 | 3,245 |

이 결과를 보면 서울 음식점 상권 업소 중 한식이 40,448개로 가장 많고, 커피점/카페가 19,703개로 그다음입니다. 이 숫자는 원본 119,158행을 업종별 개수 표로 줄였기 때문에 한눈에 읽을 수 있습니다.

검증 코드는 수업 답안 자료의 S13 코드 파일에 들어 있습니다. 실행 후에는 아래 출력처럼 주요 행수와 집계 결과가 맞는지 확인해 보세요.
