## 미션 목표

수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일을 사용해 월별 매출 추세 라인차트와 BI 도구에 넘길 수 있는 인수인계용 CSV, 즉 엑셀로도 열리는 표 파일을 저장합니다. 초보자는 엑셀 파일(.xlsx)을 받아서 먼저 열 구조를 확인해도 됩니다. 다만 아래 모범 코드는 같은 데이터를 CSV 기준으로 읽습니다.

이번 미션에서 확인할 조건은 이렇습니다.

- `date`를 날짜형으로 변환합니다.
- 월별 `amount` 합계를 계산합니다.
- 월별 주문수와 객단가도 함께 계산합니다.
- 월별 매출 라인차트를 PNG로 저장합니다.
- 월별 집계표를 CSV로 저장합니다.
- 6월 매출이 가장 높고, 2월 매출이 가장 낮은지 확인합니다.

여기서 날짜형으로 바꾸는 이유는 단순한 글자처럼 보이는 날짜를 파이썬이 실제 달력 날짜로 알아보게 만들기 위해서입니다. 그래야 1월 주문끼리, 2월 주문끼리 정확히 묶을 수 있습니다. 월별 집계는 주문 한 건 한 건을 그대로 보는 대신, 달마다 매출이 얼마나 모였는지 보는 과정입니다. 키를 잴 때 매 순간을 다 적는 것이 아니라 한 번의 숫자로 정리하듯이, 주문 데이터를 월 단위 숫자로 줄여 보는 것입니다.

## 미션 모범 코드

아래 코드는 카페 매출 미션만 따로 실행할 수 있는 최소 코드입니다. 전체 따라하기와 검증은 S14 전체 실행 코드에서 함께 확인할 수 있습니다.

코드 흐름을 먼저 눈으로 봐 주세요. `pd.read_csv()`로 카페 매출 표를 불러오고, `pd.to_datetime()`으로 날짜를 날짜형으로 바꿉니다. 그다음 `pd.Grouper(key="date", freq="MS")`로 월 시작일 기준 묶음을 만듭니다. `MS`는 Month Start, 즉 월의 시작일 기준이라는 뜻입니다. 마지막으로 매출합계, 주문수, 객단가를 만든 뒤 CSV와 차트로 저장합니다.

```python
from pathlib import Path

import matplotlib.pyplot as plt
import matplotlib.ticker as mticker
import pandas as pd
import seaborn as sns
from matplotlib import font_manager

ROOT = Path("/mnt/d/ai-data-analysis-course")
DATA = ROOT / "02_datasets"
OUT = ROOT / "06_answers/S14_outputs"
OUT.mkdir(parents=True, exist_ok=True)

font_candidates = [
    "NanumGothic",
    "NanumGothicCoding",
    "Noto Sans CJK KR",
    "Malgun Gothic",
    "AppleGothic",
    "DejaVu Sans",
]
available_fonts = {font.name for font in font_manager.fontManager.ttflist}
selected_font = next(
    (font for font in font_candidates if font in available_fonts),
    "DejaVu Sans",
)

sns.set_theme(style="whitegrid")
plt.rcParams["font.family"] = selected_font
plt.rcParams["axes.unicode_minus"] = False

cafe = pd.read_csv(DATA / "cafe_sales/cafe_sales_clean.csv")
cafe["date"] = pd.to_datetime(cafe["date"], errors="coerce")
cafe = cafe.dropna(subset=["date", "amount"]).copy()

monthly_sales = (
    cafe.groupby(pd.Grouper(key="date", freq="MS"))
    .agg(매출합계=("amount", "sum"), 주문수=("order_id", "count"))
    .reset_index()
    .rename(columns={"date": "월"})
)
monthly_sales["객단가"] = monthly_sales["매출합계"] / monthly_sales["주문수"]

monthly_sales.to_csv(
    OUT / "s14_monthly_sales.csv",
    index=False,
    encoding="utf-8-sig",
)

fig, ax = plt.subplots(figsize=(8.8, 5.2))
sns.lineplot(
    data=monthly_sales,
    x="월",
    y="매출합계",
    marker="o",
    color="#A3D576",
    linewidth=1.2,
    ax=ax,
)

ax.set_title("카페 월별 매출 추세")
ax.set_xlabel("월")
ax.set_ylabel("매출합계")
ax.yaxis.set_major_formatter(
    mticker.FuncFormatter(lambda x, _: f"{x / 1_000_000:.1f}백만")
)

plt.tight_layout()
plt.savefig(OUT / "s14_monthly_sales_trend.png", dpi=160)
plt.close(fig)

print("선택된 글꼴:", selected_font)
print(monthly_sales)
print("CSV:", OUT / "s14_monthly_sales.csv")
print("차트:", OUT / "s14_monthly_sales_trend.png")
```

`객단가`는 주문 한 건당 평균 매출입니다. 계산식은 `매출합계 / 주문수`입니다. 예를 들어 한 달 매출을 그 달 주문수로 나누면, 손님 한 번의 주문이 평균적으로 얼마였는지 볼 수 있습니다. 매출이 올랐을 때 주문수가 늘어서 오른 것인지, 주문 한 건의 금액이 커져서 오른 것인지 나눠 보기 위한 보조 숫자입니다.

`to_csv()`로 저장하는 이유도 중요합니다. 차트만 저장하면 사람은 보기 좋지만, BI 도구나 다른 분석 도구는 다시 계산하기 어렵습니다. 그래서 차트에 쓴 월별 집계표를 함께 저장해 두면, 나중에 대시보드로 넘길 때 같은 숫자를 다시 사용할 수 있습니다.

## 주요 전처리와 저장 결과

### 1. 서울 지하철 시간대 히트맵

사용 데이터는 수업 홈의 데이터 받기 페이지에서 내려받은 서울 지하철 시간대별 승하차 파일입니다.

이 실습에서는 원본 199,424행을 그대로 그리지 않습니다. 그대로 그리면 너무 많은 숫자가 한 화면에 몰려서 읽기 어렵습니다. 그래서 “어느 역의 어느 시간대 승차가 많은가?”라는 질문에 맞게 표를 줄입니다.

전처리에서 하는 일입니다.

- `승하차구분 == "승차"` 행만 사용합니다.
- `호선 + 역명`으로 `역라벨`을 만듭니다.
- 20개 시간대 열을 역별로 합산합니다.
- 총 승차 상위 10개 역만 히트맵에 사용합니다.
- 10개 역 × 20개 시간대를 긴 표로 바꾸어 200행 CSV로 저장합니다.

긴 표는 한 줄에 하나의 조합만 놓는 방식입니다. 예를 들어 “강남역, 08-09시간대, 승차인원”처럼 사전에서 단어를 찾듯이 역과 시간대를 한 줄씩 펼쳐 놓습니다. 이렇게 저장하면 BI 도구에서 다시 피벗하거나 필터링하기 쉽습니다.

실행하면 지하철 히트맵 이미지, SVG 이미지, BI에서 쓰기 좋은 지하철 시간대별 긴 표 CSV가 만들어집니다.

### 2. 강남 아파트 실거래 차트

사용 데이터는 수업 홈의 데이터 받기 페이지에서 내려받은 강남구 아파트 실거래 파일입니다.

이 데이터에서는 거래금액 분포와 면적에 따른 가격 차이를 봅니다. 처음에는 `거래금액(만원)`이 숫자처럼 보여도 쉼표가 들어간 문자열일 수 있습니다. 그래서 계산할 수 있는 숫자형으로 바꿔야 합니다.

전처리에서 하는 일입니다.

- `거래금액(만원)`의 쉼표를 제거하고 숫자형 `거래금액_만원`을 만듭니다.
- `거래금액_억원 = 거래금액_만원 / 10,000`을 만듭니다.
- `계약년월`을 날짜형 `계약월`로 변환합니다.
- 전용면적을 5개 `면적구간`으로 나눕니다.

검증 결과입니다.

```text
원본: 3,754행, 20열
분석 가능 행수: 3,754행
전용면적 범위: 12.1㎡ ~ 273.96㎡
거래금액 범위: 15,000만원 ~ 1,380,000만원
전용면적-거래금액 상관계수: r = 0.806
```

실행하면 거래금액 히스토그램, 면적구간별 박스플롯, 전용면적과 거래금액 산점도, 그리고 차트에 사용한 아파트 정제 CSV가 만들어집니다.

히스토그램은 거래금액이 어느 구간에 몰려 있는지 보는 그림입니다. 박스플롯은 면적구간별로 중앙값과 퍼짐을 한 번에 보는 그림입니다. 산점도는 한 거래를 점 하나로 찍어서 전용면적과 거래금액의 관계를 보는 그림입니다.

### 3. FRED CPI 월별 라인차트

사용 데이터는 수업 홈의 데이터 받기 페이지에서 내려받은 한국 CPI 월별 파일입니다. CPI는 소비자물가지수입니다. 물가가 시간에 따라 어떻게 움직였는지 보는 대표적인 시계열 지표라고 생각하면 됩니다.

전처리에서 하는 일입니다.

- `observation_date`를 날짜형 `기준월`로 변환합니다.
- `KORCPIALLMINMEI`를 숫자형 `소비자물가지수`로 사용합니다.
- 2000년 1월 이후 데이터만 필터링합니다.

검증 결과입니다.

```text
원본: 767행, 2열
원본 기간: 1960-01-01 ~ 2023-11-01
2000년 이후: 287행
2000년 이후 기간: 2000-01-01 ~ 2023-11-01
```

실행하면 CPI 월별 라인차트 이미지, SVG 이미지, 2000년 이후 CPI CSV가 만들어집니다.

라인차트는 시간 순서가 있는 데이터에 잘 맞습니다. CPI는 월별로 이어지는 숫자이므로 막대 하나씩 비교하기보다 선으로 이어서 흐름을 보는 편이 자연스럽습니다.

### 4. 카페 월별 매출 미션

사용 데이터는 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일입니다.

전처리에서 하는 일입니다.

- `date`를 날짜형으로 변환합니다.
- 월 시작일 기준 `pd.Grouper(key="date", freq="MS")`로 묶습니다.
- `amount` 합계와 `order_id` 개수를 집계합니다.
- `객단가 = 매출합계 / 주문수`를 계산합니다.

실행하면 월별 매출 집계 CSV, 월별 매출 추세 PNG 차트, 월별 매출 추세 SVG 차트가 만들어집니다.

여기서 월 시작일 기준으로 묶는다는 말은 2025년 1월에 속한 주문을 `2025-01-01`, 2025년 2월에 속한 주문을 `2025-02-01`이라는 대표 날짜로 모은다는 뜻입니다. 달력을 한 장씩 넘기며 주문을 바구니에 담는다고 생각하면 됩니다.
