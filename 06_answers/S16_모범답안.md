# S16 모범답안: 지역 차원으로 매출 분해

## 미션

Power BI에서 `customers`, `products`, `날짜테이블` 차원을 `orders` 팩트 테이블에 연결한 뒤, 고객 차원의 `region`으로 매출을 분해한다.

핵심은 `orders`에 지역 컬럼을 직접 만들지 않는 것이다. 지역은 `customers[region]`에서 가져오고, 매출은 관계를 따라 `orders`의 주문 행을 기준으로 계산한다.

## 정답 모델

최종 관계는 다음과 같아야 한다.

| 1쪽 테이블 | 1쪽 컬럼 | 다쪽 테이블 | 다쪽 컬럼 | 상태 |
|---|---|---|---|---|
| `customers` | `customer_id` | `orders` | `customer_id` | 활성 |
| `products` | `product_id` | `orders` | `product_id` | 활성 |
| `날짜테이블` | `Date` | `orders` | `order_date` | 활성 |

세 관계 모두 카디널리티는 1:다, 교차 필터 방향은 단일 방향이다.

## 날짜 차원 DAX

```DAX
날짜테이블 =
ADDCOLUMNS(
    CALENDAR(MIN(orders[order_date]), MAX(orders[order_date])),
    "연도", YEAR([Date]),
    "월번호", MONTH([Date]),
    "연월", FORMAT([Date], "yyyy-MM"),
    "연월정렬", YEAR([Date]) * 100 + MONTH([Date])
)
```

확인값:

| 항목 | 값 |
|---|---:|
| 첫 날짜 | 2024-01-01 |
| 마지막 날짜 | 2024-12-31 |
| 날짜 행 수 | 366 |

`날짜테이블[Date]`는 날짜 테이블로 지정한다. `날짜테이블[연월]`은 `날짜테이블[연월정렬]`로 정렬한다.

## 측정값 DAX

```DAX
Total Sales =
SUMX(
    orders,
    orders[qty] * RELATED(products[price]) * (1 - orders[discount_rate])
)
```

```DAX
Order Count =
DISTINCTCOUNT(orders[order_id])
```

```DAX
Avg Order Value =
DIVIDE([Total Sales], [Order Count])
```
