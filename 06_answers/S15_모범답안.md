# S15 모범답안: 총매출 KPI 카드

## 미션에서 확인해야 하는 답

미션의 답은 카페 매출 파일을 Power BI Desktop에 불러온 뒤, `amount`의 합계를 카드로 보여주는 것입니다. 여기서 CSV는 엑셀로도 열리는 표 파일입니다. 수업에서는 CSV를 기준으로 설명하지만, 같은 데이터를 담은 엑셀 파일(.xlsx)을 받아도 괜찮습니다.

정답 카드 값:

```text
90,492,500
```

이 숫자는 2025년 1월 1일부터 6월 30일까지의 매출을 전부 더한 값입니다. 엑셀에서 매출 열 전체에 `SUM`을 건 것과 같은 의미라고 보면 됩니다.

Power BI에서 카드가 `90.49M` 또는 `90M`처럼 보일 수 있습니다. 이때 계산이 틀린 것은 아닙니다. Power BI가 큰 숫자를 보기 좋게 줄여서 보여준 것입니다. 여기서 `M`은 백만 단위입니다. 정확한 금액을 확인하려면 카드 서식에서 표시 단위(Display units)를 `없음(None)`, 소수 자릿수(Decimal places)를 `0`으로 바꿔 보세요. 그러면 `90,492,500`으로 확인할 수 있습니다.

## Power BI에서 맞춰야 하는 설정

수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일을 사용합니다. 파일 위치를 외우는 것이 아니라, Power BI에서 “데이터 가져오기”를 눌러 받은 파일을 선택한다고 생각하면 됩니다.

| 항목 | 설정 |
|---|---|
| 데이터 원본 | 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 CSV 파일 |
| 표 이름 | `cafe_sales_clean` |
| 시각화 | 카드(Card) 또는 카드(새)(Card new) |
| 값 필드 | `amount` |
| 집계 방식 | 합계(Sum) |
| 제목 | `총매출` 또는 `총매출(원)` |
| 표시 단위 | 없음(None) |
| 소수 자릿수 | 0 |

카드에 `10,424`가 보이면 총매출이 아니라 주문 건수나 행 개수를 보여주고 있을 가능성이 높습니다. 데이터가 10,424줄이기 때문에 이런 값이 나옵니다.

카드에 약 `8,681`이 보이면 `amount`를 모두 더한 것이 아니라 평균을 낸 상태일 가능성이 높습니다. 평균은 주문 한 건당 대략 얼마인지 보는 값이고, 이번 미션의 목표인 총매출과는 다릅니다. 여기서는 반드시 `amount`의 합계를 보아야 합니다.

## 첫 페이지에 함께 있어야 하는 시각화

첫 페이지에는 총매출 숫자 하나만 덩그러니 두는 것이 아니라, 매출을 이해하는 데 필요한 세 가지 화면이 함께 있으면 좋습니다. 하나는 전체 크기를 보는 카드, 하나는 상품군끼리 비교하는 막대그래프, 하나는 날짜 흐름을 보는 꺾은선그래프입니다.

| 시각화 | 필드 배치 | 목적 |
|---|---|---|
| 총매출 KPI 카드 | `amount` 합계 | 핵심 숫자 확인 |
| 카테고리별 매출 막대그래프 | X축 `category`, Y축 `amount` 합계 | 상품군별 매출 비교 |
| 일자별 매출 꺾은선그래프 | X축 `date`, Y축 `amount` 합계 | 기간 흐름 확인 |

카테고리별 매출은 상품군별로 돈이 얼마나 나왔는지 보는 표입니다. 키를 재듯이 같은 기준으로 세워 놓고 비교하는 장면이라고 생각해 보세요. 커피가 가장 크고, 그다음이 음료, 디저트 순서입니다.

## 검증용 Python 코드

S15는 Power BI 차시이므로 별도 `S15_code.py` 파일을 제출할 필요는 없습니다. 아래 코드는 모범답안의 숫자가 실제 데이터에서 맞는지 다시 계산해 본 강사용 검산 코드입니다. 학생 실습에서는 이 코드를 따라 치는 것이 목표가 아닙니다. Power BI 화면에서 같은 값이 나오는지 확인하면 됩니다.

```python
from pathlib import Path
import pandas as pd

path = Path("/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.csv")
df = pd.read_csv(path)

print("shape:", df.shape)
print("date_min:", df["date"].min())
print("date_max:", df["date"].max())
print("total_amount:", int(df["amount"].sum()))
print("order_count:", len(df))
print("null_total:", int(df.isna().sum().sum()))

category_summary = (
    df.groupby("category", as_index=False)["amount"]
    .sum()
    .sort_values("amount", ascending=False)
)
print(category_summary.to_string(index=False))

df["date_dt"] = pd.to_datetime(df["date"])
monthly = df.groupby(df["date_dt"].dt.to_period("M"))["amount"].sum().reset_index()
monthly["date_dt"] = monthly["date_dt"].astype(str)
print(monthly.to_string(index=False))
```

여기서 `shape: (10424, 12)`는 데이터가 10,424행, 12열이라는 뜻입니다. `total_amount: 90492500`은 총매출을 쉼표 없이 출력한 값이고, 화면에서는 `90,492,500`으로 읽으면 됩니다. `null_total: 0`은 비어 있는 값이 없다는 뜻입니다.

## 제출 화면을 말로 설명한 예시

상단에는 `총매출(원)` 카드가 있고 값은 `90,492,500`입니다. 이 카드는 보고서를 열자마자 가장 먼저 확인하는 큰 숫자입니다.

왼쪽에는 `카테고리별 매출` 막대그래프가 있습니다. 커피 막대가 가장 높고, 음료, 디저트 순서로 보입니다. 이 그래프는 어떤 상품군이 매출을 가장 많이 만들었는지 비교하기 위한 화면입니다.

오른쪽 또는 아래에는 `일자별 매출` 꺾은선그래프가 있습니다. 2025년 1월 1일부터 6월 30일까지의 매출 흐름이 보입니다. 날짜 축은 사전에서 날짜를 찾듯이 시간 순서대로 읽으면 됩니다.

## 흔한 오답과 수정 방법

Power BI에서는 필드를 잘못 넣어도 그래프가 만들어지기 때문에, 숫자가 맞는지 꼭 확인해야 합니다. 모양이 그럴듯해 보여도 값 필드와 집계 방식이 다르면 답이 달라집니다.

| 오답 상태 | 원인 | 수정 |
|---|---|---|
| 카드 값이 `10,424` | 주문 건수 또는 행 개수를 표시함 | `amount`를 값에 넣고 합계로 바꾼다 |
| 카드 값이 약 `8,681` | `amount` 평균을 표시함 | 집계 방식을 합계로 바꾼다 |
| 카드 값이 `90.49M` | 표시 단위가 자동 축약됨 | 표시 단위를 없음으로 바꾼다 |
| 막대그래프 값이 카테고리 개수처럼 보임 | `amount`가 아니라 `category` 개수가 값에 들어감 | Y축 또는 값에 `amount` 합계를 넣는다 |
| 꺾은선그래프가 연도 한 점만 보임 | 날짜 계층 구조의 연도 수준만 표시됨 | X축에서 `date` 자체를 선택하거나 날짜 계층을 일 수준으로 펼친다 |

카드 값이 이상할 때는 먼저 `amount`가 들어갔는지 보세요. 그다음 `합계(Sum)`인지 확인합니다. 이 두 가지가 맞으면 대부분 정답에 가까워집니다.

날짜 그래프가 한 점만 보이면 Power BI가 날짜를 연도 단위로 묶어 버린 상태일 수 있습니다. 일자별 흐름을 보고 싶을 때는 `date` 자체를 X축에 놓거나, 날짜 계층을 일 수준까지 펼쳐 보세요.
