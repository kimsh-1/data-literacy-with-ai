# S17 모범답안

## 미션에서 만드는 것

쇼핑 주문 데이터로 시계열 KPI 대시보드 한 페이지를 만드는 답안입니다. 여기서 KPI는 매출, 주문건수, 평균주문금액처럼 화면에서 계속 확인해야 하는 중요한 숫자를 뜻합니다. 완성 화면에는 월별 총매출 흐름, 고객 등급 슬라이서, 카테고리 드릴다운, 전월대비 성장률이 함께 보여야 합니다.

Power BI 파일을 자동으로 만들어 주는 답안은 아닙니다. 여러분이 Power BI에서 직접 만든 화면이 맞는지 확인할 수 있도록 DAX 측정값과 검증값을 기준으로 제시합니다. DAX는 Power BI 안에서 계산식을 만들 때 쓰는 언어라고 보면 됩니다.

## 데이터 검증 결과

수업 홈의 데이터 받기 페이지에서 내려받은 쇼핑 주문 데이터를 pandas 3.0.3으로 열어 확인했습니다. pandas는 파이썬에서 표 데이터를 확인할 때 자주 쓰는 도구입니다. CSV는 엑셀로도 열리는 표 파일이라는 뜻이고, 초보자는 같은 데이터가 담긴 엑셀 파일(.xlsx)을 받아서 진행해도 됩니다.

| 데이터 | 실제 행수 | 열수 |
|---|---:|---:|
| 주문 데이터 | 20,000 | 6 |
| 고객 데이터 | 3,000 | 7 |
| 상품 데이터 | 200 | 5 |

데이터가 제대로 들어왔는지 볼 때는 행수부터 확인합니다. 키를 재기 전에 줄자가 맞는지 보는 것과 같습니다. 주문 데이터는 20,000행, 고객 데이터는 3,000행, 상품 데이터는 200행이어야 합니다.

추가 확인:

| 항목 | 값 |
|---|---:|
| 주문 기간 | 2024-01-01 ~ 2024-12-31 |
| 주문 월수 | 12개월 |
| 날짜테이블 행수 | 366 |
| 고객 조인 누락 | 0건 |
| 상품 조인 누락 | 0건 |
| 전체 주문건수 | 20,000 |
| 전체 판매수량 | 29,535 |
| 전체 매출 | 1,938,489,803 |
| 평균주문금액 | 96,924.49 |

주문 기간은 2024년 1월 1일부터 2024년 12월 31일까지입니다. 2024년은 윤년이라 날짜테이블이 366행인 것도 맞습니다. 고객 조인 누락과 상품 조인 누락이 모두 0건이라는 말은, 주문표에 적힌 고객과 상품을 고객표와 상품표에서 모두 찾을 수 있다는 뜻입니다. 사전에서 단어를 찾았는데 빠진 단어가 하나도 없는 상태라고 보면 됩니다.

매출 계산식:

```text
매출 = products[price] × orders[qty] × (1 - orders[discount_rate])
```

주문 데이터에는 매출이라는 완성된 열이 따로 없습니다. 그래서 상품 단가에 주문 수량을 곱하고, 할인율만큼을 빼서 매출을 계산합니다. 이 계산으로 나온 전체 매출은 **1,938,489,803원**입니다.

## 모델 관계 정답

Power BI 모델 보기에서 아래 관계가 있어야 합니다.

| 1쪽 | 다쪽 | 방향 |
|---|---|---|
| `customers[customer_id]` | `orders[customer_id]` | 단일 |
| `products[product_id]` | `orders[product_id]` | 단일 |
| `날짜테이블[Date]` | `orders[order_date]` | 단일 |

여기서 1쪽은 기준표, 다쪽은 기록표라고 생각하면 쉽습니다. 고객 한 명은 여러 번 주문할 수 있으므로 `customers`가 1쪽, `orders`가 다쪽입니다. 상품 하나도 여러 주문에 등장할 수 있으므로 `products`가 1쪽, `orders`가 다쪽입니다.

`날짜테이블`은 날짜 테이블로 표시하고, 날짜 열은 `Date`로 지정합니다. 날짜테이블은 시간 계산을 할 때 기준이 되는 달력입니다. 전월매출처럼 “바로 전 달”을 찾아야 하는 계산은 이 달력이 있어야 안정적으로 움직입니다.

## 날짜테이블 DAX

S16에서 날짜테이블을 이미 만들었다면 다시 만들 필요는 없습니다. 아직 없다면 아래 DAX로 만듭니다.

```DAX
날짜테이블 =
ADDCOLUMNS(
    CALENDAR(DATE(2024, 1, 1), DATE(2024, 12, 31)),
    "연도", YEAR([Date]),
    "월번호", MONTH([Date]),
    "연월", FORMAT([Date], "yyyy-MM"),
    "연월정렬", YEAR([Date]) * 100 + MONTH([Date])
)
```

이 식은 2024년 1월 1일부터 2024년 12월 31일까지 하루씩 들어 있는 달력을 만듭니다. `연월`은 화면에 2024-01처럼 보기 좋게 표시하려고 만든 열이고, `연월정렬`은 2024-01, 2024-02, 2024-03 순서로 제대로 정렬하기 위해 만든 숫자 열입니다.

생성 후 `날짜테이블[연월]`은 `날짜테이블[연월정렬]` 기준으로 정렬합니다.

## 측정값 모범답안

측정값은 Power BI에서 반복해서 쓰는 계산식입니다. 엑셀 피벗테이블의 값 영역에 “합계”나 “평균”을 넣는 것과 비슷하지만, Power BI에서는 그 계산식을 이름 붙여 저장해 두고 여러 시각화에서 다시 꺼내 씁니다.

### 총매출

```DAX
총매출 =
SUMX(
    orders,
    RELATED(products[price]) * orders[qty] * (1 - orders[discount_rate])
)
```

`SUMX`는 주문표를 한 줄씩 보면서 매출을 계산한 뒤 전부 더합니다. 주문표에는 수량과 할인율이 있고, 상품 단가는 상품표에 있습니다. 그래서 `RELATED(products[price])`로 연결된 상품표에서 단가를 가져옵니다.

검증값:

```text
1,938,489,803
```

총매출 카드나 표에서 이 값이 나오면 계산식과 관계 설정이 제대로 맞은 것입니다.

### 주문건수

```DAX
주문건수 =
DISTINCTCOUNT(orders[order_id])
```

주문건수는 주문번호가 몇 개 있는지 세는 값입니다. 같은 주문번호가 중복되어 들어오는 상황까지 생각해서 `DISTINCTCOUNT`를 씁니다. `DISTINCTCOUNT`는 중복을 빼고 고유한 값만 세는 함수입니다.

검증값:

```text
20,000
```

### 총수량

```DAX
총수량 =
SUM(orders[qty])
```

총수량은 주문된 상품 수량을 모두 더한 값입니다. 주문 한 건이 상품 1개일 수도 있고 2개 이상일 수도 있으므로, 주문건수와 총수량은 서로 다를 수 있습니다.

검증값:

```text
29,535
```

### 평균주문금액

```DAX
평균주문금액 =
DIVIDE([총매출], [주문건수])
```

평균주문금액은 주문 한 건당 매출이 평균적으로 얼마인지 보는 값입니다. 총매출을 주문건수로 나누면 됩니다. DAX에서는 나눗셈을 할 때 `/`를 바로 쓰기보다 `DIVIDE`를 쓰는 편이 안전합니다. 분모가 0이거나 비어 있을 때 오류를 줄여 주기 때문입니다.

검증값:

```text
96,924.49
```

### 매출비중

```DAX
매출비중 =
DIVIDE(
    [총매출],
    CALCULATE([총매출], ALL(products[category]))
)
```

매출비중은 “전체 매출 중 이 카테고리가 차지하는 몫”입니다. 예를 들어 디지털 매출비중은 디지털 매출을 전체 카테고리 매출로 나눈 값입니다.

`ALL(products[category])`는 현재 선택된 카테고리 필터를 잠시 풀어 전체 카테고리 매출을 보게 해 줍니다. 그래서 분자는 현재 카테고리 매출, 분모는 전체 카테고리 매출이 됩니다.

카테고리별 주요 검증값:

| 카테고리 | 매출 | 매출비중 |
|---|---:|---:|
| 디지털 | 673,862,749 | 34.76% |
| 패션 | 281,835,909 | 14.54% |
| 스포츠 | 217,531,145 | 11.22% |
| 뷰티 | 179,845,362 | 9.28% |
| 생활용품 | 178,042,374 | 9.18% |

디지털 카테고리는 673,862,749원으로 가장 큽니다. 매출비중 34.76%는 전체 매출 중 약 3분의 1이 디지털에서 나왔다는 뜻입니다.

### 전월매출

```DAX
전월매출 =
CALCULATE(
    [총매출],
    SAMEPERIODLASTMONTH('날짜테이블'[Date])
)
```

전월매출은 현재 보고 있는 달의 바로 전 달 매출입니다. 예를 들어 화면에서 2024-12를 보고 있다면 전월매출은 2024-11 매출입니다. `SAMEPERIODLASTMONTH`가 날짜테이블을 기준으로 한 달 전 기간을 찾아 줍니다.

### 전월대비 증감액

```DAX
전월대비증감액 =
[총매출] - [전월매출]
```

전월대비 증감액은 현재 달 매출에서 전월매출을 뺀 값입니다. 양수이면 전월보다 늘어난 것이고, 음수이면 전월보다 줄어든 것입니다.

### 전월대비 성장률

```DAX
전월대비성장률 =
DIVIDE([전월대비증감액], [전월매출])
```

전월대비 성장률은 전월매출에 비해 얼마나 늘거나 줄었는지를 비율로 보여줍니다. 1.14%라면 전월보다 1.14% 늘었다는 뜻이고, -16.26%라면 전월보다 16.26% 줄었다는 뜻입니다.

## 월별 KPI 정답

월별 표는 대시보드가 제대로 계산되는지 확인하는 기준표입니다. 카드 하나만 보면 전체 숫자만 보이지만, 월별 표를 보면 어느 달에 올라가고 내려갔는지 흐름이 보입니다.

| 연월 | 주문건수 | 판매수량 | 총매출 | 전월매출 | 전월대비 성장률 |
|---|---:|---:|---:|---:|---:|

| 2024-02 | 1,317 | 1,983 | 128,361,174 | 131,088,708 | -2.08% |
| 2024-03 | 1,504 | 2,234 | 135,686,515 | 128,361,174 | 5.71% |
| 2024-04 | 1,580 | 2,281 | 152,837,005 | 135,686,515 | 12.64% |
| 2024-05 | 1,631 | 2,411 | 164,426,614 | 152,837,005 | 7.58% |
| 2024-06 | 1,526 | 2,284 | 137,684,249 | 164,426,614 | -16.26% |
| 2024-07 | 1,682 | 2,465 | 168,414,635 | 137,684,249 | 22.32% |
| 2024-08 | 1,687 | 2,492 | 162,095,281 | 168,414,635 | -3.75% |

| 2024-10 | 1,720 | 2,508 | 158,810,201 | 150,484,525 | 5.53% |

| 2024-12 | 2,100 | 3,166 | 225,576,457 | 223,024,439 | 1.14% |

해석:

2024년 전체 매출은 1,938,489,803원입니다. 2024년 12월 매출은 225,576,457원으로 월별 매출이 가장 큽니다. 다만 전월대비 성장률이 가장 높은 달은 2024년 12월이 아니라 2024년 11월입니다.

여기서 뭐가 보이시나요? 2024년 11월은 10월 대비 40.43% 증가했습니다. 반대로 2024년 6월은 5월 대비 16.26% 감소했습니다. 그래서 “매출 금액이 가장 큰 달”과 “전월보다 가장 크게 성장한 달”은 다르게 읽어야 합니다.

2024년 1월의 전월매출과 전월대비 성장률이 비어 있는 것도 정상입니다. 데이터가 2024년 1월부터 시작하므로, 2024년 1월 앞에 비교할 2023년 12월 값이 없기 때문입니다.

## 등급 슬라이서 검증값

고객 등급 슬라이서는 고객을 VIP, 골드, 실버, 브론즈처럼 등급별로 걸러 보는 필터입니다. 슬라이서에서 등급을 하나씩 선택했을 때 총매출 카드가 아래 값과 맞는지 확인합니다.

| 등급 | 고객수 | 주문건수 | 총매출 |
|---|---:|---:|---:|
| 실버 | 1,030 | 6,373 | 603,583,519 |
| 골드 | 538 | 5,433 | 547,524,985 |
| 브론즈 | 1,125 | 4,437 | 393,948,369 |
| VIP | 210 | 3,757 | 393,432,930 |

예를 들어 `VIP`를 선택했을 때 총매출 카드가 393,432,930으로 바뀌어야 합니다. 이때 월별 선그래프와 카테고리 그래프도 VIP 고객 기준으로 같이 바뀌어야 자연스럽습니다. 슬라이서는 화면 위에 놓는 리모컨처럼, 여러 시각화를 한 번에 같은 조건으로 바꿉니다.

## 미션 대시보드 구성 정답

권장 배치:

1. 상단: 총매출, 주문건수, 평균주문금액, 전월대비 성장률 카드
2. 왼쪽: 고객 등급 슬라이서, 월 슬라이서
3. 가운데: 월별 총매출 선그래프
4. 오른쪽: 카테고리별 매출 막대그래프
5. 하단: 카테고리별 매출비중 표와 해석 문장

상단에는 한눈에 봐야 하는 숫자 카드를 둡니다. 왼쪽에는 조건을 바꾸는 슬라이서를 두면, 사용자가 먼저 필터를 고르고 나서 화면을 읽기 쉽습니다. 가운데의 월별 총매출 선그래프는 시간 흐름을 보는 자리이고, 오른쪽의 카테고리별 매출 막대그래프는 어떤 상품 묶음이 큰지 비교하는 자리입니다.

필수 동작:

- 월 슬라이서에서 `2024-12`를 선택하면 전월대비 성장률 카드가 1.14%를 표시한다.
- 고객 등급 슬라이서에서 `VIP`를 선택하면 총매출 카드가 393,432,930을 표시한다.
- 카테고리 막대그래프에서 `디지털`을 클릭하면 다른 시각화가 디지털 기준으로 필터링된다.
- 카테고리 막대그래프는 `products[category]`에서 `products[product_name]`으로 드릴다운된다.

드릴다운은 큰 묶음에서 작은 항목으로 내려가는 기능입니다. 처음에는 카테고리별 매출을 보고, 디지털을 누르면 디지털 안의 상품별 매출로 내려갑니다. 엑셀 피벗테이블에서 대분류 아래 세부 항목을 펼쳐 보는 느낌으로 이해하면 됩니다.
