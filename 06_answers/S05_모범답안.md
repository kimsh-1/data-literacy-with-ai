# S05 모범답안: 평균 왜곡 사례 찾기

## 미션 이해하기

수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일의 `sales` 시트에서 상품별 주문금액 `amount`를 비교합니다. 여기서 찾고 싶은 것은 평균이 중앙값보다 많이 높은 상품입니다. 평균만 보면 “보통 주문금액”을 실제보다 크게 볼 수 있기 때문입니다.

## 모범답안

평균 왜곡 사례로 볼 수 있는 상품은 `치즈케이크`입니다.

| 항목 | 값 |
|---|---:|
| 주문 건수 | 633 |
| 평균 주문금액 | 10,740.92 |
| 중앙값 | 6,500 |
| 최빈값 | 6,500 |
| 최솟값 | 6,500 |
| 최댓값 | 19,500 |
| 평균 - 중앙값 | 4,240.92 |
| 평균 / 중앙값 | 1.65 |

답안 문장은 아래처럼 쓰면 됩니다.

```text
치즈케이크의 평균 주문금액은 10,740.92원이지만 중앙값과 최빈값은 6,500원이다. 주문 633건 중 331건은 1개 주문이라 6,500원이고, 2개 주문 191건과 3개 주문 111건이 평균을 끌어올린다. 따라서 "보통 치즈케이크 주문은 얼마인가"를 말할 때 평균만 사용하면 실제로 가장 흔한 6,500원 주문을 과장해 볼 수 있다. 매출 계획에는 평균이 유용하지만, 보통 주문금액 설명에는 중앙값과 최빈값을 함께 보는 것이 좋다.
```

이 답안에서 봐야 할 부분은 평균과 중앙값의 차이입니다. 평균 주문금액은 10,740.92원이지만, 중앙값과 최빈값은 6,500원입니다. 중앙값은 주문금액을 작은 값부터 큰 값까지 줄 세웠을 때 가운데 있는 값이고, 최빈값은 가장 자주 나온 값입니다. 둘 다 6,500원이라는 말은 “가장 보통에 가까운 치즈케이크 주문”은 6,500원 쪽에 가깝다는 뜻입니다.

그런데 평균은 모든 주문금액을 더한 뒤 주문 건수로 나눕니다. 그래서 13,000원 주문과 19,500원 주문도 평균 계산에 그대로 들어갑니다. 마치 반 평균 키를 구할 때 키가 아주 큰 학생 몇 명이 평균을 올리는 것처럼, 2개 주문과 3개 주문이 치즈케이크 평균 주문금액을 위로 끌어올린 것입니다.

## 상위 평균 왜곡 후보

상품별 `평균 - 중앙값`을 내림차순으로 정렬하면 평균이 보통 주문금액보다 높게 보일 가능성이 큰 상품부터 볼 수 있습니다. 차이가 클수록 평균만 보고 말할 때 조심해야 합니다.

| 상품 | 주문 건수 | 평균 | 중앙값 | 평균 - 중앙값 |
|---|---:|---:|---:|---:|
| 치즈케이크 | 633 | 10,740.92 | 6,500 | 4,240.92 |
| 레몬에이드 | 644 | 9,940.99 | 6,000 | 3,940.99 |
| 크로플 | 609 | 9,112.48 | 5,500 | 3,612.48 |
| 바닐라라떼 | 1,151 | 9,098.18 | 5,500 | 3,598.18 |
| 복숭아아이스티 | 652 | 9,051.38 | 5,500 | 3,551.38 |

여기서 `치즈케이크`가 가장 위에 있습니다. 평균과 중앙값의 차이가 4,240.92로 가장 크기 때문입니다. 이 표만 보면 “치즈케이크의 평균이 높다”에서 멈추지 말고, 왜 높아졌는지 수량 분포까지 확인해야 합니다.

## 치즈케이크 수량 분포

| 주문 수량 | 주문 건수 | 주문금액 |
|---:|---:|---:|
| 1 | 331 | 6,500 |
| 2 | 191 | 13,000 |
| 3 | 111 | 19,500 |

여기서 뭐가 보이시나요? 치즈케이크는 1개 주문이 331건입니다. 전체 633건 중 절반 이상이 1개 주문이라서 중앙값과 최빈값이 6,500원으로 나옵니다.

하지만 2개 주문 191건과 3개 주문 111건도 있습니다. 두 수량을 합치면 302건입니다. 이 주문들은 13,000원과 19,500원으로 계산되기 때문에 평균을 10,740.92원까지 올립니다. 그러니 이 사례는 데이터가 틀린 것이 아닙니다. 평균이라는 계산 방식이 큰 주문금액을 함께 반영하면서 생긴 해석상의 주의점입니다.

## 엑셀 풀이 요약

`item_amount_summary` 시트에서 상품별 요약표를 만듭니다. 이 시트는 상품마다 주문금액의 평균, 중앙값, 최빈값을 나란히 놓고 비교하기 위한 계산표입니다.

상품 목록을 먼저 만듭니다. 같은 상품명이 여러 번 나오기 때문에, 상품명을 한 번씩만 뽑아 정렬합니다.

```excel
=SORT(UNIQUE(sales!$G$2:$G$10425))
```

주문 건수는 해당 상품이 몇 번 주문되었는지 세는 값입니다.

```excel
=COUNTIF(sales!$G$2:$G$10425,A2)
```

평균은 해당 상품의 주문금액을 모두 더한 뒤 주문 건수로 나눈 값입니다.

```excel
=AVERAGEIF(sales!$G$2:$G$10425,A2,sales!$J$2:$J$10425)
```

중앙값은 해당 상품의 주문금액만 골라낸 뒤, 작은 값부터 큰 값까지 줄 세웠을 때 가운데 있는 값입니다.

```excel
=MEDIAN(FILTER(sales!$J$2:$J$10425,sales!$G$2:$G$10425=A2))
```

최빈값은 해당 상품의 주문금액 중 가장 자주 나온 값입니다. “가장 흔한 주문금액”을 볼 때 도움이 됩니다.

```excel
=MODE.SNGL(FILTER(sales!$J$2:$J$10425,sales!$G$2:$G$10425=A2))
```

평균과 중앙값의 차이를 계산합니다. 이 값이 클수록 평균이 보통 주문금액보다 높게 보일 가능성이 큽니다.

```excel
=C2-D2
```

`평균 - 중앙값` 열을 내림차순으로 정렬하면 `치즈케이크`가 1위입니다. 이때 바로 결론을 내리지 말고, `qty` 분포까지 확인해 보세요. 평균이 올라간 이유가 주문 수량 때문인지 확인해야 답안 해석이 자연스럽습니다.

## 강사용 검증 코드

아래 코드는 엑셀 풀이와 같은 계산을 pandas로 다시 검산한 코드입니다. pandas는 표 데이터를 다루는 파이썬 도구입니다. 이 차시는 엑셀 차시이므로 수강생 실습용 코드는 위의 엑셀 공식이며, 별도 `S05_code.py` 산출물은 만들지 않습니다.

```python
from pathlib import Path

import pandas as pd


ROOT = Path("/mnt/d/ai-data-analysis-course")
HR_PATH = ROOT / "02_datasets/hr_employees/hr_employees_clean.csv"
CAFE_PATH = ROOT / "02_datasets/cafe_sales/cafe_sales_clean.csv"

hr = pd.read_csv(HR_PATH)
cafe = pd.read_csv(CAFE_PATH)


def first_mode(values: pd.Series):
    modes = values.mode()
    return modes.iloc[0] if not modes.empty else pd.NA


def outlier_count(values: pd.Series) -> int:
    q1 = values.quantile(0.25)
    q3 = values.quantile(0.75)
    iqr = q3 - q1
    lower = q1 - 1.5 * iqr
    upper = q3 + 1.5 * iqr
    return int(((values < lower) | (values > upper)).sum())


dept_summary = (
    hr.groupby("dept")
    .agg(
        count=("salary", "count"),
        mean=("salary", "mean"),
        median=("salary", "median"),
        mode=("salary", first_mode),
        min=("salary", "min"),
        max=("salary", "max"),
        std_dev=("salary", "std"),
        q1=("salary", lambda s: s.quantile(0.25)),
        q3=("salary", lambda s: s.quantile(0.75)),
    )
    .assign(
        range=lambda d: d["max"] - d["min"],
        iqr=lambda d: d["q3"] - d["q1"],
    )
)
dept_summary["outlier_count"] = hr.groupby("dept")["salary"].apply(outlier_count)
dept_summary = dept_summary.sort_values("mean", ascending=False)

item_summary = (
    cafe.groupby("item")
    .agg(
        count=("order_id", "count"),
        unit_price=("unit_price", "first"),
        mean=("amount", "mean"),
        median=("amount", "median"),
        mode=("amount", first_mode),
        min=("amount", "min"),
        max=("amount", "max"),
        std_dev=("amount", "std"),
    )
    .assign(
        range=lambda d: d["max"] - d["min"],
        mean_minus_median=lambda d: d["mean"] - d["median"],
        mean_to_median_ratio=lambda d: d["mean"] / d["median"],
    )
    .sort_values("mean_minus_median", ascending=False)
)

cheesecake_qty = (
    cafe.loc[cafe["item"] == "치즈케이크"]
    .groupby("qty")
    .agg(count=("order_id", "count"), amount=("amount", "first"))
)

print(f"pandas version: {pd.__version__}")
print(f"HR rows: {len(hr):,}, columns: {hr.shape[1]}")
print(f"Cafe rows: {len(cafe):,}, columns: {cafe.shape[1]}")
print("\nDepartment salary summary")
print(
    dept_summary[
        ["count", "mean", "median", "mode", "min", "max", "range", "std_dev", "outlier_count"]
    ].to_string(float_format=lambda x: f"{x:.2f}")
)
print("\nTop mean distortion candidates")
print(
    item_summary[
        ["count", "mean", "median", "mode", "mean_minus_median", "mean_to_median_ratio"]
    ]
    .head(5)
    .to_string(float_format=lambda x: f"{x:.2f}")
)
print("\nCheesecake quantity distribution")
print(cheesecake_qty.to_string())
```
