## 사용한 데이터와 검증 결과 읽기

검증은 `pandas 3.0.3`으로 실제 카페 매출 파일을 열어서 확인했습니다. 여기서 행은 엑셀에서 가로로 한 줄씩 내려가는 기록이고, 열은 세로로 이어지는 항목입니다. 주문 한 건이 한 행이라면, 행 수는 주문 기록이 몇 건인지에 가깝게 볼 수 있습니다.

CSV는 엑셀로도 열리는 표 파일입니다. 초보자라면 수업 홈의 데이터 받기 페이지에서 내려받은 엑셀 파일을 사용해도 괜찮습니다. 같은 내용의 데이터를 표 파일과 엑셀 파일 두 방식으로 확인했다고 보면 됩니다.

| 파일 | 행 | 열 | 시트 또는 파일 |
|---|---:|---:|---|
| `cafe_sales_clean.csv` | 10,424 | 12 | CSV |
| `cafe_sales_dirty.csv` | 10,439 | 12 | CSV |
| `cafe_sales_clean.xlsx` | 10,424 | 12 | `sales` |
| `cafe_sales_dirty.xlsx` | 10,439 | 12 | `sales` |

clean 데이터는 분석하기 좋게 정리된 기준 데이터입니다. dirty 데이터는 실무에서 자주 만나는 문제를 일부러 섞어 둔 연습용 데이터입니다. 두 데이터 모두 열의 구성은 같습니다. 즉, 같은 항목을 가지고 있는데 dirty 쪽에는 표기, 결측, 중복 같은 문제가 들어 있다고 보면 됩니다.

```text
order_id, date, weekday, hour, store, category, item, unit_price, qty, amount, payment, member
```

## 미션 1. 컬럼별 척도 분류 답안

척도는 “이 값으로 어떤 분석을 할 수 있는가”를 정하는 기준입니다. 숫자처럼 보여도 전부 더하고 평균 낼 수 있는 값은 아닙니다. 예를 들어 주문번호는 숫자처럼 생겼지만 평균 주문번호를 구해도 의미가 없죠. 사물함 번호처럼 “구분하기 위한 이름표”로 보는 편이 맞습니다.

아래 표는 각 컬럼을 어떤 척도로 보아야 하는지, 그리고 그 척도에서 어떤 분석이 가능한지 정리한 답안입니다.

| 컬럼 | 척도 | 가능한 분석 | 주의할 점 |
|---|---|---|---|
| `order_id` | 식별자 | 중복 확인, 주문 추적 | 숫자로 보여도 평균이나 합계는 의미 없음 |
| `date` | 날짜형 | 일별, 월별, 요일별 추세 | 날짜 형식이 섞이면 먼저 통일 필요 |
| `weekday` | 순서가 있는 범주형 | 요일별 매출, 주중·주말 비교 | 평균 요일 계산은 의미 없음 |
| `hour` | 시간형 수치 | 시간대별 주문 수, 피크 시간 확인 | 8~21시 범위의 시간대로 해석 |
| `store` | 범주형 | 지점별 매출, 지점별 주문 수 | `강남점`과 `강남 점` 같은 표기 혼재 주의 |
| `category` | 범주형 | 카테고리별 매출 | 앞뒤 공백이 있으면 다른 범주로 집계될 수 있음 |
| `item` | 범주형 | 상품별 매출, 인기 상품 | 같은 상품의 표기 통일 필요 |
| `unit_price` | 수치형 | 단가 평균, 가격대 확인 | `4,500원`처럼 문자 포함 시 숫자 변환 필요 |
| `qty` | 수치형 | 판매 수량 합계, 평균 주문 수량 | 999 같은 이상치 확인 필요 |
| `amount` | 수치형 | 총매출, 평균 주문금액 | 음수 금액 또는 단가×수량 불일치 확인 필요 |
| `payment` | 범주형 | 결제수단별 비중 | 결측값 처리 기준 필요 |
| `member` | 논리형 | 회원 여부별 매출, 회원 비율 | 결측값 처리 기준 필요 |

여기서 범주형은 이름표처럼 그룹을 나누는 값입니다. 지점명, 상품명, 결제수단처럼 “어느 그룹인가”를 보는 값이죠. 수치형은 키를 재듯이 더하고, 평균 내고, 크고 작음을 비교할 수 있는 값입니다. 날짜형은 시간의 흐름을 보는 값이고, 논리형은 예/아니오처럼 둘 중 하나로 나누는 값입니다.

## 미션 2. dirty 데이터 문제와 교정 방향 답안

dirty 데이터는 그냥 틀린 데이터라고만 보면 안 됩니다. 실무에서는 이런 문제가 섞인 표를 자주 받습니다. 중요한 것은 “어디가 문제인지 찾고, 어떤 기준으로 고칠지 정하는 것”입니다. 마치 주소록에서 같은 사람 이름이 다르게 적힌 것을 찾아 하나로 맞추는 작업과 비슷합니다.

아래 표에서는 발견한 문제, 확인된 개수, 교정 방향을 함께 봅니다. 개수는 문제의 크기를 판단하게 해 줍니다. 1건인지, 313행인지에 따라 처리의 우선순위와 조심해야 할 정도가 달라지기 때문입니다.

| 컬럼 | 발견한 문제 | 확인한 개수 | 교정 방향 |
|---|---|---:|---|
| `order_id` | 같은 주문번호가 반복됨 | 중복 주문번호 15건 | 실제 중복이면 한 건만 남기고, 재주문이면 별도 주문번호 부여 |
| 전체 행 | 완전히 동일한 행이 있음 | 13행 | 중복 제거 전후 행수와 매출 합계 비교 |
| `payment` | 결제수단 결측 | 313행 | 원자료 확인, 불명 처리, 분석 제외 중 선택 |
| `member` | 회원 여부 결측 | 209행 | 원자료 확인, 불명 처리, 분석 제외 중 선택 |
| `date` | 날짜 형식 혼합 | `/` 형식 206행, `MM-DD-YYYY` 형식 104행 | 하나의 날짜형으로 변환 |
| `unit_price` | 숫자와 문자가 섞임 | `원` 또는 쉼표 포함 157행 | `원`과 쉼표 제거 후 숫자형으로 변환 |
| `store` | `강남점`과 `강남 점` 혼재 | `강남 점` 310행 | `강남점`으로 통일 |
| `category` | 앞뒤 공백 포함 | 313행 | 앞뒤 공백 제거 |
| `item` | 아메리카노 표기 혼재 | `아메리카` 37행, `Americano` 36행, `아메 리카노` 33행 | `아메리카노`로 통일 |
| `qty` | 비현실적 수량 | 999가 2행 | 원자료 확인 후 수정 또는 제외 |
| `amount` | 음수 금액 | 1행 | 환불 데이터인지 확인, 아니면 오류로 수정 |
| `amount` | 단가×수량과 금액 불일치 | 3행 | 단가와 수량 기준으로 재계산하거나 원자료 확인 |

dirty 파일은 clean 파일보다 15행 많습니다. 그런데 여기서 조심할 점이 있습니다. “주문번호가 중복된 주문”과 “모든 컬럼 값이 완전히 같은 행”은 같은 말이 아닙니다. 주문번호만 같고 다른 값이 조금이라도 다르면 완전 동일 행은 아닐 수 있습니다.

이 데이터에서는 dirty 파일이 clean 파일보다 15행 많지만, 일부 중복 행에 다른 dirty 처리가 겹쳐 있습니다. 그래서 전체 컬럼이 완전히 같은 행은 `duplicated()` 기준 13행으로 확인됩니다. 초급 실습에서는 “주문번호가 중복된 주문 15건”과 “완전히 같은 행 13건”을 나누어 말하면 됩니다.

## tidy하게 고치는 방향 예시

tidy 데이터는 분석하기 좋게 가지런한 표입니다. 한 셀에는 한 값, 한 열에는 한 변수, 한 행에는 한 관찰이 들어가야 합니다. 이 원칙이 지켜지면 피벗테이블도 안정적으로 돌아가고, 파이썬이나 BI 도구에서도 집계가 덜 흔들립니다.

| 원칙 | dirty에서 깨진 예 | tidy하게 만드는 방향 |
|---|---|---|
| 한 셀 한 값 | `unit_price`의 `4,500원` | 값은 `4500`, 단위 표시는 셀 서식으로 처리 |
| 한 열 한 변수 | 날짜 열에 여러 형식 혼합 | 하나의 날짜형으로 통일 |
| 같은 범주는 같은 표기 | `강남점`과 `강남 점` | 표준명을 정하고 일괄 치환 |
| 계산 가능한 숫자는 숫자로 저장 | `unit_price`가 문자처럼 읽힘 | 숫자형 변환 후 합계·평균 계산 |
| 분석 단위 유지 | 같은 `order_id` 반복 | 중복 여부를 확인하고 한 주문은 한 행으로 정리 |

예를 들어 `4,500원`은 사람 눈에는 읽기 편합니다. 하지만 컴퓨터 입장에서는 숫자 4500에 글자 `원`이 붙어 있는 값입니다. 계산하려면 먼저 숫자만 남겨야 합니다. 금액 단위는 값 안에 넣기보다 엑셀의 셀 서식처럼 표시 방식으로 처리하는 편이 좋습니다.

`강남점`과 `강남 점`도 마찬가지입니다. 사람은 같은 지점이라고 금방 알아보지만, 컴퓨터는 띄어쓰기가 다르면 다른 값으로 볼 수 있습니다. 그래서 분석 전에 표준 이름을 정하고 같은 값으로 맞춰 주어야 합니다.

## 검증 코드

아래 코드는 이 모범답안의 숫자가 실제 데이터와 맞는지 확인하는 파이썬 코드입니다. 수업을 처음 듣는 단계에서는 코드를 모두 이해하지 않아도 됩니다. 지금은 “데이터를 열고, 행과 열의 개수를 확인하고, 결측·중복·표기 문제의 개수를 세는 코드”라고 보면 됩니다.

코드블록 안의 내용은 검증용 원본 그대로입니다.

```python
from pathlib import Path

import pandas as pd


ROOT = Path(__file__).resolve().parents[1]
DATA_DIR = ROOT / "02_datasets" / "cafe_sales"

CLEAN_CSV = DATA_DIR / "cafe_sales_clean.csv"
DIRTY_CSV = DATA_DIR / "cafe_sales_dirty.csv"
CLEAN_XLSX = DATA_DIR / "cafe_sales_clean.xlsx"
DIRTY_XLSX = DATA_DIR / "cafe_sales_dirty.xlsx"

EXPECTED_COLUMNS = [
    "order_id",
    "date",
    "weekday",
    "hour",
    "store",
    "category",
    "item",
    "unit_price",
    "qty",
    "amount",
    "payment",
    "member",
]


def main() -> None:
    clean = pd.read_csv(CLEAN_CSV)
    dirty = pd.read_csv(DIRTY_CSV)
    clean_xlsx = pd.read_excel(CLEAN_XLSX, sheet_name="sales")
    dirty_xlsx = pd.read_excel(DIRTY_XLSX, sheet_name="sales")

    print("파일 존재 확인")
    for path in [CLEAN_CSV, DIRTY_CSV, CLEAN_XLSX, DIRTY_XLSX]:
        print(f"- {path}: {path.exists()}")

    print("\n행수와 열수")
    print(f"clean csv: {clean.shape}")
    print(f"dirty csv: {dirty.shape}")
    print(f"clean xlsx: {clean_xlsx.shape}")
    print(f"dirty xlsx: {dirty_xlsx.shape}")

    print("\n컬럼")
    print(list(clean.columns))

    unit_price_text = dirty["unit_price"].astype(str).str.contains("원|,", regex=True, na=False)
    slash_dates = dirty["date"].astype(str).str.contains("/", regex=False, na=False)
    mdy_dash_dates = dirty["date"].astype(str).str.match(r"\d{2}-\d{2}-\d{4}", na=False)
    category_spaces = dirty["category"].astype(str) != dirty["category"].astype(str).str.strip()
    qty_numeric = pd.to_numeric(dirty["qty"], errors="coerce")
    amount_numeric = pd.to_numeric(dirty["amount"], errors="coerce")
    unit_price_numeric = pd.to_numeric(
        dirty["unit_price"].astype(str).str.replace(",", "", regex=False).str.replace("원", "", regex=False),
        errors="coerce",
    )

    issue_summary = pd.DataFrame(
        [
            ["원본 대비 증가 행", len(dirty) - len(clean)],
            ["주문번호 중복", dirty["order_id"].duplicated().sum()],
            ["완전 동일 중복 행", dirty.duplicated().sum()],
            ["payment 결측", dirty["payment"].isna().sum()],
            ["member 결측", dirty["member"].isna().sum()],
            ["날짜 YYYY/MM/DD 형식", slash_dates.sum()],
            ["날짜 MM-DD-YYYY 형식", mdy_dash_dates.sum()],
            ["unit_price 문자 포함", unit_price_text.sum()],
            ["store='강남 점'", (dirty["store"] == "강남 점").sum()],
            ["category 앞뒤 공백", category_spaces.sum()],
            ["item='아메리카'", (dirty["item"] == "아메리카").sum()],
            ["item='Americano'", (dirty["item"] == "Americano").sum()],
            ["item='아메 리카노'", (dirty["item"] == "아메 리카노").sum()],
            ["qty=999", (qty_numeric == 999).sum()],
            ["amount 음수", (amount_numeric < 0).sum()],
            ["단가×수량과 금액 불일치", ((unit_price_numeric * qty_numeric) != amount_numeric).sum()],
        ],
        columns=["항목", "확인값"],
    )

    scale_table = pd.DataFrame(
        [
            ["order_id", "식별자", "중복 확인, 주문 추적"],
            ["date", "날짜형", "일별, 월별, 요일별 추세"],
            ["weekday", "순서가 있는 범주형", "요일별 집계, 주중·주말 비교"],
            ["hour", "시간형 수치", "시간대별 주문 수"],
            ["store", "범주형", "지점별 비교"],
            ["category", "범주형", "상품 대분류별 비교"],
            ["item", "범주형", "상품별 비교"],
            ["unit_price", "수치형", "단가 계산"],
            ["qty", "수치형", "수량 합계와 평균"],
            ["amount", "수치형", "총매출과 평균 주문금액"],
            ["payment", "범주형", "결제수단별 비중"],
            ["member", "논리형", "회원 여부별 비교"],
        ],
        columns=["컬럼", "척도", "가능한 분석"],
    )

    print("\ndirty 문제 요약")
    print(issue_summary.to_string(index=False))

    print("\n척도 분류표")
    print(scale_table.to_string(index=False))

    assert clean.shape == (10424, 12)
    assert dirty.shape == (10439, 12)
    assert clean_xlsx.shape == (10424, 12)
    assert dirty_xlsx.shape == (10439, 12)
    assert list(clean.columns) == EXPECTED_COLUMNS
    assert list(dirty.columns) == EXPECTED_COLUMNS
    assert len(dirty) - len(clean) == 15
    assert dirty["order_id"].duplicated().sum() == 15
    assert dirty.duplicated().sum() == 13
    assert dirty["payment"].isna().sum() == 313
    assert dirty["member"].isna().sum() == 209
    assert slash_dates.sum() == 206
    assert mdy_dash_dates.sum() == 104
    assert unit_price_text.sum() == 157
    assert (dirty["store"] == "강남 점").sum() == 310
    assert category_spaces.sum() == 313
    assert (dirty["item"] == "아메리카").sum() == 37
    assert (dirty["item"] == "Americano").sum() == 36
    assert (dirty["item"] == "아메 리카노").sum() == 33
    assert (qty_numeric == 999).sum() == 2
    assert (amount_numeric < 0).sum() == 1
    assert ((unit_price_numeric * qty_numeric) != amount_numeric).sum() == 3

    print("\n검증 통과")


if __name__ == "__main__":
    main()
```
