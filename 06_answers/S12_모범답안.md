# S12 모범답안

## 미션 답안은 이렇게 씁니다

모범답안은 “데이터를 청소했습니다”에서 끝나면 부족합니다. 몇 행이 줄었는지, 왜 줄었는지, 그 결과 계산이 얼마나 믿을 만해졌는지를 숫자로 말해야 합니다. 키를 재듯이, 청소 전과 청소 후를 같은 기준으로 재야 변화가 보입니다.

작성 문장 예시:

```text
정제 전에는 10,439행이고, 정제 후에는 10,421행이다.
주요 제거 사유는 중복 order_id 15행, qty=999 수량 이상치 2행, amount 음수 1행이다.
판매수량 합계가 19,259개에서 17,238개로 크게 달라진 이유는 qty=999 두 행이 원본 합계를 부풀렸기 때문이다.
정제 후에는 중복 order_id가 0개이고, 전체 결측값도 0개이며, 단가 x 수량과 금액 불일치도 0건이다.
```

좋은 답안의 조건:

- 결측값을 무조건 삭제하지 않고, `payment`와 `member`를 기준에 따라 채웠다.
- `unit_price`, `qty`, `amount`, `date`를 계산 가능한 타입으로 변환했다.
- `order_id` 기준 중복을 제거했다.
- IQR 기준으로 `qty=999` 2행을 이상치로 확인했다.
- 음수 금액 1행을 제거했다.
- 정제 전후 행수와 통계 변화를 숫자로 설명했다.
- cp949 원본 파일을 `encoding="cp949"`로 읽고 날짜를 datetime으로 변환했다.

여기서 중요한 점은 “삭제를 많이 했다”가 아니라 “분석에 쓸 수 있는 표로 만들었다”는 점입니다. `payment`처럼 결제수단을 모르는 값은 주문 자체를 버릴 이유가 없으니 `미확인`으로 채웁니다. 반대로 `qty=999`처럼 카페 주문 수량으로 보기 어려운 값은 합계를 크게 흔들기 때문에 따로 확인하고 제거합니다.

## 실제로 확인한 데이터와 실행 환경

수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 CSV를 확인했습니다. CSV는 엑셀로도 열리는 표 파일입니다. 초보자는 같은 데이터의 엑셀 파일(.xlsx)을 내려받아 먼저 눈으로 열어 봐도 됩니다. 다만 아래 검증 코드는 CSV를 기준으로 실행했습니다.

검증 파일:

- 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 dirty 데이터
- 수업 홈의 데이터 받기 페이지에서 내려받은 cp949 배달 음식 원본 데이터

실제 실행 환경:

- `python3`
- pandas `3.0.3`

확인 결과:

| 파일 | 결과 |
|---|---:|
| `cafe_sales_dirty.csv` | 10,439행 x 12열 |
| 고유 `order_id` | 10,424개 |
| cp949 원본 파일 | 1,947행 x 15열 |
| cp949 원본 날짜 변환 실패 | 0건 |

행과 열은 표의 크기를 말합니다. 10,439행 x 12열이면 주문 기록이 10,439줄 있고, 각 주문에 대해 날짜, 지점, 메뉴, 금액 같은 정보가 12가지 들어 있다는 뜻입니다. 고유 `order_id`가 10,424개인데 전체 행수는 10,439행이므로, 같은 주문번호가 반복된 행이 있다는 것도 바로 의심할 수 있습니다.

## dirty 데이터에서 보이는 문제

dirty 데이터는 더럽다는 뜻이 아니라, 분석 전에 손봐야 할 문제가 섞여 있는 데이터라고 보면 됩니다. 엑셀에서 빈칸, 이상한 숫자, 같은 주문번호, 띄어쓰기 다른 메뉴명을 찾는 작업과 같습니다.

| 점검 항목 | 확인값 |
|---|---:|
| 전체 행수 | 10,439 |
| 고유 `order_id` | 10,424 |
| 중복 `order_id` 행 | 15 |
| 완전 동일 중복 행 | 13 |
| `payment` 결측 | 313 |
| `member` 결측 | 209 |
| 날짜 `YYYY/MM/DD` 형식 | 206 |
| 날짜 `MM-DD-YYYY` 형식 | 104 |
| 날짜 변환 실패 | 0 |
| `unit_price` 문자 포함 | 157 |
| `unit_price` 숫자 변환 실패 | 0 |
| `store='강남 점'` | 310 |
| `category` 앞뒤 공백 | 313 |
| 아메리카노 표기 변형 | 106 |
| `qty=999` | 2 |
| `amount` 음수 | 1 |
| 단가 x 수량과 금액 불일치 | 3 |

여기서 뭐가 보이시나요? 빈칸만 문제가 아닙니다. `강남점`과 `강남 점`처럼 사람 눈에는 같은 지점이지만 컴퓨터에는 다른 글자인 값도 있습니다. `unit_price`에 쉼표나 `원`이 섞이면 사람에게는 가격처럼 보여도 pandas에는 문자로 읽힐 수 있습니다. 그래서 계산하기 전에 숫자 타입으로 바꿔야 합니다.

## 정제 기준은 이렇게 잡습니다

정제 기준은 “내 마음대로 고치기”가 아닙니다. 왜 채우는지, 왜 지우는지 기준을 정하고 그 기준대로 처리해야 합니다. 사전에서 단어를 통일하듯이, 같은 뜻의 값은 같은 표기로 맞춰 줍니다.

| 항목 | 처리 |
|---|---|
| `payment` 결측 | `미확인`으로 채움 |
| `member` 결측 | 수업용 기준으로 `False`로 채움 |
| `store` | 앞뒤 공백 제거, 내부 공백 제거 |
| `category` | 앞뒤 공백 제거 |
| `item` | `아메리카`, `Americano`, `아메 리카노`를 `아메리카노`로 통일 |
| `unit_price` | 쉼표와 `원` 제거 후 숫자 변환 |
| `date` | `pd.to_datetime(..., format="mixed")`로 datetime 변환 |
| 중복 | `order_id` 기준으로 15행 제거 |
| 필수값 결측 | `dropna(subset=...)`; 이번 데이터에서는 0행 제거 |
| 수량 이상치 | IQR 정상범위 -0.5~3.5 밖의 2행 제거 |
| 금액 오류 | `amount <= 0` 1행 제거 |

`payment`는 결제수단을 모른다는 뜻이므로 `미확인`으로 남겨도 주문 금액 분석에는 쓸 수 있습니다. `member`는 수업용 기준으로 비회원인 `False`로 채웠습니다. 실제 회사 데이터라면 이 기준은 담당자와 확인해야 합니다.

IQR은 값이 보통 어느 범위 안에 모여 있는지 보는 방법입니다. 이번 수량 데이터에서는 정상범위가 -0.5~3.5로 계산됩니다. 카페 주문 수량이 999개인 행은 이 범위를 아주 크게 벗어나므로 이상치로 보는 것이 자연스럽습니다.

## 청소 전후 비교로 결과를 확인합니다

정제를 마친 뒤에는 꼭 전후 비교를 해야 합니다. 청소를 했는데 행수가 얼마나 줄었는지, 총매출이 이상하게 바뀌지는 않았는지, 최대 수량과 최소 금액이 정상 범위로 돌아왔는지 확인해야 합니다.

| 비교 항목 | 청소 전 | 청소 후 |
|---|---:|---:|
| 행수 | 10,439 | 10,421 |
| 고유 `order_id` | 10,424 | 10,421 |
| 총매출 | 90,600,500원 | 90,476,000원 |
| 판매수량 합계 | 19,259 | 17,238 |
| 건당 평균금액 | 8,679원 | 8,682원 |
| 건당 중앙값 | 6,500원 | 6,500원 |
| 최대 수량 | 999 | 3 |
| 최소 금액 | -5,500원 | 3,000원 |

행수는 10,439행에서 10,421행으로 줄었습니다. 줄어든 18행은 중복 `order_id` 15행, `qty=999` 이상치 2행, 음수 금액 1행입니다. 총매출은 90,600,500원에서 90,476,000원으로 크게 흔들리지 않았지만, 판매수량 합계는 19,259개에서 17,238개로 줄었습니다. 이 차이는 `qty=999` 두 행이 원래 수량 합계를 크게 부풀렸기 때문입니다.

해석:

```text
정제 후 행수는 18행 줄었다. 중복 order_id 15행, qty=999 이상치 2행, amount 음수 1행을 제거했기 때문이다. 총매출은 크게 달라지지 않았지만 판매수량 합계는 2,021개 줄었다. 이는 qty=999 두 행이 수량 합계를 크게 부풀렸기 때문이다. 정제 후에는 중복 주문번호, 전체 결측값, 단가 x 수량과 금액 불일치가 모두 0이므로 카테고리별 매출 집계에 사용할 수 있다.
```

정제 후 카테고리별 결과:

| category | 행수 | 총매출 | 판매수량 |
|---|---:|---:|---:|
| 디저트 | 2,483 | 19,331,000 | 4,054 |
| 음료 | 2,538 | 25,544,500 | 4,254 |
| 커피 | 5,400 | 45,600,500 | 8,930 |

이 표는 이제 카테고리별 매출을 비교하는 데 쓸 수 있습니다. 중복 주문번호가 남아 있거나, 금액이 음수로 들어 있거나, 단가와 수량의 곱이 금액과 맞지 않으면 이런 집계표를 믿기 어렵습니다. 정제 후에는 그런 문제를 0건으로 맞췄기 때문에 카테고리별 매출 비교로 넘어갈 수 있습니다.

## cp949 인코딩 실습 결과

인코딩은 글자를 저장하는 약속입니다. 같은 한글 파일이라도 UTF-8 약속으로 저장된 파일이 있고, cp949 약속으로 저장된 파일이 있습니다. 약속이 맞지 않으면 한글이 깨지거나 아예 파일이 열리지 않습니다.

기본값으로 읽기:

```python
pd.read_csv("02_datasets/real_korean/delivery_food/food_delivery_prediction_raw_data2_ORIGINAL_cp949.csv")
```

결과:

```text
UnicodeDecodeError
```

`UnicodeDecodeError`는 파일 안의 글자를 현재 읽는 방식으로 해석할 수 없다는 뜻입니다. 이럴 때는 데이터가 저장된 인코딩을 지정해서 다시 읽어야 합니다.

인코딩 지정:

```python
delivery = pd.read_csv(
    "02_datasets/real_korean/delivery_food/food_delivery_prediction_raw_data2_ORIGINAL_cp949.csv",
    encoding="cp949",
)
```

결과:

- 1,947행 x 15열로 정상 로드
- `일자`를 datetime으로 변환한 뒤 실패 0건
- 날짜 범위: 2013-12-01 ~ 2019-03-31

`일자`를 datetime으로 바꾸는 이유도 중요합니다. 날짜가 글자로 남아 있으면 월별 집계나 기간 필터를 하기가 불편합니다. datetime은 pandas가 날짜로 알아보는 타입이라서 월, 연도, 기간 계산을 할 수 있습니다.

## 실행한 검증 코드

아래 코드는 수업용 검증 스크립트로 실행한 내용입니다. 학생 입장에서는 코드 전체를 한 번에 외우기보다, 결측 확인, 타입 변환, 중복 제거, 이상치 제거, 검산 순서로 읽어 보세요.

실행 명령:

```bash
cd /mnt/d/ai-data-analysis-course
python3 06_answers/S12_code.py
```

수업에서 실제로 돌린 주요 실습 코드는 아래입니다.

```python
from pathlib import Path
import pandas as pd

root = Path("/mnt/d/ai-data-analysis-course")
cafe_path = root / "02_datasets/cafe_sales/cafe_sales_dirty.csv"
cp949_path = root / "02_datasets/real_korean/delivery_food/food_delivery_prediction_raw_data2_ORIGINAL_cp949.csv"

df = pd.read_csv(cafe_path, encoding="utf-8-sig")

unit_price_num = pd.to_numeric(
    df["unit_price"].astype("string")
    .str.replace(",", "", regex=False)
    .str.replace("원", "", regex=False),
    errors="coerce",
)
qty_num = pd.to_numeric(df["qty"], errors="coerce")
amount_num = pd.to_numeric(df["amount"], errors="coerce")
date_parsed = pd.to_datetime(df["date"], format="mixed", errors="coerce")

print("원본 행수:", len(df))
print("고유 order_id:", df["order_id"].nunique())
print("중복 order_id 행:", df.duplicated("order_id").sum())
print("payment 결측:", df["payment"].isna().sum())
print("member 결측:", df["member"].isna().sum())
print("날짜 변환 실패:", date_parsed.isna().sum())
print("unit_price 문자 포함:", df["unit_price"].astype("string").str.contains("원|,", regex=True, na=False).sum())
print("qty=999:", (qty_num == 999).sum())
print("amount 음수:", (amount_num < 0).sum())
print("단가 x 수량과 금액 불일치:", ((unit_price_num * qty_num) != amount_num).sum())

cleaned = df.copy()
cleaned["store"] = cleaned["store"].astype("string").str.strip().str.replace(" ", "", regex=False)
cleaned["category"] = cleaned["category"].astype("string").str.strip()
cleaned["item"] = (
    cleaned["item"]
    .astype("string")
    .str.strip()
    .replace({
        "아메리카": "아메리카노",
        "Americano": "아메리카노",
        "아메 리카노": "아메리카노",
    })
)
cleaned["payment"] = cleaned["payment"].astype("string").str.strip().fillna("미확인")
cleaned["member"] = cleaned["member"].astype("boolean").fillna(False)
cleaned["unit_price"] = pd.to_numeric(
    cleaned["unit_price"].astype("string")
    .str.replace(",", "", regex=False)
    .str.replace("원", "", regex=False),
    errors="coerce",
)
cleaned["qty"] = pd.to_numeric(cleaned["qty"], errors="coerce")
cleaned["amount"] = pd.to_numeric(cleaned["amount"], errors="coerce")
cleaned["date"] = pd.to_datetime(cleaned["date"], format="mixed", errors="coerce")

cleaned = cleaned.drop_duplicates(subset=["order_id"], keep="first").copy()
cleaned = cleaned.dropna(
    subset=["date", "store", "category", "item", "unit_price", "qty", "amount"]
).copy()

q1 = cleaned["qty"].quantile(0.25)
q3 = cleaned["qty"].quantile(0.75)
iqr = q3 - q1
lower = q1 - 1.5 * iqr
upper = q3 + 1.5 * iqr

cleaned = cleaned.loc[cleaned["qty"].between(lower, upper)].copy()
cleaned = cleaned.loc[cleaned["amount"] > 0].copy()

for col in ["order_id", "hour", "unit_price", "qty", "amount"]:
    cleaned[col] = cleaned[col].astype("int64")

cleaned["expected_amount"] = cleaned["unit_price"] * cleaned["qty"]
cleaned["date_month"] = cleaned["date"].dt.to_period("M").astype("string")

comparison = pd.DataFrame({
    "비교 항목": ["행수", "고유 order_id", "총매출", "판매수량 합계", "최대 수량", "최소 금액"],
    "청소 전": [
        len(df),
        df["order_id"].nunique(),
        amount_num.sum(),
        qty_num.sum(),
        qty_num.max(),
        amount_num.min(),
    ],
    "청소 후": [
        len(cleaned),
        cleaned["order_id"].nunique(),
        cleaned["amount"].sum(),
        cleaned["qty"].sum(),
        cleaned["qty"].max(),
        cleaned["amount"].min(),
    ],
})

print(comparison)
print("중복 order_id:", cleaned.duplicated("order_id").sum())
print("전체 결측값:", cleaned.isna().sum().sum())
print("단가 x 수량과 금액 불일치:", (cleaned["expected_amount"] != cleaned["amount"]).sum())

try:
    pd.read_csv(cp949_path)
except UnicodeDecodeError as e:
    print(type(e).__name__)

delivery = pd.read_csv(cp949_path, encoding="cp949")
delivery["일자_dt"] = pd.to_datetime(delivery["일자"], errors="coerce")

print(delivery.shape)
print(delivery["일자_dt"].isna().sum())
print(delivery["일자_dt"].min(), delivery["일자_dt"].max())
```

## 실제 실행 출력 요약

아래 출력은 정제가 제대로 되었는지 확인하는 검산표처럼 보면 됩니다. 중복 주문번호, 전체 결측값, 단가 x 수량과 금액 불일치가 모두 0이면 카테고리별 집계에 사용할 준비가 된 상태입니다.

```text

cafe_sales_dirty 원본: 10,439행 x 12열
중복 order_id 제거: 15행
필수값 결측으로 제거: 0행
qty IQR 이상치 제거: 2행
amount 0 이하 제거: 1행
정제 후 규모: 10,421행 x 14열
정제 후 중복 order_id: 0
정제 후 전체 결측값: 0
```
