## 실제로 확인한 카페 매출 데이터

수업에서 숫자를 설명하려면 먼저 데이터가 어떤 모양인지 확인해 두어야 합니다. 여기서는 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일을 기준으로 봅니다. 이 파일은 엑셀로도 열리는 표 파일인 CSV 형식입니다. 엑셀이 더 편한 분은 같은 내용의 엑셀 파일(.xlsx)을 받아도 됩니다.

오리엔테이션에서는 수강생이 코드를 직접 실행하지 않습니다. 파이썬과 Colab은 S10부터 다룹니다. 아래 내용은 강사가 미리 같은 파일을 열어, 표가 제대로 준비되어 있는지 키를 재듯 확인한 값입니다.

- 사용한 데이터: 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일
- 행과 열: 10,424행, 12열
- 날짜 범위: 2025-01-01부터 2025-06-30까지
- 결측값: 모든 열 0개
- 열 이름: `order_id`, `date`, `weekday`, `hour`, `store`, `category`, `item`, `unit_price`, `qty`, `amount`, `payment`, `member`

여기서 행은 주문 한 건이라고 보면 됩니다. 열은 그 주문을 설명하는 항목입니다. 예를 들어 `date`는 주문일, `store`는 지점, `amount`는 주문 금액입니다. 결측값이 0개라는 말은 빈칸으로 빠진 값이 없다는 뜻입니다.

간단히 집계하면 이런 숫자가 나옵니다.

| 항목 | 결과 |
|---|---:|
| 전체 주문 건수 | 10,424 |
| 전체 매출 합계 | 90,492,500 |
| 주문 1건당 평균 금액 | 8,681.2 |

주문 1건당 평균 금액 8,681.2는 주문 한 번에 평균적으로 이 정도 금액이 결제되었다는 뜻입니다. 평균은 여러 사람의 키를 모두 더한 뒤 사람 수로 나누는 것처럼, 전체 매출을 전체 주문 건수로 나누어 봅니다.

지점별 매출 합계:

| 지점 | 매출 합계 |
|---|---:|
| 강남점 | 53,425,000 |
| 홍대점 | 37,067,500 |

이 표에서는 강남점의 매출 합계가 53,425,000이고, 홍대점의 매출 합계가 37,067,500입니다. 지점별로 `amount`를 더했기 때문에 두 지점의 매출 규모를 바로 비교할 수 있습니다.

카테고리별 매출 합계:

| 카테고리 | 매출 합계 |
|---|---:|
| 커피 | 45,606,000 |
| 음료 | 25,550,000 |
| 디저트 | 19,336,500 |

카테고리별로 보면 커피의 매출 합계가 가장 큽니다. 여기서 "많이 팔렸다"를 말할 때는 수량 기준인지, 금액 기준인지 구분해야 합니다. 지금 표는 매출 합계이므로 `amount`를 더한 값입니다.

## 미션 1 모범답안. 카페 매출 파일로 분석 질문 3개 만들기

모범답안은 하나로 고정되어 있지 않습니다. 다만 좋은 분석 질문은 지금 가진 표의 열로 답할 수 있어야 합니다. 사전에서 단어를 찾듯이, 질문 속의 말이 어느 열과 연결되는지 확인해 보세요.

| 번호 | 분석 질문 | 답을 위해 필요한 열 | 판단 |
|---:|---|---|---|
| 1 | 2025년 1월부터 6월까지 지점별 총매출은 어떻게 다른가? | `date`, `store`, `amount` | 가능 |
| 2 | 2025년 1월부터 6월까지 상품 카테고리별 총매출은 어떻게 다른가? | `date`, `category`, `amount` | 가능 |
| 3 | 2025년 1월부터 6월까지 시간대별 매출 상위 시간은 언제인가? | `date`, `hour`, `amount` | 가능 |

위 질문들은 모두 카페 매출 파일에 있는 열로 답할 수 있습니다. 기간은 `date`, 지점은 `store`, 카테고리는 `category`, 시간대는 `hour`, 매출은 `amount`와 연결됩니다. 질문 안에 기간, 기준, 집계할 값이 들어 있다는 점도 좋습니다.

반대로 아래 질문은 현재 표만으로는 답하기 어렵습니다.

| 질문 | 어려운 이유 |
|---|---|
| 고객이 왜 아메리카노를 샀는가? | 구매 이유 열이 없다. |
| 고객이 매장 분위기에 만족했는가? | 만족도나 매장 분위기 평가 열이 없다. |
| 2025년 7월 매출은 반드시 얼마인가? | 현재 파일은 2025년 1월부터 6월까지의 과거 주문이다. 예측에는 추가 가정이 필요하다. |

이 질문들이 틀렸다는 뜻은 아닙니다. 다만 지금 표에는 구매 이유, 만족도, 매장 분위기 평가, 2025년 7월의 실제 매출이 들어 있지 않습니다. 분석을 시작할 때는 "궁금한가?"보다 먼저 "이 표에 그 답을 찾을 열이 있는가?"를 봅니다.

## 미션 2 모범답안. 내 업무 표 1개를 분석 질문으로 바꾸기

내 업무에서도 같은 방식으로 생각하면 됩니다. 평소 자주 보는 표를 하나 고르고, 그 표에 어떤 열이 있을지 떠올린 뒤, 그 열로 답할 수 있는 질문을 만듭니다.

예시 답안:

| 내 업무 표 | 표에 있을 것 같은 열 | 분석 질문 |
|---|---|---|
| 고객 문의 처리표 | `문의일`, `채널`, `문의유형`, `담당자`, `처리시간`, `처리상태` | 이번 달 문의유형별 평균 처리시간은 어떻게 다른가? |

이 질문은 좋은 편입니다. 무엇을 볼지, 어떤 기준으로 나눌지, 어떤 값을 계산할지가 분명하기 때문입니다.

- 대상이 명확합니다: 고객 문의 처리표
- 기준이 명확합니다: 문의유형별
- 집계할 값이 명확합니다: 평균 처리시간
- 기간이 들어 있습니다: 이번 달

단, 실제 표에 `처리시간`이 없다면 이 질문은 바로 답할 수 없습니다. 그 경우에는 처리 시작 시각과 완료 시각으로 처리시간을 계산하거나, 처리시간 열을 새로 만들어야 합니다. 분석 질문은 머릿속에서만 끝나는 문장이 아니라, 실제 표의 열과 연결되어야 합니다.

## 실습 코드

아래 파이썬 코드는 강사용 검산 코드입니다. pandas는 표 데이터를 다루는 파이썬 도구입니다. 이 코드는 카페 매출 파일을 읽고, 날짜를 날짜 형식으로 바꾼 뒤, 행 수, 열 수, 날짜 범위, 비어 있는 값, 매출 합계를 확인합니다.

S01에서 수강생이 반드시 실행해야 하는 코드는 아닙니다. 저장된 검산용 코드로 실제 실행해 정상 작동을 확인했습니다.

```python
from pathlib import Path

import pandas as pd


ROOT = Path(__file__).resolve().parents[1]
DATA_PATH = ROOT / "02_datasets" / "cafe_sales" / "cafe_sales_clean.csv"


def main():
    df = pd.read_csv(DATA_PATH)
    df["date"] = pd.to_datetime(df["date"])

    print(f"pandas_version={pd.__version__}")
    print(f"data_path={DATA_PATH}")
    print(f"rows={len(df)}")
    print(f"columns={len(df.columns)}")
    print("column_names=" + ", ".join(df.columns))
    print(f"date_range={df['date'].min().date()} ~ {df['date'].max().date()}")
    print(f"missing_total={int(df.isna().sum().sum())}")
    print(f"total_amount={int(df['amount'].sum())}")
    print(f"average_order_amount={df['amount'].mean():.1f}")

    print("\nstore_sales")
    print(df.groupby("store")["amount"].sum().sort_values(ascending=False).to_string())

    print("\ncategory_sales")
    print(df.groupby("category")["amount"].sum().sort_values(ascending=False).to_string())

    print("\ntop_5_hours_by_amount")
    print(df.groupby("hour")["amount"].sum().sort_values(ascending=False).head(5).to_string())


if __name__ == "__main__":
    main()
```
