# S07. 모범답안

## 미션 답안을 읽는 기준

미션은 “두 AI 답을 비교해서 더 믿을 만한 답을 고르는 것”입니다. 여기서 정답은 특정 AI 이름이 아닙니다. 왜 그 답을 믿을 수 있는지 설명하는 근거가 정답입니다.

AI 답을 볼 때는 멋진 문장보다 검산 가능한 흔적을 먼저 보셔야 합니다. 장부를 확인할 때 총액만 보는 것이 아니라 영수증 개수, 빠진 항목, 이상한 금액을 같이 보는 것과 같습니다. 더 믿을 만한 답은 아래 기준을 충족해야 합니다.

1. 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일의 행 수와 열 수를 먼저 확인한다.
2. 날짜 형식이 섞여 있음을 인식한다.
3. 중복 `order_id`를 어떻게 처리했는지 말한다.
4. 수량 999, 음수 금액, 단가×수량 불일치 행을 이상치로 따로 보여 준다.
5. 월별 매출 표와 차트의 숫자가 서로 맞는다.
6. 결론이 표의 숫자에서 바로 나온다.

## 기준 답안에서 적용한 처리 방식

전처리는 분석 전에 데이터를 정리하는 과정입니다. 요리하기 전에 재료를 씻고 상한 부분을 골라내는 것처럼, 매출을 계산하기 전에 날짜, 중복 주문, 이상한 금액을 먼저 확인합니다.

전처리 기준:

- `date`는 혼합 날짜 형식으로 변환했다.
- `order_id`는 주문 고유번호로 보고 중복 `order_id`는 첫 행만 남겼다.
- `unit_price`의 쉼표와 `원` 문자를 제거해 숫자로 변환했다.
- `amount <= 0`, `qty > 20`, `unit_price * qty != amount`인 행은 이상치로 표시하고 월별 매출 집계에서는 제외했다.
- `payment`, `member` 결측은 월별 매출 집계에서 제외하지 않았다.

여기서 중요한 점은 `payment`와 `member`가 비어 있어도 매출액 자체는 계산할 수 있다는 것입니다. 결제수단이나 회원 여부가 비어 있다고 해서 주문금액까지 없는 것은 아니기 때문입니다.

이 기준을 적용하면 분석용 데이터는 10,421행입니다.

## 월별 매출 추세 읽기

아래 표는 이상치와 중복 주문을 처리한 뒤, 월별로 주문건수, 판매수량, 매출액을 모은 결과입니다. 월별 매출은 달마다 매출이 어떻게 움직였는지 보는 표입니다. 키를 매달 재듯이, 매출도 같은 기준으로 월마다 재야 흐름을 비교할 수 있습니다.

| 월 | 주문건수 | 판매수량 | 매출액 | 전월대비 증감액 | 전월대비 증감률 |
|---|---:|---:|---:|---:|---:|
| 2025-01 | 1,685 | 2,784 | 14,421,000 | 0 |  |
| 2025-02 | 1,523 | 2,534 | 13,151,500 | -1,269,500 | -8.8% |
| 2025-03 | 1,716 | 2,842 | 14,735,500 | 1,584,000 | 12.0% |
| 2025-04 | 1,760 | 2,938 | 15,257,500 | 522,000 | 3.5% |
| 2025-05 | 1,800 | 2,996 | 16,094,000 | 836,500 | 5.5% |
| 2025-06 | 1,937 | 3,144 | 16,816,500 | 722,500 | 4.5% |

해석:

2025년 2월 매출은 1월보다 8.8% 감소했습니다. 이후 2025년 3월부터 6월까지 매출은 매월 증가했습니다. **2025년 6월 매출이 16,816,500원**으로 가장 크며, 주문건수와 판매수량도 가장 많습니다.

여기서 뭐가 보이시나요? 2월에 한 번 내려갔다가 3월부터 다시 올라갑니다. 그래서 “계속 증가했다”라고 말하면 틀립니다. “2월에 하락한 뒤 3월부터 6월까지 증가했다”라고 써야 표와 맞는 설명이 됩니다.

## 이상치로 제외한 행 확인하기

이상치는 단순히 값이 큰 데이터만 뜻하지 않습니다. 이번 데이터에서는 수량이 999처럼 현실적으로 보기 어렵거나, 금액이 음수이거나, 단가×수량과 주문금액이 맞지 않는 행이 문제입니다. 계산기가 맞는지 확인하듯이 `unit_price * qty`와 `amount`를 맞춰 봅니다.

| `order_id` | `date` | `item` | `unit_price` | `qty` | `amount` | 제외 이유 |
|---:|---|---|---:|---:|---:|---|
| 11496 | 2025-01-28 | 크로플 | 5,500 | 999 | 5,500 | 수량이 비현실적으로 크고 단가×수량과 금액이 맞지 않음 |
| 13013 | 2025-02-24 | 복숭아아이스티 | 5,500 | 1 | -5,500 | 금액이 음수이며 환불 여부를 설명하는 컬럼이 없음 |
| 16235 | 2025-04-23 | 콜드브루 | 5,500 | 999 | 5,500 | 수량이 비현실적으로 크고 단가×수량과 금액이 맞지 않음 |

일별 매출 합계는 IQR 기준으로 통계적 이상치가 없었습니다. IQR은 가운데에 모여 있는 보통 범위를 잡고, 그 범위에서 너무 멀리 떨어진 값을 찾는 방법입니다. 체육 시간에 키를 재고 유난히 튀는 값을 보는 것과 비슷합니다.

다만 가장 높은 일별 매출은 2025-06-21의 878,000원이었습니다. 이 값은 IQR 상한인 887,500원보다 낮아 통계적 이상치로 분류하지 않았습니다.

## 두 AI 답 비교 예시

두 AI 답을 비교할 때는 말투가 자연스러운지보다, 확인 가능한 숫자와 처리 기준이 있는지를 봅니다. 사전에서 단어를 찾듯이 하나씩 대조해 보세요.

| 비교 항목 | 답 A 예시 | 답 B 예시 | 판단 |
|---|---|---|---|
| 행 수와 열 수 | 말하지 않음 | 10,439행 × 12열 제시 | B가 더 신뢰 가능 |
| 날짜 형식 혼재 | 언급 없음 | 세 가지 날짜 형식을 모두 처리 | B가 더 신뢰 가능 |
| 중복 처리 | 기준 없음 | 중복 `order_id` 제거 기준 제시 | B가 더 신뢰 가능 |
| 이상치 처리 | "큰 매출 없음"만 언급 | 행 단위 이상치 3건 제시 | B가 더 신뢰 가능 |
| 월별 추세 | 결론만 제시 | 월별 표와 증감률 제시 | B가 더 신뢰 가능 |
| 최종 판단 | 근거 부족 | 검증 가능한 숫자 포함 | B 선택 |

제출 문장 예시:

나는 답 B를 더 신뢰합니다. 이유는 원본 행 수를 확인했고, 날짜 형식 혼재와 중복 `order_id`, 수량 999와 음수 금액을 전처리 기준에 포함했기 때문입니다. 이 데이터의 월별 매출은 2월에 하락한 뒤 3월부터 6월까지 증가하며, 6월 매출이 가장 큽니다.

## 검증용 pandas 코드

아래 코드는 AI가 만든 표가 맞는지 다시 계산해 보는 코드입니다. pandas는 표 데이터를 다루는 파이썬 도구입니다. 수업에서 직접 따라 할 때는 데이터 받기 페이지에서 카페 매출 파일을 내려받아 사용하면 되고, 초보자는 엑셀 파일(.xlsx)을 받아도 됩니다.

여기서는 검증을 위해 엑셀로도 열리는 표 파일인 CSV 데이터를 읽어 계산했습니다. 코드 안에서는 행 수, 열 수, 날짜 형식, 결측, 중복, 이상치를 차례로 확인합니다. 사람이 눈으로 세기 어려운 10,439행도 코드로는 빠르게 검산할 수 있습니다.

```python
from pathlib import Path

import pandas as pd

data_path = Path("/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_dirty.csv")
df = pd.read_csv(data_path)

expected_columns = [
    "order_id",
    "date",
    "weekday",
    "hour",
    "store",
    "category",
    "item",
    "unit_price",
    "qty",
    "amount",
    "payment",
    "member",
]

assert df.shape == (10439, 12)
assert list(df.columns) == expected_columns

date_parsed = pd.to_datetime(df["date"], format="mixed", errors="coerce")
unit_price_num = pd.to_numeric(
    df["unit_price"]
    .astype(str)
    .str.replace(",", "", regex=False)
    .str.replace("원", "", regex=False)
    .str.strip(),
    errors="coerce",
)

work = df.assign(
    date_parsed=date_parsed,
    unit_price_num=unit_price_num,
)
work["amount_mismatch"] = work["amount"] != work["unit_price_num"] * work["qty"]

date_format_counts = {
    "YYYY-MM-DD": int(df["date"].str.match(r"^\d{4}-\d{2}-\d{2}$").sum()),
    "YYYY/MM/DD": int(df["date"].str.match(r"^\d{4}/\d{2}/\d{2}$").sum()),
    "MM-DD-YYYY": int(df["date"].str.match(r"^\d{2}-\d{2}-\d{4}$").sum()),
}

quality_summary = {
    "전체 행 수": len(df),
    "전체 열 수": len(df.columns),
    "완전 중복 행": int(df.duplicated().sum()),
    "중복 order_id 추가 행": int(df["order_id"].duplicated().sum()),
    "payment 결측": int(df["payment"].isna().sum()),
    "member 결측": int(df["member"].isna().sum()),
    "store 값 '강남 점'": int((df["store"] == "강남 점").sum()),
    "앞 공백 category": int(df["category"].str.startswith(" ", na=False).sum()),
    "unit_price에 원 문자 포함": int(df["unit_price"].astype(str).str.contains("원", regex=False).sum()),
    "qty > 20": int((df["qty"] > 20).sum()),
    "amount <= 0": int((df["amount"] <= 0).sum()),
    "단가x수량과 금액 불일치": int(work["amount_mismatch"].sum()),
}

analysis = work.drop_duplicates("order_id", keep="first").copy()

anomaly_mask = (
    analysis["date_parsed"].isna()
    | (analysis["qty"] > 20)
    | (analysis["amount"] <= 0)
    | (analysis["amount_mismatch"])
)

anomalies = analysis.loc[
    anomaly_mask,
    ["order_id", "date", "item", "unit_price_num", "qty", "amount", "amount_mismatch"],
].sort_values("order_id")

cleaned = analysis.loc[~anomaly_mask].copy()
cleaned["month"] = cleaned["date_parsed"].dt.to_period("M").astype(str)

monthly = (
    cleaned.groupby("month")
    .agg(
        주문건수=("order_id", "count"),
        판매수량=("qty", "sum"),
        매출액=("amount", "sum"),
    )
    .reset_index()
)
monthly["전월대비 증감액"] = monthly["매출액"].diff().fillna(0).astype(int)
monthly["전월대비 증감률"] = monthly["매출액"].pct_change().mul(100).round(1)

daily = (
    cleaned.groupby("date_parsed")
    .agg(
        주문건수=("order_id", "count"),
        판매수량=("qty", "sum"),
        매출액=("amount", "sum"),
    )
    .reset_index()
    .rename(columns={"date_parsed": "date"})
)

q1 = daily["매출액"].quantile(0.25)
q3 = daily["매출액"].quantile(0.75)
iqr = q3 - q1
lower_bound = q1 - 1.5 * iqr
upper_bound = q3 + 1.5 * iqr
daily_outliers = daily[(daily["매출액"] < lower_bound) | (daily["매출액"] > upper_bound)]
top_daily = daily.sort_values("매출액", ascending=False).head(3)

assert len(cleaned) == 10421
assert int(cleaned["amount"].sum()) == 90476000
assert int(monthly.loc[monthly["month"] == "2025-06", "매출액"].iloc[0]) == 16816500
assert len(anomalies) == 3
assert len(daily_outliers) == 0

print("데이터 행열:", df.shape)
print("날짜 형식 수:", date_format_counts)
print("품질 점검:", quality_summary)
print()
print("[월별 매출 추세]")
print(monthly.to_string(index=False))
print()
print("[이상치로 제외한 행]")
print(anomalies.to_string(index=False))
print()
print("일별 매출 IQR 상한:", int(upper_bound))
print("일별 매출 이상치 수:", len(daily_outliers))
print("[일별 매출 상위 3일]")
print(top_daily.to_string(index=False))
```
