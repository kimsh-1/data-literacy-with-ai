# S09 모범답안

## 미션 모범답안

동료가 AI로 낸 숫자를 보고서에 바로 넣기 전에, 먼저 “계산은 맞는가”와 “그 계산을 믿어도 되는가”를 나누어 봐야 합니다. 계산기가 숫자를 맞게 더했더라도, 저울 위에 올린 재료가 잘못되어 있으면 결론도 조심해야 하는 것과 같습니다.

상황:

```text
동료가 AI로 cafe_sales_dirty.xlsx를 분석했고,
"총매출은 90,600,500원이며 6월 매출이 가장 높습니다"라고 보고했다.
이 답을 그대로 보고서에 넣어도 되는지 검토해야 한다.
```

모범 판단:

```text
원본을 그대로 합산한 총매출 90,600,500원은 엑셀 피벗 기준값과 일치한다.
그러나 dirty 파일에는 주문번호 중복, 지점명과 카테고리 표기 흔들림, qty=999, amount 음수, 단가×수량과 금액 불일치가 확인된다.
따라서 이 값은 "정제 전 원본 합계"로는 말할 수 있지만, 정제 기준을 정하기 전에는 보고서의 최종 매출로 쓰면 안 된다.
```

이 답안에서 중요한 부분은 “90,600,500원이 맞다”에서 멈추지 않는 점입니다. 엑셀 피벗테이블로 원본을 그대로 더하면 그 숫자가 나옵니다. 그래서 계산 자체는 통과로 볼 수 있습니다.

하지만 dirty 파일은 일부러 지저분하게 만든 데이터입니다. 주문이 중복되어 있거나, 같은 강남점이 `강남점`과 `강남 점`처럼 다르게 적혀 있거나, 수량이 비정상적으로 큰 값으로 들어가 있습니다. 이런 상태에서는 “원본을 그대로 더한 값”과 “보고서에 써도 되는 최종 매출”을 반드시 구분해야 합니다.

## 검증 체크리스트 작성 예

검증 체크리스트는 AI 답을 채점하는 표라고 보면 됩니다. 답이 맞는지 틀리는지만 보는 것이 아니라, 어떤 기준으로 계산했는지도 함께 봅니다. 예를 들어 금액 열로 `amount`를 썼는지, 전체 행을 다 썼는지, 중복이나 이상값을 확인했는지를 하나씩 확인합니다.

| 점검 항목 | 기준 또는 확인값 | 판정 |
|---|---:|---|
| 파일명 | `cafe_sales_dirty.xlsx` | 통과 |
| 시트명 | `sales` | 통과 |
| 행수 | 10,439행 | 통과 |
| 사용 금액 열 | `amount` | 통과 |
| 원본 총매출 | 90,600,500원 | 통과 |
| clean 총매출과 차이 | dirty가 108,000원 큼 | 확인 필요 |
| 6월 원본 매출 | 16,822,000원 | 통과 |
| 주문번호 중복 행 | 15행 | 확인 필요 |
| payment 결측 | 313행 | 확인 필요 |
| member 결측 | 209행 | 확인 필요 |
| 날짜 형식 혼재 | `YYYY/MM/DD` 206행, `MM-DD-YYYY` 104행 | 확인 필요 |
| 지점명 표기 흔들림 | `강남 점` 310행 | 확인 필요 |
| category 앞뒤 공백 | 313행 | 확인 필요 |
| 아메리카노 표기 변형 | 106행 | 확인 필요 |
| 수량 이상치 | `qty=999` 2행 | 확인 필요 |
| 음수 금액 | 1행 | 확인 필요 |
| `unit_price × qty = amount` | 불일치 3행 | 확인 필요 |

좋은 답안은 단순히 숫자를 다시 적는 답안이 아닙니다. AI가 말한 값과 피벗 기준값이 일치하는 부분은 인정하고, 그래도 최종 결론으로 쓰면 안 되는 이유를 설명해야 합니다.

- AI 답과 피벗 기준값이 일치하는 항목을 구분한다.
- "원본 그대로 합산한 값"과 "최종 확정값"을 구분한다.
- dirty 데이터의 품질 문제를 최소 3가지 이상 언급한다.
- 최종 판단을 `사용 가능`, `조건부 사용`, `사용 보류` 중 하나로 명확히 쓴다.

여기서는 `사용 보류`에 가깝습니다. 원본 합계는 말할 수 있지만, 보고서의 최종 매출로 확정하려면 중복과 이상값을 어떻게 처리할지 먼저 정해야 합니다.

## 따라하기 결과 해설

### 1. clean 파일 검산 결과

수업 홈의 데이터 받기 페이지에서 내려받은 clean 카페 매출 파일로 검산합니다. 초보자는 엑셀 파일(.xlsx)을 받아서 열어도 괜찮습니다. 검증 코드에서는 같은 내용을 담은 CSV, 즉 엑셀로도 열리는 표 파일을 사용했습니다.

[^1]: S09에서 쓰는 `cafe_sales_clean`은 S07의 `cafe_sales_dirty`를 수강 중 직접 정제해 만든 파일이 아니라, 별도로 제공된 검증용 clean 원천 파일입니다.

실제로 파일을 열어 행수, 기간, 총매출을 확인하면 아래 값이 나옵니다. 여기서 행수는 머리글을 뺀 데이터 줄 수입니다.

- 행수: 10,424행
- 열수: 12열
- 기간: 2025-01-01부터 2025-06-30까지
- 총매출: 90,492,500원
- 판매수량 합계: 17,241개

지점별 매출은 `store`별로 `amount`를 더한 값입니다. 지점별 값을 모두 더했을 때 총매출과 같아야 검산이 맞습니다.

| 지점 | 매출 |
|---|---:|
| 강남점 | 53,425,000원 |
| 홍대점 | 37,067,500원 |
| 총합계 | 90,492,500원 |

카테고리별 매출은 `category`별로 `amount`를 더한 값입니다. 상품을 커피, 음료, 디저트 바구니에 나눠 담고 각 바구니의 금액을 더하는 방식이라고 생각하면 됩니다.

| 카테고리 | 매출 |
|---|---:|
| 커피 | 45,606,000원 |
| 음료 | 25,550,000원 |
| 디저트 | 19,336,500원 |
| 총합계 | 90,492,500원 |

월별 매출은 날짜에서 월만 뽑아 묶은 뒤 `amount`를 더한 값입니다. 하루하루의 주문을 월 단위 서랍에 넣고, 서랍별 금액을 더해 보는 방식입니다.

| 월 | 매출 |
|---|---:|
| 2025-01 | 14,426,500원 |
| 2025-02 | 13,157,000원 |
| 2025-03 | 14,735,500원 |
| 2025-04 | 15,263,000원 |
| 2025-05 | 16,094,000원 |
| 2025-06 | 16,816,500원 |

해석:

```text
clean 파일에서 AI가 총매출을 90,492,500원으로 답하고, 지점별·카테고리별 합계도 위 피벗 기준값과 일치하면 계산 검산은 통과로 볼 수 있다. 다만 답변에 사용한 열이 amount인지, 전체 10,424행을 사용했는지는 함께 확인해야 한다.
```

여기서 통과라고 말할 수 있는 범위는 “계산 검산”입니다. AI가 `amount` 열을 사용했고, 전체 10,424행을 대상으로 계산했으며, 피벗 기준값과 일치한다면 clean 파일에서는 보고서에 쓸 수 있는 숫자에 가까워집니다.

### 2. dirty 파일 검산 결과

수업 홈의 데이터 받기 페이지에서 내려받은 dirty 카페 매출 파일로 같은 검산을 해 봅니다. dirty는 말 그대로 일부러 지저분하게 만든 파일입니다. 그래서 피벗과 AI의 숫자가 같아도, 데이터 자체가 믿을 만한지 한 번 더 봐야 합니다.

실제 확인 결과:

- 행수: 10,439행
- 열수: 12열
- 원본 그대로 합산한 총매출: 90,600,500원
- 판매수량 합계: 19,259개
- clean 총매출과 차이: 108,000원

dirty 원본 store 기준 매출:

| store | 매출 |
|---|---:|
| 강남 점 | 2,618,000원 |
| 강남점 | 50,858,500원 |
| 홍대점 | 37,124,000원 |
| 총합계 | 90,600,500원 |

여기서 뭐가 보이시나요? `강남점`과 `강남 점`은 사람 눈에는 같은 지점처럼 보이지만, 엑셀은 두 값을 서로 다른 이름으로 봅니다. 사전에서 `강남점`이라는 단어와 `강남 점`이라는 단어를 따로 찾는 것과 비슷합니다.

dirty 원본 category 기준 매출:

| category | 매출 |
|---|---:|
| ` 디저트 ` | 584,500원 |
| ` 음료 ` | 817,000원 |
| ` 커피 ` | 1,288,500원 |
| 디저트 | 18,772,500원 |
| 음료 | 24,746,000원 |
| 커피 | 44,392,000원 |

카테고리도 같은 문제가 있습니다. 앞뒤에 공백이 붙어 있으면 `커피`와 ` 커피 `가 서로 다른 카테고리처럼 잡힙니다. 사람은 대충 같은 말로 읽지만, 데이터는 대충 읽어 주지 않습니다.

공백을 정리해서 같은 범주로 묶으면 이렇게 보입니다.

| 기준 | 값 |
|---|---:|
| 강남점 매출 | 53,476,500원 |
| 홍대점 매출 | 37,124,000원 |
| 커피 매출 | 45,680,500원 |
| 음료 매출 | 25,563,000원 |
| 디저트 매출 | 19,357,000원 |

해석:

```text
dirty 파일은 원본을 그대로 합산하면 90,600,500원이 맞다. 그러나 지점명 `강남 점`, category 앞뒤 공백, 중복 주문, qty=999, 음수 금액이 있어 이 값을 최종 확정 매출이라고 말하면 안 된다. 피벗 검산은 "AI가 원본을 제대로 더했는가"를 확인하고, 데이터품질 점검은 "그 원본 합계를 믿어도 되는가"를 확인한다.
```

피벗 검산과 데이터품질 점검은 역할이 다릅니다. 피벗 검산은 AI가 계산을 제대로 했는지 보는 과정입니다. 데이터품질 점검은 그 계산에 사용한 원본이 믿을 만한지 보는 과정입니다.

## dirty 데이터 품질 점검 결과

dirty 파일에서는 아래 문제가 확인됩니다. 이 표는 “왜 원본 합계를 그대로 최종 매출이라고 말하면 안 되는지”를 보여 주는 근거입니다.

| 점검 항목 | 확인값 |
|---|---:|
| 원본 대비 증가 행 | 15 |
| 주문번호 중복 행 | 15 |
| 완전 동일 중복 행 | 13 |
| 중복 주문번호 종류 | 15 |
| payment 결측 | 313 |
| member 결측 | 209 |
| 날짜 `YYYY/MM/DD` 형식 | 206 |
| 날짜 `MM-DD-YYYY` 형식 | 104 |
| `unit_price` 문자 포함 | 157 |
| `store='강남 점'` | 310 |
| `category` 앞뒤 공백 | 313 |
| 아메리카노 표기 변형 | 106 |
| `qty=999` | 2 |
| `amount` 음수 | 1 |
| `unit_price × qty`와 `amount` 불일치 | 3 |

예를 들어 주문번호 중복 행이 있으면 같은 주문이 두 번 더해졌을 가능성이 있습니다. `qty=999`는 일반적인 카페 주문 수량으로 보기 어렵기 때문에 이상값으로 확인해야 합니다. `amount` 음수는 환불인지 입력 오류인지 구분해야 합니다. 이런 확인 없이 총매출만 말하면 숫자는 그럴듯해 보여도 설명이 부족합니다.

## AI에게 다시 물을 때의 모범 프롬프트

AI 답이 엑셀 피벗 기준값과 다를 때는 “틀렸어요”라고만 말하지 말고, 비교 기준을 같이 알려 주는 것이 좋습니다. 그래야 AI가 어떤 열을 다시 봐야 하는지, 어떤 행을 빠뜨렸는지 점검할 수 있습니다.

clean 파일에서 AI 답이 피벗과 다를 때:

```text
내 엑셀 피벗 기준으로 cafe_sales_clean.xlsx의 amount 총합계는 90,492,500원입니다.
방금 답과 차이가 납니다.
amount 열만 합산했는지, 전체 10,424행을 사용했는지, 어떤 필터를 적용했는지 확인해서 다시 계산해 주세요.
```

dirty 파일에서 AI가 품질 문제 없이 결론만 말했을 때는 더 구체적으로 물어야 합니다. 이때는 총매출을 다시 계산하는 것보다, 원본 데이터에 어떤 문제가 있는지 먼저 확인하게 하는 것이 좋습니다.

```text
방금 답은 원본 amount 합계만 말했습니다.
보고서에 쓰기 전에 order_id 중복, store/category 표기 흔들림, qty 이상치, amount 음수, unit_price×qty와 amount 불일치를 점검해 주세요.
각 항목의 건수와 이 문제가 총매출 해석에 주는 영향을 함께 적어 주세요.
```

이런 식으로 물으면 AI가 단순 합계만 반복하지 않고, 보고서에 필요한 검증 근거를 같이 제시하게 만들 수 있습니다.

## 실행한 검증 코드

아래 코드는 수업 답안 자료에 있는 검증 코드입니다. 초보자는 코드를 모두 이해하지 않아도 괜찮습니다. 여기서는 “엑셀 피벗으로 확인한 값과 코드로 확인한 값이 같은지”를 보는 용도로 사용합니다.

실행 명령:

```bash
python3 /mnt/d/ai-data-analysis-course/06_answers/S09_code.py
```

검산에 사용한 코드:

```python
from pathlib import Path

import pandas as pd


ROOT = Path("/mnt/d/ai-data-analysis-course")
CLEAN_CSV = ROOT / "02_datasets/cafe_sales/cafe_sales_clean.csv"
DIRTY_CSV = ROOT / "02_datasets/cafe_sales/cafe_sales_dirty.csv"

clean = pd.read_csv(CLEAN_CSV, encoding="utf-8-sig")
dirty = pd.read_csv(DIRTY_CSV, encoding="utf-8-sig")

for df in [clean, dirty]:
    df["amount_num"] = pd.to_numeric(df["amount"], errors="coerce")
    df["qty_num"] = pd.to_numeric(df["qty"], errors="coerce")
    df["unit_price_num"] = pd.to_numeric(
        df["unit_price"].astype(str).str.replace(",", "", regex=False).str.replace("원", "", regex=False),
        errors="coerce",
    )
    df["expected_amount"] = df["unit_price_num"] * df["qty_num"]
    df["date_parsed"] = pd.to_datetime(df["date"], format="mixed", errors="coerce")
    df["month"] = df["date_parsed"].dt.to_period("M").astype(str)
    df["store_norm"] = df["store"].astype(str).str.replace(" ", "", regex=False)
    df["category_norm"] = df["category"].astype(str).str.strip()

clean_total = int(clean["amount_num"].sum())
dirty_total = int(dirty["amount_num"].sum())

clean_store = clean.pivot_table(index="store", values="amount_num", aggfunc="sum")
clean_category = clean.pivot_table(index="category", values="amount_num", aggfunc="sum")
clean_monthly = clean.groupby("month")["amount_num"].sum()

dirty_store_raw = dirty.pivot_table(index="store", values="amount_num", aggfunc="sum")
dirty_category_raw = dirty.pivot_table(index="category", values="amount_num", aggfunc="sum")
dirty_store_norm = dirty.pivot_table(index="store_norm", values="amount_num", aggfunc="sum")
dirty_category_norm = dirty.pivot_table(index="category_norm", values="amount_num", aggfunc="sum")

dirty_issues = {
    "order_id_duplicated_after_first": int(dirty["order_id"].duplicated().sum()),
    "payment_missing": int(dirty["payment"].isna().sum()),
    "member_missing": int(dirty["member"].isna().sum()),
    "store_spaced": int((dirty["store"] == "강남 점").sum()),
    "category_spaced": int((dirty["category"].astype(str) != dirty["category"].astype(str).str.strip()).sum()),
    "qty_999": int((dirty["qty_num"] == 999).sum()),
    "amount_negative": int((dirty["amount_num"] < 0).sum()),
    "amount_mismatch": int((dirty["expected_amount"] != dirty["amount_num"]).sum()),
}

print(clean_total)
print(dirty_total)
print(clean_store)
print(clean_category)
print(clean_monthly)
print(dirty_store_raw)
print(dirty_category_raw)
print(dirty_store_norm)
print(dirty_category_norm)
print(dirty_issues)
```

코드가 확인한 값도 엑셀 피벗으로 본 값과 맞습니다. 그래서 이 수업에서는 피벗테이블과 코드 검산이 서로 같은 기준을 보고 있다고 이해하면 됩니다.
