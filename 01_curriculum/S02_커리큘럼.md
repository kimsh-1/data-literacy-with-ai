# S02. 좋은 데이터의 모양: tidy 데이터와 척도

## 차시 개요

데이터 분석에서 좋은 표는 보기 예쁜 표가 아니라, 컴퓨터가 규칙대로 읽을 수 있는 표입니다. 엑셀에서 색을 넣고 셀을 합치면 사람 눈에는 보기 편할 수 있지만, 피벗테이블이나 분석 도구는 그 표를 정확히 이해하지 못할 수 있습니다. 이 차시에서는 카페 매출 데이터를 보면서 “분석하기 좋은 표는 어떤 모양인가”를 확인합니다.

- 대상: 엑셀은 사용하지만 데이터 분석은 처음인 비개발자
- 선수 차시: S01
- 난이도: 1
- 시간: 40분
- 사용 데이터:
  - 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 정리본 파일
  - 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 문제 포함 파일
- 확인한 데이터 규모:
  - clean: 10,424행, 12열
  - dirty: 10,439행, 12열

여기서 clean 데이터는 분석하기 좋게 정리된 표를 뜻합니다. dirty 데이터는 같은 카페 매출 데이터 안에 실무에서 자주 생기는 문제를 일부러 넣어 둔 표입니다. 두 표를 나란히 보면 “왜 표 모양이 분석 결과에 영향을 주는지”가 훨씬 잘 보입니다.

## 학습 목표

1. 분석하기 좋은 표의 기본 모양을 설명할 수 있습니다.
2. 한 셀 한 값, 한 행 한 관찰, 한 열 한 변수의 의미를 엑셀 표에서 직접 찾을 수 있습니다.
3. 병합셀, 빈 셀, 표기 혼재, 숫자와 문자의 혼합이 분석을 방해하는 이유를 이해합니다.
4. 컬럼별로 범주형, 순서형, 수치형, 날짜형, 논리형 척도를 구분할 수 있습니다.
5. dirty 데이터에서 tidy 원칙을 어기는 지점을 찾고, 어떤 방향으로 고쳐야 하는지 말할 수 있습니다.

목표를 어렵게 느끼지 않으셔도 됩니다. 지금은 통계 공식을 외우는 시간이 아니라, 표를 볼 때 “이 표를 컴퓨터가 잘 읽을 수 있을까?” 하고 점검하는 눈을 만드는 시간입니다.

## 개념 설명

### 1. 분석용 표는 보기 좋은 보고서와 다릅니다

엑셀에서 사람에게 보여주기 좋은 표는 제목이 크게 들어가고, 셀이 병합되어 있고, 중간중간 소계나 설명이 들어가는 경우가 많습니다. 회의 자료나 보고서로 읽을 때는 이런 모양이 편할 수 있습니다. 사람이 눈으로 훑으면서 “아, 이 묶음은 같은 지점이구나” 하고 알아차릴 수 있기 때문입니다.

그런데 피벗테이블, AI 분석, 파이썬, BI 도구는 사람처럼 눈치로 읽지 않습니다. 도구는 셀 하나하나를 정해진 규칙대로 읽습니다. 셀이 비어 있으면 비어 있다고 보고, 문자가 섞여 있으면 숫자가 아니라고 보고, 같은 상품명이라도 글자가 다르면 다른 상품으로 봅니다.

분석용 표에서는 장식보다 규칙이 중요합니다. 이번 차시에서 사용할 카페 매출 데이터는 주문 한 건이 한 행입니다. `date`, `store`, `item`, `qty`, `amount`처럼 각 열은 주문을 설명하는 한 가지 속성만 담습니다. 이렇게 되어 있어야 필터, 정렬, 피벗, 집계가 안정적으로 작동합니다.

예를 들어 키를 잴 때 한 사람당 한 줄로 키를 적어야 평균 키를 계산할 수 있죠. 한 줄에 두 사람 키가 같이 적혀 있거나, 어떤 줄에는 “크다”라고만 적혀 있으면 계산이 어려워집니다. 데이터 표도 마찬가지입니다.

### 2. tidy 데이터의 세 가지 규칙

tidy 데이터는 분석하기 좋은 표의 기본 모양입니다. tidy는 “깔끔하게 정리된” 정도로 이해하시면 됩니다. 입문 단계에서는 아래 세 문장만 정확히 잡아도 충분합니다.

| 규칙 | 뜻 | 엑셀에서 깨지는 예 |
|---|---|---|
| 한 행은 한 관찰이다 | 한 줄이 주문 한 건, 고객 한 명, 직원 한 명처럼 같은 단위여야 한다 | 중간에 월별 소계 행이 끼어 있음 |
| 한 열은 한 변수다 | 한 열에는 날짜, 지점, 상품명, 수량처럼 하나의 의미만 있어야 한다 | `상품/수량`처럼 한 열에 두 의미가 섞임 |
| 한 셀은 한 값이다 | 셀 하나에는 하나의 값만 들어가야 한다 | `4,500원`처럼 숫자와 단위가 섞임, `강남점/카드`처럼 값 두 개가 한 셀에 있음 |

“한 행은 한 관찰이다”라는 말은 한 줄이 같은 종류의 대상을 설명해야 한다는 뜻입니다. 카페 매출 데이터라면 한 행은 주문 한 건이어야 합니다. 주문 데이터 사이에 “3월 합계” 같은 소계 행이 들어가면, 컴퓨터는 그 줄도 주문 한 건처럼 읽을 수 있습니다.

“한 열은 한 변수다”라는 말은 한 열에 한 가지 의미만 담아야 한다는 뜻입니다. `store` 열에는 지점만, `item` 열에는 상품명만, `qty` 열에는 수량만 있어야 합니다. 한 열에 상품명과 수량이 같이 들어가면 나중에 상품별 매출을 구하거나 수량 합계를 낼 때 다시 분리해야 합니다.

“한 셀은 한 값이다”라는 말은 셀 하나에 계산하거나 분류할 값이 하나만 들어가야 한다는 뜻입니다. `4,500원`은 사람에게는 친절해 보이지만, 컴퓨터 입장에서는 숫자 `4500`에 문자 `원`이 붙은 값입니다. 계산하려면 먼저 숫자만 남기는 정리 작업이 필요합니다.

병합셀은 이 원칙을 자주 무너뜨립니다. 사람 눈에는 위아래가 같은 그룹처럼 보이지만, 컴퓨터는 병합된 첫 셀만 값이 있고 나머지 셀은 빈 칸으로 처리하는 경우가 많습니다. 그래서 분석용 원본 표에는 병합셀을 쓰지 않는 것이 원칙입니다.

### 3. clean 데이터와 dirty 데이터를 비교해 봅니다

이번 차시의 clean 데이터는 분석용 표의 기준 예시입니다. 표가 완벽하다는 뜻보다는, 지금 단계에서 피벗테이블과 기본 분석을 안정적으로 해 볼 수 있는 모양이라는 뜻입니다.

dirty 데이터는 같은 카페 매출 데이터에 실무에서 자주 나오는 문제를 일부러 심어 둔 파일입니다. 실무 데이터는 처음부터 깔끔하게 오지 않는 경우가 많습니다. 같은 지점명이 조금 다르게 적혀 있거나, 날짜 형식이 섞여 있거나, 숫자 열에 문자가 붙어 있는 일이 자주 생깁니다. 이런 문제를 찾는 힘이 있어야 분석 결과도 믿을 수 있습니다.

clean 데이터의 컬럼은 아래처럼 구성되어 있습니다.

| 컬럼 | 의미 |
|---|---|
| `order_id` | 주문번호 |
| `date` | 주문일 |
| `weekday` | 요일 |
| `hour` | 주문 시간대 |
| `store` | 지점 |
| `category` | 상품 대분류 |
| `item` | 상품명 |
| `unit_price` | 단가 |
| `qty` | 수량 |
| `amount` | 주문금액 |
| `payment` | 결제수단 |
| `member` | 회원 여부 |

컬럼은 표의 세로줄입니다. 각 컬럼은 주문을 설명하는 질문 하나라고 보면 쉽습니다. “언제 주문했나요?”는 `date`, “어느 지점인가요?”는 `store`, “무엇을 샀나요?”는 `item`, “얼마였나요?”는 `amount`가 답합니다.

dirty 데이터에서 확인한 대표 문제는 아래 표에 모아 둡니다.

| 문제 유형 | 실제 예 | 왜 문제인가 |
|---|---|---|
| 주문번호 중복 | 같은 `order_id`가 다시 등장 | 주문 건수가 부풀 수 있음 |
| 결측 | `payment`, `member`가 비어 있음 | 결제수단별, 회원 여부별 집계가 흔들림 |
| 날짜 형식 혼합 | `2025-03-23`, `2025/03/23`, `03-23-2025` | 월별, 요일별 분석 전 날짜 인식이 불안정함 |
| 숫자와 문자 혼합 | `unit_price`에 `4,500원` 형태 포함 | 합계와 평균 계산 전에 숫자로 바꿔야 함 |
| 범주 표기 혼재 | `강남점`과 `강남 점`, `아메리카노`와 `Americano` | 같은 대상을 다른 그룹으로 집계함 |
| 앞뒤 공백 | ` 커피 `, ` 음료 ` | 눈에는 같아 보여도 컴퓨터는 다른 값으로 볼 수 있음 |
| 이상치 | `qty`가 999 | 평균과 총합을 크게 왜곡함 |
| 불가능한 값 | `amount`가 음수 | 매출 데이터의 기본 규칙과 맞지 않음 |

여기서 결측은 값이 비어 있는 상태입니다. 결제수단이 비어 있으면 카드 매출인지 현금 매출인지 알 수 없고, 회원 여부가 비어 있으면 회원과 비회원 비교가 흔들립니다.

범주 표기 혼재도 자주 나오는 문제입니다. 사람은 `강남점`과 `강남 점`을 보고 같은 지점이라고 알아차릴 수 있습니다. 하지만 컴퓨터는 글자가 정확히 다르면 다른 값으로 봅니다. 사전에서 단어를 찾듯이, 철자가 다르면 다른 항목으로 분류하는 셈입니다.

이상치도 조심해야 합니다. `qty`가 999인 주문은 실제 대량 주문일 수도 있지만, 일반적인 카페 주문에서는 입력 오류일 가능성이 높습니다. 이런 값 하나가 평균과 총합을 크게 밀어 올릴 수 있습니다. 그래서 이상한 값은 바로 지우기보다, 먼저 “왜 이런 값이 들어왔을까?”를 확인해야 합니다.

### 4. 척도는 분석 방법을 고르는 기준입니다

데이터 분석에서 컬럼을 볼 때는 값의 모양만 보면 부족합니다. “이 값으로 무엇을 할 수 있는가”를 같이 봐야 합니다. 이것이 척도 구분입니다.

척도는 쉽게 말해 데이터 값의 성격입니다. 이름표처럼 분류만 하는 값인지, 순서가 있는 값인지, 더하고 평균을 낼 수 있는 값인지에 따라 분석 방법이 달라집니다.

| 척도 | 설명 | 가능한 작업 | 예 |
|---|---|---|---|
| 범주형 | 이름표처럼 그룹을 구분한다 | 개수 세기, 그룹별 합계 | 지점, 상품명, 결제수단 |
| 순서형 | 순서가 있지만 간격이 숫자처럼 정확하지 않을 수 있다 | 정렬, 단계별 비교 | 만족도, 등급 |
| 수치형 | 더하기, 평균, 최솟값, 최댓값 계산이 가능하다 | 합계, 평균, 분포 확인 | 수량, 단가, 주문금액 |
| 날짜형 | 시간의 흐름을 나타낸다 | 일별, 월별, 요일별 추세 | 주문일 |
| 논리형 | 참과 거짓 또는 예와 아니오 | 비율 계산, 필터링 | 회원 여부 |
| 식별자 | 각 행을 구분하는 이름표다 | 중복 확인, 추적 | 주문번호 |

범주형은 이름표입니다. 지점, 상품명, 결제수단처럼 그룹을 나눌 때 씁니다. `store`가 강남점인지 홍대점인지에 따라 매출을 나눠 볼 수 있습니다.

수치형은 계산할 수 있는 값입니다. `qty`, `unit_price`, `amount`는 합계, 평균, 최솟값, 최댓값을 계산할 수 있습니다. 키를 재고 평균 키를 구하듯이, 주문금액을 모아 평균 주문금액을 구할 수 있습니다.

날짜형은 시간의 흐름을 보는 값입니다. 주문일이 날짜형으로 잘 들어가 있어야 일별 매출, 월별 매출, 요일별 매출을 볼 수 있습니다. 날짜처럼 보여도 형식이 섞여 있으면 컴퓨터가 날짜로 제대로 인식하지 못할 수 있습니다.

주의할 점은 숫자로 보인다고 모두 수치형은 아니라는 것입니다. `order_id`는 숫자로 저장되어 있을 수 있지만 평균 주문번호를 구해도 의미가 없습니다. 주문번호는 계산할 숫자가 아니라 각 주문을 구분하기 위한 식별자입니다.

## 수업 흐름

| 시간 | 내용 | 진행 방식 |
|---:|---|---|
| 0~5분 | S01 복습과 오늘의 질문 제시 | “좋은 표와 예쁜 표는 다르다”는 문제의식 공유 |
| 5~15분 | tidy 데이터 원칙 설명 | 주문 한 건을 한 행으로 보는 비유와 엑셀 비교 |
| 15~25분 | `cafe_sales_dirty.xlsx`에서 문제 찾기 | 필터, 정렬, 값 목록 확인 |
| 25~33분 | 컬럼별 척도 분류 | clean 데이터 컬럼을 기준으로 범주형, 수치형 등 분류 |
| 33~38분 | 직접 해보기 미션 | 수강생이 척도 분류표와 교정 방향 작성 |
| 38~40분 | 정리와 다음 차시 연결 | S03 피벗테이블에서 왜 tidy가 필요한지 연결 |

수업에서는 먼저 좋은 표와 예쁜 표의 차이를 확인합니다. 그다음 clean 데이터로 분석하기 좋은 표의 모양을 보고, dirty 데이터에서 실제로 어떤 문제가 분석을 방해하는지 찾아봅니다.

마지막에는 각 컬럼의 척도를 분류합니다. 이 과정은 분석 전에 하는 준비 운동에 가깝습니다. 어떤 컬럼은 더하고 평균을 낼 수 있고, 어떤 컬럼은 그룹으로 나누어 세야 하며, 어떤 컬럼은 중복 확인에만 써야 합니다. 이 구분을 해 두면 다음 차시에서 피벗테이블을 만들 때 훨씬 덜 헷갈립니다.

## 다음 차시와의 연결

S03에서는 피벗테이블로 지점별, 카테고리별, 요일별 매출 질문에 답합니다. 피벗테이블은 표를 자동으로 요약해 주는 도구입니다. 하지만 원본 표가 흐트러져 있으면 피벗테이블도 틀어진 결과를 만들 수 있습니다.

예를 들어 같은 상품명이 여러 표기로 흩어져 있으면 상품별 매출이 여러 줄로 쪼개집니다. 숫자 열에 문자가 섞여 있으면 합계나 평균 계산이 제대로 되지 않을 수 있습니다. 날짜가 제각각이면 월별, 요일별 분석 전에 날짜 인식부터 흔들립니다.

그래서 S03에서 피벗테이블을 편하게 쓰려면, 이번 차시에서 다룬 tidy 구조를 먼저 이해해야 합니다. 분석은 멋진 차트를 만드는 일에서 시작하지 않습니다. 표가 한 행, 한 열, 한 셀 단위로 바르게 정리되어 있는지 보는 일에서 시작합니다.
