# S02. 좋은 데이터의 모양: tidy 데이터와 척도

## 차시 개요

- 대상: 엑셀은 사용하지만 데이터 분석은 처음인 비개발자
- 선수 차시: S01
- 난이도: 1
- 시간: 40분
- 사용 데이터:
  - `/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.xlsx`
  - `/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_dirty.xlsx`
- 확인한 데이터 규모:
  - clean: 10,424행, 12열
  - dirty: 10,439행, 12열

## 학습 목표

1. 분석하기 좋은 표의 기본 모양을 설명할 수 있다.
2. 한 셀 한 값, 한 행 한 관찰, 한 열 한 변수의 의미를 엑셀 표에서 찾을 수 있다.
3. 병합셀, 빈 셀, 표기 혼재, 숫자와 문자의 혼합이 분석을 방해하는 이유를 이해한다.
4. 컬럼별로 범주형, 순서형, 수치형, 날짜형, 논리형 척도를 구분할 수 있다.
5. dirty 데이터에서 tidy 원칙을 어기는 지점을 찾아 교정 방향을 말할 수 있다.

## 핵심 개념

### 1. 분석용 표는 보기 좋은 보고서와 다르다

엑셀에서 사람에게 보여주기 좋은 표는 제목이 크고, 셀이 병합되어 있고, 중간중간 소계와 설명이 들어가는 경우가 많다. 이런 표는 회의 자료로는 읽기 편하지만, 피벗테이블이나 AI 분석, 파이썬, BI 도구로 넘기면 오히려 걸림돌이 된다.

분석용 표는 장식보다 규칙이 중요하다. 이번 차시에서 사용할 카페 매출 데이터는 주문 한 건이 한 행이다. `date`, `store`, `item`, `qty`, `amount`처럼 각 열은 주문을 설명하는 한 가지 속성만 담는다. 이 구조가 유지되어야 필터, 정렬, 피벗, 집계가 안정적으로 작동한다.

### 2. tidy 데이터의 세 가지 규칙

tidy 데이터는 분석하기 좋은 표의 기본형이다. 입문 단계에서는 다음 세 문장으로 기억하면 충분하다.

| 규칙 | 뜻 | 엑셀에서 깨지는 예 |
|---|---|---|
| 한 행은 한 관찰이다 | 한 줄이 주문 한 건, 고객 한 명, 직원 한 명처럼 같은 단위여야 한다 | 중간에 월별 소계 행이 끼어 있음 |
| 한 열은 한 변수다 | 한 열에는 날짜, 지점, 상품명, 수량처럼 하나의 의미만 있어야 한다 | `상품/수량`처럼 한 열에 두 의미가 섞임 |
| 한 셀은 한 값이다 | 셀 하나에는 하나의 값만 들어가야 한다 | `4,500원`처럼 숫자와 단위가 섞임, `강남점/카드`처럼 값 두 개가 한 셀에 있음 |

병합셀은 이 원칙을 자주 무너뜨린다. 사람 눈에는 위아래가 같은 그룹처럼 보이지만, 컴퓨터는 병합된 첫 셀만 값이 있고 나머지 셀은 빈 칸으로 처리하는 경우가 많다. 그래서 분석용 원본 표에는 병합셀을 쓰지 않는 것이 원칙이다.

### 3. clean 데이터와 dirty 데이터 비교

이번 차시의 clean 데이터는 분석용 표의 기준 예시다. dirty 데이터는 같은 카페 매출 데이터에 실무에서 자주 나오는 문제를 일부러 심어 둔 파일이다.

clean 데이터의 컬럼은 다음과 같다.

| 컬럼 | 의미 |
|---|---|
| `order_id` | 주문번호 |
| `date` | 주문일 |
| `weekday` | 요일 |
| `hour` | 주문 시간대 |
| `store` | 지점 |
| `category` | 상품 대분류 |
| `item` | 상품명 |
| `unit_price` | 단가 |
| `qty` | 수량 |
| `amount` | 주문금액 |
| `payment` | 결제수단 |
| `member` | 회원 여부 |

dirty 데이터에서 확인한 대표 문제는 다음과 같다.

| 문제 유형 | 실제 예 | 왜 문제인가 |
|---|---|---|
| 주문번호 중복 | 같은 `order_id`가 다시 등장 | 주문 건수가 부풀 수 있음 |
| 결측 | `payment`, `member`가 비어 있음 | 결제수단별, 회원 여부별 집계가 흔들림 |
| 날짜 형식 혼합 | `2025-03-23`, `2025/03/23`, `03-23-2025` | 월별, 요일별 분석 전 날짜 인식이 불안정함 |
| 숫자와 문자 혼합 | `unit_price`에 `4,500원` 형태 포함 | 합계와 평균 계산 전에 숫자로 바꿔야 함 |
| 범주 표기 혼재 | `강남점`과 `강남 점`, `아메리카노`와 `Americano` | 같은 대상을 다른 그룹으로 집계함 |
| 앞뒤 공백 | ` 커피 `, ` 음료 ` | 눈에는 같아 보여도 컴퓨터는 다른 값으로 볼 수 있음 |
| 이상치 | `qty`가 999 | 평균과 총합을 크게 왜곡함 |
| 불가능한 값 | `amount`가 음수 | 매출 데이터의 기본 규칙과 맞지 않음 |

### 4. 척도는 분석 방법을 고르는 기준이다

데이터 분석에서 컬럼을 볼 때는 값의 모양뿐 아니라 “이 값으로 무엇을 할 수 있는가”를 생각해야 한다. 이것이 척도 구분이다.

| 척도 | 설명 | 가능한 작업 | 예 |
|---|---|---|---|
| 범주형 | 이름표처럼 그룹을 구분한다 | 개수 세기, 그룹별 합계 | 지점, 상품명, 결제수단 |
| 순서형 | 순서가 있지만 간격이 숫자처럼 정확하지 않을 수 있다 | 정렬, 단계별 비교 | 만족도, 등급 |
| 수치형 | 더하기, 평균, 최솟값, 최댓값 계산이 가능하다 | 합계, 평균, 분포 확인 | 수량, 단가, 주문금액 |
| 날짜형 | 시간의 흐름을 나타낸다 | 일별, 월별, 요일별 추세 | 주문일 |
| 논리형 | 참과 거짓 또는 예와 아니오 | 비율 계산, 필터링 | 회원 여부 |
| 식별자 | 각 행을 구분하는 이름표다 | 중복 확인, 추적 | 주문번호 |

주의할 점은 숫자로 보인다고 모두 수치형은 아니라는 것이다. `order_id`는 숫자로 저장되어 있지만 평균 주문번호를 구해도 의미가 없다. 그러니 주문번호는 계산용 숫자가 아니라 식별자로 봐야 한다.

## 다음 차시와의 연결

S03에서는 피벗테이블로 지점별, 카테고리별, 요일별 매출 질문에 답한다. 피벗테이블이 제대로 작동하려면 이번 차시에서 다룬 tidy 구조가 먼저 필요하다. 같은 상품명이 여러 표기로 흩어져 있거나, 숫자 열에 문자가 섞여 있으면 피벗 결과가 맞아 보여도 실제로는 틀릴 수 있다.
