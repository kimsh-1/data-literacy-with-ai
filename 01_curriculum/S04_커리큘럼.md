# S04 실무 함수: VLOOKUP/XLOOKUP과 조건부 집계

## 차시 개요

- 대상: 엑셀은 사용하지만 데이터분석은 처음인 비개발자
- 선수 차시: S03
- 난이도: 2
- 시간: 50분
- 사용 데이터: `/mnt/d/ai-data-analysis-course/02_datasets/shop_orders/shop_orders_clean.xlsx`
- 사용 시트: `orders`, `customers`, `products`

이번 차시가 다루는 건 엑셀로 실무 데이터를 만질 때 거의 매번 부딪히는 문제다. 주문 목록에 들어 있는 정보는 주문번호, 고객번호, 상품번호, 수량, 할인율뿐이다. 고객이 어느 지역 사람인지, 등급이 무엇인지, 상품 단가가 얼마인지는 여기 없다. 실무 데이터는 이렇게 여러 표에 흩어져 있기 마련이라, 분석을 시작하려면 표부터 이어 붙여야 한다.

작업의 뼈대는 이렇다. `orders`를 중심 표로 삼아 `products`에서 단가를, `customers`에서 지역과 등급을 끌어온 다음 매출을 계산한다. 이어서 `SUMIF`, `COUNTIF`로 지역별 매출과 등급별 객단가를 구한다.

## 학습 목표

수강생은 이 차시가 끝나면 다음을 할 수 있어야 한다.

1. 주문 표의 상품번호와 고객번호를 기준으로 다른 표의 값을 가져올 수 있다.
2. `VLOOKUP`과 `XLOOKUP`의 차이를 설명할 수 있다.
3. 단가, 수량, 할인율을 이용해 주문별 매출을 계산할 수 있다.
4. `SUMIF`, `COUNTIF`로 조건에 맞는 합계와 건수를 계산할 수 있다.
5. 엑셀 함수 방식이 행이 많아질수록 관리하기 어려워지는 이유를 말할 수 있다.

## 데이터 구조

실제 파일 확인 결과는 다음과 같다.

- `orders`: 20,000행, 6열
  - 컬럼: `order_id`, `customer_id`, `product_id`, `order_date`, `qty`, `discount_rate`
- `customers`: 3,000행, 7열
  - 컬럼: `customer_id`, `name`, `gender`, `age`, `region`, `join_date`, `grade`
- `products`: 200행, 5열
  - 컬럼: `product_id`, `product_name`, `category`, `price`, `cost`

`orders`에는 매출 컬럼이 없다. 매출은 다음 공식으로 직접 계산한다.

```text
매출 = 단가 × 수량 × (1 - 할인율)
```

## 개념 설명

### 1. 주문 표는 영수증, 고객 표와 상품 표는 사전

`orders` 시트는 거래가 한 줄씩 쌓인 영수증 목록에 가깝다. 그런데 영수증에 적힌 건 고객번호와 상품번호뿐이다. 고객번호 `C000787`이 어느 지역 고객인지, 상품번호 `P00159`의 단가가 얼마인지 알려면 다른 표를 들여다봐야 한다.

여기서 `customers`와 `products`가 사전 노릇을 한다. 고객번호로 사전을 찾으면 지역과 등급이, 상품번호로 찾으면 상품명과 단가가 나온다. `VLOOKUP`과 `XLOOKUP`이 바로 이 사전 찾기를 대신해 주는 함수다.

### 2. VLOOKUP은 왼쪽 첫 열에서 찾아 오른쪽 값을 가져온다

`VLOOKUP`은 다음 구조로 읽는다.

```excel
=VLOOKUP(찾을값, 찾을표범위, 가져올열번호, 정확히찾기)
```

예를 들어 상품번호를 기준으로 단가를 가져오려면 다음처럼 쓴다.

```excel
=VLOOKUP(C2,products!$A:$E,4,FALSE)
```

뜻은 다음과 같다.

- `C2`: 현재 주문의 상품번호
- `products!$A:$E`: 상품 사전 범위
- `4`: 상품 사전에서 네 번째 열인 `price`
- `FALSE`: 비슷한 값이 아니라 정확히 같은 상품번호만 찾기

입문 단계에서는 마지막 인수 `FALSE`를 빠뜨리지 않는 게 중요하다. 상품번호나 고객번호처럼 정확히 맞아떨어져야 하는 키에 근사값 검색을 쓰면 엉뚱한 값이 딸려 올 수 있다.

### 3. XLOOKUP은 찾을 열과 가져올 열을 따로 지정한다

`XLOOKUP`은 최신 엑셀에서 쓸 수 있는 함수로, 구조가 한결 직관적이다.

```excel
=XLOOKUP(찾을값, 찾을열, 가져올열, 못찾을때값)
```

단가를 가져오는 공식은 다음과 같다.

```excel
=XLOOKUP(C2,products!$A:$A,products!$D:$D,"확인필요")
```

`VLOOKUP`은 표 범위에서 몇 번째 열을 가져올지 숫자로 세야 한다. `XLOOKUP`은 찾을 열과 가져올 열을 곧바로 지정하니, 열이 추가되거나 순서가 바뀌어도 공식이 덜 흔들린다.

### 4. SUMIF와 COUNTIF는 조건을 걸어 계산한다

분석 질문은 대개 “전체 매출은 얼마인가”에서 끝나지 않는다. 실무에서는 “지역별 매출은 얼마인가”, “등급별 객단가는 얼마인가”처럼 조건이 따라붙는다.

`SUMIF`는 조건에 맞는 행의 값을 합산한다.

```excel
=SUMIF(조건범위, 조건, 합계범위)
```

`COUNTIF`는 조건에 맞는 행의 개수를 센다.

```excel
=COUNTIF(조건범위, 조건)
```

등급별 객단가는 다음처럼 계산한다.

```text
등급별 객단가 = 등급별 매출 합계 ÷ 등급별 주문 건수
```

## 엑셀 비교와 한계

엑셀 함수는 눈앞의 표를 빠르게 잇고 요약하는 데 대단히 강하다. 2만 행 남짓한 주문 데이터라면 수강생이 공식이 흘러가는 과정을 직접 눈으로 좇을 수 있어 학습 효과도 좋다.

하지만 실무 데이터가 커지고 표가 늘어나면 다음 문제가 생긴다.

- 열 번호를 잘못 세면 `VLOOKUP` 결과가 조용히 틀릴 수 있다.
- 공식이 여러 열에 복사되면 어느 열이 원본이고 어느 열이 계산 결과인지 헷갈릴 수 있다.
- 파일이 무거워지고 다시 계산하는 시간이 길어진다.
- 같은 계산을 다음 달에도 반복하려면 사람이 다시 클릭하고 복사해야 한다.
- 연결해야 할 표가 세 개, 네 개로 늘어나면 공식 관리가 어려워진다.

이 차시에서는 엑셀 실무 함수로 분석의 기본 골격을 익힌다. 뒤이어 파이썬과 BI 도구를 배우고 나면, 같은 문제를 더 재현하기 쉽고 관리하기 편한 방식으로 풀 수 있다.

## 수업의 핵심 메시지

데이터분석은 필요한 정보를 한 표에 모으는 일에서 출발한다. 엑셀에서는 `VLOOKUP`이나 `XLOOKUP`으로 표를 잇고, `SUMIF`, `COUNTIF`로 조건이 붙은 질문에 답한다. 다만 함수가 늘수록 관리 비용도 불어난다. 이 한계를 몸으로 느끼는 순간이 다음 단계인 자동화 도구로 넘어가는 출발점이다.
