# S17. 시각화·상호작용·DAX 기초

## 차시 개요

- 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
- 선수 차시: S16 BI 모델링과 날짜테이블
- 난이도: 4
- 시간: 70분
- 사용 데이터: S16에서 저장한 Power BI 파일 또는 수업 홈의 데이터 받기 페이지에서 내려받은 쇼핑몰 주문 Excel 파일의 `orders`, `customers`, `products` 시트
- 사용 도구: Power BI Desktop

> 안내: Power BI Desktop은 Windows 전용입니다. Mac 수강생은 Looker Studio로 개념을 학습합니다. 전월대비 시간지능, 즉 날짜를 기준으로 전월이나 전년 같은 기간을 찾아 계산하는 기능은 Power BI 전용입니다. Looker에서는 기간비교 필드로 비슷하게 비교하고, 결과를 해석하는 방식으로 봅니다.

S16에서 만든 관계형 모델을 이제 실제로 눌러 보고 움직이는 대시보드로 바꿉니다. 관계형 모델은 주문표, 고객표, 상품표처럼 나뉘어 있는 표들이 공통 열로 연결된 구조입니다. 이 연결이 잘 되어 있으면, 고객 등급을 하나 고르는 순간 매출 카드와 월별 그래프, 카테고리 그래프가 함께 바뀝니다.

엑셀을 쓰던 감각으로 보면, 피벗테이블 여러 개와 슬라이서를 한 화면에 놓고 보는 것과 비슷합니다. 다만 Power BI에서는 계산식이 셀 하나에 들어가지 않습니다. 계산식은 모델 안에 `측정값`이라는 이름으로 저장됩니다. 그래서 총매출을 한 번 만들어 두면 카드에도 쓰고, 선그래프에도 쓰고, 표와 막대그래프에도 다시 꺼내 쓸 수 있습니다. 자주 쓰는 계산기를 책상 위에 하나 올려 두고 필요할 때마다 누르는 느낌으로 보면 됩니다.

이 차시에서는 슬라이서로 원하는 고객 등급을 골라 보고, 막대그래프에서 카테고리를 눌렀을 때 다른 시각화가 같이 바뀌는 과정을 확인합니다. 그다음 DAX 측정값으로 총매출, 매출비중, 전월대비 성장률을 만듭니다. DAX는 Power BI에서 계산식을 만드는 언어입니다. 엑셀 수식과 완전히 같지는 않지만, “무엇을 더할지”, “무엇으로 나눌지”, “어떤 조건에서 다시 계산할지”를 정하는 도구라고 보면 됩니다.

## 실제 데이터 확인

자료를 만들기 전에 pandas 3.0.3으로 수업 홈의 데이터 받기 페이지에서 내려받는 쇼핑몰 주문 파일을 열어 실제 구조를 확인했습니다.

| 시트 | 실제 행수 | 주요 컬럼 |
|---|---:|---|
| `orders` | 20,000 | `order_id`, `customer_id`, `product_id`, `order_date`, `qty`, `discount_rate` |
| `customers` | 3,000 | `customer_id`, `region`, `grade`, `join_date` |
| `products` | 200 | `product_id`, `category`, `price`, `cost` |

주문 기간은 2024년 1월 1일부터 2024년 12월 31일까지입니다. 열두 달이 빠지지 않고 모두 들어 있으므로 월별 매출 흐름을 볼 수 있습니다. `orders`를 `customers`, `products`와 조인했을 때 짝이 맞지 않는 고객이나 상품은 하나도 없었습니다(누락 0건). 여기서 조인은 표끼리 같은 ID를 기준으로 붙여 보는 일입니다. 사전에서 단어를 찾듯이, 주문표의 고객 ID를 고객표에서 찾아 고객 등급과 지역을 가져오는 방식입니다.

주문 테이블에는 매출 컬럼이 따로 없습니다. 그래서 매출은 이미 들어 있는 단가, 수량, 할인율을 사용해 직접 계산해야 합니다.

```text
매출 = 상품 단가 × 주문 수량 × (1 - 할인율)
```

검증 기준 전체 매출은 1,938,489,803원입니다. 월별 매출이 가장 큰 달은 2024년 12월 225,576,457원이며, 2024년 11월의 전월대비 성장률은 **40.43%**로 가장 큽니다. 이 값들은 나중에 여러분이 만든 측정값이 맞는지 확인하는 기준점입니다. 체중계를 맞출 때 기준 추를 올려 보듯이, 대시보드 숫자가 이 검증값과 맞는지 확인해 봅니다.

## 학습 목표

1. 슬라이서, 드릴다운, 시각화 간 상호작용의 역할을 설명할 수 있다.
2. Power BI 모델에서 DAX 측정값을 만들고 카드·차트·표에 재사용할 수 있다.
3. `SUMX`, `DIVIDE`, `CALCULATE`의 입문 수준 용도를 이해한다.
4. `날짜테이블`을 사용해 `SAMEPERIODLASTMONTH` 기반 전월대비 매출과 성장률을 계산할 수 있다.
5. 고객 등급 슬라이서와 월별 KPI를 포함한 시계열 대시보드를 만들고 결과를 해석할 수 있다.

## 꼭 알아야 할 개념

### 1. 슬라이서는 화면 위에 놓는 필터 버튼입니다

슬라이서는 원하는 조건만 골라 보게 해 주는 버튼입니다. 엑셀 피벗테이블의 슬라이서를 써본 적이 있다면 같은 감각으로 이해하면 됩니다. 예를 들어 `customers[grade]`를 슬라이서로 놓으면 VIP, 골드, 실버, 브론즈 중 원하는 등급만 눌러 볼 수 있습니다.

Power BI의 슬라이서는 한 시각화만 바꾸지 않습니다. 같은 모델에 연결된 여러 시각화를 동시에 바꿉니다. 고객 등급을 `VIP`로 선택하면 총매출 카드, 월별 선그래프, 카테고리 막대그래프가 모두 VIP 고객 기준으로 다시 계산됩니다.

여기서 중요한 점은 슬라이서가 단순히 화면을 가리는 기능이 아니라는 것입니다. 슬라이서는 “지금은 이 조건으로 다시 계산해 주세요”라고 Power BI에 알려 주는 역할을 합니다. 그래서 숫자가 바뀌는 것은 원본 데이터가 바뀌어서가 아니라, 보고 있는 조건이 바뀌었기 때문입니다.

### 2. 드릴다운은 큰 묶음에서 작은 항목으로 내려가는 기능입니다

드릴다운은 큰 분류를 먼저 보고, 그 안쪽의 세부 항목으로 내려가는 기능입니다. 엑셀 피벗에서 연도 옆의 더하기 버튼을 눌러 월을 펼치는 상황과 비슷합니다. 처음에는 큰 덩어리를 보고, 궁금한 부분만 더 자세히 들여다보는 방식입니다.

Power BI에서는 축에 여러 필드를 순서대로 올려 계층을 만듭니다. 예를 들어 막대그래프 축에 `products[category]`와 `products[product_name]`을 순서대로 넣으면 처음에는 카테고리별 매출이 보입니다. 드릴다운을 켜고 디지털 막대를 누르면 디지털 카테고리 안의 상품별 매출로 내려갑니다.

드릴다운은 “전체에서 어디가 큰가”를 본 뒤 “그 안에서 무엇이 큰가”를 이어서 볼 때 유용합니다. 큰 지도에서 도시를 고른 뒤, 그 도시 안의 동네를 확대해서 보는 것과 비슷합니다.

### 3. 상호작용은 차트끼리 서로 영향을 주는 방식입니다

Power BI에서는 한 차트를 클릭하면 다른 차트가 필터링되거나 강조됩니다. 예를 들어 카테고리 막대그래프에서 `디지털`을 누르면 월별 선그래프가 디지털 상품만의 월별 매출로 바뀝니다.

이 기능은 대시보드를 탐색할 때 매우 편리합니다. 카테고리 하나를 눌러서 그 카테고리의 월별 흐름, 총매출, 비중을 한 번에 볼 수 있기 때문입니다. 하지만 처음 배우는 사람에게는 “왜 숫자가 갑자기 바뀌었지?” 하는 혼란을 줄 수도 있습니다. 숫자가 틀린 것이 아니라, 어떤 차트를 클릭하면서 필터 조건이 생긴 것입니다.

그래서 대시보드를 만들 때는 어떤 차트가 어떤 차트를 필터링할지 의도적으로 정해야 합니다. 이번 차시에서는 `서식 > 상호 작용 편집`에서 필터, 강조, 없음이 어떻게 다른지 직접 눌러 봅니다. 필터는 선택한 항목만 남기는 방식이고, 강조는 선택한 항목을 더 진하게 보여주는 방식이며, 없음은 다른 시각화에 영향을 주지 않는 방식입니다.

### 4. DAX 측정값은 피벗 값 영역에 넣는 계산식입니다

엑셀에서는 셀에 `=SUM(...)` 같은 수식을 씁니다. 피벗테이블에서는 값 영역에 합계, 평균, 개수 같은 계산을 넣습니다. Power BI의 DAX 측정값은 이 두 가지를 합쳐 놓은 느낌입니다. 계산식을 모델 안에 한 번 저장해 두고, 필요한 시각화마다 꺼내 씁니다.

측정값은 원본 표에 새 열을 영구적으로 추가하는 것과 다릅니다. 화면에서 어떤 조건을 선택했는지에 따라 그때그때 다시 계산됩니다. 고객 등급 슬라이서에서 VIP를 고르면 VIP 기준 총매출이 되고, 전체를 선택하면 전체 총매출이 됩니다. 같은 계산식이지만, 보고 있는 조건에 따라 답이 달라지는 것입니다.

이번 차시의 기본 측정값은 아래와 같습니다.

```DAX
총매출 =
SUMX(
    orders,
    RELATED(products[price]) * orders[qty] * (1 - orders[discount_rate])
)
```

`SUMX`는 주문 행을 한 줄씩 보면서 매출을 계산한 뒤 모두 더합니다. 키를 잴 때 사람을 한 명씩 세우고 각각의 키를 확인하듯이, `SUMX`는 행을 하나씩 지나가며 계산합니다. 주문 테이블에 매출 컬럼이 없고 상품 단가가 `products` 테이블에 있기 때문에 `RELATED(products[price])`로 연결된 상품 단가를 가져옵니다.

여기서 `RELATED`는 연결된 표에서 필요한 값을 찾아오는 함수입니다. 주문표에는 상품 ID만 있고 단가는 상품표에 있으니, 상품 ID를 따라가서 단가를 찾아오는 것입니다.

### 5. 비율은 나눗셈보다 `DIVIDE`로 만듭니다

비율은 분자와 분모가 있어야 합니다. 카테고리별 매출비중을 예로 들면, 분자는 현재 카테고리의 매출이고 분모는 전체 카테고리의 매출입니다. 현재 카테고리 매출을 전체 매출로 나누면 “전체 중 이 카테고리가 얼마나 차지하는지”를 볼 수 있습니다.

```DAX
매출비중 =
DIVIDE(
    [총매출],
    CALCULATE([총매출], ALL(products[category]))
)
```

`DIVIDE`는 나눗셈을 안전하게 해 주는 함수입니다. 분모가 0이거나 비어 있으면 일반 나눗셈은 오류를 낼 수 있습니다. `DIVIDE`는 그런 상황을 더 안정적으로 처리합니다. 입문 단계에서는 DAX에서 비율을 만들 때 `/`보다 `DIVIDE`를 먼저 떠올리면 됩니다.

이 식에서 `CALCULATE`는 계산 조건을 바꾸는 역할을 합니다. `ALL(products[category])`는 카테고리 필터를 잠시 풀어 전체 카테고리 매출을 구하게 합니다. 그래서 현재 카테고리 매출을 전체 카테고리 매출과 비교할 수 있습니다.

### 6. `날짜테이블`은 시간 계산의 기준 달력입니다

주문 테이블의 `order_date`만 가지고도 월별 그래프는 그릴 수 있습니다. 하지만 전월대비, 전년동월대비, 누계처럼 시간을 비교하거나 쌓아 가는 계산은 날짜 기준이 정확해야 합니다. 이때 필요한 것이 `날짜테이블`입니다.

`날짜테이블`은 시간 계산의 기준 달력입니다. 엑셀 매출표 옆에 2024년 달력을 한 장 펼쳐 놓고, 주문일이 그 달력의 날짜에 하나씩 걸린다고 상상해 보세요. Power BI의 시간 지능 함수는 이 달력을 기준으로 “이 날짜의 전월은 어디인가”, “이 기간의 이전 기간은 어디인가”를 찾아갑니다.

주문이 없는 날짜도 달력에는 들어 있어야 합니다. 그래야 시간이 중간에 끊기지 않습니다. 매출이 없는 날을 아예 모르면, Power BI는 그날이 0인지, 데이터가 빠진 것인지 구분하기 어렵습니다.

이번 차시에서는 S16의 `날짜테이블`을 사용하되, 없을 경우 아래와 같이 2024년 전체 날짜를 가진 테이블을 만듭니다.

```DAX
날짜테이블 =
ADDCOLUMNS(
    CALENDAR(MIN(orders[order_date]), MAX(orders[order_date])),
    "연도", YEAR([Date]),
    "월번호", MONTH([Date]),
    "연월", FORMAT([Date], "yyyy-MM"),
    "연월정렬", YEAR([Date]) * 100 + MONTH([Date])
)
```

2024년은 윤년이므로 `날짜테이블`은 366행입니다. 2월 29일까지 들어 있어야 2024년 전체 날짜가 빠짐없이 들어간 것입니다.

### 7. 전월대비는 현재 필터 맥락이 있어야 해석됩니다

전월대비 성장률은 현재 보고 있는 달의 매출과 바로 전 달 매출을 비교합니다. 여기서 필터 맥락은 “지금 화면에서 어떤 조건으로 숫자를 보고 있는가”라는 뜻입니다. 월별 선그래프에서 2024년 12월 점을 보고 있다면, 현재 맥락은 2024년 12월입니다.

```DAX
전월매출 =
CALCULATE(
    [총매출],
    SAMEPERIODLASTMONTH('날짜테이블'[Date])
)

전월대비성장률 =
DIVIDE([총매출] - [전월매출], [전월매출])
```

여기서 열쇠가 되는 말은 “현재 보고 있는 달”입니다. 월별 선그래프라면 각 점마다 1월, 2월 하는 월 필터가 저절로 걸립니다. 그래서 각 달의 전월매출과 전월대비 성장률을 자연스럽게 계산할 수 있습니다.

반면 카드 하나만 덩그러니 놓아 두면 지금 어느 달을 말하는지가 모호해집니다. 카드에 전월대비 성장률을 보여줄 때는 월 슬라이서에서 한 달을 고르게 하거나, 표나 선그래프처럼 월별 맥락이 살아 있는 시각화에 넣는 편이 좋습니다. 숫자는 혼자 의미를 가지는 것이 아니라, 어떤 조건에서 계산된 숫자인지까지 같이 읽어야 합니다.

## 70분 수업 흐름

## 엑셀과 연결해서 설명할 지점

- 슬라이서는 피벗테이블 슬라이서와 비슷합니다. 다만 Power BI에서는 여러 차트를 동시에 바꿀 수 있습니다.
- 드릴다운은 피벗테이블에서 연도 아래 월, 카테고리 아래 상품을 펼치는 기능과 비슷합니다.
- DAX 측정값은 셀 수식이 아니라 모델에 저장하는 계산식입니다. 한 번 만들어 두면 여러 시각화에서 반복해서 씁니다.
- `SUMX`는 행마다 계산한 뒤 합계를 내는 방식입니다. 엑셀에서 보조열로 행별 매출을 계산하고 그 합계를 내는 과정을 한 번에 하는 느낌입니다.
- `날짜테이블`은 시간 계산을 위한 기준 달력입니다. 주문이 없는 날짜도 달력에는 있어야 합니다.
- 전월대비는 현재 보고 있는 달이라는 필터 맥락이 있어야 올바르게 읽힙니다. 어느 달을 보고 있는지 정해져야 “전월”도 정해집니다.
