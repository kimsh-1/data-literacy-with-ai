# S17. 시각화·상호작용·DAX 기초

## 차시 개요

- 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
- 선수 차시: S16 BI 모델링과 날짜테이블
- 난이도: 4
- 시간: 70분
- 사용 데이터: S16에서 저장한 Power BI 파일(`.pbix`) 또는 `02_datasets/shop_orders/shop_orders_clean.xlsx`의 `orders`, `customers`, `products` 시트
- 핵심 도구: Power BI Desktop

> 안내: Power BI Desktop은 Windows 전용이다. Mac 수강생은 Looker Studio로 개념을 학습한다. 전월대비 시간지능은 Power BI 전용이며, Looker는 기간비교 필드로 근사해 해석한다.

이번 차시는 S16에서 만든 관계형 모델을 실제로 움직이는 대시보드로 바꿔 보는 시간이다. 수강생은 슬라이서로 원하는 고객 등급을 골라 보고, 막대그래프에서 카테고리를 누르면 다른 시각화까지 함께 바뀌는 모습을 직접 확인한다. 그런 다음 DAX 측정값으로 총매출과 매출비중, 전월대비 성장률을 만든다.

엑셀 사용자의 감각으로 옮겨 보면, 피벗테이블 여러 개와 슬라이서를 한 화면에 늘어놓고 그 피벗 값의 계산식을 좀 더 정교하게 다듬는 수업이다. 차이가 있다면 Power BI에서는 계산식이 셀에 들어가지 않고 모델 안의 측정값으로 저장된다는 점이다. 덕분에 같은 측정값을 카드와 선그래프, 표, 막대그래프에서 몇 번이고 다시 꺼내 쓸 수 있다.

## 실제 데이터 확인

자료 작성 전 pandas 3.0.3으로 지정 파일을 열어 확인했다.

| 시트 | 실제 행수 | 주요 컬럼 |
|---|---:|---|
| `orders` | 20,000 | `order_id`, `customer_id`, `product_id`, `order_date`, `qty`, `discount_rate` |
| `customers` | 3,000 | `customer_id`, `region`, `grade`, `join_date` |
| `products` | 200 | `product_id`, `category`, `price`, `cost` |

주문 기간은 2024년 1월 1일부터 2024년 12월 31일까지로, 열두 달이 빠짐없이 들어 있다. `orders`를 `customers`, `products`와 조인했을 때 짝이 맞지 않는 고객이나 상품은 하나도 없었다(누락 0건). 주문 테이블에는 매출 컬럼이 따로 없어서, 매출은 아래 공식으로 직접 계산해야 한다.

```text
매출 = 상품 단가 × 주문 수량 × (1 - 할인율)
```

검증 기준 전체 매출은 1,938,489,803원이다. 월별 매출이 가장 큰 달은 2024년 12월 225,576,457원이며, 2024년 11월의 전월대비 성장률은 40.43%로 가장 크다.

## 학습 목표

1. 슬라이서, 드릴다운, 시각화 간 상호작용의 역할을 설명할 수 있다.
2. Power BI 모델에서 DAX 측정값을 만들고 카드·차트·표에 재사용할 수 있다.
3. `SUMX`, `DIVIDE`, `CALCULATE`의 입문 수준 용도를 이해한다.
4. `날짜테이블`을 사용해 `SAMEPERIODLASTMONTH` 기반 전월대비 매출과 성장률을 계산할 수 있다.
5. 고객 등급 슬라이서와 월별 KPI를 포함한 시계열 대시보드를 만들고 결과를 해석할 수 있다.

## 핵심 개념

### 1. 슬라이서는 화면 위에 놓는 필터 버튼이다

엑셀 피벗테이블의 슬라이서를 써본 사람이라면 Power BI의 슬라이서도 같은 감각으로 이해할 수 있다. 예를 들어 `customers[grade]`를 슬라이서로 놓으면 VIP, 골드, 실버, 브론즈 중 원하는 등급만 눌러 볼 수 있다.

다만 Power BI의 슬라이서는 한 시각화만 바꾸는 것이 아니라 같은 모델에 연결된 여러 시각화를 동시에 바꾼다. 고객 등급을 `VIP`로 선택하면 총매출 카드, 월별 선그래프, 카테고리 막대그래프가 모두 VIP 고객 기준으로 다시 계산된다.

### 2. 드릴다운은 큰 묶음에서 작은 항목으로 내려가는 기능이다

드릴다운은 엑셀 피벗에서 연도 옆의 더하기 버튼을 눌러 월을 펼치는 상황과 비슷하다. Power BI에서는 축에 여러 필드를 순서대로 올려 계층을 만든다.

예를 들어 막대그래프 축에 `products[category]`와 `products[product_name]`을 순서대로 넣으면 처음에는 카테고리별 매출이 보인다. 드릴다운을 켜고 디지털 막대를 누르면 디지털 카테고리 안의 상품별 매출로 내려간다.

드릴다운은 "전체에서 어디가 큰가"를 본 뒤 "그 안에서 무엇이 큰가"를 이어서 볼 때 유용하다.

### 3. 상호작용은 차트끼리 서로 영향을 주는 방식이다

Power BI에서는 한 차트를 클릭하면 다른 차트가 필터링되거나 강조된다. 예를 들어 카테고리 막대그래프에서 `디지털`을 누르면 월별 선그래프가 디지털 상품만의 월별 매출로 바뀐다.

편리한 기능이지만, 처음 배우는 사람에게는 "왜 숫자가 갑자기 바뀌었지" 하는 혼란을 줄 수도 있다. 그래서 대시보드를 짤 때는 어떤 차트가 어떤 차트를 필터링할지 처음부터 의도적으로 정해 두어야 한다. 이번 차시에서는 `서식 > 상호 작용 편집`에서 필터, 강조, 없음이 어떻게 다른지 직접 눌러 본다.

### 4. DAX 측정값은 피벗 값 영역에 넣는 계산식이다

엑셀에서는 셀에 `=SUM(...)` 같은 수식을 쓰고, 피벗테이블에서는 값 영역에 합계나 평균, 개수 같은 계산을 넣는다. Power BI의 DAX 측정값은 이 둘을 합쳐 놓은 느낌이다. 계산식을 모델 안에 한 번 저장해 두고, 필요한 시각화마다 꺼내 쓴다.

이번 차시의 기본 측정값은 아래와 같다.

```DAX
총매출 =
SUMX(
    orders,
    RELATED(products[price]) * orders[qty] * (1 - orders[discount_rate])
)
```

`SUMX`는 주문 행을 한 줄씩 보면서 매출을 계산한 뒤 모두 더한다. 주문 테이블에 매출 컬럼이 없고 상품 단가가 `products` 테이블에 있기 때문에 `RELATED(products[price])`로 연결된 상품 단가를 가져온다.

### 5. 비율은 나눗셈보다 `DIVIDE`로 만든다

비율은 분자와 분모가 있어야 한다. 예를 들어 카테고리별 매출비중은 현재 카테고리 매출을 전체 카테고리 매출로 나눈 값이다.

```DAX
매출비중 =
DIVIDE(
    [총매출],
    CALCULATE([총매출], ALL(products[category]))
)
```

`DIVIDE`는 분모가 0이거나 비어 있을 때 오류를 줄여 준다. 입문 단계에서는 DAX에서 비율을 만들 때 `/`보다 `DIVIDE`를 우선 사용한다고 기억하면 된다.

### 6. `날짜테이블`은 시간 계산의 기준 달력이다

주문 테이블의 `order_date`만 가지고도 월별 그래프는 얼마든지 그릴 수 있다. 문제는 전월대비, 전년동월대비, 누계처럼 시간을 다루는 계산이다. 이런 계산을 흔들림 없이 하려면 별도의 날짜 테이블이 있어야 한다.

엑셀 매출표 옆에 2024년 달력을 한 장 펼쳐 놓고 주문일이 그 달력의 날짜에 하나씩 걸린다고 상상하면 쉽다. Power BI의 시간 지능 함수는 이 달력을 기준으로 삼아 "이 날짜의 전월은 어디인가"를 되짚어 찾는다.

이번 차시에서는 S16의 `날짜테이블`을 사용하되, 없을 경우 아래와 같이 2024년 전체 날짜를 가진 테이블을 만든다.

```DAX
날짜테이블 =
ADDCOLUMNS(
    CALENDAR(MIN(orders[order_date]), MAX(orders[order_date])),
    "연도", YEAR([Date]),
    "월번호", MONTH([Date]),
    "연월", FORMAT([Date], "yyyy-MM"),
    "연월정렬", YEAR([Date]) * 100 + MONTH([Date])
)
```

2024년은 윤년이므로 `날짜테이블`은 366행이다.

### 7. 전월대비는 현재 필터 맥락이 있어야 해석된다

전월대비 성장률은 현재 보고 있는 달의 매출과 바로 전 달 매출을 비교한다.

```DAX
전월매출 =
CALCULATE(
    [총매출],
    SAMEPERIODLASTMONTH('날짜테이블'[Date])
)

전월대비성장률 =
DIVIDE([총매출] - [전월매출], [전월매출])
```

여기서 열쇠가 되는 말은 "현재 보고 있는 달"이다. 월별 선그래프라면 각 점마다 1월, 2월 하는 월 필터가 저절로 걸린다. 반면 카드 하나만 덩그러니 놓아 두면 지금 어느 달을 말하는지가 모호해진다. 그래서 카드로 전월대비 성장률을 보여줄 때는 월 슬라이서에서 한 달을 고르게 하거나, 표나 선그래프처럼 월별 맥락이 살아 있는 시각화에 넣는 편이 낫다.

## 엑셀과 연결해서 설명할 지점

- 슬라이서는 피벗테이블 슬라이서와 비슷하지만, 여러 차트를 동시에 바꿀 수 있다.
- 드릴다운은 피벗테이블에서 연도 아래 월, 카테고리 아래 상품을 펼치는 기능과 비슷하다.
- DAX 측정값은 셀 수식이 아니라 모델에 저장하는 계산식이다.
- `SUMX`는 행마다 계산한 뒤 합계를 내는 방식으로, 엑셀의 보조열 계산과 합계를 한 번에 하는 느낌이다.
- `날짜테이블`은 시간 계산을 위한 기준 달력이다. 주문이 없는 날짜도 달력에는 있어야 한다.
- 전월대비는 현재 보고 있는 달이라는 필터 맥락이 있어야 올바르게 읽힌다.
