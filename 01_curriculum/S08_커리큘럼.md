# S08. 좋은 분석 프롬프트 설계

## 차시 개요

- 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
- 선수 차시: S07
- 난이도: 3
- 시간: 50분
- 주 데이터: `/mnt/d/ai-data-analysis-course/02_datasets/survey_responses/survey_responses_clean.csv`
- 보조 데이터: `/mnt/d/ai-data-analysis-course/02_datasets/survey_responses/survey_responses_dirty.csv`
- 선택 데이터: `/mnt/d/ai-data-analysis-course/02_datasets/real_korean/movie_box/dacon_movies_train.csv`
- 확인된 데이터 규모: 설문 clean 1,500행 × 8열, 설문 dirty 1,510행 × 8열, 영화 train 600행 × 12열

이번 차시의 핵심은 AI에게 "분석해줘"라고 던지는 대신 제대로 된 분석 요청서를 쓰는 데 있다. 피벗테이블을 만들 때 행과 열과 값을 지정하듯, AI에게도 맥락과 질문과 출력형식을 짚어 줘야 한다. 이 세 가지가 빠지면 그럴듯한 문장이야 나오겠지만, 업무에 바로 쓸 만한 분석을 안정적으로 얻기는 어렵다.

## 학습 목표

수강생은 이 차시가 끝나면 다음을 할 수 있다.

1. 좋은 분석 프롬프트의 세 요소인 맥락, 질문, 출력형식을 설명할 수 있다.
2. 한 번에 큰 답을 요구하지 않고 코드북 설계, 행별 분류, 요약, 검증 순서로 프롬프트를 심화할 수 있다.
3. 설문 주관식 응답을 주제와 감성으로 분류하도록 AI에게 요청할 수 있다.
4. 같은 데이터에 대해 프롬프트 3개를 비교하고 어떤 프롬프트가 더 재현 가능한지 판단할 수 있다.

## 데이터 이해

사용하는 기본 파일은 `survey_responses_clean.csv`이다. 같은 내용의 xlsx 파일도 있지만, AI 파일 업로드와 파이썬 검증에는 CSV를 사용한다.

| 컬럼 | 의미 | 이번 차시에서 쓰는 방식 |
|---|---|---|
| `respondent_id` | 응답자 ID | AI 결과와 원본을 대조하는 기준 |
| `age_group` | 연령대 | 요약 해석의 보조 정보 |
| `gender` | 성별 | 이번 차시에서는 주요 분석 기준으로 쓰지 않음 |
| `nps` | 추천 의향 점수, 0-10 | 비추천, 중립, 추천 그룹 구분 |
| `satisfaction` | 만족도, 1-5 | 감성 해석의 참고값 |
| `channel` | 유입 또는 응답 채널 | 채널별 차이 해석의 보조 정보 |
| `free_text` | 주관식 응답 | 주제분류와 감성요약의 핵심 |
| `submitted_at` | 응답 제출 시각 | 이번 차시에서는 사용하지 않음 |

보조 파일인 `survey_responses_dirty.csv`는 1,510행이다. 실제 확인 결과 중복 응답자 ID 10건, NPS 결측 60건, 주관식 결측 22건, 만족도 문자 혼입 75건, `20s`와 `이십대` 같은 20대 표기 혼재 90건이 있다. 이번 차시의 본 실습은 clean 파일로 진행하되, dirty 파일은 "프롬프트에 데이터 상태를 알려야 하는 이유"를 설명할 때 사용한다.

선택 데이터인 `dacon_movies_train.csv`는 600행 × 12열이며 `genre` 컬럼이 이미 들어 있다. 시간이 남으면 "이미 사람이 만든 범주가 있는 데이터"와 "AI에게 범주를 만들게 하는 설문 데이터"를 비교하는 확장 예제로만 사용한다.

## 핵심 개념

### 1. 프롬프트는 AI에게 보내는 분석 요청서다

입문자가 흔히 쓰는 프롬프트는 "이 데이터 분석해줘"다. 피벗테이블을 만들면서 행과 열과 값 영역을 죄다 비워 둔 것과 비슷하다. AI가 뭔가 내놓기는 하겠지만, 그게 우리가 원한 질문에 답한다는 보장은 없다.

분석 프롬프트에는 최소한 세 가지가 들어가야 한다.

| 요소 | 뜻 | 설문 분석 예시 |
|---|---|---|
| 맥락 | 이 데이터가 무엇이고 왜 보는지 | 온라인 쇼핑 고객 설문이며, 주관식 불만과 칭찬을 운영 개선에 쓰려 한다 |
| 질문 | 무엇을 알고 싶은지 | `free_text`를 읽고 주제와 감성을 분류하라 |
| 출력형식 | 어떤 모양으로 돌려받을지 | `respondent_id`, 주제, 감성, 근거문구, 신뢰도 표로 작성하라 |

좋은 프롬프트는 AI를 똑똑하게 만드는 주문이 아니라, 분석 기준을 좁히고 결과를 검토 가능한 표로 받아 내기 위한 업무 지시서다.

### 2. 맥락은 AI가 기준을 세우는 재료다

같은 "배송이 늦었다"라는 문장도 분석 목적에 따라 뜻이 달라진다. 고객경험 개선 보고서라면 배송 프로세스 문제로 잡겠지만, 마케팅 보고서라면 구매 후 커뮤니케이션 문제로 볼 수도 있다.

그래서 프롬프트에는 다음 맥락을 적어야 한다.

- 데이터의 출처와 단위: 고객 설문 1행이 응답 1건임
- 주요 컬럼의 뜻: `nps`, `satisfaction`, `free_text`
- 분석 목적: 운영 개선 아이디어를 찾기 위함
- 제한 조건: 모르면 추측하지 말고 "판단 어려움"으로 표시
- 검증 기준: 결과에는 원문 근거와 응답자 ID를 포함

엑셀로 치면 맥락은 표 제목이나 컬럼 설명에 해당한다. 제목 없이 숫자만 늘어놓으면 그게 매출인지 수량인지 알 수 없는 것과 같다.

### 3. 질문은 한 번에 하나씩 좁혀야 한다

"고객 반응을 분석해줘"는 너무 넓다. 좋은 질문은 다음처럼 단계별로 좁아진다.

1. 이 설문 주관식을 분류할 주제 코드 6-8개를 먼저 제안해줘.
2. 제안한 코드북으로 표본 18행을 주제와 감성으로 분류해줘.
3. 분류 결과를 주제별 응답수, 평균 NPS, 대표 근거로 요약해줘.
4. 모호하거나 다시 검토해야 할 행을 따로 표시해줘.

순서가 중요하다. AI가 처음부터 전체 요약을 내놓으면 무슨 기준으로 분류했는지 되짚기 어렵다. 먼저 기준표를 만들고, 그 표로 행을 분류한 다음, 요약은 맨 마지막에 해야 한다.

### 4. 출력형식은 검증 가능하게 지정한다

AI가 줄글로만 답하면 읽기에는 편해도 검증하기가 까다롭다. 그래서 이번 차시에서는 표로 받는 것을 기본으로 삼는다.

권장 출력 컬럼은 다음과 같다.

| 출력 컬럼 | 이유 |
|---|---|
| `respondent_id` | 원본 행과 대조하기 위해 필요 |
| `topic` | 주관식 응답의 주요 주제 |
| `sentiment` | 긍정, 부정, 혼합, 중립 |
| `evidence` | 판단 근거가 된 원문 일부 |
| `confidence` | 높음, 중간, 낮음 |
| `needs_review` | 사람이 다시 봐야 할 행 표시 |

엑셀로 치면 출력형식은 피벗테이블 결과를 어디에 어떤 모양으로 놓을지 정하는 일이다. 표로 받아 두면 엑셀에 그대로 붙여 넣어 피벗테이블이나 필터, 정렬로 이어 갈 수 있다.

### 5. 범주분류와 감성분석은 "초안"으로 받아야 한다

주관식 응답은 숫자처럼 정답이 하나로 떨어지는 데이터가 아니다. "사용 중 오류가 있었지만 재접속 후 해결됐다"는 응답에는 앱 오류라는 부정 신호와 해결 경험이라는 긍정 신호가 뒤섞여 있다. 이런 문장을 억지로 긍정이나 부정 한쪽에만 밀어 넣으면 분석이 거칠어진다.

따라서 프롬프트에는 다음 조건을 넣는다.

- 주제는 하나만 고르되, 애매하면 `needs_review=예`로 표시한다.
- 감성은 긍정, 부정, 혼합, 중립 중 하나로 정한다.
- NPS와 주관식 문장이 어긋나면 혼합 또는 검토 필요로 표시한다.
- 각 판단에는 원문 근거를 붙인다.

AI가 잘하는 일은 텍스트를 빠르게 훑어 초안을 만들어 내는 것이다. 최종 보고까지 가려면 샘플 검토와 기준표 조정, 원본 대조가 반드시 뒤따라야 한다.

## 엑셀 기능 비교

| 하고 싶은 일 | 엑셀 중심 접근 | AI 프롬프트 접근 |
|---|---|---|
| NPS 구간 나누기 | `IF`, `IFS` 수식 | 기준을 알려주고 구간명을 붙이게 함 |
| 이미 있는 범주 집계 | 피벗테이블 | 요약표 요청 가능 |
| 주관식 의견 묶기 | 사람이 읽고 새 열에 직접 입력 | 주제 코드북을 만들고 행별 분류 요청 |
| 불만 원인 요약 | 필터 후 직접 읽기 | 부정 또는 혼합 응답의 대표 근거 요청 |
| 결과 검증 | 원본 행과 대조 | `respondent_id`, 근거문구, 검토 필요 열을 요구 |

AI가 엑셀을 대체하는 것은 아니다. AI는 주관식 텍스트를 읽어 초안을 뽑고, 엑셀은 그 초안을 다시 집계하고 검증하는 자리에 선다.

## 50분 수업 흐름

| 시간 | 내용 | 진행 방식 |
|---:|---|---|
| 0-5분 | 도입: "분석해줘"가 왜 부족한가 | 같은 데이터에 짧은 프롬프트와 구조화 프롬프트를 비교 |
| 5-12분 | 개념: 맥락, 질문, 출력형식 | 엑셀 피벗테이블의 행, 열, 값과 비교 |
| 12-24분 | 따라하기 1: 설문 표본 18행으로 주제분류 요청 | 파일 업로드 또는 표본 복사 붙여넣기 |
| 24-34분 | 따라하기 2: 단계적 심화 | 코드북 제안, 행별 분류, 요약, 검토 필요 행 요청 |
| 34-45분 | 직접 해보기: 프롬프트 3버전 A/B/C 비교 | 수강생이 직접 프롬프트 개선 |
| 45-50분 | 정리와 체크포인트 | 좋은 프롬프트의 세 요소와 검증 습관 확인 |

## 수업 중 강조할 원칙

1. AI에게 맡기기 전에 분석 목적을 먼저 정한다.
2. 프롬프트에는 맥락, 질문, 출력형식이 모두 들어가야 한다.
3. 큰 질문은 코드북 설계, 행별 분류, 요약, 검증 순서로 나눈다.
4. 주관식 분류 결과는 정답이 아니라 검토 가능한 초안이다.
5. AI 답변에는 원문 근거와 응답자 ID가 있어야 다시 확인할 수 있다.

## 차시 평가 기준

- 상: 같은 표본 또는 파일로 프롬프트 3개를 비교하고, 가장 좋은 프롬프트를 근거와 함께 설명하며, 주제·감성 분류표를 검토 가능한 형식으로 만든다.
- 중: 따라하기 프롬프트를 재현하고, 강사의 안내를 받아 출력형식과 근거 열을 포함한 결과를 얻는다.
- 하: "분석해줘" 수준의 프롬프트에서 벗어나지 못하거나, AI 결과를 원본과 대조하지 못한다.
