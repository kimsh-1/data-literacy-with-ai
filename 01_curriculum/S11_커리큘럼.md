## 꼭 잡아야 할 개념

### 1. pandas는 표를 다루는 파이썬 도구입니다

S10에서는 변수, 리스트, 딕셔너리, 조건문, 반복문을 배웠습니다. pandas는 그 문법을 표 데이터에 맞게 확장해 주는 도구입니다. 엑셀에서 셀과 열을 다루듯이, pandas에서는 행과 열이 있는 표를 코드로 다룹니다.

```python
import pandas as pd
```

여기서 `pd`는 pandas를 짧게 부르는 별명입니다. 매번 `pandas`라고 길게 쓰지 않고 `pd`라고 부르겠다는 약속입니다. 엑셀에서 `SUM`, `AVERAGE`, `FILTER` 같은 기능을 쓰듯, pandas에서는 `read_csv`, `head`, `describe`, `sort_values` 같은 기능을 씁니다.

처음에는 pandas를 어렵게 생각하지 않아도 됩니다. "엑셀에서 하던 표 확인과 필터를 파이썬 코드로 한다"라고 잡으면 충분합니다.

### 2. DataFrame은 시트, Series는 열입니다

CSV를 읽으면 pandas는 보통 DataFrame을 만듭니다. DataFrame은 행과 열이 있는 표입니다. 엑셀의 워크시트 한 장을 떠올리면 됩니다.

```python
df = pd.read_csv(path, encoding="utf-8-sig")
```

여기서 `df`는 보통 DataFrame을 담을 때 많이 쓰는 이름입니다. 이름은 바꿀 수 있지만, 입문 단계에서는 많은 예제가 `df`를 쓰기 때문에 그대로 따라가면 읽기 편합니다.

DataFrame에서 열 하나를 꺼내면 Series가 됩니다.

```python
amount_series = df["amount"]
```

엑셀로 비교하면 아래처럼 보면 됩니다.

| 엑셀 감각 | pandas 감각 |
|---|---|
| 워크시트 전체 | DataFrame |
| 열 하나 | Series |
| 행 번호 | 인덱스 |
| 열 제목 | 컬럼명 |
| 필터 조건 | 불리언 마스크 |

처음부터 DataFrame과 Series라는 이름을 완벽하게 외우려고 하지 않아도 됩니다. 먼저 "DataFrame은 표 전체", "Series는 열 하나"라는 감각을 잡아 보세요. 엑셀 파일을 열었을 때 화면 전체가 DataFrame이고, 그중 금액 열 하나만 손가락으로 짚으면 Series입니다.

### 3. CSV 읽기는 파일 선택과 인코딩을 확인하는 일입니다

CSV는 쉼표로 값을 구분해 저장한 텍스트 파일입니다. 엑셀로 열면 표처럼 보이지만, 실제로는 글자 파일입니다. 그래서 pandas로 읽을 때는 어떤 파일을 읽을지와 글자가 어떤 방식으로 저장되어 있는지를 확인해야 합니다. 이 글자 저장 방식을 인코딩이라고 부릅니다.

```python
df = pd.read_csv(path, encoding="utf-8-sig")
```

위 코드에서 `path`는 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일을 가리키는 이름이라고 보면 됩니다. `encoding="utf-8-sig"`는 한글이 깨지지 않도록 글자 저장 방식을 맞춰 읽으라는 뜻입니다.

이 과정에서 사용하는 CSV는 대부분 UTF-8로 정규화되어 있어 `encoding="utf-8-sig"`나 기본값으로 읽힙니다. 다만 한국 공공데이터나 오래된 업무 파일에는 cp949 인코딩이 남아 있기도 합니다. 이런 파일은 글자가 깨지거나 `UnicodeDecodeError`를 냅니다. 오류 이름이 낯설어도 뜻은 단순합니다. "글자 저장 방식이 맞지 않아 읽지 못했다"는 말입니다.

S11에서는 인코딩을 깊게 해결하지 않습니다. 오늘은 "CSV는 표처럼 보여도 글자 파일이고, 글자 저장 방식이 맞아야 열린다"는 정도를 기억하면 됩니다. cp949 원본을 직접 여는 실습은 S12에서 합니다.

### 4. head, info, describe는 데이터 첫 점검 도구입니다

데이터를 읽자마자 분석을 시작하지 않습니다. 먼저 어떤 표인지 확인합니다. 택배 상자를 받으면 바로 쓰기 전에 주소와 내용물을 확인하듯이, 데이터도 먼저 크기와 열 이름과 값의 상태를 봐야 합니다.

| 확인 도구 | 하는 일 | 엑셀 비교 |
|---|---|---|
| `df.head()` | 앞쪽 몇 행 보기 | 시트 맨 위 훑어보기 |
| `df.info()` | 행수, 컬럼, 결측, 타입 보기 | 열 목록과 데이터 형식 확인 |
| `df.describe()` | 숫자 열의 기초 통계 보기 | 평균, 최소, 최대 빠른 확인 |
| `df.shape` | 행과 열 개수 보기 | 시트 크기 확인 |
| `df.columns` | 컬럼명 보기 | 머리글 행 확인 |

`df.head()`는 앞쪽 몇 줄을 보여 줍니다. 엑셀 파일을 열고 맨 위 행들을 눈으로 훑는 행동과 같습니다.

`df.info()`는 행수, 컬럼, 비어 있는 값, 데이터 타입을 보여 줍니다. 여기서 데이터 타입은 숫자인지, 글자인지 같은 형식을 말합니다.

`df.describe()`는 숫자 열의 평균, 최소, 최대 같은 기초 통계를 보여 줍니다. 오늘은 이 결과를 보고 금액이 어느 정도 범위 안에 있는지 빠르게 확인합니다.

이번 데이터의 `info()`를 보면 10,424행이 모두 채워져 있고 `amount`, `qty`, `unit_price`가 숫자형으로 읽힙니다. `date`만 문자열입니다. 날짜처럼 생겼지만 아직 컴퓨터에게는 글자입니다. 이 문자열 날짜를 실제 날짜 타입으로 바꾸는 일은 다음 차시 정제 주제로 미룹니다.

### 5. 불리언 마스크는 True/False로 된 필터 조건입니다

엑셀 필터에서 지점 열을 열고 `강남점`만 체크한다고 생각해 볼까요? pandas에서는 먼저 행마다 조건을 검사합니다.

```python
gangnam_mask = df["store"] == "강남점"
```

이 코드는 각 행의 `store` 값이 `강남점`인지 비교합니다. 맞으면 True, 아니면 False가 됩니다. 결과는 True 또는 False로 된 Series입니다. 이 True/False 열을 불리언 마스크라고 부릅니다.

불리언은 True와 False처럼 참과 거짓을 나타내는 값입니다. 마스크는 필요한 부분만 보이게 가리는 도구라고 생각하면 됩니다. 얼굴팩 마스크처럼 필요한 곳만 덮는 것이 아니라, 데이터에서는 조건에 맞는 행만 통과시키는 표시라고 보면 됩니다.

```python
gangnam = df.loc[gangnam_mask]
```

`df.loc[gangnam_mask]`는 마스크가 True인 행만 남깁니다. 엑셀 필터에서 조건에 맞는 행만 보이게 한 상태와 같습니다. 여기서 `gangnam`에는 지점이 강남점인 주문만 들어갑니다.

조건이 여러 개일 때는 괄호와 `&`를 씁니다.

```python
mask = (
    (df["store"] == "강남점")
    & (df["category"] == "커피")
    & (df["amount"] >= 10000)
)
```

이 조건은 지점이 `강남점`이고, 카테고리가 `커피`이고, 금액이 10,000원 이상인 주문만 찾습니다. 세 조건을 모두 만족해야 하므로 `&`를 씁니다. 엑셀에서 필터를 여러 열에 동시에 거는 상황과 같습니다.

초보자가 자주 틀리는 부분은 두 가지입니다. 하나는 파이썬의 `and`가 아니라 pandas 조건 결합에는 `&`를 쓴다는 점입니다. 또 하나는 각 조건을 괄호로 감싼다는 점입니다. 괄호는 조건 하나하나를 묶어 주는 울타리입니다. 울타리가 없으면 pandas가 어느 조건부터 계산해야 하는지 헷갈릴 수 있습니다.

### 6. sort_values는 정렬입니다

엑셀에서 금액 열을 기준으로 내림차순 정렬하듯, pandas에서는 `sort_values`를 씁니다. 정렬은 표의 순서를 바꾸는 일입니다. 값이 큰 주문부터 보고 싶다면 금액 열을 기준으로 정렬하면 됩니다.

```python
gangnam_sorted = gangnam.sort_values("amount", ascending=False)
```

`ascending=False`는 큰 값이 위로 오게 하라는 뜻입니다. `ascending`은 오름차순이라는 뜻인데, 여기에 False를 주었으니 오름차순이 아니라 내림차순으로 보겠다는 말입니다.

같은 금액이 많을 때는 날짜나 주문번호를 보조 기준으로 추가할 수 있습니다.

```python
gangnam.sort_values(
    ["amount", "date", "order_id"],
    ascending=[False, True, True],
)
```

이 코드는 먼저 `amount`를 큰 금액부터 봅니다. 금액이 같으면 `date`를 빠른 날짜부터 보고, 날짜도 같으면 `order_id`를 작은 번호부터 봅니다. 처음에는 정렬 기준 열 이름과 오름차순/내림차순만 이해하면 됩니다.

### 7. loc는 이름 기준, iloc는 위치 기준입니다

pandas에서 행과 열을 고를 때 자주 보는 도구가 `.loc`와 `.iloc`입니다. 둘 다 데이터를 고르는 도구이지만 기준이 다릅니다.

| 도구 | 기준 | 예시 |
|---|---|---|
| `.loc` | 인덱스 라벨과 컬럼명 | `df.loc[0, "store"]` |
| `.iloc` | 숫자 위치 | `df.iloc[0, 4]` |

이번 데이터에서 첫 행의 `store`는 두 방식으로 모두 꺼낼 수 있습니다.

```python
df.loc[0, "store"]
df.iloc[0, 4]
```

둘 다 `홍대점`을 반환합니다. 하지만 의미는 다릅니다. `.loc`는 "0번 라벨 행의 store 열"입니다. 컬럼명을 직접 보고 고르는 방식입니다. `.iloc`는 "0번째 행, 4번째 열"입니다. 위치 번호로 고르는 방식입니다.

엑셀로 비유하면 `.loc`는 "store라는 제목의 열을 찾아서 보세요"에 가깝고, `.iloc`는 "왼쪽에서 5번째 열을 보세요"에 가깝습니다. 입문 단계에서는 컬럼명을 직접 쓰는 `.loc`가 읽기 쉽습니다. 위치 번호는 열 순서가 바뀌면 헷갈릴 수 있으니, 처음에는 이름으로 고르는 습관을 들이면 좋습니다.

## 엑셀과 연결해서 설명할 지점

- `df = pd.read_csv(...)`는 엑셀에서 CSV를 여는 것과 비슷합니다. pandas가 파일을 읽어서 표 형태로 가져옵니다.
- `df.head()`는 시트 맨 위 몇 줄을 눈으로 확인하는 행동입니다. 데이터가 제대로 열렸는지 먼저 보는 습관입니다.
- `df.info()`는 열 이름, 빈칸 여부, 데이터 형식을 한 번에 확인하는 점검표입니다. 분석 전에 표의 건강 상태를 보는 과정입니다.
- `df.describe()`는 숫자 열의 평균, 최소, 최대를 빠르게 보는 요약 통계입니다. 금액이 말이 되는 범위인지 확인할 수 있습니다.
- `df["store"] == "강남점"`은 엑셀 필터에서 `강남점`만 체크하기 전, 각 행이 조건에 맞는지 True/False로 표시한 것입니다.
- `df.loc[mask]`는 True인 행만 남기는 필터 결과입니다. 엑셀 필터를 적용한 뒤 보이는 행만 남긴 상태와 같습니다.
- `sort_values("amount", ascending=False)`는 금액 열을 큰 값부터 정렬하는 것입니다. 매출 금액이 큰 주문을 위에서부터 보고 싶을 때 씁니다.

- "엑셀 필터에서 조건을 체크하는 일은 pandas에서 True/False 마스크를 만드는 일과 같다."
- "`df.loc[mask]`는 조건이 True인 행만 남긴다."
- "금액이 큰 순서로 보려면 `sort_values`에 정렬할 열과 내림차순 옵션을 준다."
