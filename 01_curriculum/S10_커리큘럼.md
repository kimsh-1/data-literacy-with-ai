이 차시에서는 파이썬을 전부 배우려 하지 않습니다. 다음 차시부터 pandas를 사용할 때 문법이 낯설어서 멈추지 않도록, 최소한의 언어 감각을 먼저 익혀 봅니다. pandas는 표 데이터를 다루는 파이썬 도구입니다. 엑셀에서 표를 필터링하고 합계를 내듯이, 파이썬에서는 pandas가 그런 일을 더 큰 데이터에도 처리해 줍니다.

수강생은 Colab 노트북을 열어 셀을 실행하고, 변수에 값을 담고, 리스트와 딕셔너리로 여러 값을 표현해 봅니다. 조건문과 반복문으로는 원하는 행을 고르고 여러 값을 하나씩 계산합니다. 끝에서는 "다음 차시 pandas의 필터는 오늘 배운 조건문을 열 전체에 한 번에 적용하는 것"이라는 연결고리를 잡아 봅니다.

## 학습 목표

1. Colab 노트북을 만들고 코드 셀을 실행할 수 있다.
2. 변수, 자료형, 리스트, 딕셔너리의 역할을 엑셀 표와 비교해 설명할 수 있다.
3. `if` 조건문으로 원하는 경우만 골라낼 수 있다.
4. `for` 반복문으로 여러 값을 하나씩 처리하고 합계와 평균을 계산할 수 있다.
5. 함수는 "자주 쓰는 절차에 이름을 붙이는 것"이라는 수준으로 이해한다.

## 꼭 잡아야 할 개념

### 1. Colab은 브라우저에서 쓰는 파이썬 작업장입니다

Colab은 구글 계정으로 사용하는 온라인 노트북입니다. 설치부터 복잡하게 시작하지 않고, 브라우저에서 파이썬 코드를 실행할 수 있는 작업장이라고 보면 됩니다.

엑셀 파일이 셀 격자로 되어 있다면, Colab 노트북은 코드 셀과 텍스트 셀로 되어 있습니다. 코드 셀에는 파이썬 명령을 쓰고, 텍스트 셀에는 설명을 적습니다. 그래서 분석 과정을 한 파일 안에 "설명 + 실행 코드 + 결과" 형태로 남길 수 있습니다.

| 엑셀에서의 감각 | Colab에서의 감각 |
|---|---|
| 셀에 수식 입력 | 코드 셀에 파이썬 코드 입력 |
| Enter로 셀 이동 | `Shift + Enter`로 코드 실행 후 다음 셀 이동 |
| 워크시트 저장 | 구글 드라이브에 노트북 저장 |
| 수식 결과가 셀에 표시 | 코드 실행 결과가 셀 아래에 표시 |
| 잘못된 수식은 오류 표시 | 잘못된 코드는 빨간 오류 메시지 표시 |

Colab에서 중요한 점은 코드 셀을 위에서 아래로 실행한다는 것입니다. 위 셀에서 만든 변수는 아래 셀에서 다시 쓸 수 있습니다. 반대로 위 셀을 실행하지 않았거나 런타임을 재시작했다면, 아래 셀에서 그 변수를 찾지 못합니다.

여기서 런타임은 파이썬 코드가 실제로 돌아가는 실행 공간입니다. 칠판에 적어 둔 내용이 지워지면 다시 설명해야 하듯이, 런타임이 재시작되면 이전에 만들어 둔 변수도 다시 실행해서 만들어야 합니다.

### 2. 변수는 값에 붙이는 이름입니다

엑셀에서는 `B2` 같은 셀 주소가 값을 가리킵니다. 파이썬에서는 사람이 읽을 수 있는 이름을 붙입니다. 이름을 붙여 두면 같은 값을 나중에 다시 쓰기 쉽고, 코드도 훨씬 읽기 쉬워집니다.

```python
store_name = "강남점"
sales = 170000
```

`store_name`은 `"강남점"`이라는 글자 값을 가리킵니다. `sales`는 `170000`이라는 숫자 값을 가리킵니다. 숫자를 직접 계속 쓰는 대신 `sales`라고 이름 붙여 두면, 이 값이 매출이라는 뜻을 바로 알 수 있습니다.

변수를 만들 때의 등호 `=`는 수학의 "같다"가 아니라 "오른쪽 값을 왼쪽 이름에 담는다"에 가깝습니다. 종이 봉투 겉에 `sales`라고 써 놓고 그 안에 `170000`을 넣는다고 생각해 보세요. 조건에서 같은지 비교할 때는 `==`를 씁니다.

```python
sales = 170000      # sales에 170000을 담는다
sales == 170000     # sales가 170000인지 비교한다
```

이 차이를 초반에 잡아 두면 오류가 줄어듭니다. `=`는 담는 일이고, `==`는 같은지 묻는 일입니다.

### 3. 자료형은 값의 종류입니다

파이썬은 값의 종류를 구분합니다. 이 값이 숫자인지, 글자인지, 참과 거짓인지에 따라 할 수 있는 일이 달라집니다.

엑셀에서도 숫자처럼 보이는 글자와 진짜 숫자가 다르게 동작하죠? 예를 들어 `"170000"`처럼 따옴표 안에 들어간 값은 겉으로는 숫자처럼 보여도 글자입니다. 파이썬도 자료형이 맞아야 계산이 됩니다.

| 자료형 | 예시 | 엑셀과 비교 |
|---|---|---|
| 정수 `int` | `170000` | 소수점 없는 숫자 |
| 실수 `float` | `0.15` | 소수점 있는 숫자 |
| 문자열 `str` | `"강남점"` | 텍스트 |
| 불리언 `bool` | `True`, `False` | 조건 결과, 체크 여부 |

아래 코드는 값의 종류를 확인합니다.

```python
type(170000)
type("강남점")
type(True)
```

입문 단계에서 자료형 이름을 다 외울 필요는 없습니다. 다만 계산이 안 될 때는 "숫자인 줄 알았는데 글자였나"부터 확인하는 습관을 들여야 합니다. 자료형 확인은 데이터 분석에서 자주 하는 점검입니다. 키를 재기 전에 자가 제대로 놓였는지 보는 것과 비슷합니다.

### 4. 리스트는 한 열처럼 여러 값을 담습니다

리스트는 여러 값을 순서대로 담는 상자입니다. 엑셀에서 한 열에 매출이 아래로 쌓여 있는 모습과 비슷합니다.

```python
sales_list = [120000, 98000, 135000, 170000, 155000]
```

여기서 `sales_list`는 매출 값 여러 개를 하나로 묶어 둔 이름입니다. 대괄호 `[]` 안에 값들이 순서대로 들어 있습니다. 엑셀에서 월요일부터 금요일까지의 매출을 한 열에 적어 두는 느낌으로 보면 됩니다.

리스트에서 값의 개수는 `len()`으로 셉니다.

```python
len(sales_list)
```

합계는 `sum()`으로도 구할 수 있지만, 이 차시에서는 반복문을 익히려고 일부러 직접 더해 봅니다. 손으로 한 번 더해 보면 다음 차시 pandas의 집계가 무엇을 대신 해주는지 감이 잡힙니다. 계산기를 쓰기 전에 덧셈 원리를 한 번 확인하는 과정이라고 보면 됩니다.

### 5. 딕셔너리는 한 행처럼 이름 붙은 값을 담습니다

딕셔너리는 값마다 이름표가 붙어 있는 구조입니다. 엑셀의 한 행을 떠올리면 쉽습니다. 한 행에는 요일, 매출, 지점처럼 서로 다른 성격의 값이 함께 들어갑니다.

```python
order = {
    "요일": "월",
    "매출": 120000,
    "지점": "강남점",
}
```

여기서 `"요일"`, `"매출"`, `"지점"`은 값을 찾기 위한 이름표입니다. 파이썬에서는 이런 이름표를 키라고 부릅니다. 사전에서 단어를 찾으면 뜻이 나오듯이, 딕셔너리에서는 키를 넣으면 그에 해당하는 값이 나옵니다.

딕셔너리에서 특정 값을 꺼낼 때는 열 이름처럼 키를 씁니다.

```python
order["요일"]
order["매출"]
```

리스트 안에 딕셔너리를 여러 개 넣으면 작은 표처럼 다룰 수 있습니다.

```python
daily_sales = [
    {"요일": "월", "매출": 120000},
    {"요일": "화", "매출": 98000},
    {"요일": "수", "매출": 135000},
]
```

이 구조가 다음 차시의 pandas DataFrame으로 이어집니다. DataFrame은 이런 행과 열 구조를 훨씬 편하게 다루도록 만든 도구입니다. 지금은 작은 표를 손으로 다루는 연습을 하고, 다음 차시에는 같은 일을 더 빠르고 익숙한 표 형태로 해 봅니다.

### 6. 조건문은 필터의 출발점입니다

엑셀 필터에서 `요일 = 금` 조건을 걸면 금요일 행만 보입니다. 파이썬에서는 `if`로 조건을 씁니다. `if`는 "만약 이 조건이 맞다면 아래 일을 하세요"라는 뜻입니다.

```python
day = "금"

if day == "금":
    print("금요일입니다.")
```

`if` 뒤에 조건을 쓰고, 조건 끝에는 콜론 `:`을 붙입니다. 그 아래 실행할 코드는 들여쓰기합니다. 파이썬에서 들여쓰기는 문법입니다. 보기 좋으라고 넣는 장식이 아니라 어떤 코드가 조건 안에 들어가는지 알려주는 표시입니다.

여기서 뭐가 보이시나요? `day == "금"`은 `day`에 담긴 값이 `"금"`과 같은지 확인합니다. 맞으면 `print("금요일입니다.")`가 실행되고, 맞지 않으면 실행되지 않습니다. 엑셀 필터가 조건에 맞는 행만 보여 주는 것처럼, 조건문은 조건에 맞는 경우에만 코드를 실행합니다.

### 7. 반복문은 같은 작업을 여러 번 하는 방법입니다

엑셀에서는 수식을 한 셀에 만든 뒤 아래로 채웁니다. 파이썬에서는 `for` 반복문이 비슷한 역할을 합니다. 여러 값에 같은 작업을 차례대로 적용할 때 씁니다.

```python
for amount in sales_list:
    print(amount)
```

`sales_list` 안의 값을 하나씩 꺼내 `amount`라는 이름으로 부르는 코드입니다. 첫 번째 반복에서 `amount`는 `120000`, 두 번째 반복에서는 `98000`이 됩니다.

이때 `amount`는 반복문이 지금 보고 있는 값의 임시 이름입니다. 사람이 명단을 보면서 한 사람씩 이름을 부르듯이, 파이썬은 리스트 안의 값을 하나씩 꺼내 같은 작업을 반복합니다.

합계를 직접 구하려면 누적할 변수를 먼저 만듭니다.

```python
total = 0

for amount in sales_list:
    total = total + amount
```

`total = total + amount`는 처음 보면 어색합니다. 오른쪽의 기존 `total`에 이번 `amount`를 더한 다음, 그 값을 다시 `total`에 담으라는 뜻입니다. 장바구니에 물건 값을 하나씩 더해 총액을 갱신하는 방식과 같습니다.

### 8. 함수는 자주 쓰는 절차에 이름을 붙이는 것입니다

이 차시에서 함수는 깊게 다루지 않습니다. 다만 다음 차시부터 `read_csv()`, `head()`, `mean()` 같은 함수를 계속 만나게 되니 개념만 미리 맛봅니다.

```python
def average(numbers):
    total = 0
    for number in numbers:
        total = total + number
    return total / len(numbers)
```

함수는 여러 줄짜리 절차에 이름을 붙여 두고 다시 꺼내 쓰는 방법입니다. 엑셀의 `SUM()`이나 `AVERAGE()`처럼 이름만 부르면 정해진 계산을 해줍니다.

위 코드에서는 `average`라는 이름의 함수를 만들었습니다. `numbers`라는 여러 숫자를 받으면, 합계를 구하고 개수로 나누어 평균을 돌려줍니다. 여기서 돌려준다는 말은 계산 결과를 함수 밖에서 다시 쓸 수 있게 내보낸다는 뜻입니다.

함수를 처음부터 완벽히 만들 필요는 없습니다. 입문 단계에서는 "반복해서 쓸 절차에 이름을 붙인 것" 정도로 이해하면 충분합니다.

마지막에는 평일 행만 골라 합계와 평균을 계산합니다. 이 작업이 다음 차시 pandas 필터와 바로 이어집니다. 오늘은 작은 데이터를 손으로 처리하고, 다음에는 같은 생각을 표 전체에 적용해 봅니다.

## 엑셀과 연결해서 설명할 지점

- 변수는 셀 주소보다 읽기 쉬운 이름이다.
- 리스트는 값이 세로로 쌓인 한 열과 비슷하다.
- 딕셔너리는 열 이름이 붙은 한 행과 비슷하다.
- 반복문은 아래로 채우기처럼 같은 계산을 여러 값에 적용한다.
- 조건문은 엑셀 필터의 조건식과 비슷하다.
- pandas 필터는 `if` 조건문을 표 전체 열에 한 번에 적용하는 방식이다.

엑셀을 이미 써 본 수강생이라면 파이썬을 완전히 낯선 도구로 느끼지 않아도 됩니다. 변수는 셀 주소보다 읽기 쉬운 이름이고, 리스트는 한 열처럼 값이 모인 구조입니다. 딕셔너리는 열 이름이 붙은 한 행처럼 생각하면 됩니다.

반복문은 수식을 아래로 채우는 느낌과 닮았습니다. 조건문은 필터 조건과 닮았습니다. 이 연결을 계속 붙잡고 가면, 다음 차시에서 pandas를 배울 때 "갑자기 새 도구가 나왔다"가 아니라 "엑셀에서 하던 표 작업을 파이썬 방식으로 하는구나"라고 이해할 수 있습니다.

## 수업 후 수강생이 말할 수 있어야 하는 문장

- "변수는 값을 다시 쓰기 위해 붙이는 이름이다."
- "리스트는 여러 값을 순서대로 담고, 딕셔너리는 이름표가 붙은 값을 담는다."
- "반복문은 리스트 안의 값을 하나씩 꺼내 같은 작업을 반복한다."
- "조건문은 원하는 경우만 실행하게 하는 문법이고, pandas 필터의 바탕이 된다."

수업이 끝난 뒤에는 문법 이름만 외우는 것이 아니라, 각 문법이 왜 필요한지 말할 수 있어야 합니다. 변수는 값을 다시 쓰기 위해 필요하고, 리스트와 딕셔너리는 여러 값을 표처럼 다루기 위해 필요합니다. 반복문은 같은 작업을 여러 번 하기 위해 쓰고, 조건문은 원하는 경우만 골라 처리하기 위해 씁니다.

이 정도 감각이 있으면 다음 차시에서 pandas를 만났을 때 훨씬 덜 낯섭니다. pandas가 해 주는 일은 오늘 배운 조건, 반복, 묶음의 생각을 표 데이터에 맞게 더 편하게 만들어 주는 것입니다.
