# S19. 풀사이클 실습: AI 요약, 파이썬 그래프, BI 1페이지

## 차시 개요

- 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
- 선수 차시: S14 시각화 실전, S18 AI 요약과 BI 요약
- 난이도: 5
- 시간: 90분
- 사용 도구: Python, pandas, matplotlib, AI 요약 도구, Looker Studio 또는 Power BI Desktop
- 사용 데이터: `02_datasets/real_korean/cafe_restaurant/seoul_commercial_food_stores_202109_github.csv`
- 선택 케이스: 상권 groupby 케이스

이번 차시는 지금까지 배운 내용을 한꺼번에 길게 복습하는 시간이 아니다. 이미 짜둔 스캐폴딩을 발판 삼아, 실제 데이터분석 업무가 어떤 순서로 이어지는지 작게 한 바퀴 돌아 보는 시간이다.

오늘의 질문은 다음과 같다.

> 서울 음식 업종 점포는 어느 구에 많이 모여 있고, 구별 업종 구성은 어떻게 다른가?

원자료는 119,158행 13열짜리 서울 음식 업종 상가 데이터다. 행이 이렇게 많으니 원자료를 곧바로 차트에 밀어 넣지 않는다. 먼저 파이썬으로 구와 업종별 집계표를 만든 다음, 그 표를 AI 요약과 BI 대시보드에 넘긴다.

수업의 고정 스코프는 다음 세 단계다.

1. AI 요약: 원자료 전체가 아니라 파이썬이 만든 집계 요약을 AI에 넣고 설명문을 만든다.
2. 파이썬 핵심 그래프 1개: 점포 수 상위 10개 구 막대그래프를 만든다.
3. BI 1페이지: 구별·업종별 점포 수 CSV로 필터 가능한 1페이지 대시보드를 만든다.

## 실제 확인한 데이터 사실

`pandas.read_csv()`로 실제 확인한 결과는 다음과 같다.

| 항목 | 확인값 |
|---|---:|
| 원자료 행 수 | 119,158행 |
| 원자료 열 수 | 13열 |
| 시도명 | 모두 서울특별시 |
| 시군구 수 | 25개 |
| 업종 중분류 수 | 14개 |
| 상가업소번호 중복 | 0건 |
| 상호명 결측 | 1건 |
| 행정동명 결측 | 312건 |
| 지점명 결측 | 87,355건 |

이번 분석에서 `지점명`은 쓰지 않는다. 지점명이 많이 비었다고 해서 그 행들을 통째로 버리면 정작 분석 목적과 어긋난다. 오늘 목적은 구별 점포 수와 업종 구성을 보는 것이니, `시군구명`, `상권업종중분류명`, `상가업소번호` 같은 핵심 컬럼이 채워져 있는지를 더 눈여겨본다.

점포 수 상위 구는 실제 실행 기준으로 강남구 12,095개, 마포구 7,396개, 송파구 7,106개, 서초구 6,366개, 강서구 6,114개 순이다. 전체 커피점/카페는 19,703개이고, 업종 중분류 중 가장 많은 것은 한식 40,448개다.

## 학습 목표

1. 큰 원자료를 바로 차트에 넣지 않고, 분석 목적에 맞는 집계표로 줄일 수 있다.
2. AI 요약에 원자료 전체를 던지는 방식과 집계 결과를 근거로 요약시키는 방식의 차이를 설명할 수 있다.
3. 파이썬 코드로 재현 가능한 그래프와 BI 입력용 CSV를 만들 수 있다.
4. Looker Studio 또는 Power BI에서 같은 CSV를 사용해 1페이지 대시보드를 만들 수 있다.
5. 결과 해석에서 "점포 수"와 "매출", "수요", "성공 가능성"을 구분할 수 있다.

## 핵심 개념

### 1. 풀사이클은 거창한 프로젝트가 아니라 끊기지 않는 흐름이다

입문자에게 풀사이클이라는 말은 부담스럽게 들릴 수 있다. 하지만 오늘 다루는 풀사이클은 작다.

```text
질문 정하기
-> 데이터 확인
-> 필요한 컬럼만 고르기
-> groupby로 요약하기
-> AI로 설명 초안 만들기
-> 파이썬 그래프 1개 만들기
-> BI 1페이지로 공유하기
-> 해석의 한계 적기
```

중요한 것은 모든 도구를 많이 쓰는 것이 아니라, 도구 사이의 역할을 끊지 않는 것이다. 엑셀에서도 원자료, 피벗테이블, 차트, 보고서가 따로 놀면 보고가 어려워진다. 데이터분석에서도 같은 집계표가 AI 요약, 그래프, BI의 공통 출발점이 되어야 한다.

### 2. `groupby`는 피벗테이블의 값 요약이다

엑셀에서 "구별 점포 수"를 보려면 피벗테이블의 행 영역에 구를 넣고, 값 영역에 점포 번호 개수를 넣는다. pandas에서는 같은 일을 `groupby()`로 한다.

| 엑셀 피벗테이블 | pandas |
|---|---|
| 행 영역: 시군구명 | `groupby("시군구명")` |
| 값 영역: 상가업소번호 개수 | `.agg(점포수=("상가업소번호", "count"))` |
| 내림차순 정렬 | `.sort_values("점포수", ascending=False)` |

오늘 BI에 넘기는 표는 원자료 119,158행이 아니라 `시군구명 x 상권업종중분류명`으로 줄인 349행이다. 이 정도 크기면 BI에서 필터와 막대그래프를 다루기 쉽다.

### 3. AI는 계산기가 아니라 설명 초안 도구로 쓴다

AI에게 원자료 전체를 올리고 "분석해 줘"라고 하면 어떤 기준으로 요약했는지 확인하기 어렵다. 이번 차시에서는 파이썬이 만든 검증된 집계 결과만 AI에 넣는다.

AI에게 맡기는 일은 다음과 같다.

- 숫자를 다시 계산하는 일: 맡기지 않는다.
- 표에 있는 숫자를 바탕으로 설명문을 정리하는 일: 맡긴다.
- 분석 한계를 빠뜨리지 않게 점검하는 일: 맡긴다.

비유하면 파이썬은 계산 과정이 적힌 계산기이고, AI는 그 계산 결과를 보고서 문장으로 바꾸는 초안 작성자다. 초안 작성자는 편하지만, 원장부를 검산하지 않고 믿으면 안 된다.

### 4. 파이썬 그래프는 보고서의 핵심 그림 한 장이다

오늘 만드는 그래프는 "서울 음식 업종 점포 수 상위 10개 구" 막대그래프다. 그래프 하나로 모든 것을 설명하려고 하지 않는다. 대신 첫 질문에 가장 직접적으로 답하는 그림 하나를 정확하게 만든다.

그래프에는 점포 수와 카페 비율을 같이 표시한다. 구별 전체 규모와 카페 비중을 한 번에 보되, 카페 비율은 보조 설명으로만 사용한다. 이 데이터에는 매출, 임대료, 유동인구가 없으므로 "카페가 잘 된다"거나 "창업하기 좋다"라고 말하면 과해진다.

### 5. BI 1페이지는 최종 보고서가 아니라 탐색 가능한 요약 화면이다

BI 도구는 발표용 차트 하나보다 필터와 상호작용이 강하다. 오늘 BI 페이지에는 다음 요소를 둔다.

- 총 점포 수 카드
- 구별 점포 수 막대그래프
- 업종 중분류 필터
- 구별 업종 구성 막대그래프 또는 표
- 해석 메모 2줄

Windows 수강생은 Power BI Desktop을 사용할 수 있다. Mac 수강생은 Power BI Desktop 설치가 어렵기 때문에 Looker Studio 경로를 기본 대안으로 쓴다.

## 엑셀과 연결해서 설명할 지점

- `groupby()`는 피벗테이블과 비슷하다. 행 기준으로 묶고 값의 개수나 합계를 낸다.
- `to_csv()`는 피벗 결과를 다른 도구에 넘기기 위한 저장이다.
- AI 요약 입력용 텍스트는 엑셀 피벗 결과를 복사해 보고서 초안 작성자에게 주는 것과 비슷하다.
- 파이썬 그래프는 반복 실행 가능한 차트다. 같은 코드와 같은 데이터면 같은 그림이 나온다.
- BI 대시보드는 엑셀 차트보다 필터와 공유에 강하다. 대신 데이터 모델과 필드 형식을 맞춰야 한다.

## 해석에서 조심할 점

- 점포 수가 많다는 것은 매출이 높다는 뜻이 아니다.
- 카페 비율이 높다는 것은 카페 창업 성공률이 높다는 뜻이 아니다.
- 이 데이터는 2021년 9월 기준 상가 정보이므로 현재 상권 상황과 다를 수 있다.
- 좌표가 있지만 오늘은 지도 분석을 하지 않는다. 구별 집계와 업종 구성까지만 다룬다.
- 행정동명 결측이 312건 있으므로 행정동 단위로 깊게 내려가려면 추가 점검이 필요하다.
