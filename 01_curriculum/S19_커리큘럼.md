# S19. 풀사이클 실습: AI 요약, 파이썬 그래프, BI 1페이지

## 차시 개요

- 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
- 선수 차시: S14 시각화 실전, S18 AI 요약과 BI 요약
- 난이도: 5
- 시간: 90분
- 사용 도구: Python, pandas, matplotlib, AI 요약 도구, Looker Studio 또는 Power BI Desktop
- 사용 데이터: 수업 홈의 데이터 받기 페이지에서 내려받은 서울 음식 업종 상가 파일. CSV는 엑셀로도 열리는 표 파일입니다. 초보자는 같은 내용의 엑셀 파일(.xlsx)을 받아 열어 봐도 됩니다.
- 선택 케이스: 상권 groupby 케이스

서울 음식 업종 점포가 어느 구에 많이 모여 있는지 보고, 구마다 업종 구성이 어떻게 다른지 확인하는 한 바퀴짜리 분석 흐름입니다. 엑셀에서 원자료를 피벗테이블로 줄이고 차트와 보고서로 이어 가듯이, 여기서는 파이썬으로 집계표를 만들고 그 결과를 AI 요약과 BI 화면으로 연결해 봅니다.

이 차시는 지금까지 배운 내용을 길게 다시 보는 시간이 아닙니다. 이미 준비된 스캐폴딩 코드를 발판 삼아, 실제 데이터분석 업무가 어떤 순서로 이어지는지 작게 끝까지 밟아 보는 시간입니다.

우리가 붙잡을 질문은 이 문장입니다.

> 서울 음식 업종 점포는 어느 구에 많이 모여 있고, 구별 업종 구성은 어떻게 다른가?

원자료는 **119,158행 13열**짜리 서울 음식 업종 상가 데이터입니다. 여기서 행은 점포 하나하나의 기록이고, 열은 구 이름, 업종 이름, 점포 번호처럼 각 점포를 설명하는 항목입니다. 행이 이렇게 많으면 원자료를 곧바로 차트에 넣지 않습니다. 키를 재기 전에 사람을 줄 세우듯이, 먼저 어떤 기준으로 묶을지 정해야 합니다.

오늘은 파이썬으로 구와 업종별 집계표를 먼저 만듭니다. 집계표는 많은 원자료를 질문에 맞게 줄인 표입니다. 그다음 이 표를 AI 요약과 BI 대시보드에 넘깁니다. BI는 Business Intelligence의 줄임말로, 필터와 차트를 이용해 데이터를 살펴보는 대시보드 도구라고 생각하면 됩니다.

오늘 끝까지 이어 볼 범위는 세 가지입니다.

1. AI 요약: 원자료 전체가 아니라 파이썬이 만든 집계 요약을 AI에 넣고 설명문을 만듭니다.
2. 파이썬 핵심 그래프 1개: 점포 수 상위 10개 구 막대그래프를 만듭니다.
3. BI 1페이지: 구별·업종별 점포 수 CSV로 필터 가능한 1페이지 대시보드를 만듭니다.

## 실제 확인한 데이터 상태

`pandas.read_csv()`로 실제 데이터를 읽어 확인한 결과입니다. `pandas.read_csv()`는 파이썬에서 CSV 파일을 표처럼 불러오는 함수입니다.

| 항목 | 확인값 |
|---|---:|
| 원자료 행 수 | 119,158행 |
| 원자료 열 수 | 13열 |
| 시도명 | 모두 서울특별시 |
| 시군구 수 | 25개 |
| 업종 중분류 수 | 14개 |
| 상가업소번호 중복 | 0건 |
| 상호명 결측 | 1건 |
| 행정동명 결측 | 312건 |
| 지점명 결측 | 87,355건 |

이번 분석에서 `지점명`은 쓰지 않습니다. `지점명`이 많이 비어 있다고 해서 그 행들을 통째로 버리면, 오히려 오늘의 질문과 멀어집니다. 우리가 알고 싶은 것은 지점명이 아니라 구별 점포 수와 업종 구성입니다. 그래서 `시군구명`, `상권업종중분류명`, `상가업소번호`처럼 오늘 질문에 직접 필요한 컬럼이 채워져 있는지를 더 눈여겨봅니다.

점포 수 상위 구는 실제 실행 기준으로 강남구 12,095개, 마포구 7,396개, 송파구 7,106개, 서초구 6,366개, 강서구 6,114개 순입니다. 전체 커피점/카페는 19,703개이고, 업종 중분류 중 가장 많은 것은 한식 40,448개입니다.

## 학습 목표

1. 큰 원자료를 바로 차트에 넣지 않고, 분석 목적에 맞는 집계표로 줄일 수 있습니다.
2. AI 요약에 원자료 전체를 던지는 방식과 집계 결과를 근거로 요약시키는 방식의 차이를 설명할 수 있습니다.
3. 파이썬 코드로 재현 가능한 그래프와 BI 입력용 CSV를 만들 수 있습니다.
4. Looker Studio 또는 Power BI에서 같은 CSV를 사용해 1페이지 대시보드를 만들 수 있습니다.
5. 결과 해석에서 "점포 수"와 "매출", "수요", "성공 가능성"을 구분할 수 있습니다.

## 핵심 개념

### 1. 풀사이클은 거창한 프로젝트가 아니라 끊기지 않는 흐름입니다

입문자에게 풀사이클이라는 말은 부담스럽게 들릴 수 있습니다. 하지만 오늘 다루는 풀사이클은 작습니다. 하나의 질문을 정하고, 필요한 숫자를 만들고, 그 숫자로 설명문과 그래프와 대시보드를 만드는 흐름입니다.

흐름을 글자로 쓰면 이렇게 됩니다.

```text
질문 정하기
-> 데이터 확인
-> 필요한 컬럼만 고르기
-> groupby로 요약하기
-> AI로 설명 초안 만들기
-> 파이썬 그래프 1개 만들기
-> BI 1페이지로 공유하기
-> 해석의 한계 적기
```

여기서 중요한 부분은 도구를 많이 쓰는 것이 아닙니다. 도구 사이에서 같은 숫자가 이어져야 한다는 점입니다. 엑셀에서도 원자료, 피벗테이블, 차트, 보고서가 따로 놀면 보고하기 어렵습니다. 데이터분석에서도 같은 집계표가 AI 요약, 그래프, BI의 공통 출발점이 되어야 합니다.

### 2. `groupby`는 피벗테이블의 값 요약입니다

엑셀에서 "구별 점포 수"를 보려면 피벗테이블의 행 영역에 구를 넣고, 값 영역에 점포 번호 개수를 넣습니다. pandas에서는 같은 일을 `groupby()`로 합니다. `groupby`는 같은 값끼리 묶은 뒤 개수나 합계를 내는 기능입니다. 사전에서 같은 초성끼리 단어를 모아 찾기 쉽게 만드는 것과 비슷합니다.

| 엑셀 피벗테이블 | pandas |
|---|---|
| 행 영역: 시군구명 | `groupby("시군구명")` |
| 값 영역: 상가업소번호 개수 | `.agg(점포수=("상가업소번호", "count"))` |
| 내림차순 정렬 | `.sort_values("점포수", ascending=False)` |

오늘 BI에 넘기는 표는 원자료 119,158행이 아닙니다. `시군구명 x 상권업종중분류명`으로 줄인 349행입니다. 이렇게 줄이면 BI에서 필터를 걸고 막대그래프를 만들 때 훨씬 다루기 쉽습니다. 원자료 전체를 들고 가는 것이 성실한 분석처럼 보일 수 있지만, 질문에 맞게 줄인 표가 더 좋은 출발점일 때가 많습니다.

### 3. AI는 계산기가 아니라 설명 초안 도구로 씁니다

AI에게 원자료 전체를 올리고 "분석해 줘"라고 하면 어떤 기준으로 요약했는지 확인하기 어렵습니다. 특히 숫자가 많은 데이터에서는 AI가 어떤 숫자를 골랐고 무엇을 빠뜨렸는지 사람이 다시 검산해야 합니다. 그래서 이번 차시에서는 파이썬이 만든 검증된 집계 결과만 AI에 넣습니다.

AI에게 맡기는 일은 이렇게 나눕니다.

- 숫자를 다시 계산하는 일: 맡기지 않습니다.
- 표에 있는 숫자를 바탕으로 설명문을 정리하는 일: 맡깁니다.
- 분석 한계를 빠뜨리지 않게 점검하는 일: 맡깁니다.

비유하면 파이썬은 계산 과정이 적힌 계산기이고, AI는 그 계산 결과를 보고서 문장으로 바꾸는 초안 작성자입니다. 초안 작성자는 편합니다. 하지만 원장부를 검산하지 않고 초안 작성자의 말만 믿으면 안 됩니다. 숫자는 파이썬에서 확인하고, 문장은 AI로 다듬는다고 생각해 보세요.

### 4. 파이썬 그래프는 보고서의 핵심 그림 한 장입니다

오늘 만드는 그래프는 "서울 음식 업종 점포 수 상위 10개 구" 막대그래프입니다. 그래프 하나로 모든 것을 설명하려고 하지 않습니다. 대신 첫 질문에 가장 직접적으로 답하는 그림 하나를 정확하게 만듭니다.

그래프에는 점포 수와 카페 비율을 같이 표시합니다. 점포 수는 구별 전체 규모를 보여 줍니다. 카페 비율은 그 구의 음식 업종 안에서 커피점/카페가 어느 정도 비중을 차지하는지 보조로 보여 줍니다. 키를 잴 때 키 자체와 체형 정보를 함께 보는 느낌입니다. 다만 카페 비율은 보조 설명일 뿐입니다.

이 데이터에는 매출, 임대료, 유동인구가 없습니다. 그래서 "카페가 잘 된다"거나 "창업하기 좋다"라고 말하면 범위를 넘어섭니다. 그래프가 보여 주는 것은 점포 수 분포와 업종 구성입니다. 그래프가 보여 주지 않는 것은 말하지 않는 연습도 분석에서 중요합니다.

### 5. BI 1페이지는 최종 보고서가 아니라 탐색 가능한 요약 화면입니다

BI 도구는 발표용 차트 하나보다 필터와 상호작용이 강합니다. 같은 표를 놓고도 한식만 볼 수 있고, 커피점/카페만 볼 수도 있습니다. 엑셀 필터를 더 보기 좋게 화면으로 만든다고 생각하면 이해하기 쉽습니다.

오늘 BI 페이지에는 다음 요소를 둡니다.

- 총 점포 수 카드
- 구별 점포 수 막대그래프
- 업종 중분류 필터
- 구별 업종 구성 막대그래프 또는 표
- 해석 메모 2줄

Windows 수강생은 Power BI Desktop을 사용할 수 있습니다. Mac 수강생은 Power BI Desktop 설치가 어렵기 때문에 Looker Studio 경로를 기본 대안으로 씁니다. 어느 도구를 쓰더라도 출발점은 같습니다. 파이썬이 만든 같은 집계표를 사용해야 결과를 서로 비교할 수 있습니다.

## 엑셀과 연결해서 설명할 지점

- `groupby()`는 피벗테이블과 비슷합니다. 행 기준으로 묶고 값의 개수나 합계를 냅니다.
- `to_csv()`는 피벗 결과를 다른 도구에 넘기기 위한 저장입니다.
- AI 요약 입력용 텍스트는 엑셀 피벗 결과를 복사해 보고서 초안 작성자에게 주는 것과 비슷합니다.
- 파이썬 그래프는 반복 실행 가능한 차트입니다. 같은 코드와 같은 데이터면 같은 그림이 나옵니다.
- BI 대시보드는 엑셀 차트보다 필터와 공유에 강합니다. 대신 데이터 모델과 필드 형식을 맞춰야 합니다.

## 해석에서 조심할 점

- 점포 수가 많다는 것은 매출이 높다는 뜻이 아닙니다.
- 카페 비율이 높다는 것은 카페 창업 성공률이 높다는 뜻이 아닙니다.
- 이 데이터는 2021년 9월 기준 상가 정보이므로 현재 상권 상황과 다를 수 있습니다.
- 좌표가 있지만 오늘은 지도 분석을 하지 않습니다. 구별 집계와 업종 구성까지만 다룹니다.
- 행정동명 결측이 312건 있으므로 행정동 단위로 깊게 내려가려면 추가 점검이 필요합니다.
