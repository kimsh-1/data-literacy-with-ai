# S15. Power BI 설치, 로드, 첫 차트

## 차시 개요

- 대상: 엑셀은 쓰지만 데이터분석은 처음인 비개발자
- 선수 차시: S03 피벗테이블과 기본 집계
- 난이도: 3
- 시간: 60분
- 사용 데이터: 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 CSV, 즉 엑셀로도 열리는 표 파일
- 보조 데이터 파일: 수업 홈의 데이터 받기 페이지에서 내려받은 같은 카페 매출 엑셀 파일(.xlsx)

BI 도구는 데이터를 새로 만들어 내는 도구가 아니라, 이미 정리된 표를 사람이 보기 좋은 보고 화면으로 바꿔 주는 도구입니다. 여기서 BI는 Business Intelligence의 줄임말이고, 회사나 가게의 숫자를 보면서 판단할 수 있게 도와주는 화면이라고 생각하면 됩니다.

엑셀 피벗테이블은 내가 빠르게 합계를 내 보고 기준을 바꿔 보는 작업장에 가깝습니다. Power BI는 그 결과를 다른 사람이 계속 열어 보고, 필터를 바꾸고, 그래프를 보면서 이해할 수 있는 보고 화면에 가깝습니다. 엑셀에서 계산한 숫자를 종이에 적어 두는 것과, 벽에 붙인 현황판으로 만들어 두는 것의 차이를 떠올리면 됩니다.

수강생은 Power BI Desktop을 설치하거나 실행한 뒤, 수업 홈의 데이터 받기 페이지에서 내려받은 정제된 카페 매출 파일을 불러옵니다. 그리고 첫 보고서 페이지에 총매출 카드, 카테고리별 매출 막대그래프, 일자별 매출 꺾은선그래프를 만듭니다. Mac 수강생은 Power BI Desktop을 직접 설치하기 어렵기 때문에, 이번 시간에는 화면 구조와 개념을 따라오고 S18에서 Looker Studio로 직접 실습합니다.

## 학습 목표

1. Power BI Desktop이 Windows용 보고서 제작 도구라는 점을 알고 설치 또는 실행할 수 있다.
2. CSV 파일과 엑셀 파일을 Power BI에 불러오는 경로를 설명할 수 있다.
3. 필드 목록, 시각화 패널, 캔버스의 역할을 엑셀 피벗테이블과 비교해 이해한다.
4. `category`, `date`, `amount` 필드를 사용해 막대그래프와 꺾은선그래프를 만들 수 있다.
5. `amount`의 합계를 KPI 카드로 표시하고, 표시 단위를 조정해 총매출을 확인할 수 있다.

## 데이터 확인 결과

수업에서 쓰는 카페 매출 파일은 `python3`와 `pandas 3.0.3`으로 열어 미리 확인한 데이터입니다. 여기서 확인한다는 말은, 표가 몇 줄인지, 기간이 어디부터 어디까지인지, 비어 있는 값이 있는지를 미리 검사했다는 뜻입니다. 키를 재기 전에 줄자가 제대로 놓였는지 보는 것과 같습니다.

| 항목 | 확인값 |
|---|---:|
| 행 수 | 10,424 |
| 열 수 | 12 |
| 기간 | 2025-01-01부터 2025-06-30까지 |
| 총매출 `amount` 합계 | 90,492,500 |
| 주문수 | 10,424 |
| 수량 `qty` 합계 | 17,241 |
| 결측값 총합 | 0 |

행 수는 주문 기록이 몇 줄 있는지를 뜻합니다. 이 데이터에는 10,424줄이 있고, 주문수도 10,424건입니다. 기간은 2025-01-01부터 2025-06-30까지이므로, 반년 동안의 카페 매출 흐름을 보는 자료라고 이해하면 됩니다. 결측값은 비어 있는 값이라는 뜻인데, 결측값 총합이 0이므로 이번 실습에서는 비어 있는 칸 때문에 그래프가 어긋날 가능성이 낮습니다.

주요 열은 이렇게 읽으면 됩니다.

| 열 이름 | 의미 | Power BI에서 주로 쓰는 위치 |
|---|---|---|
| `date` | 주문일 | 꺾은선그래프의 X축 |
| `category` | 상품군 | 막대그래프의 X축 |
| `amount` | 주문금액 | 그래프 값, KPI 카드 값 |
| `store` | 지점 | 선택 필터 또는 범례 |
| `order_id` | 주문 번호 | 주문 건수 확인 |

`date`는 언제 팔렸는지를 보는 열입니다. 시간의 흐름을 봐야 하므로 꺾은선그래프의 X축에 놓습니다. `category`는 커피, 음료, 디저트처럼 무엇이 팔렸는지를 나누는 기준입니다. 항목끼리 비교해야 하므로 막대그래프의 X축에 놓습니다. `amount`는 주문금액이므로 그래프의 높이나 카드의 큰 숫자로 들어갑니다.

## 핵심 개념 1. Power BI Desktop은 보고서 제작 도구입니다

Power BI라는 이름은 여러 곳에서 나옵니다. 처음 배우는 사람은 여기서 가장 많이 헷갈립니다. 같은 이름 안에 보고서를 만드는 프로그램, 웹에 올려 공유하는 서비스, 조직에서 쓰는 유료 기능이 함께 있기 때문입니다.

| 구분 | 이번 차시에서의 의미 |
|---|---|
| Power BI Desktop | Windows PC에 설치해 보고서 파일을 만드는 프로그램 |
| `.pbix` 파일 | Power BI Desktop에서 저장하는 보고서 파일 |
| Power BI Service | 웹에 게시하고 공유하는 서비스. S18에서 공유 개념을 다룬다 |
| Power BI Pro, Fabric, Copilot | 조직 공유와 고급 기능에 필요한 유료 또는 조직 환경. 오늘은 쓰지 않는다 |

이번 차시에서 손으로 다룰 대상은 Power BI Desktop입니다. 내 컴퓨터에서 데이터 파일을 열고, 차트를 배치하고, 보고서 한 페이지를 만드는 데 집중합니다. 계정으로 공유하기, 웹에 게시하기, 다른 사람에게 권한을 주는 일은 오늘의 목표가 아닙니다.

Power BI Desktop은 Windows용 도구입니다. Microsoft 공식 다운로드 페이지도 Windows용 Desktop 설치를 안내하고, 강의 환경도 Windows 10이나 Windows 11 PC를 기준으로 봅니다. Mac 사용자는 억지로 우회 설치를 시도하지 않아도 됩니다. 이번 시간에는 어떤 화면에서 어떤 일을 하는지 눈으로 익히고, S18에서 Looker Studio를 사용해 같은 보고서 개념을 직접 실습합니다.

## 핵심 개념 2. 로드는 표를 Power BI 안으로 가져오는 일입니다

엑셀은 파일을 열면 바로 셀이 보입니다. Power BI는 조금 다릅니다. 먼저 데이터를 가져와야 하고, 이 과정을 로드라고 부릅니다. 로드는 표를 Power BI 보고서 안에서 사용할 수 있게 등록하는 일입니다.

피벗테이블을 만들 때 원본 표 범위를 먼저 지정했던 일을 떠올려 보세요. Power BI에서도 먼저 원본 데이터를 지정합니다. 다만 Power BI는 원본 표를 그대로 고치는 프로그램이라기보다, 그 표를 바탕으로 카드와 그래프가 놓인 화면을 만드는 프로그램입니다.

| 엑셀에서의 감각 | Power BI에서의 감각 |
|---|---|
| 엑셀 파일 열기 | 데이터 가져오기 |
| 표 범위 선택 | CSV 또는 엑셀 시트 선택 |
| 피벗테이블 필드 목록 | 데이터 패널의 필드 목록 |
| 행, 열, 값 영역 | X축, Y축, 값, 범례 영역 |
| 피벗 차트 | 시각화 개체 |

오늘 다루는 카페 매출 데이터는 이미 깨끗하게 정리돼 있습니다. 그래서 Power Query에서 행을 지우거나 열을 새로 만드는 정제 작업은 하지 않습니다. 대신 `date`가 날짜로 인식되는지, `amount`가 숫자로 인식되는지만 확인합니다. 날짜가 날짜로 잡혀야 시간 흐름을 제대로 그릴 수 있고, 금액이 숫자로 잡혀야 합계를 낼 수 있습니다.

CSV 파일로 불러와도 되고, 초보자는 같은 내용의 엑셀 파일(.xlsx)을 받아서 실습해도 됩니다. 다만 한 보고서 안에 둘을 동시에 넣으면 같은 데이터가 두 번 들어간 것처럼 헷갈릴 수 있으니, 수업에서는 하나를 기준으로 진행합니다.

## 핵심 개념 3. 시각화는 필드를 놓는 방식으로 만듭니다

Power BI 화면 가운데의 흰 영역이 캔버스입니다. 캔버스는 보고서 종이라고 보면 됩니다. 카드, 막대그래프, 꺾은선그래프 같은 시각화 개체를 이 종이 위에 올려놓습니다. 오른쪽에는 시각화 패널과 데이터 패널이 있습니다. 시각화 패널에서는 그래프 모양을 고르고, 데이터 패널에서는 표의 열을 골라 그래프에 넣습니다.

시각화를 만들 때는 세 가지를 차례로 생각해 봅니다.

1. 무엇을 비교할 것인가: `category`, `store`, `date`
2. 어떤 숫자를 볼 것인가: `amount`, `qty`, `order_id`
3. 어떤 모양이 적절한가: 카드, 막대, 꺾은선

막대그래프는 항목을 나란히 견줄 때 좋습니다. 카테고리별 매출은 커피, 음료, 디저트처럼 항목을 나눠 비교하는 일이므로 막대그래프에 잘 맞습니다. 이때 `category`마다 `amount` 합계를 세워서 높이를 비교합니다.

시간 흐름에는 꺾은선그래프가 어울립니다. 일자별 매출이라면 `date` 순서대로 `amount` 합계를 이어 봅니다. 선이 올라가면 매출이 늘어난 날이고, 선이 내려가면 매출이 줄어든 날입니다. 날짜를 따라 체온을 기록하면 몸 상태의 흐름이 보이는 것과 같은 감각입니다.

KPI 카드는 핵심 숫자 하나를 크게 보여주는 자리입니다. KPI는 Key Performance Indicator의 줄임말로, 꼭 봐야 하는 대표 지표라는 뜻입니다. 이번 보고서에서는 총매출이 그 역할을 합니다. 여러 그래프를 보기 전에 “그래서 전체 매출이 얼마인가요?”라는 질문에 바로 답해 주는 숫자입니다.

## 핵심 개념 4. KPI 카드는 큰 셀 하나라고 생각합니다

엑셀에서 `=SUM(매출범위)`를 계산한 뒤, 그 셀을 크게 키워 보고서 맨 위에 놓으면 보는 사람이 제일 먼저 총매출을 확인할 수 있습니다. Power BI의 카드도 같은 역할을 합니다. 많은 숫자 중에서 가장 먼저 봐야 하는 숫자 하나를 크게 보여줍니다.

이번 미션은 `amount` 합계를 카드로 띄우는 것입니다. 맞는 값은 **90,492,500**입니다. 이 숫자는 카페 매출 파일의 모든 주문금액을 더한 총매출입니다. 한 주문씩 영수증 금액을 전부 더한 값이라고 생각하면 됩니다.

Power BI는 기본 설정에서 큰 숫자를 `90M`이나 `90.49M`처럼 줄여 보여줄 때가 있습니다. 여기서 `M`은 million, 즉 백만 단위입니다. 화면을 간단하게 보이게 하려는 기능이지만, 처음 검산할 때는 오히려 헷갈립니다. 정확히 맞는지 확인하려면 표시 단위를 `없음(None)`으로 바꾸고 소수 자릿수를 `0`으로 맞춥니다. 그러면 총매출을 원래 숫자 그대로 확인할 수 있습니다.

## 엑셀과 연결해서 설명할 지점

- Power BI의 데이터 패널은 피벗테이블 필드 목록과 비슷하다.
- 막대그래프의 X축은 피벗테이블의 행 영역처럼 비교 기준을 놓는 자리이다.
- 막대그래프의 Y축 또는 값 영역은 피벗테이블의 값 영역처럼 합계나 개수를 놓는 자리이다.
- KPI 카드는 엑셀에서 `SUM` 결과를 크게 보여주는 셀과 비슷하다.
- Power BI는 화면을 만드는 도구이므로, 원본 데이터를 고치는 일은 엑셀 정제, Power Query, 파이썬과 역할을 나누어 생각한다.

엑셀을 이미 써 봤다면 Power BI가 완전히 낯선 도구는 아닙니다. 피벗테이블에서 행 영역에 기준을 놓고, 값 영역에 숫자를 놓았던 경험을 떠올리면 됩니다. Power BI에서는 그 자리가 X축, Y축, 값, 범례 같은 이름으로 바뀌어 있을 뿐입니다.

다만 Power BI는 원본 데이터를 고치는 데 초점을 둔 도구가 아닙니다. 원본을 정리하는 일은 엑셀 정제, Power Query, 파이썬이 더 잘 맡습니다. Power BI는 정리된 데이터를 사람에게 보여 주는 보고 화면을 만드는 쪽에 가깝습니다. 재료 손질과 접시 담기가 다른 일인 것처럼 역할을 나누어 생각하면 쉽습니다.

- "막대그래프는 항목 비교, 꺾은선그래프는 시간 흐름, 카드는 핵심 숫자에 쓴다."
- "`amount`를 카드에 넣고 합계로 요약하면 총매출을 볼 수 있다."

이 문장들을 말할 수 있으면 화면에서 무엇을 누르는지만 외운 것이 아니라, 왜 그 행동을 하는지도 이해한 상태입니다. `amount`를 카드에 넣는 이유는 주문금액을 모두 더해 총매출을 보기 위해서입니다. `category`를 막대그래프에 놓는 이유는 상품군별로 매출을 비교하기 위해서입니다. `date`를 꺾은선그래프에 놓는 이유는 날짜가 지나며 매출이 어떻게 움직였는지 보기 위해서입니다.

## 완료 판단

수강생이 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일을 Power BI에 불러와 총매출 KPI 카드를 `90,492,500`원에 맞추고, 같은 페이지에 카테고리별 매출 막대그래프와 일자별 매출 꺾은선그래프까지 배치했다면 완료로 봅니다.
