> 이 스펙이 차시 생성의 유일 계약. 각 차시 에이전트는 자기 차시 블록 + 이 상단 공통규칙을 읽고 생성한다.
> v2 변경: 실데이터 3표 실증검증 결과 반영 — 붕괴된 "기온↔수요 상관"축 교체(C1), cp949 인코딩 정규화(M1), S10 확장, S16 날짜차원, S18 Looker 주력, S19 축소, 환경 선수조건 추가.

## 공통 규칙 (전 차시 강제)
- 산문 전부 윤문 대상, **이모지 금지**, 사람이 쓴 강의노트 톤.
- 데이터: 합성(cafe_sales/shop_orders/survey_responses/hr_employees) + 실데이터(REAL_DATA_CATALOG.md, 전부 UTF-8 정규화 완료).
- **모범답안 코드는 지정 파일로 실제 실행해 통과한 것만 제출**(추정 금지). 파일명·전처리를 블록에 명시된 대로.
- 난이도 1~5: **파트 내 단조 증가, 새 도구(파이썬 S10·BI S15)는 진입장벽 리셋**.
- 대용량 실데이터(수만~20만행)는 시각화 전 **groupby 집계 또는 특정 역/기간 필터/`.sample()`** 로 축소 후 사용.

## 검증된 데이터 사실 (실측 기준 — 이걸로만 실습 설계)
- **cafe_sales**(합성): clean ~10,400행 + dirty(함정 8종), date 컬럼 有. 메인 골격.
- **shop_orders**(합성): orders/customers/products 3표, 참조무결성 확인, order_date 2024년 12개월 → 전월대비 DAX 가능. 단 orders에 매출 컬럼 없음 → 단가(products)·지역(customers) VLOOKUP 후 매출=단가×수량×(1−할인) 계산 필요.
- **hr_employees**(합성): 1,200행. **심슨의 역설 실증 확인** — 전체 남>여(+3.4M)이나 5개 부서 전부 여>남. clean/dirty.
- **survey_responses**(합성 1,500행): dirty에 중복10·nps결측·척도문자혼입(매우만족 등)·age_group 표기혼재(20대/이십대/20s) 확인.
- **molit_apt_sale_seoul_gangnam_2024.csv**(실 3,754행, UTF-8): `전용면적(㎡)`·`거래금액(만원)` → **강한 양의 상관**. S06 상관·S14 추세선·S20 회귀맛보기의 핵심 소재.
- **배달콜↔기온은 상관 없음(r≈0.015)** — 상관 실습에 쓰지 말 것. 단, "있을 것 같지만 없다"는 상관≠인과 반례로만 언급.
- **seoul_subway daily_hourly_2025**(19.9만행): 역명×시간대 집계 후 히트맵. **weather와 조인 금지**(연도 불일치).
- **cafe_restaurant 상가업소**(11.9만행): groupby 빠름, 시각화는 상위 N만.
- **FRED CPI/환율/실업**(월별 시계열), **seoul_living_pop 자치구 일별**(대용량 집계용).
