# AI 기반 데이터 분석 수업 — 마스터플랜 (로지컬)

> 이 문서는 **내부 설계도**다. 수강생·강사에게 나가는 콘텐츠(커리큘럼 설명·핸드아웃·강사 대본·슬라이드 카피)는 별도로 작성하며, **전부 윤문(gn-voice/humanize)** 을 거쳐 마감한다. 이 플랜 자체는 문어체로 간결하게 유지한다.

---

## 1. 수업 정체성

- **한 줄 정의**: 엑셀은 쓰지만 데이터 분석·코딩·통계는 처음인 사람이, AI를 지렛대 삼아 엑셀 → 파이썬 → BI 대시보드까지 한 사이클을 스스로 돌리게 만드는 실전 과정.
- **대상**: 완전 입문(비개발자). 엑셀 기본 조작(셀 입력, 간단한 함수)은 해봤다는 전제만 둔다.
- **핵심 약속**: "코드를 못 짜도 AI가 짜준다. 대신 **무엇을 물어보고, 그 답이 맞는지 검증하는 눈**은 사람이 가진다."

## 2. 설계 원칙

1. **하나의 데이터가 수업 전체를 관통한다.** 카페 매출 데이터를 메인 골격으로, 엑셀 정리 → AI 분석 → 파이썬 심화 → BI 대시보드까지 같은 데이터를 계속 쓴다. 도구가 바뀌어도 데이터는 그대로라, 학습자는 "도구의 차이"에만 집중한다.
2. **매 단계 검증을 붙인다.** AI가 준 답을 엑셀 피벗이나 파이썬으로 교차확인하는 습관을 처음부터 심는다 (환각 방어).
3. **난이도는 데이터로 통제한다.** 예시 파일을 clean/dirty 두 티어로 만들어, "깨끗한 데이터로 개념 익히기 → 지저분한 데이터로 실전 감각" 순으로 태운다.
4. **읽기 30 / 손 70.** 파이썬·BI 파트는 반드시 직접 따라 친다. 핸드아웃에 "따라 하기" 블록과 "직접 해보기" 미션을 분리 표기.
5. **모든 글은 윤문.** 수강생이 읽는 문장은 AI 티 없이, 사람이 쓴 강의 노트처럼.

## 3. 도구 스택 (비개발자 기준 고정)

| 단계 | 도구 | 선정 이유 |
|---|---|---|
| 엑셀 정리 | MS Excel (피벗·함수) | 이미 쓰는 도구. 데이터 감각의 출발점 |
| AI 분석 | ChatGPT(Advanced Data Analysis) / Claude / Copilot | 코드 없이 자연어로 분석. "AI 기반"의 핵심 |
| 파이썬 | Google Colab + Pandas + matplotlib/seaborn | 설치 부담 없는 웹 실습. 자료집 1·2부와 연결 |
| BI | **Power BI Desktop**(주, Windows 전용), **Looker Studio**(무료 게시·공유·Mac 대안) | Power BI는 엑셀 native·무료 Desktop이나 게시/공유·Copilot은 조직계정·Fabric/Premium 필요 → 무료 배포는 Looker Studio |

> 주의(critic 반영): Power BI Copilot·Service 게시는 유료/조직계정 전제. Mac 수강생과 무료 배포는 Looker Studio 경로. 상세는 SESSIONS_SPEC.md S18.

> 권위 계약은 **SESSIONS_SPEC.md(20차시 v2)**. 아래 구조와 어긋나면 SESSIONS_SPEC이 우선.

## 4. 커리큘럼 구조 (6 파트 / 20차시)

각 차시 = 학습목표 3개 이내 + 사용 데이터 + 따라 하기 + 직접 해보기 + 한 줄 정리.

### Part 0. 오리엔테이션 & 데이터 마인드셋 (1차시)
- 0-1. 데이터 분석이란 무엇이고, 엑셀만으로는 왜 벽에 부딪히나 / AI 시대 분석가의 작업 흐름(엑셀→AI→파이썬→BI) 지도 그리기
- 사용 데이터: `cafe_sales_clean`(맛보기)

### Part 1. 엑셀에서 데이터 감각 잡기 (3차시)
- 1-1. 좋은 데이터의 모양 — tidy data, 한 셀 한 값, 병합셀이 왜 독인가, 데이터 척도(범주/순서/수치)
- 1-2. 피벗테이블 — 집계·그룹·교차표로 "질문에 답하기"
- 1-3. 실무 함수 — VLOOKUP/XLOOKUP, 조건부(IF/SUMIF/COUNTIF), 엑셀의 한계 체험(큰 데이터·반복작업)
- 사용 데이터: `cafe_sales_clean`, `shop_orders`(3테이블)

### Part 2. 숫자에 속지 않는 통계 직관 (2차시)
- 2-1. 중심(평균·중앙값·최빈)·산포(범위·표준편차)·분포·박스플롯 — 수식보다 그림
- 2-2. 상관 vs 인과, 이상치, p-value/신뢰구간의 감(感), 심슨의 역설
- 사용 데이터: `cafe_sales_clean`, `hr_employees`

### Part 3. AI로 분석하기 (노코드) (3차시)
- 3-1. 엑셀을 AI에 올리고 자연어로 분석시키기 — 요약·집계·추세
- 3-2. 좋은 분석 프롬프트 설계 — 맥락·질문·출력형식 3요소 / 이상치·상관 뽑기
- 3-3. **AI 답 검증하기** — 피벗으로 교차확인, 환각·계산오류 잡기, AI가 만든 차트 다듬기
- 사용 데이터: `cafe_sales_dirty`, `survey_responses`

### Part 4. 파이썬 & Pandas 제대로 (5차시)
- 4-1. Colab 환경 + 파이썬 기초 문법(변수·자료형·자료구조·조건/반복/함수)
- 4-2. Pandas 입문 — Series/DataFrame, 읽기·탐색(info/describe/head), 선택·필터·정렬
- 4-3. 데이터 정제 — 결측치·이상치(IQR)·타입변환·중복·문자열(.str)
- 4-4. 그룹화·집계·재구조화 — groupby·pivot_table·merge(=VLOOKUP)·melt
- 4-5. 시각화 — matplotlib/seaborn로 히스토그램·박스플롯·산점도·히트맵·시계열
- 사용 데이터: `cafe_sales_dirty`, `shop_orders`, `web_traffic`

### Part 5. BI 대시보드 (핵심, 4차시)
- 5-1. Power BI Desktop 설치·엑셀 로드, 화면 구조, 첫 차트
- 5-2. 데이터 모델링 — 테이블 관계 맺기(주문↔고객↔상품), 스타 스키마 감
- 5-3. 시각화·상호작용 — 슬라이서·드릴다운·KPI 카드, DAX 기초 측정값(합계·비율·전월대비)
- 5-4. Copilot로 대시보드 가속 + 배포·공유 / (비교) Looker Studio 한 바퀴
- 사용 데이터: `shop_orders`, `web_traffic`, `cafe_sales_clean`

### Part 6. 종합 캡스톤 (2차시)
- 6-1. 풀사이클 실습 — 원데이터 → 엑셀 정리 → AI 분석 → 파이썬 심화 → Power BI 대시보드
- 6-2. 인사이트 보고 — "그래서 뭘 하면 되나" 한 장 요약, 발표
- 사용 데이터: `capstone_*`(통합 케이스)

## 5. 데이터셋 설계 (상세는 DATASETS_MANIFEST.md)

| 데이터셋 | 도메인 | clean | dirty | 주 용도 |
|---|---|---|---|---|
| cafe_sales | 카페 매출(메인 골격) | ✅ | ✅ | 전 파트 관통 |
| shop_orders | 쇼핑몰 주문/고객/상품 3테이블 | ✅ | — | merge/관계, BI 모델링 |
| survey_responses | 설문 응답 | — | ✅ | 범주·텍스트, AI 분석 |
| hr_employees | 인사 데이터(심슨역설 실증) | ✅ | ✅ | 분포·상관·통계 |

> web_traffic·capstone 합성셋은 폐기(생성 안 함). 시계열은 실데이터(FRED 물가·seoul_subway)로, 캡스톤은 실데이터 케이스(상권 cafe_restaurant / 강남 아파트 molit_apt)로 대체 — SESSIONS_SPEC v2 기준.

- 모든 파일은 **BI-ready**(시트당 표 1개, 헤더 1행, 병합셀 금지, long format)로 제작.
- dirty 버전은 함정을 **의도적으로** 심는다: 결측·중복·형식불일치(날짜/숫자)·오타 범주·이상치·공백·혼합타입.
- 재현성: 모든 데이터는 `02_datasets/_generators/`의 파이썬 스크립트로 생성(시드 고정). 원본이 날아가도 재생성 가능.

## 7. 제작 순서 (워커 분산 전제)

1. **[현재] Phase 0** — 골격·마스터플랜·데이터셋 매니페스트 + 대표 데이터셋 1종(cafe_sales) 실제 생성으로 품질 증명
2. Phase 1 — 나머지 데이터셋 전량 생성(생성 스크립트 병렬)
3. Phase 2 — 차시별 커리큘럼 문서 초안 → 윤문
4. Phase 3 — 핸드아웃 + 강사 대본 → 윤문
5. Phase 4 — 슬라이드 덱(deck-factory)
6. Phase 5 — 모범답안·해설, 전체 정합성 점검

> 각 Phase는 사용자 확인 후 진행. "완료" 과장 금지 — 실제 파일 생성/검증 결과로만 보고.
