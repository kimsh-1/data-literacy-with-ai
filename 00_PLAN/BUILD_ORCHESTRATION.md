# 대규모 병렬 빌드 오케스트레이션 (실행 계획)

> 지시: 데이터 수집 함대 완료 후 → 전체 수업을 대규모 병렬로 빌드. **"완료했다"는 주장을 믿지 말고 전부 실증 검증.** 산출물 전량 단일 폴더 `D:\ai-data-analysis-course\`.
> 단일 홈 폴더: `/mnt/d/ai-data-analysis-course/` (이미 존재, 여기에 전부 저장)

## 실행 주체 배분 (워커 분산)
- **메인(오케스트레이터)**: 단계 전환·통합·최종 판정. 나.
- **Workflow 팬아웃 에이전트**: 차시별 생성·검증·윤문 병렬.
- **codex 함대(동시성 15)**: 배치 생산·수리 분산.
- **전용 에이전트 타입**: 윤문=`gn-voice-monolith`/`humanize-monolith`, 내용불변검증=`content-fidelity-auditor`, AI티=`ai-tell-detector`/`naturalness-reviewer`, 논리검증=`critic`.

## Phase 0 — 데이터 통합 (함대 완료 직후, 메인)
1. `02_datasets/real_korean/*/` 수집물 집계 → 진짜 파일만 남기고 (pandas 재검증) `REAL_DATA_CATALOG.md` 작성.
2. 부족 도메인은 합성 데이터셋(gen_*.py)으로 보강 (cafe_sales 외 shop_orders·survey·web_traffic·hr·capstone).
3. **데이터 확정 목록** → Phase 1 설계 입력.

## Phase 1 — 전체 논리성 + 난이도 설계 (메인 + critic 3표 검증)
- 산출: `00_PLAN/SESSIONS_SPEC.md` — 18차시 잠금 스펙. 차시마다:
  `{번호, 제목, 학습목표(≤3), 선수차시(연결성), 사용데이터셋(실/합성), 난이도(1~5), 핵심실습, 미션, 예상시간}`
- 난이도 곡선 **단조 증가** 설계, 데이터셋 재사용 맵(같은 데이터가 파트 관통).
- `critic` 에이전트 3표 다수결로 논리성·순서 검증 → 반려 시 수정 후 재검.
- **이 스펙이 팬아웃의 유일 계약.**

## Phase 2 — 차시별 콘텐츠 생성 (팬아웃: 차시당 1에이전트 ×18 병렬)
각 에이전트가 자기 차시 폴더에만 씀(파일 격리):
- `01_curriculum/S{n}_*.md` — 커리큘럼(학습목표·흐름·개념설명)
- `03_handouts/S{n}_*.md` — 수강생 핸드아웃(따라하기 블록 / 직접해보기 미션)
- `04_scripts/S{n}_*.md` — 강사 대본(말할 것·시연 포인트·예상질문)
- `06_answers/S{n}_*.md` + 실습코드 — 모범답안/해설
- **실데이터 실제 로드해 코드가 돌아가는지 확인**한 것만 제출.

## Phase 3 — 논리성·연결성 점검 (팬아웃 검증 + 적대적, 완료 불신)
- 차시별 검증 에이전트: 선수지식 충족? 다음 차시 연결? 난이도 급점프? 데이터 일관? 코드 실제 실행되나?
- cross-차시 정합성 1에이전트: 전체 흐름 갭/중복/모순/난이도 역전 적발.
- **실증**: 주장 말고 실제 파일 열람 + 실습코드 실행 결과로 판정.
- 결함 리스트 → codex 배치로 수리 → 재검(dry될 때까지).

## Phase 4 — 윤문 (팬아웃 per 콘텐츠)
- 학생/강사용 산문 전량: `gn-voice-monolith` 또는 `humanize-monolith` 태움.
- 직후 `content-fidelity-auditor`로 **의미·수치·개념 불변** 검증(과윤문/내용훼손 롤백).
- 제외: 코드·데이터·스펙·내부문서(MASTERPLAN/이 문서)는 윤문 안 함.

## Phase 5 — 슬라이드 + 최종 정합성 감사
- `deck-factory`로 파트/모듈별 슬라이드 덱 → `05_slides/`.
- 최종 감사 에이전트(deck-factory-audit 교훈): 산출물 인벤토리 대조, **껍데기·빈슬롯·미완 적발**. 통과 못 하면 final 아님.

## 콘텐츠 작성 규칙 (전 Phase 강제)
- **이모지 전면 금지.** 커리큘럼·핸드아웃·강사대본·슬라이드·모범답안·내부문서 어디에도 이모지를 쓰지 않는다. 원자료(자료집 1·2부)가 이모지 헤더를 썼더라도 이식하지 않는다. 강조는 텍스트/볼드/번호로만.
- 불릿 남발·기계적 병렬·과한 구분선 자제. 사람이 쓴 강의노트 톤.
- 영어 인용·번역투 최소화(윤문 단계에서 재검).

## 게이트 원칙 (재점검 프로토콜)
1. 어떤 단계도 "완료" 텍스트로 통과시키지 않는다 — 파일 존재 + 내용 실검 + 코드 실행으로만.
2. 자기구현 불신: 생성 에이전트와 검증 에이전트 분리, 검증은 적대적으로.
3. 게이트 3함정(합성출처·무측정채점·스텁export) 경계.
