## 12-22분. DataFrame과 Series

시연 코드:

```python
print(type(df))

amount_series = df["amount"]
print(type(amount_series))
print(len(amount_series))
```

말할 것:

DataFrame은 표 전체입니다. 엑셀로 치면 시트 한 장입니다. Series는 그 안에 들어 있는 열 하나입니다. 그래서 `df["amount"]`라고 쓰면 표 전체에서 금액 열만 꺼내는 뜻입니다.

처음부터 DataFrame, Series라는 이름을 완벽하게 외우려고 하면 어렵습니다. 오늘은 DataFrame은 표 전체, Series는 열 하나라는 감각만 정확히 잡아도 충분합니다.

칠판 또는 화면에 적을 비교:

| 엑셀 | pandas |
|---|---|
| 시트 전체 | DataFrame |
| 열 하나 | Series |
| 머리글 | columns |
| 행 번호 | index |

질문 유도:

`df["amount"]`는 표 전체일까요, 열 하나일까요?

기대 답:

열 하나입니다. 그래서 Series입니다.

## 22-32분. head, info, describe로 첫 점검하기

시연 코드:

```python
print(df.head())
df.info()
print(df[["unit_price", "qty", "amount"]].describe())
```

말할 것:

`head()`는 맨 위 몇 행을 보는 명령입니다. 엑셀 파일을 열고 상단 몇 줄을 눈으로 확인하는 행동과 같습니다. 데이터가 제대로 열렸는지, 열 이름이 예상대로 보이는지 먼저 보는 겁니다.

`info()`는 행수, 열수, 빈칸 여부, 데이터 타입을 보여줍니다. 이번 데이터는 10,424행이고 모든 열이 10,424개 non-null로 보입니다. non-null은 비어 있지 않다는 뜻입니다. 즉 이번 정제 파일에는 빈칸이 없습니다.

`describe()`는 숫자 열의 기초 통계를 보여줍니다. 키를 잴 때 최솟값, 최댓값, 평균을 보는 것처럼, 숫자 열이 어느 범위에 있는지 빠르게 확인하는 도구입니다. 금액의 최소값은 3,000원, 최대값은 19,500원입니다. 수량은 1개에서 3개 사이로 보입니다.

시연 포인트:

- `df.info()`는 결과를 화면에 직접 출력하므로 `print(df.info())`를 쓰면 마지막에 `None`이 보일 수 있다고 알려준다.
- `date`가 문자열로 보이는 점을 짚되, 날짜 변환은 다음 차시에서 다룬다고 선을 긋는다.

## 32-44분. 강남점만 남기는 불리언 필터

시연 코드:

```python
print(df["store"].value_counts())
```

말할 것:

먼저 지점 열에 어떤 값이 있는지 확인해 봅니다. 사전에서 단어가 몇 번 나왔는지 세듯이, `value_counts()`는 값별 개수를 세어 줍니다. 강남점은 6,196행, 홍대점은 4,228행입니다.

시연 코드:

```python
gangnam_mask = df["store"] == "강남점"

print(gangnam_mask.head(10))
```

말할 것:

이 코드는 아직 데이터를 줄인 것이 아닙니다. 각 행이 강남점인지 아닌지만 True와 False로 표시한 상태입니다. 이것을 불리언 마스크라고 부릅니다.

엑셀에서는 지점 열 필터를 열고 `강남점`만 체크합니다. pandas에서는 그 체크 결과를 True/False 열로 먼저 만든다고 생각하시면 됩니다. 조건에 맞으면 True, 아니면 False입니다.

시연 코드:

```python
gangnam = df.loc[gangnam_mask]

print(gangnam.shape)
print(gangnam.head())
```

말할 것:

`df.loc[gangnam_mask]`는 True인 행만 남깁니다. 결과 행수는 6,196행입니다. 여기서 비로소 엑셀 필터를 적용한 것과 같은 상태가 됩니다.

강조:

- 조건을 만드는 줄과 조건을 적용하는 줄을 분리해서 보여준다.
- `gangnam_mask` 자체를 출력해 True/False가 보이게 한다.
- 이 시점에서 "엑셀 필터 = 불리언 마스크"를 다시 말한다.

## 44-51분. 금액이 큰 순서로 정렬하기

시연 코드:

```python
display_cols = ["order_id", "date", "store", "category", "item", "qty", "amount"]

gangnam_sorted = gangnam.sort_values(
    ["amount", "date", "order_id"],
    ascending=[False, True, True],
)

print(gangnam_sorted.loc[:, display_cols].head(10))
```

말할 것:

엑셀에서 금액 열을 내림차순으로 정렬하는 것과 같습니다. `ascending=False`는 큰 값이 위로 오게 하라는 뜻입니다. 금액이 큰 주문부터 보고 싶을 때 쓰는 방식입니다.

정렬 기준이 세 개 있는 이유는 같은 금액이 여러 번 나올 수 있기 때문입니다. 먼저 금액이 큰 순서로 보고, 같은 금액이면 날짜와 주문번호 순서로 보게 했습니다. 오늘은 `amount`를 내림차순으로 정렬한다는 감각을 먼저 잡으면 됩니다.

시연 포인트:

- `display_cols`는 원본 데이터를 바꾸는 것이 아니라 화면에 보여줄 열만 고르는 장치라고 설명한다.
- `sort_values`는 원본 `df`를 자동으로 덮어쓰지 않는다고 말한다. 정렬 결과를 계속 쓰려면 변수에 담아야 한다.

## 51-56분. loc와 iloc 비교하기

시연 코드:

```python
print(df.loc[0, "store"])
print(df.iloc[0, 4])
```

말할 것:

둘 다 `홍대점`이 나옵니다. 하지만 고르는 방식은 다릅니다. `.loc`는 이름 기준이고, `.iloc`는 위치 기준입니다.

시연 코드:

```python
print(df.loc[0:2, ["date", "store", "amount"]])
print(df.iloc[0:3, [1, 4, 9]])
```

말할 것:

입문 단계에서는 `.loc`가 더 읽기 쉽습니다. `store`, `amount`처럼 컬럼명을 직접 쓰기 때문입니다. `.iloc`는 몇 번째 행, 몇 번째 열인지 위치를 알아야 합니다. 열 순서가 바뀌면 헷갈릴 수 있습니다.

주의:

`.loc[0:2]`는 2가 포함되고, `.iloc[0:3]`은 3이 포함되지 않습니다. 오늘은 이 차이를 억지로 외우기보다, `.loc`는 이름 기준, `.iloc`는 위치 기준이라는 점을 먼저 잡습니다.

미션:

강남점 주문 중 카테고리가 커피이고, 주문 금액이 10,000원 이상인 행만 추출합니다. 그 결과를 금액이 큰 순서로 정렬하고, 행수와 총금액과 평균금액을 출력해 봅니다.

- 결과 행수, 총금액, 평균금액이 모두 출력되어야 한다.

## 예상 질문과 답

### 질문 1. 왜 `df["store"] = "강남점"`이 아니라 `==`를 쓰나요?

답:

`=`는 값을 넣는 기호입니다. `==`는 같은지 비교하는 기호입니다. 필터 조건은 값을 바꾸는 일이 아니라, 이 행의 지점이 강남점인지 묻는 일입니다. 그래서 `==`를 씁니다.

### 질문 2. 조건을 여러 개 붙일 때 왜 `and`가 아니라 `&`를 쓰나요?

답:

pandas에서는 조건 하나가 True/False 하나로 끝나지 않습니다. 행이 10,424개이면 True/False도 10,424개가 생깁니다. 이렇게 True/False로 된 열 전체끼리 조건을 합칠 때는 `&`를 씁니다. 그리고 각 조건은 괄호로 감싸 주세요.

### 질문 3. `loc`와 `iloc` 중 무엇을 먼저 쓰면 되나요?

답:

입문 단계에서는 `.loc`를 먼저 쓰면 됩니다. 컬럼명을 직접 쓰기 때문에 코드가 더 읽기 쉽습니다. `.iloc`는 행과 열의 숫자 위치가 꼭 필요할 때 사용합니다.

### 질문 4. `head()`로 본 5행이 전체를 대표하나요?

답:

대표한다고 보면 안 됩니다. `head()`는 파일이 제대로 열렸는지, 컬럼이 예상대로 있는지 확인하는 첫 점검 도구입니다. 앞의 5행만 보고 전체 매출이 이렇다고 판단하면 위험합니다. 대표성이나 분포는 `describe`, `value_counts`, 이후 집계로 확인합니다.

### 질문 5. 정렬하면 원본 데이터 순서가 바뀌나요?

답:

`sort_values` 결과를 새 변수에 담으면 원본 `df`는 그대로 있습니다. 원본을 바꾸려면 덮어쓰는 코드를 직접 써야 합니다. 오늘은 원본을 유지하고 `gangnam_sorted`, `mission` 같은 새 변수에 담습니다.
