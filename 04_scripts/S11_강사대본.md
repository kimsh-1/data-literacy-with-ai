## 12-22분. DataFrame과 Series

시연 코드:

```python

```

말할 것:

DataFrame은 표 전체다. 엑셀로 치면 시트 한 장이다. Series는 그 안의 열 하나여서, `df["amount"]`라고 쓰면 금액 열만 꺼낸다.

처음부터 자료구조 이름을 외우려고 하면 어렵다. 오늘은 DataFrame은 표, Series는 열이라는 감각만 정확히 잡으면 된다.

칠판 또는 화면에 적을 비교:

| 엑셀 | pandas |
|---|---|
| 시트 전체 | DataFrame |
| 열 하나 | Series |
| 머리글 | columns |
| 행 번호 | index |

질문 유도:

`df["amount"]`는 표 전체인가, 열 하나인가.

기대 답:

열 하나다. 그래서 Series다.

## 32-44분. 강남점 불리언 필터

시연 코드:

```python
print(df["store"].value_counts())
```

말할 것:

먼저 지점에 어떤 값이 있는지 확인한다. 강남점은 6,196행, 홍대점은 4,228행이다.

시연 코드:

```python
gangnam_mask = df["store"] == "강남점"

print(gangnam_mask.head(10))
```

말할 것:

이 코드는 데이터를 아직 줄이지 않았다. 각 행이 강남점인지 아닌지만 True와 False로 표시했다. 이것을 불리언 마스크라고 부른다.

엑셀에서는 지점 열 필터에서 `강남점`만 체크한다. pandas에서는 그 체크 결과를 True/False 열로 먼저 만든다고 생각하면 된다.

시연 코드:

```python
gangnam = df.loc[gangnam_mask]

print(gangnam.shape)
print(gangnam.head())
```

말할 것:

`df.loc[gangnam_mask]`는 True인 행만 남긴다. 결과 행수는 6,196행이다. 여기서 비로소 엑셀 필터를 적용한 것과 같은 상태가 된다.

강조:

- 조건을 만드는 줄과 조건을 적용하는 줄을 분리해서 보여준다.
- `gangnam_mask` 자체를 출력해 True/False가 보이게 한다.
- 이 시점에서 "엑셀 필터 = 불리언 마스크"를 다시 말한다.

## 44-51분. 금액 정렬

시연 코드:

```python
display_cols = ["order_id", "date", "store", "category", "item", "qty", "amount"]

gangnam_sorted = gangnam.sort_values(
    ["amount", "date", "order_id"],
    ascending=[False, True, True],
)

print(gangnam_sorted.loc[:, display_cols].head(10))
```

말할 것:

엑셀에서 금액 열을 내림차순 정렬하는 것과 같다. `ascending=False`는 큰 값이 위로 오게 한다는 뜻이다.

정렬 기준이 세 개 있는 이유는 같은 금액이 많기 때문이다. 먼저 금액이 큰 순서로 보고, 같은 금액이면 날짜와 주문번호 순서로 보게 했다. 오늘의 핵심은 첫 번째 기준인 `amount` 내림차순이다.

시연 포인트:

- `display_cols`는 원본 데이터를 바꾸는 것이 아니라 화면에 보여줄 열만 고르는 장치라고 설명한다.
- `sort_values`는 원본 `df`를 자동으로 덮어쓰지 않는다고 말한다. 결과를 변수에 담아야 계속 쓸 수 있다.

## 51-56분. loc와 iloc

시연 코드:

```python
print(df.loc[0, "store"])
print(df.iloc[0, 4])
```

말할 것:

둘 다 `홍대점`이 나온다. 하지만 `.loc`는 이름 기준이고, `.iloc`는 위치 기준이다.

시연 코드:

```python
print(df.loc[0:2, ["date", "store", "amount"]])
print(df.iloc[0:3, [1, 4, 9]])
```

말할 것:

입문 단계에서는 `.loc`가 더 읽기 쉽다. `store`, `amount`처럼 컬럼명을 직접 쓰기 때문이다. `.iloc`는 위치를 정확히 알아야 하므로 열 순서가 바뀌면 헷갈릴 수 있다.

주의:

`.loc[0:2]`는 2가 포함되고, `.iloc[0:3]`은 3이 포함되지 않는다. 오늘은 깊게 외우기보다, `.loc`는 이름 기준, `.iloc`는 위치 기준이라는 차이를 잡는다.

## 56-60분. 직접 해보기 안내

미션:

강남점 주문 중 카테고리가 커피이고, 주문 금액이 10,000원 이상인 행만 추출한다. 그 결과를 금액 큰 순서로 정렬하고, 행수와 총금액과 평균금액을 출력한다.

힌트 코드:

```python
mission_mask = (
    (df["store"] == "강남점")
    & (df["category"] == "커피")
    & (df["amount"] >= 10000)
)
```

수강생에게 안내할 점:

- 조건이 3개이므로 각 조건을 괄호로 감싼다.
- 조건 사이에는 `&`를 쓴다.
- `and`를 쓰면 pandas Series 조건에서는 오류가 난다.
- 결과 행수, 총금액, 평균금액이 모두 출력되어야 한다.

완료 판단 체크포인트:

수강생이 `강남점`, `커피`, `amount >= 10000` 조건으로 1,362행을 추출하고 총금액 16,664,500원과 평균금액 12,235.32원을 출력하면 완료로 본다.

## 예상 질문과 답

### 질문 1. 왜 `df["store"] = "강남점"`이 아니라 `==`를 쓰나요?

답:

`=`는 값을 넣는 기호이고, `==`는 같은지 비교하는 기호다. 필터 조건은 값을 바꾸는 일이 아니라 같은지 묻는 일이므로 `==`를 쓴다.

### 질문 2. 조건을 여러 개 붙일 때 왜 `and`가 아니라 `&`를 쓰나요?

답:

pandas에서는 조건 하나가 True/False 하나가 아니라 True/False로 된 열 전체다. 이 열끼리 조건을 결합할 때는 `&`를 쓰고, 각 조건은 괄호로 감싼다.

### 질문 3. `loc`와 `iloc` 중 무엇을 먼저 쓰면 되나요?

답:

입문 단계에서는 `.loc`를 먼저 쓰면 된다. 컬럼명을 직접 쓰기 때문에 코드가 더 읽기 쉽다. `.iloc`는 행과 열의 숫자 위치가 필요할 때 사용한다.

### 질문 4. `head()`로 본 5행이 전체를 대표하나요?

답:

대표한다고 보면 안 된다. `head()`는 파일이 제대로 열렸는지, 컬럼이 예상대로 있는지 확인하는 첫 점검 도구다. 대표성이나 분포는 `describe`, `value_counts`, 이후 집계로 확인한다.

### 질문 5. 정렬하면 원본 데이터 순서가 바뀌나요?

답:

`sort_values` 결과를 새 변수에 담으면 원본 `df`는 그대로 있다. 원본을 바꾸려면 덮어쓰는 코드를 직접 써야 한다. 오늘은 원본을 유지하고 `gangnam_sorted`, `mission` 같은 새 변수에 담는다.
