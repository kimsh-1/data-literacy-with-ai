자료 작성 전 pandas로 확인한 기준값:

```text
orders: 20,000행

products: 200행
주문 기간: 2024-01-01 ~ 2024-12-31
날짜테이블 필요 행수: 366행
전체 매출: 1,938,489,803
2024-12 매출: 225,576,457
2024-12 전월대비 성장률: 1.14%
```

## 0-7분. 도입과 S16 모델 복습

말할 것:

지난 시간에는 여러 테이블을 하나의 모델로 엮었다. 오늘은 그 모델을 대시보드로 펼쳐 보이고, 클릭하면 숫자가 함께 움직이도록 만든다. 엑셀로 치면 피벗테이블 여러 개를 한 화면에 늘어놓고 슬라이서로 묶는 작업이다.

오늘 눈여겨볼 차이는 계산식이 어디에 놓이느냐다. 엑셀은 셀에 수식을 적지만, Power BI는 모델 안에 측정값을 만들어 둔다. 이 측정값 하나를 카드와 표, 그래프에서 두고두고 다시 쓴다.

시연:

1. `orders`, `customers`, `products`가 불러와져 있는지 확인한다.
2. 모델 보기에서 `customers[customer_id]`와 `orders[customer_id]` 관계를 확인한다.
3. `products[product_id]`와 `orders[product_id]` 관계를 확인한다.
4. 날짜테이블과 `orders[order_date]` 관계를 확인한다.

강조:

주문 테이블에는 매출 컬럼이 아예 없다. 매출은 상품 단가와 수량, 할인율을 조합해 직접 만들어 내야 한다. 바로 이 지점이 오늘 DAX를 꺼내 드는 이유다.

## 7-17분. 기본 측정값 만들기

시연 DAX:

```DAX
총매출 =
SUMX(
    orders,
    RELATED(products[price]) * orders[qty] * (1 - orders[discount_rate])
)
```

말할 것:

`SUMX`는 주문 행을 한 줄씩 훑으며 계산하고, 맨 끝에서 그 값을 전부 더한다. 엑셀에서 보조열에 행별 매출을 만들어 두고 합계를 내던 일을 한 번에 처리한다고 보면 된다.

`RELATED(products[price])`는 지금 이 주문의 상품ID와 연결된 상품 테이블에서 단가를 끌어온다는 뜻이다. 그러니 모델 관계가 먼저 잡혀 있어야 한다.

시연:

1. `orders` 테이블을 선택한다.
2. `모델링 > 새 측정값`을 누른다.
3. 총매출 측정값을 입력한다.
4. 카드 시각화에 넣어 1,938,489,803이 나오는지 확인한다.

이어 만들 측정값:

```DAX
주문건수 =
DISTINCTCOUNT(orders[order_id])
```

```DAX
총수량 =
SUM(orders[qty])
```

```DAX
평균주문금액 =
DIVIDE([총매출], [주문건수])
```

말할 것:

나눗셈은 웬만하면 `DIVIDE`로 처리한다. 분모가 비어 있거나 0이어도 오류를 덜 낸다. 입문 단계에서는 비율이든 평균이든 `DIVIDE`로 만든다고 외워 두면 된다.

## 17-27분. 매출비중 측정값

시연 DAX:

```DAX
매출비중 =
DIVIDE(
    [총매출],
    CALCULATE([총매출], ALL(products[category]))
)
```

말할 것:

카테고리별 매출비중은 지금 카테고리의 매출을 전체 카테고리 매출로 나눈 값이다. 표에서 디지털 행을 짚고 있다면 분자는 디지털 매출, 분모는 모든 카테고리를 합친 매출이 된다.

`CALCULATE`는 계산 맥락 자체를 바꿔 주는 함수다. 오늘은 여기까지만 하고 깊이 파고들지 않는다. 지금은 "분모를 전체 카테고리 기준으로 다시 계산한다" 정도로만 잡아 두면 충분하다.

시연:

1. 테이블 시각화를 만든다.
2. `products[category]`, `[총매출]`, `[매출비중]`을 넣는다.
3. 매출 내림차순으로 정렬한다.
4. 디지털 34.76%가 나오는지 확인한다.

## 27-40분. 카드, 선그래프, 막대그래프, 등급 슬라이서

시연:

1. 카드 3개를 만든다. 총매출, 주문건수, 평균주문금액을 배치한다.
2. 꺾은선형 차트를 만들고 X축에 `날짜테이블[연월]`, Y축에 `[총매출]`을 넣는다.
3. 막대그래프를 만들고 축에 `products[category]`, 값에 `[총매출]`을 넣는다.
4. 슬라이서를 만들고 `customers[grade]`를 넣는다.

말할 것:

슬라이서는 화면 위에 얹어 놓는 필터 버튼이다. 등급에서 VIP를 누르면 모든 시각화가 VIP 고객 기준으로 새로 계산된다. 여기서 놓치면 안 되는 것은, 숫자가 그냥 가려지는 게 아니라 측정값이 그 필터 조건 아래에서 처음부터 다시 계산된다는 점이다.

확인 질문:

VIP를 눌렀을 때 총매출 카드가 393,432,930 언저리로 바뀌는지 함께 보자. 실버로 바꾸면 603,583,519 근처가 나온다. 이 값이 안 맞으면 관계가 끊겼거나, 슬라이서가 엉뚱한 필드를 보고 있을 공산이 크다.

## 40-50분. 드릴다운과 상호작용 편집

시연:

1. 카테고리 막대그래프의 축에 `products[product_name]`을 두 번째 필드로 추가한다.
2. 드릴다운 아이콘을 켠다.
3. `디지털` 막대를 눌러 상품명 수준으로 내려간다.
4. 위로 올라가기 아이콘으로 돌아온다.

말할 것:

드릴다운은 큰 묶음에서 작은 항목으로 한 칸 내려가는 기능이다. 전체 카테고리를 훑고 나서 디지털 안쪽에서 어떤 상품이 큰지 곧바로 이어 보는 흐름이다.

상호작용 시연:

1. 카테고리 막대그래프를 선택한다.
2. `서식 > 상호 작용 편집`을 누른다.
3. 월별 선그래프에 필터, 강조, 없음 아이콘이 나타나는 것을 보여준다.
4. 필터와 강조의 차이를 클릭해 보여준다.

말할 것:

대시보드는 한 곳을 클릭하면 다른 차트가 따라 바뀐다. 이 연동은 기본값으로 알아서 생기지만, 잘 만든 대시보드는 여기에 그냥 맡기지 않고 어디를 어디에 걸지 직접 정해 둔다. 오늘은 카테고리를 누르면 월별 선그래프와 KPI 카드가 그 카테고리 기준으로 바뀌도록 잡아 둔다.

## 50-60분. 전월대비 DAX

말할 것:

이제 시간 계산으로 넘어간다. 전월대비는 그냥 바로 위 행을 빼는 게 아니라, 날짜테이블을 기준 삼아 현재 달의 전월이 어디인지 찾아 계산한다. S16에서 날짜테이블을 미리 만들어 둔 이유가 여기에 있다.

시연 DAX:

```DAX
전월매출 =
CALCULATE(
    [총매출],
    SAMEPERIODLASTMONTH('날짜테이블'[Date])
)
```

```DAX
전월대비증감액 =
[총매출] - [전월매출]
```

```DAX
전월대비성장률 =
DIVIDE([전월대비증감액], [전월매출])
```

시연:

1. 월별 테이블을 만든다.
2. `날짜테이블[연월]`, `[총매출]`, `[전월매출]`, `[전월대비성장률]`을 넣는다.
3. 2024-01은 전월매출이 비어 있음을 보여준다.
4. 2024-11 성장률 40.43%, 2024-12 성장률 1.14%를 확인한다.

말할 것:

1월의 전월매출이 비어 있는 건 오류가 아니다. 이 데이터에도, 날짜테이블에도 2023년 12월이 아예 없어서 그렇다. 알 수 없는 값을 억지로 0으로 메우기보다, 비어 있는 채로 두는 편이 오히려 정직하다.

카드 시연:

1. 월 슬라이서를 만든다.
2. `2024-12`를 선택한다.
3. 전월대비 성장률 카드를 만들고 1.14%가 표시되는지 확인한다.

강조:

전월대비 카드는 지금 어느 달을 보고 있는지가 정해져야 비로소 읽힌다. 그래서 월 슬라이서와 짝지어 쓰는 편이 안전하다.

## 60-68분. 직접 해보기 안내

미션:

시계열 KPI 대시보드 한 페이지를 완성한다.

수강생에게 안내할 조건:

- 총매출, 주문건수, 평균주문금액, 전월대비 성장률 카드가 있어야 한다.
- 월별 총매출 선그래프가 있어야 한다.
- 고객 등급 슬라이서가 있어야 한다.
- 월 슬라이서에서 2024-12를 선택하면 전월대비 성장률 1.14%가 보여야 한다.
- 카테고리별 매출 막대그래프는 상품명으로 드릴다운되어야 한다.
- 카테고리별 매출비중 표가 있어야 한다.
- 해석 문장 2개를 적어야 한다.

돌아다니며 볼 것:

- `orders[order_date]`가 날짜 형식인지 확인한다.
- 날짜테이블과 주문일 관계가 연결되어 있는지 확인한다.
- `[총매출]`을 계산열이 아니라 측정값으로 만들었는지 확인한다.
- 전월대비 성장률 형식이 백분율인지 확인한다.
- 슬라이서가 `customers[grade]`를 보고 있는지 확인한다.

## 68-70분. 마무리

말할 것:

오늘 만든 대시보드는 그냥 예쁜 그림이 아니다. 관계형 모델 위에 측정값을 올리고, 그 측정값이 필터 맥락에 따라 스스로 다시 계산되도록 짜 놓은 결과물이다. 슬라이서를 누르든 막대를 클릭하든, 같은 측정값이 매번 다른 조건에서 새로 계산된다.

다음 단계에서는 이 대시보드를 남들과 나눠 보기 쉬운 형태로 다듬고, Power BI를 쓰기 힘든 환경을 위해 Looker Studio 경로도 함께 짚어 본다.

완료 판단 체크포인트:

수강생이 월 슬라이서에서 2024-12를 선택했을 때 전월대비 성장률 1.14%를 표시하고, 등급 슬라이서와 카테고리 드릴다운이 함께 작동하는 시계열 KPI 대시보드를 완성하면 완료로 본다.

## 예상 질문과 답

질문: 날짜테이블 없이 `orders[order_date]`만 써도 월별 그래프가 나오는데 왜 날짜테이블이 필요한가.

답: 월별 합계만 보려면 주문일만으로도 된다. 하지만 전월대비나 전년동월대비, 누계 같은 시간 계산은 주문이 없는 날짜까지 채운 기준 달력이 있어야 흔들림 없이 돌아간다. 날짜테이블을 굳이 따로 두는 이유가 그것이다.

질문: 2024-01의 전월매출이 비어 있는 것은 잘못된 것인가.

답: 잘못된 게 아니다. 데이터도 날짜테이블도 2024년부터 시작하니 2023년 12월 값 자체가 없다. 견줄 대상이 없어서 비어 있는 것이니, 이게 정상이다.

질문: 전월대비 성장률 카드가 이상하게 보인다.

답: 카드는 현재 월이라는 맥락이 있어야 제대로 나온다. 월 슬라이서에서 한 달만 고르거나, 월별 표나 선그래프처럼 월이 축으로 들어간 시각화에서 확인한다.

질문: `SUM` 대신 왜 `SUMX`를 쓰는가.

답: 주문 테이블에 매출 컬럼이 없어서다. 행마다 상품 단가와 수량, 할인율을 곱해 매출을 먼저 만든 뒤 합쳐야 하니 `SUMX`가 필요하다.

질문: `RELATED`가 오류가 난다.

답: `orders[product_id]`와 `products[product_id]` 관계가 아예 없거나 엉뚱하게 이어졌을 가능성이 크다. 모델 보기에서 관계부터 확인한다.

질문: 슬라이서를 눌러도 어떤 차트는 바뀌지 않는다.

답: 그 차트의 상호작용이 `없음`으로 잡혀 있거나, 슬라이서와 이어지지 않은 테이블만 바라보고 있을 수 있다. `서식 > 상호 작용 편집`과 모델 관계를 함께 확인한다.
