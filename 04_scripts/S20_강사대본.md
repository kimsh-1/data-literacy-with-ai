```text
분석 결과는 읽는 사람이 움직일 수 있어야 합니다. 지난 시간에는 데이터를 열고, 요약하고, 차트를 만들었습니다. 그런데 실무에서는 "차트를 만들었습니다"에서 끝나지 않습니다. 보고를 받는 사람은 바로 묻습니다. "그래서 뭘 하면 되나요?" 이 질문에 답할 수 있게 바꾸는 작업이 인사이트 보고입니다.
```

시연 포인트:

- 결과물이 없는 수강생은 강사 예시를 사용한다고 안내한다. 이때 수업 홈의 데이터 받기 페이지에서 서울 음식점·카페 상권 파일을 내려받아 쓸 수 있다고 말한다.
- 오늘 제출물은 1페이지 보고서, 3분 발표 원고, 동료 피드백 반영이라고 명확히 말한다.

판서 또는 화면 문장:

```text
발견: 구별 점포수는 강남구 12,095개, 마포구 7,396개, 송파구 7,106개 순으로 많다.
시사점: 상권 후보는 전체 점포수만 보지 말고 구와 업종 조합을 함께 보아야 한다.
액션: 강남구, 마포구, 송파구의 한식과 커피점/카페 점포수를 먼저 비교한다.
```

엑셀 비교 설명:

```text
엑셀에서 피벗테이블을 만들었다고 보고가 끝나는 것은 아닙니다. 피벗테이블은 근거입니다. 보고서는 그 근거를 사람이 판단할 수 있는 말로 바꾼 문서입니다. 피벗테이블이 재료라면, 보고서는 그 재료로 만든 한 접시의 설명이라고 보면 됩니다.
```

## 따라하기 시연 15-30분

### 1. 코드 실행과 숫자 확인

말할 것:

```text
먼저 보고서에 쓸 숫자가 실제 데이터에서 나온 것인지 확인해 봅니다. 보고서에 들어가는 숫자는 기억이나 감으로 쓰면 안 됩니다. 사전에서 단어를 찾듯이, 데이터에서 숫자를 찾아 확인한 뒤에 써야 합니다.
```

터미널에서 실행:

```bash
cd /mnt/d/ai-data-analysis-course
python3 06_answers/S20_code.py
```

시연 포인트:

- 원자료 행수 119,158행을 확인한다.
- 구별 점포수 상위 3개와 구별·업종 조합 상위 값을 확인한다.
- 보고서에는 원자료 전체가 아니라 집계 결과를 넣어야 한다는 점을 짚어 준다. 119,158행을 그대로 보여 주는 것이 아니라, 강남구 12,095개처럼 읽을 수 있는 숫자로 줄여야 한다고 설명한다.
- 강사용 예시 1페이지 보고서가 생성된 것을 보여준다. 학생에게는 파일 경로를 외우게 하지 말고, 결과물이 만들어졌다는 점만 보여준다.

### 2. 제목 고치기

말할 것:

```text
"서울 상권 분석"은 주제입니다. 보고서 제목으로는 아직 약합니다. 제목만 읽어서는 어떤 결론인지 알기 어렵기 때문입니다. 보고서 제목은 결론에 가까워야 합니다. 오늘 예시는 "서울 음식점·카페 상권은 강남구에 가장 집중되어 있으며, 구별 우선순위는 한식과 커피점/카페 조합부터 확인해야 한다"입니다.
```

시연 포인트:

- 제목만 보고도 발표의 방향을 알 수 있는지 묻는다.
- 제목이 길어도 괜찮다고 말해 준다. 두루뭉술한 짧은 제목보다는 구체적인 긴 제목이 낫다.

### 3. 시각화 선택

말할 것:

```text
이 케이스에서 보고 싶은 것은 구와 업종 조합별 점포수 비교입니다. 어느 구에 어떤 업종이 많은지 크기를 비교해야 하므로 막대그래프가 가장 잘 맞습니다. 아파트 대안 예시처럼 두 숫자의 관계를 보는 경우라면 산점도, 즉 점들이 흩어진 그래프와 추세선, 즉 점들의 전체 방향을 보여 주는 선을 씁니다. 그때는 방향과 강도인 상관계수까지만 읽으면 됩니다. 차트는 예쁜 것을 고르는 작업이 아닙니다. 결론을 가장 바로 보여주는 증거를 고르는 작업입니다.
```

시연 포인트:

- S19 구별·업종 점포수 막대그래프 또는 BI 화면을 보고서 중앙에 둔다고 설명한다.
- 차트 3개를 늘어놓기보다 중심 차트 1개와 보조 표 1개가 낫다고 설명한다.

## 3분 발표 구조 30-40분

말할 것:

```text
3분 발표는 생각보다 짧습니다. 분석 과정을 처음부터 설명하면 시간이 부족합니다. 그래서 결론부터 말하고, 그 결론을 받쳐 주는 근거 두 개를 말한 뒤, 마지막에 액션 하나를 말합니다. 듣는 사람이 길을 잃지 않게 표지판을 먼저 세우는 방식입니다.
```

발표 예시:

```text
서울 음식점·카페 상권은 강남구에 가장 집중되어 있으며, 구별 우선순위는 한식과 커피점/카페 조합부터 확인해야 합니다.
119,158행을 집계한 결과 구별 점포수는 강남구 12,095개, 마포구 7,396개, 송파구 7,106개 순입니다.
구별·업종 조합에서는 강남구 한식 3,917개, 마포구 한식 2,835개, 송파구 한식 2,626개, 강남구 커피점/카페 2,253개가 상위에 있습니다.
따라서 상권 조사 담당자는 강남구, 마포구, 송파구를 먼저 비교하고, 각 구에서 한식과 커피점/카페의 경쟁 강도를 확인한 뒤 후보지를 좁히는 방식이 적절합니다.
```

강조:

- 수치 2개 이상
- 차트 1개
- 액션 1개 이상
- 한계 1문장

## 예상 질문과 답

### 질문 1. 차트를 여러 개 넣으면 더 성실해 보이지 않나요?

답:

```text
분석 파일에는 여러 차트가 있어도 됩니다. 하지만 1페이지 보고서에는 결론을 가장 잘 보여주는 차트가 우선입니다. 차트가 많아질수록 읽는 사람이 무엇을 봐야 하는지 놓칠 수 있습니다. 보고서는 만든 것을 모두 보여 주는 장부가 아니라, 판단에 필요한 증거를 골라 보여 주는 문서입니다.
```

### 질문 2. 상권 데이터에서 점포수가 많으면 좋은 상권이라는 뜻인가요?

답:

```text
그렇게 단정하면 안 됩니다. 점포수는 상권 규모와 경쟁 강도를 함께 보여주는 출발점입니다. 점포수가 많다는 것은 사람들이 많이 모이는 지역일 수도 있지만, 이미 경쟁이 센 지역일 수도 있습니다. 그래서 점포수가 많은 구와 업종 조합을 먼저 확인하되, 실제 판단에는 임대료, 유동인구, 매출, 배달 수요, 폐업률 같은 추가 정보가 필요합니다.
```

### 질문 3. 액션을 쓰기 어렵습니다. 어떻게 시작하나요?

답:

```text
누가 무엇을 다음에 할지로 시작해 보세요. 예를 들어 "상권 조사 담당자는 강남구와 마포구의 한식·커피점/카페 점포수를 비교표로 만든다"처럼 쓰면 됩니다. "추가 분석 필요"는 방향이고, 액션은 행동입니다. 보고서를 읽은 사람이 바로 할 수 있는 일을 문장으로 적어야 합니다.
```

### 질문 4. 동료가 제 결론에 동의하지 않으면 어떻게 하나요?

답:

```text
동의 여부보다 근거와 연결이 중요합니다. 동료가 제목을 다르게 이해했다면 제목을 고칩니다. 숫자가 부족하다고 하면 발견을 보강합니다. 액션이 너무 넓다고 하면 대상을 좁힙니다. 피드백은 내 결론을 포기하라는 뜻이 아니라, 읽는 사람이 오해한 지점을 찾아 주는 신호로 보면 됩니다.
```

### 질문 5. 상권 데이터를 선택한 사람도 같은 구조를 쓰면 되나요?

답:

```text
오늘 기본 예시가 그 구조입니다. 발견에는 구별 점포수, 업종별 상위 N개, 구별·업종 조합 같은 집계 숫자를 넣습니다. 시사점은 어느 구와 업종에 집중할지로 바꾸고, 액션은 상권 조사 우선순위나 추가 비교표 작성으로 쓰면 됩니다. 단, 원자료 119,158행을 그대로 차트에 넣지 말고 먼저 집계해야 합니다.
```
