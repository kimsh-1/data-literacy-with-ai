## 5-12분. 한글폰트와 기본 테마 설정

시연 코드:

```python
from pathlib import Path

import matplotlib.pyplot as plt
import pandas as pd
import seaborn as sns
from matplotlib import font_manager

ROOT = Path("/mnt/d/ai-data-analysis-course")
DATA = ROOT / "02_datasets"
OUT = ROOT / "06_answers/S14_outputs"
OUT.mkdir(parents=True, exist_ok=True)

font_candidates = [
    "NanumGothic",
    "NanumGothicCoding",
    "Noto Sans CJK KR",
    "Malgun Gothic",
    "AppleGothic",
    "DejaVu Sans",
]

available_fonts = {font.name for font in font_manager.fontManager.ttflist}
selected_font = next(
    (font for font in font_candidates if font in available_fonts),
    "DejaVu Sans",
)

sns.set_theme(style="whitegrid")
plt.rcParams["font.family"] = selected_font
plt.rcParams["axes.unicode_minus"] = False

print(selected_font)
```

말할 것:

한글 차트에서 글자가 네모로 깨지면 데이터가 잘못된 것이 아니다. matplotlib이 한글 글꼴을 찾지 못한 것이다. 그래서 오늘은 차트를 그리기 전에 글꼴부터 잡는다.

`axes.unicode_minus=False`는 음수 부호가 깨지는 것을 막기 위한 설정이다. 오늘 차트에는 음수가 많지 않지만, 분석 환경을 처음 잡을 때 같이 설정해두면 좋다.

시연 포인트:

- `selected_font`가 `NanumGothic` 또는 `NanumGothicCoding`으로 나오는지 확인한다.
- 수강생 환경에서 `DejaVu Sans`만 나오면 한글이 깨질 수 있다고 안내한다.

## 12-25분. 지하철 시간대 히트맵

시연 코드:

```python
subway = pd.read_csv(
    DATA / "real_korean/seoul_subway/seoulmetro_daily_hourly_2025.csv"
)

time_cols = [
    "06시이전", "06-07시간대", "07-08시간대", "08-09시간대",
    "09-10시간대", "10-11시간대", "11-12시간대", "12-13시간대",
    "13-14시간대", "14-15시간대", "15-16시간대", "16-17시간대",
    "17-18시간대", "18-19시간대", "19-20시간대", "20-21시간대",
    "21-22시간대", "22-23시간대", "23-24시간대", "24시이후",
]

ride = subway[subway["승하차구분"] == "승차"].copy()
ride["역라벨"] = ride["호선"].astype(str) + " " + ride["역명"].astype(str)

station_hour = ride.groupby("역라벨")[time_cols].sum()
top_stations = station_hour.sum(axis=1).sort_values(ascending=False).head(10).index
heatmap_wide = station_hour.loc[top_stations, time_cols]
```

말할 것:

이 파일은 199,424행이다. 이 정도 데이터는 pandas로 읽을 수 있지만, 그대로 그래프로 그리면 좋은 차트가 되지 않는다. 먼저 질문을 정한다. "승차 인원이 많은 주요 역은 시간대별로 어떤 패턴을 보이는가"가 오늘의 질문이다.

그래서 `승하차구분`이 `승차`인 행만 남긴다. 그 다음 역과 시간대별로 합계를 낸다. 마지막으로 총 승차 상위 10개 역만 남긴다. 이것이 큰 데이터를 차트용 표로 줄이는 과정이다.

시연 코드:

```python
fig, ax = plt.subplots(figsize=(13, 6))
sns.heatmap(
    heatmap_wide / 1_000_000,
    cmap="Blues",
    linewidths=0.4,
    cbar_kws={"label": "연간 승차인원(백만 명)"},
    ax=ax,
)
ax.set_title("서울 지하철 주요 역의 시간대별 승차 패턴")
ax.set_xlabel("시간대")
ax.set_ylabel("역")
ax.tick_params(axis="x", labelrotation=45)
plt.tight_layout()
plt.savefig(OUT / "s14_subway_heatmap.png", dpi=160)
```

시연 포인트:

- 히트맵은 피벗 테이블에 조건부 서식을 입힌 것과 같다고 설명한다.
- 색이 진할수록 승차 인원이 많다는 것을 먼저 읽게 한다.
- `weather` 데이터와 조인하지 않는다. 연도 불일치 때문에 이번 수업의 설계 대상이 아니다.

예상 결과:

```text
원본 199,424행
승차 99,645행
히트맵 표 10행 × 20열
상위 역 예: 2호선 잠실(송파구청), 2호선 강남, 2호선 홍대입구, 1호선 서울역
```

말할 것:

국토부 실거래 데이터의 거래금액은 쉼표가 들어간 문자열이다. 계산하고 그래프로 그리려면 숫자로 바꿔야 한다. 만원 단위는 숫자가 너무 커서 차트용으로 억원 단위를 함께 만든다.

```python
fig, ax = plt.subplots(figsize=(9, 5))
```

## 38-48분. 산점도와 추세선

시연 코드:

```python
corr = apt["전용면적(㎡)"].corr(apt["거래금액_만원"])

fig, ax = plt.subplots(figsize=(9, 5.5))
sns.regplot(
    data=apt,
    x="전용면적(㎡)",
    y="거래금액_억원",
    ci=None,
    scatter_kws={"s": 28, "alpha": 0.5},
    line_kws={"color": "#1F2430"},
    ax=ax,
)
ax.set_title(f"전용면적과 거래금액의 관계: r={corr:.3f}")
ax.set_xlabel("전용면적(㎡)")
ax.set_ylabel("거래금액(억원)")
plt.tight_layout()
```

말할 것:

산점도에서는 한 거래가 점 하나다. 오른쪽으로 갈수록 전용면적이 넓고, 위로 갈수록 거래금액이 높다. 점들이 대체로 오른쪽 위로 올라가면 양의 관계가 있다는 뜻이다.

이번 데이터의 상관계수는 `r = 0.806`이다. 강한 양의 상관이다. 하지만 이 숫자는 전용면적만이 가격의 원인이라는 뜻이 아니다. 단지, 위치, 층, 연식, 거래 시점 같은 변수가 함께 작용한다.

시연 포인트:

- 엑셀 산점도에서 추세선을 추가하는 것과 연결한다.
- `ci=None`은 추세선 주변의 신뢰구간 그림을 숨겨 입문자가 선과 점에 집중하게 하는 설정이라고 설명한다.
- 산점도는 원자료 행 하나하나가 의미 있을 때 적합하다고 설명한다.

## 예상 질문과 답변

### Q1. 한글이 계속 깨지면 어떻게 하나요?

A. 먼저 나눔폰트가 설치되어 있는지 확인한다. 설치 후에도 깨지면 matplotlib 폰트 캐시가 이전 상태를 기억하고 있을 수 있다. Colab에서는 런타임을 다시 시작하고, 로컬에서는 캐시 삭제 후 다시 실행한다. 수업에서는 `selected_font` 출력값이 `NanumGothic` 또는 `NanumGothicCoding`인지 확인한다.

### Q2. 지하철 데이터는 왜 전체 역을 다 그리지 않나요?

A. 전체 역을 모두 그리면 글자가 겹쳐 읽을 수 없다. 좋은 시각화는 많은 데이터를 한 화면에 억지로 넣는 것이 아니라 질문에 맞게 줄이는 것이다. 오늘은 총 승차 상위 10개 역의 시간대 패턴을 보는 것이 목표다.

### Q3. 산점도에 점이 너무 많이 겹치면 어떻게 하나요?

A. `alpha`로 투명도를 낮추거나, 필요한 경우 표본을 뽑거나, 구간별 집계로 바꿀 수 있다. 오늘 아파트 데이터는 3,754행이라 투명도 조정만으로도 관계를 읽을 수 있다.

### Q4. 상관계수 0.806이면 전용면적이 가격의 원인인가요?

A. 아니다. 상관은 함께 움직이는 정도를 말할 뿐 원인을 증명하지 않는다. 같은 면적이라도 단지, 위치, 층, 연식, 거래 시점 등이 가격에 영향을 준다.

### Q5. 왜 CSV 저장에 `utf-8-sig`를 쓰나요?

A. 원본 데이터는 UTF-8로 정규화되어 있다. 다만 수강생이 Excel이나 BI 도구에서 파일을 열 때 한글 열 이름이 깨지는 경우가 있어 전달용 CSV는 `utf-8-sig`로 저장한다. 데이터 내용이 바뀌는 것은 아니다.

### Q6. Power BI에서도 같은 차트를 다시 만들 수 있나요?

A. 가능하다. 오늘 저장한 CSV는 BI에서 바로 불러올 수 있는 차트용 표다. S15 이후에는 Power BI나 Looker Studio에서 이 집계표를 불러와 대시보드 차트로 배치할 수 있다.
