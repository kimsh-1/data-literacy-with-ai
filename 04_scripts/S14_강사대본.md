강조:

그래프는 표를 예쁘게 꾸미는 일이 아닙니다. 질문에 답하기 위해 표를 눈으로 읽기 쉬운 모양으로 바꾸는 일입니다. 큰 공공데이터를 통째로 그리려고 하지 말고, 먼저 필요한 크기로 줄여야 합니다. 키를 잴 때 몸 전체를 재는 것이 아니라 키라는 한 가지 길이를 재듯이, 차트도 질문에 필요한 정보만 꺼내서 봅니다.

## 12-25분. 지하철 시간대 히트맵

시연 코드:

```python
subway = pd.read_csv(
    DATA / "real_korean/seoul_subway/seoulmetro_daily_hourly_2025.csv"
)

time_cols = [
    "06시이전", "06-07시간대", "07-08시간대", "08-09시간대",
    "09-10시간대", "10-11시간대", "11-12시간대", "12-13시간대",
    "13-14시간대", "14-15시간대", "15-16시간대", "16-17시간대",
    "17-18시간대", "18-19시간대", "19-20시간대", "20-21시간대",
    "21-22시간대", "22-23시간대", "23-24시간대", "24시이후",
]

ride = subway[subway["승하차구분"] == "승차"].copy()
ride["역라벨"] = ride["호선"].astype(str) + " " + ride["역명"].astype(str)

station_hour = ride.groupby("역라벨")[time_cols].sum()
top_stations = station_hour.sum(axis=1).sort_values(ascending=False).head(10).index
heatmap_wide = station_hour.loc[top_stations, time_cols]
```

말할 것:

이 지하철 파일은 199,424행입니다. 이 정도 데이터는 pandas로 읽을 수 있지만, 그대로 그래프로 그리면 좋은 차트가 되지 않습니다. 역도 많고 날짜도 많고 시간대도 많아서, 화면에 올리는 순간 무엇을 봐야 할지 알기 어렵습니다.

그래서 먼저 질문을 정합니다. 오늘의 질문은 “승차 인원이 많은 주요 역은 시간대별로 어떤 패턴을 보이는가”입니다. 질문이 정해지면 필요한 행과 열이 보입니다.

먼저 `승하차구분`이 `승차`인 행만 남깁니다. 지하철을 탄 사람의 패턴을 보려는 것이기 때문입니다. 그다음 역과 시간대별로 합계를 냅니다. 마지막으로 총 승차 인원이 많은 상위 10개 역만 남깁니다. 이렇게 큰 데이터를 차트용 표로 줄입니다. 큰 책에서 필요한 페이지에 책갈피를 꽂는 것과 비슷합니다.

시연 코드:

```python
fig, ax = plt.subplots(figsize=(13, 6))
sns.heatmap(
    heatmap_wide / 1_000_000,
    cmap="Blues",
    linewidths=0.4,
    cbar_kws={"label": "연간 승차인원(백만 명)"},
    ax=ax,
)
ax.set_title("서울 지하철 주요 역의 시간대별 승차 패턴")
ax.set_xlabel("시간대")
ax.set_ylabel("역")
ax.tick_params(axis="x", labelrotation=45)
plt.tight_layout()
plt.savefig(OUT / "s14_subway_heatmap.png", dpi=160)
```

시연 포인트:

- 히트맵은 피벗 테이블에 조건부 서식을 입힌 것과 같다고 설명한다.
- 색이 진할수록 승차 인원이 많다는 것을 먼저 읽게 한다.
- `weather` 데이터와 조인하지 않는다. 연도 불일치 때문에 이번 수업의 설계 대상이 아니다.

예상 결과:

```text
원본 199,424행
승차 99,645행
히트맵 표 10행 × 20열
상위 역 예: 2호선 잠실(송파구청), 2호선 강남, 2호선 홍대입구, 1호선 서울역
```

말할 것:

여기서 히트맵 표는 10행 × 20열입니다. 행은 역이고, 열은 시간대입니다. 색이 진한 칸은 그 역의 그 시간대에 승차 인원이 많다는 뜻입니다. 숫자가 너무 크기 때문에 `1_000_000`으로 나누어서 백만 명 단위로 보여 줍니다. 1,000원짜리를 100만 원 단위로 묶어서 읽으면 큰돈을 더 쉽게 읽는 것과 같은 원리입니다.

## 25-38분. 히스토그램과 박스플롯

시연 코드:

```python
apt = pd.read_csv(
    DATA / "real_korean/realestate_apt/molit_apt_sale_seoul_gangnam_2024.csv"
)

apt["거래금액_만원"] = pd.to_numeric(
    apt["거래금액(만원)"].astype(str).str.replace(",", "", regex=False),
    errors="coerce",
)
apt["거래금액_억원"] = apt["거래금액_만원"] / 10_000
apt["계약월"] = pd.to_datetime(apt["계약년월"].astype(str), format="%Y%m")
apt = apt.dropna(subset=["전용면적(㎡)", "거래금액_만원", "계약월"]).copy()
```

말할 것:

국토부 실거래 데이터의 거래금액은 쉼표가 들어간 문자열입니다. 예를 들어 숫자처럼 보이지만, 컴퓨터 입장에서는 글자로 들어 있을 수 있습니다. 계산하고 그래프로 그리려면 먼저 진짜 숫자로 바꿔야 합니다.

`str.replace(",", "", regex=False)`는 쉼표를 지우는 과정입니다. 그다음 `pd.to_numeric()`으로 숫자로 바꿉니다. 만원 단위는 숫자가 너무 커서 차트에서 읽기 어렵기 때문에, 억원 단위 열도 함께 만듭니다. 같은 가격표라도 “250,000만 원”보다 “25억 원”이라고 읽는 편이 훨씬 편하죠.

시연 코드:

```python
fig, ax = plt.subplots(figsize=(9, 5))
sns.histplot(data=apt, x="거래금액_억원", bins=36, ax=ax)
ax.set_title("강남구 아파트 거래금액 분포")
ax.set_xlabel("거래금액(억원)")
ax.set_ylabel("거래건수")
plt.tight_layout()
```

말할 것:

히스토그램은 개별 단지를 비교하는 차트가 아닙니다. 값이 어느 구간에 몰려 있는지 보는 차트입니다. 엑셀의 도수분포표를 그래프로 표현한 것과 비슷합니다.

여기서 x축은 거래금액 구간이고, y축은 그 구간에 들어간 거래건수입니다. 막대가 높은 구간은 그 가격대의 거래가 많았다는 뜻입니다. 사람 키를 150cm대, 160cm대, 170cm대처럼 나누어 몇 명씩 있는지 세는 것과 같은 방식입니다.

시연 코드:

```python
apt["면적구간"] = pd.cut(
    apt["전용면적(㎡)"],
    bins=[0, 40, 60, 85, 120, float("inf")],
    labels=["40㎡ 이하", "40-60㎡", "60-85㎡", "85-120㎡", "120㎡ 초과"],
)

fig, ax = plt.subplots(figsize=(10, 5))
sns.boxplot(data=apt, x="면적구간", y="거래금액_억원", ax=ax)
ax.set_title("면적구간별 거래금액의 범위 비교")
ax.set_xlabel("면적구간")
ax.set_ylabel("거래금액(억원)")
plt.tight_layout()
```

말할 것:

박스플롯은 처음 보면 조금 낯설 수 있습니다. 가운데 선은 중앙값입니다. 중앙값은 데이터를 작은 값부터 큰 값까지 줄 세웠을 때 가운데에 있는 값입니다. 평균과 비슷하게 대표값 역할을 하지만, 아주 비싼 거래 한두 건에 덜 흔들립니다.

상자의 위아래는 중간 50% 범위입니다. 점처럼 따로 표시되는 값은 다른 거래보다 멀리 떨어진 값입니다. 평균 하나만 볼 때보다 그룹별 퍼짐을 더 잘 볼 수 있습니다. 면적구간별로 가격이 어느 정도 범위에 있는지 비교할 때 유용합니다.

## 38-48분. 산점도와 추세선

시연 코드:

```python
corr = apt["전용면적(㎡)"].corr(apt["거래금액_만원"])

fig, ax = plt.subplots(figsize=(9, 5.5))
sns.regplot(
    data=apt,
    x="전용면적(㎡)",
    y="거래금액_억원",
    ci=None,
    scatter_kws={"s": 28, "alpha": 0.5},
    line_kws={"color": "#1F2430"},
    ax=ax,
)
ax.set_title(f"전용면적과 거래금액의 관계: r={corr:.3f}")
ax.set_xlabel("전용면적(㎡)")
ax.set_ylabel("거래금액(억원)")
plt.tight_layout()
```

말할 것:

산점도에서는 한 거래가 점 하나입니다. 오른쪽으로 갈수록 전용면적이 넓고, 위로 갈수록 거래금액이 높습니다. 점들이 대체로 오른쪽 위로 올라가면, 전용면적이 넓을수록 거래금액도 높아지는 경향이 있다는 뜻입니다.

이번 데이터의 상관계수는 `r = 0.806`입니다. 강한 양의 상관입니다. 여기서 상관계수는 두 숫자가 함께 움직이는 정도를 -1부터 1 사이의 값으로 나타낸 숫자입니다. 1에 가까울수록 같이 올라가는 관계가 강합니다.

하지만 이 숫자가 전용면적만이 가격의 원인이라는 뜻은 아닙니다. 같은 면적이라도 단지, 위치, 층, 연식, 거래 시점 같은 변수가 함께 작용합니다. 산점도와 상관계수는 관계를 보여 주는 도구이지, 원인을 판결하는 도구는 아닙니다.

시연 포인트:

- 엑셀 산점도에서 추세선을 추가하는 것과 연결한다.
- `ci=None`은 추세선 주변의 신뢰구간 그림을 숨겨 입문자가 선과 점에 집중하게 하는 설정이라고 설명한다.
- 산점도는 원자료 행 하나하나가 의미 있을 때 적합하다고 설명한다.

`observation_date`를 날짜형으로 바꾸는 이유도 여기에 있습니다. 라인차트는 x축 순서가 중요합니다. 날짜가 그냥 글자로 들어 있으면 컴퓨터가 시간 순서로 다루지 못할 수 있습니다. 날짜형으로 바꾸면 “이 값은 2000년 1월, 그다음은 2000년 2월”처럼 시간의 흐름을 제대로 이해하게 됩니다.

오늘은 경제 해석 수업이 아니라 시계열 차트 만들기 수업입니다. 그래서 물가가 왜 올랐는지 깊게 해석하기보다, 월별 데이터가 있을 때 어떤 방식으로 선 그래프를 만들고 저장하는지에 집중합니다.

시연 포인트:

- `observation_date`를 날짜형으로 바꾸는 이유를 설명한다.
- 라인차트는 x축 순서가 중요하므로 날짜형 변환이 중요하다고 말한다.
- 오늘은 경제 해석 수업이 아니라 시계열 차트 만들기 수업이라고 범위를 정한다.

`utf-8-sig`는 Excel이나 일부 BI 도구가 한글 열 이름을 안정적으로 읽게 해줍니다. 데이터 내용이 바뀌는 것은 아닙니다. 한글 이름표가 깨지지 않도록 포장 방식을 조금 더 친절하게 맞추는 것이라고 보면 됩니다.

## 62-70분. 미션 안내

미션:

수업 홈의 데이터 받기 페이지에서 카페 매출 정리 파일을 내려받아 사용한다. 월별 매출 추세 라인차트와 CSV를 저장한다. CSV가 낯선 학생에게는 엑셀 파일(.xlsx)을 받아도 된다고 안내한다.

힌트 코드:

```python
cafe = pd.read_csv(DATA / "cafe_sales/cafe_sales_clean.csv")
cafe["date"] = pd.to_datetime(cafe["date"])

monthly_sales = (
    cafe.groupby(pd.Grouper(key="date", freq="MS"))
    .agg(매출합계=("amount", "sum"), 주문수=("order_id", "count"))
    .reset_index()
    .rename(columns={"date": "월"})
)
```

말할 것:

이번 미션은 새로운 문법을 많이 쓰는 문제가 아닙니다. 오늘 배운 순서를 스스로 반복하는 문제입니다. 날짜를 날짜형으로 바꾸고, 월별로 집계하고, 라인차트를 저장하고, CSV도 저장하면 됩니다.

여기서 `pd.Grouper(key="date", freq="MS")`는 날짜를 월 단위로 묶겠다는 뜻입니다. `MS`는 월의 시작일 기준으로 묶는다는 의미입니다. 주문이 10,424건 있어도, 월별로 묶으면 2025년 1월부터 2025년 6월까지 6행으로 줄어듭니다. 영수증을 한 장씩 보는 대신 월별 매출 장부로 합치는 과정이라고 생각하면 됩니다.

정답 확인값:

```text
월별 집계 결과: 6행
2025년 2월 매출: 13,157,000원
2025년 6월 매출: 16,816,500원
```

완료 판단 체크포인트 1문장:

수강생이 `s14_monthly_sales.csv`와 `s14_monthly_sales_trend.png`를 저장하고, 2025년 6월 매출 `16,816,500`원이 가장 높으며 2월 매출 `13,157,000`원이 가장 낮다고 설명하면 완료로 본다.

## 예상 질문과 답변

### Q1. 한글이 계속 깨지면 어떻게 하나요?

A. 먼저 나눔폰트가 설치되어 있는지 확인합니다. 설치 후에도 깨지면 matplotlib 폰트 캐시가 이전 상태를 기억하고 있을 수 있습니다. Colab에서는 런타임을 다시 시작하고, 로컬에서는 캐시를 지운 뒤 다시 실행합니다. 수업에서는 `selected_font` 출력값이 `NanumGothic` 또는 `NanumGothicCoding`인지 확인합니다.

### Q2. 지하철 데이터는 왜 전체 역을 다 그리지 않나요?

A. 전체 역을 모두 그리면 글자가 겹쳐서 읽을 수 없습니다. 좋은 시각화는 많은 데이터를 한 화면에 억지로 넣는 것이 아니라, 질문에 맞게 줄이는 것입니다. 오늘은 총 승차 상위 10개 역의 시간대 패턴을 보는 것이 목표입니다.

### Q3. 산점도에 점이 너무 많이 겹치면 어떻게 하나요?

A. `alpha`로 투명도를 낮출 수 있습니다. 그래도 너무 복잡하면 필요한 만큼 표본을 뽑거나, 구간별 집계로 바꿀 수도 있습니다. 오늘 아파트 데이터는 3,754행이라 투명도 조정만으로도 관계를 읽을 수 있습니다.

### Q4. 상관계수 0.806이면 전용면적이 가격의 원인인가요?

A. 아닙니다. 상관은 두 숫자가 함께 움직이는 정도를 말할 뿐, 원인을 증명하지 않습니다. 같은 면적이라도 단지, 위치, 층, 연식, 거래 시점 등이 가격에 영향을 줍니다. 그래서 “전용면적과 거래금액은 강하게 함께 움직인다”까지 말할 수 있고, “전용면적이 가격을 전부 결정한다”라고 말하면 안 됩니다.

### Q5. 왜 CSV 저장에 `utf-8-sig`를 쓰나요?

A. 원본 데이터는 UTF-8로 정리되어 있습니다. 다만 수강생이 Excel이나 BI 도구에서 파일을 열 때 한글 열 이름이 깨지는 경우가 있습니다. 전달용 CSV는 `utf-8-sig`로 저장하면 한글 열 이름을 더 안정적으로 읽습니다. 데이터 내용이 바뀌는 것은 아닙니다.

### Q6. Power BI에서도 같은 차트를 다시 만들 수 있나요?

A. 가능합니다. 오늘 저장한 CSV는 BI에서 바로 불러올 수 있는 차트용 표입니다. S15 이후에는 Power BI나 Looker Studio에서 이 집계표를 불러와 대시보드 차트로 배치할 수 있습니다.
