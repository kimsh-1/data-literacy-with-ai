from pathlib import Path

말할 것:

이 상황은 실무에서 매우 흔하다. 한 파일에 모든 정보가 들어 있지 않고, 여러 표에 나뉘어 있다. 그래서 분석 전에 표를 붙이는 일이 필요하다.

시연 포인트:

- `orders.columns`, `customers.columns`, `products.columns`를 보여준다.
- 공통 키인 `customer_id`, `product_id`를 수강생이 직접 찾게 한다.
- `orders`에 매출 컬럼이 없다는 점을 분명히 짚는다.

## 15-27분. `merge`로 정보 붙이고 매출 계산

시연 코드:

```python
merged = (
    orders.merge(customers, on="customer_id", how="left")
    .merge(products, on="product_id", how="left")
)

print(merged.shape)
print(merged[["region", "category", "price"]].isna().sum())
```

말할 것:

`merge`는 pandas의 VLOOKUP이라고 생각하면 된다. 주문표를 기준으로 고객표에서 지역을 가져오고, 상품표에서 카테고리와 단가를 가져온다. `how="left"`는 왼쪽 표인 주문표의 행을 유지하겠다는 뜻이다.

병합을 했으면 반드시 확인할 것이 있다. 첫째, 행수가 예상대로 유지되었는가. 둘째, 새로 붙인 값에 빈 값이 생겼는가. 이번 데이터는 20,000행이 그대로 유지되고, 지역·카테고리·단가 누락값이 0개다.

시연 코드:

```python
merged["sales"] = merged["price"] * merged["qty"] * (1 - merged["discount_rate"])
print(merged["sales"].sum())
```

말할 것:

매출은 원래 파일에 없었다. 단가는 상품표에서 왔고, 수량과 할인율은 주문표에 있었다. 여러 표를 붙인 뒤에야 매출 계산이 가능해졌다.

## 27-40분. `groupby`와 `agg`로 다중 집계

시연 코드:

```python
region_category_summary = (
    merged.groupby(["region", "category"])
    .agg(
        주문건수=("order_id", "nunique"),
        판매수량=("qty", "sum"),
        매출합계=("sales", "sum"),
        평균주문금액=("sales", "mean"),
    )
    .reset_index()
    .sort_values("매출합계", ascending=False)
)

print(region_category_summary.head(10))
```

말할 것:

`groupby(["region", "category"])`는 지역으로 먼저 나누고, 그 안에서 카테고리로 다시 나눈다는 뜻이다. 엑셀 피벗에서 행 영역에 지역과 카테고리를 넣은 상태를 떠올리면 된다.

`agg`는 값 영역에 여러 계산을 넣는 부분이다. 주문건수는 주문번호의 고유 개수, 판매수량은 수량 합계, 매출합계는 매출 합계, 평균주문금액은 매출 평균이다.

시연 포인트:

- `reset_index()`를 빼고 한 번 실행해 인덱스가 계층처럼 보이는 모습을 보여준다.
- `reset_index()`를 다시 붙여 일반 표처럼 다루기 쉽게 만든다고 설명한다.
- 상위 결과가 `서울특별시 × 디지털`임을 확인한다.

예상 결과:

```text
서울특별시 디지털 매출합계 210,356,387
경기도 디지털 매출합계 188,110,354
```

## 40-48분. `pivot_table`로 교차표 만들기

시연 코드:

```python
pivot = pd.pivot_table(
    merged,
    index="region",
    columns="category",
    values="sales",
    aggfunc="sum",
    fill_value=0,
    margins=True,
    margins_name="합계",
)

print(pivot.round(0).astype("int64").head())
```

말할 것:

`groupby` 결과는 긴 표다. 그런데 지역을 행에 두고 카테고리를 열에 두면 비교가 쉬워진다. 이것이 피벗 테이블이다.

`index`는 행, `columns`는 열, `values`는 계산할 숫자, `aggfunc`는 계산 방식이다. `fill_value=0`은 빈 조합을 0으로 채운다. `margins=True`는 합계 행과 합계 열을 만든다.

강조:

피벗은 분석 결과를 보기 좋게 펼치는 방법이다. 원자료를 피벗 형태로만 저장하는 것이 항상 좋은 것은 아니다. 다음 `melt`에서 그 반대 방향을 본다.

## 48-58분. `melt`로 wide를 long으로 바꾸기

시연 코드:

```python
pop = pd.read_csv(
    DATA / "real_korean/population_kosis/mois_legal_dong_gender_age_population_20260531.csv"
)

id_cols = ["법정동코드", "기준연월", "시도명", "시군구명", "읍면동명", "리명"]
age_gender_cols = [
    col
    for col in pop.columns
    if (col.endswith("남자") or col.endswith("여자")) and col not in ["남자", "여자"]
]

seoul_pop = pop[pop["시도명"] == "서울특별시"].copy()

long_pop = seoul_pop.melt(
    id_vars=id_cols,
    value_vars=age_gender_cols,
    var_name="연령성별",
    value_name="인구수",
)
```

말할 것:

이 인구 파일은 18,623행, 231열이다. 열이 매우 많다. `0세남자`, `1세남자`, `2세남자`처럼 분석 기준이 열 이름 속에 들어 있다.

`melt`는 열 이름에 들어 있던 정보를 값으로 내리는 작업이다. 엑셀 파워쿼리의 "열 피벗 해제"와 비슷하다.

이번 실습에서는 전체를 한 번에 길게 만들지 않고 서울특별시 452행만 골랐다. 그 결과 100,344행이 된다. 큰 파일에서는 먼저 필요한 지역이나 기간을 필터링하고, 그 다음 구조를 바꾸는 습관이 중요하다.

시연 코드:

```python
long_pop["성별"] = long_pop["연령성별"].str.extract("(남자|여자)$")
long_pop["연령"] = (
    long_pop["연령성별"].str.replace("(남자|여자)$", "", regex=True).str.strip()
)

print(long_pop[["시군구명", "읍면동명", "연령", "성별", "인구수"]].head())
```

시연 포인트:

- `id_vars`는 그대로 남길 열이라고 설명한다.
- `value_vars`는 세로로 내릴 열이라고 설명한다.
- `연령성별`에서 `성별`과 `연령`을 다시 분리하면 이후 분석이 쉬워진다고 설명한다.

## 예상 질문과 답변

### Q1. `merge`와 `join`은 같은 건가요?

비슷한 일을 하지만 오늘은 `merge`만 기억하면 된다. `merge`는 열의 값을 기준으로 두 표를 붙이는 방법이고, 엑셀의 VLOOKUP과 가장 연결하기 쉽다.

### Q2. 왜 `how="left"`를 쓰나요?

주문표를 기준으로 분석하기 때문이다. 주문 20,000행을 유지한 채 고객 정보와 상품 정보를 붙이고 싶다. 만약 기준을 잘못 잡으면 행이 줄거나 늘어날 수 있다.

### Q3. `groupby` 뒤에 왜 `reset_index()`를 붙이나요?

`groupby` 결과의 묶음 기준은 인덱스로 들어간다. 화면으로 보기에는 괜찮지만, 이후 정렬하거나 다시 병합할 때 일반 열로 있는 편이 쉽다. 그래서 수업에서는 `reset_index()`를 붙여 표처럼 다룬다.

### Q4. `pivot_table`과 `groupby`는 무엇이 다른가요?

둘 다 집계한다. `groupby`는 긴 요약표를 만들고, `pivot_table`은 행과 열이 나뉜 교차표를 만든다. 엑셀 피벗처럼 비교표를 만들고 싶으면 `pivot_table`이 편하다.

### Q5. 왜 넓은 표를 굳이 긴 표로 바꾸나요?

분석 도구는 보통 "열 하나는 의미 하나"인 표를 좋아한다. `0세남자`, `1세남자`처럼 열 이름에 값이 들어 있으면 필터링과 그래프 작성이 불편하다. `연령`, `성별`, `인구수` 열로 나누면 분석하기 쉬워진다.

### Q6. 큰 파일을 전부 `melt`하면 안 되나요?

가능할 때도 있지만 입문 수업에서는 먼저 줄이는 습관이 중요하다. 이번 인구 파일 전체를 222개 연령·성별 열로 `melt`하면 400만 행이 넘는다. 오늘은 서울특별시만 필터링해 100,344행으로 연습한다.

### Q7. 평균 생활인구와 최대 생활인구는 어떻게 다르게 읽나요?

평균 생활인구는 기간 전체에서 보통 어느 정도 사람이 있었는지를 보여준다. 최대 생활인구는 특정일·특정 시간대의 피크를 포함한 값이므로, 일상적인 규모보다 행사나 업무지구 특성의 영향을 더 크게 받을 수 있다.
