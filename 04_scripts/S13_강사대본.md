## 15-27분. `merge`로 정보 붙이고 매출 계산하기

시연 코드:

```python
merged = (
    orders.merge(customers, on="customer_id", how="left")
    .merge(products, on="product_id", how="left")
)

print(merged.shape)
print(merged[["region", "category", "price"]].isna().sum())
```

말할 것:

`merge`는 pandas에서 표와 표를 붙이는 방법입니다. 엑셀을 써 보셨다면 VLOOKUP을 떠올리면 됩니다. 주문표를 기준으로 고객표에서 지역을 가져오고, 상품표에서 카테고리와 단가를 가져옵니다.

여기서 `how="left"`는 왼쪽 표인 주문표의 행을 유지하겠다는 뜻입니다. 지금 우리는 주문 20,000건을 기준으로 분석하려고 합니다. 그래서 주문표의 행이 사라지면 안 됩니다.

표를 붙인 뒤에는 꼭 확인해야 합니다. 행수가 예상대로 유지되었는지 봅니다. 그리고 새로 붙인 지역, 카테고리, 단가에 빈 값이 생겼는지 봅니다. 이번 데이터는 20,000행이 그대로 유지되고, 지역·카테고리·단가 누락값이 0개입니다. 조회가 잘 된 상태라고 볼 수 있습니다.

시연 코드:

```python
merged["sales"] = merged["price"] * merged["qty"] * (1 - merged["discount_rate"])
print(merged["sales"].sum())
```

말할 것:

매출은 원래 주문 파일에 들어 있지 않았습니다. 단가는 상품표에서 가져왔고, 수량과 할인율은 주문표에 있었습니다. 그래서 표를 붙인 뒤에야 매출을 계산할 수 있습니다.

계산식은 단가에 수량을 곱하고, 할인율만큼 빼는 방식입니다. 물건 가격표와 구매 수량, 할인 쿠폰을 같이 놓고 실제 결제 금액을 계산하는 것과 같습니다.

## 27-40분. `groupby`와 `agg`로 여러 기준 집계하기

시연 코드:

```python
region_category_summary = (
    merged.groupby(["region", "category"])
    .agg(
        주문건수=("order_id", "nunique"),
        판매수량=("qty", "sum"),
        매출합계=("sales", "sum"),
        평균주문금액=("sales", "mean"),
    )
    .reset_index()
    .sort_values("매출합계", ascending=False)
)

print(region_category_summary.head(10))
```

말할 것:

`groupby(["region", "category"])`는 지역으로 먼저 나누고, 그 안에서 카테고리로 다시 나누라는 뜻입니다. 엑셀 피벗 테이블에서 행 영역에 지역을 넣고, 그 아래에 카테고리를 넣는 장면을 떠올리면 됩니다.

`agg`는 각 묶음마다 무엇을 계산할지 정하는 부분입니다. 주문건수는 주문번호의 고유 개수입니다. 같은 주문번호가 중복되어도 하나로 세겠다는 뜻입니다. 판매수량은 수량 합계이고, 매출합계는 매출의 합계입니다. 평균주문금액은 주문 한 건당 매출이 보통 어느 정도였는지 보는 값입니다.

여기서 중요한 점은 “무엇으로 묶고, 무엇을 계산하는가”입니다. 키를 재듯이 사람을 키 구간으로 나누고 평균 키를 구할 수 있죠. 여기서는 지역과 카테고리로 주문을 나누고, 그 안에서 건수와 수량과 매출을 계산합니다.

시연 포인트:

- `reset_index()`를 빼고 한 번 실행해 인덱스가 계층처럼 보이는 모습을 보여준다.
- `reset_index()`를 다시 붙여 일반 표처럼 다루기 쉽게 만든다고 설명한다.
- 상위 결과가 `서울특별시 × 디지털`임을 확인한다.

예상 결과:

```text
서울특별시 디지털 매출합계 210,356,387
경기도 디지털 매출합계 188,110,354
```

## 40-48분. `pivot_table`로 지역×카테고리 교차표 만들기

시연 코드:

```python
pivot = pd.pivot_table(
    merged,
    index="region",
    columns="category",
    values="sales",
    aggfunc="sum",
    fill_value=0,
    margins=True,
    margins_name="합계",
)

print(pivot.round(0).astype("int64").head())
```

말할 것:

`groupby` 결과는 세로로 긴 표입니다. 그런데 지역을 행에 두고, 카테고리를 열에 두면 지역별 차이를 한눈에 비교하기 쉽습니다. 이렇게 행과 열이 만나는 형태로 만든 요약표가 피벗 테이블입니다.

`index`는 행에 둘 기준입니다. `columns`는 열에 둘 기준입니다. `values`는 계산할 숫자이고, `aggfunc`는 계산 방식입니다. 여기서는 매출을 합계로 계산합니다. `fill_value=0`은 해당 지역과 카테고리 조합이 없을 때 빈칸 대신 0을 넣겠다는 뜻입니다. `margins=True`는 합계 행과 합계 열을 만들어 줍니다.

강조:

피벗은 분석 결과를 보기 좋게 펼치는 방법입니다. 원자료를 항상 피벗 모양으로 저장하는 것이 좋은 것은 아닙니다. 사람이 보기에는 넓은 표가 편할 때가 있지만, 분석 도구는 긴 표를 더 잘 다루는 경우가 많습니다. 바로 다음에 그 반대 방향인 `melt`를 봅니다.

## 48-58분. `melt`로 넓은 표를 긴 표로 바꾸기

시연 코드:

```python
pop = pd.read_csv(
    DATA / "real_korean/population_kosis/mois_legal_dong_gender_age_population_20260531.csv"
)

id_cols = ["법정동코드", "기준연월", "시도명", "시군구명", "읍면동명", "리명"]
age_gender_cols = [
    col
    for col in pop.columns
    if (col.endswith("남자") or col.endswith("여자")) and col not in ["남자", "여자"]
]

seoul_pop = pop[pop["시도명"] == "서울특별시"].copy()

long_pop = seoul_pop.melt(
    id_vars=id_cols,
    value_vars=age_gender_cols,
    var_name="연령성별",
    value_name="인구수",
)
```

말할 것:

수업 홈의 데이터 받기 페이지에서 내려받은 법정동 연령·성별 인구 파일을 읽습니다. 이 인구 파일은 18,623행, 231열입니다. 행도 많지만 특히 열이 매우 많습니다. `0세남자`, `1세남자`, `2세남자`처럼 분석 기준이 열 이름 속에 들어 있습니다.

`melt`는 열 이름에 들어 있던 정보를 값으로 아래로 내리는 작업입니다. 엑셀 파워쿼리의 “열 피벗 해제”와 비슷합니다. 넓게 펼쳐진 표를 길게 쌓는다고 생각하면 됩니다.

이번 실습에서는 전체를 한 번에 길게 만들지 않고 서울특별시 452행만 먼저 골랐습니다. 그 결과 100,344행이 됩니다. 큰 파일에서는 먼저 필요한 지역이나 기간만 고르고, 그 다음 표의 모양을 바꾸는 습관이 중요합니다. 한꺼번에 다 펼치면 화면도 복잡해지고, 초보자는 무엇을 보고 있는지 놓치기 쉽습니다.

시연 코드:

```python
long_pop["성별"] = long_pop["연령성별"].str.extract("(남자|여자)$")
long_pop["연령"] = (
    long_pop["연령성별"].str.replace("(남자|여자)$", "", regex=True).str.strip()
)

print(long_pop[["시군구명", "읍면동명", "연령", "성별", "인구수"]].head())
```

시연 포인트:

- `id_vars`는 그대로 남길 열이라고 설명한다.
- `value_vars`는 세로로 내릴 열이라고 설명한다.
- `연령성별`에서 `성별`과 `연령`을 다시 분리하면 이후 분석이 쉬워진다고 설명한다.

## 예상 질문과 답변

### Q1. `merge`와 `join`은 같은 건가요?

비슷한 일을 하지만 오늘은 `merge`만 기억하면 됩니다. `merge`는 열의 값을 기준으로 두 표를 붙이는 방법입니다. 엑셀의 VLOOKUP과 연결해서 이해하기 가장 쉽습니다. 주문표의 고객ID로 고객표에서 지역을 찾아오듯이, 한 표의 값을 기준으로 다른 표의 정보를 가져옵니다.

### Q2. 왜 `how="left"`를 쓰나요?

주문표를 기준으로 분석하기 때문입니다. 주문 20,000행을 유지한 채 고객 정보와 상품 정보를 붙이고 싶습니다. 기준을 잘못 잡으면 행이 줄어들 수도 있고, 반대로 늘어날 수도 있습니다. 그래서 병합 뒤에는 항상 행수가 그대로인지 확인해야 합니다.

### Q3. `groupby` 뒤에 왜 `reset_index()`를 붙이나요?

`groupby` 결과의 묶음 기준은 인덱스로 들어갑니다. 화면으로 보기에는 괜찮지만, 이후에 정렬하거나 다시 병합할 때는 일반 열로 있는 편이 다루기 쉽습니다. 그래서 수업에서는 `reset_index()`를 붙여서 다시 평범한 표처럼 만듭니다.

### Q4. `pivot_table`과 `groupby`는 무엇이 다른가요?

둘 다 집계를 합니다. `groupby`는 보통 세로로 긴 요약표를 만들고, `pivot_table`은 행과 열이 나뉜 교차표를 만듭니다. 엑셀 피벗처럼 지역을 행에 두고 카테고리를 열에 두어 비교하고 싶을 때는 `pivot_table`이 편합니다.

### Q5. 왜 넓은 표를 굳이 긴 표로 바꾸나요?

분석 도구는 보통 “열 하나는 의미 하나”인 표를 좋아합니다. `0세남자`, `1세남자`처럼 열 이름 안에 값이 들어 있으면 필터링과 그래프 작성이 불편합니다. `연령`, `성별`, `인구수` 열로 나누면 “20세만 보기”, “여자만 보기”, “연령별 합계 보기” 같은 작업이 훨씬 쉬워집니다.

### Q6. 큰 파일을 전부 `melt`하면 안 되나요?

할 수 있는 경우도 있습니다. 다만 입문 수업에서는 먼저 줄이는 습관이 중요합니다. 이번 인구 파일 전체를 222개 연령·성별 열로 `melt`하면 400만 행이 넘습니다. 오늘은 서울특별시만 필터링해서 100,344행으로 연습합니다. 필요한 부분을 먼저 고른 뒤 모양을 바꾸는 순서가 안전합니다.

### Q7. 평균 생활인구와 최대 생활인구는 어떻게 다르게 읽나요?

평균 생활인구는 기간 전체에서 보통 어느 정도 사람이 있었는지를 보여줍니다. 최대 생활인구는 특정일·특정 시간대의 피크를 포함한 값입니다. 그래서 일상적인 규모를 보려면 평균을 보고, 사람이 가장 많이 몰렸던 순간의 규모를 보려면 최대를 봅니다. 업무지구나 행사 같은 특성은 최대값에 더 크게 드러날 수 있습니다.
