## 25-40분. 리스트와 딕셔너리

시연 코드:

```python
sales_list = [120000, 98000, 135000, 170000, 155000]

print(sales_list)
print(len(sales_list))
```

말할 것:

리스트는 여러 값을 순서대로 담는다. 엑셀 한 열에 매출이 아래로 쌓인 모습과 비슷하다. `len()`은 값이 몇 개인지 세는 함수다.

시연 코드:

```python
one_day = {
    "요일": "월",
    "매출": 120000,
}

print(one_day["요일"])
print(one_day["매출"])
```

말할 것:

딕셔너리는 이름표가 붙은 값이다. 엑셀의 한 행과 비슷해서, 이 행에 `요일` 열과 `매출` 열이 있다고 보면 된다.

강조:

리스트는 대괄호 `[]`, 딕셔너리는 중괄호 `{}`를 쓴다. 처음에는 괄호 모양이 헷갈리는 게 당연하다. 오늘은 모양을 외우기보다 어떤 상황에서 쓰는지만 잡으면 된다.

## 40-55분. 반복문으로 합계 계산

시연 코드:

```python
total_sales = 0

for amount in sales_list:
    total_sales = total_sales + amount

print(total_sales)
```

말할 것:

엑셀에서는 첫 행에 수식을 쓰고 아래로 채운다. 파이썬에서는 반복문이 그 역할을 맡아 같은 작업을 여러 값에 적용한다.

`for amount in sales_list`는 리스트에서 값을 하나씩 꺼내 이번 값을 `amount`라고 부르겠다는 뜻이다. 첫 번째 반복에서 `amount`는 120000, 다음 반복에서는 98000이다.

`total_sales = total_sales + amount`는 이상하게 보일 수 있다. 오른쪽의 기존 합계에 이번 매출을 더한 뒤 그 값을 다시 합계 변수에 담으라는 뜻이다.

시연 포인트:

- 일부러 콜론을 빼고 `SyntaxError`를 보여준다.
- 일부러 들여쓰기를 지우고 `IndentationError`를 보여준다.
- 오류 메시지는 마지막 줄부터 읽는다고 안내한다.

## 55-65분. 평균 계산과 함수 맛보기

시연 코드:

```python
average_sales = total_sales / len(sales_list)

print(average_sales)
```

말할 것:

평균은 합계를 개수로 나눈 값이다. 엑셀 `AVERAGE` 함수가 해주던 일을 지금은 손으로 써본 셈이다. 나중에는 파이썬에서도 평균 구하는 함수나 pandas의 `.mean()`을 쓰게 된다.

선택 시연 코드:

```python
def average(numbers):
    total = 0
    for number in numbers:
        total = total + number
    return total / len(numbers)

print(average(sales_list))
```

말할 것:

함수는 자주 쓰는 절차에 이름을 붙이는 방법이다. 오늘은 함수를 많이 만들지는 않는다. 다만 다음 시간부터 `read_csv()`, `head()`, `mean()` 같은 이름을 계속 보게 되니, 함수를 "정해진 일을 해주는 도구" 정도로만 이해하면 된다.

## 65-78분. 조건문과 요일 필터

시연 코드:

```python
day = "금"

if day == "금":
    print("금요일 데이터입니다.")
```

말할 것:

조건문은 엑셀 필터에 빗대면 이해하기 쉽다. 엑셀에서 요일이 금요일인 행만 보이게 하듯, 파이썬에서는 `if day == "금"`이라고 쓴다.

여기서 `=`와 `==`는 반드시 구분한다. `=`는 값을 담고, `==`는 같은지 묻는다.

시연 코드:

```python
daily_sales = [
    {"요일": "월", "매출": 120000},
    {"요일": "화", "매출": 98000},
    {"요일": "수", "매출": 135000},
    {"요일": "목", "매출": 170000},
    {"요일": "금", "매출": 155000},
    {"요일": "토", "매출": 210000},
    {"요일": "일", "매출": 190000},
]

for row in daily_sales:
    if row["요일"] == "금":
        print(row)
```

말할 것:

이 코드는 작은 표에서 한 행씩 꺼내 보고 요일이 금요일이면 출력한다. 바로 이 구조가 다음 시간 pandas 필터의 바탕이다. 다음 시간에는 이 조건을 행마다 직접 반복하는 대신, pandas가 열 전체에 한 번에 적용하게 한다.

## 78-87분. 직접 해보기 안내

- 조건을 만족하면 `weekday_rows.append(row)`로 모은다.
- 동시에 `weekday_total`에 매출을 더한다.

## 87-90분. 다음 차시 브리지

말할 것:

오늘 마지막 코드는 사실상 필터다. 행을 하나씩 보고 조건에 맞으면 남겼다. 다음 차시 pandas에서는 같은 일을 훨씬 짧게 쓴다.

예를 들어 오늘은 이렇게 생각했다.

```python
if row["요일"] in weekday_names:
    ...
```

다음 시간에는 표 전체에 대해 이런 식으로 생각한다.

```python
df[df["weekday"].isin(["Mon", "Tue", "Wed", "Thu", "Fri"])]
```

문법 모양은 달라지지만 핵심은 같다. 조건을 세우고 거기에 맞는 행만 남긴다.

## 예상 질문과 답변

### Q1. 파이썬 문법을 다 외워야 하나요?

아니다. 오늘 외울 것은 많지 않다. 변수는 이름, 리스트는 여러 값, 딕셔너리는 이름표 붙은 값, 조건문은 필터, 반복문은 같은 작업 반복. 이 감각만 잡으면 된다. 정확한 문법은 실습하면서 찾아보면 된다.

### Q2. 왜 `sum()`을 쓰지 않고 반복문으로 직접 더하나요?

실무에서는 `sum()`을 쓰면 된다. 오늘 직접 더해 보는 건 반복문이 값을 하나씩 어떻게 처리하는지 눈으로 보기 위해서다. 이 감각이 있어야 나중에 pandas가 알아서 해주는 일이 무엇인지 알 수 있다.

### Q3. `=`와 `==`가 계속 헷갈립니다.

처음에는 다들 헷갈린다. `=`는 변수에 값을 넣을 때, `==`는 같은지 물어볼 때라고 기억하자. 오류가 나면 이 둘부터 확인한다.

### Q4. 들여쓰기는 꼭 해야 하나요?

파이썬에서는 반드시 해야 한다. 들여쓰기가 코드의 소속을 나타내기 때문이다. `if`나 `for` 아래에서 실행할 코드는 한 단계 안쪽으로 들여쓴다.

### Q5. Colab을 닫으면 변수가 사라지나요?

노트북 파일 자체는 저장되지만, 실행 중이던 변수는 런타임이 꺼지면 사라진다. 다시 열면 위에서부터 필요한 셀을 순서대로 실행해 주어야 한다.

### Q6. 오늘 배운 것으로 실제 데이터 분석을 할 수 있나요?

오늘 문법만으로도 작은 리스트는 처리할 수 있다. 다만 실제 CSV는 행과 열이 많으니 다음 차시부터 pandas를 쓴다. 오늘 배운 조건문과 반복문이 그 pandas 문법을 이해하는 바탕이 된다.
