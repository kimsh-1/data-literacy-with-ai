## 25-40분. 리스트와 딕셔너리로 값 묶기

시연 코드:

```python
sales_list = [120000, 98000, 135000, 170000, 155000]

print(sales_list)
print(len(sales_list))
```

말할 것:

리스트는 여러 값을 순서대로 담는 구조입니다. 엑셀에서 매출 값이 한 열에 아래로 쌓여 있는 모습을 떠올리면 쉽습니다. 여기서는 매출 5개가 한 줄에 들어 있지만, 느낌은 엑셀의 한 열과 비슷합니다.

`len()`은 값이 몇 개인지 세는 함수입니다. 키를 잴 때 자를 대듯이, 리스트에 `len()`을 대면 안에 값이 몇 개 들어 있는지 알려줍니다.

시연 코드:

```python
one_day = {
    "요일": "월",
    "매출": 120000,
}

print(one_day["요일"])
print(one_day["매출"])
```

말할 것:

딕셔너리는 이름표가 붙은 값입니다. 엑셀 표의 한 행을 떠올려 보세요. 한 행 안에 `요일`이라는 칸도 있고, `매출`이라는 칸도 있습니다. 딕셔너리는 이렇게 이름을 보고 값을 꺼낼 수 있게 해 줍니다.

`one_day["요일"]`은 `one_day`라는 한 행에서 `요일` 칸을 꺼내는 느낌입니다. 사전에서 단어를 찾듯이, 이름을 넣으면 그에 해당하는 값을 찾습니다.

강조:

리스트는 대괄호 `[]`, 딕셔너리는 중괄호 `{}`를 씁니다. 처음에는 괄호 모양이 헷갈리는 게 자연스럽습니다. 지금은 괄호를 외우려고 애쓰기보다, 리스트는 여러 값이 순서대로 들어간 것, 딕셔너리는 이름표가 붙은 값이라고 구분해 봅니다.

## 40-55분. 반복문으로 합계 계산하기

시연 코드:

```python
total_sales = 0

for amount in sales_list:
    total_sales = total_sales + amount

print(total_sales)
```

말할 것:

엑셀에서는 첫 행에 수식을 쓰고 아래로 채우기를 합니다. 파이썬에서는 반복문이 비슷한 일을 합니다. 같은 작업을 여러 값에 차례대로 적용해 주는 문법입니다.

`for amount in sales_list`는 `sales_list` 안에서 값을 하나씩 꺼내겠다는 뜻입니다. 그리고 이번에 꺼낸 값을 잠깐 `amount`라고 부르겠다는 뜻입니다. 첫 번째 반복에서 `amount`는 120000이고, 다음 반복에서는 98000이 됩니다.

`total_sales = total_sales + amount`는 처음 보면 이상합니다. 왼쪽과 오른쪽에 같은 이름이 있기 때문입니다. 이 문장은 오른쪽의 기존 합계에 이번 매출을 더한 뒤, 그 결과를 다시 `total_sales`에 담으라는 뜻입니다. 장바구니에 물건 값을 하나씩 더해 가듯이 합계가 점점 커집니다.

시연 포인트:

- 일부러 콜론을 빼고 `SyntaxError`를 보여준다.
- 일부러 들여쓰기를 지우고 `IndentationError`를 보여준다.
- 오류 메시지는 마지막 줄부터 읽는다고 안내한다.

## 55-65분. 평균 계산과 함수 맛보기

시연 코드:

```python
average_sales = total_sales / len(sales_list)

print(average_sales)
```

말할 것:

평균은 합계를 개수로 나눈 값입니다. 엑셀의 `AVERAGE` 함수가 해주던 일을 지금은 우리가 직접 써 본 셈입니다.

여기서는 `total_sales`가 합계이고, `len(sales_list)`가 개수입니다. 그래서 둘을 나누면 평균 매출이 나옵니다. 나중에는 파이썬에서도 평균을 구하는 함수나 pandas의 `.mean()`을 쓰게 됩니다. 지금은 평균이 어떻게 만들어지는지 한 번 손으로 확인해 보는 단계입니다.

선택 시연 코드:

```python
def average(numbers):
    total = 0
    for number in numbers:
        total = total + number
    return total / len(numbers)

print(average(sales_list))
```

말할 것:

함수는 자주 쓰는 절차에 이름을 붙이는 방법입니다. 같은 계산을 여러 번 해야 할 때, 매번 긴 코드를 다시 쓰지 않고 이름으로 불러 쓸 수 있게 해 줍니다.

오늘 함수를 많이 만들지는 않습니다. 다만 다음 시간부터 `read_csv()`, `head()`, `mean()` 같은 이름을 계속 보게 됩니다. 그때 함수는 "정해진 일을 해주는 도구"라고 생각하면 됩니다. 엑셀에서 `SUM()`을 쓰면 합계를 구해 주는 것과 비슷합니다.

## 65-78분. 조건문과 요일 필터 만들기

시연 코드:

```python
day = "금"

if day == "금":
    print("금요일 데이터입니다.")
```

말할 것:

조건문은 엑셀 필터에 빗대면 이해하기 쉽습니다. 엑셀에서 요일이 금요일인 행만 보이게 하려면 필터 조건을 걸죠. 파이썬에서는 `if day == "금"`이라고 씁니다.

여기서 `=`와 `==`는 꼭 구분해야 합니다. `=`는 값을 담는 문법입니다. `==`는 두 값이 같은지 묻는 문법입니다. `day == "금"`은 `day` 안의 값이 `"금"`과 같나요, 하고 물어보는 문장입니다.

시연 코드:

```python
daily_sales = [
    {"요일": "월", "매출": 120000},
    {"요일": "화", "매출": 98000},
    {"요일": "수", "매출": 135000},
    {"요일": "목", "매출": 170000},
    {"요일": "금", "매출": 155000},
    {"요일": "토", "매출": 210000},
    {"요일": "일", "매출": 190000},
]

for row in daily_sales:
    if row["요일"] == "금":
        print(row)
```

말할 것:

이 코드는 작은 표에서 한 행씩 꺼내 보는 코드입니다. `daily_sales`는 여러 행이 들어 있는 작은 표처럼 볼 수 있습니다. 반복문은 그중 한 행씩 꺼내서 `row`라고 부릅니다.

그리고 `row["요일"] == "금"`으로 이번 행의 요일이 금요일인지 확인합니다. 조건이 맞으면 그 행을 출력합니다. 바로 이 구조가 다음 시간에 배울 pandas 필터의 바탕입니다. 다음 시간에는 행마다 직접 반복문을 쓰는 대신, pandas가 열 전체에 조건을 한 번에 적용하게 합니다.

## 78-87분. 직접 해보기 안내

미션:

`daily_sales`에서 평일인 `월`, `화`, `수`, `목`, `금`만 골라 `weekday_rows`에 담고, 평일 매출 합계와 평균을 출력한다.

수강생에게 안내할 조건:

- `weekday_names = ["월", "화", "수", "목", "금"]`를 만든다.
- 반복문으로 `daily_sales`를 한 행씩 본다.
- `row["요일"] in weekday_names` 조건을 사용한다.
- 조건을 만족하면 `weekday_rows.append(row)`로 모은다.
- 동시에 `weekday_total`에 매출을 더한다.

말할 것:

이번에는 금요일 하나만 고르는 것이 아니라 평일 5개를 고릅니다. 먼저 평일 이름을 리스트로 만들어 둡니다. 그다음 한 행씩 보면서, 그 행의 요일이 평일 이름 목록 안에 있는지 확인합니다.

`in`은 목록 안에 들어 있는지 묻는 문법입니다. 출석부에서 이름을 찾듯이, `row["요일"] in weekday_names`는 이번 요일이 평일 목록 안에 있나요, 하고 확인합니다.

조건에 맞는 행은 `weekday_rows.append(row)`로 모읍니다. `append()`는 리스트 맨 뒤에 값을 하나 더 붙이는 도구입니다. 동시에 매출은 `weekday_total`에 계속 더해 줍니다.

완료 판단 체크포인트:

수강생이 반복문과 조건문으로 평일 5행만 골라 `평일 행수 5`, `평일 매출 합계 678000`, `평일 매출 평균 135600.0`을 출력하면 완료로 본다.

## 87-90분. 다음 차시로 연결하기

말할 것:

오늘 마지막 코드는 사실상 필터입니다. 행을 하나씩 보고, 조건에 맞는 행만 남겼습니다. 다음 차시 pandas에서는 같은 일을 훨씬 짧게 씁니다.

예를 들어 오늘은 이렇게 생각했습니다.

```python
if row["요일"] in weekday_names:

```

문법 모양은 달라지지만 하는 일은 같습니다. 조건을 세우고, 그 조건에 맞는 행만 남깁니다. 오늘은 손으로 한 행씩 확인했다면, 다음 시간에는 pandas가 표 전체를 한 번에 훑어 준다고 보면 됩니다.

## 예상 질문과 답변

### Q1. 파이썬 문법을 다 외워야 하나요?

아닙니다. 오늘 배운 문법을 전부 외우려고 하지 않아도 됩니다. 변수는 값에 붙이는 이름, 리스트는 여러 값, 딕셔너리는 이름표 붙은 값, 조건문은 필터, 반복문은 같은 작업 반복이라고 이해하면 됩니다. 정확한 문법은 실습하면서 다시 찾아봐도 괜찮습니다.

### Q2. 왜 `sum()`을 쓰지 않고 반복문으로 직접 더하나요?

실무에서는 `sum()`을 쓰면 됩니다. 오늘 직접 더해 보는 이유는 반복문이 값을 하나씩 어떻게 처리하는지 눈으로 보기 위해서입니다. 이 과정을 한 번 봐야 나중에 pandas가 자동으로 해주는 일이 무엇인지 더 잘 이해할 수 있습니다.

### Q3. `=`와 `==`가 계속 헷갈립니다.

처음에는 많이 헷갈립니다. `=`는 변수에 값을 넣을 때 씁니다. `==`는 두 값이 같은지 물어볼 때 씁니다. 코드가 예상대로 움직이지 않거나 오류가 나면, 이 둘을 먼저 확인해 보세요.

### Q4. 들여쓰기는 꼭 해야 하나요?

파이썬에서는 반드시 해야 합니다. 들여쓰기가 코드의 소속을 나타내기 때문입니다. `if`나 `for` 아래에서 실행할 코드는 한 단계 안쪽으로 들여씁니다. 보기 좋으라고 맞추는 것이 아니라, 파이썬이 코드를 이해하기 위해 필요한 문법입니다.

### Q5. Colab을 닫으면 변수가 사라지나요?

노트북 파일 자체는 저장됩니다. 하지만 실행 중이던 변수는 런타임이 꺼지면 사라질 수 있습니다. 다시 노트북을 열었을 때 아래쪽 셀에서 변수를 찾지 못한다면, 위에서부터 필요한 셀을 순서대로 다시 실행해 주세요.
