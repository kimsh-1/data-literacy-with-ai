# S15 강사대본: Power BI 설치, 로드, 첫 차트

## 수업 전 확인

- 강사용 PC가 Windows인지 확인한다.
- Power BI Desktop이 실행되는지 확인한다.
- 설치가 안 된 경우 Microsoft Store 또는 Microsoft 공식 다운로드 페이지에 접속 가능한지 확인한다.
- 수업 홈의 데이터 받기 페이지에서 엑셀로도 열리는 표 파일인 CSV 형식의 카페 매출 파일과, 같은 내용의 엑셀 파일을 내려받을 수 있는지 확인한다.
- 카페 매출 CSV 파일의 실제 확인값을 칠판이나 강사용 메모에 적어 둔다.
  - 행 수: 10,424
  - 열 수: 12
  - 기간: 2025-01-01부터 2025-06-30까지
  - 총매출: 90,492,500
- 실습은 CSV 파일을 기준으로 진행하지만, 초보자는 같은 데이터의 엑셀 파일(.xlsx)을 받아도 된다고 안내한다.
- Mac 수강생에게 Power BI Desktop은 Windows용이며, S18에서 Looker Studio로 직접 대체 실습한다고 먼저 안내한다.

## 0-8분. BI 도구를 왜 쓰는지 잡아 주기

말할 것:

BI는 Business Intelligence의 줄임말입니다. 오늘 수업에서는 어렵게 생각하지 말고, 데이터를 한 번 연결해 두고 중요한 숫자와 그래프를 한 화면에 모아 보는 보고서 도구라고 잡으면 됩니다.

엑셀과 AI, 파이썬을 거쳐 왔죠. 그런데 보고할 때마다 파일을 열고, 피벗테이블을 다시 만지고, 차트를 다시 정리하는 일은 꽤 번거롭습니다. BI 도구를 쓰면 표를 한 번 불러온 뒤에 총매출, 카테고리별 매출, 날짜별 매출 같은 화면을 만들어 둘 수 있습니다. 체중계를 매번 새로 만드는 것이 아니라, 올라서면 바로 숫자가 보이게 만들어 두는 느낌입니다.

Power BI Desktop은 Windows에서 보고서를 만드는 프로그램입니다. 오늘은 공유하거나 웹에 게시하는 단계까지 가지 않습니다. 내 PC에서 첫 보고서 파일을 만들고, 차트가 어떻게 만들어지는지만 확실히 봅니다. Mac 사용자는 설치를 억지로 따라오지 않아도 됩니다. 화면 구조와 용어만 익혀 두면, S18에서 Looker Studio로 같은 개념을 직접 실습합니다.

엑셀과 연결해서 설명:

엑셀 피벗테이블에서 행, 열, 값 영역을 채웠던 기억을 떠올려 보세요. Power BI에서는 그 자리가 X축, Y축, 값, 범례라는 이름으로 보입니다. 이름만 조금 다를 뿐입니다. 비교 기준을 놓고, 볼 숫자를 놓고, 어떤 모양으로 보여줄지 고르는 원리는 같습니다.

## 8-18분. 설치 확인과 화면 구성 보기

시연:

1. Windows 시작 메뉴에서 `Power BI Desktop`을 검색한다.
2. 실행한다.
3. 로그인 화면이 나오면 오늘은 로컬 보고서 작성 실습이라는 점을 말하고, 강사 PC 환경에 맞게 닫거나 건너뛴다.
4. 빈 보고서 화면에서 캔버스, 데이터 패널, 시각화 패널, 리본 메뉴를 차례로 가리킨다.

말할 것:

가운데 흰 화면이 보고서 캔버스입니다. 우리가 만드는 차트와 카드가 여기에 올라갑니다. 오른쪽 데이터 패널은 엑셀 피벗테이블의 필드 목록과 비슷합니다. 어떤 열을 쓸 수 있는지 보여 주는 자리입니다. 그 옆 시각화 패널은 차트 종류를 고르는 자리입니다.

위쪽 리본 메뉴에는 데이터를 가져오거나 저장하는 버튼이 있습니다. 엑셀에서 메뉴를 눌러 기능을 찾는 것과 비슷하게 보시면 됩니다.

주의:

수강생 화면이 영어로 떠 있을 수 있다. 메뉴를 짚을 때는 한국어와 영어를 나란히 불러 준다. 이를테면 `데이터 가져오기`와 괄호 속 `Get data`를 같이 말하는 식이다.

## 18-28분. CSV 불러오기와 엑셀 불러오기 흐름 비교

시연 코드가 아니라 클릭 순서:

1. `홈(Home)`을 누른다.
2. `데이터 가져오기(Get data)`를 누른다.
3. `텍스트/CSV(Text/CSV)`를 선택한다.
4. 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 CSV 파일을 선택한다.
5. 미리보기에서 열 이름을 확인한다.
6. `로드(Load)`를 누른다.

말할 것:

오늘 데이터는 이미 정리된 상태라서 바로 로드합니다. 로드는 표를 Power BI 안으로 가져오는 일입니다. 엑셀에서 파일을 열면 셀이 바로 보이지만, Power BI에서는 먼저 데이터를 가져온 뒤 그 데이터를 재료로 차트를 만듭니다.

현업에서는 이 미리보기 단계가 중요합니다. 열 이름이 제대로 보이는지, 날짜가 날짜로 읽히는지, 금액이 숫자로 읽히는지 확인해야 합니다. 오늘 파일은 10,424행, 12열이고 결측값, 즉 비어 있는 값이 없는 것으로 확인했습니다.

데이터 패널에서 읽을 열:

`order_id`, `date`, `weekday`, `hour`, `store`, `category`, `item`, `unit_price`, `qty`, `amount`, `payment`, `member`

말할 것:

오늘 가장 중요한 열은 `amount`입니다. 총매출 카드도, 두 그래프의 값도 모두 이 열에서 나옵니다. `amount`는 주문금액이라고 보면 됩니다.

`category`는 막대그래프에서 커피, 음료, 디저트를 비교하는 기준이 됩니다. `date`는 꺾은선그래프에서 시간 흐름을 보는 기준이 됩니다. 사전에서 단어를 찾아 뜻을 보듯이, Power BI에서는 열 이름을 보고 어떤 역할을 맡길지 정합니다.

엑셀 파일로 불러오는 흐름도 보여 주기:

같은 데이터의 엑셀 파일도 수업 홈의 데이터 받기 페이지에서 받을 수 있다. 새 보고서에서 엑셀 파일을 불러온다면 `데이터 가져오기`, `Excel 통합 문서`, 카페 매출 엑셀 파일 선택, `sales` 시트 선택 순서로 고른다. 하지만 지금 보고서에는 CSV만 쓴다. 같은 데이터를 두 번 불러오면 숫자가 겹쳐서 헷갈리기 쉽다.

## 28-38분. 카테고리별 매출 막대그래프 만들기

시연:

1. 시각화 패널에서 `묶은 세로 막대형 차트(Clustered column chart)`를 선택한다.
2. `category`를 X축에 넣는다.
3. `amount`를 Y축 또는 값에 넣는다.
4. `amount`가 합계로 들어갔는지 확인한다.
5. 제목을 `카테고리별 매출`로 바꾼다.

말할 것:

막대그래프는 항목을 비교할 때 씁니다. 여기서는 커피, 음료, 디저트의 매출을 견줍니다. 키를 잴 때 사람을 나란히 세워 놓으면 누가 더 큰지 바로 보이죠. 막대그래프도 비슷합니다. 항목별 숫자를 나란히 세워 놓고 비교합니다.

엑셀 피벗테이블로 치면 `category`를 행에 넣고, `amount`를 값에 넣은 뒤 차트로 보는 것과 같습니다. 여기서 중요한 점은 `amount`가 합계로 들어가야 한다는 것입니다. 평균이나 개수로 들어가면 우리가 보려는 매출 합계가 아닙니다.

검산용 숫자:

- 커피: 45,606,000
- 음료: 25,550,000
- 디저트: 19,336,500

시연 포인트:

Power BI가 값을 `45.61M`처럼 줄여 보여줘도 틀린 것이 아닙니다. 표시 단위가 자동으로 잡힌 것뿐입니다. `M`은 million, 즉 백만 단위로 줄여 보인다는 뜻입니다. 오늘 미션의 카드에서는 이 표시 단위를 바꿔 정확한 숫자를 띄웁니다.

## 38-48분. 일자별 매출 꺾은선그래프 만들기

시연:

1. 캔버스 빈 곳을 클릭한다.
2. `꺾은선형 차트(Line chart)`를 선택한다.
3. `date`를 X축에 넣는다.
4. `amount`를 Y축 또는 값에 넣는다.
5. 제목을 `일자별 매출`로 바꾼다.

말할 것:

꺾은선그래프는 시간 흐름을 볼 때 씁니다. 막대그래프가 항목을 나란히 비교하는 도구라면, 꺾은선그래프는 날짜 순서대로 숫자가 올라가고 내려가는 모습을 보여주는 도구입니다.

오늘 데이터는 2025년 1월 1일부터 6월 30일까지의 주문 기록입니다. 그래서 `date`를 축에 놓으면 날짜별 매출의 오르내림이 드러납니다. 어느 날 매출이 높았는지, 어느 구간에서 낮아졌는지 눈으로 따라갈 수 있습니다.

날짜 축 문제 대응:

Power BI가 `date`를 날짜 계층 구조로 넣으면 연도, 분기, 월, 일 단위가 딸려 나온다. 오늘은 일자별 흐름을 보려는 것이니, X축 필드의 드롭다운에서 `date` 자체를 고르도록 안내한다. 수강생 화면마다 조금씩 다르니 강사 화면을 천천히 짚어 준다.

말할 것:

날짜 계층 구조는 날짜를 연도, 분기, 월, 일로 접어서 보는 기능입니다. 편할 때도 있지만, 오늘처럼 하루하루의 흐름을 보려면 `date` 자체를 골라야 합니다. 화면이 다르게 보이는 수강생이 있으면 당황하지 말고, X축에 어떤 날짜 필드가 들어갔는지부터 같이 봅니다.

## 48-56분. 직접 해보기: 총매출 KPI 카드

미션 안내:

이제 총매출을 카드로 만들어 봅니다. 카드는 큰 숫자 하나를 보여주는 시각화입니다. 엑셀에서 `SUM`으로 합계를 구한 뒤, 그 결과를 큰 글씨로 적어 두는 것과 같습니다.

이 카드에 들어갈 숫자는 `amount`의 합계입니다. 오늘 맞춰야 할 값은 **90,492,500**입니다. 이 숫자가 맞으면 데이터를 제대로 불러왔고, 집계 방식도 맞게 잡은 것입니다.

수강생 수행:

1. 캔버스 빈 곳을 클릭한다.
2. `카드(Card)` 또는 `카드(새)(Card new)`를 선택한다.
3. `amount`를 값 영역에 넣는다.
4. 집계 방식이 합계인지 확인한다.
5. 제목을 `총매출` 또는 `총매출(원)`으로 바꾼다.
6. 표시 단위를 `없음(None)`으로 바꾼다.
7. 소수 자릿수를 0으로 맞춘다.

기대 결과:

카드 값이 `90,492,500`이면 맞다.

돌아다니며 볼 것:

- 수강생이 `amount` 대신 `order_id`를 넣지 않았는지 확인한다.
- 카드 값이 `10,424`로 보이면 주문 건수를 넣었을 가능성이 높다.
- 카드 값이 `90.49M`으로 보이면 표시 단위만 바꾸면 된다.
- 값이 평균으로 들어가면 약 `8,681` 수준으로 보일 수 있다. 집계를 합계로 바꾸게 한다.

완료 판단 체크포인트:

수강생이 총매출 카드에 `90,492,500`을 띄우고, 같은 페이지에 카테고리별 매출 막대그래프와 일자별 매출 꺾은선그래프까지 배치했다면 이 차시는 완료로 본다.

2. `저장(Save)`을 누른다.

## 예상 질문과 답변

### Q1. Power BI Desktop은 꼭 설치해야 하나요?

A. Windows에서 직접 실습하려면 필요합니다. Power BI Desktop은 Windows용 보고서 제작 프로그램입니다. Mac 수강생은 설치하지 않고 화면 구조와 필드 배치 개념을 따라오면 됩니다. 직접 실습은 S18의 Looker Studio에서 이어 갑니다.

### Q2. Power BI와 엑셀 차트는 무엇이 다른가요?

A. 엑셀 차트는 파일 안에서 빠르게 그려 보기 좋습니다. Power BI는 여러 시각화를 한 페이지에 모으고, 필터와 상호작용을 얹어 반복해서 보는 보고서에 강합니다. 엑셀은 계산하고 확인하는 작업장에 가깝고, Power BI는 다른 사람이 계속 들여다보는 보고 화면에 가깝다고 보면 됩니다.

### Q3. 로그인해야 하나요?

A. 오늘 실습은 수업 홈의 데이터 받기 페이지에서 받은 카페 매출 파일을 불러와 Desktop에서 보고서를 만드는 과정입니다. 공유나 게시를 하지 않으니 조직 계정 기능은 건드리지 않습니다. 다만 설치 방식이나 회사 PC 정책에 따라 로그인 화면이 뜰 수는 있습니다.

### Q4. CSV와 엑셀 중 무엇을 써야 하나요?

A. 오늘 결과를 맞추기 위해 CSV를 기준으로 삼습니다. 엑셀 파일도 같은 데이터라 `sales` 시트를 고르면 불러올 수 있습니다. 초보자는 엑셀 파일이 더 익숙할 수 있지만, 한 보고서에 CSV와 엑셀을 함께 넣으면 같은 데이터가 겹치니 피합니다.

### Q5. 카드 숫자가 정답보다 작게 보입니다.

A. 세 가지를 차례로 봅니다. 값에 `amount`가 들어갔는지, 집계가 합계인지, 표시 단위가 저절로 `M`으로 축약돼 있지는 않은지 확인합니다. 카드가 작게 보인다고 바로 틀린 것은 아닙니다. `90.49M`처럼 보이면 표시 방식만 줄여서 보여주는 상태일 수 있습니다.

### Q6. 날짜 그래프가 이상합니다.

A. 먼저 `date`가 날짜로 인식됐는지 봅니다. X축에 날짜 계층 구조가 들어가면 연도나 월 단위로 접혀 보이기도 합니다. 일자별로 펴 보려면 X축에서 `date` 자체를 고릅니다. 날짜 그래프는 시간 순서가 중요하니, 축에 어떤 필드가 들어갔는지부터 확인해 보세요.

## 평가 루브릭

| 수준 | 판단 기준 |
|---|---|
| 상 | CSV를 직접 로드하고, 카드 값이 `90,492,500`임을 확인하며, 막대와 꺾은선그래프의 의미를 설명한다 |
| 중 | 강사 절차를 따라 세 시각화를 만들지만, 표시 단위나 집계 방식 검산은 도움을 받아 수정한다 |
| 하 | 파일 로드 또는 필드 배치 단계에서 멈추며, 총매출 카드 값을 확인하지 못한다 |
