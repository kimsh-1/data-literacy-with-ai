# S04 강사대본: 실무 함수

## 수업 전 준비

- 파일 확인: `/mnt/d/ai-data-analysis-course/02_datasets/shop_orders/shop_orders_clean.xlsx`
- 시트 확인: `orders`, `customers`, `products`
- 실제 데이터 행수:
  - `orders`: 20,000행
  - `customers`: 3,000행
  - `products`: 200행
- 실습 결과 검산값:
  - 전체 주문 건수: 20,000
  - 전체 매출: 1,938,489,803
  - VIP 객단가: 104,719.97

## 도입: 0-5분

오늘 다루는 건 엑셀 실무에서 가장 손이 자주 가는 함수 묶음이다. 주문 데이터를 열어 보면 주문번호, 고객번호, 상품번호, 수량, 할인율은 있는데 지역도 단가도 없다. 지역별 매출을 내려면 지역과 매출이 둘 다 있어야 하니, 지금 표 하나만으로는 답이 안 나온다.

실무에서는 흔한 장면이다. 거래 기록 따로, 고객 정보 따로, 상품 정보 따로 놓여 있다. 분석하려면 이 표들부터 연결해야 한다. 오늘 목표는 `orders`를 가운데 두고 `customers`와 `products`에서 필요한 값을 끌어오는 것이다.

## 데이터 구조 설명: 5-12분

시연 포인트:

1. `orders` 시트를 연다.
2. `customer_id`, `product_id`, `qty`, `discount_rate`를 짚는다.
3. `customers` 시트에서 `customer_id`, `region`, `grade`를 짚는다.
4. `products` 시트에서 `product_id`, `price`를 짚는다.

말할 내용:

`orders`는 주문이 한 줄씩 쌓인 거래 기록이다. 그런데 주문표만 봐서는 `P00159`가 얼마짜리인지 알 도리가 없다. 상품번호를 들고 `products`라는 사전을 뒤져야 한다. 고객도 똑같다. `C000787`이 어느 지역, 어떤 등급 고객인지는 `customers`에서 찾아야 나온다.

여기서 기억할 단어가 키다. 표와 표를 이을 때 기준이 되는 값을 키라고 부른다. 오늘은 `product_id`와 `customer_id`가 그 키다.

## VLOOKUP 시연: 12-25분

시연 포인트:

1. `orders` 시트 G:L 열에 새 제목을 만든다.
2. G2, H2, I2에 상품 관련 VLOOKUP을 입력한다.
3. J2, K2에 고객 관련 VLOOKUP을 입력한다.
4. 한두 줄 결과를 눈으로 확인한 뒤 아래로 채운다.

입력 공식:

```excel
=VLOOKUP(C2,products!$A:$E,2,FALSE)
```

```excel
=VLOOKUP(C2,products!$A:$E,3,FALSE)
```

```excel
=VLOOKUP(C2,products!$A:$E,4,FALSE)
```

```excel
=VLOOKUP(B2,customers!$A:$G,5,FALSE)
```

```excel
=VLOOKUP(B2,customers!$A:$G,7,FALSE)
```

말할 내용:

`VLOOKUP`은 세로로 늘어선 표에서 값을 찾는 함수다. 인수는 순서대로 찾을 값, 찾을 표, 가져올 열 번호, 그리고 정확히 찾을지 여부다. 고객번호와 상품번호는 정확히 일치해야 하니 맨 끝에는 `FALSE`를 넣는다.

수강생이 가장 헷갈려 하는 건 세 번째 인수다. `products!$A:$E` 범위에서는 A열이 1번, B열이 2번, C열이 3번, D열이 4번이다. 그러니 단가인 `price`를 가져오려면 4를 쓴다.

## XLOOKUP 비교: 25-30분

시연 포인트:

1. VLOOKUP으로 이미 계산한 값을 지우지 않는다.
2. 빈 셀에 XLOOKUP 예시를 보여 주거나 화면에 공식만 설명한다.

예시 공식:

```excel
=XLOOKUP(C2,products!$A:$A,products!$D:$D,"확인필요")
```

말할 내용:

`XLOOKUP`은 찾을 열과 가져올 열을 따로 지정하니 열 번호를 셀 일이 없다. 최신 엑셀이라면 이쪽이 읽기 쉽고 덜 헷갈린다. 다만 회사마다 엑셀 버전이 제각각이라, 오늘은 VLOOKUP을 기본으로 익히고 XLOOKUP은 한 수 나은 대안으로만 알아 둔다.

## 매출 계산: 30-36분

시연 포인트:

1. L2 셀에 매출 공식을 입력한다.
2. 첫 세 행의 값이 기준과 맞는지 확인한다.
3. L열 전체를 채운다.

입력 공식:

```excel
=I2*E2*(1-F2)
```

말할 내용:

매출은 단가에 수량을 곱한 뒤 할인율을 반영한 값이다. 할인율이 5%면 고객은 정가의 95%를 낸다. 그래서 `1 - 할인율`을 곱한다.

확인할 값:

- L2: 22,900
- L3: 35,910
- L4: 105,800

이 값이 어긋나면 십중팔구 단가를 가져온 열이 틀렸거나 수량 컬럼을 잘못 참조한 경우다. 이 데이터의 수량 컬럼은 `qty`이고 E열에 있다.

## 지역별 SUMIF/COUNTIF: 36-43분

시연 포인트:

1. `region_summary` 시트를 만든다.
2. 지역 목록을 만들고 중복 제거한다.
3. `COUNTIF`, `SUMIF`, 객단가 공식을 입력한다.
4. 전체 합계를 검산한다.

입력 공식:

```excel
=COUNTIF(orders!$J$2:$J$20001,A2)
```

```excel
=SUMIF(orders!$J$2:$J$20001,A2,orders!$L$2:$L$20001)
```

```excel
=C2/B2
```

말할 내용:

`SUMIF`는 조건에 맞는 값만 골라 더한다. 여기서는 J열 지역이 A2와 같은 행만 추려 L열 매출을 합친다. `COUNTIF`는 그 같은 조건의 주문 건수를 센다. 매출을 주문 건수로 나누면 객단가다.

검산 기준:

- 주문 건수 합계: 20,000
- 매출 합계: 1,938,489,803
- 서울특별시 매출: 570,482,124
- 경기도 매출: 530,726,511

## 정리: 48-50분

말할 내용:

오늘 한 일이 곧 데이터분석의 기본 흐름이다. 필요한 값을 여러 표에서 끌어와 한 표로 모으고, 매출이라는 계산 컬럼을 새로 만들고, 끝으로 조건별 합계와 건수를 구해 질문에 답했다.

엑셀은 이 과정을 눈으로 좇으며 따라가기에 좋다. 그러나 함수가 불어나고 파일이 무거워지면 실수가 끼어들 자리도 그만큼 늘어난다. 그래서 다음부터는 같은 작업을 더 되풀이하기 쉽게 만드는 도구를 배운다.

## 예상 질문과 답변

Q. 왜 `FALSE`를 꼭 넣어야 하나요?

A. 고객번호와 상품번호는 정확히 같은 값을 찾아야 한다. `FALSE`를 빼면 엑셀이 비슷한 값을 찾는 방식으로 움직일 수 있고, 그러면 엉뚱한 값이 딸려 올 위험이 있다.

Q. VLOOKUP과 XLOOKUP 중 무엇을 쓰면 되나요?

A. 회사 엑셀이 XLOOKUP을 지원하면 XLOOKUP이 읽기 편하다. 다만 VLOOKUP은 오래된 파일이나 조직에서도 여전히 흔하게 쓰이니, 읽고 고칠 줄은 알아야 한다.

Q. 숫자가 정답과 조금 다릅니다.

A. 먼저 L열 매출 공식이 `=I2*E2*(1-F2)`인지 본다. 다음으로 I열 단가가 `products`의 네 번째 열인 `price`를 제대로 가져오는지 확인한다. 지역이나 등급별 집계라면 조건 범위와 합계 범위가 서로 다른 열을 가리키고 있어야 한다.

Q. 지역 목록은 꼭 중복 제거로 만들어야 하나요?

A. 꼭 그렇진 않다. 다만 입문 단계에서는 눈으로 확인하기 쉬워 중복 제거를 쓴다. 최신 엑셀이라면 `=SORT(UNIQUE(orders!J2:J20001))`로도 만들 수 있다.

Q. 객단가는 고객 1명당 평균 구매액인가요?

A. 이 실습에서 말하는 객단가는 주문 1건당 평균 매출이다. 고객 1명당 평균 구매액을 보려면 고객별로 먼저 묶는 계산이 따로 필요하다.
