# S02 강사대본. 좋은 데이터의 모양

## 수업 전 준비

- 강사용 PC에서 아래 파일이 열리는지 확인한다.
  - `/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.xlsx`
  - `/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_dirty.xlsx`
- 두 파일 모두 `sales` 시트가 있다.
- 실측 기준:
  - clean: 10,424행, 12열
  - dirty: 10,439행, 12열
- 엑셀 행 번호는 머리글 때문에 각각 10,425와 10,440까지 보일 수 있다.

## 도입: 0~5분

말할 것:

“지난 시간에는 데이터 분석이 무엇인지, 우리가 앞으로 엑셀, AI, 파이썬, BI를 어떻게 연결해서 쓸지 큰 지도를 봤습니다. 오늘은 그 도구들이 공통으로 좋아하는 표의 모양을 봅니다. 엑셀을 잘 쓰는 분일수록 예쁘게 꾸민 표에 익숙할 수 있습니다. 그런데 분석용 표는 예쁜 표가 아니라 규칙적인 표여야 합니다.”

“오늘의 기준은 간단합니다. 한 행은 주문 한 건, 한 열은 한 가지 속성, 한 셀은 한 값입니다.”

시연 포인트:

- `cafe_sales_clean.xlsx`를 열고 한 행을 가로로 읽는다.
- “이 한 줄이 주문 한 건입니다”라고 짚는다.
- `amount`는 주문금액, `qty`는 수량처럼 한 열이 하나의 의미만 갖는다고 말한다.

## 개념 설명: 5~15분

말할 것:

“분석용 데이터에서 병합셀은 되도록 쓰지 않습니다. 보고서에서는 병합셀이 보기 좋지만, 분석 도구는 병합된 영역을 사람처럼 읽지 못하는 경우가 많습니다. 위쪽 셀에만 값이 있고 아래쪽은 빈 값으로 보일 수 있습니다.”

“또 하나 중요한 것은 한 셀 한 값입니다. `4,500원`은 사람에게는 친절하지만 컴퓨터에게는 숫자 4500이 아니라 글자가 섞인 값일 수 있습니다. 그러면 합계와 평균 계산을 바로 할 수 없습니다.”

“숫자로 보이는 값도 다 같은 숫자가 아닙니다. `order_id`는 숫자처럼 보이지만 평균 주문번호를 구하는 것은 의미가 없습니다. 주문번호는 계산할 숫자가 아니라 행을 구분하는 이름표입니다.”

판서 또는 화면 메모:

| 원칙 | 한 문장 설명 |
|---|---|
| 한 행 한 관찰 | 주문 한 건이 한 줄 |
| 한 열 한 변수 | 날짜, 지점, 상품명, 수량을 따로 |
| 한 셀 한 값 | 숫자와 단위, 값 두 개를 섞지 않음 |
| 병합셀 금지 | 빈 값과 구조 오류를 만들 수 있음 |

## dirty 데이터 시연: 15~25분

시연 순서:

1. `cafe_sales_dirty.xlsx`를 연다.
2. 1행에 필터를 켠다.
3. `store` 필터를 열어 `강남점`과 `강남 점`을 보여준다.
4. `item` 필터에서 `아메리카노`, `아메리카`, `아메 리카노`, `Americano`를 보여준다.
5. `category` 필터에서 앞뒤 공백이 있는 값을 보여준다.
6. `unit_price` 필터 검색창에 `원`을 입력해 `4,500원` 형태를 보여준다.
7. `payment`와 `member`의 빈 값을 보여준다.
8. `qty`에서 999를 필터링한다.
9. `amount`에서 0보다 작은 값을 필터링한다.

말할 것:

“이 파일은 망가진 파일이 아니라 연습용 파일입니다. 실제 업무 데이터에서 이 정도 문제는 흔합니다. 중요한 건 처음부터 완벽한 표를 바라는 게 아니라, 어떤 문제가 분석 결과를 흔드는지 알아보는 눈입니다.”

“`강남점`과 `강남 점`은 사람에게는 같은 지점입니다. 그러나 피벗테이블은 둘을 다른 지점으로 볼 수 있습니다. 이 상태로 지점별 매출을 구하면 강남점 매출이 둘로 갈라집니다.”

“`unit_price`에 `원`이 붙어 있으면 사람에게는 읽기 쉽지만, 분석에서는 숫자 열이 문자 열로 바뀔 수 있습니다. 가격 열은 가격만 담아야 합니다.”

주의할 점:

- 이 차시에서는 완전한 정제 실습을 하지 않는다.
- 목표는 문제를 고치는 기술보다 문제를 알아보는 기준이다.
- 결측값을 무조건 삭제한다고 말하지 않는다. 결측은 삭제, 대체, 별도 범주, 원자료 확인 중 선택해야 한다고 설명한다.

## 척도 분류 활동: 25~33분

말할 것:

“이제 각 컬럼이 어떤 종류의 값인지 분류해 보겠습니다. 척도는 어려운 통계 용어처럼 들리지만, 실제로는 이 컬럼으로 무엇을 할 수 있는지 정하는 기준입니다.”

진행:

- 수강생에게 3분간 `order_id`부터 `member`까지 척도를 적게 한다.
- 이후 함께 모범 분류를 맞춘다.

강사용 기준:

| 컬럼 | 척도 | 설명 |
|---|---|---|
| `order_id` | 식별자 | 주문을 구분하는 이름표 |
| `date` | 날짜형 | 일별, 월별, 요일별 분석 가능 |
| `weekday` | 순서가 있는 범주형 | 요일 순서가 있지만 평균 요일은 의미 없음 |
| `hour` | 시간형 수치 | 시간대 정렬, 시간대별 집계 가능 |
| `store` | 범주형 | 지점 그룹 |
| `category` | 범주형 | 상품 대분류 |
| `item` | 범주형 | 상품명 |
| `unit_price` | 수치형 | 단가 계산 가능 |
| `qty` | 수치형 | 수량 합계, 평균 가능 |
| `amount` | 수치형 | 매출 합계, 평균 가능 |
| `payment` | 범주형 | 결제수단 그룹 |
| `member` | 논리형 | 회원 여부 |

강조할 말:

“척도 분류는 정답 맞히기 시험이 아닙니다. 중요한 것은 왜 그렇게 보았는지 설명하는 것입니다. 예를 들어 `weekday`는 범주형으로 봐도 되고, 월요일부터 일요일까지 순서가 있는 범주로 봐도 됩니다. 다만 평균을 내는 숫자는 아닙니다.”

## 예상 질문과 답변

### 질문 1. 병합셀을 이미 많이 쓰고 있는데 전부 없애야 하나요?

답변:

보고용 시트와 분석용 원본 시트를 분리하면 된다. 보고용 시트에서는 병합셀을 사용할 수 있지만, 피벗테이블이나 분석에 연결할 원본 시트에서는 병합셀을 피하는 것이 좋다.

### 질문 2. `4,500원`처럼 쓰면 엑셀에서도 계산되는 경우가 있는데 왜 문제인가요?

답변:

엑셀 화면에서는 계산되는 것처럼 보여도 다른 도구로 옮기거나 CSV로 저장하면 문자로 처리될 수 있다. 분석용 원본에는 숫자만 저장하고, `원` 표시는 셀 서식으로 처리하는 편이 안전하다.

### 질문 3. 결측값은 그냥 삭제하면 되나요?

답변:

항상 삭제하는 것은 아니다. 결측이 적고 분석에 큰 영향이 없으면 삭제할 수 있지만, 특정 지점이나 특정 결제수단에서만 결측이 많다면 중요한 신호일 수 있다. 삭제, 대체, 별도 범주 처리, 원자료 확인 중에서 목적에 맞게 결정한다.

### 질문 4. `weekday`는 범주형인가요, 순서형인가요?

답변:

요일 이름은 그룹을 나누는 값이므로 범주형이다. 다만 월요일부터 일요일까지 자연스러운 순서가 있으므로 “순서가 있는 범주형”으로 다룰 수 있다. 평균 요일을 계산하는 수치형은 아니다.

### 질문 5. `member`는 참과 거짓인데 범주형인가요?

답변:

회원과 비회원 두 그룹을 나누므로 범주형으로 볼 수 있고, 값이 `True`와 `False` 두 개뿐이므로 논리형으로도 부른다. 분석에서는 회원 비율, 회원 여부별 매출 비교에 자주 쓴다.

## 완료 판단 체크포인트

수강생이 `order_id`를 식별자라고 설명하고, dirty 데이터에서 같은 범주의 표기 혼재와 숫자·문자 혼합 문제를 각각 하나 이상 찾아 교정 방향을 말하면 완료로 판단한다.
