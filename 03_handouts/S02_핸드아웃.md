# S02 핸드아웃. 좋은 데이터의 모양

## 오늘 사용할 데이터

분석하기 좋은 표는 사람이 보기 좋게 꾸민 표와 다릅니다. 엑셀에서 제목을 크게 만들고, 셀을 병합하고, 중간에 소계를 넣으면 사람 눈에는 편할 수 있습니다. 그런데 피벗테이블이나 분석 도구는 그런 장식보다 “한 줄이 무엇을 뜻하는지”, “한 열에 어떤 값이 들어 있는지”를 더 중요하게 봅니다.

수업 홈의 데이터 받기 페이지에서 아래 두 파일을 내려받아 사용합니다.

- 카페 매출 정리본(clean) 파일
- 카페 매출 문제 연습본(dirty) 파일

clean은 분석하기 좋게 정리된 상태를 뜻하고, dirty는 일부러 문제를 섞어 둔 연습용 상태를 뜻합니다. 엑셀로도 열리는 표 파일인 CSV가 함께 보이더라도, 처음 실습하는 분은 엑셀 파일(.xlsx)을 내려받아도 됩니다.

두 파일 모두 `sales` 시트가 있으며 컬럼은 12개입니다. clean 파일은 10,424행, dirty 파일은 10,439행입니다. 엑셀에서는 머리글 행까지 포함되므로 화면상 마지막 행 번호는 각각 10,425와 10,440으로 보일 수 있습니다. 화면의 마지막 행 번호가 실제 데이터 건수보다 1 크게 보이는 이유는 1행이 데이터가 아니라 컬럼 이름이 적힌 머리글이기 때문입니다.

## 따라하기 1. 분석하기 좋은 표 모양 확인

1. 수업 홈의 데이터 받기 페이지에서 카페 매출 정리본(clean) 파일을 내려받아 엽니다.
   - 이 파일은 문제가 적은 기준 표입니다. 좋은 표의 모양을 먼저 봐야 나중에 문제 있는 표가 왜 불편한지 비교할 수 있습니다.
2. `sales` 시트를 확인합니다.
   - 한 엑셀 파일 안에 여러 장의 종이가 들어 있다고 생각하면 쉽습니다. 지금은 그중 매출 데이터가 들어 있는 `sales` 시트를 보는 것입니다.
3. 1행의 머리글을 왼쪽부터 읽습니다.
   - 머리글은 각 열의 이름표입니다. 사전에서 단어를 찾듯이, 이 이름표를 보면 각 열이 무엇을 기록하는지 알 수 있습니다.
   - `order_id`, `date`, `weekday`, `hour`, `store`, `category`, `item`, `unit_price`, `qty`, `amount`, `payment`, `member`
4. 2행을 가로로 읽습니다.
   - 2행은 실제 주문 데이터가 시작되는 첫 줄입니다. 이 줄 하나가 주문 한 건을 설명하는지 확인해 보세요.
   - 한 행이 주문 한 건을 설명하는지 확인합니다.
5. 각 열을 세로로 훑어봅니다.
   - 세로로 보면 한 열 안에 같은 종류의 값만 들어 있는지 확인할 수 있습니다. 키를 재는 줄자에는 키만 적혀 있어야 하듯이, 한 열에는 한 가지 의미의 값만 들어 있어야 합니다.
   - `date`에는 날짜만 있습니다.
   - `store`에는 지점명만 있습니다.
   - `qty`에는 수량만 있습니다.
   - `amount`에는 주문금액만 있습니다.

확인한 내용: 이 표는 “주문 한 건이 한 행, 주문의 속성이 각 열”인 구조입니다. 이런 구조에서는 피벗테이블과 분석 도구가 값을 안정적으로 읽을 수 있습니다.

## 따라하기 2. dirty 표에서 문제 찾기

1. 수업 홈의 데이터 받기 페이지에서 카페 매출 문제 연습본(dirty) 파일을 내려받아 엽니다.
   - 이 파일에는 실무에서 자주 만나는 데이터 문제가 들어 있습니다. 일부러 문제를 찾아보면서 분석 전에 무엇을 고쳐야 하는지 익힙니다.
2. `sales` 시트에서 1행 아무 셀이나 클릭합니다.
   - 1행은 머리글입니다. 머리글을 클릭해 두면 엑셀이 각 열 이름을 기준으로 필터를 걸기 쉽습니다.
3. 엑셀 상단 메뉴에서 `데이터`를 누릅니다.
   - 데이터 메뉴에는 정렬, 필터처럼 표를 점검할 때 자주 쓰는 도구가 모여 있습니다.
4. `필터`를 클릭합니다.
   - 필터를 켜면 각 열에 어떤 값들이 들어 있는지 목록으로 볼 수 있습니다.
5. 각 컬럼의 필터 버튼을 눌러 값 목록을 확인합니다.
   - 값 목록은 그 열의 작은 요약표처럼 볼 수 있습니다. 같은 값인지, 다른 값인지, 비어 있는 값이 있는지 여기서 먼저 눈에 들어옵니다.

### 2-1. 지점 표기 문제 찾기

1. `store` 컬럼의 필터 버튼을 누릅니다.
   - `store`는 지점을 나타내는 열입니다. 지점별 매출을 계산하려면 지점 이름이 흔들리지 않아야 합니다.
2. 값 목록에서 다음 값이 함께 있는지 확인합니다.
   - 값 목록에 비슷한 이름이 나란히 보이면 표기 문제가 있는지 의심해 볼 수 있습니다.
   - `강남점`
   - `강남 점`
   - `홍대점`
3. `강남점`과 `강남 점`은 같은 지점으로 보아야 합니다.
   - 사람은 같은 지점이라고 알아차리지만, 컴퓨터는 띄어쓰기가 다르면 다른 이름으로 볼 수 있습니다.

기록할 내용: `store`는 범주형 컬럼이지만 같은 범주의 표기가 섞여 있습니다.

### 2-2. 상품명 표기 문제 찾기

1. `item` 컬럼의 필터 버튼을 누릅니다.
   - `item`은 상품명을 나타내는 열입니다. 상품별 매출을 보려면 같은 상품이 같은 이름으로 적혀 있어야 합니다.
2. 검색창에 `아메`를 입력합니다.
   - 전체 상품명을 다 훑기 어렵기 때문에, 일부 글자를 넣어 비슷한 상품명만 모아 봅니다.
3. 다음처럼 비슷하지만 다른 값이 있는지 확인합니다.
   - 비슷한 이름이 여러 개 보이면 같은 상품의 매출이 여러 그룹으로 나뉠 수 있습니다.
   - `아메리카노`
   - `아메리카`
   - `아메 리카노`
4. 필터 검색창에 `Americano`도 입력해 봅니다.
   - 한글 이름과 영어 이름이 함께 쓰이면 사람은 같은 상품으로 볼 수 있어도 엑셀은 다른 상품으로 계산합니다.

기록할 내용: 같은 상품이 여러 이름으로 기록되면 상품별 매출이 나뉘어 계산됩니다.

### 2-3. 카테고리 앞뒤 공백 찾기

1. `category` 컬럼의 필터 버튼을 누릅니다.
   - `category`는 커피, 음료, 디저트처럼 상품을 큰 묶음으로 나누는 열입니다.
2. 값 목록에서 `커피`와 ` 커피 `처럼 앞뒤 공백이 있는 값이 따로 보이는지 확인합니다.
   - 눈으로는 거의 같아 보여도, 앞뒤에 빈칸이 들어가면 엑셀은 다른 값으로 취급할 수 있습니다.
3. `음료`, `디저트`도 같은 방식으로 확인합니다.
   - 한 열에서 같은 문제가 반복되는지 보는 과정입니다. 한 군데에서 공백 문제가 보이면 다른 카테고리에도 있을 수 있습니다.

기록할 내용: 사람 눈에는 거의 같아도 컴퓨터는 공백이 있으면 다른 값으로 처리할 수 있습니다.

### 2-4. 단가 컬럼의 숫자와 문자 혼합 찾기

1. `unit_price` 컬럼의 필터 버튼을 누릅니다.
   - `unit_price`는 단가입니다. 단가는 합계나 평균 계산에 쓰이므로 숫자로 다뤄져야 합니다.
2. 검색창에 `원`을 입력합니다.
   - 숫자 열에 문자 `원`이 섞여 있는지 빠르게 찾기 위해 검색합니다.
3. `4,500원`, `5,500원`처럼 숫자와 문자가 함께 들어간 값을 확인합니다.
   - 사람에게는 읽기 편하지만, 계산 도구는 이것을 순수한 숫자로 보지 못할 수 있습니다.

기록할 내용: 단가 열은 계산에 쓰는 수치형이어야 하므로 `원`과 쉼표를 제거하고 숫자로 맞춰야 합니다.

### 2-5. 날짜 형식 혼합 찾기

1. `date` 컬럼의 값을 훑어봅니다.
   - 날짜 열은 일별, 월별, 요일별 분석의 기준이 됩니다. 그래서 날짜로 제대로 인식되는지가 중요합니다.
2. 다음처럼 날짜 형식이 섞여 있는지 확인합니다.
   - 같은 날짜라도 적는 방식이 섞이면 엑셀이 일부 값을 날짜가 아니라 글자로 볼 수 있습니다.
   - `2025-03-23`
   - `2025/03/23`
   - `03-23-2025`

기록할 내용: 날짜가 여러 형식으로 섞이면 월별 추세나 요일 분석 전에 날짜형으로 통일해야 합니다.

### 2-6. 결측값 찾기

1. `payment` 컬럼의 필터 버튼을 누릅니다.
   - `payment`는 결제수단입니다. 결제수단별 매출을 보려면 비어 있는 값이 있는지 먼저 확인해야 합니다.
2. 값 목록에서 `(빈 셀)` 또는 비어 있는 항목을 선택합니다.
   - 필터에서 빈 셀만 고르면 결제수단이 빠진 행을 따로 볼 수 있습니다.
3. 결제수단이 비어 있는 행을 확인합니다.
   - 빈 값이 몇 건인지, 어떤 주문에서 비었는지 보는 과정입니다.
4. 필터를 해제합니다.
   - 다음 열을 확인하려면 이전 필터를 풀어야 전체 데이터를 다시 볼 수 있습니다.
5. `member` 컬럼에서도 같은 방식으로 빈 셀을 확인합니다.
   - `member`도 회원 여부를 나타내는 중요한 열입니다. 같은 방법으로 비어 있는 값을 찾아 봅니다.

기록할 내용: 비어 있는 값은 무시할지, 별도 범주로 둘지, 원자료를 확인해 채울지 결정해야 합니다.

### 2-7. 수량 이상치 찾기

1. `qty` 컬럼의 필터 버튼을 누릅니다.
   - `qty`는 수량입니다. 수량은 합계와 평균에 바로 영향을 주기 때문에 이상한 값이 있는지 봐야 합니다.
2. 값 목록에서 `999`가 있는지 확인합니다.
   - 카페 주문에서 수량 `999`는 일반적인 주문이라고 보기 어렵습니다. 이런 값은 이상치, 즉 너무 튀는 값으로 의심합니다.
3. `999`만 선택해 해당 행을 봅니다.
   - 이상한 값이 실제로 어떤 주문에 들어 있는지 행 전체를 확인해야 입력 오류인지 판단할 수 있습니다.

기록할 내용: 카페 주문 수량이 999인 값은 입력 오류일 가능성이 높습니다. 평균과 총합을 왜곡할 수 있습니다.

### 2-8. 음수 금액 찾기

1. `amount` 컬럼의 필터 버튼을 누릅니다.
   - `amount`는 주문금액입니다. 매출 분석에서 가장 자주 합산하는 열이라 값의 방향이 중요합니다.
2. 숫자 필터에서 `보다 작음`을 선택합니다.
   - 음수만 찾기 위해 숫자 조건을 거는 단계입니다.
3. 기준값에 `0`을 입력합니다.
   - `0`보다 작은 값은 음수입니다. 이렇게 조건을 걸면 음수 금액 행만 볼 수 있습니다.
4. 음수 금액 행이 있는지 확인합니다.
   - 환불처럼 따로 설명된 데이터가 아니라면, 매출 금액이 음수인 행은 원인을 확인해야 합니다.

기록할 내용: 환불 데이터라는 별도 설명이 없다면 매출 금액이 음수인 행은 확인이 필요합니다.

## 따라하기 3. 컬럼별 척도 분류

## 스스로 점검하기

아래 문장에 답할 수 있으면 오늘 차시를 통과한 것입니다. 병합셀은 여러 셀을 하나처럼 합쳐 보이게 하는 기능입니다. 사람 눈에는 보기 좋아도, 분석 도구는 병합된 셀 아래쪽을 빈칸으로 읽는 경우가 많습니다. 그래서 분석용 원본 표에서는 병합셀을 피해야 합니다.

- “왜 병합셀은 분석용 원본 표에 쓰지 않는가?”
- “왜 `order_id`는 숫자처럼 보여도 수치형 분석 대상이 아닌가?”
- “왜 `4,500원`은 사람이 보기에는 편하지만 분석에는 불편한가?”
- “같은 상품명이 여러 표기로 들어가면 피벗테이블 결과가 어떻게 틀어지는가?”
