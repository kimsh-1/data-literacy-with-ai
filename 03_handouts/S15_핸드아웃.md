# S15 핸드아웃: Power BI 설치, 로드, 첫 차트

## 오늘 사용할 것

- Windows PC
- Power BI Desktop
- 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일
  - 엑셀로도 열리는 표 파일인 CSV 파일
  - 같은 데이터를 담은 엑셀 파일

Power BI Desktop은 Windows용 보고서 제작 프로그램입니다. 표 데이터를 불러와서 카드, 막대그래프, 꺾은선그래프 같은 보고 화면을 만드는 도구라고 보면 됩니다. Mac 수강생은 오늘 화면 구조와 개념을 눈으로 따라오면 되고, 직접 실습은 S18의 Looker Studio에서 이어 갑니다.

공식 설치 페이지는 아래 두 곳입니다.

- Microsoft Power BI 다운로드: `https://www.microsoft.com/en-us/power-platform/products/power-bi/downloads`
- Microsoft Store Power BI Desktop: `https://apps.microsoft.com/detail/9ntxr16hnw1t`

오늘 실습 화면은 카페 매출 CSV 파일을 기준으로 맞춥니다. 엑셀 파일은 같은 데이터를 담은 보조 파일입니다. CSV가 낯설면 초보자는 엑셀 파일(.xlsx)을 받아도 됩니다. 다만 한 Power BI 파일에 CSV와 엑셀을 동시에 불러오지는 않습니다. 같은 재료를 냄비 두 개에 동시에 넣으면 맛을 비교하기 어려운 것처럼, 오늘은 한 가지 방식으로만 맞춰 봅니다.

## 따라하기 1: Power BI Desktop 열기

### 1단계. 설치 확인

1. Windows 시작 메뉴를 엽니다.  
   설치된 프로그램은 시작 메뉴에서 찾는 것이 가장 빠릅니다.
2. `Power BI Desktop`을 검색합니다.  
   정확한 프로그램 이름으로 찾으면 비슷한 Microsoft 앱과 헷갈리지 않습니다.
3. 프로그램이 있으면 실행합니다.  
   화면을 먼저 열어야 데이터 가져오기와 차트 만들기를 따라 할 수 있습니다.
4. 프로그램이 없으면 강사의 안내에 따라 Microsoft Store 또는 Microsoft 공식 다운로드 페이지에서 설치합니다.  
   Power BI Desktop은 설치형 프로그램이라서, PC에 없으면 보고서 파일을 만들 수 없습니다.

처음 실행하면 로그인 화면이 뜰 수 있습니다. 오늘은 로컬 파일로 보고서를 만드는 실습이니, 강사 안내에 맞춰 닫거나 건너뛰면 됩니다. 조직 PC라면 보안 정책 때문에 설치나 로그인이 막히기도 합니다. 이럴 때는 프로그램 문제가 아니라 회사나 학교 PC의 설정 문제일 수 있습니다.

### 2단계. 빈 보고서 화면 확인

Power BI Desktop을 열면 화면을 바로 만지기 전에 영역 이름부터 확인합니다. 지도에서 현재 위치를 먼저 보는 것과 같습니다. 어디에 차트를 놓고, 어디에서 열을 고르고, 어디에서 차트 모양을 바꾸는지 알아야 뒤에서 덜 헤맵니다.

| 화면 영역 | 볼 것 |
|---|---|
| 가운데 흰 영역 | 차트를 놓는 캔버스 |
| 오른쪽 데이터 패널 | 불러온 표와 열 목록 |
| 오른쪽 시각화 패널 | 막대그래프, 꺾은선그래프, 카드 선택 |
| 위쪽 리본 메뉴 | 데이터 가져오기, 저장, 새 시각적 개체 |

## 따라하기 2: CSV 파일 불러오기

### 1단계. 데이터 가져오기

1. 위쪽 메뉴에서 `홈(Home)`을 누릅니다.  
   Power BI에서 데이터를 불러오는 명령은 주로 홈 메뉴에 모여 있습니다.
2. `데이터 가져오기(Get data)`를 누릅니다.  
   이 버튼은 Power BI에게 “이제 분석할 표를 가져오겠다”라고 알려 주는 입구입니다.
3. `텍스트/CSV(Text/CSV)`를 선택합니다.  
   카페 매출 CSV 파일은 행과 열로 된 표 파일이므로 이 선택지를 사용합니다.
4. 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 CSV 파일을 선택합니다.  
   오늘 사용할 매출 표를 정확히 골라야 뒤에서 총매출과 그래프 값이 맞습니다.
5. 미리보기 화면이 열리면 열 이름을 확인합니다.  
   열 이름을 보는 이유는 잘못된 파일을 골랐는지 바로 알아차리기 위해서입니다.
6. `로드(Load)`를 누릅니다.  
   로드를 눌러야 표가 Power BI 안으로 들어오고, 오른쪽 데이터 패널에서 열을 끌어다 쓸 수 있습니다.

### 2단계. 데이터 패널 확인

오른쪽 데이터 패널에 `cafe_sales_clean` 표가 생겼는지 확인합니다. 표를 펼쳐 아래 열이 보이면 정상입니다. 사전에서 낱말을 찾듯이, 이 패널에서 우리가 그래프에 넣을 열을 고르게 됩니다.

```text
order_id
date
weekday
hour
store
category
item
unit_price
qty
amount
payment
member
```

확인할 것:

- `amount`는 숫자 열이어야 합니다. 매출을 더해야 하므로 글자가 아니라 숫자로 인식되어야 합니다.
- `date`는 날짜 열로 인식되는 것이 좋습니다. 날짜가 제대로 잡혀야 일자별 흐름을 선으로 볼 수 있습니다.
- 오늘은 정제 실습이 아니므로 복잡한 변환은 하지 않습니다. 이미 정리된 표를 Power BI에 올려 첫 보고 화면을 만드는 데 집중합니다.

`date`가 날짜로 인식되지 않으면, 강사 화면을 따라 `변환 데이터(Transform data)`에서 형식을 날짜로 바꾼 뒤 `닫기 및 적용(Close & Apply)`을 누릅니다. 날짜 형식을 맞추는 일은 달력 자를 대고 순서를 맞추는 작업이라고 생각하면 됩니다.

## 따라하기 3: 엑셀 파일 로드 경로 확인하기

이 단계는 엑셀 파일을 불러오는 메뉴 위치만 눈에 익히는 용도입니다. 이미 CSV를 불러온 파일에는 엑셀을 덧붙이지 않습니다. 같은 표를 두 번 넣으면 결과가 두 배로 보이거나, 어떤 표를 기준으로 봐야 하는지 헷갈릴 수 있습니다.

새 Power BI 파일에서 엑셀을 불러올 때는 아래 순서를 따릅니다.

1. `홈(Home)`을 누릅니다.  
   외부 데이터를 가져오는 시작점은 CSV와 마찬가지로 홈 메뉴입니다.
2. `데이터 가져오기(Get data)`를 누릅니다.  
   Power BI에 새 표를 연결하겠다는 뜻입니다.
3. `Excel 통합 문서(Excel workbook)`를 선택합니다.  
   엑셀 파일은 CSV와 가져오는 입구가 다르기 때문에 이 항목을 고릅니다.
4. 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 엑셀 파일을 선택합니다.  
   CSV와 같은 내용을 담은 엑셀 파일로 메뉴 흐름만 확인합니다.
5. 탐색기(Navigator)에서 `sales` 시트를 선택합니다.  
   엑셀 파일 안에는 여러 시트가 있을 수 있으므로, 어떤 시트를 표로 가져올지 정해야 합니다.
6. `로드(Load)`를 누릅니다.  
   이 버튼을 눌러야 선택한 시트가 Power BI의 데이터 패널에 들어옵니다.

CSV와 엑셀은 같은 데이터입니다. 실습 결과를 맞추기 위해, 이후 단계는 CSV로 불러온 `cafe_sales_clean` 표를 기준으로 삼습니다.

## 따라하기 4: 카테고리별 매출 막대그래프 만들기

### 1단계. 막대그래프 추가

1. 시각화 패널에서 `묶은 세로 막대형 차트(Clustered column chart)`를 선택합니다.  
   막대그래프는 항목끼리 크기를 비교할 때 좋습니다.
2. 캔버스에 빈 막대그래프가 생기면 오른쪽 필드 영역을 봅니다.  
   빈 차트는 아직 재료가 올라가지 않은 접시와 같습니다. 이제 기준과 숫자를 넣어야 합니다.
3. 데이터 패널에서 `category`를 `X축(X-axis)`으로 끌어다 놓습니다.  
   `category`는 커피, 음료, 디저트처럼 무엇끼리 비교할지 정하는 기준입니다.
4. 데이터 패널에서 `amount`를 `Y축(Y-axis)` 또는 `값(Values)`으로 끌어다 놓습니다.  
   `amount`는 매출 금액이므로, 막대의 높이를 만드는 숫자입니다.

### 2단계. 집계 방식 확인

`amount`가 `합계(Sum)`로 들어갔는지 확인합니다. 매출 비교는 “얼마나 팔렸는가”를 보는 일이므로 합계가 맞습니다. 평균이나 개수로 잡히면 질문이 달라집니다. 예를 들어 평균은 주문 한 번당 금액을 보는 것이고, 개수는 몇 건인지를 세는 것입니다.

그래프 제목을 아래처럼 바꿉니다.

```text
카테고리별 매출
```

예상되는 카테고리별 매출 합계:

| category | amount 합계 |
|---|---:|
| 커피 | 45,606,000 |
| 음료 | 25,550,000 |
| 디저트 | 19,336,500 |

화면의 표시 단위에 따라 `45.61M`처럼 줄여 보이기도 합니다. 여기서 `M`은 백만 단위라는 뜻입니다. 정확한 숫자를 확인할 때는 표시 단위를 조정하면 됩니다.

## 따라하기 5: 일자별 매출 꺾은선그래프 만들기

### 1단계. 꺾은선그래프 추가

1. 캔버스의 빈 공간을 클릭합니다.  
   새 그래프를 만들 자리부터 선택해야 기존 막대그래프가 바뀌지 않습니다.
2. 시각화 패널에서 `꺾은선형 차트(Line chart)`를 선택합니다.  
   꺾은선그래프는 시간에 따라 숫자가 어떻게 움직이는지 볼 때 씁니다.
3. `date`를 `X축(X-axis)`으로 끌어다 놓습니다.  
   날짜는 왼쪽에서 오른쪽으로 흐르는 시간 자 역할을 합니다.
4. `amount`를 `Y축(Y-axis)` 또는 `값(Values)`으로 끌어다 놓습니다.  
   날짜마다 매출 합계가 어느 정도인지 선의 높이로 보게 됩니다.

### 2단계. 날짜 축 확인

Power BI가 `date`를 `날짜 계층 구조(Date hierarchy)`로 넣을 때가 있습니다. 일자별 흐름을 보려면 X축 필드의 드롭다운을 눌러 `date` 자체를 고릅니다. 화면이나 버전에 따라 메뉴 이름은 조금씩 다릅니다.

날짜 계층 구조는 연도, 분기, 월처럼 큰 단위로 접어 보는 방식입니다. 오늘은 하루하루의 매출 흐름을 보는 것이 목적이므로, 접힌 달력 대신 날짜 그대로를 축에 놓는다고 생각하면 됩니다.

그래프 제목을 아래처럼 바꿉니다.

```text
일자별 매출
```

데이터 기간은 `2025-01-01`부터 `2025-06-30`까지입니다.

## 직접 해보기: 총매출 KPI 카드 만들기

목표:

`amount`의 합계를 카드로 표시하고, 총매출이 `90,492,500`원과 일치하게 만듭니다. KPI 카드는 큰 셀 하나라고 생각하면 됩니다. 엑셀에서 합계 결과를 크게 보여 주듯이, 보고서에서 가장 먼저 봐야 할 숫자를 크게 보여 주는 자리입니다.

### 수행 조건

1. 캔버스의 빈 공간을 클릭합니다.  
   새 카드를 만들기 위해 기존 그래프 선택을 풀어 줍니다.
2. 시각화 패널에서 `카드(Card)` 또는 `카드(새)(Card new)`를 선택합니다.  
   카드는 숫자 하나를 크게 보여 주는 시각화입니다.
3. 데이터 패널에서 `amount`를 카드의 값 영역에 끌어다 놓습니다.  
   총매출은 주문금액 열인 `amount`를 더해서 만듭니다.
4. 값이 `합계(Sum of amount)`인지 확인합니다.  
   총매출을 보려면 평균이나 개수가 아니라 합계여야 합니다.
5. 카드 제목을 아래처럼 설정합니다.  
   제목을 붙여 두면 보는 사람이 이 숫자가 무엇인지 바로 압니다.

```text
총매출
```

6. 서식에서 표시 단위(Display units)를 `없음(None)`으로 바꿉니다.  
   Power BI가 숫자를 `90M`처럼 줄여 보이면 검산하기 어렵기 때문입니다.
7. 소수 자릿수(Decimal places)를 `0`으로 바꿉니다.  
   원 단위 금액을 볼 때는 소수점이 필요하지 않습니다.

### 정답 기준

카드 값이 아래 숫자와 일치해야 합니다.

```text
90,492,500
```

Power BI가 원 단위를 자동으로 붙여 주지 않으면, 제목을 `총매출(원)`으로 적어 두면 됩니다.

| 증상 | 확인할 것 |
|---|---|
| `amount` 합계가 이상하다 | `amount`가 숫자 열인지, 집계 방식이 합계인지 확인한다 |
| 카드가 `10,424`처럼 보인다 | `amount`가 아니라 `order_id` 개수가 들어갔을 수 있다 |
