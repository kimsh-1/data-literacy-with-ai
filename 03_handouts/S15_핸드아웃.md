# S15 핸드아웃: Power BI 설치, 로드, 첫 차트

## 오늘 사용할 것

- Windows PC
- Power BI Desktop
- 데이터 파일
  - `/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.csv`
  - `/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.xlsx`

Power BI Desktop은 Windows용 프로그램이다. Mac 수강생은 오늘 화면 구조와 개념만 눈으로 따라오면 되고, 실습은 S18의 Looker Studio에서 이어 간다.

공식 설치 경로:

- Microsoft Power BI 다운로드: `https://www.microsoft.com/en-us/power-platform/products/power-bi/downloads`
- Microsoft Store Power BI Desktop: `https://apps.microsoft.com/detail/9ntxr16hnw1t`

실습 기준은 `cafe_sales_clean.csv`다. 엑셀 파일은 같은 데이터를 담은 보조 파일로, 엑셀 로드 경로를 확인할 때만 쓴다. 한 Power BI 파일에 CSV와 엑셀을 동시에 불러오지는 않는다.

## 따라하기 1: Power BI Desktop 열기

### 1단계. 설치 확인

1. Windows 시작 메뉴를 연다.
2. `Power BI Desktop`을 검색한다.
3. 프로그램이 있으면 실행한다.
4. 프로그램이 없으면 강사의 안내에 따라 Microsoft Store 또는 Microsoft 공식 다운로드 페이지에서 설치한다.

처음 실행하면 로그인 화면이 뜰 수 있다. 오늘은 로컬 파일로 보고서를 만드는 실습이니, 강사 안내에 맞춰 닫거나 건너뛰면 된다. 조직 PC라면 보안 정책 탓에 설치나 로그인이 막히기도 한다.

### 2단계. 빈 보고서 화면 확인

Power BI Desktop을 열면 다음 영역을 확인한다.

| 화면 영역 | 볼 것 |
|---|---|
| 가운데 흰 영역 | 차트를 놓는 캔버스 |
| 오른쪽 데이터 패널 | 불러온 표와 열 목록 |
| 오른쪽 시각화 패널 | 막대그래프, 꺾은선그래프, 카드 선택 |
| 위쪽 리본 메뉴 | 데이터 가져오기, 저장, 새 시각적 개체 |

## 따라하기 2: CSV 파일 불러오기

### 1단계. 데이터 가져오기

1. 위쪽 메뉴에서 `홈(Home)`을 누른다.
2. `데이터 가져오기(Get data)`를 누른다.
3. `텍스트/CSV(Text/CSV)`를 선택한다.
4. 아래 파일을 선택한다.

```text
/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.csv
```

5. 미리보기 화면이 열리면 열 이름을 확인한다.
6. `로드(Load)`를 누른다.

### 2단계. 데이터 패널 확인

오른쪽 데이터 패널에 `cafe_sales_clean` 표가 생겼는지 확인한다. 표를 펼쳐 아래 열이 보이면 정상이다.

```text
order_id
date
weekday
hour
store
category
item
unit_price
qty
amount
payment
member
```

확인할 것:

- `amount`는 숫자 열이어야 한다.
- `date`는 날짜 열로 인식되는 것이 좋다.
- 오늘은 정제 실습이 아니므로 복잡한 변환은 하지 않는다.

`date`가 날짜로 인식되지 않으면, 강사 화면을 따라 `변환 데이터(Transform data)`에서 형식을 날짜로 바꾼 뒤 `닫기 및 적용(Close & Apply)`을 누른다.

## 따라하기 3: 엑셀 파일 로드 경로 확인하기

이 단계는 경로만 눈에 익히는 용도다. 이미 CSV를 불러온 파일에는 엑셀을 덧붙이지 않는다.

새 Power BI 파일에서 엑셀을 불러올 때는 다음 순서를 따른다.

1. `홈(Home)`을 누른다.
2. `데이터 가져오기(Get data)`를 누른다.
3. `Excel 통합 문서(Excel workbook)`를 선택한다.
4. 아래 파일을 선택한다.

```text
/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.xlsx
```

5. 탐색기(Navigator)에서 `sales` 시트를 선택한다.
6. `로드(Load)`를 누른다.

CSV와 엑셀은 같은 데이터다. 실습 결과를 맞추기 위해, 이후 단계는 CSV로 불러온 `cafe_sales_clean` 표를 기준으로 삼는다.

## 따라하기 4: 카테고리별 매출 막대그래프 만들기

### 1단계. 막대그래프 추가

1. 시각화 패널에서 `묶은 세로 막대형 차트(Clustered column chart)`를 선택한다.
2. 캔버스에 빈 막대그래프가 생기면 오른쪽 필드 영역을 본다.
3. 데이터 패널에서 `category`를 `X축(X-axis)`으로 끌어다 놓는다.
4. 데이터 패널에서 `amount`를 `Y축(Y-axis)` 또는 `값(Values)`으로 끌어다 놓는다.

### 2단계. 집계 방식 확인

`amount`가 `합계(Sum)`로 들어갔는지 확인한다. 평균이나 개수로 잡히면 총매출 비교가 어긋난다.

그래프 제목을 아래처럼 바꾼다.

```text
카테고리별 매출
```

예상되는 카테고리별 매출 합계:

| category | amount 합계 |
|---|---:|
| 커피 | 45,606,000 |
| 음료 | 25,550,000 |
| 디저트 | 19,336,500 |

화면의 표시 단위에 따라 `45.61M`처럼 줄여 보이기도 한다.

## 따라하기 5: 일자별 매출 꺾은선그래프 만들기

### 1단계. 꺾은선그래프 추가

1. 캔버스의 빈 공간을 클릭한다.
2. 시각화 패널에서 `꺾은선형 차트(Line chart)`를 선택한다.
3. `date`를 `X축(X-axis)`으로 끌어다 놓는다.
4. `amount`를 `Y축(Y-axis)` 또는 `값(Values)`으로 끌어다 놓는다.

### 2단계. 날짜 축 확인

Power BI가 `date`를 `날짜 계층 구조(Date hierarchy)`로 넣을 때가 있다. 일자별 흐름을 보려면 X축 필드의 드롭다운을 눌러 `date` 자체를 고른다. 화면이나 버전에 따라 메뉴 이름은 조금씩 다르다.

그래프 제목을 아래처럼 바꾼다.

```text
일자별 매출
```

데이터 기간은 `2025-01-01`부터 `2025-06-30`까지이다.

## 직접 해보기: 총매출 KPI 카드 만들기

목표:

`amount`의 합계를 카드로 표시하고, 총매출이 `90,492,500`원과 일치하게 만든다.

### 수행 조건

1. 캔버스의 빈 공간을 클릭한다.
2. 시각화 패널에서 `카드(Card)` 또는 `카드(새)(Card new)`를 선택한다.
3. 데이터 패널에서 `amount`를 카드의 값 영역에 끌어다 놓는다.
4. 값이 `합계(Sum of amount)`인지 확인한다.
5. 카드 제목을 아래처럼 설정한다.

```text
총매출
```

6. 서식에서 표시 단위(Display units)를 `없음(None)`으로 바꾼다.
7. 소수 자릿수(Decimal places)를 `0`으로 바꾼다.

### 정답 기준

카드 값이 아래 숫자와 일치해야 한다.

```text
90,492,500
```

Power BI가 원 단위를 자동으로 붙여 주지 않으면, 제목을 `총매출(원)`으로 적어 두면 된다.

## 막혔을 때 확인할 것

| 증상 | 확인할 것 |
|---|---|
| `amount` 합계가 이상하다 | `amount`가 숫자 열인지, 집계 방식이 합계인지 확인한다 |
| 카드가 `10,424`처럼 보인다 | `amount`가 아니라 `order_id` 개수가 들어갔을 수 있다 |
| 날짜 축이 한 점만 보인다 | X축에 `date` 자체가 아니라 연도 계층만 들어갔는지 확인한다 |
| 그래프가 비어 있다 | 시각화 개체를 선택한 상태에서 필드를 넣었는지 확인한다 |
| CSV가 깨져 보인다 | 수업 배포본은 UTF-8 정규화 파일이어야 한다. 파일 경로를 다시 확인한다 |
