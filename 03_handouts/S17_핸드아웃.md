작업 루트 기준 경로는 다음과 같다.

```text

```


## 오늘 만들 결과물

한 페이지 대시보드에 아래 요소를 만든다.

- 평균주문금액 카드
- 전월대비 성장률 카드
- 월별 총매출 선그래프
- 카테고리별 매출 막대그래프

- 고객 등급 슬라이서
- 월 슬라이서
- 카테고리에서 상품명으로 내려가는 드릴다운

## 따라하기 1: 데이터 불러오기와 관계 확인

### 1단계. S16 파일 열기 또는 Excel 파일 불러오기

1. S16에서 저장한 Power BI 파일(`.pbix`)이 있으면 그 파일을 열고 2단계로 이동한다.
2. 새로 시작한다면 Power BI Desktop을 연다.
3. `홈 > 데이터 가져오기 > Excel 통합 문서`를 누른다.
4. `shop_orders_clean.xlsx` 파일을 연다.
5. 탐색기 창에서 아래 시트를 모두 선택한다.

```text
orders
customers
products
```

6. 미리보기에서 한글과 날짜가 정상으로 보이면 `로드`를 누른다.

### 2단계. 데이터 형식 확인

왼쪽의 `데이터 보기` 또는 `모델 보기`에서 형식을 확인한다.

| 테이블 | 컬럼 | 형식 |
|---|---|---|
| `orders` | `order_date` | 날짜 |
| `orders` | `qty` | 정수 |
| `orders` | `discount_rate` | 십진수 |
| `products` | `price` | 정수 또는 통화 |
| `customers` | `grade` | 텍스트 |

`order_date`가 날짜가 아니라 텍스트로 보이면 해당 열을 선택하고 `열 도구 > 데이터 형식 > 날짜`로 바꾼다.

### 3단계. 관계 확인

`모델 보기`에서 아래 관계가 있는지 확인한다.

| 1쪽 테이블 | 다쪽 테이블 | 관계 |
|---|---|---|
| `customers[customer_id]` | `orders[customer_id]` | 1 대 다 |
| `products[product_id]` | `orders[product_id]` | 1 대 다 |

관계가 없다면 `customer_id`끼리, `product_id`끼리 드래그해서 연결한다. 필터 방향은 기본값인 단일 방향으로 둔다.

## 따라하기 2: 날짜테이블 만들기

S16에서 `날짜테이블`을 이미 만들어 두었다면 제대로 있는지 확인만 하고 넘어간다. 아직 없다면 아래 순서대로 만든다.

1. 상단 메뉴에서 `모델링 > 새 테이블`을 누른다.
2. 수식 입력창에 아래 DAX를 붙여 넣는다.

```DAX
날짜테이블 =
ADDCOLUMNS(
    CALENDAR(MIN(orders[order_date]), MAX(orders[order_date])),
    "연도", YEAR([Date]),
    "월번호", MONTH([Date]),
    "연월", FORMAT([Date], "yyyy-MM"),
    "연월정렬", YEAR([Date]) * 100 + MONTH([Date])
)
```

3. `날짜테이블[Date]`와 `orders[order_date]`를 연결한다.
4. `날짜테이블`을 선택하고 `테이블 도구 > 날짜 테이블로 표시`를 누른다.
5. 날짜 열은 `Date`를 선택한다.
6. `날짜테이블[연월]` 열을 선택하고 `열 도구 > 열 기준 정렬`에서 `연월정렬`을 선택한다.

확인할 것:

```text
날짜테이블 행 수: 366
주문 기간: 2024-01-01 ~ 2024-12-31
```

## 따라하기 3: 기본 측정값 만들기

측정값은 `orders` 테이블에 만들어도 되고, 측정값만 모으는 별도 테이블에 만들어도 된다. 오늘은 나중에 찾기 쉽도록 `orders` 테이블을 선택한 상태에서 만든다.

### 1단계. 총매출

`모델링 > 새 측정값`을 누르고 아래 수식을 입력한다.

```DAX
총매출 =
SUMX(
    orders,
    RELATED(products[price]) * orders[qty] * (1 - orders[discount_rate])
)
```

형식은 `통화` 또는 `천 단위 구분 기호가 있는 정수`로 바꾼다.

검증값:

```text
총매출: 1,938,489,803
```

### 2단계. 주문건수와 총수량

새 측정값을 두 개 더 만든다.

```DAX
주문건수 =
DISTINCTCOUNT(orders[order_id])
```

```DAX
총수량 =
SUM(orders[qty])
```

검증값:

```text
주문건수: 20,000
총수량: 29,535
```

### 3단계. 평균주문금액

```DAX
평균주문금액 =
DIVIDE([총매출], [주문건수])
```

검증값:

```text
평균주문금액: 약 96,924
```

## 따라하기 4: 매출비중 만들기

카테고리별 매출비중을 만들기 위해 새 측정값을 추가한다.

```DAX
매출비중 =
DIVIDE(
    [총매출],
    CALCULATE([총매출], ALL(products[category]))
)
```

이 측정값은 표나 막대그래프에서 `products[category]`와 나란히 놓고 볼 때라야 뜻이 산다. 형식은 `백분율`로 두고, 소수 자릿수는 1자리나 2자리로 맞춘다.

상위 카테고리 검증값:

| 카테고리 | 매출 | 매출비중 |
|---|---:|---:|
| 디지털 | 673,862,749 | 34.76% |
| 패션 | 281,835,909 | 14.54% |
| 스포츠 | 217,531,145 | 11.22% |

## 따라하기 5: 기본 시각화 만들기

### 1단계. KPI 카드 만들기

1. 보고서 보기로 이동한다.
2. `카드` 시각화를 추가한다.
3. 필드에 `[총매출]`을 넣는다.
4. 같은 방식으로 `[주문건수]`, `[평균주문금액]` 카드를 만든다.

권장 제목:

```text
총매출
주문건수
평균주문금액
```

### 2단계. 월별 매출 선그래프 만들기

1. `꺾은선형 차트`를 추가한다.
2. X축에 `날짜테이블[연월]`을 넣는다.
3. Y축에 `[총매출]`을 넣는다.
4. X축이 2024-01부터 2024-12 순서로 보이는지 확인한다.

검증값:

```text
2024-01 매출: 131,088,708
2024-12 매출: 225,576,457
```

### 3단계. 카테고리별 매출 막대그래프 만들기

1. `묶은 세로 막대형 차트` 또는 `묶은 가로 막대형 차트`를 추가한다.
2. 축에 `products[category]`를 넣는다.
3. 값에 `[총매출]`을 넣는다.
4. 매출 내림차순으로 정렬한다.

디지털 카테고리가 가장 위에 오면 정상이다.

### 4단계. 카테고리별 매출비중 표 만들기

1. `테이블` 시각화를 추가한다.
2. 필드에 `products[category]`, `[총매출]`, `[매출비중]`을 넣는다.
3. `[총매출]` 내림차순으로 정렬한다.

## 따라하기 6: 고객 등급 슬라이서 만들기

1. `슬라이서` 시각화를 추가한다.
2. 필드에 `customers[grade]`를 넣는다.
3. 슬라이서 형식은 목록 또는 타일 중 편한 방식으로 둔다.
4. `VIP`를 눌러 전체 시각화가 VIP 고객 기준으로 바뀌는지 확인한다.

등급별 총매출 검증값:

| 등급 | 총매출 |
|---|---:|
| 실버 | 603,583,519 |
| 골드 | 547,524,985 |
| 브론즈 | 393,948,369 |
| VIP | 393,432,930 |

## 따라하기 7: 드릴다운 만들기

카테고리에서 상품명으로 내려가는 드릴다운을 만든다.

1. 카테고리별 매출 막대그래프를 선택한다.
2. 축 영역에 `products[category]` 아래로 `products[product_name]`을 추가한다.
3. 시각화 상단의 드릴다운 아이콘을 켠다.
4. `디지털` 막대를 클릭한다.
5. 디지털 카테고리 안의 상품별 매출로 내려가는지 확인한다.
6. 위로 올라가기 아이콘을 눌러 다시 카테고리 수준으로 돌아온다.

드릴다운은 한마디로 대분류에서 세부 상품으로 한 단계 내려가는 동작이라고 보면 된다.

## 따라하기 8: 상호작용 편집하기

1. 카테고리별 매출 막대그래프를 선택한다.
2. 상단 메뉴에서 `서식 > 상호 작용 편집`을 누른다.
3. 월별 선그래프 위에 표시되는 아이콘을 확인한다.
4. `필터`, `강조`, `없음`을 각각 눌러 차이를 본다.
5. 오늘 실습에서는 카테고리 막대그래프가 월별 선그래프와 KPI 카드를 필터링하도록 둔다.

해석:

- 필터: 선택한 항목만 남긴다.
- 강조: 선택한 항목의 비중을 진하게 보여준다.
- 없음: 다른 시각화에 영향을 주지 않는다.

## 따라하기 9: 전월대비 DAX 만들기

### 1단계. 전월매출

새 측정값을 만든다.

```DAX
전월매출 =
CALCULATE(
    [총매출],
    SAMEPERIODLASTMONTH('날짜테이블'[Date])
)
```

### 2단계. 전월대비 증감액

```DAX
전월대비증감액 =
[총매출] - [전월매출]
```

### 3단계. 전월대비 성장률

```DAX
전월대비성장률 =
DIVIDE([전월대비증감액], [전월매출])
```

형식은 `백분율`, 소수 자릿수는 2자리로 설정한다.

### 4단계. 월별 표로 검증하기

1. `테이블` 시각화를 추가한다.
2. 필드에 `날짜테이블[연월]`, `[총매출]`, `[전월매출]`, `[전월대비성장률]`을 넣는다.
3. 2024-01은 전월매출이 비어 있어야 한다.
4. 2024-12의 전월대비 성장률이 1.14%인지 확인한다.

주요 검증값:

| 연월 | 총매출 | 전월매출 | 전월대비 성장률 |
|---|---:|---:|---:|
| 2024-01 | 131,088,708 | 비어 있음 | 비어 있음 |
| 2024-11 | 223,024,439 | 158,810,201 | 40.43% |
| 2024-12 | 225,576,457 | 223,024,439 | 1.14% |

### 5단계. 월 슬라이서와 성장률 카드 만들기

1. `슬라이서` 시각화를 추가한다.
2. 필드에 `날짜테이블[연월]`을 넣는다.
3. 슬라이서에서 `2024-12`를 선택한다.
4. `카드` 시각화를 추가하고 `[전월대비성장률]`을 넣는다.

카드에 1.14%가 뜨면 제대로 된 것이다. 여러 달을 한꺼번에 선택하면 전월대비 카드가 무엇을 가리키는지 읽기 애매해지니, 입문 단계에서는 한 달씩만 골라서 읽는다.

## 직접 해보기: 시계열 KPI 대시보드

아래 조건을 만족하는 대시보드 한 페이지를 완성한다.

목표:

1. 월별 총매출 선그래프를 만든다.
2. 총매출, 주문건수, 평균주문금액, 전월대비 성장률 카드를 만든다.
3. 고객 등급 슬라이서를 만든다.
4. 월 슬라이서를 만들고 `2024-12`를 선택했을 때 전월대비 성장률 1.14%가 보이게 한다.
5. 카테고리별 매출 막대그래프에서 상품명 수준으로 드릴다운할 수 있게 한다.
6. 카테고리별 매출비중 표를 만든다.
7. 상호작용 편집으로 카테고리 막대그래프가 월별 선그래프와 KPI 카드를 필터링하도록 둔다.
8. 대시보드 아래쪽 또는 별도 텍스트 상자에 해석 문장 2개를 적는다.

해석 문장 예시:

```text
2024년 전체 매출은 1,938,489,803원이다.
2024년 11월은 전월대비 성장률이 40.43%로 가장 높고, 2024년 6월은 -16.26%로 가장 낮다.
```

제출 전 체크:

- 총매출 카드가 전체 선택 상태에서 1,938,489,803을 보여주는가.
- 월 슬라이서에서 2024-12만 선택하면 전월대비 성장률 카드가 1.14%를 보여주는가.
- 고객 등급 슬라이서를 누르면 모든 시각화가 함께 바뀌는가.
- 카테고리 막대그래프에서 상품명으로 드릴다운할 수 있는가.
- 2024-01의 전월매출이 비어 있는 것을 오류로 오해하지 않았는가.
