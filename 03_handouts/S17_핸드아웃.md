# S17 핸드아웃: 시각화·상호작용·DAX 기초

## 오늘 사용할 파일

S16에서 저장해 둔 Power BI 파일이 있다면 그 파일을 그대로 이어서 씁니다. 이미 관계와 날짜테이블을 만들어 둔 상태라서, 오늘은 대시보드와 DAX 측정값에 바로 집중할 수 있습니다.

처음부터 다시 시작해야 한다면 수업 홈의 데이터 받기 페이지에서 내려받은 쇼핑몰 주문 정리 엑셀 통합 문서를 사용합니다. 이 파일 안에는 주문, 고객, 상품을 담은 세 표가 들어 있습니다. 파일 이름이나 폴더 경로를 외우는 것이 목표가 아니라, Power BI에서 어떤 표를 가져와 어떻게 연결해서 보는지가 오늘의 목표입니다.

> 안내: Power BI Desktop은 Windows 전용입니다. Mac 수강생은 Looker Studio로 개념을 학습합니다. 전월대비 시간지능은 Power BI 전용이며, Looker는 기간비교 필드로 비슷하게 해석해 봅니다.

## 오늘 만들 결과물

한 페이지 대시보드에 아래 요소를 놓아 봅니다. 카드에는 숫자 하나를 크게 보여주고, 그래프에는 흐름과 비교를 보여줍니다. 슬라이서는 보고 싶은 조건을 고르는 필터 버튼이라고 생각하면 됩니다.

- 총매출 카드
- 주문건수 카드
- 평균주문금액 카드
- 전월대비 성장률 카드
- 월별 총매출 선그래프
- 카테고리별 매출 막대그래프
- 카테고리별 매출비중 표
- 고객 등급 슬라이서
- 월 슬라이서
- 카테고리에서 상품명으로 내려가는 드릴다운

## 따라하기 1: 데이터 불러오기와 관계 확인

Power BI는 표를 따로따로 가져온 뒤, 표 사이의 연결선을 보고 계산합니다. 엑셀에서 여러 시트를 VLOOKUP으로 이어서 보는 것과 비슷하지만, Power BI에서는 관계를 한 번 만들어 두면 여러 시각화가 그 관계를 같이 사용합니다.

### 1단계. S16 파일 열기 또는 엑셀 파일 불러오기

1. S16에서 저장한 Power BI 파일이 있으면 그 파일을 열고 2단계로 이동합니다. 이미 만든 관계와 날짜테이블을 이어서 쓰면 같은 작업을 다시 하지 않아도 됩니다.
2. 새로 시작한다면 Power BI Desktop을 엽니다. Power BI에서 표를 가져오고 대시보드를 만들 준비를 하는 단계입니다.
3. `홈 > 데이터 가져오기 > Excel 통합 문서`를 누릅니다. 수업 홈의 데이터 받기 페이지에서 받은 엑셀 파일을 Power BI로 불러오기 위해서입니다.
4. 데이터 받기 페이지에서 내려받은 쇼핑몰 주문 정리 엑셀 파일을 엽니다. 오늘 계산할 주문, 고객, 상품 정보가 이 파일 안에 들어 있습니다.
5. 탐색기 창에서 아래 시트를 모두 선택합니다. 세 표를 함께 가져와야 주문의 고객과 상품을 연결해서 매출을 계산할 수 있습니다.

```text
orders
customers
products
```

6. 미리보기에서 한글과 날짜가 정상으로 보이면 `로드`를 누릅니다. 글자가 깨지지 않고 날짜가 제대로 보인다는 것은 Power BI가 데이터를 읽을 준비가 되었다는 뜻입니다.

### 2단계. 데이터 형식 확인

왼쪽의 `데이터 보기` 또는 `모델 보기`에서 형식을 확인합니다. 숫자는 숫자로, 날짜는 날짜로 잡혀 있어야 계산과 정렬이 제대로 됩니다. 키를 잴 때 줄자가 필요하듯이, Power BI도 값의 종류를 알아야 알맞게 계산할 수 있습니다.

| 테이블 | 컬럼 | 형식 |
|---|---|---|
| `orders` | `order_date` | 날짜 |
| `orders` | `qty` | 정수 |
| `orders` | `discount_rate` | 십진수 |
| `products` | `price` | 정수 또는 통화 |
| `customers` | `grade` | 텍스트 |

`order_date`가 날짜가 아니라 텍스트로 보이면 해당 열을 선택하고 `열 도구 > 데이터 형식 > 날짜`로 바꿉니다. 날짜가 텍스트로 남아 있으면 월별 그래프나 전월대비 계산이 어긋날 수 있습니다.

### 3단계. 관계 확인

`모델 보기`에서 아래 관계가 있는지 확인합니다. 관계는 표와 표를 이어 주는 연결선입니다. 주문표에는 고객 번호와 상품 번호가 있고, 고객표와 상품표에는 그 번호의 자세한 설명이 들어 있다고 보면 됩니다.

| 1쪽 테이블 | 다쪽 테이블 | 관계 |
|---|---|---|
| `customers[customer_id]` | `orders[customer_id]` | 1 대 다 |
| `products[product_id]` | `orders[product_id]` | 1 대 다 |

관계가 없다면 `customer_id`끼리, `product_id`끼리 드래그해서 연결합니다. 필터 방향은 기본값인 단일 방향으로 둡니다. 이렇게 해야 고객 등급이나 상품 카테고리를 눌렀을 때 주문 매출이 자연스럽게 따라 바뀝니다.

## 따라하기 2: 날짜테이블 만들기

날짜테이블은 시간 계산을 위한 기준 달력입니다. 주문일만으로도 월별 매출은 볼 수 있지만, 전월대비처럼 “바로 전 달”을 찾아야 하는 계산에는 별도의 달력이 있으면 훨씬 안정적입니다.

S16에서 `날짜테이블`을 이미 만들어 두었다면 제대로 있는지 확인만 하고 넘어갑니다. 아직 없다면 아래 순서대로 만듭니다.

1. 상단 메뉴에서 `모델링 > 새 테이블`을 누릅니다. 기존 데이터를 고치는 것이 아니라, 시간 계산에 쓸 새 달력 표를 만들기 위해서입니다.
2. 수식 입력창에 아래 DAX를 붙여 넣습니다. DAX는 Power BI 모델 안에 계산식을 저장할 때 쓰는 언어입니다.

```DAX
날짜테이블 =
ADDCOLUMNS(
    CALENDAR(MIN(orders[order_date]), MAX(orders[order_date])),
    "연도", YEAR([Date]),
    "월번호", MONTH([Date]),
    "연월", FORMAT([Date], "yyyy-MM"),
    "연월정렬", YEAR([Date]) * 100 + MONTH([Date])
)
```

3. `날짜테이블[Date]`와 `orders[order_date]`를 연결합니다. 달력의 날짜와 실제 주문일을 이어야 “이 주문은 어느 날짜의 주문인가”를 Power BI가 알 수 있습니다.
4. `날짜테이블`을 선택하고 `테이블 도구 > 날짜 테이블로 표시`를 누릅니다. 이 표가 일반 표가 아니라 시간 계산의 기준 달력이라는 표시를 해 주는 단계입니다.
5. 날짜 열은 `Date`를 선택합니다. Power BI가 어느 열을 실제 날짜로 사용할지 지정해 주어야 합니다.
6. `날짜테이블[연월]` 열을 선택하고 `열 도구 > 열 기준 정렬`에서 `연월정렬`을 선택합니다. 이렇게 해야 2024-01부터 2024-12까지 월이 사전식이 아니라 시간 순서대로 정렬됩니다.

확인할 것:

```text
날짜테이블 행 수: 366
주문 기간: 2024-01-01 ~ 2024-12-31
```

2024년은 윤년이라 날짜가 366개입니다. 이 숫자가 맞으면 주문 기간 전체를 덮는 달력이 제대로 만들어진 것입니다.

## 따라하기 3: 기본 측정값 만들기

측정값은 Power BI 시각화에서 반복해서 꺼내 쓰는 계산식입니다. 엑셀 피벗테이블의 값 영역에 들어가는 “합계”, “개수”, “평균”을 내가 직접 계산식으로 만들어 둔다고 생각하면 됩니다.

측정값은 `orders` 테이블에 만들어도 되고, 측정값만 모으는 별도 테이블에 만들어도 됩니다. 오늘은 나중에 찾기 쉽도록 `orders` 테이블을 선택한 상태에서 만듭니다.

### 1단계. 총매출

총매출은 오늘 대시보드의 기준 숫자입니다. 주문 테이블에 매출 컬럼이 따로 없기 때문에, 상품 단가와 주문 수량과 할인율을 이용해 직접 계산해야 합니다.

`모델링 > 새 측정값`을 누르고 아래 수식을 입력합니다.

```DAX
총매출 =
SUMX(
    orders,
    RELATED(products[price]) * orders[qty] * (1 - orders[discount_rate])
)
```

`SUMX`는 주문을 한 줄씩 보면서 계산한 뒤 전부 더하는 함수입니다. 사전에서 단어 뜻을 찾듯이 `RELATED(products[price])`는 연결된 상품표에서 해당 상품의 단가를 가져옵니다.

형식은 `통화` 또는 `천 단위 구분 기호가 있는 정수`로 바꿉니다. 매출은 사람이 바로 읽어야 하는 숫자라서 1938489803처럼 붙어 보이면 해석하기 어렵습니다.

검증값:

```text
총매출: 1,938,489,803
```

### 2단계. 주문건수와 총수량

주문건수는 주문서가 몇 건인지 세는 값이고, 총수량은 팔린 물건의 개수를 모두 더한 값입니다. 두 값은 서로 다릅니다. 주문 1건에 상품 수량이 2개 들어 있을 수도 있기 때문입니다.

새 측정값을 두 개 더 만듭니다.

```DAX
주문건수 =
DISTINCTCOUNT(orders[order_id])
```

```DAX
총수량 =
SUM(orders[qty])
```

`DISTINCTCOUNT`는 중복을 빼고 고유한 주문 번호를 셉니다. `SUM`은 수량 열의 값을 그대로 더합니다.

검증값:

```text
주문건수: 20,000
총수량: 29,535
```

### 3단계. 평균주문금액

평균주문금액은 주문 1건당 평균 매출입니다. 전체 매출을 주문건수로 나누면 됩니다.

```DAX
평균주문금액 =
DIVIDE([총매출], [주문건수])
```

`DIVIDE`는 나눗셈을 안전하게 해 주는 함수입니다. 분모가 비거나 0이 되는 상황에서 오류가 덜 나도록 도와줍니다.

검증값:

```text
평균주문금액: 약 96,924
```

## 따라하기 4: 매출비중 만들기

매출비중은 “전체 매출 중 이 카테고리가 차지하는 몫”입니다. 전체 파이에서 한 조각이 얼마나 큰지 보는 계산이라고 생각하면 됩니다.

카테고리별 매출비중을 만들기 위해 새 측정값을 추가합니다.

```DAX
매출비중 =
DIVIDE(
    [총매출],
    CALCULATE([총매출], ALL(products[category]))
)
```

이 측정값은 표나 막대그래프에서 `products[category]`와 나란히 놓고 볼 때라야 뜻이 살아납니다. 현재 보고 있는 카테고리의 매출을 전체 카테고리 매출로 나누기 때문입니다. 형식은 `백분율`로 두고, 소수 자릿수는 1자리나 2자리로 맞춥니다.

상위 카테고리 검증값:

| 카테고리 | 매출 | 매출비중 |
|---|---:|---:|
| 디지털 | 673,862,749 | 34.76% |
| 패션 | 281,835,909 | 14.54% |
| 스포츠 | 217,531,145 | 11.22% |

## 따라하기 5: 기본 시각화 만들기

이제 계산한 측정값을 화면에 꺼내 놓아 봅니다. 같은 `[총매출]` 측정값이라도 카드에 넣으면 큰 숫자 하나가 되고, 선그래프에 넣으면 월별 흐름이 됩니다.

### 1단계. KPI 카드 만들기

카드는 숫자 하나를 크게 보여주는 시각화입니다. 대표 지표를 한눈에 확인할 때 씁니다.

1. 보고서 보기로 이동합니다. 실제 대시보드 화면을 구성하기 위해서입니다.
2. `카드` 시각화를 추가합니다. 숫자 하나를 크게 보여줄 자리를 만드는 단계입니다.
3. 필드에 `[총매출]`을 넣습니다. 대시보드에서 가장 먼저 확인할 전체 매출을 보여주기 위해서입니다.
4. 같은 방식으로 `[주문건수]`, `[평균주문금액]` 카드를 만듭니다. 매출 규모, 주문 수, 주문 1건당 금액을 함께 보면 숫자를 더 정확히 해석할 수 있습니다.

권장 제목:

```text
총매출
주문건수
평균주문금액
```

### 2단계. 월별 매출 선그래프 만들기

선그래프는 시간에 따른 흐름을 볼 때 좋습니다. 매출이 어느 달에 올라가고 내려가는지 눈으로 확인할 수 있습니다.

1. `꺾은선형 차트`를 추가합니다. 월별 매출 흐름을 선으로 이어서 보기 위해서입니다.
2. X축에 `날짜테이블[연월]`을 넣습니다. 가로축에 시간을 놓아야 월별 변화를 읽을 수 있습니다.
3. Y축에 `[총매출]`을 넣습니다. 세로축에 매출 금액을 놓아야 각 달의 매출 크기를 비교할 수 있습니다.
4. X축이 2024-01부터 2024-12 순서로 보이는지 확인합니다. 월 순서가 틀어지면 그래프가 시간 흐름을 제대로 보여주지 못합니다.

검증값:

```text
2024-01 매출: 131,088,708
2024-12 매출: 225,576,457
```

### 3단계. 카테고리별 매출 막대그래프 만들기

막대그래프는 항목끼리 크기를 비교할 때 좋습니다. 어떤 카테고리가 많이 팔렸는지 바로 볼 수 있습니다.

1. `묶은 세로 막대형 차트` 또는 `묶은 가로 막대형 차트`를 추가합니다. 카테고리별 매출 크기를 막대 길이로 비교하기 위해서입니다.
2. 축에 `products[category]`를 넣습니다. 비교할 기준을 상품 카테고리로 잡는 단계입니다.
3. 값에 `[총매출]`을 넣습니다. 각 카테고리의 매출 합계를 막대로 보여주기 위해서입니다.
4. 매출 내림차순으로 정렬합니다. 가장 큰 항목이 먼저 보이면 어디가 큰지 빠르게 읽을 수 있습니다.

디지털 카테고리가 가장 위에 오면 정상입니다.

### 4단계. 카테고리별 매출비중 표 만들기

표는 정확한 숫자와 비율을 함께 확인할 때 좋습니다. 그래프가 모양으로 비교한다면, 표는 숫자를 직접 확인하는 자리입니다.

1. `테이블` 시각화를 추가합니다. 카테고리명, 매출, 비중을 줄 단위로 확인하기 위해서입니다.
2. 필드에 `products[category]`, `[총매출]`, `[매출비중]`을 넣습니다. 카테고리별로 금액과 비율을 나란히 보기 위해서입니다.
3. `[총매출]` 내림차순으로 정렬합니다. 매출이 큰 카테고리부터 읽으면 대시보드 해석이 쉬워집니다.

## 따라하기 6: 고객 등급 슬라이서 만들기

슬라이서는 화면 위에 놓는 필터 버튼입니다. `VIP`를 누르면 VIP 고객의 매출만 보고, `골드`를 누르면 골드 고객의 매출만 보는 식입니다.

1. `슬라이서` 시각화를 추가합니다. 사용자가 보고 싶은 조건을 직접 고를 수 있게 하기 위해서입니다.
2. 필드에 `customers[grade]`를 넣습니다. 고객 등급별로 대시보드를 나누어 보기 위해서입니다.
3. 슬라이서 형식은 목록 또는 타일 중 편한 방식으로 둡니다. 수업에서는 기능을 익히는 것이 목적이므로 읽기 쉬운 형태를 고르면 됩니다.
4. `VIP`를 눌러 전체 시각화가 VIP 고객 기준으로 바뀌는지 확인합니다. 슬라이서가 카드, 선그래프, 막대그래프에 함께 적용되는지 확인하는 단계입니다.

등급별 총매출 검증값:

| 등급 | 총매출 |
|---|---:|
| 실버 | 603,583,519 |
| 골드 | 547,524,985 |
| 브론즈 | 393,948,369 |
| VIP | 393,432,930 |

## 따라하기 7: 드릴다운 만들기

드릴다운은 큰 묶음에서 작은 항목으로 한 단계 내려가는 기능입니다. 카테고리에서 끝내지 않고, 그 안의 상품명까지 들어가 보는 동작입니다.

카테고리에서 상품명으로 내려가는 드릴다운을 만듭니다.

1. 카테고리별 매출 막대그래프를 선택합니다. 어느 시각화에 드릴다운 기능을 넣을지 정하기 위해서입니다.
2. 축 영역에 `products[category]` 아래로 `products[product_name]`을 추가합니다. 카테고리 다음 단계가 상품명이라는 계층을 만들기 위해서입니다.
3. 시각화 상단의 드릴다운 아이콘을 켭니다. 막대를 눌렀을 때 세부 단계로 내려갈 수 있게 하는 설정입니다.
4. `디지털` 막대를 클릭합니다. 가장 큰 카테고리 안에서 어떤 상품이 매출을 만들었는지 보기 위해서입니다.
5. 디지털 카테고리 안의 상품별 매출로 내려가는지 확인합니다. 큰 묶음에서 세부 상품으로 화면이 바뀌면 드릴다운이 제대로 작동한 것입니다.
6. 위로 올라가기 아이콘을 눌러 다시 카테고리 수준으로 돌아옵니다. 세부 화면을 본 뒤 다시 전체 비교 화면으로 돌아오는 연습입니다.

드릴다운은 한마디로 대분류에서 세부 상품으로 한 단계 내려가는 동작이라고 보면 됩니다.

## 따라하기 8: 상호작용 편집하기

상호작용은 차트끼리 서로 영향을 주는 방식입니다. 한 차트를 클릭했을 때 다른 차트가 같이 바뀌는 이유가 바로 상호작용입니다.

1. 카테고리별 매출 막대그래프를 선택합니다. 이 막대그래프를 눌렀을 때 다른 시각화가 어떻게 반응할지 정하기 위해서입니다.
2. 상단 메뉴에서 `서식 > 상호 작용 편집`을 누릅니다. 시각화끼리 영향을 주는 방식을 직접 고치기 위해서입니다.
3. 월별 선그래프 위에 표시되는 아이콘을 확인합니다. 현재 막대그래프가 선그래프에 어떤 방식으로 작용하는지 눈으로 확인하는 단계입니다.
4. `필터`, `강조`, `없음`을 각각 눌러 차이를 봅니다. 같은 선택이라도 결과가 어떻게 달라지는지 직접 봐야 헷갈리지 않습니다.
5. 오늘 실습에서는 카테고리 막대그래프가 월별 선그래프와 KPI 카드를 필터링하도록 둡니다. 카테고리를 누르면 그 카테고리만 남겨서 월별 흐름과 KPI를 다시 계산해 보기 위해서입니다.

해석:

- 필터: 선택한 항목만 남깁니다.
- 강조: 선택한 항목의 비중을 진하게 보여줍니다.
- 없음: 다른 시각화에 영향을 주지 않습니다.

## 따라하기 9: 전월대비 DAX 만들기

전월대비는 현재 보고 있는 달의 매출을 바로 전 달 매출과 비교하는 계산입니다. 12월을 보고 있다면 11월과 비교하고, 11월을 보고 있다면 10월과 비교합니다. 그래서 날짜테이블과 주문일의 관계가 꼭 필요합니다.

### 1단계. 전월매출

전월매출은 현재 달보다 한 달 전의 매출입니다. 현재 달을 기준으로 “한 칸 뒤 달”을 찾아오는 계산이라고 생각하면 됩니다.

새 측정값을 만듭니다.

```DAX
전월매출 =
CALCULATE(
    [총매출],
    SAMEPERIODLASTMONTH('날짜테이블'[Date])
)
```

### 2단계. 전월대비 증감액

전월대비 증감액은 이번 달 매출에서 전월매출을 뺀 값입니다. 양수이면 늘어난 것이고, 음수이면 줄어든 것입니다.

```DAX
전월대비증감액 =
[총매출] - [전월매출]
```

### 3단계. 전월대비 성장률

성장률은 증감액을 전월매출로 나눈 값입니다. 금액 차이만 보면 규모가 큰 달이 더 커 보일 수 있으니, 비율로도 함께 봅니다.

```DAX
전월대비성장률 =
DIVIDE([전월대비증감액], [전월매출])
```

형식은 `백분율`, 소수 자릿수는 2자리로 설정합니다. 성장률은 0.0114보다 1.14%처럼 보이는 편이 사람이 읽기 쉽습니다.

### 4단계. 월별 표로 검증하기

표로 먼저 검증하면 계산이 맞는지 차분히 확인할 수 있습니다. 그래프나 카드보다 행별 숫자를 비교하기 쉽기 때문입니다.

1. `테이블` 시각화를 추가합니다. 월별 계산 결과를 줄 단위로 확인하기 위해서입니다.
2. 필드에 `날짜테이블[연월]`, `[총매출]`, `[전월매출]`, `[전월대비성장률]`을 넣습니다. 현재 달 매출, 전월매출, 성장률을 한 줄에서 비교하기 위해서입니다.
3. 2024-01은 전월매출이 비어 있어야 합니다. 데이터가 2024-01부터 시작하므로 그보다 앞선 전월이 없기 때문입니다.
4. 2024-12의 전월대비 성장률이 1.14%인지 확인합니다. 마지막 달의 값이 맞으면 시간 계산이 제대로 연결되었는지 확인하는 데 도움이 됩니다.

주요 검증값:

| 연월 | 총매출 | 전월매출 | 전월대비 성장률 |
|---|---:|---:|---:|
| 2024-01 | 131,088,708 | 비어 있음 | 비어 있음 |
| 2024-11 | 223,024,439 | 158,810,201 | 40.43% |
| 2024-12 | 225,576,457 | 223,024,439 | 1.14% |

### 5단계. 월 슬라이서와 성장률 카드 만들기

성장률 카드는 어느 달을 기준으로 보는지가 분명해야 합니다. 그래서 월 슬라이서로 한 달을 고른 뒤 카드를 읽어 봅니다.

1. `슬라이서` 시각화를 추가합니다. 보고 싶은 달을 직접 고를 수 있게 하기 위해서입니다.
2. 필드에 `날짜테이블[연월]`을 넣습니다. 월 단위로 대시보드를 필터링하기 위해서입니다.
3. 슬라이서에서 `2024-12`를 선택합니다. 2024-12를 기준으로 전월대비 성장률을 확인하기 위해서입니다.
4. `카드` 시각화를 추가하고 `[전월대비성장률]`을 넣습니다. 선택한 달의 성장률을 큰 숫자 하나로 확인하기 위해서입니다.

카드에 1.14%가 뜨면 제대로 된 것입니다. 여러 달을 한꺼번에 선택하면 전월대비 카드가 무엇을 가리키는지 읽기 애매해지니, 입문 단계에서는 한 달씩만 골라서 읽습니다.

## 직접 해보기: 시계열 KPI 대시보드

아래 조건을 만족하는 대시보드 한 페이지를 완성합니다. 지금까지 만든 계산식과 시각화를 한 화면에 놓고, 슬라이서와 드릴다운이 제대로 움직이는지 확인해 보세요.

목표:

1. 월별 총매출 선그래프를 만듭니다. 시간에 따라 매출이 어떻게 변했는지 보기 위해서입니다.
2. 총매출, 주문건수, 평균주문금액, 전월대비 성장률 카드를 만듭니다. 대시보드에서 가장 먼저 읽을 숫자를 한눈에 보기 위해서입니다.
3. 고객 등급 슬라이서를 만듭니다. 고객 등급별로 매출 흐름과 카테고리 구성이 어떻게 달라지는지 보기 위해서입니다.
4. 월 슬라이서를 만들고 `2024-12`를 선택했을 때 전월대비 성장률 1.14%가 보이게 합니다. 성장률 카드가 어떤 달을 기준으로 계산되는지 분명히 하기 위해서입니다.
5. 카테고리별 매출 막대그래프에서 상품명 수준으로 드릴다운할 수 있게 합니다. 큰 카테고리 안에서 어떤 상품이 매출을 만들었는지 내려가 보기 위해서입니다.
6. 카테고리별 매출비중 표를 만듭니다. 금액뿐 아니라 전체에서 차지하는 비율까지 함께 보기 위해서입니다.
7. 상호작용 편집으로 카테고리 막대그래프가 월별 선그래프와 KPI 카드를 필터링하도록 둡니다. 카테고리를 눌렀을 때 관련 숫자와 흐름이 같이 바뀌게 하기 위해서입니다.
8. 대시보드 아래쪽 또는 별도 텍스트 상자에 해석 문장 2개를 적습니다. 숫자를 만드는 데서 끝내지 않고, 그 숫자가 무슨 뜻인지 말로 설명하는 연습입니다.

해석 문장 예시:

```text
2024년 전체 매출은 1,938,489,803원이다.
2024년 11월은 전월대비 성장률이 40.43%로 가장 높고, 2024년 6월은 -16.26%로 가장 낮다.
```

제출 전 체크:

- 총매출 카드가 전체 선택 상태에서 1,938,489,803을 보여주는지 확인해 보세요. 기준 숫자가 맞아야 다른 시각화도 믿고 볼 수 있습니다.
- 월 슬라이서에서 2024-12만 선택하면 전월대비 성장률 카드가 1.14%를 보여주는지 확인해 보세요. 월 선택과 시간 계산이 제대로 연결되었는지 보는 기준입니다.
- 고객 등급 슬라이서를 누르면 모든 시각화가 함께 바뀌는지 확인해 보세요. 슬라이서가 대시보드 전체 필터로 작동하는지 확인하는 단계입니다.
- 카테고리 막대그래프에서 상품명으로 드릴다운할 수 있는지 확인해 보세요. 대분류에서 세부 상품으로 내려가는 기능이 살아 있어야 합니다.
- 2024-01의 전월매출이 비어 있는 것을 오류로 오해하지 않았는지 확인해 보세요. 2024-01 앞에는 비교할 전월 데이터가 없기 때문에 비어 있는 것이 맞습니다.
