# S01 핸드아웃. 데이터 분석이란, AI 시대의 작업 흐름 + 환경 세팅

## 오늘 할 일

데이터 분석은 표에 들어 있는 값으로 업무 질문에 답하는 일입니다. 오늘은 어려운 분석 기법을 배우기보다, 분석을 시작하기 전에 꼭 잡아야 하는 감각을 먼저 익혀 봅니다.

오늘 확인할 내용은 세 가지입니다.

1. 이 표로 어떤 질문에 답할 수 있는지 판단합니다.
2. 엑셀, AI, 파이썬, BI가 각각 언제 필요한지 큰 지도를 잡습니다.
3. 앞으로 수업을 따라가기 위한 환경을 확인합니다.

분석은 계산부터 시작하지 않습니다. 먼저 “이 표에 내가 궁금한 답을 찾을 재료가 있나요?”를 확인합니다. 키를 재려면 줄자가 있어야 하듯이, 데이터 분석도 답에 필요한 열이 표 안에 있어야 합니다.

## 준비물

- 노트북
- Google 계정 (파이썬 파트인 S10부터 Google Colab에 사용합니다. 오늘은 준비만 해 두면 됩니다.)
- 수업에서 지정한 AI 도구 계정
- 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 파일

CSV는 엑셀로도 열리는 표 파일입니다. 열었을 때 엑셀이 더 편한 분은 같은 데이터의 엑셀 파일을 받아서 사용해도 됩니다.

확인할 점은 세 가지입니다.

- Power BI Desktop은 Windows에서만 설치해 사용합니다. Mac 사용자는 Part 5에서 Looker Studio 경로를 따릅니다.
- 엑셀은 Microsoft 365를 권장합니다. 뒤 차시에서 FILTER, SORT 등 동적 배열 함수를 사용합니다.
- AI 파일 업로드와 데이터 분석 기능은 유료 구독이 필요한 경우가 있습니다. 수업에서는 표준 도구를 기준으로 설명하되, 가능한 무료 대안도 함께 안내합니다.

## 따라하기 1. 카페 매출 파일 내려받기

1. 수업 홈의 데이터 받기 페이지를 엽니다.  
   실습에 쓸 표를 먼저 준비해야, 뒤에서 같은 화면을 보며 따라갈 수 있습니다.

2. 카페 매출 파일을 내려받습니다.  
   오늘은 이 파일 하나로 “어떤 질문에 답할 수 있는지”를 연습합니다.

3. 내려받은 파일을 찾기 쉬운 위치에 둡니다.  
   파일을 바로 열 수 있어야 실습 흐름이 끊기지 않습니다.

4. 파일 이름을 보고 카페 매출 데이터인지 확인합니다.  
   엉뚱한 파일을 열면 열 이름과 값이 달라져서 설명을 따라가기 어렵습니다.

파일이 보이면 오늘 실습 데이터 준비는 완료입니다.

## 따라하기 2. 엑셀로 표 관찰하기

1. 카페 매출 파일을 엑셀로 엽니다.  
   엑셀로 먼저 보는 이유는 표의 모양을 눈으로 확인하기 쉽기 때문입니다.

2. 첫 행의 열 이름을 확인합니다.  
   열 이름은 이 표가 어떤 정보를 들고 있는지 알려 주는 표지판입니다.

```text
order_id, date, weekday, hour, store, category, item, unit_price, qty, amount, payment, member
```

3. 각 열이 무엇을 뜻하는지 옆 사람에게 설명해 봅니다.  
   사전에서 단어 뜻을 찾듯이, 열 이름의 뜻을 알아야 질문을 만들 수 있습니다.

| 열 이름 | 뜻 |
|---|---|
| `order_id` | 주문 번호 |
| `date` | 주문일 |
| `weekday` | 요일 |
| `hour` | 주문 시간대 |
| `store` | 지점 |
| `category` | 상품 카테고리 |
| `item` | 상품명 |
| `unit_price` | 단가 |
| `qty` | 수량 |
| `amount` | 주문 금액 |
| `payment` | 결제수단 |
| `member` | 멤버십 여부 |

4. 아래 질문에 답해 봅니다. 정확한 계산보다 “어느 열을 봐야 하는지”를 생각하는 것이 중요합니다.  
   분석 질문은 보통 열 이름으로 바꿀 수 있어야 계산으로 이어집니다.

| 질문 | 봐야 할 열 |
|---|---|
| 어느 지점의 매출이 더 높은가? | `store`, `amount` |
| 어떤 카테고리가 많이 팔리는가? | `category`, `qty` 또는 `amount` |
| 어느 시간대에 매출이 높은가? | `hour`, `amount` |
| 멤버십 고객과 비멤버십 고객의 매출은 다른가? | `member`, `amount` |

여기서 뭐가 보이시나요? “지점”을 비교하려면 `store`가 필요하고, “매출”을 보려면 `amount`가 필요합니다. 질문 속 말과 표의 열 이름을 연결하는 습관이 분석의 출발점입니다.

## 따라하기 3. 데이터로 답할 수 있는 질문 구분하기

다음 질문을 보고, 현재 카페 주문 표만으로 답할 수 있는지 판단해 보세요.  
이 단계가 필요한 이유는, 데이터에 없는 내용을 억지로 계산하려고 하면 그럴듯하지만 틀린 답이 나오기 때문입니다.

| 질문 | 판단 | 이유 |
|---|---|---|
| 2025년 1월부터 6월까지 지점별 총매출은? | 가능 | `date`, `store`, `amount`가 있다. |
| 상품 카테고리별 주문 수량은? | 가능 | `category`, `qty`가 있다. |
| 손님이 카페 분위기에 만족했는가? | 어려움 | 만족도나 분위기 평가 열이 없다. |
| 다음 달 매출은 얼마인가? | 제한적 | 과거 데이터는 있지만 미래 값은 없다. 예측 모델과 추가 가정이 필요하다. |
| 왜 특정 메뉴를 골랐는가? | 어려움 | 구매 이유를 묻는 열이나 설문이 없다. |

분석을 시작할 때는 “궁금한가?”보다 “이 표의 열로 답할 수 있는가?”를 먼저 확인합니다.

예를 들어 손님의 만족도를 알고 싶다면 만족도 점수나 설문 응답 열이 있어야 합니다. 구매 이유를 알고 싶다면 구매 이유를 물어본 기록이 있어야 합니다. 표에 없는 값은 사전에서 없는 단어를 찾는 것과 비슷해서, 분석 도구를 바꾼다고 바로 생기지 않습니다.

## 직접 해보기. 오늘의 미션

### 미션 1. 카페 매출 표로 분석 질문 3개 만들기

카페 매출 파일로 답할 수 있는 질문을 3개 작성합니다. 질문에는 가능한 한 다음 네 가지가 들어가야 합니다.

- 대상
- 기준
- 비교하거나 집계할 값
- 기간

대상은 무엇을 볼 것인지입니다. 기준은 무엇으로 나누어 볼 것인지입니다. 비교하거나 집계할 값은 합계, 개수, 평균처럼 숫자로 확인할 값입니다. 기간은 언제부터 언제까지 볼 것인지를 뜻합니다.

작성 예시는 이렇게 볼 수 있습니다.

```text
2025년 1월부터 6월까지 지점별 총매출은 어떻게 다른가?
```

이 문장에는 카페 주문이라는 대상, 지점별이라는 기준, 총매출이라는 집계 값, 2025년 1월부터 6월까지라는 기간이 들어 있습니다.

아래 표를 채워 보세요.

| 번호 | 분석 질문 | 답을 위해 필요한 열 |
|---:|---|---|
| 1 |  |  |
| 2 |  |  |
| 3 |  |  |

### 미션 2. 내 업무 표 1개를 질문으로 바꾸기

내가 평소 자주 보는 표를 하나 떠올립니다. 매출표, 고객문의표, 출석표, 재고표, 캠페인 성과표 같은 것이면 됩니다.

그 표로 답하고 싶은 질문을 1개 작성해 보세요. 이때도 “무엇을 기준으로 나누어 볼지”, “어떤 숫자를 비교할지”를 같이 생각하면 질문이 훨씬 분명해집니다.

| 내 업무 표 | 표에 있을 것 같은 열 | 분석 질문 |
|---|---|---|
|  |  |  |

예를 들어 고객문의표라면 문의일, 문의 유형, 처리 상태, 담당자 같은 열이 있을 수 있습니다. 그러면 “지난달 문의 유형별 처리 완료 건수는 어떻게 다른가?”처럼 질문을 만들 수 있습니다.

## 제출 기준

아래 세 가지가 들어가면 제출 완료입니다.

- 카페 매출 표로 만든 분석 질문 3개
- 각 질문에 필요한 열 이름
- 내 업무 표 1개와 그 표로 답하고 싶은 질문 1개

제출할 때는 질문만 쓰고 끝내지 말고, 그 질문에 답하려면 어떤 열을 봐야 하는지도 함께 적어 보세요. 질문과 열 이름이 연결되어야 실제 분석으로 넘어갈 수 있습니다.

## 막힐 때 확인할 것

- 파일이 안 보이면 수업 홈의 데이터 받기 페이지에서 카페 매출 파일을 다시 내려받습니다.
- 질문이 너무 막연하면 “어떤 열을 기준으로, 어떤 숫자를 비교할지”부터 적습니다.
- 데이터에 없는 내용을 묻고 있다면, 필요한 열을 새로 적어 봅니다.

막히는 것은 자연스럽습니다. 분석 질문을 만드는 일은 머릿속 궁금증을 표의 열 이름으로 바꾸는 연습입니다. 처음에는 느리게 하셔도 됩니다. 지금은 계산을 빨리 하는 것보다, 이 표로 답할 수 있는 질문과 답하기 어려운 질문을 구분하는 감각이 더 중요합니다.
