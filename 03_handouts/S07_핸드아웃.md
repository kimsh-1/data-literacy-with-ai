# S07. 엑셀을 AI에 올려 자연어로 분석

## 사용할 파일

AI 분석 도구에는 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 엑셀 파일을 올립니다. 초보자는 엑셀 파일(.xlsx)로 진행하면 됩니다. 파일을 올린다는 말은, AI에게 표 데이터를 직접 읽을 수 있게 건네주는 일입니다.

- 시트 이름: `sales`
- 데이터 규모: 10,439행 × 12열
- 주요 컬럼: `order_id`, `date`, `store`, `category`, `item`, `unit_price`, `qty`, `amount`, `payment`, `member`

여기서 행은 주문 기록 한 줄, 열은 주문번호나 날짜처럼 데이터의 항목이라고 보면 됩니다. 키를 잴 때 키, 몸무게, 나이를 따로 적듯이, 데이터도 열마다 역할이 나뉩니다.

업로드 기능이 없는 계정이라면 수업 화면에 제시된 샘플 답안과 비교하면 됩니다. 실제 회사 파일, 고객 정보, 개인정보가 담긴 파일은 허가 없이 외부 AI에 올리지 않습니다. 이 원칙은 실습보다 더 중요합니다.

## 오늘의 목표

AI에게 엑셀 파일을 업로드하고 세 가지 결과를 받아 봅니다. 그냥 “분석해 줘”라고 맡기는 것이 아니라, AI가 어떤 기준으로 계산했는지 확인하는 연습입니다.

1. 데이터 구조와 품질 점검
2. 월별 매출 추세 표
3. 월별 매출 선그래프와 이상치 설명

마지막에는 두 AI 답안을 비교합니다. 더 말이 그럴듯한 답이 아니라, 행 수와 전처리 기준, 숫자를 확인할 수 있는 답을 더 믿어야 합니다.

## 따라하기 1. 파일 업로드 후 데이터 구조 확인하기

분석을 시작하기 전에 데이터가 어떤 모양인지 먼저 봅니다. 요리를 하기 전에 재료가 몇 개 있고 상한 것은 없는지 확인하는 것과 같습니다.

### 1단계. 파일 업로드

이 단계는 AI가 실제 데이터를 읽을 수 있게 준비하는 과정입니다. 파일을 올리지 않으면 AI는 표 안의 숫자를 직접 확인할 수 없습니다.

1. 파일 업로드와 데이터 분석이 가능한 AI 채팅창을 엽니다.
2. 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 엑셀 파일을 업로드합니다.
3. 업로드가 끝나면 아래 프롬프트를 그대로 입력합니다.

```text
첨부한 cafe_sales_dirty.xlsx 파일의 sales 시트를 먼저 점검해 주세요.

아직 결론을 내리지 말고 다음 항목만 표로 알려 주세요.
1. 행 수와 열 수
2. 컬럼 목록
3. 각 컬럼의 의미를 한 줄씩 추정한 설명
4. 결측값이 있는 컬럼과 결측 개수
5. 중복 order_id 개수
6. 월별 매출 분석 전에 주의해야 할 데이터 품질 이슈
```

### 2단계. 확인할 결과

이 단계는 AI가 파일을 제대로 읽었는지 확인하는 과정입니다. 계산기로 답을 구했더라도, 먼저 숫자를 제대로 눌렀는지 확인해야 하는 것과 같습니다.

AI 답에서 최소한 아래 내용이 나와야 합니다.

| 확인 항목 | 기대값 |
|---|---:|
| 행 수 | 10,439 |
| 열 수 | 12 |
| `payment` 결측 | 313 |
| `member` 결측 | 209 |
| 중복 `order_id` 추가 행 | 15 |

결측값은 비어 있는 값입니다. 예를 들어 결제수단이 적혀 있어야 하는 칸이 비어 있으면 `payment` 결측입니다. 중복 `order_id`는 같은 주문번호가 여러 번 나온 경우입니다. 주문번호는 주문마다 붙는 이름표이므로, 같은 이름표가 반복되면 먼저 확인해야 합니다.

AI가 행 수를 말하지 않거나, 바로 매출 결론부터 말하면 다시 질문합니다.

```text
결론보다 검증이 먼저입니다. 행 수, 열 수, 결측값, 중복 order_id, 날짜 형식 문제를 먼저 확인한 뒤 다시 답해 주세요.
```

## 따라하기 2. 월별 매출 추세와 이상치 요청하기

이제 월별 매출을 계산해 봅니다. 여기서 중요한 점은 “월별 매출”이라는 결과만 요구하지 않고, 계산 전에 어떤 행을 정리하고 어떤 행을 제외할지 함께 알려 주는 것입니다.

### 1단계. 월별 매출 분석 기준 알려 주기

이 단계는 AI에게 업무 지시서를 주는 과정입니다. 같은 데이터라도 중복 주문을 남기느냐 지우느냐에 따라 매출액이 달라질 수 있습니다.

아래 프롬프트를 그대로 입력합니다. 이번 차시에서 가장 중요한 실습입니다.

```text
이제 월별 매출 추세를 분석해 주세요.

반드시 아래 기준을 적용해 주세요.
- date는 YYYY-MM-DD, YYYY/MM/DD, MM-DD-YYYY 형식이 섞여 있을 수 있으니 모두 날짜로 해석해 주세요.
- order_id는 주문 고유번호이므로 중복 order_id는 한 건만 남겨 주세요.
- unit_price에 쉼표나 '원' 문자가 붙어 있으면 숫자로 바꿔 주세요.
- amount가 0 이하인 행, qty가 20보다 큰 행, unit_price * qty와 amount가 맞지 않는 행은 이상치로 따로 보여 주고 월별 매출 집계에서는 제외해 주세요.
- payment와 member 결측은 매출 집계에서 제외하지 마세요.

결과는 다음 순서로 보여 주세요.
1. 적용한 전처리 기준 요약
2. 월별 주문건수, 판매수량, 매출액, 전월대비 증감액, 전월대비 증감률 표
3. 월별 매출 선그래프
4. 이상치로 제외한 행 목록
5. 분석 결론 3문장
```

전처리는 분석 전에 데이터를 계산하기 좋은 상태로 손보는 일입니다. 날짜를 날짜로 읽게 하고, 단가에 붙은 `원` 같은 글자를 숫자에서 떼어 내고, 말이 안 되는 행을 따로 빼는 과정이라고 보면 됩니다.

### 2단계. AI 답에서 확인할 핵심 수치

이 단계는 AI가 기준대로 계산했는지 숫자로 확인하는 과정입니다. 말로는 맞아 보여도, 표의 숫자가 다르면 다시 물어봐야 합니다.

전처리 기준을 위와 같이 적용했다면 흐름은 아래와 같아야 합니다.

| 월 | 주문건수 | 매출액 | 전월대비 증감률 |
|---|---:|---:|---:|
| 2025-01 | 1,685 | 14,421,000 |  |
| 2025-02 | 1,523 | 13,151,500 | -8.8% |
| 2025-03 | 1,716 | 14,735,500 | 12.0% |
| 2025-04 | 1,760 | 15,257,500 | 3.5% |
| 2025-05 | 1,800 | 16,094,000 | 5.5% |
| 2025-06 | 1,937 | 16,816,500 | 4.5% |

이 표에서는 2025-02의 전월대비 증감률이 -8.8%입니다. 1월보다 2월 매출이 줄었다는 뜻입니다. 반대로 2025-03부터 2025-06까지는 전월대비 증감률이 모두 양수입니다. 앞달보다 매출이 늘었다고 읽으면 됩니다.

해석의 중심은 아래 세 문장입니다.

- 2025년 2월에 매출이 1월보다 줄었다.
- 2025년 3월부터 6월까지는 매출이 매월 증가했다.
- 2025년 6월 매출이 16,816,500원으로 가장 크다.

## 따라하기 3. 차트를 다시 요청하기

표만으로도 추세를 읽을 수 있지만, 선그래프를 만들면 흐름이 눈에 더 잘 들어옵니다. 다만 차트는 예쁘게 보이는 것보다 축과 숫자가 맞는 것이 먼저입니다.

### 1단계. 축과 기준을 분명히 적어 차트 요청하기

이 단계는 차트가 어떤 값을 그려야 하는지 고정하는 과정입니다. x축은 시간, y축은 매출액처럼 역할을 정해 주어야 그래프를 믿고 볼 수 있습니다.

AI가 차트를 만들었더라도 축과 기준이 불분명하면 다시 요청합니다.

```text
월별 매출액을 선그래프로 다시 만들어 주세요.

차트 조건:
- x축은 2025-01부터 2025-06까지 월입니다.
- y축은 매출액 합계입니다.
- 각 점에 매출액 값을 표시해 주세요.
- 그래프 아래에 "2월 하락 후 3월부터 6월까지 증가"라는 해석을 붙여 주세요.
```

차트가 표와 다른 값을 가리키면 표의 전처리 기준부터 다시 확인합니다. 표와 차트, 결론에 나온 숫자는 서로 어긋나면 안 됩니다. 같은 체중을 재는데 체중계마다 숫자가 다르면 먼저 저울을 확인하듯이, AI 답도 표와 차트의 숫자를 맞춰 봐야 합니다.

## 따라하기 4. 이상치 설명을 요청하기

이상치는 단순히 “매출이 큰 달”이라는 뜻만은 아닙니다. 수량이 999처럼 현실적으로 이상하거나, 금액이 음수이거나, 단가와 수량을 곱한 값이 주문금액과 맞지 않는 행도 이상치입니다.

### 1단계. 제외한 행을 직접 보여 달라고 요청하기

이 단계는 AI가 어떤 데이터를 빼고 계산했는지 확인하는 과정입니다. 계산에서 빠진 행을 보지 않으면, 매출 표가 왜 그렇게 나왔는지 알기 어렵습니다.

AI가 이상치를 "매출이 큰 달"만으로 설명하면 부족합니다. 아래처럼 행 단위 이상치를 다시 묻습니다.

```text
이번 분석에서 제외한 이상치 행을 order_id, date, item, unit_price, qty, amount, 제외 이유 순서로 표로 보여 주세요.

또한 일별 매출 합계를 기준으로 통계적으로 유난히 높은 날이나 낮은 날이 있었는지도 확인해 주세요. 기준을 함께 설명해 주세요.
```

확인해야 할 행 단위 이상치는 아래 3건입니다.

| `order_id` | 문제 |
|---:|---|
| 11496 | `qty`가 999이고 `unit_price * qty`와 `amount`가 맞지 않음 |
| 13013 | `amount`가 -5,500원 |
| 16235 | `qty`가 999이고 `unit_price * qty`와 `amount`가 맞지 않음 |

여기서 `unit_price * qty`는 단가에 수량을 곱한 값입니다. 커피 한 잔이 5,500원이고 2잔 팔렸다면 주문금액은 11,000원이 되어야 합니다. 이 계산이 맞지 않으면 그 행은 매출 집계에 바로 넣지 말고 따로 확인해야 합니다.

## 직접 해보기. 두 AI 답 비교

같은 파일과 같은 질문을 두 AI에게 넣고 답을 비교합니다. AI 도구가 하나뿐이라면 같은 AI에서 아래 두 프롬프트를 각각 실행해 비교합니다.

짧은 질문은 빠르지만 기준이 빠질 수 있습니다. 긴 질문은 조금 번거롭지만 AI가 무엇을 처리했는지 확인하기 쉽습니다.

### 비교용 프롬프트 A

```text
이 파일에서 월별 매출 추세와 이상치를 분석해 줘.
```

### 비교용 프롬프트 B

따라하기 2의 긴 프롬프트를 사용합니다.

### 제출할 표

아래 표를 채워 보세요. 빈칸에는 “예”, “아니오”, 또는 확인한 내용을 짧게 적으면 됩니다. 사전에서 단어를 찾듯이, AI 답에서 근거가 되는 문장과 숫자를 찾아 적어 봅니다.

| 비교 항목 | 답 A | 답 B | 더 믿을 답 |
|---|---|---|---|
| 행 수와 열 수를 말했는가 |  |  |  |
| 날짜 형식 혼재를 처리했는가 |  |  |  |
| 중복 `order_id` 처리 기준을 말했는가 |  |  |  |
| 수량 999, 음수 금액을 이상치로 표시했는가 |  |  |  |
| 월별 매출 표의 최고월이 2025-06인가 |  |  |  |
| 결론이 표와 차트의 숫자와 맞는가 |  |  |  |

### 제출할 결론 문장

아래 형식으로 3문장을 작성합니다. 좋은 결론은 멋진 문장보다 근거가 보이는 문장입니다.

```text
나는 답 __를 더 신뢰한다.
이유는 __________ 때문이다.
이 데이터의 월별 매출은 2월에 하락한 뒤 3월부터 6월까지 증가하며, 6월 매출이 가장 크다.
```

## 자주 생기는 문제

### 파일 업로드 버튼이 보이지 않는 경우

계정이나 도구 설정에 따라 업로드 기능이 없을 수 있습니다. 그럴 때는 수업 화면을 보며 따라가고, 직접 해보기는 수업에서 제공되는 샘플 답안과 비교합니다. 업로드가 안 된다고 해서 실습의 목적이 사라지는 것은 아닙니다. 어떤 기준을 확인해야 하는지 읽는 연습만으로도 충분히 의미가 있습니다.

### AI가 차트를 만들지 못하는 경우

먼저 월별 표부터 만들게 하고, 그 표를 기준으로 선그래프를 요청합니다. 차트가 끝내 안 되더라도 표만으로 추세는 충분히 읽을 수 있습니다. 표에서 2025-02는 내려가고, 2025-03부터 2025-06까지 올라가는지만 확인해도 큰 흐름은 잡을 수 있습니다.

### AI 답마다 숫자가 조금 다른 경우

전처리 기준이 달라졌을 가능성이 큽니다. 중복을 지웠는지, 이상치를 뺐는지, 날짜 형식을 빠짐없이 읽었는지 확인합니다. 같은 재료로 계산했는데 결과가 다르면, 대개 계산 전에 뺀 행이나 남긴 행이 서로 다릅니다.

### AI가 "이상치 없음"이라고 말하는 경우

어떤 기준의 이상치인지 물어봅니다. 일별 매출 합계 기준으로는 이상치가 없을 수 있지만, 행 단위에서는 수량 999와 음수 금액이 있습니다. 이상치는 기준을 정해 놓고 찾아야 합니다. 기준이 없으면 AI가 “없다”고 말해도 그대로 믿기 어렵습니다.
