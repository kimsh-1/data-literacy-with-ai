## 따라하기 1: 패키지와 한글폰트 설정

차트를 그리기 전에 공통 설정을 먼저 합니다. 이 단계가 필요한 이유는 차트 제목, 축 이름, 저장 위치, 한글 글꼴을 한 번에 준비해 두어야 뒤 실습에서 같은 설정을 반복하지 않아도 되기 때문입니다.

아래 코드를 실행해 보세요. 필요한 도구를 불러오고, 차트 결과를 저장할 위치를 준비한 뒤, 현재 컴퓨터에서 사용할 수 있는 한글 글꼴을 찾아 선택합니다.

```python
from pathlib import Path

import matplotlib.pyplot as plt
import matplotlib.ticker as mticker
import pandas as pd
import seaborn as sns
from matplotlib import font_manager

ROOT = Path("/mnt/d/ai-data-analysis-course")
DATA = ROOT / "02_datasets"
OUT = ROOT / "06_answers/S14_outputs"
OUT.mkdir(parents=True, exist_ok=True)

font_candidates = [
    "NanumGothic",
    "NanumGothicCoding",
    "Noto Sans CJK KR",
    "Malgun Gothic",
    "AppleGothic",
    "DejaVu Sans",
]

available_fonts = {font.name for font in font_manager.fontManager.ttflist}
selected_font = next(
    (font for font in font_candidates if font in available_fonts),
    "DejaVu Sans",
)

sns.set_theme(style="whitegrid")
plt.rcParams["font.family"] = selected_font
plt.rcParams["axes.unicode_minus"] = False

print("선택된 글꼴:", selected_font)
```

한글이 네모로 보이면 데이터가 잘못된 것이 아니라, matplotlib이 한글 글꼴을 찾지 못한 상태일 수 있습니다. 이럴 때는 나눔폰트를 설치해 보세요. Ubuntu, WSL, Colab에서는 아래 명령을 참고합니다.

```bash

```

Colab에서는 명령 앞에 `!`를 붙여 실행합니다. 설치 후에도 한글이 깨지면 런타임을 다시 시작하거나 matplotlib 폰트 캐시를 지운 뒤 다시 실행해 보세요. 글꼴 설정은 이름표를 붙이는 일과 비슷합니다. 이름표가 제대로 붙어야 차트를 다른 사람이 읽을 수 있습니다.

## 따라하기 2: 서울 지하철 시간대 히트맵 만들기

수업 홈의 데이터 받기 페이지에서 서울 지하철 시간대별 이용 파일을 내려받아 진행합니다. 이 단계가 필요한 이유는 큰 원자료를 바로 그리지 않고, 질문에 맞는 작은 표로 줄인 뒤 색으로 패턴을 보기 위해서입니다.

서울 지하철 시간대별 이용 데이터는 199,424행입니다. 이 많은 행을 그대로 그래프로 그리면 사람이 읽기 어렵습니다. 그래서 승차 데이터만 고르고, 역별·시간대별 합계로 줄입니다. 마치 긴 영수증 전체를 보는 대신 품목별 합계만 다시 적는 것과 같습니다.

```python
subway_path = DATA / "real_korean/seoul_subway/seoulmetro_daily_hourly_2025.csv"
subway = pd.read_csv(subway_path)

time_cols = [
    "06시이전",
    "06-07시간대",
    "07-08시간대",
    "08-09시간대",
    "09-10시간대",
    "10-11시간대",
    "11-12시간대",
    "12-13시간대",
    "13-14시간대",
    "14-15시간대",
    "15-16시간대",
    "16-17시간대",
    "17-18시간대",
    "18-19시간대",
    "19-20시간대",
    "20-21시간대",
    "21-22시간대",
    "22-23시간대",
    "23-24시간대",
    "24시이후",
]

ride = subway[subway["승하차구분"] == "승차"].copy()
ride["역라벨"] = ride["호선"].astype(str) + " " + ride["역명"].astype(str)

station_hour = ride.groupby("역라벨")[time_cols].sum()
top_stations = station_hour.sum(axis=1).sort_values(ascending=False).head(10).index
heatmap_wide = station_hour.loc[top_stations, time_cols]

print(subway.shape)
print(heatmap_wide.shape)
```

확인할 결과:

```text
지하철 원본: 199,424행, 26열
히트맵 데이터: 10행, 20열
```

여기서 10행은 승차 인원이 많은 상위 10개 역이고, 20열은 시간대입니다. 이제 이 숫자표를 색으로 바꿔 봅니다. 히트맵은 값이 큰 칸을 더 진한 색으로 보여주는 차트입니다.

```python
fig, ax = plt.subplots(figsize=(13, 6))
sns.heatmap(
    heatmap_wide / 1_000_000,
    cmap="Blues",
    linewidths=0.4,
    cbar_kws={"label": "연간 승차인원(백만 명)"},
    ax=ax,
)

ax.set_title("서울 지하철 주요 역의 시간대별 승차 패턴")
ax.set_xlabel("시간대")
ax.set_ylabel("역")
ax.tick_params(axis="x", labelrotation=45)

plt.tight_layout()
plt.savefig(OUT / "s14_subway_heatmap.png", dpi=160)
plt.show()
```

색이 진한 칸은 해당 역과 해당 시간대의 승차 인원이 많다는 뜻입니다. 숫자를 하나하나 읽는 대신 색의 진하기로 사람이 빠르게 패턴을 볼 수 있게 만드는 것이 히트맵의 장점입니다.

BI에서 쓰기 좋도록 히트맵용 데이터를 긴 표로 저장합니다. 이 단계가 필요한 이유는 BI 도구가 보통 “역, 시간대, 값”처럼 한 행에 하나의 관측값이 들어 있는 긴 표를 다루기 쉽기 때문입니다.

```python
heatmap_long = (
    heatmap_wide.reset_index()
    .melt(id_vars="역라벨", var_name="시간대", value_name="연간승차인원")
    .assign(연간승차인원_백만=lambda df: df["연간승차인원"] / 1_000_000)
)

heatmap_long.to_csv(
    OUT / "s14_subway_station_hour_heatmap.csv",
    index=False,
    encoding="utf-8-sig",
)

print(heatmap_long.head())
```

`heatmap_long`은 200행입니다. 행 10개 역과 열 20개 시간대가 “역 하나, 시간대 하나, 승차인원 하나”라는 긴 표로 바뀌었기 때문입니다. 넓은 표를 긴 표로 바꾸는 과정은 가로로 펼쳐진 출석부를 이름표 목록으로 다시 적는 것과 비슷하다고 보면 됩니다.

## 따라하기 3: 강남 아파트 거래금액 히스토그램

수업 홈의 데이터 받기 페이지에서 강남 아파트 실거래 파일을 내려받아 진행합니다. 이 단계가 필요한 이유는 가격 데이터가 글자처럼 들어와 있을 때, 계산할 수 있는 숫자로 바꾸어야 그래프를 그릴 수 있기 때문입니다.

아파트 실거래 데이터의 `거래금액(만원)`은 쉼표가 들어간 문자열입니다. 사람 눈에는 금액처럼 보이지만, 컴퓨터 입장에서는 아직 글자입니다. 쉼표를 빼고 숫자로 바꾼 뒤, 억원 단위 열을 만들어 보겠습니다. 억원 단위로 바꾸면 큰 금액을 더 편하게 읽을 수 있습니다.

```python
apt_path = DATA / "real_korean/realestate_apt/molit_apt_sale_seoul_gangnam_2024.csv"
apt = pd.read_csv(apt_path)

apt["거래금액_만원"] = pd.to_numeric(
    apt["거래금액(만원)"].astype(str).str.replace(",", "", regex=False),
    errors="coerce",
)
apt["거래금액_억원"] = apt["거래금액_만원"] / 10_000
apt["계약월"] = pd.to_datetime(apt["계약년월"].astype(str), format="%Y%m")

apt = apt.dropna(subset=["전용면적(㎡)", "거래금액_만원", "계약월"]).copy()

print(apt.shape)
print(apt[["전용면적(㎡)", "거래금액_만원", "거래금액_억원"]].head())
```

확인할 결과:

```text
분석 가능 행수: 3,754행
```

이제 히스토그램을 그립니다. 히스토그램은 “어느 가격대에 거래가 많이 몰려 있는가”를 보는 차트입니다. 사람 키를 150cm대, 160cm대, 170cm대처럼 구간으로 나누어 세는 것과 비슷합니다.

```python
fig, ax = plt.subplots(figsize=(9, 5))
sns.histplot(

ax.set_title("강남구 아파트 거래금액 분포")

plt.tight_layout()
plt.savefig(OUT / "s14_apt_price_histogram.png", dpi=160)
```

히스토그램은 정확한 단지별 비교를 하려는 차트가 아닙니다. 전체 거래가 낮은 가격대에 많은지, 중간 가격대에 많은지, 높은 가격대까지 넓게 퍼져 있는지를 볼 때 적합합니다.

## 따라하기 4: 면적구간별 박스플롯

전용면적을 구간으로 나누어 봅니다. 이 단계가 필요한 이유는 면적을 하나하나 따로 보면 너무 잘게 쪼개져 있어서, 비슷한 크기끼리 묶어야 가격 범위를 비교하기 쉽기 때문입니다.

`pd.cut()`은 숫자를 구간으로 자르는 도구입니다. 줄자로 키를 잰 뒤 “160cm대”, “170cm대”로 묶는 것처럼, 전용면적을 `40㎡ 이하`, `40-60㎡`, `60-85㎡`, `85-120㎡`, `120㎡ 초과`로 나눕니다.

```python
apt["면적구간"] = pd.cut(
    apt["전용면적(㎡)"],
    bins=[0, 40, 60, 85, 120, float("inf")],
    labels=["40㎡ 이하", "40-60㎡", "60-85㎡", "85-120㎡", "120㎡ 초과"],
)

print(apt["면적구간"].value_counts(sort=False))
```

박스플롯을 그립니다. 박스플롯은 그룹별 중앙값과 퍼짐을 한 번에 보여주는 차트입니다.

```python
fig, ax = plt.subplots(figsize=(10, 5))
sns.boxplot(
    data=apt,
    x="면적구간",
    y="거래금액_억원",
    color="#F5BACC",
    ax=ax,
)

ax.set_title("면적구간별 거래금액의 범위 비교")
ax.set_xlabel("면적구간")
ax.set_ylabel("거래금액(억원)")

plt.tight_layout()
plt.savefig(OUT / "s14_apt_area_boxplot.png", dpi=160)
plt.show()
```

엑셀 피벗에서는 그룹별 평균만 보는 경우가 많습니다. 그런데 평균 하나만 보면 같은 평균 안에서도 값이 넓게 퍼져 있는지, 좁게 모여 있는지 알기 어렵습니다. 박스플롯은 가운데 값과 퍼짐을 함께 보여주기 때문에 면적구간별 가격 차이를 더 입체적으로 읽을 수 있습니다.

## 따라하기 5: 산점도와 추세선

전용면적과 거래금액의 관계를 산점도와 추세선으로 봅니다. 이 단계가 필요한 이유는 두 숫자가 함께 움직이는지, 예를 들어 면적이 커질수록 거래금액도 커지는 경향이 있는지 확인하기 위해서입니다.

산점도는 데이터 한 행을 점 하나로 찍는 차트입니다. 여기서는 x축에 전용면적, y축에 거래금액을 둡니다. 점들이 오른쪽 위로 올라가는 모양이면 면적이 큰 집일수록 가격도 높아지는 경향이 있다고 읽을 수 있습니다.

```python
corr = apt["전용면적(㎡)"].corr(apt["거래금액_만원"])
print(f"상관계수 r = {corr:.3f}")

fig, ax = plt.subplots(figsize=(9, 5.5))
sns.regplot(
    data=apt,
    x="전용면적(㎡)",
    y="거래금액_억원",
    ci=None,
    scatter_kws={"s": 28, "alpha": 0.5, "color": "#F0986E"},
    line_kws={"color": "#1F2430"},
    ax=ax,
)

ax.set_title(f"전용면적과 거래금액의 관계: r={corr:.3f}")
ax.set_xlabel("전용면적(㎡)")
ax.set_ylabel("거래금액(억원)")

plt.tight_layout()
plt.savefig(OUT / "s14_apt_scatter_trend.png", dpi=160)
plt.show()
```

확인할 결과:

```text
상관계수 r = 0.806
```

상관계수 r은 두 숫자가 얼마나 함께 움직이는지 요약한 값입니다. 여기서는 r = 0.806이므로 강한 양의 상관으로 볼 수 있습니다. 다만 상관계수가 높아도 원인을 증명한 것은 아닙니다. 같은 면적이라도 단지, 위치, 층, 연식, 거래 시점이 가격에 영향을 줄 수 있습니다. 그래서 “면적이 가격과 함께 움직이는 경향이 있다”라고 말할 수는 있지만, “면적 하나가 가격을 결정한다”라고 말하면 지나친 해석입니다.

아파트 차트에 쓴 데이터를 저장합니다. 이 단계가 필요한 이유는 원자료 전체가 아니라 차트에 필요한 열을 정리한 표를 BI 도구나 다른 사람에게 넘기기 위해서입니다.

```python
apt[
    ["시군구", "단지명", "계약월", "전용면적(㎡)", "거래금액_만원", "거래금액_억원", "면적구간"]
].to_csv(
    OUT / "s14_apt_chart_ready.csv",
    index=False,
    encoding="utf-8-sig",
)
```

`index=False`는 pandas가 자동으로 붙인 행 번호를 파일에 따로 저장하지 않겠다는 뜻입니다. `encoding="utf-8-sig"`는 엑셀이나 일부 BI 도구에서 한글 열 이름을 더 안정적으로 읽게 도와줍니다.

## 따라하기 6: FRED CPI 월별 라인차트

수업 홈의 데이터 받기 페이지에서 한국 소비자물가지수 월별 파일을 내려받아 진행합니다. 이 단계가 필요한 이유는 시간 순서가 있는 데이터는 막대보다 선으로 이어 보아야 흐름을 읽기 쉽기 때문입니다.

CPI는 소비자물가지수입니다. 물가가 시간이 지나며 어떻게 변했는지 보는 지표라고 생각하면 됩니다. 월별 시계열은 라인차트로 봅니다. 원본 767행 중 2000년 이후만 사용합니다.

```python
cpi_path = DATA / "real_korean/prices_econ/fred_korea_cpi_all_items_monthly.csv"
cpi = pd.read_csv(cpi_path)

cpi["기준월"] = pd.to_datetime(cpi["observation_date"])
cpi["소비자물가지수"] = pd.to_numeric(cpi["KORCPIALLMINMEI"], errors="coerce")

cpi_recent = cpi[cpi["기준월"] >= "2000-01-01"][["기준월", "소비자물가지수"]].copy()

print(cpi.shape)
print(cpi_recent.shape)
print(cpi_recent.head())
```

확인할 결과:

```text
CPI 원본: 767행, 2열
2000년 이후: 287행
```

라인차트를 저장합니다. 선이 위로 올라가면 소비자물가지수가 상승한 것이고, 선이 완만해지면 상승 속도가 느려진 것으로 읽을 수 있습니다.

```python
fig, ax = plt.subplots(figsize=(10, 5))
sns.lineplot(
    data=cpi_recent,
    x="기준월",
    y="소비자물가지수",
    color="#5477C4",
    ax=ax,
)

ax.set_title("한국 소비자물가지수 월별 추세")
ax.set_xlabel("기준월")
ax.set_ylabel("소비자물가지수")

plt.tight_layout()
plt.savefig(OUT / "s14_cpi_line.png", dpi=160)
plt.show()

cpi_recent.to_csv(OUT / "s14_cpi_recent.csv", index=False, encoding="utf-8-sig")
```

차트만 저장하는 것이 아니라 차트에 사용한 표도 함께 저장했습니다. 나중에 BI 도구에서 같은 차트를 다시 만들거나, 다른 사람이 숫자를 검토해야 할 때 이 정리된 표가 필요합니다.

## 직접 해보기: 월별 매출 추세 라인차트 저장

수업 홈의 데이터 받기 페이지에서 카페 매출 정제 파일을 내려받아 진행하세요. 이 실습이 필요한 이유는 앞에서 배운 “날짜 변환, 월별 집계, 라인차트 저장, BI용 CSV 저장” 흐름을 스스로 한 번 연결해 보기 위해서입니다.

목표:

1. `date`를 날짜형으로 변환합니다. 그래야 컴퓨터가 글자를 실제 날짜로 읽고 월별로 묶을 수 있습니다.
2. 월별 `amount` 합계를 계산합니다. 주문 한 건 한 건을 보는 대신 월별 매출이라는 요약표를 만들기 위해서입니다.
3. 월별 매출 라인차트를 저장합니다. 시간 흐름에 따라 매출이 올라갔는지 내려갔는지 눈으로 보기 위해서입니다.
4. 월별 집계표를 CSV로 저장합니다. BI 도구에 넘길 수 있는 정리된 표를 만들기 위해서입니다.
5. 6월이 가장 높고 2월이 가장 낮은지 확인합니다. 그래프를 그린 뒤에는 숫자로도 결과가 맞는지 확인해야 합니다.

힌트 코드:

```python
cafe = pd.read_csv(DATA / "cafe_sales/cafe_sales_clean.csv")
cafe["date"] = pd.to_datetime(cafe["date"])

monthly_sales = (
    cafe.groupby(pd.Grouper(key="date", freq="MS"))
    .agg(매출합계=("amount", "sum"), 주문수=("order_id", "count"))
    .reset_index()
    .rename(columns={"date": "월"})
)

print(monthly_sales)
```

저장 결과는 두 가지입니다. 월별 집계표는 `s14_monthly_sales.csv`라는 이름으로 저장하고, 월별 매출 추세 차트는 `s14_monthly_sales_trend.png`라는 이름으로 저장해 보세요. 저장 위치는 앞에서 공통 설정으로 준비한 출력 위치를 그대로 사용하면 됩니다.

완료 기준:

- `monthly_sales`가 6행이다.
- 2025년 6월 매출이 `16,816,500`원이다.
- 2025년 2월 매출이 `13,157,000`원이다.
- CSV 파일과 PNG 차트 파일이 실제로 저장되어 있다.

이 핸드오프, 즉 정제 결과를 CSV로 저장해 BI로 넘기는 과정은 S19 풀사이클에서 BI 입력으로 실제 재현됩니다.
