# S18 핸드아웃: Looker Studio로 대시보드 공유하기

## 오늘 사용할 것

- Chrome 또는 Edge 브라우저
- Google 계정 또는 Gmail 계정
- Looker Studio: `https://lookerstudio.google.com/`
- 데이터 파일

```text
/mnt/d/ai-data-analysis-course/02_datasets/survey_responses/survey_responses_clean.csv
```

Power BI Service 게시와 Power BI Copilot은 조직 계정이나 유료 환경을 요구할 때가 있다. 그래서 오늘 표준 실습은 Looker Studio로 간다. 개인 Gmail 계정 하나만 있어도 이 길로 미션을 끝낼 수 있다.

## 오늘 만들 대시보드

설문 응답 1,500건을 사용해 다음 요소가 있는 1페이지 대시보드를 만든다.

| 위치 | 요소 | 확인할 값 또는 내용 |
|---|---|---|
| 상단 | 응답 수 카드 | 1,500 |
| 상단 | 평균 NPS 카드 | 약 6.86 |
| 상단 | 평균 만족도 카드 | 약 3.62 |
| 상단 | NPS 점수 카드 | 약 -4.73 |

수업 화면이 영어로 떠 있을 수 있으니, 메뉴 이름은 한국어와 영어를 나란히 확인하며 따라간다.

## 따라하기 1. Looker Studio 열기

### 1단계. 로그인

1. 브라우저를 연다.
2. 아래 주소로 이동한다.

```text
https://lookerstudio.google.com/
```

3. Google 계정으로 로그인한다.
4. 처음 접속한 경우 약관이나 국가 설정 화면이 나올 수 있다. 강사의 안내에 따라 진행한다.

### 2단계. 빈 보고서 만들기

1. 왼쪽 위 또는 시작 화면에서 `만들기(Create)`를 누른다.
2. `보고서(Report)` 또는 `빈 보고서(Blank report)`를 선택한다.
3. 데이터 추가 화면이 나타나면 다음 단계로 넘어간다.

## 따라하기 2. CSV 파일 연결하기

### 1단계. 파일 업로드 커넥터 선택

1. 데이터 연결 화면에서 `파일 업로드(File Upload)` 또는 `CSV 파일 업로드(CSV file upload)`를 찾는다.
2. Google 제공 커넥터인지 확인한다.
3. 커넥터를 선택한다.

### 2단계. CSV 업로드

1. `파일 업로드` 또는 `데이터세트 추가(Add data set)`를 누른다.
2. 아래 파일을 선택한다. 브라우저 파일 선택 창에서는 Windows 경로를 사용한다.

```text
Windows: D:\ai-data-analysis-course\02_datasets\survey_responses\survey_responses_clean.csv
WSL 참고: /mnt/d/ai-data-analysis-course/02_datasets/survey_responses/survey_responses_clean.csv
```

3. 업로드가 끝나면 `연결(Connect)`을 누른다.
4. 필드 목록이 보이면 `보고서에 추가(Add to report)`를 누른다.

공식 문서에 따르면 CSV 업로드 파일은 UTF-8로 인코딩된 표 형식 데이터여야 한다. 수업 배포본은 이미 UTF-8로 맞춰 두었으니 따로 변환할 것은 없다.

### 3단계. 필드 확인

필드 목록에서 아래 열이 보이는지 확인한다.

```text
respondent_id
age_group
gender
nps
satisfaction
channel
free_text
submitted_at
```

필드 유형은 다음처럼 맞춘다.

| 필드 | 권장 유형 |
|---|---|
| `respondent_id` | 텍스트 |
| `age_group` | 텍스트 |
| `gender` | 텍스트 |
| `nps` | 숫자 |
| `satisfaction` | 숫자 |
| `channel` | 텍스트 |
| `free_text` | 텍스트 |
| `submitted_at` | 날짜 및 시간 |

`submitted_at`이 텍스트로 남아 있어도 오늘 필수 미션에는 지장이 없다. 시간이 남는다면 날짜 및 시간 유형으로 바꿔 기간 필터를 하나 더 얹어 본다.

## 따라하기 3. 계산 필드 만들기

### 1단계. NPS 그룹 만들기

데이터 원본 또는 보고서 편집 화면에서 `필드 추가(Add a field)`를 누른다.

필드 이름:

```text
NPS_그룹
```

수식:

```text
CASE
  WHEN nps >= 9 THEN "추천 고객"
  WHEN nps >= 7 THEN "중립 고객"
  ELSE "비추천 고객"
END
```

저장한다. 이 필드는 0-10점짜리 NPS 원점수를 세 그룹으로 갈라 준다.

### 2단계. NPS 점수 계산용 필드 만들기

다시 `필드 추가(Add a field)`를 누른다.

필드 이름:

```text
NPS_점수_행
```

수식:

```text
CASE
  WHEN nps >= 9 THEN 100
  WHEN nps <= 6 THEN -100
  ELSE 0
END
```

저장한다. 이 필드는 추천 고객을 100, 중립 고객을 0, 비추천 고객을 -100으로 바꿔 준다. 그런 다음 이 필드의 평균을 내면 그대로 NPS 점수가 된다.

### 3단계. 만족도 구간 만들기

필수는 아니지만, 나중에 필터나 색상 구분에 요긴하게 쓸 수 있다.

필드 이름:

```text
만족도_구간
```

수식:

```text
CASE
  WHEN satisfaction >= 4 THEN "만족"
  WHEN satisfaction = 3 THEN "보통"
  ELSE "불만족"
END
```

| 상황 | 권장 설정 |

4. `링크 복사(Copy link)`를 누른다.

## Power BI Copilot 선택 시연을 볼 때의 기준

Power BI Copilot은 모든 사용자에게 알아서 켜지는 기능이 아니다. 조직의 Fabric 관리자 설정, 유료 Fabric이나 Premium 용량, 워크스페이스 권한이 맞물려야 열린다. 수업 중 강사 환경에 이게 갖춰져 있다면 다음 정도만 가볍게 시연한다.

```text
이 보고서의 주요 변화를 한 문단으로 요약해줘.
NPS가 낮은 채널을 중심으로 설명해줘.
```

Copilot이 내놓는 답도 결론이 아니라 초안일 뿐이다. 핵심 숫자는 원본 데이터와 카드 값으로 다시 맞춰 봐야 한다.
