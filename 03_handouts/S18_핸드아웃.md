# S18 핸드아웃: Looker Studio로 대시보드 공유하기

## 오늘 사용할 것

- Chrome 또는 Edge 브라우저
- Google 계정 또는 Gmail 계정
- Looker Studio: `https://lookerstudio.google.com/`
- 수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 파일

Looker Studio에 올릴 때는 엑셀로도 열리는 표 파일인 CSV를 사용합니다. 엑셀 화면에서 먼저 데이터를 확인하고 싶은 분은 같은 내용의 엑셀 파일(.xlsx)을 받아서 열어 봐도 됩니다.

Power BI Service에 보고서를 게시하거나 Power BI Copilot을 쓰려면 조직 계정이나 유료 환경이 필요한 경우가 있습니다. 그래서 오늘 표준 실습은 Looker Studio로 진행합니다. 개인 Gmail 계정 하나만 있어도 보고서를 만들고 공유 링크까지 확인할 수 있습니다.

## 오늘 만들 대시보드

설문 응답 1,500건을 사용해 1페이지 대시보드를 만듭니다. 대시보드는 여러 장의 표를 따로 보는 대신, 중요한 숫자와 차트를 한 화면에서 보는 보고서라고 생각하시면 됩니다.

| 위치 | 요소 | 확인할 값 또는 내용 |
|---|---|---|
| 상단 | 응답 수 카드 | 1,500 |
| 상단 | 평균 NPS 카드 | 약 6.86 |
| 상단 | 평균 만족도 카드 | 약 3.62 |
| 상단 | NPS 점수 카드 | 약 -4.73 |
| 본문 | NPS 그룹별 막대그래프 | 추천, 중립, 비추천 고객 수 |
| 본문 | 채널별 표 | 채널별 응답 수, 평균 NPS, 평균 만족도 |
| 본문 | 연령대별 막대그래프 | 연령대별 평균 만족도 |

NPS는 고객이 우리 서비스를 다른 사람에게 추천할 마음이 얼마나 있는지를 0점부터 10점까지로 표시한 점수입니다. 수업 화면이 영어로 보일 수 있으니, 메뉴 이름은 한국어와 영어를 나란히 확인하며 따라가 보세요.

## 따라하기 1. Looker Studio 열기

### 1단계. 로그인

이 단계는 보고서를 웹에서 만들고 저장하기 위해 내 Google 계정으로 작업 공간에 들어가는 과정입니다.

1. 브라우저를 연다.
2. 아래 주소로 이동한다.

```text
https://lookerstudio.google.com/
```

3. Google 계정으로 로그인한다.
4. 처음 접속한 경우 약관이나 국가 설정 화면이 나올 수 있다. 강사의 안내에 따라 진행한다.

약관 화면이나 국가 설정 화면은 처음 쓰는 도구에서 기본 정보를 확인하는 절차입니다. 한 번 지나가면 보통 다시 나오지 않습니다.

### 2단계. 빈 보고서 만들기

이 단계는 아직 아무 차트도 없는 빈 도화지를 여는 과정입니다.

1. 왼쪽 위 또는 시작 화면에서 `만들기(Create)`를 누른다.
2. `보고서(Report)` 또는 `빈 보고서(Blank report)`를 선택한다.
3. 데이터 추가 화면이 나타나면 다음 단계로 넘어간다.

빈 보고서를 먼저 만들고 나서 데이터를 연결합니다. 엑셀에서 새 통합 문서를 열고 표를 붙여 넣는 흐름과 비슷하게 보면 됩니다.

## 따라하기 2. CSV 파일 연결하기

### 1단계. 파일 업로드 커넥터 선택

이 단계는 Looker Studio에게 “내가 가진 표 파일을 직접 올릴게요”라고 알려 주는 과정입니다.

1. 데이터 연결 화면에서 `파일 업로드(File Upload)` 또는 `CSV 파일 업로드(CSV file upload)`를 찾는다.
2. Google 제공 커넥터인지 확인한다.
3. 커넥터를 선택한다.

커넥터는 데이터를 어디에서 가져올지 정하는 문이라고 생각하시면 됩니다. 오늘은 웹 광고 계정이나 데이터베이스가 아니라, 우리가 내려받은 설문 응답 파일을 직접 올립니다.

### 2단계. CSV 업로드

이 단계는 수업 홈의 데이터 받기 페이지에서 받은 설문 응답 파일을 Looker Studio에 넣는 과정입니다.

1. `파일 업로드` 또는 `데이터세트 추가(Add data set)`를 누른다.
2. 수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 CSV 파일을 선택한다.
3. 업로드가 끝나면 `연결(Connect)`을 누른다.
4. 필드 목록이 보이면 `보고서에 추가(Add to report)`를 누른다.

공식 문서에 따르면 CSV 업로드 파일은 UTF-8로 인코딩된 표 형식 데이터여야 합니다. UTF-8은 한글이 깨지지 않게 글자를 저장하는 방식이라고 이해하시면 됩니다. 수업에서 받는 설문 응답 파일은 이미 맞춰져 있으니 따로 변환하지 않아도 됩니다.

### 3단계. 필드 확인

이 단계는 Looker Studio가 각 열을 제대로 알아봤는지 확인하는 과정입니다. 엑셀에서 열 이름과 데이터 형식을 눈으로 확인하는 것과 같습니다.

필드 목록에서 아래 열이 보이는지 확인한다.

```text
respondent_id
age_group
gender
nps
satisfaction
channel
free_text
submitted_at
```

필드 유형은 다음처럼 맞춘다.

| 필드 | 권장 유형 |
|---|---|
| `respondent_id` | 텍스트 |
| `age_group` | 텍스트 |
| `gender` | 텍스트 |
| `nps` | 숫자 |
| `satisfaction` | 숫자 |
| `channel` | 텍스트 |
| `free_text` | 텍스트 |
| `submitted_at` | 날짜 및 시간 |

`respondent_id`처럼 계산할 필요가 없는 값은 텍스트로 봅니다. `nps`와 `satisfaction`은 평균을 내야 하므로 숫자로 봐야 합니다. `submitted_at`이 텍스트로 남아 있어도 오늘 필수 미션에는 지장이 없습니다. 시간이 남는다면 날짜 및 시간 유형으로 바꿔 기간 필터를 하나 더 얹어 보세요.

## 따라하기 3. 계산 필드 만들기

### 1단계. NPS 그룹 만들기

이 단계는 0점부터 10점까지 흩어져 있는 NPS 점수를 추천, 중립, 비추천 세 묶음으로 나누는 과정입니다.

데이터 원본 또는 보고서 편집 화면에서 `필드 추가(Add a field)`를 누른다.

필드 이름:

```text
NPS_그룹
```

수식:

```text
CASE
  WHEN nps >= 9 THEN "추천 고객"
  WHEN nps >= 7 THEN "중립 고객"
  ELSE "비추천 고객"
END
```

저장한다. 이 필드는 0-10점짜리 NPS 원점수를 세 그룹으로 갈라 줍니다. 키를 잴 때 171.2cm, 168.5cm를 그대로 외우기보다 “160cm대, 170cm대”로 묶어 보면 더 빨리 보이는 것과 같은 원리입니다.

### 2단계. NPS 점수 계산용 필드 만들기

이 단계는 NPS 공식에 맞게 각 응답을 100, 0, -100으로 바꿔 평균을 낼 준비를 하는 과정입니다.

다시 `필드 추가(Add a field)`를 누른다.

필드 이름:

```text
NPS_점수_행
```

수식:

```text
CASE
  WHEN nps >= 9 THEN 100
  WHEN nps <= 6 THEN -100
  ELSE 0
END
```

저장한다. 추천 고객은 100, 중립 고객은 0, 비추천 고객은 -100으로 바뀝니다. 그런 다음 이 필드의 평균을 내면 NPS 점수가 됩니다. 사람마다 점수를 먼저 붙이고 반 평균을 내는 방식이라고 보면 됩니다.

### 3단계. 만족도 구간 만들기

이 단계는 만족도 1점부터 5점까지의 숫자를 만족, 보통, 불만족이라는 말로 읽기 쉽게 바꾸는 과정입니다.

필수는 아니지만, 나중에 필터나 색상 구분에 요긴하게 쓸 수 있다.

필드 이름:

```text
만족도_구간
```

수식:

```text
CASE
  WHEN satisfaction >= 4 THEN "만족"
  WHEN satisfaction = 3 THEN "보통"
  ELSE "불만족"
END
```

숫자만 있을 때보다 말로 된 구간이 있으면 차트를 읽는 사람이 빠르게 이해합니다. 사전에서 단어 뜻을 찾아 붙이듯, 숫자에 이름표를 붙이는 작업입니다.

## 따라하기 4. 카드 4개 만들기

### 1단계. 응답 수 카드

이 단계는 이 대시보드가 몇 명의 응답을 바탕으로 만들어졌는지 맨 위에서 바로 보이게 하는 과정입니다.

1. 상단 메뉴에서 `차트 추가(Add a chart)`를 누른다.
2. `스코어카드(Scorecard)`를 선택한다.
3. 캔버스 상단 왼쪽에 배치한다.
4. 측정항목에 `respondent_id`를 넣는다.
5. 집계 방식을 `고유 개수(Count Distinct)` 또는 `개수(Count)`로 설정한다.
6. 제목을 아래처럼 바꾼다.

```text
응답 수
```

정답 기준은 **1,500**입니다. 이 숫자가 다르면 파일을 잘못 올렸거나 같은 데이터를 중복으로 올렸을 가능성이 있습니다.

### 2단계. 평균 NPS 카드

이 단계는 전체 응답자의 추천 의향이 평균적으로 어느 정도인지 한눈에 보이게 하는 과정입니다.

1. 스코어카드를 하나 더 만든다.
2. 측정항목에 `nps`를 넣는다.
3. 집계 방식을 `평균(Average)`으로 바꾼다.
4. 소수 자릿수를 2자리로 맞춘다.
5. 제목을 아래처럼 바꾼다.

```text
평균 NPS
```

정답 기준은 약 `6.86`입니다. 합계가 아니라 평균입니다. 시험 점수 전체 합을 보는 것이 아니라 반 평균을 보는 상황과 같습니다.

### 3단계. 평균 만족도 카드

이 단계는 전체 응답자의 만족도를 1점부터 5점 사이의 평균값으로 확인하는 과정입니다.

1. 스코어카드를 하나 더 만든다.
2. 측정항목에 `satisfaction`을 넣는다.
3. 집계 방식을 `평균(Average)`으로 바꾼다.
4. 소수 자릿수를 2자리로 맞춘다.
5. 제목을 아래처럼 바꾼다.

```text
평균 만족도
```

정답 기준은 약 `3.62`입니다. 만족도는 한 사람의 의견보다 전체 흐름을 보는 숫자라서 평균으로 보는 것이 자연스럽습니다.

### 4단계. NPS 점수 카드

이 단계는 추천 고객 비율과 비추천 고객 비율의 차이를 하나의 점수로 보여 주는 과정입니다.

1. 스코어카드를 하나 더 만든다.
2. 측정항목에 `NPS_점수_행`을 넣는다.
3. 집계 방식을 `평균(Average)`으로 바꾼다.
4. 소수 자릿수를 2자리로 맞춘다.
5. 제목을 아래처럼 바꾼다.

```text
NPS 점수
```

정답 기준은 약 `-4.73`입니다. 앞에서 만든 계산 필드를 평균으로 집계해야 이 값이 나옵니다. `NPS_점수_행`을 합계로 보면 전혀 다른 큰 숫자가 나오니 조심해 보세요.

## 따라하기 5. 기본 차트 만들기

### 1단계. NPS 그룹별 막대그래프

이 단계는 추천, 중립, 비추천 고객이 각각 몇 명인지 막대 길이로 비교하는 과정입니다.

1. `차트 추가(Add a chart)`를 누른다.
2. `막대그래프(Bar chart)` 또는 `세로 막대그래프(Column chart)`를 선택한다.
3. 차원에는 `NPS_그룹`을 넣는다.
4. 측정항목에는 `respondent_id` 개수 또는 `Record Count`를 넣는다.
5. 제목을 아래처럼 바꾼다.

```text
NPS 그룹별 응답 수
```

검산용 응답 수:

| NPS 그룹 | 응답 수 |
|---|---:|
| 비추천 고객 | 552 |
| 추천 고객 | 481 |
| 중립 고객 | 467 |

여기서 뭐가 보이시나요? 추천 고객도 많지만, 비추천 고객이 552명으로 더 많습니다. 그래서 NPS 점수가 약 `-4.73`으로 내려갑니다.

### 2단계. 채널별 표

이 단계는 어떤 유입 채널에서 응답이 많이 들어왔고, 채널별 평균 점수가 어떻게 다른지 표로 비교하는 과정입니다.

1. `차트 추가(Add a chart)`를 누른다.
2. `표(Table)`를 선택한다.
3. 차원에는 `channel`을 넣는다.
4. 측정항목에는 다음 세 가지를 넣는다.
   - `respondent_id` 개수
   - `nps` 평균
   - `satisfaction` 평균
5. 응답 수 기준 내림차순으로 정렬한다.
6. 제목을 아래처럼 바꾼다.

```text
채널별 설문 요약
```

상위 채널은 `네이버 검색`이며 응답 수는 `435`입니다. 표는 막대그래프보다 숫자를 정확히 확인하기 좋습니다. 그래서 채널별 비교처럼 여러 값을 함께 볼 때 잘 맞습니다.

### 3단계. 연령대별 평균 만족도 막대그래프

이 단계는 연령대에 따라 평균 만족도가 어떻게 달라지는지 비교하는 과정입니다.

1. `차트 추가(Add a chart)`를 누른다.
2. `막대그래프` 또는 `세로 막대그래프`를 선택한다.
3. 차원에는 `age_group`을 넣는다.
4. 측정항목에는 `satisfaction` 평균을 넣는다.
5. 제목을 아래처럼 바꾼다.

```text
연령대별 평균 만족도
```

평균 만족도는 40대와 50대가 상대적으로 높게, 10대가 낮게 나옵니다. 이 말은 원인을 단정하는 설명이 아닙니다. 화면에 보이는 차이를 그대로 읽은 관찰입니다. “왜 그럴까?”는 추가 데이터나 인터뷰가 있어야 말할 수 있습니다.

## 따라하기 6. 공유 링크 만들기

### 1단계. 보고서 이름 바꾸기

이 단계는 내가 만든 보고서를 나중에 찾기 쉽고, 제출할 때 누구 것인지 알아보기 쉽게 만드는 과정입니다.

화면 왼쪽 위 보고서 제목을 아래처럼 바꾼다.

```text
S18_설문_공유_대시보드_본인이름
```

보고서 이름은 파일 이름표와 같습니다. 여러 사람이 같은 수업을 듣기 때문에 본인 이름을 넣어 두면 확인이 쉬워집니다.

### 2단계. 보기 모드 확인

이 단계는 내가 만든 화면이 보는 사람 입장에서 어떻게 보이는지 미리 확인하는 과정입니다.

1. 오른쪽 위 `보기(View)`를 눌러 보고서가 어떻게 보이는지 확인한다.
2. 다시 `수정(Edit)`으로 돌아온다.

편집 화면에서는 차트 테두리와 설정 패널이 보입니다. 보기 화면은 실제 공유받은 사람이 보는 화면입니다. 제출 전에는 보기 화면에서 숫자와 차트가 자연스럽게 보이는지 확인해 보세요.

### 3단계. 공유 설정

이 단계는 링크를 받은 사람이 보고서를 볼 수 있는 권한을 여는 과정입니다. 공유는 파일을 보내는 일이 아니라, 문을 열어 주는 일에 가깝습니다.

1. 오른쪽 위 `공유(Share)`를 누른다.
2. 수업 상황에 맞게 둘 중 하나를 선택한다.

| 상황 | 권장 설정 |
|---|---|
| 같은 반 수강생끼리 검증 | 특정 동료 Gmail을 `보기 가능(Viewer)`으로 추가 |
| 강사가 링크 제출만 확인 | `링크가 있는 모든 사용자 보기 가능`으로 설정 |
| 회사 또는 실제 업무 데이터 | 조직 내부 또는 특정 사용자만 보기 가능 |

3. 권한이 `편집자(Editor)`가 아니라 `보기 가능(Viewer)`인지 확인한다.
4. `링크 복사(Copy link)`를 누른다.

회사나 실제 업무 데이터라면 아무에게나 열리는 링크를 만들면 안 됩니다. 오늘 수업 데이터는 민감 정보가 없는 실습용 데이터이지만, 권한을 확인하는 습관은 그대로 가져가야 합니다.

### 4단계. 링크 검증

이 단계는 내가 복사한 링크가 실제로 다른 환경에서도 열리는지 확인하는 과정입니다.

아래 둘 중 하나로 확인한다.

| 방법 | 확인할 것 |
|---|---|
| 시크릿 창에서 열기 | 로그인 없이 또는 다른 계정으로 보기 화면이 열리는가 |
| 옆자리 동료에게 보내기 | 동료가 편집 화면이 아니라 보기 화면으로 들어오는가 |

검증할 때 다음 숫자가 보이면 정상이다.

```text
응답 수: 1,500
평균 NPS: 약 6.86
평균 만족도: 약 3.62
NPS 점수: 약 -4.73
```

링크를 만들었다고 공유가 끝난 것은 아닙니다. 실제로 열어 봐야 합니다. 문 손잡이를 만들고 나서 문이 열리는지 한 번 당겨 보는 과정이라고 생각하시면 됩니다.

## 직접 해보기. 공유 링크 생성과 검증

### 미션

Looker Studio 대시보드의 공유 링크를 만들고, 다른 환경에서 열리는지 검증해 보세요.

이 미션의 목적은 예쁜 차트를 만드는 것에서 끝나지 않습니다. 다른 사람이 보기 권한으로 안전하게 열 수 있는 보고서를 완성하는 데 있습니다.

### 조건

1. 보고서 제목이 `S18_설문_공유_대시보드_본인이름` 형식이다.
2. 수업 홈의 데이터 받기 페이지에서 내려받은 설문 응답 파일을 데이터 원본으로 사용했다.
3. 상단 카드에 응답 수, 평균 NPS, 평균 만족도, NPS 점수가 있다.
4. 응답 수가 `1,500`으로 보인다.
5. NPS 점수 카드가 약 `-4.73`으로 보인다.
6. 공유 권한은 보기 권한이다.
7. 링크를 시크릿 창 또는 동료 계정에서 열어 검증했다.

조건을 하나씩 확인해 보세요. 특히 응답 수와 NPS 점수는 대시보드가 제대로 만들어졌는지 확인하는 기준 숫자입니다.

빈칸을 채우면서 본인이 직접 검증한 값을 적어 보세요. 링크만 제출하면 실제로 열리는지 알 수 없기 때문에, 검증 방법과 카드 값까지 함께 적습니다.

## 막혔을 때 확인할 것

| 증상 | 확인할 것 |
|---|---|
| CSV 업로드가 실패한다 | 파일이 UTF-8 CSV인지, 파일 경로가 맞는지 확인한다 |
| 한글이 깨진다 | 수업 배포본이 아닌 다른 파일을 올렸는지 확인한다 |
| 응답 수가 1,500보다 크다 | 같은 CSV를 같은 데이터셋에 중복 업로드했을 수 있다 |
| 평균 NPS가 합계처럼 크게 나온다 | 집계 방식이 평균인지 확인한다 |
| NPS 점수가 이상하다 | `NPS_점수_행`을 평균으로 집계했는지 확인한다 |
| 링크를 받은 사람이 못 연다 | 공유 설정에서 Viewer 권한을 열었는지 확인한다 |
| 받은 사람이 편집할 수 있다 | 권한이 Editor로 되어 있는지 확인하고 Viewer로 바꾼다 |

문제가 생기면 먼저 숫자를 봅니다. 응답 수가 `1,500`이 아니면 데이터 연결 쪽을 의심하고, 평균 NPS나 NPS 점수가 이상하면 집계 방식이나 계산 필드를 확인해 보세요. 화면이 안 열리면 차트 문제가 아니라 공유 권한 문제일 가능성이 큽니다.

## Power BI Copilot 선택 시연을 볼 때의 기준

Power BI Copilot은 모든 사용자에게 자동으로 켜지는 기능이 아닙니다. 조직의 Fabric 관리자 설정, 유료 Fabric이나 Premium 용량, 워크스페이스 권한이 맞아야 열립니다. 수업 중 강사 환경에 이 조건이 갖춰져 있다면 아래와 같은 질문을 가볍게 시연할 수 있습니다.

```text
이 보고서의 주요 변화를 한 문단으로 요약해줘.
NPS가 낮은 채널을 중심으로 설명해줘.
```

Copilot이 내놓는 답도 최종 결론이 아니라 초안입니다. 사람이 다시 숫자를 확인해야 합니다. 특히 응답 수 `1,500`, 평균 NPS 약 `6.86`, 평균 만족도 약 `3.62`, NPS 점수 약 `-4.73`처럼 기준이 되는 숫자는 원본 데이터와 카드 값으로 다시 맞춰 봐야 합니다.
