## 따라하기 2: 문자열, 타입, 결측값 정리

이제 계산 가능한 표로 만들 차례입니다. 원본 `df`는 그대로 두고 `cleaned`라는 복사본을 만듭니다. 원본을 남겨 두면 정제 전후를 비교할 수 있습니다. 지우개로 바로 지우는 대신, 원본 종이를 복사해 놓고 고치는 느낌입니다.

```python
cleaned = df.copy()

cleaned["store"] = cleaned["store"].astype("string").str.strip().str.replace(" ", "", regex=False)
cleaned["category"] = cleaned["category"].astype("string").str.strip()
cleaned["item"] = (
    cleaned["item"]
    .astype("string")
    .str.strip()
    .replace({
        "아메리카": "아메리카노",
        "Americano": "아메리카노",
        "아메 리카노": "아메리카노",
    })
)

cleaned["payment"] = cleaned["payment"].astype("string").str.strip().fillna("미확인")
cleaned["member"] = cleaned["member"].astype("boolean").fillna(False)
```

이 코드는 세 가지 일을 합니다.

- `store`, `category`, `item`은 그룹명이 흔들리지 않게 정리합니다. 같은 지점과 같은 메뉴가 한 그룹으로 모이게 하려는 작업입니다.
- `payment` 빈칸은 주문을 버리지 않고 `미확인`으로 채웁니다. 결제수단을 모른다고 해서 매출 기록 전체를 버리면 손실이 커집니다.
- `member` 빈칸은 이번 실습 기준으로 비회원 `False`로 채웁니다. 실제 업무에서는 담당자와 기준을 정해야 하지만, 오늘은 수업 기준을 명확히 두고 진행합니다.

## 따라하기 3: 숫자와 날짜 타입 바꾸기

타입은 값의 종류입니다. 숫자처럼 보이는 문자를 진짜 숫자로 바꾸고, 날짜처럼 보이는 문자를 진짜 날짜로 바꿉니다. 그래야 합계, 평균, 월별 집계가 제대로 됩니다.

```python
cleaned["unit_price"] = pd.to_numeric(
    cleaned["unit_price"].astype("string")
    .str.replace(",", "", regex=False)
    .str.replace("원", "", regex=False),
    errors="coerce",
)
cleaned["qty"] = pd.to_numeric(cleaned["qty"], errors="coerce")
cleaned["amount"] = pd.to_numeric(cleaned["amount"], errors="coerce")
cleaned["date"] = pd.to_datetime(cleaned["date"], format="mixed", errors="coerce")

cleaned.dtypes
```

확인할 점입니다.

- `date`가 `datetime64` 계열로 바뀝니다. 이제 월별, 요일별, 기간별 분석을 할 수 있습니다.
- `unit_price`, `qty`, `amount`가 숫자형으로 바뀝니다. 이제 더하기와 평균 계산이 가능합니다.

`errors="coerce"`는 바꿀 수 없는 값이 나오면 오류로 멈추지 않고 결측값으로 표시하라는 뜻입니다. 문제 값을 찾아내기 위한 안전장치라고 보면 됩니다.

## 따라하기 4: 중복, 결측, 이상치 제거

이 단계에서는 분석에 방해되는 행을 제거합니다. 중복 주문은 한 번만 남기고, 꼭 필요한 값이 비어 있는 행은 제외합니다.

```python
before = len(cleaned)
cleaned = cleaned.drop_duplicates(subset=["order_id"], keep="first").copy()
print("중복 제거 행수:", before - len(cleaned))

before = len(cleaned)
cleaned = cleaned.dropna(
    subset=["date", "store", "category", "item", "unit_price", "qty", "amount"]
).copy()
print("필수값 결측으로 제거한 행수:", before - len(cleaned))
```

기대 결과입니다.

- 중복 `order_id` 제거: 15행
- 필수값 결측으로 제거: 0행

이번 데이터에서는 필수값 결측으로 삭제되는 행이 없습니다. 그래도 `dropna(subset=...)` 패턴은 중요합니다. 실제 업무에서는 날짜나 금액이 비어 있는 행이 자주 나오기 때문입니다.

### IQR로 수량 이상치 찾기

이상치는 다른 값들과 너무 멀리 떨어진 값입니다. 무조건 나쁜 값이라는 뜻은 아니고, 확인해야 할 값이라는 뜻입니다. IQR은 가운데 50% 값의 범위를 이용해 너무 튀는 값을 찾는 방법입니다.

```python
q1 = cleaned["qty"].quantile(0.25)
q3 = cleaned["qty"].quantile(0.75)
iqr = q3 - q1
lower = q1 - 1.5 * iqr
upper = q3 + 1.5 * iqr

print(q1, q3, iqr, lower, upper)

qty_outlier = ~cleaned["qty"].between(lower, upper)
cleaned.loc[qty_outlier, ["order_id", "date", "store", "item", "qty", "amount"]]
```

기대 결과입니다.

- Q1: 1.0
- Q3: 2.0
- IQR: 1.0
- 정상범위: -0.5 ~ 3.5
- `qty=999`인 2행이 이상치로 잡힙니다.

수량 999는 정상범위 -0.5 ~ 3.5에서 크게 벗어납니다. 카페에서 한 주문에 999개를 산 것으로 보기 어렵기 때문에 이번 실습에서는 제거합니다.

이상치와 음수 금액을 제거합니다. 이 단계는 수량과 매출 합계가 비현실적인 값에 끌려가지 않게 하기 위해 필요합니다.

```python
cleaned = cleaned.loc[~qty_outlier].copy()
cleaned = cleaned.loc[cleaned["amount"] > 0].copy()

for col in ["order_id", "hour", "unit_price", "qty", "amount"]:
    cleaned[col] = cleaned[col].astype("int64")

cleaned["expected_amount"] = cleaned["unit_price"] * cleaned["qty"]
cleaned["date_month"] = cleaned["date"].dt.to_period("M").astype("string")
```

`expected_amount`는 단가와 수량을 곱해 다시 계산한 금액입니다. 영수증 금액을 한 번 더 검산하는 것과 같습니다. `date_month`는 월별로 묶기 위해 만든 값입니다.

## 따라하기 7: cp949 파일을 encoding 옵션으로 읽기

인코딩은 글자를 저장하는 약속입니다. 같은 한글이라도 어떤 약속으로 저장했는지에 따라 Python이 읽지 못할 수 있습니다. 이번에는 일부러 cp949 방식의 한글 파일을 읽어 보며 오류와 해결 방법을 확인합니다.

먼저 기본값으로 읽어 봅니다. 이 단계는 일부 파일이 왜 바로 열리지 않는지 직접 보기 위해 필요합니다.

```python
cp949_path = root / "02_datasets/real_korean/delivery_food/food_delivery_prediction_raw_data2_ORIGINAL_cp949.csv"

try:
    pd.read_csv(cp949_path)

    print(type(e).__name__)
```

기대 결과입니다.

```text
UnicodeDecodeError
```

`UnicodeDecodeError`는 글자를 해석하는 방식이 맞지 않아서 생기는 오류입니다. 데이터가 망가졌다는 뜻이 아니라, 읽을 때 맞는 인코딩을 알려 달라는 신호로 보면 됩니다.

이제 인코딩을 지정해 다시 읽습니다. cp949로 저장된 파일은 `encoding="cp949"`를 알려 주면 정상적으로 열립니다.

```python
delivery = pd.read_csv(cp949_path, encoding="cp949")

print(delivery.shape)
print(delivery.head())
print(delivery.columns)
```

기대 결과입니다.

- 데이터 크기: 1,947행 x 15열
- 첫 열: `일자`
- 한글 컬럼이 정상적으로 보입니다.

날짜를 datetime으로 바꿉니다. 날짜가 문자열로 남아 있으면 기간 검색이나 월별 집계가 불편합니다.

```python
delivery["일자_dt"] = pd.to_datetime(delivery["일자"], errors="coerce")

print(delivery["일자_dt"].isna().sum())
print(delivery["일자_dt"].min())
print(delivery["일자_dt"].max())
```

기대 결과입니다.

- 날짜 변환 실패: 0건
- 기간: 2013-12-01 ~ 2019-03-31

이제 `일자_dt`를 기준으로 기간을 자르거나 월별로 묶을 수 있습니다. 날짜를 날짜 타입으로 바꿔 두면 사전에서 단어를 찾듯이 원하는 기간을 정확히 찾을 수 있습니다.
