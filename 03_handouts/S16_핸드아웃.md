# S16 핸드아웃: 데이터 모델링

## 실습 목표

이번 실습에서는 `shop_orders`의 세 표를 Power BI에서 관계로 연결한다. 마지막에는 날짜 차원 테이블을 만들고, 고객 차원의 지역 필드로 매출을 분해한다.

## 사용할 파일

기준 데이터는 S17과 같은 `shop_orders` 세 파일이다.

```text
02_datasets/shop_orders/orders.csv
02_datasets/shop_orders/customers.csv
02_datasets/shop_orders/products.csv
```

Power BI Desktop 실습에서는 같은 세 표가 들어 있는 Excel 통합 문서를 사용한다. Windows에서는 아래 파일 경로를 사용한다.

```text
D:\ai-data-analysis-course\02_datasets\shop_orders\shop_orders_clean.xlsx
```

같은 파일의 작업 루트 기준 경로는 다음과 같다.

```text
/mnt/d/ai-data-analysis-course/02_datasets/shop_orders/shop_orders_clean.xlsx
```

> 안내: Mac 수강생은 Power BI 대신 Looker Studio로 개념만 학습한다. 스타스키마와 시간지능은 1:1로 대체되지 않는다.

파일에는 세 개의 시트가 있다.

| 시트 | 행 수 | 역할 |
|---|---:|---|
| `orders` | 20,000 | 주문 내역, 팩트 테이블 |
| `customers` | 3,000 | 고객 정보, 차원 테이블 |
| `products` | 200 | 상품 정보, 차원 테이블 |

## 따라하기

### 1단계. Power BI에 세 표 불러오기

1. Power BI Desktop을 연다.
2. 홈 탭에서 데이터 가져오기 또는 Get data를 클릭한다.
3. Excel 통합 문서를 선택한다.
4. 아래 파일을 연다.

```text
D:\ai-data-analysis-course\02_datasets\shop_orders\shop_orders_clean.xlsx
```

5. 탐색기 창에서 다음 시트를 모두 선택한다.
   - `orders`
   - `customers`
   - `products`
6. 데이터 변환을 클릭한다.

### 2단계. 데이터 형식 확인하기

Power Query 편집기에서 각 표의 데이터 형식을 확인한다.

`orders`:

| 컬럼 | 권장 형식 |
|---|---|
| `order_id` | 텍스트 |
| `customer_id` | 텍스트 |
| `product_id` | 텍스트 |
| `order_date` | 날짜 |
| `qty` | 정수 |
| `discount_rate` | 소수 |

`customers`:

| 컬럼 | 권장 형식 |
|---|---|
| `customer_id` | 텍스트 |
| `name` | 텍스트 |
| `gender` | 텍스트 |
| `age` | 정수 |
| `region` | 텍스트 |
| `join_date` | 날짜 |
| `grade` | 텍스트 |

`products`:

| 컬럼 | 권장 형식 |
|---|---|
| `product_id` | 텍스트 |
| `product_name` | 텍스트 |
| `category` | 텍스트 |
| `price` | 정수 |
| `cost` | 정수 |

확인이 끝나면 닫기 및 적용을 클릭한다.

### 3단계. 모델 보기에서 표 역할 확인하기

왼쪽의 모델 보기를 클릭한다.

오늘 만들 모델은 다음 구조다.

```text
customers      1 -> *  orders  * <- 1  products
                             *
                             ^
                             1
                       날짜테이블
```

읽는 방법은 다음과 같다.

- `orders`: 주문이 한 줄씩 쌓인 팩트 테이블
- `customers`: 고객번호별 지역과 등급을 가진 고객 차원
- `products`: 상품번호별 카테고리와 단가를 가진 상품 차원
- `날짜테이블`: 주문일을 연월로 묶기 위한 날짜 차원

### 4단계. 고객-주문 관계 만들기

자동으로 관계가 만들어져 있으면 설정만 확인한다. 없으면 직접 만든다.

1. 모델 보기에서 `customers[customer_id]`를 `orders[customer_id]`로 끌어 놓는다.
2. 관계 설정 창에서 다음을 확인한다.

| 항목 | 값 |
|---|---|
| 카디널리티 | 일대다 또는 1:* |
| 1쪽 | `customers` |
| 다쪽 | `orders` |
| 교차 필터 방향 | 단일 |
| 이 관계를 활성화 | 체크 |

확인을 클릭한다.

### 5단계. 상품-주문 관계 만들기

1. `products[product_id]`를 `orders[product_id]`로 끌어 놓는다.
2. 관계 설정 창에서 다음을 확인한다.

| 항목 | 값 |
|---|---|
| 카디널리티 | 일대다 또는 1:* |
| 1쪽 | `products` |
| 다쪽 | `orders` |
| 교차 필터 방향 | 단일 |
| 이 관계를 활성화 | 체크 |

확인을 클릭한다.

### 6단계. 날짜 차원 테이블 만들기

리본 메뉴에서 모델링 또는 테이블 도구의 새 테이블을 클릭한다. 수식 입력줄에 아래 DAX를 그대로 입력한다.

```DAX
날짜테이블 =
ADDCOLUMNS(
    CALENDAR(MIN(orders[order_date]), MAX(orders[order_date])),
    "연도", YEAR([Date]),
    "월번호", MONTH([Date]),
    "연월", FORMAT([Date], "yyyy-MM"),
    "연월정렬", YEAR([Date]) * 100 + MONTH([Date])
)
```

확인할 값:

- `날짜테이블` 행 수: 366
- 첫 날짜: 2024-01-01
- 마지막 날짜: 2024-12-31

### 7단계. 날짜 테이블로 지정하기

1. 오른쪽 필드 창에서 `날짜테이블`을 선택한다.
2. 테이블 도구에서 날짜 테이블로 표시 또는 Mark as date table을 클릭한다.
3. 날짜 컬럼으로 `날짜테이블[Date]`를 선택한다.
4. 확인을 클릭한다.

날짜 테이블은 날짜가 빠짐없이 이어져 있어야 하고, 날짜 컬럼에 중복과 결측이 없어야 한다. 이 실습의 `날짜테이블`은 2024년 모든 날짜를 하루 단위로 가지고 있으므로 조건을 만족한다.

### 8단계. 날짜-주문 관계 만들기

모델 보기에서 `날짜테이블[Date]`를 `orders[order_date]`로 끌어 놓는다.

| 항목 | 값 |
|---|---|
| 카디널리티 | 일대다 또는 1:* |
| 1쪽 | `날짜테이블` |
| 다쪽 | `orders` |
| 교차 필터 방향 | 단일 |
| 이 관계를 활성화 | 체크 |

확인을 클릭한다.

### 9단계. 정렬 열 지정하기

연월은 글자로 되어 있어서 정렬 기준을 따로 지정해 두는 편이 안전하다.

1. 데이터 보기에서 `날짜테이블`을 연다.
2. `연월` 컬럼을 선택한다.
3. 열 도구에서 열 기준 정렬을 클릭하고 `연월정렬`을 선택한다.

### 10단계. 매출 측정값 만들기

`orders` 테이블을 선택하고 새 측정값을 만든다. 아래 DAX를 그대로 입력한다.

```DAX
Total Sales =
SUMX(
    orders,
    orders[qty] * RELATED(products[price]) * (1 - orders[discount_rate])
)
```

이 측정값은 주문 한 줄마다 다음 계산을 한 뒤 모두 더한다.

```text
상품 단가 × 수량 × (1 - 할인율)
```

검산용 측정값도 만든다.

```DAX
Order Count =
DISTINCTCOUNT(orders[order_id])
```

```DAX
Avg Order Value =
DIVIDE([Total Sales], [Order Count])
```

### 11단계. 카드로 전체 값 검산하기

보고서 보기로 이동한다.

1. 카드 시각화를 추가한다.
2. 값에 `[Total Sales]`를 넣는다.
3. 다른 카드에 `[Order Count]`를 넣는다.

검산 기준:

| 항목 | 값 |
|---|---:|
| `Order Count` | 20,000 |
| `Total Sales` | 1,938,489,803 |

값이 다르면 관계가 빠졌거나 `Total Sales` 수식의 `RELATED(products[price])` 부분이 제대로 작동하지 않은 것이다.

### 12단계. 지역별 매출 표 만들기

1. 테이블 시각화를 추가한다.
2. 필드를 다음 순서로 넣는다.

| 위치 | 필드 |
|---|---|
| 첫 번째 열 | `customers[region]` |
| 두 번째 열 | `[Order Count]` |
| 세 번째 열 | `[Total Sales]` |
| 네 번째 열 | `[Avg Order Value]` |

3. `[Total Sales]` 기준 내림차순으로 정렬한다.

검산 기준:

| region | Order Count | Total Sales |
|---|---:|---:|
| 서울특별시 | 5,711 | 570,482,124 |
| 경기도 | 5,442 | 530,726,511 |
| 부산광역시 | 1,268 | 132,704,477 |

여기서 눈여겨볼 대목은 `orders`에 `region` 컬럼이 없다는 사실이다. 지역은 `customers`에서 가져오지만, 매출은 관계를 따라 `orders` 기준으로 계산된다.

## 직접 해보기

### 미션. 지역 차원으로 매출 분해하기

아래 조건을 만족하는 보고서 페이지를 만든다.

1. 막대 차트를 추가한다.
2. 축에는 `customers[region]`을 넣는다.
3. 값에는 `[Total Sales]`를 넣는다.
4. 도구 설명에는 `[Order Count]`, `[Avg Order Value]`를 넣는다.
5. 슬라이서를 추가하고 `날짜테이블[연월]`을 넣는다.
6. 선택 사항으로 범례에 `products[category]`를 넣어 지역별 매출을 상품 카테고리로 더 나눈다.

제출할 내용:

- 전체 매출
- 매출 1위 지역과 금액
- 매출 2위 지역과 금액
- 지역이 몇 개로 나뉘었는지
- `날짜테이블[연월]` 슬라이서가 지역별 매출을 바꾸는지 여부

통과 기준:

- 전체 매출이 1,938,489,803으로 나온다.
- 지역은 17개로 나온다.
- 매출 1위는 서울특별시, 2위는 경기도로 나온다.
- 슬라이서의 월을 바꾸면 지역별 매출 값이 함께 바뀐다.
