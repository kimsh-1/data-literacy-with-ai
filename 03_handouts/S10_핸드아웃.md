## 따라하기 1: Colab 노트북 만들기

Colab은 브라우저에서 쓰는 파이썬 작업장입니다. 프로그램을 따로 설치하지 않아도 코드 셀에 파이썬 코드를 쓰고 바로 실행할 수 있습니다.

노트북 이름을 바꿔 두면 나중에 구글 드라이브에서 찾기 쉽습니다. 수업 중 만든 파일이 여러 개가 되면 이름이 곧 라벨 역할을 합니다.

### 2단계. 첫 코드 실행하기

이 단계는 Colab에서 코드가 어떻게 실행되는지 확인하는 과정입니다. 아직 계산을 하려는 것이 아니라, “내가 쓴 명령이 셀 아래에 결과로 나타난다”는 감각을 잡아 봅니다.

첫 번째 코드 셀에 아래 코드를 입력한다.

```python
print("안녕하세요. 파이썬을 시작합니다.")
```

실행 방법은 둘 중 하나를 사용한다.

- 코드 셀 왼쪽의 실행 버튼을 누른다.
- 키보드에서 `Shift + Enter`를 누른다.

셀 아래에 아래 문장이 나오면 정상이다.

```text
안녕하세요. 파이썬을 시작합니다.
```

`print()`는 괄호 안의 값을 화면에 보여 주는 함수입니다. 엑셀에서 셀에 결과가 보이듯이, Colab에서는 실행 결과가 코드 셀 아래에 보입니다.

## 따라하기 2: 변수와 자료형 확인하기

이 단계는 값에 이름을 붙이고, 그 값이 어떤 종류인지 확인하는 과정입니다. 엑셀에서 `B2` 셀에 매출이 들어 있듯이, 파이썬에서는 사람이 읽기 쉬운 이름에 값을 담아 둡니다.

새 코드 셀을 만들고 아래 코드를 입력한다.

```python
store_name = "강남점"
today_sales = 170000
discount_rate = 0.1
is_weekend = False

print(store_name)
print(today_sales)
print(discount_rate)
print(is_weekend)
```

실행하면 변수에 담긴 값이 순서대로 출력된다.

여기서 변수는 값을 다시 쓰기 위해 붙이는 이름입니다. `store_name`이라는 이름에는 `"강남점"`이 들어 있고, `today_sales`라는 이름에는 `170000`이 들어 있습니다. 등호 `=`는 수학의 “같다”가 아니라 “오른쪽 값을 왼쪽 이름에 담는다”에 가깝습니다.

이어서 새 코드 셀에 아래 코드를 입력한다.

```python
print(type(store_name))
print(type(today_sales))
print(type(discount_rate))
print(type(is_weekend))
```

출력에서 확인할 것:

- `str`: 글자
- `int`: 정수
- `float`: 소수점이 있는 숫자
- `bool`: 참 또는 거짓

자료형은 값의 종류입니다. 엑셀에서도 숫자처럼 보이는 글자와 진짜 숫자가 다르게 동작하죠? 파이썬도 마찬가지입니다. 계산이 안 될 때는 “이 값이 정말 숫자인가?”를 먼저 확인해 보면 좋습니다.

## 따라하기 3: 리스트로 매출 여러 개 담기

이 단계는 여러 값을 하나의 묶음으로 담아 보는 과정입니다. 엑셀에서 한 열에 월요일부터 금요일까지 매출이 세로로 쌓여 있는 모습을 떠올리면 쉽습니다.

새 코드 셀에 아래 코드를 입력한다.

```python
sales_list = [120000, 98000, 135000, 170000, 155000]

print(sales_list)
print(len(sales_list))
```

`sales_list`는 월요일부터 금요일까지 매출 5개를 담은 리스트라고 보면 된다. `len()`은 값이 몇 개인지 세는 함수다.

여기서 `len(sales_list)`의 결과는 리스트 안에 들어 있는 값의 개수입니다. 사람 수를 세듯이, 데이터가 몇 개 들어 있는지 세는 도구라고 보면 됩니다.

## 따라하기 4: 반복문으로 매출 합계 구하기

이 단계는 리스트 안의 값을 하나씩 꺼내 더하는 과정입니다. 엑셀에서는 `SUM` 함수가 한 번에 합계를 내지만, 여기서는 파이썬이 값을 어떤 순서로 처리하는지 보기 위해 일부러 직접 더해 봅니다.

이번에는 `sum()` 함수 대신 반복문으로 하나씩 더해 본다. 새 코드 셀에 아래 코드를 입력한다.

```python
total_sales = 0

for amount in sales_list:
    total_sales = total_sales + amount

print(total_sales)
```

기대 결과:

```text
678000
```

코드를 읽는 순서는 다음과 같다.

1. `total_sales = 0`으로 합계를 담을 변수를 만든다.
2. `sales_list`에서 값을 하나씩 꺼내 `amount`라고 부른다.
3. 꺼낸 값을 `total_sales`에 계속 더한다.
4. 반복이 끝나면 `total_sales`에는 전체 합계가 남는다.

`total_sales = total_sales + amount`는 처음 보면 이상해 보일 수 있습니다. 오른쪽의 기존 합계에 이번 매출을 더한 뒤, 그 결과를 다시 `total_sales`에 담는다는 뜻입니다. 장바구니에 물건 가격을 하나씩 더해 가는 모습과 비슷합니다.

주의할 점:

- `for` 줄 끝에는 콜론 `:`을 붙인다.
- 반복할 코드는 반드시 들여쓰기한다.
- 들여쓰기는 보통 스페이스 4칸이다.

파이썬에서 들여쓰기는 보기 좋으라고 넣는 장식이 아닙니다. 어떤 코드가 반복문 안에 들어가는지 알려 주는 문법입니다.

## 따라하기 5: 평균 구하기

이 단계는 앞에서 만든 합계와 개수를 이용해 평균을 계산하는 과정입니다. 평균은 “전체를 몇 개로 나눈 값”입니다. 키 평균을 구할 때 전체 키 합계를 사람 수로 나누는 것과 같습니다.

새 코드 셀에 아래 코드를 입력한다.

```python
average_sales = total_sales / len(sales_list)

print(average_sales)
```

기대 결과:

```text
135600.0
```

엑셀의 `AVERAGE` 함수가 속으로 하던 일을 손으로 옮겨 쓴 셈이다.

```text
평균 = 합계 / 개수
```

결과가 `135600.0`처럼 소수점이 붙어 나오는 이유는 나눗셈 결과를 파이썬이 소수점이 있는 숫자로 다루기 때문입니다. 값 자체는 평일 5일 매출의 평균을 뜻합니다.

## 따라하기 6: 딕셔너리로 한 행 표현하기

이 단계는 이름표가 붙은 값을 하나의 묶음으로 담아 보는 과정입니다. 리스트가 한 열처럼 여러 값을 담는다면, 딕셔너리는 엑셀 표의 한 행처럼 “요일은 월, 매출은 120000”처럼 열 이름과 값을 함께 담습니다.

새 코드 셀에 아래 코드를 입력한다.

```python
one_day = {
    "요일": "월",
    "매출": 120000,
}

print(one_day)
print(one_day["요일"])
print(one_day["매출"])
```

딕셔너리는 이름표를 붙여 값을 담는다. 엑셀 표의 한 행을 떠올리면 쉽다.

| 요일 | 매출 |
|---|---:|
| 월 | 120000 |

`one_day["요일"]`은 딕셔너리에서 `"요일"`이라는 이름표가 붙은 값을 꺼내는 코드입니다. 사전에서 단어를 찾듯이, 키를 넣으면 그 키에 해당하는 값을 찾습니다.

## 따라하기 7: 조건문으로 금요일만 확인하기

이 단계는 조건에 맞을 때만 코드를 실행하는 연습입니다. 엑셀 필터에서 요일이 금요일인 행만 보이게 하는 것과 비슷합니다.

새 코드 셀에 아래 코드를 입력한다.

```python
day = "금"

if day == "금":
    print("금요일 데이터입니다.")
```

`=`와 `==`를 구분한다.

- `=`: 값을 변수에 담는다.
- `==`: 두 값이 같은지 비교한다.

`if day == "금":`은 “day에 들어 있는 값이 금과 같다면”이라는 뜻입니다. 조건이 맞으면 들여쓰기된 줄이 실행되고, 조건이 맞지 않으면 그 줄은 건너뜁니다.

## 따라하기 8: 리스트와 딕셔너리로 작은 표 만들기

이 단계는 여러 행을 가진 작은 표를 파이썬 코드로 표현하는 과정입니다. 딕셔너리 하나가 한 행이라면, 딕셔너리를 여러 개 담은 리스트는 여러 행이 모인 표처럼 볼 수 있습니다.

새 코드 셀에 아래 코드를 입력한다.

```python
daily_sales = [
    {"요일": "월", "매출": 120000},
    {"요일": "화", "매출": 98000},
    {"요일": "수", "매출": 135000},
    {"요일": "목", "매출": 170000},
    {"요일": "금", "매출": 155000},
    {"요일": "토", "매출": 210000},
    {"요일": "일", "매출": 190000},
]

for row in daily_sales:
    print(row["요일"], row["매출"])
```

요일과 매출이 한 줄씩 출력된다. 여기서 `row`는 반복문이 지금 보고 있는 한 행이다.

반복문은 `daily_sales` 안의 행을 위에서부터 하나씩 꺼냅니다. 첫 번째 반복에서 `row`는 월요일 행이고, 두 번째 반복에서 `row`는 화요일 행입니다. 이렇게 한 행씩 보면서 필요한 값만 꺼내면, 나중에 더 큰 표도 같은 방식으로 다룰 수 있습니다.

## 직접 해보기: 평일 데이터만 필터링하기

이 실습은 조건문과 반복문을 함께 쓰는 연습입니다. 표에서 평일 행만 골라낸 뒤, 그 행들의 매출 합계와 평균을 계산해 봅니다.

아래 조건을 만족하는 코드를 작성한다.

목표:

1. `daily_sales`에서 평일인 `월`, `화`, `수`, `목`, `금` 행만 고른다.
2. 고른 행을 `weekday_rows` 리스트에 담는다.
3. 평일 매출 합계와 평균을 계산한다.
4. 평일 행수, 평일 합계, 평일 평균을 출력한다.

힌트 코드:

```python
weekday_names = ["월", "화", "수", "목", "금"]
weekday_rows = []
weekday_total = 0

for row in daily_sales:
    if row["요일"] in weekday_names:
        weekday_rows.append(row)
        weekday_total = weekday_total + row["매출"]

weekday_average = weekday_total / len(weekday_rows)

print(weekday_rows)
print(weekday_total)
print(weekday_average)
```

기대 결과:

```text
평일 행수: 5
평일 매출 합계: 678000
평일 매출 평균: 135600.0
```

여기서 `weekday_names`는 평일 이름을 모아 둔 기준표입니다. `row["요일"] in weekday_names`는 지금 보고 있는 행의 요일이 그 기준표 안에 있는지 확인합니다. 조건에 맞는 행은 `weekday_rows`에 담고, 그 행의 매출은 `weekday_total`에 더합니다.

기대 결과처럼 보이게 하려면 어떤 값을 출력해야 하는지도 함께 생각해 보세요. 평일 행수는 `weekday_rows`에 담긴 행의 개수이고, 평일 매출 합계는 `weekday_total`, 평일 매출 평균은 `weekday_average`입니다.

## 오류가 났을 때 먼저 볼 것

오류가 나면 빨간 메시지의 맨 아래 줄부터 읽는다.

처음에는 빨간 글씨가 나오면 당황할 수 있습니다. 하지만 오류는 실패 표시가 아니라 파이썬이 “여기부터 다시 보세요”라고 짚어 주는 안내문에 가깝습니다. 특히 맨 아래 줄에는 오류 이름이 나오는 경우가 많습니다.

자주 만나는 오류:

| 오류 | 흔한 원인 | 확인할 것 |
|---|---|---|
| `NameError` | 변수를 만들기 전에 사용함 | 위 셀을 실행했는가 |
| `SyntaxError` | 문법이 틀림 | 콜론, 따옴표, 괄호가 빠졌는가 |
| `IndentationError` | 들여쓰기가 틀림 | `if`, `for` 아래 줄이 들여쓰기 되었는가 |
| `TypeError` | 자료형이 맞지 않음 | 숫자와 글자를 섞어 계산하지 않았는가 |

오류는 실패가 아니라 파이썬이 어느 줄을 다시 보라고 짚어 주는 신호다.
