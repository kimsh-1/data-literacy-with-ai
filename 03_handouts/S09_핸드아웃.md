# S09 핸드아웃: AI 답 검증하기

## 사용할 파일

수업 홈의 데이터 받기 페이지에서 카페 매출 파일 두 개를 내려받아 사용합니다. 데이터 받기 페이지에 CSV가 함께 보인다면, CSV는 엑셀로도 열리는 표 파일입니다. 처음 실습하는 분은 엑셀 파일(.xlsx)을 받아서 따라 해도 됩니다.

1. clean 실습 파일
   - 카페 매출 clean 파일
2. dirty 실습 파일
   - 카페 매출 dirty 파일

AI 도구의 파일 업로드 기능은 계정이나 요금제에 따라 다를 수 있습니다. 파일을 올릴 수 없으면 AI에는 프롬프트와 검증 체크리스트 작성만 맡기고, 실제 계산은 엑셀 피벗테이블로 해 보세요.

## 오늘의 목표

AI가 데이터 파일을 분석해 준 답을 그대로 믿지 않고, 엑셀 피벗테이블로 다시 계산해서 맞는지 확인합니다. 숫자는 말로 들으면 그럴듯해 보이지만, 업무에서는 한 번 더 재는 과정이 필요합니다. 키를 눈대중으로 보지 않고 줄자로 재듯이, AI 답도 피벗테이블로 다시 재 봅니다.

검증할 질문은 네 가지입니다.

1. 전체 행수와 기간은 맞는가?
2. 총매출은 맞는가?
3. 지점별, 카테고리별 매출은 맞는가?
4. dirty 데이터에서는 중복, 결측, 표기 흔들림, 이상값을 확인했는가?

## 따라하기 1. clean 파일에서 AI 답과 피벗을 대조하기

### 1단계. AI에게 clean 파일 분석 요청하기

이 단계는 AI가 어떤 숫자를 내놓는지 먼저 받아 두기 위해 필요합니다. 단, 여기서 받은 숫자는 아직 정답이 아니라 검토할 후보라고 생각합니다.

AI 도구에 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 clean 파일을 업로드합니다.

그다음 아래 프롬프트를 그대로 입력합니다.

```text
첨부한 cafe_sales_clean.xlsx의 sales 시트를 실제로 계산해서 분석해 주세요.

다음 항목을 표로 답해 주세요.
1. 전체 데이터 행수와 날짜 기간
2. amount 열의 총합계
3. store별 amount 합계
4. category별 amount 합계
5. 월별 amount 합계

반드시 계산에 사용한 열 이름을 함께 적어 주세요.
값을 추정하지 말고, 확인하지 못한 값은 "확인 필요"라고 적어 주세요.
```

AI 답에서 숫자만 따로 메모해 보세요. 이 숫자는 아직 확정값이 아닙니다. 이제 엑셀로 같은 질문을 다시 계산해 볼 겁니다.

### 2단계. 엑셀에서 clean 파일 열기

이 단계는 AI가 분석했다는 데이터와 내가 엑셀에서 확인하는 데이터가 같은지 맞춰 보기 위해 필요합니다. 시작 파일이 다르면 뒤의 모든 숫자가 달라집니다.

1. 엑셀을 여세요.
2. `파일` > `열기`를 누르세요.
3. 데이터 받기 페이지에서 내려받은 카페 매출 clean 파일을 여세요.
4. 시트 이름이 `sales`인지 확인하세요.
5. 행수는 머리글을 제외하고 10,424행입니다.

### 3단계. 총매출 피벗 만들기

이 단계는 전체 매출이라는 가장 큰 숫자를 먼저 확인하기 위해 필요합니다. 총매출이 맞지 않으면 지점별, 카테고리별 숫자도 믿기 어렵습니다.

1. `sales` 시트의 데이터 안 아무 셀이나 클릭하세요.
2. `삽입` > `피벗 테이블`을 클릭하세요.
3. 위치는 `새 워크시트`를 선택하고 `확인`을 누르세요.
4. 피벗 필드에서 `amount`를 `값` 영역에 넣으세요.
5. 값 이름이 `합계: amount`인지 확인하세요.

기준값:

| 항목 | 피벗 기준값 |
|---|---:|
| 총매출 | 90,492,500원 |

AI가 이 값과 다르게 답했다면 아직 통과가 아닙니다. 같은 데이터를 봤는지, `amount` 열을 합계로 더했는지 다시 확인해야 합니다.

### 4단계. 지점별 매출 검산하기

이 단계는 총매출이 어느 지점에서 나왔는지 나누어 확인하기 위해 필요합니다. 전체 합계가 맞아도, 지점별로 나누는 과정에서 AI가 잘못 묶을 수 있습니다.

같은 피벗테이블에서 필드를 아래처럼 배치합니다.

| 필드 | 놓을 영역 |
|---|---|
| `store` | 행 |
| `amount` | 값 |

기준값:

| 지점 | 매출 |
|---|---:|
| 강남점 | 53,425,000원 |
| 홍대점 | 37,067,500원 |
| 총합계 | 90,492,500원 |

지점별 매출의 합계가 총매출과 같아야 합니다. 강남점과 홍대점 매출을 더했을 때 총합계가 맞는지 꼭 확인해 보세요.

### 5단계. 카테고리별 매출 검산하기

이 단계는 상품 종류별 매출을 AI가 제대로 나누었는지 확인하기 위해 필요합니다. 카테고리는 사전에서 단어를 찾듯이 같은 글자끼리 묶이기 때문에, 이름이 조금만 달라도 다른 항목으로 잡힐 수 있습니다.

새 피벗테이블을 만들거나, 기존 피벗의 행 영역을 `category`로 바꿉니다.

| 필드 | 놓을 영역 |
|---|---|
| `category` | 행 |
| `amount` | 값 |

기준값:

| 카테고리 | 매출 |
|---|---:|
| 커피 | 45,606,000원 |
| 음료 | 25,550,000원 |
| 디저트 | 19,336,500원 |
| 총합계 | 90,492,500원 |

### 6단계. 월별 매출 검산하기

이 단계는 날짜를 월 단위로 묶어서 매출 흐름을 확인하기 위해 필요합니다. 하루하루의 날짜를 그대로 보면 너무 잘게 나뉘기 때문에, 월만 뽑아 새 기준을 만듭니다.

월별 매출은 날짜에서 월만 뽑아야 합니다.

1. `sales` 시트로 돌아갑니다.
2. 마지막 열 오른쪽에 새 열을 만들고 머리글을 `month`로 입력합니다.
3. 첫 데이터 행에 아래 수식을 입력합니다.

```excel
=LEFT(B2,7)
```

4. 마지막 행까지 채웁니다.
5. 새 피벗테이블을 만듭니다.
6. 필드를 아래처럼 배치합니다.

| 필드 | 놓을 영역 |
|---|---|
| `month` | 행 |
| `amount` | 값 |

기준값:

| 월 | 매출 |
|---|---:|
| 2025-01 | 14,426,500원 |
| 2025-02 | 13,157,000원 |
| 2025-03 | 14,735,500원 |
| 2025-04 | 15,263,000원 |
| 2025-05 | 16,094,000원 |
| 2025-06 | 16,816,500원 |
| 총합계 | 90,492,500원 |

### 7단계. AI 답과 피벗 기준값 비교표 작성하기

이 단계는 AI 답을 말로만 확인하지 않고, 내 기준값과 나란히 놓고 판단하기 위해 필요합니다. 숫자를 표에 넣으면 어디가 맞고 어디가 다른지 눈에 바로 보입니다.

아래 표를 채웁니다.

| 점검 항목 | AI 답 | 피벗 기준값 | 판정 |
|---|---:|---:|---|
| 행수 |  | 10,424행 |  |
| 총매출 |  | 90,492,500원 |  |
| 강남점 매출 |  | 53,425,000원 |  |
| 홍대점 매출 |  | 37,067,500원 |  |
| 커피 매출 |  | 45,606,000원 |  |
| 2025-06 매출 |  | 16,816,500원 |  |

판정은 `일치`, `불일치`, `확인 필요` 중 하나로 씁니다.

AI 답이 다르면 아래처럼 다시 질문합니다.

```text
내 엑셀 피벗 기준으로 amount 합계는 90,492,500원입니다.
방금 답과 차이가 납니다.
amount 열만 합산했는지, 필터를 적용했는지, 분석한 행수가 10,424행인지 확인해서 다시 계산해 주세요.
```

이렇게 물으면 AI에게 막연히 다시 하라고 하는 것이 아니라, 어느 열을 썼는지, 필터가 있었는지, 행수가 맞는지 확인하게 만드는 질문이 됩니다.

## 따라하기 2. dirty 파일에서 같은 검증을 해 보기

### 1단계. AI에게 dirty 파일 분석 요청하기

이 단계는 데이터가 지저분할 때 AI가 숫자만 말하는지, 품질 문제까지 확인하는지 보기 위해 필요합니다. dirty 데이터에서는 총매출 하나만 맞아도 충분하지 않습니다.

AI 도구에 수업 홈의 데이터 받기 페이지에서 내려받은 카페 매출 dirty 파일을 업로드합니다.

아래 프롬프트를 입력합니다.

```text
첨부한 cafe_sales_dirty.xlsx의 sales 시트를 분석해 주세요.
이 파일은 일부러 지저분한 데이터일 수 있습니다.

총매출을 말하기 전에 아래 품질 점검을 먼저 해 주세요.
1. 전체 행수
2. order_id 중복
3. payment와 member 결측
4. 날짜 형식 혼재
5. store와 category 표기 흔들림 또는 앞뒤 공백
6. qty 이상값
7. amount 음수
8. unit_price × qty와 amount 불일치

그다음 원본을 그대로 합산한 amount 총합계와, 이 숫자를 최종 결론으로 써도 되는지에 대한 판단을 적어 주세요.
```

### 2단계. dirty 파일을 엑셀에서 열기

이 단계는 AI가 본 dirty 데이터와 내가 검산할 dirty 데이터가 같은지 확인하기 위해 필요합니다. clean 파일과 행수가 다르기 때문에 처음부터 구분해야 합니다.

1. 엑셀에서 데이터 받기 페이지에서 내려받은 카페 매출 dirty 파일을 여세요.
2. 시트 이름이 `sales`인지 확인하세요.
3. 행수는 머리글을 제외하고 10,439행입니다.

### 3단계. 원본 그대로 총매출 확인하기

이 단계는 정제하지 않은 상태에서 단순히 더하면 얼마가 나오는지 확인하기 위해 필요합니다. 여기서 구한 값은 최종 매출이 아니라, 원본을 그대로 합산한 값입니다.

피벗테이블을 만들고 `amount`를 `값` 영역에 넣습니다.

기준값:

| 항목 | 원본 그대로 합산한 값 |
|---|---:|
| 총매출 | 90,600,500원 |
| 판매수량 합계 | 19,259개 |

clean의 총매출 90,492,500원과 비교하면 dirty 원본 합계가 108,000원 더 큽니다. 이 차이는 그냥 넘기면 안 됩니다. 왜 더 커졌는지 데이터 안에서 찾아봐야 합니다.

### 4단계. 지점 표기 흔들림 확인하기

이 단계는 같은 지점이 서로 다른 이름으로 들어가 있지는 않은지 확인하기 위해 필요합니다. 엑셀은 사람처럼 뜻을 짐작하지 않고, 글자가 다르면 다른 값으로 봅니다.

피벗 필드를 아래처럼 배치합니다.

| 필드 | 놓을 영역 |
|---|---|
| `store` | 행 |
| `amount` | 값 |

dirty 원본 기준 결과:

| store | 매출 |
|---|---:|
| 강남 점 | 2,618,000원 |
| 강남점 | 50,858,500원 |
| 홍대점 | 37,124,000원 |
| 총합계 | 90,600,500원 |

`강남점`과 `강남 점`은 사람 눈에는 같은 지점처럼 보이지만, 피벗테이블에서는 다른 값으로 잡힙니다. AI도 이 차이를 놓칠 수 있습니다.

### 5단계. 카테고리 앞뒤 공백 확인하기

이 단계는 카테고리 이름 앞뒤에 숨어 있는 공백 때문에 같은 항목이 갈라지는지 확인하기 위해 필요합니다. 눈으로는 비슷해 보여도, 데이터는 글자 하나와 공백 하나까지 따로 봅니다.

피벗 필드를 아래처럼 배치합니다.

| 필드 | 놓을 영역 |
|---|---|
| `category` | 행 |
| `amount` | 값 |

dirty 원본 기준 결과:

| category | 매출 |
|---|---:|
| ` 디저트 ` | 584,500원 |
| ` 음료 ` | 817,000원 |
| ` 커피 ` | 1,288,500원 |
| 디저트 | 18,772,500원 |
| 음료 | 24,746,000원 |
| 커피 | 44,392,000원 |

앞뒤 공백 때문에 카테고리가 3개가 아니라 6개처럼 보입니다.

### 6단계. 이상값 빠르게 찾기

이 단계는 매출을 크게 흔들 수 있는 이상한 값을 눈으로 빠르게 찾기 위해 필요합니다. 수량이 너무 크거나 금액이 음수인 행은 전체 합계를 왜곡할 수 있습니다.

엑셀 필터를 켭니다.

1. `qty` 열 필터에서 큰 값 순으로 정렬합니다.
2. `qty=999`인 행이 2개 있는지 확인합니다.
3. `amount` 열 필터에서 작은 값 순으로 정렬합니다.
4. `amount=-5500`인 행이 1개 있는지 확인합니다.
5. `store` 필터에서 `강남 점`이 따로 있는지 확인합니다.
6. `category` 필터에서 앞뒤 공백이 있는 항목이 따로 보이는지 확인합니다.

## 직접 해보기. AI 답 검증 체크리스트 만들기

이제 직접 판단하는 연습을 해 봅니다. AI가 말한 문장이 그럴듯해 보여도, 보고서에 넣기 전에는 어떤 근거로 통과시킬지 정해야 합니다.

상황:

```text
동료가 AI로 cafe_sales_dirty.xlsx를 분석했고,
"총매출은 90,600,500원이며 6월 매출이 가장 높습니다"라고 보고했다.
이 답을 그대로 보고서에 넣어도 되는지 검토해야 한다.
```

해야 할 일:

1. 검증 체크리스트를 최소 8개 항목으로 만듭니다.
2. 각 항목에 `통과`, `불일치`, `확인 필요` 중 하나를 씁니다.
3. 마지막에 이 AI 답을 보고서에 그대로 써도 되는지 3문장으로 판단합니다.

체크리스트에 반드시 들어갈 항목:

- 파일명과 시트명
- 행수
- 날짜 기간
- 사용한 금액 열
- 총매출
- 그룹별 합계와 총합계 일치 여부
- 주문번호 중복
- 결측
- 지점명과 카테고리 표기 흔들림
- 수량 이상값
- 음수 금액
- `unit_price × qty = amount` 확인

제출 문장 예시:

```text
원본을 그대로 더한 총매출 90,600,500원은 피벗과 일치한다.
하지만 dirty 파일에는 주문번호 중복, category 공백, qty=999, amount 음수 같은 문제가 있다.
따라서 이 숫자는 원본 합계로는 말할 수 있지만, 정제 기준을 정하기 전에는 최종 매출로 보고서에 쓰면 안 된다.
```
