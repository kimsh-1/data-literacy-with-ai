# S03. 피벗테이블로 질문에 답하기

## 사용할 파일

엑셀에서 아래 파일을 연다.

`/mnt/d/ai-data-analysis-course/02_datasets/cafe_sales/cafe_sales_clean.xlsx`

- 시트 이름: `sales`
- 데이터 규모: 10,424행 × 12열
- 이번 차시에서 주로 사용할 컬럼: `store`, `category`, `weekday`, `hour`, `amount`, `qty`, `order_id`

## 오늘의 목표

피벗테이블을 이용해 다음 두 질문에 답한다.

1. 지점별로 어떤 카테고리가 매출을 많이 만드는가?
2. 요일과 시간대에 따라 매출은 어떻게 달라지는가?

마지막에는 직접 "가장 잘 팔리는 시간대"를 찾는다.

## 피벗테이블 기본 구조

피벗테이블 필드 목록에는 보통 네 영역이 있다.

| 영역 | 넣는 것 | 예시 |
|---|---|---|
| 행 | 세로로 나눌 기준 | `store`, `weekday`, `hour` |
| 열 | 가로로 나눌 기준 | `category`, `hour` |
| 값 | 계산할 숫자 | `amount` 합계, `qty` 합계 |
| 필터 | 일부만 볼 조건 | 특정 지점, 특정 카테고리 |

중요한 질문은 "무엇별로 나눌 것인가"와 "무엇을 계산할 것인가"이다.

## 따라하기 1. 지점×카테고리 매출 피벗 만들기

질문: 지점별로 커피, 음료, 디저트 매출이 어떻게 다른가?

### 1단계. 피벗테이블 삽입

1. `sales` 시트에서 데이터 안의 아무 셀이나 클릭한다.
2. 상단 메뉴에서 `삽입`을 클릭한다.
3. `피벗 테이블`을 클릭한다.
4. 표/범위가 자동으로 잡히는지 확인한다.
5. 위치는 `새 워크시트`를 선택한다.
6. `확인`을 클릭한다.

### 2단계. 필드 배치

오른쪽의 피벗테이블 필드 목록에서 다음처럼 끌어다 놓는다.

| 필드 | 놓을 영역 |
|---|---|
| `store` | 행 |
| `category` | 열 |
| `amount` | 값 |

값 영역에 `합계: amount`가 보이면 맞게 설정된 것이다.

### 3단계. 금액 형식 정리

1. 피벗테이블 안의 매출 숫자 중 하나를 오른쪽 클릭한다.
2. `값 필드 설정`을 클릭한다.
3. 집계 방식이 `합계`인지 확인한다.
4. `표시 형식`을 클릭한다.
5. `숫자`를 선택하고 `1000 단위 구분 기호 사용`을 켠다.
6. 소수 자릿수는 `0`으로 설정한다.

### 4단계. 결과 읽기

실제 데이터 기준 결과는 다음과 같다.

| 지점 | 디저트 | 음료 | 커피 | 총합계 |
|---|---:|---:|---:|---:|
| 강남점 | 11,604,000 | 15,281,500 | 26,539,500 | 53,425,000 |
| 홍대점 | 7,732,500 | 10,268,500 | 19,066,500 | 37,067,500 |
| 총합계 | 19,336,500 | 25,550,000 | 45,606,000 | 90,492,500 |

해석 문장 예시는 다음과 같다.

강남점과 홍대점 모두 커피 매출이 가장 크다. 전체 매출은 강남점이 53,425,000원으로 홍대점을 앞선다.

## 따라하기 2. 요일×시간대 매출 피벗 만들기

질문: 요일과 시간대에 따라 매출이 어떻게 달라지는가?

### 1단계. 새 피벗테이블 만들기

1. 다시 `sales` 시트로 이동한다.
2. 데이터 안의 아무 셀이나 클릭한다.
3. `삽입` > `피벗 테이블`을 클릭한다.
4. 위치는 `새 워크시트`를 선택한다.
5. `확인`을 클릭한다.

### 2단계. 필드 배치

| 필드 | 놓을 영역 |
|---|---|
| `weekday` | 행 |
| `hour` | 열 |
| `amount` | 값 |

값 영역은 `합계: amount`로 둔다. `hour` 값은 8부터 21까지 보인다. 여기서는 각 숫자를 8시, 9시, 10시처럼 해석한다.

### 3단계. 숫자 표시 형식 정리

따라하기 1과 같은 방식으로 매출 숫자에 1000 단위 구분 기호를 적용한다.

### 4단계. 교차표 읽기

교차표는 한 칸씩 외우는 표가 아니라 흐름을 읽는 표다. 다음 순서로 본다.

1. 숫자가 큰 시간대가 어느 열에 몰리는지 본다.
2. 특정 요일에서 유난히 큰 값이 있는지 본다.
3. 총합계를 보고 전체 흐름을 확인한다.

실제 데이터에서는 13시 매출이 가장 크다. 요일과 시간대를 함께 보면 일요일 13시가 2,260,500원으로 가장 큰 조합이다.

요일 값은 원본 데이터에 영어 약어로 들어 있다.

| 값 | 의미 |
|---|---|
| Mon | 월요일 |
| Tue | 화요일 |
| Wed | 수요일 |
| Thu | 목요일 |
| Fri | 금요일 |
| Sat | 토요일 |
| Sun | 일요일 |

## 직접 해보기. 가장 잘 팔리는 시간대 찾기

질문: 매출액 합계 기준으로 가장 잘 팔리는 시간대는 언제인가?

### 조건

새 피벗테이블을 만들어 아래처럼 구성한다.

| 필드 | 놓을 영역 |
|---|---|
| `hour` | 행 |
| `amount` | 값 |
| `qty` | 값 |
| `order_id` | 값 |

설정은 다음처럼 맞춘다.

- `amount`: 합계
- `qty`: 합계
- `order_id`: 개수

`amount` 합계를 기준으로 내림차순 정렬한다.

### 제출할 답

아래 형식으로 한 문장을 작성한다.

`매출액 합계 기준 가장 잘 팔리는 시간대는 __시이며, 매출은 ______원, 판매수량은 ______개, 주문건수는 ______건이다.`

### 추가 질문

시간이 남으면 다음도 확인한다.

1. 판매수량 기준 1위 시간대도 같은가?
2. 주문건수 기준 1위 시간대도 같은가?
3. 요일×시간대를 함께 볼 때 가장 큰 조합은 무엇인가?

## 자주 생기는 문제

### 값이 합계가 아니라 개수로 나오는 경우

값 영역의 필드 이름 옆 화살표를 누르고 `값 필드 설정`으로 들어간다. `합계`, `개수`, `평균` 중 원하는 계산 방식을 선택한다.

### 요일 순서가 월, 화, 수 순서가 아닌 경우

이번 차시에서는 정렬 순서보다 값의 크기를 읽는 것이 중요하다. 순서가 달라도 해석에는 지장이 없다. 날짜와 요일 정렬은 이후 차시에서 더 다룬다.

### 피벗테이블 필드 목록이 보이지 않는 경우

피벗테이블 안의 아무 셀이나 클릭한다. 그래도 보이지 않으면 상단의 `피벗 테이블 분석` 메뉴에서 `필드 목록`을 켠다.
