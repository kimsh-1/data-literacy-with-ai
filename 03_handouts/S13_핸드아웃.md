## 따라하기 1: 주문 3개 표 읽고 병합하기

이 단계에서는 주문 내역, 고객 정보, 상품 정보라는 세 개의 표를 먼저 읽습니다. 왜냐하면 주문 표만으로는 고객 지역이나 상품 단가를 알 수 없기 때문입니다.

수업 홈의 데이터 받기 페이지에서 주문 내역 파일, 고객 정보 파일, 상품 정보 파일을 내려받아 준비해 보세요. 새 파이썬 파일이나 Colab 노트북을 열고 아래 코드를 실행합니다.

```python
from pathlib import Path
import pandas as pd

ROOT = Path("/mnt/d/ai-data-analysis-course")
DATA = ROOT / "02_datasets"

orders = pd.read_csv(DATA / "shop_orders/orders.csv")
customers = pd.read_csv(DATA / "shop_orders/customers.csv")
products = pd.read_csv(DATA / "shop_orders/products.csv")

print(orders.shape)
print(customers.shape)
print(products.shape)
print(orders.head())
```

확인할 결과:

```text
orders: 20,000행, 6열
customers: 3,000행, 7열
products: 200행, 5열
```

여기서 `shape`는 표의 크기를 알려 줍니다. 키를 잴 때 키와 몸무게를 먼저 확인하듯이, 데이터 분석에서도 행과 열의 개수를 먼저 봅니다. 주문 표는 20,000행, 고객 표는 3,000행, 상품 표는 200행입니다.

주문 표에는 `customer_id`와 `product_id`만 있고, 고객 지역과 상품 단가는 없습니다. 주문번호만 보고는 고객이 어느 지역 사람인지, 상품 가격이 얼마인지 알 수 없는 상태입니다. 그래서 고객 표와 상품 표를 주문 표에 붙입니다. 엑셀에서 VLOOKUP으로 고객 정보와 상품 정보를 끌어오는 장면과 같습니다.

```python
merged = (
    orders.merge(customers, on="customer_id", how="left")
    .merge(products, on="product_id", how="left")
)

print(merged.shape)
print(merged[["region", "category", "price"]].isna().sum())
```

확인할 결과:

```text
병합 후: 20,000행, 17열
region      0
category    0
price       0
```

병합 후에도 행이 20,000행으로 유지되는지 꼭 봅니다. 주문이 20,000건이었는데 병합 후 행이 갑자기 늘거나 줄면, 표를 붙이는 기준이 잘못되었을 가능성이 있습니다.

`region`, `category`, `price`의 누락값이 모두 0이라는 말은 고객 지역, 상품 카테고리, 상품 단가를 빠짐없이 가져왔다는 뜻입니다. `how="left"`는 주문 표의 20,000행을 기준으로 그대로 유지하겠다는 뜻입니다. 주문 표를 중심에 놓고, 옆에서 필요한 정보를 찾아 붙이는 방식입니다.

## 따라하기 2: 매출 계산하기

이 단계에서는 주문마다 매출을 직접 계산합니다. 주문 파일에는 매출 컬럼이 없고, 매출을 만들 재료만 따로 들어 있습니다.

상품 단가는 상품 표에서 가져온 `price`에 있고, 주문 수량은 `qty`에 있으며, 할인율은 `discount_rate`에 있습니다. 그래서 세 값을 이용해 매출을 계산합니다.

```python
merged["sales"] = merged["price"] * merged["qty"] * (1 - merged["discount_rate"])

print(merged[["order_id", "price", "qty", "discount_rate", "sales"]].head())
print(merged["sales"].sum())
```

확인할 결과:

```text
전체 매출 합계: 1,938,489,803
```

계산식은 아래와 같다.

```text
매출 = 단가 * 수량 * (1 - 할인율)
```

`1 - 할인율`은 실제로 고객이 결제하는 비율입니다. 예를 들어 할인율이 0.1이면 10% 할인이라는 뜻이고, 고객은 원래 가격의 90%를 냅니다. 그래서 단가에 수량을 곱한 뒤 할인 후 비율을 다시 곱합니다.

전체 매출 합계가 1,938,489,803으로 나오면 계산이 잘 된 것입니다. 이 숫자는 20,000건 주문의 매출을 모두 더한 값입니다.

## 따라하기 3: 지역×카테고리별 다중 집계하기

이 단계에서는 20,000건의 주문을 지역과 카테고리 기준으로 요약합니다. 원자료를 한 줄씩 보는 대신, "서울특별시의 디지털 상품 매출은 얼마인가?", "경기도의 패션 주문은 몇 건인가?"처럼 비교하기 쉬운 표로 줄이는 작업입니다.

지역과 카테고리를 동시에 기준으로 묶습니다.

```python
region_category_summary = (
    merged.groupby(["region", "category"])
    .agg(
        주문건수=("order_id", "nunique"),
        판매수량=("qty", "sum"),
        매출합계=("sales", "sum"),
        평균주문금액=("sales", "mean"),
    )
    .reset_index()
    .sort_values("매출합계", ascending=False)
)

print(region_category_summary.head(10))
```

대표 결과:

```text
region    category  주문건수  판매수량      매출합계
서울특별시  디지털       612   658  210356387
경기도     디지털       514   565  188110354
서울특별시  패션         990  1442   86206229
경기도     패션         884  1292   75963357
```

읽는 법:

- `groupby(["region", "category"])`: 지역별로 나눈 뒤 카테고리별로 다시 나눈다.
- `주문건수=("order_id", "nunique")`: 주문번호의 고유 개수를 센다.
- `판매수량=("qty", "sum")`: 수량을 합한다.
- `매출합계=("sales", "sum")`: 매출을 합한다.
- `평균주문금액=("sales", "mean")`: 주문 한 건의 평균 매출을 구한다.

`nunique`는 같은 주문번호를 중복해서 세지 않고, 서로 다른 주문번호가 몇 개인지 세는 방법입니다. 사전에서 같은 단어를 여러 번 만나도 단어 종류는 한 번만 세는 것과 비슷합니다.

결과를 보면 서울특별시 디지털 카테고리는 주문건수 612건, 판매수량 658개, 매출합계 210356387입니다. 이 표는 "누가 많이 팔렸는가"보다 "어떤 지역과 어떤 카테고리 조합이 큰가"를 보는 표입니다.

## 따라하기 4: 지역×카테고리 피벗 테이블 만들기

이 단계에서는 같은 요약을 더 보기 편한 가로형 표로 바꿉니다. 엑셀 피벗 테이블에서 지역을 행에 놓고, 카테고리를 열에 놓는다고 생각해 보세요.

이번에는 엑셀 피벗처럼 지역을 행, 카테고리를 열에 놓습니다.

```python
pivot = pd.pivot_table(
    merged,
    index="region",
    columns="category",
    values="sales",
    aggfunc="sum",
    fill_value=0,
    margins=True,
    margins_name="합계",
)

pivot_display = pivot.round(0).astype("int64")
print(pivot_display.head())
```

자주 쓰는 인자:

| 인자 | 뜻 | 엑셀 피벗과 비교 |
|---|---|---|
| `index` | 행 기준 | 행 영역 |
| `columns` | 열 기준 | 열 영역 |
| `values` | 계산할 값 | 값 영역 |
| `aggfunc` | 계산 방법 | 합계, 평균, 개수 |
| `fill_value` | 빈 칸 대체값 | 빈 셀에 0 표시 |

`index="region"`은 행 방향에 지역을 놓겠다는 뜻입니다. `columns="category"`는 열 방향에 카테고리를 놓겠다는 뜻입니다. `values="sales"`와 `aggfunc="sum"`은 매출을 합계로 계산하겠다는 뜻입니다.

`fill_value=0`은 해당 지역과 카테고리 조합이 없을 때 빈칸 대신 0을 넣습니다. 빈칸이 많으면 표를 읽기 어렵기 때문에, 집계표에서는 이렇게 0으로 채우는 일이 자주 있습니다.

`margins=True`와 `margins_name="합계"`는 행과 열의 전체 합계를 붙입니다. 엑셀 피벗 테이블의 총합계와 비슷합니다.

## 따라하기 5: 231열 wide 인구 데이터를 long으로 바꾸기

이 단계에서는 열이 옆으로 아주 많이 펼쳐진 표를 분석하기 쉬운 세로형 표로 바꿉니다. wide 데이터는 열이 넓게 펼쳐진 표이고, long 데이터는 값이 한 열 안으로 길게 내려오는 표입니다.

이번 파일은 18,623행, 231열입니다. 열이 너무 많으므로 수업에서는 서울특별시 행만 먼저 골라서 `melt`합니다. 전체를 한꺼번에 바꾸기 전에 필요한 부분만 작게 잘라 보는 연습입니다.

수업 홈의 데이터 받기 페이지에서 법정동 연령·성별 인구 파일을 내려받아 준비해 보세요.

```python
pop = pd.read_csv(
    DATA / "real_korean/population_kosis/mois_legal_dong_gender_age_population_20260531.csv"
)

print(pop.shape)
print(pop.columns[:15])
```

확인할 결과:

```text
(18623, 231)
```

`(18623, 231)`은 18,623행, 231열이라는 뜻입니다. 행보다 열이 매우 많죠. 이런 표는 눈으로 볼 때는 보고서처럼 느껴질 수 있지만, 분석할 때는 필터링이나 그래프 만들기가 불편합니다.

이제 서울특별시만 남기고, `0세남자`, `1세남자` 같은 연령·성별 열을 세로로 모읍니다. 여러 칸에 흩어진 값을 한 줄로 접어 넣는다고 생각하면 됩니다.

```python
id_cols = ["법정동코드", "기준연월", "시도명", "시군구명", "읍면동명", "리명"]

age_gender_cols = [
    col
    for col in pop.columns
    if (col.endswith("남자") or col.endswith("여자")) and col not in ["남자", "여자"]
]

seoul_pop = pop[pop["시도명"] == "서울특별시"].copy()

long_pop = seoul_pop.melt(
    id_vars=id_cols,
    value_vars=age_gender_cols,
    var_name="연령성별",
    value_name="인구수",
)

long_pop["성별"] = long_pop["연령성별"].str.extract("(남자|여자)$")
long_pop["연령"] = (
    long_pop["연령성별"].str.replace("(남자|여자)$", "", regex=True).str.strip()
)

print(seoul_pop.shape)
print(len(age_gender_cols))
print(long_pop.shape)
print(long_pop[["시도명", "시군구명", "읍면동명", "연령", "성별", "인구수"]].head())
```

확인할 결과:

```text
서울특별시 wide: 452행, 231열
melt 대상 연령/성별 열: 222개
서울특별시 long: 100,344행, 10열
```

`melt`를 하면 열 이름이 값으로 내려옵니다. 넓은 표에서는 `0세남자`가 열 이름이었지만, 긴 표에서는 `연령성별` 열 안의 값이 됩니다.

서울특별시만 골랐을 때 wide 데이터는 452행, 231열입니다. 여기에서 연령/성별 열 222개를 세로로 모으면 100,344행, 10열이 됩니다. 행은 늘어나지만 열은 줄어듭니다. 분석할 때는 이렇게 길어진 표가 더 편한 경우가 많습니다.

## 따라하기 6: 서울 음식점 상권 업종 상위 10개 집계하기

이 단계에서는 큰 상가업소 표를 업종별 개수 표로 줄입니다. 원본을 그대로 많이 출력하면 눈에 들어오지 않기 때문에, 먼저 "업종별로 몇 개나 있는가"를 세어 봅니다.

상가업소 파일은 119,158행입니다. 수업 홈의 데이터 받기 페이지에서 서울 음식점 상가업소 파일을 내려받아 준비해 보세요. 원본을 그대로 보지 말고 먼저 업종별 개수로 줄입니다.

```python
stores = pd.read_csv(

top10 = (
```

확인할 대표 결과:

`groupby().size()`는 각 그룹에 행이 몇 개 있는지 세는 가장 단순한 집계입니다. 여기서는 `상권업종중분류명`이 같은 행끼리 모은 뒤, 업종마다 업소가 몇 개 있는지 셉니다.

결과에서 한식은 40,448개, 커피점/카페는 19,703개입니다. 119,158행짜리 원본을 바로 읽는 것보다, 이렇게 상위 업종부터 보면 데이터의 큰 모양이 먼저 보입니다.

## 직접 해보기: 자치구별 생활인구 집계

이번에는 직접 자치구별 생활인구 요약표를 만들어 봅니다. 생활인구는 실제 거주지 기준 인구가 아니라, 특정 시간대에 그 지역에 머무르는 사람 수를 추정한 값입니다.

사용 파일은 수업 홈의 데이터 받기 페이지에서 내려받은 서울 생활인구 일별 파일입니다.

목표:

1. 생활인구 파일을 읽습니다. 표를 읽어야 행수와 열을 확인할 수 있습니다.
2. `기준일ID`를 날짜로 바꿉니다. 날짜처럼 보이는 숫자를 진짜 날짜 형식으로 바꾸면 기간 계산과 정렬이 쉬워집니다.
3. `시군구명`이 `서울시`인 전체 합계 행은 제외합니다. 자치구 25개만 비교해야 하므로 전체 합계 행이 섞이면 안 됩니다.
4. `시군구명`별로 아래 값을 계산합니다. 자치구마다 관측 기간과 평균 규모를 한눈에 보기 위한 요약입니다.
   - 관측일수: 날짜의 고유 개수
   - 평균생활인구: `총생활인구수` 평균
   - 평균주간인구: `주간인구수(09~18)` 평균

아래 표는 실습 중 자주 만나는 상황입니다. 오류가 나면 먼저 메시지를 끝까지 읽고, 표의 원인과 해결 방법을 차례로 대조해 보세요.

| 오류 상황 | 원인 | 해결 |
|---|---|---|
